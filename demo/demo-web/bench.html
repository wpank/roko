What is this? Roko's benchmark lab runs your coding agent against standardized tasks (SWE-bench style), measures pass rates, costs, and speed, then records everything so the system learns which models and strategies work best. Think of it as a training gym for your AI coding assistant.

Test Suite

Choose which coding challenges to run. Each suite tests different capabilities.

Smoke Tests
2 built-in tasks for fast verification. Ensures the benchmark harness itself works before running expensive model evaluations.
2 tasks ~30s $0.00
SWE-bench Lite
300 curated Python bug fixes from real open-source repos. The industry standard for evaluating coding agents.
300 tasks ~4h ~$15-80
SWE-bench Verified
500 human-verified instances. Higher quality labels than Lite, considered the gold standard for leaderboard scores.
500 tasks ~8h ~$30-150
Custom Dataset
Bring your own JSONL dataset. Each row defines a repo, problem statement, gold patch, and test command.
JSONL any size

Agent Strategy

How should roko approach each task? Strategies differ in context, cost, and learning.

Minimal
Only the problem statement and validation command. Tests raw model reasoning with zero context injection.
low context cheapest
Context-Enriched
Adds source file contents from the target repo. Gives the model more information to work with.
medium context moderate
Neuro-Augmented
Source context + knowledge store queries. The system injects relevant past learnings to improve accuracy.
self-learning higher cost
Full Cascade
Neuro context + cascade routing + gate replanning. The full roko pipeline — picks models per-task and retries failures.
adaptive highest accuracy

      How self-learning works: Every benchmark run generates episodes (what happened),
      efficiency events (cost/speed per turn), and C-factor snapshots (composite reliability).
      The system uses these to: route tasks to better models, adjust gate thresholds, inject relevant past knowledge,
      and replan on failure. Performance improves automatically over repeated runs.
