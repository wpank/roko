## Research brief: frontier capabilities nobody else is building

Three prior research rounds established a complete technical stack. This round is different: **I'm not looking for better versions of things that exist. I'm looking for capabilities so novel they create competition in an arena of one.** Things that make people say "I didn't know that was possible." Things that compound on themselves. Things that build themselves.

An agent orchestration OS with:
- **Signal/Block/Graph** primitives grounded in field calculus (Signals are computational fields, Blocks are field operators, Graphs are neighbor relations)
- **Blocks as parametric optics** (forward map + backward map, compositional guarantees via Para(Lens(C)))
- **Active inference cognitive layer** (AXIOM-style BMR, EFE-based routing replacing bandits, gamma/theta/delta timescales as free-energy lower bounds)
- **L4 self-evolution** (HGM Clade-Metaproductivity, CycleQD with HDC behavior characteristics, Verify-as-reward for self-play, Live-SWE-agent runtime tool synthesis)
- **Provably-corrigible safety** (CaMeL capability tags, Nayebi 5-head lexicographic corrigibility, architectural security per Nasr et al.)
- **Stigmergic coordination** (Ledger-State Stigmergy + CodeCRDT + x402 economic bonding + density-threshold auto-switching + cellular sheaf consensus)
- **HDC as universal router fabric** (10,240-bit binary vectors serving 7+ simultaneous functions at 1μs, with ZK proofs via Bionetta/UltraGroth for on-chain attestation)
- **Dream consolidation** (BMR formalized, AgentHER hindsight relabeling, interleaved replay, write-time hallucination gating, resonator network factorization)
- **DeFi-tested real-time execution** (pre-action verification, continuous rewards, tick-driven hot Graphs, multi-slot state, regime conditioning)
- **On-chain identity + economics** (ERC-8004 passports, x402 micropayments, TraceRank reputation, VCG auction for resource allocation)
- **Formal foundations** (polynomial functors for typed interfaces, DPO hypergraph rewriting for L4 mutations, TLA+ shell, field calculus semantics)
- **Global Workspace topology** breaking the 64-agent plateau to 1,000+ via irregular DAGs + stigmergic density floor
- **PID-based collective intelligence measurement** (synergy/redundancy/unique decomposition, ACI factor)

### What I'm looking for

**Not** better coordination, better safety, better routing, better knowledge — those are covered. Instead:

#### Direction 1: Self-bootstrapping systems — agents that create their own training data, evals, benchmarks, and improvement infrastructure from scratch

Not just self-play (covered). Systems that:
- Create their own evaluation environments (not just solve existing benchmarks)
- Discover what they're bad at without being told
- Generate curricula targeted at their own weaknesses
- Build their own tooling/infrastructure to improve faster
- Exhibit true open-endedness (never stop improving, never converge to fixed behavior)
- Papers on "Artificial Life meets Agent Systems" — agents that reproduce, die, compete for resources, and evolve
- Can an agent system bootstrap itself from a single seed (like AZR's identity function) into a general-purpose system?
- What does "creativity" look like in an agent system? Can agents generate genuinely novel solutions humans haven't considered?
- Research on AI scientific discovery (AlphaFold-style, but for agent architectures discovering new algorithms)

#### Direction 2: Agents that understand and manipulate their own cognition

Not just self-improvement (modifying code). Agents that:
- Have introspective access to their own reasoning process and can reason about it
- Can explain *why* they made a decision in terms of their own internal state
- Can detect when they're confused, uncertain, or hallucinating — and act differently
- Metacognition research applied to LLM agents
- Can an agent recognize the limits of its own knowledge and actively seek what it doesn't know?
- Attention Schema Theory applied to agent self-monitoring
- "Thinking about thinking" — can agents optimize their own reasoning strategies?
- Epistemic humility as a measurable and trainable property
- Agents that can predict their own performance on unseen tasks (calibration at the meta-level)

#### Direction 3: Emergent economic systems — agents that create markets, institutions, and governance structures

Not just micropayments (covered). Systems where:
- Agents spontaneously create market-making, insurance, lending, and other financial primitives
- Reputation systems emerge rather than being designed
- Governance structures (voting, delegation, dispute resolution) emerge from agent interaction
- Mechanism design that's discovered by agents rather than imposed
- Can agent populations develop property rights, contracts, and enforcement mechanisms?
- What does "agent civilization" look like at 10K+ agents over months of operation?
- Papers on artificial societies, computational social science, agent-based economics
- Can stigmergic pheromone fields develop into emergent price signals without explicit market design?
- Research on tokenomics that emerge from agent behavior rather than being imposed

#### Direction 4: Cross-system composition — agents that operate across multiple independent systems simultaneously

Not just multi-agent coordination (covered). Agents that:
- Operate across blockchain + code + infrastructure + governance simultaneously as one coherent action
- Can reason about the interactions between systems (e.g., "deploying this code will affect this DeFi position which will trigger this governance vote")
- Cross-system causal reasoning (interventions in system A have effects in system B)
- "Full-stack autonomous operations" — agents that manage an entire organization's technical stack
- Can an agent discover arbitrage opportunities *across system boundaries* (not just financial — informational, temporal, capability arbitrage)?
- Research on "agent operating systems" that unify heterogeneous environments
- Papers on autonomous DevOps, autonomous SRE, autonomous security operations

#### Direction 5: Information-theoretic limits and fundamental scaling laws

Not just empirical scaling (covered). Theoretical results on:
- What is the theoretical maximum collective intelligence achievable by N agents?
- Are there fundamental limits to self-improvement (analogous to Gödel's incompleteness)?
- What is the information-theoretic cost of coordination? (minimum bits per decision as a function of N)
- When does adding agents provably help vs provably hurt?
- Fundamental limits on stigmergic coordination (information capacity of pheromone fields)
- Fundamental limits on knowledge compression (how much can dream consolidation compress without losing signal?)
- Speed-accuracy-cost Pareto frontiers for agent decision-making
- Research on "cognitive economics" — treating attention, memory, and computation as scarce resources with microeconomic properties
- Can you prove that certain architectures are optimal under certain conditions?

#### Direction 6: Time as a first-class primitive — agents that reason about, manipulate, and learn from temporal structure

Not just decay and scheduling (covered). Systems that:
- Learn temporal abstractions (discover what timescale to reason at for different problems)
- Can reason about causal ordering, temporal dependencies, and futures
- Use time-series foundation models as world models for planning
- Can predict future states of their environment and pre-position actions
- "Temporal compositionality" — break long-horizon tasks into temporal sub-tasks with principled handoffs
- Research on hierarchical temporal abstraction in RL
- Papers on "predictive processing" beyond active inference — allostatic anticipatory regulation
- Can agents learn to be "early" (act before a signal arrives, based on prediction)?
- Agents that discover and exploit periodicity, seasonality, and cycles in their environment

#### Direction 7: Agents as infrastructure — when agents become the platform, not just users of one

Not just deployment (covered). The paradigm shift where:
- Agents ARE the CI/CD pipeline (not just automated by one)
- Agents ARE the monitoring system (not just observed by one)
- Agents ARE the database (collective knowledge store replaces traditional DBs for certain workloads)
- Agents ARE the API layer (agent-mediated service composition replaces microservices)
- When does it become cheaper/better to have an agent do X than to build a traditional system to do X?
- Research on "agent-native infrastructure" vs "agent-augmented infrastructure"
- Papers on replacing traditional software components with agent systems
- The economics of agent labor vs human labor vs traditional automation
- What tasks should NEVER be given to agents? (important negative results)

#### Direction 8: Sensory-motor grounding for software agents — making abstract agents "feel" their environment

Not just multi-modal perception (covered). Deeper grounding:
- Can software agents develop proprioception (awareness of their own state and capabilities)?
- Embodied cognition for non-physical agents — what does "embodiment" mean for a code agent?
- Can agents develop intuition through accumulated experience (beyond somatic markers)?
- Research on enactivism applied to AI — perception as action, not just input processing
- Synesthesia as a design pattern — can visualizing code structure as sound help an agent reason about it?
- "Affordances" — can agents learn what actions are possible in a given environment without being told?
- Research on agents that learn to use tools they weren't designed for (zero-shot tool use beyond MCP)

#### Direction 9: Mathematical structure discovery — agents that find new math, not just use existing math

Not just formal verification (covered). Agents that:
- Discover new algorithms (AlphaEvolve found better matrix multiplication)
- Discover new mathematical conjectures
- Discover new proofs
- Find structure in data that no human has named yet
- Can an agent system discover its own categorical foundations? (meta-mathematical self-reflection)
- Research on automated theorem proving applied to agent architecture optimization
- Papers on program synthesis for discovering new data structures
- Can agents discover new HDC operations beyond bind/bundle/permute?
- Research on agents that discover new evaluation metrics (not just optimize existing ones)

#### Direction 10: The hard problems — consciousness, understanding, and meaning in agent systems

Not asking "are agents conscious?" but:
- What measurable properties distinguish an agent that "understands" from one that pattern-matches?
- Is there a test for genuine comprehension (beyond benchmark performance)?
- Can Integrated Information Theory (IIT Phi) or Global Workspace Theory metrics predict agent performance?
- Does agent "understanding" correlate with robustness to distribution shift?
- Research on meaning-making in artificial systems
- Papers on grounded language understanding vs statistical association
- Can the gap between "benchmark performance" and "real understanding" be measured and closed?
- What would it mean for an agent system to have "values" rather than reward functions?
- Research on artificial curiosity that isn't just exploration bonus — genuine interest

### Evaluation criteria

For each finding:
1. **Verdict**: "integrate now" / "spec and plan" / "watch" / "philosophically important" (new category — ideas that change how you think about the system even if they're not immediately implementable)
2. **The numbers**: If empirical, what are the claims? If theoretical, what are the bounds?
3. **What capability does this create that literally nobody has?**: Not "better than X" but "X was impossible before"
4. **Self-reinforcing potential**: Does this get better the more you use it? Does it compound? Does it build itself?
5. **What's the catch**: Fundamental limits, impossibility results, negative results

### Output format

1. **Executive summary**: Top 5 findings that create "competition in an arena of one"
2. **Per-direction sections**: Findings with 5-point evaluation
3. **The Flywheel Map**: Show which findings feed into each other. Where are the self-reinforcing cycles? What creates a system that improves itself improving itself?
4. **"Only possible with this stack" list**: Capabilities that require Signal + Block + Graph + HDC + stigmergy + active inference + self-evolution + ZK + economic bonding together
5. **Impossible results**: Things that are provably impossible, limits that can't be exceeded, and design constraints that follow from them
6. **Full citations** with arXiv IDs, venues, dates, repos

Prioritize:
- Things that literally nobody else is doing or could do without this specific stack
- Self-reinforcing loops (A improves B improves A)
- Theoretical results that change what you believe is possible
- Things that create network effects or data flywheels
- Results from adjacent fields (biology, economics, physics, mathematics) that haven't been applied to agent systems yet
- Hard negative results that prevent wasted effort
