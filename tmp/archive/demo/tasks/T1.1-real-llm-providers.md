## What to implement

### `ClaudeApiProvider`

```rust
/// Calls the Anthropic Messages API.
///
/// Set `ANTHROPIC_API_KEY` in the environment. Model defaults to
/// `claude-sonnet-4-20250514` but can be overridden via `ANTHROPIC_MODEL`.
pub struct ClaudeApiProvider {
    client: reqwest::Client,
    api_key: String,
    model: String,
}
```

Implementation notes:
- Constructor: read `ANTHROPIC_API_KEY` from env (return `Err` if missing). Read `ANTHROPIC_MODEL`
  from env or use `"claude-sonnet-4-20250514"` as default.
- `fill` method:
  - Build a user message: `format!("slot: {}\ncontext: {}", req.slot, req.context)`
  - System prompt: `"Respond with ONLY valid JSON matching the requested schema. No explanation, no markdown fences."`
  - POST to `https://api.anthropic.com/v1/messages` with headers:
    - `x-api-key: {api_key}`
    - `anthropic-version: 2023-06-01`
    - `content-type: application/json`
  - Request body:
    ```json
    {
      "model": "{model}",
      "max_tokens": 256,
      "temperature": 0,
      "system": "...",
      "messages": [{ "role": "user", "content": "..." }]
    }
    ```
  - Parse response: `response["content"][0]["text"]` is a string; parse it as JSON.
  - Log with `tracing::debug!(slot = %req.slot, model = %self.model, "claude api call")`.

### `OllamaProvider`

```rust
/// Calls a local Ollama instance.
///
/// `OLLAMA_URL` defaults to `http://localhost:11434`.
/// `OLLAMA_MODEL` defaults to `gemma3:7b`.
pub struct OllamaProvider {
    client: reqwest::Client,
    base_url: String,
    model: String,
}
```

Implementation notes:
- Constructor reads `OLLAMA_URL` and `OLLAMA_MODEL` from env with the defaults above.
- `fill` method:
  - Build prompt: `format!("Respond with ONLY valid JSON.\n\nslot: {}\ncontext: {}", req.slot, req.context)`
  - POST to `{base_url}/api/generate`:
    ```json
    {
      "model": "{model}",
      "prompt": "...",
      "stream": false,
      "temperature": 0
    }
    ```
  - Parse: `response["response"]` is the text string; parse it as JSON.
  - Strip markdown fences if present: if the text starts with ` ``` `, strip the first and last line.
  - Log with `tracing::debug!(slot = %req.slot, model = %self.model, "ollama call")`.

### `MultiProvider`

```rust
/// Round-robins across a list of labelled providers.
pub struct MultiProvider {
    providers: Vec<(String, Arc<dyn LlmProvider>)>,
    cursor: std::sync::atomic::AtomicUsize,
}
```

Implementation notes:
- `pub fn new(providers: Vec<(String, Arc<dyn LlmProvider>)>) -> Self`
- `pub fn label_for_index(&self, i: usize) -> &str` â€” returns the label of the provider at index
  `i % providers.len()`
- `fill` method: `let i = self.cursor.fetch_add(1, Ordering::Relaxed); providers[i % len].1.fill(req)`
- Log which provider handled the call:
  `tracing::info!(provider = %label, slot = %req.slot, "multi-provider dispatch")`

### Factory function

```rust
/// Build a provider from a backend name.
///
/// - `"stub"` â†’ [`StubLlm`]
```

For `"multi"`: construct a `ClaudeApiProvider` and an `OllamaProvider`, wrap both in a

Return `Err` for unknown backend strings.

## Slot-to-prompt table (for `ClaudeApiProvider` and `OllamaProvider`)

Both providers use the same slot string in their prompt. The model sees:

```
slot: {slot_name}
context: {context_json}
```

That is sufficient. The system prompt instructs the model to return only valid JSON matching the
schema. You do not need to encode the schema per slot â€” the context JSON already contains the
relevant state.
