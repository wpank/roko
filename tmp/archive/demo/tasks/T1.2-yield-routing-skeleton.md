## Context

`roko-demo` is a manifest-driven orchestrator that deploys contracts, runs fixture setup, and
executes scripted "spines" — Rust functions that simulate on-chain agent behaviour. Each spine
calls an `LlmProvider` to fill decision slots: bid amounts, vote decisions, route proposals, etc.

This task builds the `yield-routing` scenario from scratch. It deploys all six contracts
(MockERC20, AgentRegistry, WorkerRegistry, BountyMarket, ConsortiumValidator, InsightBoard),
registers 5 workers with distinct model labels, seeds initial knowledge on InsightBoard, and
runs a two-round job lifecycle. LLM decisions use `StubLlm` throughout — real providers, fee
distribution, and event streaming are wired in subsequent tasks.

## Architecture recap

### Scenario trait

```rust
#[async_trait]
pub trait Scenario: Send + Sync {
    fn name(&self) -> &'static str;
    fn register_fixtures(&self, _registry: &mut FixtureRegistry) {}
    async fn spine(
        &self,
        ctx: Arc<ChainCtx>,
        manifest: &ScenarioManifest,
        llm: Arc<dyn LlmProvider>,
    ) -> anyhow::Result<()>;
}
```

### ChainCtx (key methods)

```rust
ctx.wallet_provider("worker0")?    // returns Arc<DynProvider>
ctx.read_provider()?               // returns Arc<DynProvider>
ctx.address_of("BountyMarket")?    // returns Address
ctx.wallet_address("poster0")?     // returns Address
```

### LlmProvider

```rust
pub struct LlmRequest { pub slot: String, pub context: serde_json::Value }

#[async_trait]
pub trait LlmProvider: Send + Sync {
    async fn fill(&self, req: LlmRequest) -> anyhow::Result<serde_json::Value>;
}
```

Slots used in this scenario:

| Slot | Expected shape | StubLlm returns |
|---|---|---|
| `route_proposal` | `{ "hops": [...], "expected_output": number }` | `{ "hops": ["0xaaa","0xbbb"], "expected_output": 1000 + n*10 }` where n is the call counter |
| `approve` | `{ "approve": bool, "reason": string }` | alternates true/false |

#### Phase 1: Prepare

Mint, register workers, promote validators, register in AgentRegistry.

```
// Token amounts
const POSTER_MINT:    u128 = 1_000_000 * 1e18 as u128;
const WORKER_MINT:    u128 =    10_000 * 1e18 as u128;
const VALIDATOR_MINT: u128 =    10_000 * 1e18 as u128;
const WORKER_STAKE:   u128 =     1_000 * 1e18 as u128;
const INSIGHT_FUND:   u128 =   100_000 * 1e18 as u128;
```

Steps, in order:

1. Mint `POSTER_MINT` DAEJI to `poster0`.
2. For each of `worker0`–`worker4`: mint `WORKER_MINT`.
3. For each of `validator0`–`validator2`: mint `VALIDATOR_MINT`.
4. Mint `INSIGHT_FUND` to the InsightBoard contract address (needed for reward payouts).
5. For each of `worker0`–`worker4`:
   - Call `WorkerRegistry.register(worker_address, WORKER_STAKE)` using that worker's wallet.
   - Call `AgentRegistry.register(worker_address, capabilities_string)` where
     `capabilities_string = format!("model:{} task:routing", AGENT_MODELS[i])`.
6. For each of `validator0`–`validator2`:
   - Call `WorkerRegistry.register(validator_address, WORKER_STAKE)`.
   - Call `WorkerRegistry.updateReputation(validator_address, true)` **30 times** using the
     deployer wallet. This promotes the validator to Trusted tier, matching the pattern in
     `consortium.rs` lines 140–160.
7. `poster0` calls `MockERC20.approve(bounty_market_address, U256::MAX)`.

Log each phase with `tracing::info!`. Example:
```rust
tracing::info!("phase 1: prepare — minting tokens and registering agents");
```

Handle already-registered errors gracefully. If `register` returns an error whose message
contains `"already registered"`, log a warning and continue. Match the pattern from
`job_board.rs` line 112.

#### Phase 2: Seed knowledge (3 initial insights)

```rust
let seed_insights = [
    "Aave V3 USDC/ETH pool: deep liquidity, stable rate 3.2%",
    "Morpho yields 4.1% but TVL thin — slippage risk above 50K",
    "Compound V3 safest choice for large trades above 200K notional",
];
```

For each insight string:
1. Compute `content_hash = keccak256(insight.as_bytes())`.
2. `worker0` calls `InsightBoard.post(content_hash)`.
3. Capture the `insight_id` from the return value (or read `nextInsightId() - 1` after posting).
4. For each of `validator0`–`validator2`: call `InsightBoard.confirm(insight_id)`.

Store the three insight strings in a `Vec<String>` for use in later LLM context.

Log: `tracing::info!("phase 2: seeding {} insights", seed_insights.len())`.

#### Phase 3: Post job (helper — called in both rounds)

Extract a helper function or closure:

```rust
async fn post_job(
    ctx: &ChainCtx,
    round: u32,
    prior_insights: &[String],
    llm: &Arc<dyn LlmProvider>,
) -> anyhow::Result<JobResult>
```

Where `JobResult` is a local struct:
```rust
struct JobResult {
    job_id: u64,
    winner_idx: usize,
    route_proposals: Vec<serde_json::Value>,
}
```

Steps inside `post_job`:

1. **Post job.** `poster0` calls `BountyMarket.postJob(spec_hash, bounty)`:
   - `spec_hash = keccak256(format!("yield-routing-round-{}", round).as_bytes())`
   - `bounty = 1_000 * 1e18 as u128`
   - Capture `job_id = BountyMarket.nextJobId() - 1` after the call.

2. **Agent bidding.** For each worker index `i` in 0..5:
   - Build context JSON:
     ```json
     {
       "round": <round>,
       "worker": "worker<i>",
       "model": "<AGENT_MODELS[i]>",
       "job": "Route 100K USDC → ETH, maximize output",
       "prior_insights": [...]
     }
     ```
   - Call `llm.fill(LlmRequest { slot: "route_proposal".into(), context })`.
   - Parse `expected_output` from the response as `u64`.
   - Store the full response in `route_proposals[i]`.

3. **Select winner.** `winner_idx = route_proposals.iter().enumerate().max_by_key(|(_, v)| v["expected_output"].as_u64().unwrap_or(0)).map(|(i, _)| i).unwrap_or(0)`.

4. **Assign winner.** Deployer calls `BountyMarket.assign(job_id, worker_winner_address)`.

5. **Return** `JobResult { job_id, winner_idx, route_proposals }`.

Log the winner: `tracing::info!(round, winner = %format!("worker{}", winner_idx), "job assigned")`.

#### Phase 4: Execute and validate (helper — called in both rounds)

```rust
async fn execute_and_validate(
    ctx: &ChainCtx,
    result: &JobResult,
    llm: &Arc<dyn LlmProvider>,
) -> anyhow::Result<()>
```

Steps:

1. **Submit.** Winner calls `BountyMarket.submit(job_id, result_hash)`:
   - `result_hash = keccak256(serde_json::to_vec(&route_proposals[winner_idx])?.as_slice())`

2. **Mine a block** before assembling the committee. This is required for blockhash entropy.
   Match the exact pattern from `consortium.rs` line 186. If that pattern uses a provider call
   to advance the block, copy it exactly.

3. **Assemble committee.** Deployer calls `ConsortiumValidator.assembleCommittee(job_id)`.

4. **Vote.** Read `ConsortiumValidator.getMembers(job_id)` to get the committee member addresses.
   For each member address, find its wallet name (match address against `validator0`–`validator2`),
   then call `llm.fill` with slot `"approve"` and context containing `job_id` and `winner_idx`.
   Parse `{ "approve": bool }`. That validator calls `ConsortiumValidator.vote(job_id, approve)`.

5. **Resolve.** Deployer calls `BountyMarket.resolve(job_id)`.

Log: `tracing::info!(round = result.job_id, "job resolved")`.

#### Phase 5: Post winner insight

After each round, the winner posts a new insight:

```rust
let insight_text = format!(
    "Round {} winner ({}): route achieved expected_output={}",
    round,
    AGENT_MODELS[winner_idx],
    route_proposals[winner_idx]["expected_output"]
);
let hash = keccak256(insight_text.as_bytes());
winner_wallet.call(InsightBoard.post(hash));
// validators confirm
for v in 0..3 {
    validator_wallet.call(InsightBoard.confirm(new_insight_id));
}
```

Add the new insight text to `prior_insights` before starting round 2.

#### Phase 6: Round 2

Run `post_job` and `execute_and_validate` again, this time passing the updated `prior_insights`
(original 3 + the insight from round 1).

Log final summary:
```rust
tracing::info!(
    round1_winner = %format!("worker{} ({})", r1.winner_idx, AGENT_MODELS[r1.winner_idx]),
    round2_winner = %format!("worker{} ({})", r2.winner_idx, AGENT_MODELS[r2.winner_idx]),
    "yield-routing complete — 2 rounds finished"
);
```
