## Contract to implement

### `contracts/src/FeeDistributor.sol`

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.26;

import { IERC20 } from "@openzeppelin/contracts/token/ERC20/IERC20.sol";

/// @title FeeDistributor — splits job bounty payments across stakeholder roles.
/// @notice Pulls `totalAmount` from the caller and distributes it 40/30/20/10 to
///         validators, data providers, the winning agent, and treasury.
contract FeeDistributor {
    IERC20 public immutable token;
    address public treasury;

    uint256 public constant VALIDATOR_BPS    = 4_000; // 40%
    uint256 public constant DATA_PROVIDER_BPS = 3_000; // 30%
    uint256 public constant AGENT_BPS        = 2_000; // 20%
    uint256 public constant TREASURY_BPS     = 1_000; // 10%

    /// @notice Cumulative DAEJI credited to each address across all distributions.
    mapping(address => uint256) public cumulativeEarnings;

    event FeesDistributed(
        uint256 indexed jobId,
        uint256 totalAmount,
        uint256 validatorShare,
        uint256 dataProviderShare,
        uint256 agentShare,
        uint256 treasuryShare
    );

    event EarningsCredited(address indexed recipient, uint256 amount, string role);

    error EmptyValidators();
    error ZeroAmount();
    error ZeroAddress();

    constructor(address token_, address treasury_);

    /// @notice Pull `totalAmount` from caller and distribute 40/30/20/10.
    /// @param jobId         Job identifier (used only in the event).
    /// @param totalAmount   Total DAEJI to distribute. Caller must pre-approve.
    /// @param agent         Winning agent address — receives 20%.
    /// @param validators    Validator addresses — split 40% equally.
    ///                      If empty, their share goes to treasury.
    /// @param dataProviders Data provider addresses — split 30% equally.
    ///                      If empty, their share goes to treasury.
    function distribute(
        uint256 jobId,
        uint256 totalAmount,
        address agent,
        address[] calldata validators,
        address[] calldata dataProviders
    ) external;

    /// @notice Transfer treasury role. Only current treasury may call.
    function setTreasury(address newTreasury) external;
}
```

#### `distribute` implementation requirements

1. Revert with `ZeroAmount` if `totalAmount == 0`.
2. Call `token.transferFrom(msg.sender, address(this), totalAmount)` — pull tokens from caller.
3. Calculate shares:
   ```
   validatorShare    = totalAmount * VALIDATOR_BPS    / 10_000
   dataProviderShare = totalAmount * DATA_PROVIDER_BPS / 10_000
   agentShare        = totalAmount * AGENT_BPS        / 10_000
   ```
4. Compute treasury share as the remainder to avoid losing dust to integer division:
   ```
   treasuryShare = totalAmount - validatorShare - dataProviderShare - agentShare
   ```
5. If `validators.length > 0`: divide `validatorShare` equally among them; any remainder wei goes to treasury. Otherwise: add `validatorShare` to `treasuryShare`.
6. If `dataProviders.length > 0`: divide `dataProviderShare` equally among them; any remainder wei goes to treasury. Otherwise: add `dataProviderShare` to `treasuryShare`.
7. Transfer `agentShare` to `agent`; update `cumulativeEarnings[agent]`; emit `EarningsCredited(agent, agentShare, "agent")`.
8. Transfer `treasuryShare` (final, after any remainder roll-ups) to `treasury`; update `cumulativeEarnings[treasury]`; emit `EarningsCredited(treasury, treasuryShare, "treasury")`.
9. For each validator and data provider, update `cumulativeEarnings` and emit `EarningsCredited` with the appropriate role string (`"validator"` or `"data-provider"`).
10. Emit `FeesDistributed(jobId, totalAmount, validatorShare, dataProviderShare, agentShare, treasuryShare)` — use the base shares (before remainder roll-up) so event amounts are predictable.

#### `setTreasury` implementation requirements

- Revert with `NotAuthorized` (define this error) if `msg.sender != treasury`.
- Revert with `ZeroAddress` if `newTreasury == address(0)`.
- Update `treasury`.

#### Setup

```solidity

FeeDistributor internal dist;

address internal treasury  = address(0x7EA5);

address internal val1      = address(0xA11CE);

function setUp() public {

    // Fund the caller and pre-approve the distributor.

    token.approve(address(dist), type(uint256).max);
```

#### `test_distribute_basic`

Distribute 1000 DAEJI with 3 validators and 2 data providers.

Expected balances after the call:
- Each validator: `1000 * 4000 / 10000 / 3 = 133 ether` (integer division; 1 wei remainder goes to treasury)
- Each data provider: `1000 * 3000 / 10000 / 2 = 150 ether`
- Agent: `1000 * 2000 / 10000 = 200 ether`
- Treasury: `1000 * 1000 / 10000 + 1 (validator remainder) = 101 ether` (adjust based on actual arithmetic)

Assert token balances for each recipient. Assert `caller` balance decreased by 1000 ether.

#### `test_distribute_empty_validators`

Pass an empty `validators` array. The validator share (40%) should go entirely to treasury.

Assert treasury receives at least 50% of the total (40% validators + 10% base treasury).

#### `test_distribute_empty_data_providers`

Pass an empty `dataProviders` array. The data provider share (30%) should go entirely to treasury.

Assert treasury receives at least 40% of the total (30% data providers + 10% base treasury).

#### `test_distribute_single_validator`

Pass one validator in the array. That validator should receive the full 40% validator share.

Assert `token.balanceOf(val1) == totalAmount * 4000 / 10000`.

#### `test_cumulative_earnings`

Call `distribute` twice with the same addresses. Assert that `dist.cumulativeEarnings(agent)` equals the sum of both agent shares.

#### `test_distribute_events`

Use `vm.expectEmit` to assert the `FeesDistributed` event is emitted with correct parameters for a known input (e.g., 1000 ether, 1 validator, 1 data provider).

The expected amounts:
- validatorShare = 400 ether
- dataProviderShare = 300 ether
- agentShare = 200 ether
- treasuryShare = 100 ether (before any dust)

#### `test_distribute_rounding`

Use a `totalAmount` that does not divide evenly — for example, `1001 ether` with 3 validators and 2 data providers.

After the call, assert that the total of all recipient balances equals exactly `1001 ether`. No tokens may be lost.

Use a helper: sum `token.balanceOf` for all recipients and compare to `totalAmount`.
