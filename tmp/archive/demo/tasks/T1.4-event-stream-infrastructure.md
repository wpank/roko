## Context

`roko-demo` is a manifest-driven orchestrator that deploys contracts, runs fixtures, and executes
scripted "spines" — Rust functions that simulate on-chain agent behaviour. Scenarios run today
without emitting any structured events. The only observability is `tracing` output.

backends (NDJSON to stdout, WebSocket broadcast), a composite fan-out emitter, and CLI flags to
select the mode at runtime. The emitter is created in `main.rs` but is not yet wired into any
scenario spine — that happens in T2.1.

### 1. `crates/roko-demo/src/events.rs`

```rust
use serde::{Deserialize, Serialize};
use std::sync::Arc;

/// All events emitted during demo scenario execution.
#[derive(Debug, Clone, Serialize, Deserialize)]
#[serde(tag = "type", rename_all = "snake_case")]
pub enum DemoEvent {
    RoundStarted { round: u32, total_rounds: u32 },
    RoundCompleted { round: u32 },
    JobPosted { job_id: u64, description: String, bounty_amount: String },
    AgentBid {
        agent_id: String,
        model: String,
        expected_output: f64,
        strategy_summary: String,
    },
    KnowledgeQueried { agent_id: String, insights_found: usize },
    JobAssigned { job_id: u64, agent_id: String },
    ExecutionStarted { agent_id: String, route: serde_json::Value },
    ExecutionCompleted {
        agent_id: String,
        output_amount: f64,
        gas_used: u64,
    },
    ValidationVote { validator_id: String, approved: bool, reason: String },
    ValidationComplete { job_id: u64, accepted: bool },
    FeesDistributed {
        job_id: u64,
        validator_share: String,
        data_provider_share: String,
        agent_share: String,
        treasury_share: String,
    },
    InsightPosted { agent_id: String, insight_id: u64, content: String },
    InsightConfirmed { confirmer_id: String, insight_id: u64 },
    ReputationUpdated { agent_id: String, old_rep: f64, new_rep: f64 },
    CFactorMeasured {
        run_a_output: f64,
        run_b_output: f64,
        improvement_pct: f64,
    },
    ScenarioStarted { name: String, agents: Vec<AgentInfo> },
    ScenarioCompleted { name: String, duration_ms: u64 },
    Error { message: String },
}

/// Agent metadata included in `ScenarioStarted`.
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct AgentInfo {
    pub id: String,
    pub model: String,
    pub wallet: String,
}

/// Trait for event emission backends.
#[async_trait::async_trait]
pub trait EventEmitter: Send + Sync {
    async fn emit(&self, event: DemoEvent);
}

/// No-op emitter — used when `--events none`.
pub struct NullEmitter;

/// NDJSON emitter — writes one JSON line per event to stdout.
pub struct NdjsonEmitter;

/// Composite emitter — fans out to multiple backends concurrently.
pub struct CompositeEmitter {
    pub backends: Vec<Arc<dyn EventEmitter>>,
}
```

Implement `EventEmitter` for each type:

**`NullEmitter`**: no-op.

```rust
#[async_trait::async_trait]
impl EventEmitter for NullEmitter {
    async fn emit(&self, _event: DemoEvent) {}
}
```

**`NdjsonEmitter`**: serialize to JSON, print to stdout with `println!`, which flushes implicitly
on each call because each call acquires stdout lock. Use `serde_json::to_string` and propagate
with a `tracing::warn!` on serialization error rather than panicking.

```rust
#[async_trait::async_trait]
impl EventEmitter for NdjsonEmitter {
    async fn emit(&self, event: DemoEvent) {
        match serde_json::to_string(&event) {
            Ok(line) => println!("{line}"),
            Err(e) => tracing::warn!("failed to serialize event: {e}"),
        }
    }
}
```

**`CompositeEmitter`**: call `emit` on all backends concurrently using `futures::future::join_all`
or `tokio::join!`. Because the number of backends is dynamic, use
`futures::future::join_all`:

```rust
#[async_trait::async_trait]
impl EventEmitter for CompositeEmitter {
    async fn emit(&self, event: DemoEvent) {
        let futs = self.backends.iter().map(|b| b.emit(event.clone()));
        futures::future::join_all(futs).await;
    }
}
```

Note: `futures` is available transitively through `tokio`. If it does not resolve, use a
sequential loop instead:

```rust
for backend in &self.backends {
    backend.emit(event.clone()).await;
}
```

The sequential fallback is acceptable; prefer it if the `futures` crate is not directly
importable.

### 2. `crates/roko-demo/src/ws_server.rs`

```rust
use std::sync::Arc;
use tokio::sync::broadcast;
use tokio_tungstenite::tungstenite::Message;

use crate::events::{DemoEvent, EventEmitter};

/// WebSocket event emitter — broadcasts JSON-serialized events to all connected clients.
pub struct WsEmitter {
    tx: broadcast::Sender<String>,
}

impl WsEmitter {
    /// Start a WebSocket server on `0.0.0.0:{port}`.
    ///
    /// Returns an `Arc<WsEmitter>` and a `JoinHandle` for the server task. Drop the handle
    /// to stop accepting new connections (existing connections are closed when the process exits).
    pub async fn start(port: u16) -> anyhow::Result<(Arc<Self>, tokio::task::JoinHandle<()>)> {
        let (tx, _rx) = broadcast::channel(1000);
        let emitter = Arc::new(Self { tx: tx.clone() });

        let listener = tokio::net::TcpListener::bind(("0.0.0.0", port)).await?;
        tracing::info!(port, "WebSocket event server listening");

        let handle = tokio::spawn(async move {
            loop {
                match listener.accept().await {
                    Ok((stream, addr)) => {
                        tracing::debug!(%addr, "ws client connected");
                        let rx = tx.subscribe();
                        tokio::spawn(handle_connection(stream, rx));
                    }
                    Err(e) => {
                        tracing::warn!("ws accept error: {e}");
                    }
                }
            }
        });

        Ok((emitter, handle))
    }
}

#[async_trait::async_trait]
impl EventEmitter for WsEmitter {
    async fn emit(&self, event: DemoEvent) {
        match serde_json::to_string(&event) {
            Ok(json) => {
                // send returns Err only when there are no receivers — that is fine
                let _ = self.tx.send(json);
            }
            Err(e) => tracing::warn!("ws: failed to serialize event: {e}"),
        }
    }
}

async fn handle_connection(
    stream: tokio::net::TcpStream,
    mut rx: broadcast::Receiver<String>,
) {
    let ws_stream = match tokio_tungstenite::accept_async(stream).await {
        Ok(ws) => ws,
        Err(e) => {
            tracing::debug!("ws handshake failed: {e}");
            return;
        }
    };

    use futures_util::SinkExt;

    let (mut sink, _source) = futures_util::StreamExt::split(ws_stream);

    loop {
        match rx.recv().await {
            Ok(json) => {
                if sink.send(Message::Text(json.into())).await.is_err() {
                    // client disconnected
                    break;
                }
            }
            Err(broadcast::error::RecvError::Lagged(n)) => {
                tracing::warn!("ws client lagged by {n} messages");
            }
            Err(broadcast::error::RecvError::Closed) => break,
        }
    }
}
```

`tokio-tungstenite` re-exports `futures-util` internally. Import from `futures_util` directly —
it is a transitive dependency. If the import does not resolve, add `futures-util = { workspace =
true }` to `Cargo.toml`, or use the `futures` crate equivalents.

### 3. Factory function

Place this in `crates/roko-demo/src/events.rs` after the emitter implementations:

```rust
/// Build an `EventEmitter` from a mode string.
///
/// - `"none"` → [`NullEmitter`]
/// - `"ndjson"` → [`NdjsonEmitter`]
/// - `"ws"` → [`WsEmitter`] bound to `ws_port`
/// - `"both"` → [`CompositeEmitter`] with NDJSON + WebSocket
pub async fn create_emitter(
    mode: &str,
    ws_port: u16,
) -> anyhow::Result<Arc<dyn EventEmitter>> {
    match mode {
        "none" => Ok(Arc::new(NullEmitter)),
        "ndjson" => Ok(Arc::new(NdjsonEmitter)),
        "ws" => {
            let (emitter, _handle) = crate::ws_server::WsEmitter::start(ws_port).await?;
            Ok(emitter)
        }
        "both" => {
            let (ws, _handle) = crate::ws_server::WsEmitter::start(ws_port).await?;
            Ok(Arc::new(CompositeEmitter {
                backends: vec![Arc::new(NdjsonEmitter), ws],
            }))
        }
        other => anyhow::bail!("unknown event mode: {other}. Valid: none, ndjson, ws, both"),
    }
}
```

Note: the `JoinHandle` is intentionally dropped here. The server task continues running because
the spawned future holds its own reference to the `TcpListener`. This is fine for the demo's
lifetime.

## Serialization contract

`DemoEvent` uses `#[serde(tag = "type", rename_all = "snake_case")]`. This means every variant
serializes to an object with a `"type"` field set to the snake_case variant name. Examples:

```json
{"type":"job_posted","job_id":1,"description":"Find best swap route","bounty_amount":"100000000"}
{"type":"scenario_started","name":"job-board","agents":[{"id":"agent-0","model":"stub","wallet":"0x..."}]}
{"type":"round_completed","round":1}
```

The NDJSON output is one such object per line, with no trailing comma or array wrapper.
