## Context

The yield-routing scenario runs two rounds. In each round, five workers race to submit the best
routing proposal for a 100K USDC â†’ ETH swap. The winner posts an insight about their strategy
to `InsightBoard`; the next round's workers read those insights before bidding.

This is the knowledge loop: query before â†’ act â†’ post after. It is also what makes C-Factor
measurable: round 2 agents see more knowledge and (with a real LLM) produce better output.

This task wires the query and post steps into the existing yield-routing spine and emits the
corresponding `DemoEvent` variants for each step.

## Step 2 â€” `InsightData` struct and `query_insights` function

In `crates/roko-demo/src/scenarios/yield_routing.rs`, add near the top of the file (after
imports, before the `YieldRouting` struct):

```rust
/// A single insight read from InsightBoard before agent bidding.
#[derive(Clone, Debug)]
struct InsightData {
    id: u64,
    content: String,
    pheromone: u64,
}
```

Add the query function. It reads all insights the board currently holds and returns them sorted
by pheromone (highest first) so the LLM context lists the most-confirmed insights at the top:

```rust
async fn query_insights(ctx: &ChainCtx) -> anyhow::Result<Vec<InsightData>> {
    let board_addr = ctx.address_of("InsightBoard")?;
    let provider = ctx.wallet_provider("deployer")?;
    let board = InsightBoard::new(board_addr, provider);

    let next_id: U256 = board.nextInsightId().call().await?;
    let count = next_id.to::<u64>();

    let mut insights = Vec::with_capacity(count as usize);
    for id in 0..count {
        // getInsight reverts on unknown id â€” treat any error as "skip".
        match board.getInsight(U256::from(id)).call().await {
            Ok(ret) => {
                // ret.uri holds the content URI; for demo purposes the URI *is*
                // the content string (posted as "demo://yield-routing:<content>").
                // Strip the scheme prefix if present.
                let raw_uri = ret.uri;
                let content = raw_uri
                    .strip_prefix("demo://yield-routing:")
                    .unwrap_or(&raw_uri)
                    .to_string();
                insights.push(InsightData {
                    id,
                    content,
                    pheromone: ret.pheromone,
                });
            }
            Err(e) => {
                tracing::warn!(id, "getInsight failed: {e}");
            }
        }
    }
    // Sort descending by pheromone so strongest signal appears first.
    insights.sort_by(|a, b| b.pheromone.cmp(&a.pheromone));
    Ok(insights)
}
```

`InsightBoard` must be imported from `crate::bindings`. The `U256` and `keccak256` imports come
from `alloy::primitives`.

## Step 4 â€” Post new insight after the winning agent is resolved

After the round's `BountyMarket::resolve` call completes (the job is accepted), the winning
agent posts a new insight about what they learned. The insight content is LLM-generated:

```rust
let insight_text = llm
    .fill(LlmRequest {
        slot: "insight_content".into(),
        context: serde_json::json!({
            "round": round,
            "job_id": job_id.to_string(),
            "winning_proposal": winning_proposal,
            "prior_insights_count": insights.len(),
        }),
    })
    .await?
    .as_str()
    .unwrap_or("prefer lower-slippage pools")
    .to_string();

// URI encodes the content inline so T2.2's query can recover it without off-chain storage.
let uri = format!("demo://yield-routing:{insight_text}");
let content_hash = keccak256(insight_text.as_bytes());

let winner_provider = ctx.wallet_provider(&winner_name)?;
let board_via_winner = InsightBoard::new(board_addr, winner_provider);
let post_receipt = board_via_winner
    .post(content_hash.into(), uri)
    .send()
    .await?
    .get_receipt()
    .await?;

// Read back the ID from the contract (nextInsightId - 1 after post).
let new_insight_id: u64 = board.nextInsightId().call().await?.to::<u64>() - 1;

events.emit(DemoEvent::InsightPosted {
    agent_id: winner_name.clone(),
    insight_id: new_insight_id,
    content: insight_text.clone(),
}).await;
tracing::info!(round, insight_id = new_insight_id, "insight posted");

// Two other workers confirm the insight to grow pheromone weight.
for confirmer_idx in 0..2usize {
    // Use a worker other than the winner.
    let idx = (winner_idx + confirmer_idx + 1) % WORKERS;
    let confirmer_name = format!("worker{idx}");
    let confirmer_provider = ctx.wallet_provider(&confirmer_name)?;
    let board_via_confirmer = InsightBoard::new(board_addr, confirmer_provider);
    board_via_confirmer
        .confirm(U256::from(new_insight_id))
        .send()
        .await?
        .watch()
        .await?;
    events.emit(DemoEvent::InsightConfirmed {
        confirmer_id: confirmer_name,
        insight_id: new_insight_id,
    }).await;
}
```

`winning_proposal` is the JSON value returned by the winner's `llm.fill` call for
`route_proposal`; store it in a local before the submission step.
