## Context

Running a scenario today requires an upfront prepare phase: mint tokens, approve spending,
register in `WorkerRegistry`, register in `AgentRegistry`. A new participant who wants to join
mid-session cannot do so without modifying the spine.

This task adds `roko-demo register-agent`, a self-contained subcommand that completes the entire
onboarding flow for a single agent in one call. The command reads existing deployment addresses
from `deployments.json`, drives the three contract calls, and prints a formatted confirmation
with timing. It demonstrates that joining the Nunchi network takes under 60 seconds.

## Contract bindings reference

All bindings are in `crates/roko-demo/src/bindings.rs`.

```rust
// MockERC20
fn mint(address to, uint256 amount);
fn approve(address spender, uint256 amount) -> bool;
fn balanceOf(address account) -> uint256;

// WorkerRegistry
fn register(uint256 stake);          // called by the worker wallet
fn reputationOf(address) -> uint256;
fn tier(address) -> uint8;
fn setAuthorized(address, bool);

// AgentRegistry
fn register(string capabilities, bytes32 passportHash);
fn isActive(address) -> bool;
fn registeredCount() -> uint256;
```

`WorkerRegistry.register` takes only `stake` — the caller is `msg.sender`. The wallet calling
`register` becomes the registered worker.

## What to implement

### 1. New subcommand variant

In `crates/roko-demo/src/main.rs`, add `RegisterAgent` to the `Cmd` enum:

```rust
/// Register a new agent in one step: mint DAEJI, stake bond, register in both registries.
RegisterAgent {
    /// Human-readable agent name (used in output only).
    #[arg(long)]
    name: String,

    /// LLM model this agent runs. Stored as capabilities string.
    #[arg(long, default_value = "gemma-7b")]
    model: String,

    /// Stake amount in whole DAEJI tokens (scaled by 1e18 internally).
    #[arg(long, default_value_t = 1000u64)]
    stake: u64,

    /// Wallet name from demo/wallets.toml (e.g. "worker4").
    #[arg(long)]
    wallet: String,

    /// Scenario name — used to locate deployments.json.
    #[arg(long, default_value = "yield-routing")]
    scenario: String,
},
```

### 2. Handler function

Add a free async function in `main.rs`:

```rust
async fn register_agent_cmd(cli: &Cli, args: &RegisterAgentArgs) -> anyhow::Result<()>
```

Where `RegisterAgentArgs` is a struct carrying the fields destructured from the enum variant,
or inline the logic directly in the `match` arm — whichever pattern the existing handlers use.
Follow the style already in the file.

#### Registration flow

1. **Record start time** with `std::time::Instant::now()`.

2. **Load deployments.** Read `cli.runtime_dir.join("deployments.json")`. Deserialize using
   `DeployedSuite` (or the type that `deploy.rs` writes). Extract the addresses for `MockERC20`,
   `WorkerRegistry`, and `AgentRegistry`.

   If `deployments.json` does not exist, bail with a clear error:
   ```
   deployments not found — run `roko-demo deploy yield-routing` first
   ```

3. **Build provider.** Construct a wallet provider for `args.wallet` using the same pattern as
   `agents_cmd`. The deployer wallet must also be available for the `mint` call (minting is
   restricted to the deployer in `MockERC20`).

4. **Mint DAEJI.** From the deployer wallet, call:
   ```rust
   MockERC20.mint(worker_address, stake_amount_scaled)
   ```
   where `stake_amount_scaled = U256::from(args.stake) * U256::from(10u128.pow(18))`.

5. **Approve WorkerRegistry.** From the worker wallet, call:
   ```rust
   MockERC20.approve(worker_registry_address, stake_amount_scaled)
   ```

6. **Register in WorkerRegistry.** From the worker wallet, call:
   ```rust
   WorkerRegistry.register(stake_amount_scaled)
   ```
   If the call returns an error whose message contains `"already registered"`, log a warning
   and continue rather than aborting.

7. **Register in AgentRegistry.** From the worker wallet, call:
   ```rust
   AgentRegistry.register(
       capabilities_string,
       passport_hash,
   )
   ```
   where:
   - `capabilities_string = format!("model:{} task:routing", args.model)`
   - `passport_hash = keccak256(capabilities_string.as_bytes())` (use `alloy::primitives::keccak256`)

   If the call returns an error whose message contains `"already registered"`, log a warning
   and continue.

8. **Read post-registration state.** Using the read provider:
   - `WorkerRegistry.reputationOf(worker_address)` → `rep`
   - `WorkerRegistry.tier(worker_address)` → `tier_num`

9. **Map tier number to label:**
   ```rust
   let tier_label = match tier_num {
       0 => "Unregistered",
       1 => "Probation",
       2 => "Standard",
       3 => "Trusted",
       4 => "Elite",
       _ => "Unknown",
   };
   ```

10. **Print confirmation** to stdout (not tracing — this is user-facing output):
    ```
    Agent registered!
      Name:   my-trader
      Model:  gemma-7b
      Wallet: worker4
      Stake:  1000 DAEJI
      Tier:   Standard
      Time:   2.3s
      Ready to bid on jobs.
    ```
    Format using `println!`. Compute elapsed with `start.elapsed().as_secs_f64()` and format to
    one decimal place: `format!("{:.1}s", elapsed)`.

### 3. Wire the subcommand in `main`

In the `match cli.cmd` block, add a branch that calls `register_agent_cmd`. Follow the pattern
of existing branches exactly.

## Output format (exact)

```
Agent registered!
  Name:   {name}
  Model:  {model}
  Wallet: {wallet}
  Stake:  {stake} DAEJI
  Tier:   {tier_label}
  Time:   {elapsed}
  Ready to bid on jobs.
```

Use two-space indentation, a colon, and enough spaces after the label to align values at column 10
(name field width: 8 characters including colon). Matching the example above is sufficient — pixel
perfect alignment is not required.
