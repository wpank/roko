## Context

Every scenario in `roko-demo` today is orchestrated: a single Rust function drives agent actions
in a deterministic script — post job, assign winner, submit, resolve, repeat. There is no
competition, no timing, and no independent agent decision-making.

This task adds an `autonomous` subcommand that runs agents as independent concurrent tasks. Each
agent polls `BountyMarket` for open jobs, queries `InsightBoard` for context, asks an `LlmProvider`
for a bid decision, submits a bid, and executes the job if assigned. A separate poster task
periodically publishes new jobs. No central coordinator decides who wins — agents compete on the
bid value they return from the LLM.

The mode is a clean test of the full agent loop: is the system safe to run unscripted?

## Architecture

Two kinds of Tokio tasks run concurrently:

```
poster_task  ──→  BountyMarket.postJob × total_jobs  (one post every `interval` seconds)

agent_task_0 ─┐
agent_task_1 ─┤  each: poll → query knowledge → bid → execute → post insight
agent_task_2 ─┤
agent_task_3 ─┤
agent_task_4 ─┘
```

There is no message passing between agent tasks. Each agent uses its own wallet provider and
reads shared chain state independently.

Assignment coordination: `BountyMarket.assign` requires either the poster or the resolver to call
it. Because agents cannot assign themselves, the poster task is also responsible for assignment.
After each `postJob`, the poster waits for bids (polled via on-chain state or a shared
`tokio::sync::broadcast` channel), selects the best bidder by `expected_output`, and calls
`assign`. This makes the poster an auctioneer, not a controller — agents still drive bidding and
execution independently.

#### Shared state

use std::sync::Arc;
use tokio::sync::{broadcast, Mutex};

/// Bid submitted by one agent for one job.
#[derive(Debug, Clone)]
pub struct AgentBid {
    pub job_id: u64,
    pub agent_index: usize,     // 0..agents
    pub expected_output: f64,
}

/// Shared bid collection for one open job.
/// The poster creates one per job and drops it after assignment.
pub struct BidCollector {

#### Agent loop

```rust
pub async fn agent_loop(
    ctx: Arc<ChainCtx>,
    agent_index: usize,
    model: String,
    llm: Arc<dyn LlmProvider>,
    events: Arc<dyn EventEmitter>,
    bid_tx: Arc<Mutex<Option<broadcast::Sender<AgentBid>>>>,
    // `bid_tx` is Some while a job is open, None otherwise.
    // The poster sets it when a job is posted and clears it after assignment.
    timeout: std::time::Duration,
) -> anyhow::Result<()>
```

Loop body:

1. **Check for an open job.** Lock `bid_tx`. If `None`, sleep 500ms and retry. If `Some(tx)`,
   release the lock and proceed.

2. **Query InsightBoard.** Call `InsightBoard.nextInsightId()` to get the total count. If
   count > 0, read up to the last 3 insight IDs by calling a local helper that fetches the
   `contentHash` for each ID. Emit `DemoEvent::KnowledgeQueried { agent_id, insights_found }`.

3. **Ask the LLM.** Build an `LlmRequest` with slot `"route_proposal"` and context:
   ```json
   {
     "worker": "worker{i}",
     "model": "{model}",
     "job": "Route 100K USDC -> ETH, maximize output",
     "insights_found": N
   }
   ```
   Parse the response as a `RouteProposal` (same struct as in `yield_routing.rs`).

4. **Submit bid.** Lock `bid_tx` again. If still `Some`, send an `AgentBid` through the channel.
   Log: `tracing::info!(agent = %format!("worker{agent_index}"), expected_output = proposal.expected_output, "bid submitted")`.
   Emit `DemoEvent::AgentBid { agent_id, model, expected_output, strategy_summary }`.

5. **Wait for assignment.** Poll `BountyMarket.stateOf(job_id)` every 200ms. The state transitions
   are: `Funded (0) → Assigned (1) → Submitted (2) → Terminal (3/4)`. Wait until state != 0.
   If state moves to `Assigned`:
   - Check if the assigned worker is this agent: `BountyMarket.workerOf(job_id) == ctx.wallet_address(worker_name)`.
   - If not this agent, go back to step 1 (wait for the next job).
   - If this agent, proceed to execution.

6. **Execute.** Call `BountyMarket.submit(job_id, result_hash)` from this agent's wallet:
   ```rust
   let result_hash = keccak256(serde_json::to_vec(&proposal.hops)?.as_slice());
   ```
   Emit `DemoEvent::ExecutionStarted` and `DemoEvent::ExecutionCompleted`.

7. **Post insight.** Call `InsightBoard.post(content_hash)` where:
   ```rust
   let insight = format!("worker{agent_index} round result: expected_output={:.2}", proposal.expected_output);
   let content_hash = keccak256(insight.as_bytes());
   ```
   Emit `DemoEvent::InsightPosted`.

8. **Loop.** Go back to step 1.

Exit the loop when `timeout` elapses (check `Instant::now() > start + timeout` at the top of
each iteration).

#### Poster loop

```rust
pub async fn poster_loop(
    ctx: Arc<ChainCtx>,
    bid_tx_slot: Arc<Mutex<Option<broadcast::Sender<AgentBid>>>>,
    interval: std::time::Duration,
    total_jobs: u32,
    agent_count: usize,
    events: Arc<dyn EventEmitter>,
) -> anyhow::Result<()>
```

Loop body (repeat `total_jobs` times, then return):

1. **Sleep `interval`.** Use `tokio::time::sleep(interval).await`.

2. **Post job.** `poster0` calls `BountyMarket.postJob(spec_hash, bounty)`:
   - `spec_hash = keccak256(format!("autonomous-job-{n}").as_bytes())`
   - `bounty = 1_000 * 10u128.pow(18)`
   - Capture `job_id = BountyMarket.nextJobId() - 1`.
   Emit `DemoEvent::JobPosted { job_id, description, bounty_amount }`.

3. **Open the bid window.** Create a new `broadcast::channel(agent_count + 4)` and put the
   sender in `bid_tx_slot`:
   ```rust
   let (tx, _) = broadcast::channel(agent_count + 4);
   *bid_tx_slot.lock().await = Some(tx);
   ```

4. **Wait for bids.** Sleep for 2 seconds (hardcoded bid window). Then subscribe to the channel
   and drain all bids sent during that window. Collect into `Vec<AgentBid>`.

5. **Close the bid window.** Set `*bid_tx_slot.lock().await = None`.

6. **Select winner.** Find the bid with the highest `expected_output`. If no bids were received,
   log a warning and continue to the next job.

7. **Assign.** Deployer calls `BountyMarket.assign(job_id, worker_address)` where
   `worker_address = ctx.wallet_address(format!("worker{}", winner.agent_index))`.
   Emit `DemoEvent::JobAssigned { job_id, agent_id }`.

8. **Validate.** Deployer calls `ConsortiumValidator.assembleCommittee(job_id)`. For each
   validator in `validator0`–`validator2`, call `ConsortiumValidator.vote(job_id, true)`.
   Then call `BountyMarket.resolve(job_id)`.
   Emit `DemoEvent::ValidationComplete { job_id, accepted: true }`.

#### Public entry point

```rust
pub async fn run_autonomous(
    ctx: Arc<ChainCtx>,
    llm: Arc<dyn LlmProvider>,
    events: Arc<dyn EventEmitter>,
    agents: u32,
    jobs: u32,
    interval: std::time::Duration,
    timeout: std::time::Duration,
) -> anyhow::Result<()>
```

Steps:

1. Emit `DemoEvent::ScenarioStarted { name: "autonomous".to_string(), agents: ... }`.
2. Create `bid_tx_slot: Arc<Mutex<Option<broadcast::Sender<AgentBid>>>> = Arc::new(Mutex::new(None))`.
3. Spawn `agents` agent tasks using `tokio::spawn`.
4. Spawn one poster task using `tokio::spawn`.
5. Await all tasks with `tokio::try_join!` or `futures::future::try_join_all`. Propagate the first
   error, if any.
6. Emit `DemoEvent::ScenarioCompleted { name: "autonomous".to_string(), duration_ms }`.
7. Return `Ok(())`.

Use `tokio::time::timeout(timeout, ...)` around the outer join to enforce the scenario time limit.
On timeout, log a warning and return `Ok(())` rather than an error — a partial run is not a failure.
