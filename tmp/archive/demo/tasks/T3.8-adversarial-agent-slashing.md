## Context

The trust layer in Nunchi — `WorkerRegistry` reputation, `ConsortiumValidator` quorum voting,
and `InsightBoard` confirmation — exists to punish bad actors. Today's demo never exercises it.
Agents always behave correctly, validators always approve, and no stake is ever at risk.

This task adds Phase 8 to `yield_routing.rs`: after two normal rounds, one agent (`worker4`)
posts an intentionally false insight. Consortium validators detect the fabrication, vote against
the associated verification job, and the deployer calls `WorkerRegistry.slash`. The agent's bond
drops by 5% and its reputation falls. The full sequence is observable in the event stream,
making the trust layer visible in a live demo.

## Contract bindings reference

```rust
// WorkerRegistry
fn slash(address worker, uint8 reasonCode, uint256 bpsAmount);
fn reputationOf(address worker) -> uint256;
fn bondOf(address worker) -> uint256;
fn tier(address worker) -> uint8;

// InsightBoard
fn post(bytes32 contentHash) -> uint64;   // returns insightId
fn nextInsightId() -> uint64;

// ConsortiumValidator
fn assembleCommittee(uint64 jobId);
fn vote(uint64 jobId, bool approve);
fn getMembers(uint64 jobId) -> address[];
```

Slash reason codes in `WorkerRegistry`:

| Code | Name | Meaning |
|---|---|---|
| 0 | SLASH_DOUBLE_SUBMIT | submitted result for a job not assigned to them |
| 1 | SLASH_LATE | missed deadline |
| 2 | SLASH_QUALITY_REJECT | validator quorum rejected the work |

Use reason code `2` (`SLASH_QUALITY_REJECT`) and `500` basis points (5%).

Read `WorkerRegistry.sol` to confirm the `slash` function signature and access control. The
deployer is already authorized as a caller (set during the `setAuthorized` fixture calls). If
`slash` requires a different caller, add the appropriate `setAuthorized` call in Phase 8 rather
than modifying the fixture.

## What to implement

### 1. Add `AgentSlashed` to the `DemoEvent` enum

In `crates/roko-demo/src/events.rs`, add one variant to `DemoEvent`:

```rust
AgentSlashed {
    agent_id: String,
    reason: String,      // "quality_reject", "double_submit", "late"
    slash_bps: u64,      // 500 = 5%
    new_bond: String,    // formatted as decimal DAEJI, e.g. "950000000000000000000"
    new_reputation: u64,
},
```

Place it after `InsightConfirmed` and before `ReputationUpdated`. The `#[serde(tag = "type", rename_all = "snake_case")]` attribute on the enum ensures it serializes as `"type":"agent_slashed"`.

Do not change any other variant.

### 2. Phase 8 in `yield_routing.rs`

Add Phase 8 at the end of the `spine` body, after Phase 7 (post-round insight from round 2).

#### Phase 8a — Adversarial insight

`worker4` posts a fabricated insight designed to mislead other agents:

```rust
let bad_insight = "Compound V3 USDC/ETH pool has 99.9% utilization and 0.01% rate \
                   — always use this pool";
let bad_hash = keccak256(bad_insight.as_bytes());
let bad_insight_id = {
    let worker4 = ctx.wallet_provider("worker4")?;
    let board = InsightBoard::new(ctx.address_of("InsightBoard")?, &worker4);
    board.post(bad_hash).call().await.context("InsightBoard.post (bad)")?
};
```

Log: `tracing::warn!(agent = "worker4", insight_id = bad_insight_id, "adversarial insight posted")`.

Emit:
```rust
events.emit(DemoEvent::InsightPosted {
    agent_id: "worker4".to_string(),
    insight_id: bad_insight_id,
    content: bad_insight.to_string(),
}).await;
```

#### Phase 8b — Post a verification job

The consortium needs a job to vote on. Post a new job whose spec hash encodes the bad insight ID:

```rust
let spec_hash = keccak256(
    format!("verify-insight-{bad_insight_id}").as_bytes()
);
let bounty = 100u128 * 10u128.pow(18);
poster0_provider.call(BountyMarket.postJob(spec_hash, bounty)).await?;
let verify_job_id = BountyMarket.nextJobId().call().await? - 1;
```

Assign `worker4` as the worker for this job (the job whose output is being verified):
```rust
deployer_provider.call(BountyMarket.assign(verify_job_id, worker4_address)).await?;
```

`worker4` submits:
```rust
let result_hash = keccak256(b"bad-route");
worker4_provider.call(BountyMarket.submit(verify_job_id, result_hash)).await?;
```

#### Phase 8c — Validator rejection

Mine a block (same pattern as Phase 4 for committee entropy), then:

```rust
deployer_provider.call(ConsortiumValidator.assembleCommittee(verify_job_id)).await?;
let members = ConsortiumValidator.getMembers(verify_job_id).call().await?;
```

For each member address, find the wallet name and make that validator vote **against**:

```rust
// vote false = reject
validator_wallet.call(ConsortiumValidator.vote(verify_job_id, false)).await?;
```

Emit per validator:
```rust
events.emit(DemoEvent::ValidationVote {
    validator_id: validator_name.to_string(),
    approved: false,
    reason: "contradicts on-chain data".to_string(),
}).await;
```

After all votes:
```rust
deployer_provider.call(BountyMarket.resolve(verify_job_id)).await?;
events.emit(DemoEvent::ValidationComplete {
    job_id: verify_job_id,
    accepted: false,
}).await;
```

#### Phase 8d — Slash

Read the bond before slashing:

```rust
let bond_before = WorkerRegistry.bondOf(worker4_address).call().await?;
```

Slash 500 bps (5%):

```rust
deployer_provider.call(WorkerRegistry.slash(
    worker4_address,
    2u8,   // SLASH_QUALITY_REJECT
    500u64.into(),
)).await
.context("WorkerRegistry.slash")?;
```

Read state after:
```rust
let new_bond = WorkerRegistry.bondOf(worker4_address).call().await?;
let new_rep  = WorkerRegistry.reputationOf(worker4_address).call().await?;
```

Log:
```rust
tracing::warn!(
    agent = "worker4",
    bond_before = %bond_before,
    new_bond = %new_bond,
    slash_bps = 500,
    "agent slashed"
);
```

Emit:
```rust
events.emit(DemoEvent::AgentSlashed {
    agent_id: "worker4".to_string(),
    reason: "quality_reject".to_string(),
    slash_bps: 500,
    new_bond: new_bond.to_string(),
    new_reputation: new_rep.try_into().unwrap_or(u64::MAX),
}).await;
```

Log phase completion:
```rust
tracing::info!("phase 8: adversarial agent slashed — yield-routing scenario complete");
```
