# Agent Runbook - 10 Dreams

Batch `10` is a **status-regeneration and frontier-tagging** pass.

## Default posture

- trust `crates/roko-dreams` first
- treat Doc 16 as the main stale artifact
- prefer correcting ownership, status, and scope over adding dream code
- treat the helper shell script as prompt preparation unless it is explicitly upgraded to execute batches

## What good work looks like

- make shipped trigger, replay, imagination, hypnagogia, and threat surfaces visible
- remove stale `roko-golem` ownership claims
- distinguish shipped runtime from shipped support infrastructure
- mark frontier material as target-state
- leave one obvious source index and one obvious status note for later agents

## What to avoid

- do not implement missing runtime features
- do not widen into oneirography, mesh sharing, or safety execution work
- do not describe frontier designs in present tense

## Deliverable standard

Each real execution batch should leave:

- explicit status or banner updates
- verification commands and results
- clear deferrals for out-of-scope findings
- a concise pass/fail-style outcome in the agent report
