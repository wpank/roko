Implemented the concrete `roko-agent` parity gaps and kept the patch additive.

The main runtime fixes are:
- MCP discovery now passes configured server `env` into spawned MCP processes and `find_mcp_config` now falls back to `~/.mcp.json`.
- Translator selection now supports capability/profile-based routing for Gemini native models via `GeminiTranslator`, while preserving the existing slug-based entrypoint.
- Tool-loop checkpoints now persist and restore `SessionState`, so resumed runs keep provider conversation/session IDs.
- Added the doc-compat `AgentResult::all_engrams()` alias.
- Exposed the pool/task-runner surfaces the docs name at the crate root, made `WarmEntry` public, added `MultiAgentPool` alias methods (`evict_stale_warm`, `kill_by_plan`, `kill_by_role`), and added `AgentTask.priority` with a `with_priority` builder.

Verification passed:
- `cargo check -p roko-agent`
