## Rules

1. **Search before writing.** `grep -rn 'TypeName' crates/ --include='*.rs' | grep -v target/` — this codebase has duplicates from parallel development.
2. **Wire, don't build.** If existing code just needs to be called, call it. Don't rebuild.
3. **Compile clean.** Every verify command must pass. `-D warnings` means fix the warning, don't `#[allow]` it unless the rule is genuinely wrong.
4. **Write-scope discipline.** Stay inside the listed write scope. If you need a 1-line fix in an adjacent crate for compilation, make it and document it.
5. **Naming conventions.** Use `neuro` (not grimoire), `Korai` (not styx/Styx), `fleet` (not clade). No death/mortality concepts.
6. **No destructive git.** Never force-push, reset, or rm outside the worktree. The runner handles branch lifecycle.
7. **Phase 2+ stubs.** If a doc describes chain/daimon/dreams functionality, add a stub type with doc comment — no full logic.
