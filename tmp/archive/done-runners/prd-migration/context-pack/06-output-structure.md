## Required files in your topic folder

1. **`INDEX.md`** — the topic index. Required for every topic. Must contain:
   - `# <Topic Title>` heading
   - One-paragraph topic summary
   - `## Prerequisites` section (optional) listing sub-docs or topics to read first
   - `## Contents` section with a markdown table or list linking every sub-doc
   - `## Cross-references` section listing related topic folders
   - `## Generation Notes` section at the bottom with metadata about your generation run
   - Minimum 50 lines

2. **Sub-docs** — at least 10 focused sub-docs per topic (15-20 for rich topics).
   Each is a standalone file named `NN-<slug>.md`:
   - Two-digit numeric prefix for ordering (00, 01, 02, ...)
   - Descriptive slug in kebab-case
   - `.md` extension
   - Example: `00-vision-and-thesis.md`, `03-synapse-traits.md`, `12-provenance-and-attestation.md`

## Minimum content requirements per sub-doc

- **Lines**: 200 minimum, 500-2000 target for rich content.
- **Sections**: at least 3 level-2 headings.
- **Citations**: at least 2-3 citations per sub-doc (unless the sub-doc is purely implementation-focused).
- **Code samples**: if the sub-topic involves Rust traits, types, or functions, include full Rust code samples.
- **Cross-references**: link to at least 2 other sub-docs or topics.
- **Abstract**: non-empty, 2-4 paragraphs.

## Minimum content requirements per topic

- **Sub-docs**: at least 10, ideally 12-18.
- **Total line count**: at least 2500 lines across all files in the topic folder.
- **INDEX.md**: at least 50 lines, references every sub-doc.
- **Citations**: at least 15 citations total across the topic (fewer is OK for low-citation topics like tools, deployment, interfaces).

## File naming conventions

- Sub-docs are numbered from `00` upward.
- The numeric prefix reflects logical reading order, not priority.
- Slugs use kebab-case with no spaces or special characters.
- Avoid abbreviations unless they're universally known (e.g., `hdc`, `pad`, `vcg`).
- Example good names: `03-universal-cognitive-loop.md`, `07-tier-progression.md`
- Example bad names: `UnivLoop.md`, `03_universal_loop.MD`, `final_v2.md`

## Writing order (recommended)

1. Read all context-pack files (you're doing this now).
2. Read the refactoring-prd files listed in your prompt.
3. Read the SOURCE-INDEX.md entry for your topic.
4. Read the legacy PRD files, research/tmp files, implementation-plan files, and
   reference code files listed in the SOURCE-INDEX entry.
5. Plan your sub-doc breakdown (aim for 10-18 sub-docs).
6. Write each sub-doc fully before moving to the next one. Do not leave half-finished drafts.
7. Write the INDEX.md last, after all sub-docs exist.
8. Run your mental self-check against `04-writing-rules.md` Rule 15.
9. Add the Generation Notes section to INDEX.md.

## Parallel agents

You are one of up to 22 agents running in parallel. Each agent owns exactly one topic.
Agents do not share state. Do not write files outside your topic folder. Do not read
files from other topic folders to compose cross-references — use forward references by
path without verifying they exist.

The runner script generates the master `INDEX.md` at `docs/INDEX.md` after all topics
finish. You do not need to write this file.
