# Agent Runbook — 10 Dreams

Batch `10` is a **status-regeneration and frontier-tagging pass**.

## Default posture

- Trust `crates/roko-dreams` first.
- Treat Doc 16 as the main stale artifact.
- Prefer correcting ownership, status, and scope over adding new dream code.

## What good work looks like

- make shipping dream triggers, CLI, hypnagogia, and threat simulation visible,
- remove stale `roko-golem` ownership claims,
- split mixed docs into shipping vs partial vs future,
- leave later agents with one obvious status document and one obvious source index.

## What to avoid

- do not implement missing runtime features,
- do not widen into oneirography, mesh sharing, or safety systems,
- do not overfit docs to speculative future plans.

## Deliverable standard

Each batch should leave:

- explicit status or banner updates,
- verification commands and results,
- deferrals for out-of-scope findings,
- a clear `PASS`, `FAIL`, or `BLOCKED` result.
