# Agent Runbook — 12 Interfaces

Batch `12` is a **status-calibration and scope-discipline pass**.

## Default posture

- Trust checked-in CLI/server/TUI code before trusting the interface PRDs.
- Treat the shipping core and the innovation halo as different classes of work.
- Prefer narrowing or downgrading overclaiming docs over trying to “save” them with speculative implementation assumptions.

## What good work looks like

- explicit status correction for CLI, server, sidecar, and TUI,
- explicit frontier banners for Spectre, portal, A2UI, sonification, UX innovation, and IDE work,
- explicit treatment of `roko new`, standalone `roko explain`, and serve-port drift,
- acceptance criteria that can be checked with grep and source inspection,
- clear deferrals when work belongs to a later frontend/IDE/audio pass.

## What to avoid

- do not invent missing frontend, visualization, or IDE runtime surfaces,
- do not treat README claims as equal to source code when they conflict,
- do not overstate route behavior when only route-module presence was verified,
- do not widen a parity batch into a product design project.

## Deliverable standard

Each batch should leave:

- clear scope and out-of-scope boundaries,
- explicit verification commands,
- concrete acceptance criteria,
- stable notes for the next overnight agent.
