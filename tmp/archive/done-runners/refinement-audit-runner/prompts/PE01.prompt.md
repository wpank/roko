This section covers the orchestration layer: PlanRunner, DAG executor, speculative execution, worktree isolation, merge queue, snapshot/recovery persistence, conductor integration, and domain-aware task foundations.

## Current State

- `PlanRunner` has 50+ fields and runs the plan-execute-gate-persist loop end-to-end.
- `UnifiedTaskDag` is fully implemented with `waves()`, `cpm_analysis()`, `DagMutation`, but runtime still schedules through flat `TaskTracker`.
- `SpeculativeExecution` exists in executor API but not in runtime dispatcher.
- Snapshots are functional but not hardened against corruption.
- `EventLog` has hash-chained integrity but recovery does not validate it.
- Worktree lifecycle is implemented but activity/health not exercised in unattended runs.

## Implementation

Follow the 6 batches (O1-O6) defined in `tmp/docs-parity/01/BATCHES.md`:

1. **O1**: Harden snapshot + recovery integrity in `crates/roko-orchestrator/src/executor/snapshot.rs` and `recovery.rs`.
2. **O5**: Exercise worktree activity/health in `crates/roko-orchestrator/src/worktree.rs` and `orchestrate.rs`.
3. **O2**: Dispatch speculative execution for real in `crates/roko-cli/src/orchestrate.rs` and `executor/mod.rs`.
4. **O3**: Wire DAG capabilities into runtime scheduling via `crates/roko-orchestrator/src/dag.rs` and `orchestrate.rs`.
