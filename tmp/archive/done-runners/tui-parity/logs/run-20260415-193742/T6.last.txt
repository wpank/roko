Implemented per-agent route/context metrics across the TUI in [state.rs](/Users/will/dev/nunchi/roko/roko/.roko/worktrees/tui-parity-run-20260415-193742/crates/roko-cli/src/tui/state.rs), [agents_view.rs](/Users/will/dev/nunchi/roko/roko/.roko/worktrees/tui-parity-run-20260415-193742/crates/roko-cli/src/tui/views/agents_view.rs), and [dashboard_view.rs](/Users/will/dev/nunchi/roko/roko/.roko/worktrees/tui-parity-run-20260415-193742/crates/roko-cli/src/tui/views/dashboard_view.rs). `TuiState` now carries a documented `RouteMetrics` map keyed by agent id, populated from the latest efficiency event per agent in polling mode and preserved/synthesized in connected mode. Since the real event type does not expose literal `tier`, `context_limit`, or `focus_score`, the mapping is adapted from repository reality: model from the event slug, tier inferred from operating frequency/model family, context usage from total tokens, context limit from known model windows, and focus from cascade-router confidence when available with a prompt-retention fallback.

In F3, the agent output pane now has a compact metrics bar above the output stream showing model, context usage, focus, and tier, with context utilization color-coded and focus dimmed/brightened by score. In the dashboard Agents sub-tab, there is now a compact `Routes` table between the parallel pool and the output panel with `Agent`, `Model`, and `Tier` columns sourced from the shared route-metrics state.
