## Task

Create a segment parser that classifies agent output lines into semantic types,
groups consecutive same-type lines, caches the rendered result, and integrates
the styled output into the agents view right panel.

## Implementation

### 1. Create `segment.rs` (new file)

```rust
/// Semantic segment types for agent output
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum SegmentKind {
    Thinking,    // Natural language reasoning
    Heading,     // Markdown headers
    ToolUse,     // Tool invocations
    Code,        // Code blocks
    Success,     // Positive outcomes
    Error,       // Failures
    Blank,       // Empty lines
    TurnMarker,  // Turn boundaries
}

pub struct SegmentGroup {
    pub kind: SegmentKind,
    pub lines: Vec<String>,
}

pub struct CachedRender {
    pub last_len: usize,
    pub groups: Vec<SegmentGroup>,
    pub styled_lines: Vec<ratatui::text::Line<'static>>,
}
```

**Parsing rules** (from Mori reference):
- `Heading`: starts with `# `, `## `, or `### `
- `ToolUse`: starts with `▸ `, `> `, `$ `, or starts with `Running `, `Reading `, `Writing `, `Editing `, `Created `
- `Code`: between ``` fences, or 4+ leading spaces after blank
- `Success`: contains `✓`, `PASS`, `APPROVE`
- `Error`: contains `ERROR`, `FAILED`, `error[`, starts with `error`
- `TurnMarker`: matches `────` pattern
- `Blank`: empty or whitespace-only
- `Thinking`: everything else

**Preprocessing**:
- Insert newlines before "Now ", "Let ", "I'll " to break long paragraphs
- Split lines >120 chars at sentence boundaries (`. `)

**Grouping**: consecutive lines of the same kind collapse into one `SegmentGroup`.

### 2. Style each segment kind

Use ROSEDUST theme colors:
- `Thinking`: default foreground (dim white)
- `Heading`: bold, rose highlight
- `ToolUse`: cyan/blue
- `Code`: green on dark background
- `Success`: bright green
- `Error`: bright red
- `Blank`: empty
- `TurnMarker`: dim horizontal rule

### 3. CachedRender integration

Add a `HashMap<String, CachedRender>` to TuiState keyed by agent_id.
On each frame, check if the agent's output length changed; if so, re-parse
and cache. Otherwise reuse cached lines.

### 4. Wire into agents_view.rs

Replace the raw text rendering in the right panel with the cached styled lines.
Each group gets a subtle header line showing the segment kind.
