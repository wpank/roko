## Task

Add per-agent context metrics and routing information display, showing model,
tier, context usage, and focus score in a compact metrics bar above agent output
and as columns in the dashboard agent sub-tab.

## Implementation

### 1. Add RouteMetrics struct (`state.rs`)

```rust
pub struct RouteMetrics {
    pub model: String,          // e.g., "sonnet-4"
    pub tier: String,           // e.g., "fast", "balanced", "deep"
    pub context_used: u64,      // tokens consumed
    pub context_limit: u64,     // model context window
    pub focus_score: f64,       // 0.0-1.0 relevance score
}
```

Add `pub route_metrics: HashMap<String, RouteMetrics>` to TuiState (keyed by agent_id).

### 2. Populate from efficiency events

In `update_from_snapshot()`:
```rust
// Read efficiency events from DashboardData.efficiency_events
// Extract model, tier, context usage per agent
for event in &data.efficiency_events {
    if let Some(agent_id) = &event.agent_id {
        self.route_metrics.entry(agent_id.clone()).or_insert_with(|| RouteMetrics {
            model: event.model.clone().unwrap_or_default(),
            tier: event.tier.clone().unwrap_or_default(),
            context_used: event.tokens_used.unwrap_or(0),
            context_limit: event.context_limit.unwrap_or(200_000),
            focus_score: event.focus_score.unwrap_or(0.0),
        });
    }
}
```

Adapt the field names to match the actual `AgentEfficiencyEvent` struct in `dashboard.rs`.

### 3. Render metrics bar in agents_view.rs

Above the agent output panel, render a compact one-line metrics bar:
```
[sonnet-4] ctx: 45k/200k │ focus: 0.87 │ tier: balanced
```

Use ROSEDUST theme colors:
- Model name: bold
- Context usage: color-coded by utilization (green <50%, yellow <80%, red >=80%)
- Focus score: dim if low, bright if high

### 4. Add model/tier columns to dashboard agent sub-tab

In `dashboard_view.rs`, if there's an agent listing sub-tab, add columns for
model and tier from route_metrics.
