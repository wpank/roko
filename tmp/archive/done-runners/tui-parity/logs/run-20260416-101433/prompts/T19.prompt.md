## Task

Add integration tests that verify T9's real-dispatch implementation actually dispatches to the LLM backend (not echo).

## Tests to add

### 1. `message_with_mock_dispatcher_returns_real_content`

Build a mock `ToolDispatcher` that returns a fixed `ChatResponse { content: "Hello, test", ... }`. Start agent-server with this dispatcher. POST `/message` with prompt `"ping"`. Assert response JSON `.response == "Hello, test"` (not `"<agent_id>: ping"`).

### 2. `message_without_dispatcher_returns_503`

Start agent-server WITHOUT a dispatcher. POST `/message`. Assert response is 503 with JSON `.error` containing "no configured dispatcher".

### 3. `message_dispatch_error_returns_502`

Build a mock dispatcher that always fails with `Err(...)`. Start agent-server. POST `/message`. Assert response is 502 with JSON `.error` containing "dispatch failed".

### 4. `stream_with_mock_dispatcher_streams_chunks`

Mock streaming dispatcher that emits three chunks `["Hello", ", ", "world"]` then `done`. Connect to `/stream` via tungstenite. Send text `"hi"`. Assert received three WS frames with content chunks, followed by a frame with `{"done": true}`.

### 5. `message_preserves_context`

POST `/message` with `context: {"thread": "xyz"}`. Assert response JSON `.context.thread == "xyz"` is preserved.

## Mock dispatcher scaffold

If `roko-agent::dispatcher::ToolDispatcher` is not easily mockable, build a small test-only wrapper:

```rust
struct MockDispatcher {
    response: ChatResponse,
    should_fail: bool,
}

#[async_trait]
impl DispatchLike for MockDispatcher {
    async fn dispatch(&self, _req: ChatRequest) -> Result<ChatResponse, DispatchError> {
        if self.should_fail { return Err(DispatchError::NotConfigured); }
        Ok(self.response.clone())
    }
}
```

Use a trait-based plumbing in T9 so T19 can swap in mocks without pulling real LLM clients.
