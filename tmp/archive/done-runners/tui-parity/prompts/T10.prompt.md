## Task

Populate four currently-dead fields on `TuiState` from the `DashboardData` snapshot inside `update_from_snapshot()` so the corresponding widgets show real data:

1. **gate_results** — copy `DashboardData.gate_results` into `TuiState.gate_results`. Adapter may be needed if field names differ.
2. **token_history + token_rate** — derive per-agent rolling token count from episodes or efficiency stream.
3. **orchestrator_state / iteration / phase** — populate from executor snapshot (header strip reads these).
4. **partial plan progress** — replace the `if completed { total } else { 0 }` jump with `tasks_done` from actual task statuses so the progress bar shows intermediate values.

## Implementation

### 1. gate_results bridge

Find `DashboardData.gate_results` in `dashboard.rs`. Find `TuiState.gate_results` in `state.rs`. Map each entry 1:1 inside `update_from_snapshot()`. If struct shapes differ (`summary` vs `output`), add an `impl From<DashboardGateResult> for GateResultEntry`.

### 2. token_history / token_rate

Inside `update_from_snapshot()`, iterate over `snap.episodes` (or efficiency events if available) and push token counts into `self.token_history[agent_id]`. Keep the deque bounded at 120 samples. Compute `token_rate` as `(latest - earliest) / elapsed_seconds` for the most-recent 60-sample window. Update `token_rate` for the currently selected agent.

### 3. Orchestrator state / iteration / phase

Find where executor snapshot state is exposed in `dashboard.rs` (likely `DashboardData.executor` or similar). Copy into `TuiState.orchestrator_state`, `TuiState.iteration`, `TuiState.phase`. If the snapshot doesn't carry these, extend `DashboardData` to include them (source: `.roko/state/executor.json`).

### 4. Partial plan progress

In `state.rs` around line 739 (search for `tasks_done`), replace binary logic:

```rust
let tasks_done = if completed { p.task_count } else { 0 };
```

with:

```rust
let tasks_done = p.tasks.iter().filter(|t| t.is_done()).count();
```

(If `PlanEntry.tasks` is empty at this point, that's fixed by T11 — for T10 just prepare the structure.)

1. Never reset navigation fields (`selected_plan`, `selected_agent`, scroll offsets, `active_tab`) — only update data fields.
2. If a data source is missing, fall back to sensible defaults (empty Vec, 0) — do not panic or unwrap.
3. Token history deque must be bounded (120 samples max) to avoid unbounded memory.
4. Keep changes minimal: this batch only wires existing fields. No new struct definitions except adapters.
