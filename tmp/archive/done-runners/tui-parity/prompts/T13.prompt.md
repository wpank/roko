## Task

Three modal problems to fix in one batch:

1. **Modals open empty** â€” WaveOverview, QueueOverview, TaskPicker pass `Vec::new()` instead of real data from `TuiState`.
2. **PlanDetail modal has no render path** â€” `show_plan_detail` flag exists, but no `ModalState::PlanDetail` variant and no rendering when Enter pressed on a plan.
3. **Key intercepts leak** â€” modal-open keys (help, plan_detail, wave_overview, agent_pool) don't intercept Tab, arrows, j/k, etc. â€” they leak to the background tab.

## Implementation

### 1. Pass real data into modal constructors

Search for modal construction sites. For each:

- `WaveOverview::new(waves: Vec::new())` â†’ `WaveOverview::new(tui_state.execution_waves.clone())`
- `QueueOverview::new(milestones: Vec::new())` â†’ use `tui_state.milestones` (or compute from plans)
- `TaskPicker::new(tasks: Vec::new())` â†’ `TaskPicker::new(tui_state.available_tasks.clone())`

### 2. Add `ModalState::PlanDetail` variant

In `state.rs`:

```rust
pub enum ModalState {
    // ...existing
    PlanDetail { plan_id: String },
}
```

Create `modals/plan_detail.rs` if missing with a `render()` function that takes `&PlanEntry` and draws title, status, tasks, timing, failure summary.

### 3. Wire Enter-on-plan â†’ open PlanDetail

In `input.rs` key handler for Enter when focus is on plan tree:

```rust
if let Some(plan) = selected_plan_id {
    state.active_modal = Some(ModalState::PlanDetail { plan_id: plan });
}
```

### 4. Fix modal key intercepts

In `input.rs:292-302` (the intercept section), extend the match to include:

- `show_wave_overview`
- `show_help`
- `show_plan_detail` (or `ModalState::PlanDetail` once unified in T14)
- `show_agent_pool_modal`

Intercept pattern: when any modal is open, only Escape/q/Ctrl-C + modal-internal keys (j/k scroll, arrows) reach the handler â€” everything else is swallowed.

## Rules

1. Modal data must be passed by clone (modals hold their own snapshot, don't borrow from TuiState).
2. PlanDetail modal must handle missing plan gracefully (render "Plan not found" instead of panicking).
3. Ctrl-C must still force-quit even when a modal is open (handled before modal intercept).
4. Depends on T10 (for populated state to pass into modals).
