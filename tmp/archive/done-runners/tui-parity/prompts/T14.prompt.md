## Task

Retire the legacy `show_*` booleans on `TuiState` in favor of the canonical `active_modal: Option<ModalState>` enum. Today both systems coexist and can desync (e.g. `show_plan_detail = true` while `active_modal = None`).

## Implementation

### 1. Inventory boolean flags

Grep for every `show_*: bool` field on `TuiState`:

```bash
grep -n 'show_.*: bool' crates/roko-cli/src/tui/state.rs
```

Expected list: `show_plan_detail`, `show_wave_overview`, `show_help`, `show_agent_pool_modal`, `show_queue_overview`, `show_task_picker`, etc.

### 2. Ensure `ModalState` covers all cases

Each boolean should have a corresponding `ModalState` variant. After T13, `PlanDetail` exists. Verify all others have variants (`Help`, `WaveOverview`, `AgentPool`, `QueueOverview`, `TaskPicker`, `Confirm`, etc.).

### 3. Replace writes

For each `state.show_X = true` site → `state.active_modal = Some(ModalState::X)`.
For each `state.show_X = false` site → `state.active_modal = None`.

Use grep to find all writes:

```bash
grep -rn 'show_plan_detail\|show_wave_overview\|show_help\|show_agent_pool_modal\|show_queue_overview\|show_task_picker' crates/roko-cli/src/tui/
```

### 4. Replace reads

For each `if state.show_X` site → `matches!(state.active_modal, Some(ModalState::X))` or `if let Some(ModalState::X) = &state.active_modal`.

### 5. Delete the booleans

Once no callers remain, remove the fields from `TuiState`.

1. Depends on T13 — do not start until T13 has added `PlanDetail` variant.

4. Run `cargo clippy -D warnings` — catches any field left unused.
