## Implementation process

For each field:

1. Grep for usages: `grep -rn 'field_name' crates/roko-cli/src/tui/ --include='*.rs'`
2. Identify which is canonical (used by active render/input code).
3. Delete the non-canonical one + update any write sites.
4. Run `cargo check -p roko-cli` after each removal.

For each duplicate type:

1. Find both definitions.
2. Verify the state.rs version has all fields from the widgets version (extend if not).
3. Replace widgets version with `pub use` re-export.
4. Run `cargo check`.
