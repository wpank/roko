### learning.rs test refactor

Find all `.expect(` in the test module:

```bash
grep -n '\.expect(' crates/roko-serve/src/routes/learning.rs
```

For each, replace:

```rust
fs::write(&path, data).expect("write failed");
```

with:

```rust
fs::write(&path, data).map_err(|e| format!("write failed: {e}")).unwrap();
```

or better, use `?` with a `Result<(), Box<dyn Error>>` return type on the test function (requires `#[tokio::test(flavor = "multi_thread")]` form that supports `?`).

## Rules

1. Tests must run via `cargo test -p roko-serve --lib` and `cargo test -p roko-mcp-code --lib`.
2. Use `tower::ServiceExt::oneshot` for HTTP-layer tests — do not spin up a real network server.
3. Mock external dependencies (worker callbacks, file system) with tempdirs where possible.
4. Every test must have a failure message that explains what broke.
5. No dependency on actual `.roko/` directory state — each test uses its own tempdir.
