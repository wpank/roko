## Task

Add a "Procs" sub-tab to the dashboard showing per-agent process metrics
(PID, CPU%, MEM, state, uptime) with sparkline visualizations.

## Implementation

### 1. Add ProcessMetrics struct (`state.rs`)

```rust
pub struct ProcessMetrics {
    pub pid: u32,
    pub role: String,
    pub cpu_pct: f32,
    pub mem_bytes: u64,
    pub state: String,       // "running", "sleeping", "stopped"
    pub uptime_secs: f64,
    pub cpu_history: VecDeque<f32>,   // last 60 samples for sparkline
    pub mem_history: VecDeque<u64>,
}
```

Add `pub process_metrics: Vec<ProcessMetrics>` to TuiState.

### 2. Expose active_pids from ProcessSupervisor (`roko-runtime`)

In `crates/roko-runtime/src/process.rs`, add:
```rust
impl ProcessSupervisor {
    pub fn active_pids(&self) -> Vec<(u32, String)> {
        // Returns (pid, role) for all tracked processes
    }
}
```

### 3. Sys metrics collection thread

In App, extend the existing sys_rx thread (or add a new one) to collect
per-PID metrics via the `sysinfo` crate:

```rust
use sysinfo::{System, Pid, ProcessesToUpdate};

// Every 2 seconds:
let mut sys = System::new();
for (pid, role) in active_pids {
    sys.refresh_processes(ProcessesToUpdate::Some(&[Pid::from(pid as usize)]));
    if let Some(proc) = sys.process(Pid::from(pid as usize)) {
        // Collect cpu_usage(), memory(), status(), run_time()
    }
}
```

### 4. Render Procs sub-tab (`dashboard_view.rs`)

Add a "Procs" sub-tab (cycle with `p` key or as a panel in the dashboard).
Render as a table:

```
┌─────────────────────────────────────────────────────────────┐
│ PID     Role          CPU%    MEM       State     Uptime    │
│ 12345   implementer   45.2%   128 MB    running   5m 23s   │
│ 12346   conductor     12.1%    64 MB    running   5m 20s   │
│ 12347   auto-fixer     0.0%    32 MB    sleeping  2m 10s   │
└─────────────────────────────────────────────────────────────┘
```

### 5. Per-agent sparklines

Use the braille module to render small CPU/MEM sparklines inline or
in an expanded row below each process entry.

## Dependencies

- **T1**: Uses connected mode for live process data
- **T3**: Process data can include approval-blocked state

## Rules

1. Check if `sysinfo` is already a dependency. If not, add it to roko-cli's Cargo.toml.
2. The Procs sub-tab should be accessible via a key binding (e.g., `p` in dashboard view).
3. Sparkline history should be bounded (e.g., last 60 samples = 2 minutes at 2s interval).
4. If ProcessSupervisor is not available (standalone mode), show "No process data" message.
