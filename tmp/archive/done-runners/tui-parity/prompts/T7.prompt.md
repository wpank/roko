Read these files first:

- `tmp/tui-parity/context-pack/00-TUI-PARITY-RULES.md`

- `crates/roko-cli/src/tui/widgets/` (all files)

## Task

Remove dead fields, structs, and code from the TUI module. This is a cleanup
batch — net code deletion, no new features.

## Fields and structs to investigate and remove

### 1. `agents_by_id: HashMap<String, AgentState>` + `AgentState` struct

Search for all usages of `agents_by_id` and `AgentState` in the TUI module.
If they are not read by any view/widget/modal, remove them.

### 2. `token_burn_history: HashMap<String, VecDeque<u64>>` + `TokenBurnEntry` struct

Search for `token_burn_history` and `TokenBurnEntry`. If the token sparkline
widget uses a different data source, remove these.

### 3. `plan_detail_content: String` and `plan_summary_content: String`

These may have been replaced by computed values from PlanEntry. Search for
reads — if no view renders them, remove.

### 4. `parallel_run` field

Check if `parallel_run` is still used. It may have been replaced by
`pipeline_run_state` or `execution_waves`.

### 5. Legacy aliases: `task_total`, `task_done`

Search for these. If they're computed from `task_counts()` method instead,
remove the stored fields.

### 6. Deduplicate git subprocess calls

Check if multiple places spawn `git` subprocesses independently. If so,
consolidate into the single `git_rx` background thread pattern.

## Process

For each candidate:

1. **Search**: `grep -rn 'field_name' crates/roko-cli/src/tui/ --include='*.rs'`
2. **Classify**: Is it written but never read? Written and read but with stale data?
3. **Remove**: Delete the field, struct, and all write sites. Fix compilation errors.
4. **Verify**: `cargo check -p roko-cli`

## Write scope

- `crates/roko-cli/src/tui/state.rs`
- `crates/roko-cli/src/tui/app.rs`
- `crates/roko-cli/src/tui/views/*`
- `crates/roko-cli/src/tui/widgets/*`

## Rules

1. Only remove code that is truly dead (not read by any active code path).
2. Do NOT remove fields that are read by views/widgets, even if the data
   looks stale — those need to be fixed, not deleted.
3. Run clippy after removals to catch any new dead_code warnings.
4. This batch has NO dependencies — it can run independently.

## Done when

- `cargo check -p roko-cli` succeeds
- `cargo clippy -p roko-cli --no-deps -- -D warnings` passes
- Net LOC change is negative (more deleted than added)
- No dead_code warnings in TUI module
