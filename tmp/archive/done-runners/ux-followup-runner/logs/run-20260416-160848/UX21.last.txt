Implemented UX21.

`roko-agent-server` now exposes `GET /logs?tail=N` via [crates/roko-agent-server/src/features/logs.rs](/Users/will/dev/nunchi/roko/roko/.roko/worktrees/ux-followup-run-20260416-160848/crates/roko-agent-server/src/features/logs.rs). `tail` defaults to `200` and is capped at `2000`. Missing log files return `204 No Content`; existing logs are scrubbed with `LogScrubber` and tailed with bounded memory instead of loading the whole file. The sidecar log path lives on `AgentState` and defaults to `.roko/agents/<agent_id>/log`, with a builder/state override added in [crates/roko-agent-server/src/lib.rs](/Users/will/dev/nunchi/roko/roko/.roko/worktrees/ux-followup-run-20260416-160848/crates/roko-agent-server/src/lib.rs) and [crates/roko-agent-server/src/state.rs](/Users/will/dev/nunchi/roko/roko/.roko/worktrees/ux-followup-run-20260416-160848/crates/roko-agent-server/src/state.rs). I also added a small adjacent hook in [crates/roko-agent-server/src/features/messaging.rs](/Users/will/dev/nunchi/roko/roko/.roko/worktrees/ux-followup-run-20260416-160848/crates/roko-agent-server/src/features/messaging.rs) so a successful `POST /message` appends a scrubbed sidecar log line; that gives the required testable path from message to log tail.

`roko-serve` now proxies that surface at `GET /api/agents/{id}/logs?tail=N` in [crates/roko-serve/src/routes/agents.rs](/Users/will/dev/nunchi/roko/roko/.roko/worktrees/ux-followup-run-20260416-160848/crates/roko-serve/src/routes/agents.rs). The proxy forwards the query, auth token, upstream status, body, and content type. I did not add the optional CLI wrapper.

Verified with `cargo check -p roko-agent-server -p roko-serve`, `cargo test -p roko-agent-server --lib`, `cargo test -p roko-serve agent_logs_ -- --nocapture`, and `cargo clippy -p roko-agent-server -p roko-serve --no-deps -- -D warnings`.
