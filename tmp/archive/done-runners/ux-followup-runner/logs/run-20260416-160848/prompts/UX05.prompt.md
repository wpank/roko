## Task

The TUI's polling fallback at `crates/roko-cli/src/tui/app.rs:357-361` is a
bug. The user has confirmed it is **not** an intended fallback; standalone
mode should subscribe to an in-process `StateHub` unconditionally.

Today:
```rust
if app.snapshot_rx.is_none() && app.last_refresh.elapsed() > Duration::from_secs(1) {
    app.data.refresh().await?;
    app.tui_state.update_from_snapshot(&app.data);
    app.last_refresh = Instant::now();
}
```
And `snapshot_rx` is `None` in the default ctor (line 412). So standalone
`roko dashboard` hits this branch every tick.

## Implementation

### 1. Spawn an in-process StateHub when none is supplied

In `crates/roko-cli/src/tui/app.rs`, factor the ctor so
`new()` / `new_with_page()` also start a local `SharedStateHub`:

```rust
impl App {
    #[must_use]
    pub fn new(root: impl AsRef<Path>) -> Self {
        let state_hub = roko_core::SharedStateHub::new_in_process();
        Self::new_connected_with_state_hub(root, None, state_hub)
    }

    #[must_use]
    pub fn new_with_page(root: impl AsRef<Path>, initial_page: Option<PageId>) -> Self {
        let state_hub = roko_core::SharedStateHub::new_in_process();
        Self::new_connected_with_state_hub(root, initial_page, state_hub)
    }
}
```

If `roko-core` does not yet have `SharedStateHub::new_in_process`, add it.
It should:
- construct a `StateHub`
- wrap in `Arc`
- return a handle that can be cloned + whose `Receiver` is always
  up-to-date.

### 2. Populate the hub from `.roko/` on construction

For standalone mode, the hub is empty until something publishes to it.
Bootstrap by walking `.roko/` once at construction:

```rust
impl SharedStateHub {
    pub fn new_in_process() -> Self {
        let hub = StateHub::new();
        Self(Arc::new(hub))
    }

    pub fn bootstrap_from_workdir(&self, workdir: &Path) -> Result<()> {
        let snapshot = DashboardSnapshot::load_from_workdir(workdir)?;
        self.0.apply_snapshot(snapshot);
        Ok(())
    }
}
```

Call `bootstrap_from_workdir` immediately after construction in the app
ctor. Subsequent updates come from UX06 (file watcher) and UX07/UX08
(incremental tailers).

### 3. Delete the polling fallback branch

Remove the `if app.snapshot_rx.is_none()` branch at `app.rs:357-361`. After
this batch, the TUI's only refresh path is the `snapshot_rx` one that
already exists for connected mode.

### 4. Keep `snapshot_rx` non-optional (if possible)

Since the TUI now always has a hub, `snapshot_rx: Option<watch::Receiver<_>>`
can collapse to `snapshot_rx: watch::Receiver<_>`. Simpler. Optional — if it
cascades too far, keep `Option` and just ensure it is `Some` in every ctor.

### 5. Unit test: the polling branch is unreachable

Add a unit test that constructs an `App::new(tempdir)` and asserts
`app.snapshot_rx.is_some()` (or, after the collapse, simply that the field
exists).
