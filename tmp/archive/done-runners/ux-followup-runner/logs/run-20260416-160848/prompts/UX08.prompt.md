## Task

`.roko/task-outputs/` holds one file per in-flight task tail. Today the TUI
walks the directory on every refresh + re-reads every file. Switch to a
per-file map of cursors so only modified files get re-tailed.

## Implementation

### 1. `TaskOutputCursors` — per-file tail manager

`crates/roko-cli/src/tui/task_outputs.rs` (new; may already exist as a
leaf module — extend or replace `load_task_outputs` call sites):

```rust
pub struct TaskOutputCursors {
    base_dir: PathBuf,
    cursors: HashMap<String, TaskOutputCursor>,  // key = task_id (filename stem)
}

struct TaskOutputCursor {
    path: PathBuf,
    tail: RingBuffer<String>,   // bounded to N lines, e.g. 400
    cursor: JsonlCursor,        // if task output is line-oriented
}

impl TaskOutputCursors {
    pub fn new(base_dir: impl Into<PathBuf>) -> Self {
        Self { base_dir: base_dir.into(), cursors: HashMap::new() }
    }

    /// Walk the dir once; add new tasks, drop stale ones.
    pub fn reconcile(&mut self) -> std::io::Result<()> {
        let mut seen: HashSet<String> = HashSet::new();
        if self.base_dir.exists() {
            for entry in std::fs::read_dir(&self.base_dir)? {
                let e = entry?;
                let Some(name) = e.file_name().to_str().map(|s| s.to_string()) else { continue };
                seen.insert(name.clone());
                self.cursors.entry(name.clone())
                    .or_insert_with(|| TaskOutputCursor {
                        path: e.path(),
                        tail: RingBuffer::new(400),
                        cursor: JsonlCursor::new(e.path()),
                    });
            }
        }
        self.cursors.retain(|k, _| seen.contains(k));
        Ok(())
    }

    /// Incrementally tail each known file.
    pub fn tick(&mut self) -> std::io::Result<()> {
        for cursor in self.cursors.values_mut() {
            for line in cursor.cursor.read_new_lines()? {
                cursor.tail.push(line);
            }
        }
        Ok(())
    }

    pub fn tail_for(&self, task_id: &str) -> Option<&[String]> {
        self.cursors.get(task_id).map(|c| c.tail.as_slice())
    }
}
```

### 2. Plug into notify events

Two event kinds matter:
- **directory changed**: call `reconcile` (a task entered / left the dir)
- **file changed**: call `tick` (incremental tail)

Both are covered by UX06's `FsRefresh::Coalesced` event fired on any change
inside `.roko/`. Call both on every refresh:

```rust
self.task_outputs.reconcile()?;
self.task_outputs.tick()?;
```

This is cheap — `reconcile` just compares filename sets.

### 3. `backfill_agent_output_tail` adjustment

`dashboard.rs` has a `backfill_agent_output_tail` function that attaches
tails from `task_outputs` to episodes. Keep the helper; it now reads from
`TaskOutputCursors::tail_for` instead of the `HashMap<PathBuf, Vec<String>>`
the old loader produced.

### 4. Bound each tail

Ring buffer at 400 lines per task (configurable via constant). This caps
memory regardless of task runtime.

## Rules

1. **No unbounded memory.** Each ring buffer caps at 400 lines.
2. **Stale files reaped.** If a task finishes + its file gets deleted, the
   cursor is dropped within one `reconcile` tick.
3. **Hot-path cheap.** `reconcile` is O(dir contents); `tick` reads only
   new bytes.
4. **No disk-read inside render.** Views read from in-memory ring buffers.
5. **Preserve backfill behaviour.** Episodes that don't bring their own
   `agent_output_tail` still get populated via `backfill_agent_output_tail`.
