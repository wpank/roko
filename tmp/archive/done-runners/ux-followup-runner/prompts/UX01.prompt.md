## Task

Close the learn → replan loop: when a task hits the gate failure limit, emit
a `PlanRevision` event, capture the failing gate verdict + tail of logs, and
re-invoke the plan generator with that failure context as additional prompt
input. Dedupe by a task-hash so the same failure does not trigger infinite
re-plans.

## Current state (evidence)

- `run_gate_rung` at `orchestrate.rs:11423-11461` dispatches
  Compile / Test / Clippy gates and returns `Vec<Verdict>`.
- Verdicts are persisted via `FileSubstrate::put` at `orchestrate.rs:11170-11188`
  as `Kind::GateVerdict` signals — no consumer today.
- When a task fails N attempts, the executor marks it `failed` and continues.
- `roko-learn::episode_logger::Episode` already records `gate_verdicts` +
  `failure_reason` per turn.
- No `PlanRevision` / `RokoEvent` variant exists in `roko-runtime::event_bus`.

### 3. Subscribe + re-plan in a new `roko-orchestrator::replanning` module

Create `crates/roko-orchestrator/src/replanning.rs` (or inside orchestrate.rs
if that's where the DAG executor lives — pick the right home):

```rust
pub struct ReplanSubscriber {
    event_rx: broadcast::Receiver<RokoEvent>,
    plan_gen: Arc<PlanGenerator>,
    dedupe: DashMap<String, u32>,   // key = (plan_id, task_id) + failure hash
    max_replans_per_plan: u32,      // default 2
    replans_seen: DashMap<String, u32>,
}

impl ReplanSubscriber {
    pub async fn run(mut self) {
        while let Ok(evt) = self.event_rx.recv().await {
            if let RokoEvent::PlanRevision { plan_id, task_id, reason, failing_verdicts, log_tail, .. } = evt {
                let hash = failure_hash(&plan_id, &task_id, &failing_verdicts);
                if *self.dedupe.entry(hash.clone()).or_insert(0) >= 1 {
                    tracing::info!(plan_id, task_id, "duplicate failure; skipping replan");
                    continue;
                }
                self.dedupe.insert(hash, 1);

                let total = *self.replans_seen.entry(plan_id.clone()).or_insert(0);
                if total >= self.max_replans_per_plan {
                    tracing::warn!(plan_id, "max replans reached; giving up");
                    continue;
                }
                self.replans_seen.entry(plan_id.clone()).and_modify(|c| *c += 1);

                let augmentation = format_failure_prompt(&task_id, &reason, &failing_verdicts, &log_tail);
                match self.plan_gen.regenerate(&plan_id, augmentation).await {
                    Ok(path) => tracing::info!(plan_id, path = %path.display(), "replan succeeded"),
                    Err(err) => tracing::error!(plan_id, %err, "replan failed"),
                }
            }
        }
    }
}
```

## Rules

1. **Dedupe first.** Same `(plan_id, task_id, failure_hash)` must not trigger
   more than one replan. Persist the dedupe set to `.roko/learn/replans.json`
   so restarts do not re-trigger.
2. **Cap per plan.** `max_replans_per_plan = 2` (configurable). After that,
   the task stays `failed` and the executor moves on.
3. **Never lose the failure.** The original failure episode + gate verdicts
   stay in `.roko/episodes.jsonl`. The replan event is additional, not a
   replacement.
4. **No infinite loops.** If the replan itself fails the gate, the per-plan
   cap plus the dedupe prevent runaway retries.
5. **Small commits.** If the work exceeds the batch budget, land the event
   variant + emitter first, put the subscriber behind a `replan_on_gate_failure`
   toml flag that defaults false, and leave a follow-up.
