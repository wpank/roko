## Task

Replace the 500 ms `std::thread::sleep` loop at
`crates/roko-cli/src/tui/app.rs:529-548` with a `notify` crate watcher over
`.roko/`. Debounce bursts 200 ms; emit one refresh signal per debounce window.

## Current state

```rust
// app.rs:526-548
let (data_tx, data_rx) = std::sync::mpsc::channel::<DashboardData>();
std::thread::Builder::new()
    .name("tui-data-refresh".into())
    .spawn(move || {

            let data = DashboardData::load_best_effort(&data_workdir);
```

This:
- burns CPU + IO every 500 ms regardless of change
- has unbounded channel size if the UI stalls (see UX11)
- cannot distinguish "nothing changed" from "changed"

Use `notify-debouncer-full` for a pre-built debouncer.

### 2. New `fs_watch.rs` module

`crates/roko-cli/src/tui/fs_watch.rs`:

```rust
use notify::RecursiveMode;
use notify_debouncer_full::{new_debouncer, DebounceEventResult};
use std::sync::mpsc;
use std::time::Duration;

pub struct FsWatchHandle {
    _debouncer: notify_debouncer_full::Debouncer<notify::RecommendedWatcher, notify_debouncer_full::FileIdMap>,
    pub rx: mpsc::Receiver<FsRefresh>,
}

#[derive(Debug, Clone)]
pub enum FsRefresh {
    Coalesced,  // one or more events in the debounce window
}

pub fn watch_roko_dir(workdir: &Path) -> anyhow::Result<FsWatchHandle> {
    let roko_dir = workdir.join(".roko");
    if !roko_dir.exists() {
        std::fs::create_dir_all(&roko_dir)?;
    }
    let (tx, rx) = mpsc::channel();
    let mut deb = new_debouncer(Duration::from_millis(200), None, move |res: DebounceEventResult| {
        if res.is_ok() {
            let _ = tx.send(FsRefresh::Coalesced);
        }
    })?;
    deb.watcher().watch(&roko_dir, RecursiveMode::Recursive)?;
    Ok(FsWatchHandle { _debouncer: deb, rx })
}
```

### 3. Fallback poller

Some filesystems (network mounts, bind mounts on macOS) don't emit events.
Detect at startup and fall back to a 1 s poll explicitly logged:

```rust
pub fn watch_roko_dir_with_fallback(workdir: &Path) -> FsWatchHandle {
    match watch_roko_dir(workdir) {
        Ok(h) => h,
        Err(err) => {
            tracing::warn!(%err, "notify unavailable; falling back to 1 s poll");
            spawn_poll_fallback(workdir, Duration::from_secs(1))
        }
    }
}
```

## Rules

1. **Fallback required.** If `notify` fails to initialise (CI sandboxes,
   exotic filesystems), fall back to a 1 s poll *with a startup `tracing::warn!`*
   so the operator knows.
2. **Debounce 200 ms.** Bursts (e.g. many files written in one transaction)
   → one refresh.
3. **Thread lifecycle.** The `Debouncer` must outlive the App; store it on
   `App` (not dropped at ctor return).
4. **Bound the channel.** Use `std::sync::mpsc` here is OK because we only
   send `FsRefresh::Coalesced` after debouncing, so max outstanding = 1 per
   UI frame. If drift possible, use `sync_channel(4)`.
5. **No regression in tests.** CI runners on Linux must still succeed; the
   `notify` crate supports Linux inotify + macOS FSEvents + Windows
   ReadDirectoryChangesW.
