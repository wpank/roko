## Task

The TUI's Agents tab today derives agent status from the periodically-parsed
`executor.json` (stale up to ~500 ms). `roko-agent-server` already exposes a
`/stream` WebSocket (T9 landed in PR #13). Subscribe to it for each visible
agent, push live chunks into a new `TuiState::agent_streams` ring buffer,
and render the live tail in the Agents tab detail panel.

## Implementation

### 1. New WS client module

`crates/roko-cli/src/tui/ws_client.rs`:

```rust
use tokio_tungstenite::{connect_async, tungstenite::Message};

pub struct AgentStreamClient {
    agent_id: String,
    url: String,
    rx: tokio::sync::mpsc::Receiver<StreamChunk>,
    _handle: tokio::task::JoinHandle<()>,
}

pub enum StreamChunk {
    Text(String),
    Done { session: Option<String> },
    Disconnected,
}

impl AgentStreamClient {
    pub fn connect(agent_id: impl Into<String>, base_url: &str) -> Self {
        let agent_id = agent_id.into();
        let url = format!("{}/agents/{}/stream", base_url.trim_end_matches('/'), agent_id);
        let (tx, rx) = tokio::sync::mpsc::channel(128);
        let handle = tokio::spawn(stream_loop(url.clone(), tx));
        Self { agent_id, url, rx, _handle: handle }
    }

    pub fn try_recv(&mut self) -> Option<StreamChunk> {
        self.rx.try_recv().ok()
    }
}

async fn stream_loop(url: String, tx: tokio::sync::mpsc::Sender<StreamChunk>) {
    let mut backoff = Duration::from_secs(1);
    loop {
        match connect_async(&url).await {
            Ok((mut ws, _)) => {
                backoff = Duration::from_secs(1);
                while let Some(msg) = ws.next().await {
                    let Ok(msg) = msg else { break };
                    match msg {
                        Message::Text(t) => {
                            if t.contains("\"done\":true") {
                                let _ = tx.send(StreamChunk::Done { session: parse_session(&t) }).await;
                            } else {
                                let _ = tx.send(StreamChunk::Text(t)).await;
                            }
                        }
                        Message::Close(_) => break,
                        _ => {}
                    }
                }
            }
            Err(err) => tracing::debug!(%err, %url, "ws connect failed; retrying"),
        }
        let _ = tx.send(StreamChunk::Disconnected).await;
        tokio::time::sleep(backoff).await;
        backoff = (backoff * 2).min(Duration::from_secs(30));
    }
}
```

### 2. Extend `TuiState`

```rust
// crates/roko-cli/src/tui/state.rs
pub struct TuiState {
    // existing fields…
    pub agent_streams: HashMap<String, AgentStream>,
}

pub struct AgentStream {
    pub chunks: RingBuffer<String>,  // cap 200
    pub connected: bool,
    pub last_chunk_at: Option<Instant>,
}

impl TuiState {
    pub fn push_agent_chunk(&mut self, agent_id: &str, chunk: String) {
        let entry = self.agent_streams.entry(agent_id.to_string()).or_insert_with(|| AgentStream {
            chunks: RingBuffer::new(200),
            connected: true,
            last_chunk_at: None,
        });
        entry.chunks.push(chunk);
        entry.last_chunk_at = Some(Instant::now());
    }
}
```

### 3. Wire into Agents view

On tab activation (Agents tab = F3 in current layout — verify in `app.rs`):
- For each visible agent, call `AgentStreamClient::connect(agent.id, base_url)`.
- Store clients in `app.agent_stream_clients: HashMap<String, AgentStreamClient>`.
- On tab-exit, drop the clients (which cancels the background tasks via the
  handle's Drop).

In the main event loop, poll each client:

```rust
for (id, client) in app.agent_stream_clients.iter_mut() {
    while let Some(chunk) = client.try_recv() {
        match chunk {
            StreamChunk::Text(t) => app.tui_state.push_agent_chunk(id, t),
            StreamChunk::Disconnected => app.tui_state.mark_disconnected(id),
            StreamChunk::Done { .. } => app.tui_state.mark_done(id),
        }
    }
}
```

## Rules

1. **Bounded buffers.** 200 chunks / agent; older chunks drop.
2. **No leaks on tab switch.** Drop clients when leaving the Agents tab.
3. **Reconnect with back-off.** 1 s initial, doubles, caps at 30 s.
4. **No blocking.** WS I/O on `tokio::spawn`; UI thread only calls
   `try_recv()`.
5. **Graceful degradation.** If the sidecar is unreachable, the panel shows
   "connecting…" not an error spam.
