## Task

A second background thread shells out to `git` every 3 s. Replace with a
`notify` watcher over `.git/HEAD` (+ `.git/refs/heads/*` for branch
updates). Only re-run `collect_git_data` when the watcher fires.

## Implementation

### 1. New git watcher

`crates/roko-cli/src/tui/git_watch.rs`:

```rust
use notify::RecursiveMode;
use notify_debouncer_full::new_debouncer;
use std::sync::mpsc;
use std::time::Duration;

pub struct GitWatchHandle {
    _debouncer: notify_debouncer_full::Debouncer<notify::RecommendedWatcher, notify_debouncer_full::FileIdMap>,
    pub rx: mpsc::Receiver<()>,
}

pub fn watch_git(workdir: &Path) -> anyhow::Result<GitWatchHandle> {
    let git_dir = workdir.join(".git");
    if !git_dir.is_dir() {
        anyhow::bail!("not a git repo");
    }
    let (tx, rx) = mpsc::channel();
    let mut deb = new_debouncer(Duration::from_millis(500), None, move |res| {
        if res.is_ok() { let _ = tx.send(()); }
    })?;
    // Watch the head + refs; this covers commits, checkouts, fetches.
    for p in [git_dir.join("HEAD"), git_dir.join("refs/heads"), git_dir.join("packed-refs")] {
        if p.exists() {
            deb.watcher().watch(&p, RecursiveMode::Recursive).ok();
        }
    }
    Ok(GitWatchHandle { _debouncer: deb, rx })
}
```

### 2. Wire into `App`

Replace the 3 s polling thread:

```rust
let git_watch = watch_git(&self.workdir).ok();
self.git_rx = git_watch.as_ref().map(|h| h.rx);
self._git_watcher = git_watch;   // keep alive
```

In the main event loop, on watcher fire:

```rust
if let Some(rx) = &self.git_rx {
    if rx.try_recv().is_ok() {
        let bg = collect_git_bg_data();
        // push to StateHub or directly to TuiState
    }
}
```

### 3. Initial read at startup

Run `collect_git_bg_data` once immediately so the Git tab is populated before
the first watcher event.

### 4. Fallback for non-git workdirs

If `.git/` is missing (e.g. a detached installation), log a single
`tracing::info!` and mark the Git tab as "not a git repo". No panic.

### 5. Delete the 3 s polling thread

Remove the `tui-git-refresh` `std::thread::spawn` block at `app.rs:554-596`.

## Rules

1. **No new deps.** `notify` + `notify-debouncer-full` already added by UX06.
2. **No panics.** Fallback quietly if not a git repo.
3. **Still updates on commit / checkout / fetch.** Verify by
   `git commit --allow-empty` and watching the Git tab update in < 1 s.
4. **Debounce 500 ms.** Git operations often touch many refs at once.
5. **Keep the existing `GitBgData` shape.** Downstream widgets unchanged.
