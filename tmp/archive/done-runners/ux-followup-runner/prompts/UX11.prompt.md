## Task

Two small hardening items:

1. **Backpressure on the TUI channels.** `std::sync::mpsc::channel` is
   unbounded; if the UI thread stalls, the producer grows the queue
   indefinitely. Swap to `tokio::sync::watch` (single-slot) for data that
   should be "latest wins", or `std::sync::mpsc::sync_channel(4)` for
   event streams.

2. **Durable `DashboardGenerationState` across process restarts.** Today
   it lives in a process-local `OnceLock<Mutex<HashMap<PathBuf, ...>>>`
   (dashboard.rs:104-110). Persist to `.roko/state/dashboard-gen.json` so
   multi-tab or cross-process consumers get a stable monotonic counter.

## Implementation

### 1. Backpressure — choose the right primitive

- **Data refresh (latest-wins)**: `DashboardData` is fully re-derivable from
  `.roko/`; losing an intermediate refresh is fine. Use
  `tokio::sync::watch::channel(DashboardData::empty())`:

  ```rust
  let (data_tx, data_rx) = tokio::sync::watch::channel(DashboardData::empty());
  ```

- **Git data (low cadence)**: 1 event per ~second at most. Use
  `std::sync::mpsc::sync_channel(4)` so producer back-offs gently on a
  stalled UI.

  ```rust
  let (git_tx, git_rx) = std::sync::mpsc::sync_channel::<GitBgData>(4);
  ```

- **FS refresh (coalesced)**: from UX06. Already bounded (single message
  per debounce window) — add `sync_channel(4)` defensively.

### 3. Durable gen counter

`crates/roko-cli/src/tui/dashboard_gen.rs` (new):

```rust
#[derive(Debug, Clone, Default, Serialize, Deserialize)]
pub struct DashboardGenerationState {
    pub fingerprint: u64,
    pub generation: u64,
}

pub struct DurableGenCounter {
    path: PathBuf,
    state: Mutex<HashMap<PathBuf, DashboardGenerationState>>,
}

impl DurableGenCounter {
    pub fn load(workdir: &Path) -> Self {
        let path = workdir.join(".roko/state/dashboard-gen.json");
        let state = if let Ok(body) = std::fs::read_to_string(&path) {
            serde_json::from_str(&body).unwrap_or_default()
        } else {
            Default::default()
        };
        Self { path, state: Mutex::new(state) }
    }

    pub fn next(&self, root: &Path, stamps: DashboardDataStamps) -> u64 {
        let fingerprint = stamps.fingerprint();
        let mut guard = self.state.lock().unwrap();
        let entry = guard.entry(root.to_path_buf()).or_default();
        if entry.fingerprint != fingerprint {
            entry.fingerprint = fingerprint;
            entry.generation += 1;
            self.persist(&guard);
        }
        entry.generation
    }

    fn persist(&self, guard: &HashMap<PathBuf, DashboardGenerationState>) {
        if let Some(parent) = self.path.parent() {
            let _ = std::fs::create_dir_all(parent);
        }
        let tmp = self.path.with_extension("json.tmp");
        if let Ok(body) = serde_json::to_string_pretty(guard) {
            if std::fs::write(&tmp, body).is_ok() {
                let _ = std::fs::rename(&tmp, &self.path);
            }
        }
    }
}
```

### 4. Replace the `OnceLock` use site

`dashboard.rs:next_dashboard_data_generation` becomes:

```rust
fn next_dashboard_data_generation(root: &Path, stamps: DashboardDataStamps) -> u64 {
    static GEN: OnceLock<DurableGenCounter> = OnceLock::new();
    GEN.get_or_init(|| DurableGenCounter::load(root)).next(root, stamps)
}
```

(Note: if multiple workdirs are in play in one process, key by workdir.
The durable counter already handles that via `HashMap<PathBuf, …>`.)

## Rules

1. **No protocol change for downstream consumers.** The only observable
   diff is "generation no longer resets across restarts".
2. **Atomic writes.** Use `rename(tmp, path)` pattern so a crash mid-write
   never leaves a truncated JSON.
3. **Single-slot for latest-wins.** Do not use unbounded channels anywhere
   in `tui/app.rs` after this batch.
4. **Back-compat.** On first run, `.roko/state/dashboard-gen.json` is
   missing → starts at generation 1 (same as today).
