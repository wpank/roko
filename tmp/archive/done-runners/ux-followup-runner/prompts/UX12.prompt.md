## Task

Add schema versioning to the resume snapshot + a migration framework that
walks `v0 → v1 → v2 → …` on load. Without this, future schema changes brick
in-flight runs.

### 2. Migration module

`crates/roko-cli/src/snapshot_migrate.rs` (new):

```rust
pub fn upgrade(raw: serde_json::Value) -> anyhow::Result<ExecutorSnapshot> {
    let version = raw.get("schema_version").and_then(|v| v.as_u64()).unwrap_or(0) as u32;
    let migrated = match version {
        0 => migrate_v0_to_v1(raw)?,
        v if v == CURRENT_SCHEMA_VERSION => raw,
        v if v > CURRENT_SCHEMA_VERSION => {
            anyhow::bail!(
                "snapshot schema v{} is newer than this build supports (v{}); \
                upgrade roko-cli or restore an older snapshot",
                v,
                CURRENT_SCHEMA_VERSION,
            );
        }
        other => anyhow::bail!("unknown snapshot schema_version: {other}"),
    };
    Ok(serde_json::from_value(migrated)?)
}

fn migrate_v0_to_v1(mut raw: serde_json::Value) -> anyhow::Result<serde_json::Value> {
    // v0 had no schema_version; just inject the field.
    if let Some(obj) = raw.as_object_mut() {
        obj.insert("schema_version".into(), serde_json::json!(1));
    }
    Ok(raw)
}
```

## Rules

1. **Forward-only migrations.** No reverse v1 → v0 path.
2. **Fail loud on newer-than-build.** Clear diagnostic, not a panic.
3. **Do not mutate existing snapshots on disk on read.** Upgrade happens
   in memory; the next `save_snapshot_atomic` rewrites at the current version.
4. **Atomic writes preserved.** `save_snapshot_atomic` still writes via
   `tmp + rename`.
