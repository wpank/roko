## Task

`--resume <snapshot.json>` today loads the snapshot and re-runs plan
discovery against the supplied `plans/` directory. If a plan was renamed
or deleted between save and resume, the plan id from the snapshot does not
match any discovered plan — downstream code silently skips or explodes deep
inside the executor.

Add a reconciliation pass that walks the snapshot's `plan_ids` against the
discovered plan set and surfaces a clear error on mismatch.

## Implementation

### 1. Helper

In `crates/roko-cli/src/orchestrate.rs` (or a new
`snapshot_reconcile.rs`):

```rust
pub fn reconcile_snapshot_vs_plans(
    snapshot: &ExecutorSnapshot,
    discovered: &[PlanDescriptor],
) -> Result<(), SnapshotReconcileError> {
    let discovered_ids: HashSet<&str> = discovered.iter().map(|p| p.id.as_str()).collect();
    let missing: Vec<String> = snapshot
        .plan_ids()
        .filter(|id| !discovered_ids.contains(id.as_str()))
        .cloned()
        .collect();

    if !missing.is_empty() {
        return Err(SnapshotReconcileError::PlanIdsMissing {
            missing,
            discovered: discovered.iter().map(|p| p.id.clone()).collect(),
        });
    }
    Ok(())
}
```

Error type:

```rust
#[derive(Debug, thiserror::Error)]
pub enum SnapshotReconcileError {
    #[error("resume snapshot references {missing:?} but plans/ has {discovered:?}. \
             Rename or prune the snapshot before resuming.")]
    PlanIdsMissing { missing: Vec<String>, discovered: Vec<String> },
}
```

### 2. Call site

Where `plan run --resume` loads the snapshot and discovers plans, insert:

```rust
let snapshot = snapshot_migrate::upgrade(raw)?;
let discovered = discover_plans(&plans_dir)?;
reconcile_snapshot_vs_plans(&snapshot, &discovered)?;
// then hand off to executor
```

### 3. Diagnostic output

`SnapshotReconcileError` should print with enough context that an operator
can act without reading source code. Example:

```
Error: resume snapshot references plans [foo, bar] but plans/ has
[baz, qux]. Rename or prune the snapshot before resuming.
  - Snapshot path: .roko/state/executor.json
  - Plans root: plans/
  - Missing: foo, bar
```

## Rules

1. **No silent skip.** Before this batch, resume would continue with a
   partial plan set; after, it refuses to start.
2. **Additive plans OK.** A plan newly added since the snapshot is not an
   error.
3. **Missing plans are errors.** Unless the operator passes a new flag
   (`--allow-snapshot-drift`) — do **not** add that flag in this batch;
   let them fix their state first.
4. **Keep UX12's upgrade call in the same sequence.** Upgrade → reconcile
   → execute.
