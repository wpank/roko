## Task

Three related hardening moves on `ProcessSupervisor`:

1. **SIGTERM-then-SIGKILL escalation** — today `shutdown` calls
   `Child::kill()` which is SIGKILL on Unix. Add a graceful SIGTERM step
   with a grace period before the hard kill.
2. **`impl Drop for ProcessSupervisor`** — today a parent-task panic leaves
   children orphaned. A Drop impl that calls `kill_all()` synchronously
   reaps them.
3. **`CancellationToken` plumbing** — `spawn` should accept an optional
   `CancellationToken` so callers can co-operatively cancel children.

## Implementation

### 1. SIGTERM escalation (Unix only)

```rust
// crates/roko-runtime/src/process.rs
use nix::sys::signal::{kill, Signal};
use nix::unistd::Pid;

impl ManagedChild {
    pub async fn shutdown(&mut self) -> ProcessOutcome {
        #[cfg(unix)]
        {
            if let Some(pid) = self.child.id() {
                let _ = kill(Pid::from_raw(pid as i32), Signal::SIGTERM);
            }
        }
        match tokio::time::timeout(self.grace_period, self.child.wait()).await {
            Ok(Ok(status)) => self.outcome(Some(status), false),
            _ => self.force_kill().await,
        }
    }
}
```

Windows fallback (`cfg(windows)`): keep the current `Child::kill()` path;
Windows does not have SIGTERM.

### 2. Drop impl

```rust
impl Drop for ProcessSupervisor {
    fn drop(&mut self) {
        let children = std::mem::take(&mut self.children);
        if !children.is_empty() {
            tracing::warn!(count = children.len(), "ProcessSupervisor dropped with live children; force-killing");
            for (_, handle) in children.into_iter() {
                // Use a synchronous force-kill; no async available in Drop.
                #[cfg(unix)]
                {
                    if let Some(pid) = handle.child.id() {
                        let _ = nix::sys::signal::kill(nix::unistd::Pid::from_raw(pid as i32), nix::sys::signal::Signal::SIGKILL);
                    }
                }
                #[cfg(windows)]
                {
                    let _ = handle.child.start_kill();
                }
            }
        }
    }
}
```

### 3. CancellationToken plumbing

Extend `ProcessSupervisor::spawn` to accept an optional token:

```rust
pub struct SpawnOptions {
    // existing fields
    pub cancellation: Option<CancellationToken>,
}

impl ProcessSupervisor {
    pub fn spawn(&self, opts: SpawnOptions) -> Result<ProcessHandle> {
        // existing body
        // then:
        if let Some(token) = opts.cancellation.clone() {
            let handle_id = id;
            let me = self.clone();
            tokio::spawn(async move {
                token.cancelled().await;
                me.shutdown(handle_id).await;
            });
        }
        Ok(handle)
    }
}
```

Callers in `crates/roko-cli/src/orchestrate.rs` (and anywhere else
`PlanRunner` spawns agents) gain the option to pass a token down from the
top-level CLI Ctrl-C handler.

## Rules

1. **Windows compat preserved.** Guard Unix signals behind `cfg(unix)`.
2. **Grace period configurable.** Default 5 s; exposed via `SpawnOptions`
   (already exists as `grace_period`).
3. **No deadlocks in Drop.** Drop runs synchronously; do not `tokio::spawn`
   or await inside it. Use `nix::signal::kill` directly.
4. **Idempotent shutdown.** Calling `shutdown` twice must not panic.
5. **No behaviour change for callers that don't pass a token.** All
   `cancellation` fields default to `None`.
