## Task

Today `run_gate_rung` writes every `Verdict` into `FileSubstrate` as a
`Kind::GateVerdict` signal (`orchestrate.rs:11170-11188`). No reader
consumes these signals. Add:

1. A reader that walks the substrate by kind.
2. A per-gate rolling pass/fail aggregator.
3. A "Verdicts" panel on the TUI Gate tab that renders a per-gate
   sparkline timeline.

## Implementation

### 1. Reader helper

```rust
// crates/roko-cli/src/tui/verdicts.rs (new)
pub struct VerdictsAggregator {
    substrate: Arc<FileSubstrate>,
    last_seen_cursor: SubstrateCursor,
    per_gate: HashMap<String, GateStats>,
}

pub struct GateStats {
    pub name: String,
    pub buckets: TrendBuckets, // bucket_size=1h, N=24 buckets
    pub recent_failures: RingBuffer<FailureEntry>,
}

pub struct FailureEntry {
    pub ts: chrono::DateTime<chrono::Utc>,
    pub task_id: String,
    pub artifacts: Option<PathBuf>,
}

impl VerdictsAggregator {
    pub fn tick(&mut self) -> Result<()> {
        for signal in self.substrate.query(Kind::GateVerdict, self.last_seen_cursor)? {
            self.ingest(signal);
        }
        Ok(())
    }
}
```

### 2. TrendBuckets primitive

Lightweight:
```rust
#[derive(Debug, Clone, Default, Serialize, Deserialize)]
pub struct TrendBuckets {
    pub bucket_size_secs: u64,
    pub slots: VecDeque<Bucket>,
}

pub struct Bucket {
    pub start: chrono::DateTime<chrono::Utc>,
    pub pass: u32,
    pub fail: u32,
}
```

Consider hoisting to a shared location (`crates/roko-learn/src/trend.rs`)
since UX17 + UX22 need the same shape.

### 3. Render on Gate tab

In `dashboard_view.rs` or a new `views/gate_view.rs`:

- Small grid: one row per gate (compile, test, clippy, …)
- 24 vertical bars per row; bar height = `pass / (pass+fail)` per bucket
- Recent failures list in a side panel

### 4. StateHub extension

Add to `DashboardSnapshot`:

```rust
pub gate_trends: HashMap<String, TrendBuckets>,
pub gate_recent_failures: Vec<FailureEntry>,
```

Populate from the `VerdictsAggregator` on every refresh.

## Rules

1. **Read-only over substrate.** Do not rewrite verdicts; only aggregate.
2. **Cursor-based.** Never re-scan from start after the first tick.
3. **Bounded memory.** Buckets = 24 × 1 h; recent failures ring = 50.
4. **Graceful if no verdicts.** Empty trend renders as "no data yet".
