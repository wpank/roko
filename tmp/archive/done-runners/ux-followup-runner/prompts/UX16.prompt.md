## Task

`roko-conductor::diagnosis` computes diagnoses today (orchestrate.rs invokes
it), but the output is invisible to the operator — no TUI widget and no
HTTP endpoint. Add both:

1. `DashboardSnapshot.diagnoses: Vec<DiagnosisSummary>` populated by the
   conductor on every cycle.
2. TUI panel on a new "Diagnosis" tab or within the existing dashboard
   page.
3. `GET /api/diagnosis/recent` endpoint.

## Implementation

### 1. Publish diagnoses into StateHub

```rust
// inside orchestrate.rs after a conductor tick
if let Some(diagnosis) = conductor.latest_diagnosis() {
    let summary = DiagnosisSummary::from(&diagnosis);
    state_hub.apply(DashboardEvent::Diagnosis(summary));
}
```

Add `DashboardEvent::Diagnosis(DiagnosisSummary)` in `roko-core`.

### 2. Summary type

```rust
// crates/roko-core/src/dashboard_snapshot.rs
pub struct DiagnosisSummary {
    pub id: String,
    pub ts: chrono::DateTime<chrono::Utc>,
    pub severity: DiagnosisSeverity,
    pub subject: String,      // e.g. "agent stuck", "plan DAG deadlock"
    pub detail: String,       // short human-readable explanation
    pub suggested_action: Option<String>,
    pub intervention_taken: Option<String>,
}

pub enum DiagnosisSeverity { Info, Warn, Alert }
```

Bounded ring in `DashboardSnapshot`:

```rust
pub diagnoses: VecDeque<DiagnosisSummary>,  // cap 50
```

### 3. TUI widget

Add a "Diagnosis" sub-section to the dashboard Conductor area (if there is
one) or a dedicated tab. Render one row per diagnosis with colour per
severity (rose tones per ROSEDUST theme).

### 4. HTTP endpoint

`crates/roko-serve/src/routes/diagnosis.rs` (new):

```rust
pub async fn recent(State(state): State<Arc<AppState>>) -> Json<Vec<DiagnosisSummary>> {
    let snap = state.state_hub.snapshot_borrow();
    Json(snap.diagnoses.iter().cloned().collect())
}
```

Register: `GET /api/diagnosis/recent`.

## Rules

1. **Bounded ring.** 50 diagnoses max; older drop.
2. **No duplicate rows.** Dedup by `DiagnosisSummary::id` before pushing.
3. **Non-blocking.** Publishing to StateHub must not slow the conductor
   tick.
4. **JSON schema stable.** `DiagnosisSummary` serde `#[serde(default)]` on
   new fields; additive only.
