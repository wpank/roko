## Task

The roko-core obs kernel defines the canonical metric surface. The per-agent
sidecar (`roko-agent-server`) records its own counters/histograms in
`state.rs`. Today nothing ensures the field names + label sets agree between
them — a metric named `agent_turn_total` on one path could be `turns_total`
on the other, producing two unrelated time series in any external store.

Audit the union, pick canonical names, and have one path re-export the
other (or share a `MetricSchema` trait).

## Implementation

### 1. Audit step (document findings in final message)

Grep every counter / histogram declaration in both:
- `crates/roko-core/src/obs/` (`metrics.rs`, `histograms.rs`)
- `crates/roko-agent-server/src/state.rs`
- `crates/roko-serve/src/routes/status.rs`

Produce a table in the final message:

| Metric (canonical) | roko-core emits? | agent-server emits? | labels |

### 2. Shared schema trait

`crates/roko-core/src/obs/schema.rs` (new):

```rust
pub const AGENT_TURN_TOTAL: &str = "agent_turn_total";
pub const AGENT_TURN_LATENCY_MS: &str = "agent_turn_latency_ms";
pub const GATE_VERDICT_TOTAL: &str = "gate_verdict_total";
// … one const per metric name

pub trait MetricSchema {
    fn schema_version() -> u32;
}
```

### 3. Agent-server adopts the constants

Replace any hand-written strings in `roko-agent-server/src/state.rs` with
references to the constants above:

```rust
- self.metrics.counter("turns_total").inc();
+ self.metrics.counter(roko_core::obs::schema::AGENT_TURN_TOTAL).inc();
```

### 4. Regression test

```rust
// crates/roko-core/tests/metric_schema.rs

fn all_agent_server_metrics_use_canonical_names() {
    // scan crates/roko-agent-server/src/state.rs file content at compile
    // time via include_str!, grep for raw string literal uses in metric
    // calls that are not in the SCHEMA const set.
}
```

### 5. `/metrics` aggregator diff

Confirm `roko-serve/src/routes/status.rs` (or equivalent) union of the two
sources uses the same field names. Any that differ get renamed in the
emitter (prefer the `roko-core` name).

## Rules

1. **No metric renames visible to external scrape targets.** Prefer the
   existing `roko-core` names; rename on the agent-server side.
2. **Backward-compatible aliases if needed.** If a field must rename,
   emit both old + new for 1 release cycle and note it in final message.
3. **No new deps.** Use only what's in workspace.
