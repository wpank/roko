## Task

`roko-serve` already exposes `/api/agents/topology` but the TUI has no
topology view. Add a compact nested-list / tree widget that renders the
agent → pool → active-tasks topology on demand (Ctrl+T keybinding or a new
sub-tab on the Agents page).

## Implementation

### 1. Topology type

Reuse whatever struct `roko-serve` returns (probably
`AgentTopology { pools: Vec<Pool>, orphans: Vec<Agent> }`). Re-export from
`roko-core` if not already shared.

### 2. StateHub field

```rust
pub agent_topology: AgentTopology,
```

Populated by a background fetch (Agents tab activation triggers a one-shot
fetch via the aggregator client) OR by the HTTP handler pushing into
StateHub.

### 3. Widget

Render as a nested list:

```
Agent Topology
├── pool: default
│   ├── agent-01 [idle]
│   │   ├── task: implement-x (running)
│   │   └── task: review-y    (queued)
│   └── agent-02 [busy]
└── orphans
    └── agent-03
```

Use box-drawing chars; no external graph lib needed.

### 4. Keybinding

`Ctrl+T` from anywhere → toggle topology modal / sub-tab.

## Rules

1. **Read-only widget.** No mutations via the topology panel.
2. **Refresh on tab switch.** One fetch on activation; don't poll.
3. **Graceful if `/api/agents/topology` returns 404** (older `roko serve`
   versions). Show "topology not available" placeholder.
