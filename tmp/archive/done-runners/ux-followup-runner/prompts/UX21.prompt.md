## Task

`roko-agent-server` exposes `/health`, `/stats`, `/message`, `/stream`,
`/predictions`, `/research`, `/tasks` — but no way to tail the agent's
own log file. Add `GET /logs?tail=N` on the sidecar + proxy via aggregator
at `GET /api/agents/{id}/logs?tail=N`.

## Implementation

### 1. Sidecar feature module

`crates/roko-agent-server/src/features/logs.rs` (new):

```rust
pub async fn logs_handler(
    State(state): State<Arc<AgentState>>,
    Query(q): Query<LogsQuery>,
) -> Result<Json<LogsResponse>, (StatusCode, Json<Value>)> {
    let tail = q.tail.unwrap_or(200).min(2000);
    let path = state.log_path();
    match tail_file(&path, tail) {
        Ok(lines) => Ok(Json(LogsResponse { lines, path: path.display().to_string() })),
        Err(err) => Err((StatusCode::INTERNAL_SERVER_ERROR, Json(json!({ "error": err.to_string() })))),
    }
}

fn tail_file(path: &Path, n: usize) -> io::Result<Vec<String>> {
    let content = std::fs::read_to_string(path)?;
    Ok(content.lines().rev().take(n).map(String::from).collect::<Vec<_>>()
        .into_iter().rev().collect())
}
```

Route: `.route("/logs", get(logs_handler))` in
`roko-agent-server/src/lib.rs`.

### 2. `AgentState::log_path`

Add a log-path field to `AgentState` (seeded at boot from `.roko/agents/<id>/log`
or wherever the sidecar writes its log). If no log file yet, return a
`204 No Content` with empty lines.

### 3. Aggregator proxy

`crates/roko-serve/src/routes/agents.rs`:

```rust
.route("/:id/logs", get(agents::proxy_logs))
```

```rust
pub async fn proxy_logs(
    State(state): State<Arc<AppState>>,
    Path(id): Path<String>,
    Query(q): Query<LogsQuery>,
) -> Result<Json<LogsResponse>, ProxyError> {
    let url = state.agent_sidecar_url(&id)?;
    let client = reqwest::Client::new();
    let resp = client.get(format!("{url}/logs")).query(&q).send().await?;
    Ok(resp.json().await?)
}
```

### 4. CLI wrapper

Optional but useful: `roko agent logs <id> --tail N` in
`crates/roko-cli/src/main.rs`.

## Rules

1. **Tail only, never full file.** `?tail` caps at 2000.
2. **No secrets leak.** If log lines are scrubbed at write time, no extra
   handling; otherwise run them through `roko-core::obs::scrub`.
3. **No unbounded read.** If a log is pathological (multi-GB), still read
   only the last `tail` lines (via `rev_lines` crate or `read_to_string` +
   `tail` — accept that multi-GB is pathological).
4. **Aggregator surface idempotent.** GET only; no side-effects.
