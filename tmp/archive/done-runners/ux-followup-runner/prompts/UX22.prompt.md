## Task

`GET /api/c-factor/trend?window=24h|7d` driven by a rolling scan of
`.roko/learn/c-factor.jsonl`, plus a trend widget on the Learning tab.

### 2. HTTP route

`crates/roko-serve/src/routes/learning.rs`:

```rust
.route("/c-factor/trend", get(cfactor_trend_handler))
```

```rust
pub async fn cfactor_trend_handler(
    State(state): State<Arc<AppState>>,
    Query(q): Query<TrendQuery>,
) -> Json<Vec<CFactorBucket>> {
    let (bucket, n) = match q.window.as_deref() {
        Some("7d") => (chrono::Duration::hours(1), 168),
        _ => (chrono::Duration::hours(1), 24),
    };
    let path = state.workdir.join(".roko/learn/c-factor.jsonl");
    match cfactor_trend(&path, bucket, n) {
        Ok(buckets) => Json(buckets),
        Err(_) => Json(Vec::new()),
    }
}
```

## Rules

1. **Cheap.** Same SLA as UX17 (≤ 50 ms for 10 k events).
2. **Default window = 24 h** if query param absent.
3. **Empty if file missing.** Return `[]`, not 404.
4. **Don't duplicate `/api/metrics/c_factor`** — the new endpoint is
   additive.
