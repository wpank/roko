## Task

`PlaybookStore::new` is called at boot (`orchestrate.rs:1584`) but
`store.query(…)` is never invoked. Wire the store into the prompt builder's
"memory" layer so the agent sees relevant playbooks on each dispatch.

## Implementation

### 1. Pick the seam

Three candidates:
- **A**: Prompt builder's memory layer (`roko-compose::system_prompt_builder`)
- **B**: Dispatcher pre-prompt hook (`roko-agent::dispatcher::ToolDispatcher`)
- **C**: New `roko-orchestrator::pre_dispatch` hook

Recommendation: **A**. The memory layer already composes context per agent
turn; playbooks belong there semantically.

### 2. Query API

```rust
// crates/roko-learn/src/playbook.rs (extend)
impl PlaybookStore {
    pub fn query(&self, ctx: &QueryContext) -> Vec<Playbook> {
        // Rank playbooks by similarity to the current task prompt + recent
        // episodes. Return top-N.
    }
}

pub struct QueryContext {
    pub task_id: String,
    pub task_title: String,
    pub role: String,
    pub recent_episodes: usize,   // how many recent episodes to consider
    pub max_results: usize,       // cap 5
}
```

### 3. Wire into `SystemPromptBuilder::memory_layer`

```rust
fn memory_layer(&self, ctx: &BuilderContext) -> LayerText {
    let mut text = String::new();
    text.push_str(&self.render_recent_episodes(ctx));
    text.push_str(&self.render_playbooks(ctx));
    LayerText { role: "memory".into(), text }
}

fn render_playbooks(&self, ctx: &BuilderContext) -> String {
    let Some(store) = &self.playbook_store else { return String::new(); };
    let query = QueryContext {
        task_id: ctx.task_id.clone(),
        task_title: ctx.task_title.clone(),
        role: ctx.role.clone(),
        recent_episodes: 10,
        max_results: 3,
    };
    let playbooks = store.query(&query);
    render_playbooks_markdown(&playbooks)
}
```

### 4. Pass the store into the builder

`SystemPromptBuilder::new` grows an optional
`playbook_store: Option<Arc<PlaybookStore>>` parameter. Defaults to `None`
to preserve existing tests.

### 5. Integration test


## Rules

1. **Optional store.** Missing store → empty playbook section, no crash.
2. **Cap at 3 playbooks per turn.** Avoid blowing out the context window.
3. **Stable ordering.** Sort by rank descending, then by id ascending.
4. **No circular deps.** `roko-compose` does not yet import
   `roko-learn`; check Cargo.toml and add if needed. If the cycle is
   impossible, pass the rendered playbook markdown through the builder
   args instead of the store.
