## Task

`SafetyLayer::check_pre_execution` receives the calling role but does not
enforce a per-tool ACL. Add a role→tools whitelist parsed from `roko.toml`
and rejected invocations on mismatch.

## Implementation

### 1. `roko.toml` schema

```toml
[agent.implementer]
model = "claude-opus-4-6"
tools = ["read", "edit", "bash", "git-*"]

[agent.reviewer]
model = "claude-sonnet-4-6"
tools = ["read"]
```

Glob support: `bash`, `git-*`, `*` (allow all), `!git-push` (deny — optional).

### 3. SafetyLayer lookup

```rust
pub struct SafetyLayer {
    role_tools: HashMap<String, ToolWhitelist>,   // NEW
    // existing fields…
}

struct ToolWhitelist(Vec<glob::Pattern>);

impl ToolWhitelist {
    fn matches(&self, tool: &str) -> bool {
        self.0.iter().any(|p| p.matches(tool))
    }
}

impl SafetyLayer {
    pub async fn check_pre_execution(&self, inv: &Invocation, role: &str) -> Result<(), SafetyError> {
        // existing checks…
        if let Some(wl) = self.role_tools.get(role) {
            if !wl.matches(&inv.tool) {
                return Err(SafetyError::ToolNotInRoleWhitelist {
                    role: role.to_string(),
                    tool: inv.tool.clone(),
                });
            }
        }
        Ok(())
    }
}
```

### 4. Absent whitelist → permissive

If a role has no `tools` entry in `roko.toml`, allow all. This preserves
backwards compat.

## Rules

1. **Glob patterns** via the `glob` crate. `git-*` matches `git-commit`,
   `git-push`, etc.
2. **Permissive default.** No `tools` entry → all allowed.
3. **Deny patterns deferred.** `!git-push` syntax captured in final
   message as a follow-up; don't implement negation in this batch.
4. **No runtime reload.** Whitelist is fixed at dispatcher construction.
