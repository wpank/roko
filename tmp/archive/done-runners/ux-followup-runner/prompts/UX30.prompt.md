## Task

Codex's 2025 protocol extensions (multi-reasoning, draft-frame semantics)
are in flux. No test harness today confirms our `codex_agent` handles the
full frame set. Build a replay-based conformance harness driven by a
captured session fixture.

## Implementation

### 1. Fixture capture

`crates/roko-agent/testdata/codex/sessions/ten-turn-basic.jsonl`:
- one line per over-the-wire frame
- includes: `draft`, `reasoning_delta`, `tool_call`, `tool_result`,
  `final`

(If a real session can be captured, use it. Otherwise hand-craft a
fixture matching the current Codex spec as of April 2026.)

### 2. Replay driver

`crates/roko-agent/tests/codex_conformance.rs`:

```rust
#[tokio::test]
async fn codex_ten_turn_basic_replays_cleanly() {
    let fixture = load_fixture("ten-turn-basic.jsonl");
    let mut mock_http = MockHttp::new(fixture.frames.clone());
    let agent = CodexAgent::from_mock(mock_http.url());
    let resp = agent.send_turn(mock_req("please help")).await.unwrap();

    assert_eq!(resp.content, fixture.expected_content);
    assert_eq!(resp.reasoning, fixture.expected_reasoning);
    assert_eq!(resp.tool_calls.len(), fixture.expected_tool_calls);
    assert_eq!(resp.session, Some(fixture.expected_session));
}
```

### 3. MockHttp helper

A small `MockHttp` inside the test file that serves the frames in sequence
on the next `POST /v1/chat/completions` (or whichever path Codex uses).

### 4. Future extensions (as follow-up notes)

- Streaming harness (`streaming-basic.jsonl`)
- Tool-call loop harness (`tool-call-multi.jsonl`)
- Error / retry harness (`error-after-reasoning.jsonl`)

This batch lands just the 10-turn basic. Each extension can be its own

## Rules

1. **Offline-only.** The test must not require a real Codex endpoint.
2. **Record-replay, not mock-all.** The frame fixture is the source of
   truth; assertions derive from it.
3. **Deterministic.** Same fixture → same pass/fail.
4. **Small bound.** Fixture < 50 KB to keep repo lean.
