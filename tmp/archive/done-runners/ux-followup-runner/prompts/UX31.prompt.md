## Task

Cursor's agent API supports streaming but our `cursor_agent.rs` has only
non-streaming send_turn. Port the Claude streaming pattern so Cursor
implements `send_turn_streaming` on `LlmBackend`.

## Implementation

### 1. Add `send_turn_streaming`

In `crates/roko-agent/src/cursor_agent.rs`:

```rust
#[async_trait::async_trait]
impl LlmBackend for CursorAgent {
    async fn send_turn_streaming(
        &self,
        req: ChatRequest,
        mut on_chunk: Box<dyn FnMut(ChatStreamChunk) + Send>,
    ) -> Result<ChatResponse> {
        let url = format!("{}/v1/chat/completions", self.base_url);
        let mut stream = self.http.post(&url).json(&req).send().await?.bytes_stream();
        let mut accumulated = String::new();
        while let Some(bytes) = stream.next().await {
            let frame = parse_sse_frame(&bytes?)?;
            on_chunk(frame.clone());
            accumulated.push_str(&frame.delta_text);
            if frame.is_final { break; }
        }
        Ok(ChatResponse::from_accumulated(accumulated))
    }
}
```

### 2. SSE parser

If Cursor uses SSE (`data: {json}\n\n`) like the OpenAI compat layer, reuse
`crates/roko-agent/src/provider/openai_compat.rs` helpers. If it uses a
different protocol (WebSocket, NDJSON), follow the Cursor docs / inspect
the server's `content-type`.

### 3. Tool-call streaming

If a tool call arrives mid-stream, emit a `ChatStreamChunk::ToolCall` and
keep accumulating subsequent text until the final frame.

## Rules

1. **Don't break non-streaming.** `send_turn` keeps working; streaming is
   additive.
2. **Graceful partial frame handling.** Drop malformed frames with a
   tracing warn; never panic.
3. **Back-pressure aware.** If `on_chunk` is slow, do not buffer the whole
   stream — await the callback each iteration.
4. **Close the HTTP response.** Drop stream cleanly on termination.
