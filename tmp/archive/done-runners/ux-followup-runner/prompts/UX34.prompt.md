## Implementation

### 1. Integration test

`crates/roko-learn/tests/cascade_router_integration.rs` (new):

```rust
#[tokio::test]
async fn cascade_selects_cheaper_backend_after_learning() {
    let tmp = tempfile::TempDir::new().unwrap();
    let dispatcher = ToolDispatcher::new_with_two_mocks(
        MockBackend::expensive_success(),
        MockBackend::cheap_success(),
    );
    let mut router = CascadeRouter::new(
        tmp.path().join("cascade.json"),
        CascadeConfig::test_defaults(),
    );

    for i in 0..20 {
        let choice = router.choose(&dispatcher).await;
        let req = ChatRequest::mock(format!("turn {i}"));
        let resp = dispatcher.dispatch_with(choice, req).await.unwrap();
        router.record_outcome(choice, &resp).await;
    }

    // After 20 turns, the router should heavily prefer the cheap backend.
    let state = router.load_state().unwrap();
    assert!(state.choice_counts["cheap"] > 12);
    assert_eq!(state.schema_version, cascade_router::CURRENT_SCHEMA_VERSION);

    // The persisted JSON should match the in-memory state.
    let on_disk: CascadeRouterState = serde_json::from_slice(&std::fs::read(tmp.path().join("cascade.json")).unwrap()).unwrap();
    assert_eq!(on_disk, state);
}
```

## Rules

1. **Offline.** No network; mock dispatcher only.
2. **Temp dir isolation.** `tempfile::TempDir` per test.
3. **Deterministic.** Use a fixed RNG seed so choice sequence is
   reproducible.
4. **Under 5 s per test.** If cascade convergence is slow, lower the
   config thresholds via `CascadeConfig::test_defaults()`.
