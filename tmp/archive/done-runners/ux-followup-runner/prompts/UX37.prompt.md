## Task

CLAUDE.md claims `SystemPromptBuilder (6-layer prompts) — Wired`. No test
asserts all 6 layers compose for each role. Add a snapshot test per role
that captures the rendered prompt + diffs against a golden file.

## Implementation

### 1. Enumerate roles

Grep `crates/roko-compose/src/templates/` for the canonical role list.
Expected set: implementer, reviewer, planner, researcher, tester (+ others
the repo ships).

### 2. Snapshot test scaffold

Use the `insta` crate (commonly used for this):

```rust
// crates/roko-compose/tests/system_prompt_snapshot.rs
use roko_compose::system_prompt_builder::{SystemPromptBuilder, BuilderContext};

#[test]
fn implementer_prompt_matches_golden() {
    let ctx = BuilderContext::test_fixture_for("implementer");
    let prompt = SystemPromptBuilder::new().build(&ctx);
    insta::assert_snapshot!("implementer", prompt);
}

// similar tests for reviewer, planner, researcher, tester
```

### 3. Fixture builder

`BuilderContext::test_fixture_for(role)` returns a deterministic context
(fixed timestamps, canned recent episodes, canned playbooks, canned tools).

### 4. Golden files

`crates/roko-compose/tests/snapshots/*.snap` — committed, diffed on each
run.

### 5. Layer enumeration in the rendered prompt

The snapshot output should make each of the 6 layers identifiable by a
header comment:

```
# ROLE
…
# SKILLS
…
# TOOLS
…
# CONTEXT
…
# MEMORY
…
# POLICY
…
```

If the current builder does not emit per-layer headers, add them (in
comment form that the LLM ignores but humans can scan).

### 6. Drift-detection

Running the test should show the diff vs golden; acceptance requires
`cargo insta accept` once the output is reviewed.

## Rules

1. **Deterministic fixtures.** Fixed timestamps + content; no time / RNG.
2. **Snapshot-on-accept.** First run produces the snapshot; review + accept.
3. **6 layers all present.** Each snapshot must contain all 6 layer
   headers; test fails if any is missing.
4. **No network.** `BuilderContext` carries all data in-process.

  accepting snapshots.
- `cargo clippy -p roko-compose --no-deps -- -D warnings` passes.
