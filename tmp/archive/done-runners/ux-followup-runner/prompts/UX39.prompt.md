## Task

All `roko-serve` routes use bare `Json<T>` extraction. A malformed POST
body is caught only by the inner deserializer; server returns the raw serde
error in the response. Add:

1. A validation layer on `Json<T>` via `validator` crate (or a custom guard).
2. A shared `ApiError` response type with consistent shape.
3. OpenAPI surface via `utoipa`.

## Implementation

### 1. Validator wrapper

```rust
// crates/roko-serve/src/extract.rs (new)
use axum::{extract::FromRequest, http::Request};
use serde::de::DeserializeOwned;
use validator::Validate;

pub struct ValidJson<T>(pub T);

#[async_trait::async_trait]
impl<T, S, B> FromRequest<S, B> for ValidJson<T>
where
    T: DeserializeOwned + Validate,
    S: Send + Sync,
    B: axum::body::HttpBody + Send + 'static,
{
    type Rejection = ApiError;
    async fn from_request(req: Request<B>, state: &S) -> Result<Self, Self::Rejection> {
        let axum::Json(value) = axum::Json::<T>::from_request(req, state).await.map_err(ApiError::parse)?;
        value.validate().map_err(ApiError::validation)?;
        Ok(ValidJson(value))
    }
}
```

### 2. Shared `ApiError`

```rust
// crates/roko-serve/src/errors.rs (new)
pub struct ApiError {
    pub status: StatusCode,
    pub code: String,
    pub message: String,
    pub details: Option<serde_json::Value>,
}

impl IntoResponse for ApiError { … }

impl ApiError {
    pub fn parse(err: JsonRejection) -> Self { … }
    pub fn validation(err: ValidationErrors) -> Self { … }
    pub fn not_found(msg: &str) -> Self { … }
    pub fn internal(err: impl Display) -> Self { … }
}
```

### 3. Request struct attributes

```rust
#[derive(Deserialize, Validate)]
pub struct CreateAgentRequest {
    #[validate(length(min = 1, max = 128))]
    pub id: String,
    pub model: String,
    // …
}
```

Apply to the ~15 POST bodies the server accepts.

### 4. Replace `Json<T>` in POST handlers with `ValidJson<T>`

Diff handler signatures:

```rust
- pub async fn create_agent(Json(req): Json<CreateAgentRequest>) -> …
+ pub async fn create_agent(ValidJson(req): ValidJson<CreateAgentRequest>) -> …
```

### 5. OpenAPI via utoipa

```rust
// crates/roko-serve/src/openapi.rs (new)
#[derive(utoipa::OpenApi)]
#[openapi(
    paths(agents::list, agents::create, agents::message, …),
    components(schemas(CreateAgentRequest, AgentSummary, …)),
)]
pub struct ApiDoc;
```

Expose at `/api/openapi.json`:

```rust
.route("/api/openapi.json", get(|| async { Json(ApiDoc::openapi()) }))
```

### 6. CI schema-drift check

Optional but useful: `cargo xtask openapi-check` that regenerates the
`openapi.json` and fails if the checked-in copy drifts.

## Rules

1. **Non-breaking.** Response bodies for successful paths stay the same.
2. **Error bodies unified.** `ApiError` shape is: `{ code, message,
   details? }` with matching `status` HTTP code.
3. **Not a rewrite.** Add validation to POST bodies that need it; don't
   retrofit GET handlers that have no body.
4. **OpenAPI as additional artefact.** The CI check is a follow-up if this
   batch is time-constrained.
