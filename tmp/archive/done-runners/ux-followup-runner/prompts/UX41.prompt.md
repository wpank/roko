## Task

No coverage measurement in CI today. The current tree may not even have a
`.github/workflows/` directory yet. Add a minimal `cargo llvm-cov` workflow
scaffold plus a local helper script so the repo has a concrete coverage
surface instead of just a TODO.

## Implementation

### 1. Create the workflow from scratch if needed

`.github/workflows/coverage.yml`:

```yaml
name: coverage
on:
  pull_request:
  push: { branches: [main] }

jobs:
  coverage:
    runs-on: ubuntu-latest
    timeout-minutes: 45
    steps:
      - uses: actions/checkout@v4
      - uses: dtolnay/rust-toolchain@stable
        with: { components: llvm-tools-preview }
      - uses: taiki-e/install-action@cargo-llvm-cov
      - run: rustup override set stable
      - run: cargo llvm-cov --workspace --html --output-dir target/llvm-cov --ignore-filename-regex='(tests/|target/|testdata/|benches/)'
      - run: cargo llvm-cov --workspace --summary-only > coverage-summary.txt
      - uses: actions/upload-artifact@v4
        with:
          name: coverage-html
          path: target/llvm-cov
      - uses: actions/upload-artifact@v4
        with:
          name: coverage-summary
          path: coverage-summary.txt
```

If `.github/` or `.github/workflows/` does not exist, create it. Do not
assume an existing CI scaffold.

### 2. Local helper

`tools/coverage.sh`:


## Rules

1. **No gating.** CI must **not** fail on coverage below a threshold.
2. **45 min budget.** If the run exceeds, document but do not retry.
3. **Reasonable exclusions.** Ignore `tests/`, `target/`, `testdata/`,
   `benches/` so coverage reflects library code only.
4. **Artifact-only.** No external upload to codecov/coveralls in this
   batch (follow-up).
5. **Repo reality first.** If there is no existing CI directory, the batch
   must create the minimal structure instead of erroring on a missing path.
