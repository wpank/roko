## Task

Two related hygiene sweeps:

1. **Remove `#![allow(clippy::missing_errors_doc)]` / `missing_panics_doc`
   from every crate.** Add the missing `# Errors` / `# Panics` sections
   instead of suppressing.

2. **Audit timeout-based test assertions** (e.g.
   `with_timeout_ms(100)` at `crates/roko-agent/src/exec.rs:504`) and
   stabilise via mock-clock or `CI=true` multiplier.

## Implementation

### Part 1: clippy docs

Sequence per crate:

1. Remove `#![allow(clippy::missing_errors_doc)]` from `lib.rs`.
2. Run `cargo clippy -p <crate> --no-deps -- -D warnings -D clippy::missing_errors_doc`.
3. For each hit, add `# Errors` doc block:

   ```rust
   /// Returns the thing.
   ///
   /// # Errors
   ///
   /// Returns [`ThingError::NotFound`] if the thing is absent.
   pub fn get_thing() -> Result<Thing, ThingError> { … }
   ```

4. Same for `missing_panics_doc` with `# Panics`.

Prioritize crates where the allow was newly added (git blame will show).
At a minimum cover: `roko-cli`, `roko-serve`, `roko-agent`, `roko-agent-server`,
`roko-gate`, `roko-learn`, `roko-compose`.

### Part 2: timeout-flake audit

Enumerate timeout-based test assertions:
```bash
grep -rn "with_timeout_ms\|timeout.*Duration::from_millis([1-9][0-9]*)\b" crates/ --include="*.rs"
```

For each:
- **Preferred**: switch to `tokio::time::pause()` + `advance()` for
  deterministic mock-clock behaviour.
- **Fallback**: multiply the timeout by 10 under `CI=true`:

  ```rust
  fn scale_test_timeout(ms: u64) -> u64 {
      if std::env::var("CI").map(|v| v == "true").unwrap_or(false) {
          ms * 10
      } else {
          ms
      }
  }
  ```

## Write scope

- Every crate's `lib.rs` or `main.rs` that currently suppresses
  `clippy::missing_*`
- Doc comments across the listed crates
- `crates/roko-agent/src/exec.rs` (timeout scaling)
- Other tests with bare `100 ms` timeouts identified in the audit

## Rules

1. **No new allows.** Do not re-suppress what you removed.
2. **Doc accuracy.** `# Errors` lists the actual error variants, not
   placeholders.
3. **Tests pass on slow CI.** A 10x scale factor under `CI=true` is the
   fallback; mock clock preferred.
4. **Single commit per crate.** Easier to review / revert.

## Done when

- `cargo clippy --workspace --no-deps -- -D warnings -D clippy::missing_errors_doc -D clippy::missing_panics_doc`
  passes.
- `cargo test --workspace --no-run` compiles.
- Any remaining `#![allow(clippy::missing_*)]` directive has a TODO
  comment referencing a follow-up ticket.
- Final message lists: crates touched, timeout audit count + resolutions,
  remaining allows (with rationale).
