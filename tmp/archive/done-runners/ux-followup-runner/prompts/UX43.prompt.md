## Task

The checklist claims ~33 % done but was never mechanically verified. Build
a walker tool that, for each item, greps the current codebase for the
relevant call-site / type / file and produces a fresh "done/not" annotated
copy.

## Implementation

### 1. Tool crate / script

`tools/mori-parity-check/` (new rust binary crate) OR
`tools/mori-parity-check.sh` (plain bash with ripgrep). Pick whichever is
idiomatic for the repo. Prefer the bash variant for simplicity.

```bash
#!/usr/bin/env bash
# tools/mori-parity-check.sh
set -euo pipefail

CHECKLIST=bardo-backup/tmp/roko-progress/MORI-PARITY-CHECKLIST.md
OUTPUT=tmp/mori-parity-verified.md

regenerate_checklist() {
    python3 tools/mori-parity-check/verify.py \
        --checklist "$CHECKLIST" \
        --repo "$(pwd)" \
        --output "$OUTPUT"
}

regenerate_checklist
```

### 2. Python verifier

`tools/mori-parity-check/verify.py`:

```python
"""
Walks MORI-PARITY-CHECKLIST.md, extracts ✓/✗ items + the code references
embedded in each bullet (e.g. `apps/mori/src/agent/connection.rs:2400`),
uses the Mori appendix to map those anchors into the current Roko tree,
greps the current repo for the same symbol in `crates/`, and writes an
updated checklist with verified ticks.
"""
import argparse
import re
import subprocess
from pathlib import Path

MORI_REF = re.compile(r'`(apps/mori/\S+?\.rs)(?::(\d+))?`')
SYMBOL_HINT = re.compile(r'`([A-Z][A-Za-z0-9_]+|[a-z][a-z_]+\w*)`')

def verify_item(line: str, repo: Path) -> tuple[str, str]:
    # Extract the key symbol, grep crates/
    symbols = SYMBOL_HINT.findall(line)
    for s in symbols:
        hit = subprocess.run(["rg", "--no-heading", "-l", s, str(repo / "crates")],
                             capture_output=True, text=True)
        if hit.returncode == 0 and hit.stdout.strip():
            return "✅", hit.stdout.splitlines()[0]
    return "❌", ""

# Walk the checklist, emit verified copy
```

### 3. Output format

Write to `tmp/mori-parity-verified.md` with the same structure as the input
but:
- ✅ items include the file where the symbol was found
- ❌ items include "not found in crates/"
- Header includes: total items, ✅ count, ❌ count, % complete

Weekly GitHub Action that runs the tool + opens a PR with the fresh

## Rules

1. **Heuristic, not exact.** Symbol grep is imperfect. Document the false-
   positive rate in the tool's README.
2. **Read-only over `bardo-backup/`.** Do not modify the source
   MORI-PARITY-CHECKLIST.md.
3. **Stable output.** Same input → same output (sort hits).
4. **Reasonable runtime.** ≤ 60 s on a modern laptop over the full
   checklist.

## Done when

- `bash tools/mori-parity-check.sh` produces `tmp/mori-parity-verified.md`.
- The output shows an explicit ✅/❌ per item with evidence.
- Final message reports: total items, verified ✅ count, ✗ count, %
  complete, tool runtime.
