## Implementation

### 1. Post-mortem

Inspect the 2026-04-16 run logs and `BATCHES.md` + `common.sh` diff to
find why the runner stopped at T14. Write findings to
`tmp/tui-parity/POSTMORTEM-20260416.md`.

Likely suspect: a hardcoded max-batch count or a crashing step without
error propagation.

### 2. Env knobs

In `tmp/tui-parity/lib/common.sh`:

```bash
: "${TUI_PARITY_MAX_BATCHES:=0}"   # 0 = unlimited
: "${TUI_PARITY_MAX_RETRIES:=${UX_MAX_RETRIES:-2}}"
```

In `run-tui-parity.sh`, log them at startup:

```bash
log_info "runner" "TUI_PARITY_MAX_BATCHES=$TUI_PARITY_MAX_BATCHES TUI_PARITY_MAX_RETRIES=$TUI_PARITY_MAX_RETRIES"
```

And honour `TUI_PARITY_MAX_BATCHES > 0` in the main batch loop (same
pattern as this runner's `UX_MAX_BATCHES`).

### 3. Log retention

`.gitignore`:

```
tmp/tui-parity/logs/run-*/
!tmp/tui-parity/logs/run-*/manifest.env
!tmp/tui-parity/logs/run-*/status.tsv
```

(Or simpler: ignore the whole directory — since `tmp/` is already
gitignored per `59835802`, this may already be covered.)

Retention script `tmp/tui-parity/tools/rotate-logs.sh`:

```bash
#!/usr/bin/env bash
# Keep the 5 most recent runs; delete older.
ls -1dt tmp/tui-parity/logs/run-* 2>/dev/null \
    | tail -n +6 \
    | xargs -I{} rm -rf {}
```

Wire into `run-tui-parity.sh` preflight so every run rotates first.

### 4. CI dry-run

`.github/workflows/tui-parity-dry-run.yml` (new):

```yaml
name: tui-parity dry-run
on:
  pull_request:
    paths:
      - 'tmp/tui-parity/**'
      - 'tmp/ux-followup-runner/**'
jobs:
  dry-run:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: dry-run tui-parity runner
        run: bash tmp/tui-parity/run-tui-parity.sh --dry-run
      - name: dry-run ux-followup runner
        run: bash tmp/ux-followup-runner/run-ux-followup.sh --dry-run
```

### 5. Trailer on silent spawn_failed

Harden `lib/spawn.sh` (and `lib/spawn.sh` here, parallel edit) so that
every invocation, if no result file is written + codex returns 0, emits
a trailer to `status.tsv`:

```tsv
<ts> <batch> <attempt> no_result_file codex returned 0 but wrote no result
```

## Rules

1. **Backwards compatible.** Existing invocations without `TUI_PARITY_MAX_*`
   set behave as today.
2. **Rotation never deletes an in-flight run.** Check for a live
   `current-batch.env` before deleting.
3. **CI step is informational.** Dry-run does not need Codex; it only
   snapshots prompts.
4. **No runtime rust changes.**
