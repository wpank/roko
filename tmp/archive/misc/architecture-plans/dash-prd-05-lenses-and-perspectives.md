````markdown
## Why this document matters

The same underlying data means different things depending on whose vantage point you take. A leaderboard looks different when you're viewing your own agents, your team's agents, all agents in an arena, all agents in a domain, or all agents on the network. A cost breakdown looks different at the agent level, the fleet level, and the network level. A knowledge entry has different implications depending on whether you're its author, a consumer, or a challenger.

The dashboard could handle this by making many different pages for the same underlying concept: "My Agents Leaderboard," "Team Leaderboard," "Arena Leaderboard," "Network Leaderboard." This would be a mistake. It duplicates work, produces inconsistent behavior between near-identical pages, and fragments the user's mental model.

The correct approach is to treat vantage point as a first-class UI concept called a **lens**. A lens is a filter on the data a page displays. The same page supports multiple lenses; the user switches between them with a visible control; the underlying data and interactions remain the same.

This document specifies the five canonical lenses, what each lens means, how lenses compose with pages and with filters, how the UI makes the current lens obvious, and how lenses interact with permissions and privacy.

Subsequent documents (specifically the page specifications in Section IV) declare which lenses each page supports.

---
````

````markdown
**Lenses are not modes.** A mode implies the whole UI changes. Lenses change only the scope of the current page. Navigation, layout, controls, and affordances remain the same.
````
