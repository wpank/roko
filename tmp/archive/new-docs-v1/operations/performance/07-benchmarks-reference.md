# Benchmarks Reference

> The per-subsystem benchmark suites, how to run them, and where to find historical
> results.

**Status**: Shipping
**Crate**: `roko-core`, `roko-fs`, `roko-orchestrator`
**Depends on**: [01-latency-budgets.md](01-latency-budgets.md)
**Last reviewed**: 2026-04-19

## TL;DR

Run all benchmarks with `cargo bench --workspace`. Historical results are in
[`status/benchmarks.md`](../../status/benchmarks.md). Compare before/after any change
to a hot path.

## Benchmark Suite Layout

Benchmarks live next to the crate they test, in `crates/<crate>/benches/`:

```
crates/
  roko-core/
    benches/
      engram.rs         ← Engram construction, BLAKE3 hashing, field mutation
      score.rs          ← Score arithmetic (7-axis compute, clamp, decay step)
      hdc.rs            ← HDC bind, bundle, permute, hamming_distance
      decay.rs          ← All four decay models over 1K/10K/100K steps
  roko-fs/
    benches/
      substrate.rs      ← JSONL append (single, batched), GC scan
  roko-orchestrator/
    benches/
      context.rs        ← Context assembly (cold/warm index, various sizes)
      dag.rs            ← DAG scheduling (100/1000 task graphs)
  roko-learn/
    benches/
      router.rs         ← CascadeRouter T0/T1/T2 dispatch latency
      bandit.rs         ← Thompson Sampling update, arm selection
      hdc_search.rs     ← HDC similarity search (10K/100K/1M entries)
```

## Running Benchmarks

### All benchmarks

```bash
cargo bench --workspace
```

Output goes to `target/criterion/`. Criterion produces HTML reports at
`target/criterion/report/index.html`.

### Single subsystem

```bash
cargo bench -p roko-core
```

### Single benchmark group

```bash
cargo bench -p roko-core --bench hdc -- hdc_hamming
```

### With baseline comparison

```bash
# Save baseline
cargo bench --workspace -- --save-baseline before

# Make your change, then:
cargo bench --workspace -- --baseline before
```

Criterion will report the percentage change and statistical significance for each
benchmark.

## Key Benchmarks and Their Thresholds

These are the benchmarks that must pass on every PR (checked by the regression
detector — see [08-regression-detection.md](08-regression-detection.md)):

| Benchmark | Current p99 | Regression threshold |
|-----------|------------|---------------------|
| `engram::bench_engram_new` | 280 ns | > 350 ns |
| `score::bench_score_compute` | 80 ns | > 100 ns |
| `hdc::bench_hdc_hamming` | 65 ns | > 85 ns |
| `hdc::bench_hdc_bind` | 18 ns | > 25 ns |
| `substrate::bench_jsonl_append_single` | 60 µs | > 100 µs |
| `substrate::bench_jsonl_append_batch_100` | 180 µs | > 300 µs |
| `hdc_search::bench_search_100k` | 3 ms | > 6 ms |
| `context::bench_assembly_warm_small` | 150 ms | > 300 ms |

The regression threshold is a 25% increase over the current value (not the original
baseline), to accommodate natural variance while catching meaningful regressions.

## Benchmark Machines

Criterion benchmarks run on the CI machine (GitHub Actions, `ubuntu-latest`, 2-core x86-64,
4 GB RAM). Results will differ on your local machine. When comparing across machines:

- Relative comparisons (before/after within a machine) are valid.
- Absolute numbers (comparing to the table above) should be within 2× for a modern
  developer machine.

For authoritative numbers, run on the CI machine or a dedicated benchmark machine with
consistent CPU frequency scaling disabled:

```bash
echo "performance" | sudo tee /sys/devices/system/cpu/cpu*/cpufreq/scaling_governor
cargo bench --workspace
```

## Adding a Benchmark

New benchmarks must be added when:

1. A new hot-path operation is introduced (any function called in the 5 hot paths).
2. The threshold for an existing benchmark is changed.
3. A new subsystem that has per-Engram or per-task processing is added.

Template for a new benchmark:

```rust
// crates/roko-new/benches/my_operation.rs
use criterion::{black_box, criterion_group, criterion_main, Criterion};
use roko_new::MyType;

fn bench_my_operation(c: &mut Criterion) {
    let input = MyType::new(/* ... */);
    c.bench_function("my_operation", |b| {
        b.iter(|| {
            black_box(input.do_something());
        })
    });
}

criterion_group!(benches, bench_my_operation);
criterion_main!(benches);
```

Add it to `Cargo.toml`:

```toml
[[bench]]
name    = "my_operation"
harness = false
```

## Historical Results

Historical benchmark results are tracked in [`status/benchmarks.md`](../../status/benchmarks.md).
That file is updated on every release. It contains:

- The raw p50/p95/p99 numbers for each benchmark.
- The trend chart (HTML; generated by Criterion's report).
- Notes on any benchmark changes (e.g. when the HDC vector size changed from 4,096 to
  10,240 bits, all HDC benchmarks were reset).
