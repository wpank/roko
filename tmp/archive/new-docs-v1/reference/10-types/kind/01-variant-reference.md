# Kind — Variant Reference

> A description of every Kind variant: its purpose, typical Body type, and usage context.

**Status**: Shipping  
**Crate**: `roko-core`  
**Depends on**: [Overview](00-overview.md)  
**Last reviewed**: 2026-04-19

---

## AgentOutput

**Purpose**: Records the primary result of an agent invocation — the answer, artifact, or
decision produced by the agent for a task.

**Typical Body**: `Body::Text` (a written response), `Body::Json` (structured result), or
`Body::Structured` (typed fields).

**Lifetime**: Working tier. Retrieved when reviewing what an agent concluded about a task.
Reinforced when the same task type is encountered again.

**Notes**: Gate pipeline reads `AgentOutput` Engrams to assess agent quality over time.

---

## GateVerdict

**Purpose**: Records the outcome of a Gate evaluation — pass, fail, or conditional pass —
along with the score and the Engram that was evaluated.

**Typical Body**: `Body::Json` containing `{verdict, score, engram_id, reason}`.

**Lifetime**: Working tier. Verdicts are consulted during learning loops and post-mortems.

**Notes**: `GateVerdict` Engrams are the input to the learning loop's policy update. They
must not be modified after creation (immutable audit record).

---

## ToolTrace

**Purpose**: A verbatim record of a tool invocation: the tool name, input arguments,
output, and duration.

**Typical Body**: `Body::Json` or `Body::Structured`.

**Lifetime**: Transient tier. Fast decay — a tool trace from last week is rarely needed.

**Notes**: ToolTrace Engrams are the primary input to the `ErrorRecord` creation pipeline
when a tool fails.

---

## KnowledgeEntry

**Purpose**: A durable factual claim or domain knowledge item that the agent should
retain across sessions.

**Typical Body**: `Body::Text` (prose claim) or `Body::Structured` (typed fact).

**Lifetime**: Consolidated tier. Slow decay; reinforced frequently by recall.

**Notes**: The primary target of HDC near-duplicate filtering — overlapping knowledge
entries should be deduplicated before insertion.

---

## Prediction

**Purpose**: A forward-looking belief about a future state of the environment, a system,
or a task outcome.

**Typical Body**: `Body::Json` containing `{subject, expected_value, confidence, deadline_ms}`.

**Lifetime**: Working tier with Exponential decay. Predictions become stale after their
deadline; the system scores prediction accuracy via `GateVerdict` Engrams.

---

## Observation

**Purpose**: A raw sensory input — a log entry, an API response, an event from the
environment — before any interpretation.

**Typical Body**: `Body::Text` (raw log line), `Body::Json` (structured event), or
`Body::Binary` (raw bytes).

**Lifetime**: Transient tier. Short Exponential decay (default 1 day).

**Notes**: Observations are typically processed into higher-level Engrams (KnowledgeEntry,
Prediction) and then allowed to decay.

---

## Plan

**Purpose**: A structured sequence of intended actions to accomplish a goal.

**Typical Body**: `Body::Structured` (list of steps + goal) or `Body::Json`.

**Lifetime**: Working tier with Step decay (sprint-scoped). Plans are valid during their
planning epoch and discounted after.

---

## Episode

**Purpose**: A record of a completed task session — start time, end time, key events,
and outcome.

**Typical Body**: `Body::Json` or `Body::Structured`.

**Lifetime**: Consolidated tier. Episodes are the long-term episodic memory of the agent.

**Notes**: Episodes reference their component ToolTrace and AgentOutput Engrams in their
`lineage` field.

---

## Reflection

**Purpose**: An introspective assessment by an agent about its own performance, knowledge
gaps, or strategy adjustments.

**Typical Body**: `Body::Text` or `Body::Structured`.

**Lifetime**: Persistent tier. The slowest-decaying Kind — reflects the agent's
self-model which should be very stable.

**Notes**: Reflections are generated by the Reflection subsystem at the end of completed
episodes. They feed back into the learning loop.

---

## Pheromone

**Purpose**: A stigmergic signal deposited by an agent for others to follow — a pointer
to an opportunity, a warning, or a recommended path.

**Typical Body**: `Body::Structured` with fields `{kind, scope, intensity, payload}`.

**Lifetime**: Transient tier with fast Exponential decay (default 1 hour).

**Notes**: Pheromone Engrams are the mechanism for inter-agent coordination. See the
cross-pollination research on stigmergic bandits for the full protocol.

---

## Metric

**Purpose**: A point-in-time numeric measurement or health indicator.

**Typical Body**: `Body::Json` containing `{name, value, unit, labels}`.

**Lifetime**: Transient tier with Linear decay (default 1 day lifetime).

**Notes**: Metrics are typically exported to an observability backend before they expire.
The Metric Kind in the Substrate is for agent-internal metrics, not system telemetry.

---

## ContextAssembly

**Purpose**: A snapshot of the context window assembled for a specific inference call —
which Engrams were included, their weights, and the total token count.

**Typical Body**: `Body::Json` or `Body::Structured`.

**Lifetime**: Transient tier with Linear decay (default 30-minute lifetime — session-scoped).

**Notes**: ContextAssembly Engrams are used for debugging "why did the agent say X" queries
and for context-aware caching.

---

## ModelSelection

**Purpose**: Records which model was selected for a task and why.

**Typical Body**: `Body::Json` containing `{model_id, reason, task_kind, latency_budget_ms}`.

**Lifetime**: Working tier. Model selection records inform the ModelSelection operator's
learning loop.

---

## ErrorRecord

**Purpose**: A structured record of an error event with full context: the error type,
the Engram or operation that failed, the stack trace, and recovery steps taken.

**Typical Body**: `Body::Structured` or `Body::Json`.

**Lifetime**: Working tier. Errors are reviewed during post-mortems and feed into the
Gate pipeline's error rate tracking.

---

## Custom(String)

**Purpose**: Application-specific Kind for use cases not covered by the 15 built-in variants.

**Typical Body**: Application-defined.

**Lifetime**: Defaults to Working tier Demurrage. Creators should override decay explicitly.

**Notes**: The string name should follow the convention `"crate-name/kind-name"` to prevent
collisions. The name is used verbatim as the canonical bytes for the ContentHash.

---

## Open Questions

- Should `Pheromone` be a separate top-level subsystem rather than a Kind? Currently
  modelled as a Kind for simplicity.
- Should `ContextAssembly` reference its component Engrams via `lineage`? Currently
  the component list is stored in the body, not lineage.
