## Intent

Create the narrow acceptance contract that future Roko-agent tasks must satisfy. This batch should establish a real, typed done gate. It should not attempt to implement the whole self-hosting system.

## Implementation Requirements

- Locate the current gate, plan validation, review, and parity tracking surfaces.
- Add or refine a typed acceptance contract that can express:
  - compile/test/lint requirements;
  - no-stub production path requirements;
  - structured agent output requirements;
  - review verdict requirements;
  - retry/reflection/replan requirements;
  - parity ledger evidence requirements.
- Make missing or malformed acceptance data fail closed.
- Add a narrow CLI or library validation path if one already naturally exists.
- Persist or expose enough structured data for later batches to attach outcomes.
- Avoid broad rewrites of unrelated gate or orchestrator code.

## Acceptance

- Future batches can point at this contract instead of relying on prompt-only instructions.
- The contract distinguishes success, failure, blocked, retry, replan, and human-needed outcomes.
- No new production path returns success only because a stub exists.
- Tests cover parsing/validation/fail-closed behavior.
