# RT01 - Structured Review Verdict Parsing and Fail-Closed Contracts

## Read First

- `tmp/architecture-plans/arch-20-orchestrator-gaps.md`
- `tmp/architecture-plans/08-end-to-end-acceptance.md`
- code handling agent review output in `crates/roko-gate`, `crates/roko-orchestrator`, and `crates/roko-cli`
- the RT00 acceptance contract implementation

## Intent

Replace brittle free-text review handling with a structured review verdict contract that orchestration and gates can act on.

## Implementation Requirements

- Find all review verdict paths and any string parsing that decides pass/fail.
- Add a typed `ReviewVerdict`-style structure, or extend the closest existing equivalent.
- Include status, confidence, blocking findings, non-blocking findings, required next action, evidence paths, and raw-output reference.
- Parse agent output into the typed structure using structured APIs where possible.
- Treat missing, ambiguous, contradictory, or unparsable verdicts as failed/needs-human, never as passed.
- Preserve raw review text for audit/debugging.
- Add focused tests for valid verdicts, invalid verdicts, missing fields, and fail-closed behavior.

## Acceptance

- Orchestrator/gate code consumes typed verdict data for pass/fail decisions.
- Free-text-only success is not enough to pass review.
- Failed parsing produces a structured failure outcome.
- The implementation leaves room for later reviewer role/model policy.
