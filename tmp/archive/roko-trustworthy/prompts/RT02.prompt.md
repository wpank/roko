# RT02 - Compile Error Classification and Pre-Agent Cargo-Fix Path

## Read First

- `tmp/architecture-plans/arch-20-orchestrator-gaps.md`
- `tmp/architecture-plans/06-architecture-implementation.md`
- compile/test gate code in `crates/roko-gate`
- agent retry and self-host paths in `crates/roko-agent`, `crates/roko-orchestrator`, and `crates/roko-cli`

## Intent

Before spawning another agent, Roko should classify compile failures and run deterministic, scoped cargo-fix paths where safe.

## Implementation Requirements

- Locate compile/test/lint gate output capture.
- Add structured compile failure classification for at least:
  - syntax/import/type errors;
  - missing dependency/feature errors;
  - borrow/lifetime errors;
  - test expectation failures;
  - external/environment failures;
  - unknown failures.
- Add a safe pre-agent remediation path for deterministic cargo fix or formatting steps where the repo already supports them.
- Ensure remediation records what command ran, what changed, and whether a later agent retry is still needed.
- Do not run broad destructive commands or silently rewrite unrelated files.
- Add tests with representative compiler output fixtures or focused synthetic examples.

## Acceptance

- Compile failures become structured data usable by retries and replanning.
- Deterministic fixes happen before expensive agent retries when safe.
- Unknown or unsafe fixes fail closed into retry/replan rather than pass.
- Logs preserve enough detail to debug the original failure.

## Suggested Verification

```bash
cargo check -p roko-gate -p roko-agent -p roko-cli
cargo test -p roko-gate --test compile_real_project --no-run
cargo test -p roko-cli --test e2e_self_host --no-run
```
