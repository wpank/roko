## Intent

After a gate fails or repeatedly passes, Roko should create structured reflection data and promote repeated lessons into playbook candidates.

## Implementation Requirements

- Find any existing reflection, playbook, or conductor diagnosis surfaces.
- Add a post-gate reflection record with:
  - trigger gate;
  - outcome;
  - failure pattern ids or pass evidence;
  - proposed lesson;
  - confidence/evidence count;
  - admission status.
- Keep reflection separate from automatic policy mutation.
- Add promotion rules for playbook candidates that require repeated evidence.
- Ensure reflections are bounded and audit-friendly.
- Add tests for reflection creation, repeated evidence promotion, and rejection of low-evidence rules.

## Acceptance

- Gate outcomes can create reflection records.
- Playbook rules are candidates until admitted by evidence.
- One failed run cannot silently rewrite global behavior.
- Reflection data can feed future prompt/context policies.
