## Intent

Long self-hosting runs need reusable agent/session paths and robust resume after interruption.

## Implementation Requirements

- Identify the current agent spawn/session lifecycle.
- Add a warm-agent or reusable-session abstraction only where it fits the existing backend model.
- Persist enough invocation/session metadata to resume or diagnose interrupted runs.
- Ensure cancellation and timeout behavior leaves a clear state rather than ambiguous success.
- Add bounded reuse rules so stale context or old prompts cannot silently bleed into a new task.
- Add tests for session metadata, timeout/cancel state, and resume validation.

## Acceptance

- Interruption creates a resumable or clearly failed state.
- Warm reuse is opt-in and scoped by task/session policy.
- Old context cannot leak into a new task without explicit policy.
- The implementation remains compatible with multiple provider backends.
