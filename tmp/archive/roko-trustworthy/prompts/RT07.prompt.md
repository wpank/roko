## Intent

When gates fail in ways that retries cannot solve, Roko should revise the plan rather than rerun the same failing prompt.

## Implementation Requirements

- Locate plan execution, plan validation, and gate failure handling.
- Add a structured `needs_replan` path from gate outcomes/failure classifications.
- Record why replanning is needed, including failure pattern ids and blocking findings.
- Add a plan revision request/record that can be consumed by a future planner agent.
- Ensure simple deterministic failures still retry/fix before replan.
- Add tests for retry-vs-replan selection and plan revision record creation.

## Acceptance

- Gate failures can produce a plan revision signal.
- Replanning is based on structured evidence, not log text alone.
- Replan requests preserve enough context to resume after interruption.
- The self-hosting loop can distinguish retry, replan, blocked, and human-needed states.
