## Intent

Create the observations needed before contextual bandits are introduced. This batch records provider/model outcomes; it should not yet make aggressive adaptive routing decisions.

## Implementation Requirements

- Identify provider/model selection and invocation result paths.
- Add a structured outcome record with provider, model, task type, role id if available, gate result, latency, retry count, and cost/token fields where available.
- Persist or expose rolling pass-rate summaries through existing learning surfaces.
- Avoid global policy mutation in this batch.
- Add tests for outcome recording, aggregation, and missing optional metadata.

## Acceptance

- Model/provider outcomes are available as structured telemetry.
- Gate outcomes can be joined to model/provider choices.
- The data model has stable action ids for future bandits.
- No adaptive policy changes happen without explicit later batch work.

cargo check -p roko-learn -p roko-agent -p roko-serve

cargo test -p roko-agent --test provider_integration --no-run
