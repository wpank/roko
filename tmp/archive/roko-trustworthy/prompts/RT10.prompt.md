# RT10 - Remove Live Mori/Bardo Prompt Leakage from Roko Roles

## Read First

- `tmp/architecture-plans/generalized-cybernetic-agent-architecture-gaps.md`
- relevant role/prompt code in `crates/roko-agent`, `crates/roko-compose`, `crates/roko-orchestrator`, and `crates/roko-cli`
- any role manifest or prompt templates currently used by architect, implementer, scribe, reviewer, or similar roles

## Intent

Roko roles should not depend on live Mori/Bardo-specific prompt paths or hardcoded identity text. Historical references can remain in docs, but runtime role behavior should be Roko-owned and configurable.

## Implementation Requirements

- Search for Mori/Bardo references in runtime role and prompt paths.
- Distinguish historical docs from live runtime behavior.
- Remove or isolate live prompt leakage from Roko role execution.
- Replace leaked prompt text with Roko-neutral role content or manifest references.
- Add regression tests or snapshots that prevent built-in runtime roles from pulling Mori/Bardo prompts.
- Do not delete useful historical documentation unless it is incorrectly wired into runtime behavior.

## Acceptance

- Architect, implementer, scribe, reviewer, and related roles no longer load live Mori/Bardo runtime prompts.
- Runtime code can explain where each role prompt comes from.
- Remaining Mori/Bardo references are docs/reference-only or explicitly isolated.
- Tests cover the runtime leak prevention path.

## Suggested Verification

```bash
cargo check -p roko-compose -p roko-orchestrator -p roko-agent -p roko-cli
cargo test -p roko-agent --test role_tools --no-run
cargo test -p roko-compose --lib --no-run
```
