# RT11 - RoleProfile and PromptPolicy Manifest Contracts

## Read First

- `tmp/architecture-plans/generalized-cybernetic-agent-architecture-gaps.md`
- RT10 leak-removal implementation
- role/prompt/config code in `crates/roko-core`, `crates/roko-compose`, and `crates/roko-agent`
- config docs in `tmp/architecture-plans/arch-16-config.md`

## Intent

Introduce the manifest contracts that make roles and prompts configurable, composable, and versioned.

## Implementation Requirements

- Add or refine a `RoleProfile` type that describes role id, version, objectives, responsibilities, default prompt policy, context policy, tools/capabilities, output schema, and gate expectations.
- Add or refine a `PromptPolicy` type that describes ordered sections, inclusion rules, budgets, provenance, experiment ids, and fallback behavior.
- Implement manifest loading/validation using structured parsing.
- Add compatibility with existing built-in roles without requiring a giant migration in this batch.
- Add tests for valid manifests, invalid manifests, defaults, version handling, and fail-closed validation.

## Acceptance

- Roles and prompt policies can be represented outside hardcoded Rust match arms.
- Invalid manifests fail with actionable errors.
- Built-in roles have a path to migrate to manifests in RT12.
- The types are stable enough for `CognitiveWorkspace` and context bidders.

cargo check -p roko-core -p roko-compose -p roko-agent
