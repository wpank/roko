# RT12 - Manifest-Backed Built-In Roles

## Read First

- `tmp/architecture-plans/generalized-cybernetic-agent-architecture-gaps.md`
- RT10 and RT11 implementations
- built-in role code/templates in `crates/roko-agent`, `crates/roko-compose`, and `crates/roko-cli`

## Intent

Move built-in roles such as architect, implementer, scribe, reviewer, and planner onto manifest-backed `RoleProfile` and `PromptPolicy` definitions.

## Implementation Requirements

- Inventory current built-in roles and prompt sources.
- Create manifest entries for the core roles already used by self-hosting flows.
- Make runtime role loading prefer manifests, with clear fallback behavior only where needed for compatibility.
- Include role output schema expectations and gate expectations where the current behavior already implies them.
- Add tests that load each built-in manifest and verify no live Mori/Bardo prompt source is used.
- Keep the first migration narrow; do not migrate every possible future role if it risks broad churn.

## Acceptance

- Architect, implementer, scribe, and reviewer-style roles can be loaded from manifests.
- The role loader exposes role id/version and prompt policy id.
- Broken role manifests fail closed.
- Existing self-hosting flows still compile and can resolve their roles.

## Suggested Verification

cargo check -p roko-agent -p roko-compose -p roko-cli
cargo test -p roko-agent --test role_tools --no-run
cargo test -p roko-cli --test agent_config --no-run
