# RT13 - CognitiveWorkspace Audit Object and Context Provenance

## Read First

- `tmp/architecture-plans/generalized-cybernetic-agent-architecture-gaps.md`
- RT11 manifest contracts
- prompt/context code in `crates/roko-compose`
- orchestration code in `crates/roko-orchestrator`
- core types in `crates/roko-core`

## Intent

Add a `CognitiveWorkspace`-style audit object for each agent invocation so context, role, prompt, model, tools, outputs, and gates can be inspected together.

## Implementation Requirements

- Add or refine an invocation audit structure that records:
  - task contract;
  - role profile id/version;
  - prompt policy id/version;
  - context policy id/version when available;
  - included context sections with provenance and budget;
  - rejected context candidates with reasons;
  - provider/model choice;
  - capability/tool grants;
  - output parse result;
  - gate outcomes and reward observations.
- Wire the object at the natural compose/orchestrator boundary.
- Persist, emit, or expose the object through an existing event/substrate path.
- Add tests for provenance retention and redaction/boundedness where applicable.

## Acceptance

- One agent invocation can be audited without scraping logs.
- Context inclusion decisions are visible.
- Later telemetry and bidder batches can attach outcomes to this object.
- The object is structured and serializable.

## Suggested Verification

cargo check -p roko-compose -p roko-core -p roko-orchestrator
cargo test -p roko-compose --lib --no-run
cargo test -p roko-core --lib --no-run
