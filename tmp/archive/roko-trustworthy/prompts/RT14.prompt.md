# RT14 - ContextEngine Bidder Registry with Cold-Start Policies

## Read First

- `tmp/architecture-plans/generalized-cybernetic-agent-architecture-gaps.md`
- RT05 scoped context implementation
- RT13 `CognitiveWorkspace` implementation
- context code in `crates/roko-compose`, `crates/roko-learn`, and `crates/roko-orchestrator`

## Intent

Create the first composable context bidder registry. Start with deterministic cold-start policies. Adaptive learning comes later.

## Implementation Requirements

- Introduce a context bidder trait/interface or align with an existing one.
- Add a registry that can combine candidates from multiple bidders.
- Implement at least static bidders for:
  - task contract requirements;
  - relevant docs/source maps;
  - recent failure patterns if RT03 data is present;
  - role prompt/policy requirements.
- Add deterministic scoring/allocation with budgets and clear rejection reasons.
- Ensure all selected/rejected candidates are recorded into `CognitiveWorkspace`.
- Add tests for registry ordering, budget behavior, and provenance.

## Acceptance

- Context assembly is no longer a single hardcoded blob path.
- Bidders are composable and can be registered independently.
- Cold-start policy is deterministic and testable.
- The design leaves a clean seam for `LearningBidder` in RT16.
