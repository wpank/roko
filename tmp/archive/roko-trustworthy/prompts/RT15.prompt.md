## Read First

- telemetry/learning code in `crates/roko-learn`

- API/job outcome surfaces in `crates/roko-serve`

## Intent

Persist enough prompt/context section metadata and gate outcomes to support later adaptive policy. This batch creates observations, not adaptive mutation.

## Implementation Requirements

- Add stable identifiers for prompt sections and context sections.
- Record section inclusion, budget use, source/provenance, and experiment id if present.
- Join section metadata to gate outcomes and review verdicts.
- Add outcome events that can be aggregated by section, role, provider/model, and task type.
- Keep raw prompt text out of metrics unless explicitly needed and safe.
- Add tests for section metadata recording and outcome aggregation.

## Acceptance

- A gate result can be traced back to the prompt/context sections included in the invocation.
- Telemetry has stable action ids for future bandits.
- Sensitive or huge text is not blindly copied into learning metrics.
- No adaptive policy change is made yet.
