# RT16 - LearningBidder Posteriors and Attention Policy Controls

## Read First

- `tmp/architecture-plans/generalized-cybernetic-agent-architecture-gaps.md`
- RT14 bidder registry
- RT15 telemetry implementation
- learning/context code in `crates/roko-learn`, `crates/roko-compose`, and `crates/roko-orchestrator`

## Intent

Use accumulated outcomes to bias context selection. Start conservatively with posteriors and attention controls, not full global policy rewriting.

## Implementation Requirements

- Add a `LearningBidder` or equivalent bidder that reads section effectiveness summaries.
- Use simple Beta/Thompson-style or count-based posteriors where the data supports it.
- Keep cold-start fallback deterministic.
- Add guardrails:
  - minimum evidence before influence;
  - max budget share;
  - decay or recency window where appropriate;
  - safe fallback when telemetry is missing.
- Record posterior values or decision reasons into `CognitiveWorkspace`.
- Add tests for cold start, evidence threshold, posterior bias, and fallback behavior.

## Acceptance

- Learning can influence context selection in a bounded, auditable way.
- The bidder can be disabled or falls back safely.
- Low-evidence observations do not dominate prompt assembly.
- Decision metadata is persisted for later debugging.
