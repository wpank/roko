## Intent

Make the runtime/control plane durable enough for long self-hosting runs: restart, observe, cancel, resume, and configure.

## Implementation Requirements

- Identify the most critical runtime durability gap left after RT06.
- Add or refine durable state for running tasks/jobs/sessions.
- Ensure cancellation, timeout, and process supervision have clear terminal states.
- Expose enough state through CLI/API/TUI-adjacent surfaces for operators to diagnose a long run.
- Validate config loading for the new runtime/control behavior.
- Add tests for restart/resume metadata, cancellation, and state exposure.

## Acceptance

- Long runs have observable status and terminal states.
- Resume validation catches incompatible or stale state.
- Runtime config is explicit and validated.
- Operators can identify what is running, failed, blocked, or waiting.
