# RT19 - Finish the Trustworthy Self-Hosting Workflow Loop

## Read First

- `tmp/architecture-plans/05-self-hosting.md`
- `tmp/architecture-plans/08-end-to-end-acceptance.md`
- all implemented RT00-RT18 surfaces relevant to gates, roles, prompts, context, telemetry, and resume
- self-host paths in `crates/roko-cli`, `crates/roko-orchestrator`, `crates/roko-gate`, `crates/roko-compose`, `crates/roko-learn`, and `crates/roko-agent`

## Intent

Wire the pieces into an end-to-end self-hosting loop that can generate a plan, dispatch agents, assemble context, parse outputs, run gates, retry/replan, resume, and record learning signals.

## Implementation Requirements

- Identify the narrowest existing self-host entry point.
- Wire the RT00 acceptance contract into task completion.
- Use manifest-backed roles and scoped context assembly where available.
- Ensure agent output is parsed structurally before gates decide success.
- Ensure gate failures flow into retry, reflection, replan, or human-needed outcomes.
- Ensure outcome telemetry is persisted for later learning.
- Add an end-to-end smoke test or no-run compile target that exercises the full workflow with mocks.

## Acceptance

- There is a coherent self-hosting path, not just independent parts.
- The path can resume or clearly fail after interruption.
- Failures do not produce false success.
- The path emits enough structured evidence for future Roko-agent implementation batches.
