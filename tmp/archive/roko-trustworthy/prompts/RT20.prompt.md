## Intent

Prepare the queue that Roko can use to implement the rest of the architecture without turning the backlog into one giant unsupervised wave.

## Implementation Requirements

- Convert `06-architecture-implementation.md` into task packets or a queue format that the self-hosting path can consume.
- Order packets by:
  1. config/schema and architecture contracts;
  2. agent runtime;
  3. extensions/connectors/feeds;
  4. gateway/model routing;
  5. knowledge/learning/neuro;
  6. gates/evals/arenas;
  7. chain/registries;
  8. groups/coordination;
  9. dashboard projections;
  10. visual composition and authoring.
- Each packet needs an acceptance contract, source docs, likely crates, verification commands, and dependency metadata.
- Add validation for the queue format if the repo has a natural place for it.
- Do not implement the whole queue in this batch.

## Acceptance

- Roko can enumerate architecture task packets in dependency order.
- Each packet is small enough for a future agent run.
- Acceptance gates are attached to each packet.
- The queue integrates with the self-hosting path or has a clear CLI/API entry point.
