# RT21 - Docs Parity Enforcement After Runtime Gates

## Read First

- `tmp/architecture-plans/07-docs-parity-closure.md`
- RT20 queue handoff
- docs parity runners in `tmp/docs-parity2` and `tmp/refinement-audit-runner`
- plan/gate code in `crates/roko-cli`, `crates/roko-gate`, and `crates/roko-orchestrator`

## Intent

Docs parity should become enforceable by the self-hosting runtime instead of being a loose manual checklist.

## Implementation Requirements

- Identify existing parity ledger/checklist surfaces.
- Add or refine a parity enforcement packet format with doc requirement id, implementation evidence, test/gate evidence, and status.
- Wire parity checks into plan validation or gate evaluation where appropriate.
- Ensure missing parity evidence fails as `blocked` or `needs_work`, not success.
- Add tests for parity row validation and missing-evidence failure.
- Keep dashboard/product parity late unless backend projection requirements are needed by the runtime.

## Acceptance

- Implemented doc requirements can be mapped to evidence.
- Missing evidence is detectable by gates.
- Future architecture packets can require parity closure.
- Docs parity does not bypass the RT00 acceptance contract.

## Suggested Verification

```bash
cargo check -p roko-cli -p roko-gate -p roko-orchestrator
cargo test -p roko-cli --test plan_validate --no-run
cargo test -p roko-gate --lib --no-run
```
