## Intent

Make dashboard/product work consume stable backend projections. Do not let dashboard needs force unstable self-hosting internals.

## Implementation Requirements

- Identify projection contracts needed by dashboard docs.
- Add or refine backend projection types/endpoints for agent state, plan state, gate state, and learning/policy state where already supported by runtime data.
- Keep projection contracts versioned and documented.
- Add tests for API/projection serialization and missing-state behavior.
- Do not start a broad frontend implementation in this batch.

## Acceptance

- Dashboard-facing surfaces read from stable runtime/orchestrator state.
- Product surfaces do not depend on private prompt/context internals.
- Missing data is represented explicitly rather than faked.
- API tests or serialization tests cover the projection contract.
