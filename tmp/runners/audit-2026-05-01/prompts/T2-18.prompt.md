## Exact changes (one commit per section, but can be batched in one PR)

For **each** of the seven sections, do:

### 1. Delete the struct from its owner file

E.g., for `OneirographyConfig` in `crates/roko-core/src/config/tools.rs`:

Find:
```rust
#[derive(Debug, Clone, Serialize, Deserialize)]
#[serde(default)]
pub struct OneirographyConfig {
    pub field_a: ...,
    // ...
}

impl Default for OneirographyConfig {
    fn default() -> Self { ... }
}
```

Delete entirely.

### 2. Delete the field on `RokoConfig`

`crates/roko-core/src/config/schema.rs`:

Find:
```rust
pub struct RokoConfig {
    // ...
    pub oneirography: OneirographyConfig,
    // ...
}
```

Delete `pub oneirography: OneirographyConfig,`.

In the `Default` impl for `RokoConfig`:
```rust
impl Default for RokoConfig {
    fn default() -> Self {
        Self {
            // ...
            oneirography: OneirographyConfig::default(),
            // ...
        }
    }
}
```

Delete the `oneirography: ...` line.

### 3. Remove `use` import

In `schema.rs`, find and remove:
```rust
use crate::config::tools::OneirographyConfig;
```

### 4. Remove from hot-reload diff

`crates/roko-core/src/config/hot_reload.rs`:

Find any block that compares the field, e.g.:
```rust
if old.oneirography != new.oneirography {
    diff.push(ConfigChange::Oneirography);
}
```

Delete that block. Also delete the `ConfigChange::Oneirography` variant if it exists, and any consumer of that variant.

### 5. Remove from compat migration

`crates/roko-core/src/config/compat.rs`:

Find any migration entry mentioning the section, e.g.:
```rust
// Migrate v1 → v2: rename old.oneiro to oneirography
```

Delete those lines. If a `match version { ... }` arm is empty after removal, remove the arm too.

### 6. Remove `[oneirography]` block from `roko.toml`

Find the `[oneirography]` section header and all keys under it, until the next section header or EOF. Delete.

### 7. Repeat for the remaining six sections

`DemurrageConfig`, `AttentionConfig`, `ImmuneConfig`, `TemporalConfig`, `GoalsConfig` all live in `crates/roko-core/src/config/learning.rs`. Same pattern; one struct + Default at a time.

`EnergyConfig` lives in `crates/roko-core/src/config/budget.rs`. Same pattern.
