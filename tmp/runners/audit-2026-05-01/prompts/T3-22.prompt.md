# T3-22: Flip ServeAuthConfig::default() enabled to true

## Task
Change `ServeAuthConfig::default().enabled` from `false` to `true`. Update the `roko init` config template to set it back to `false` for local development with a comment. Update tests that assume default-off.

## Runner Context
Runner audit-2026-05-01, group T3. No dependencies. Wave 1.

## Source plan
`tmp/subsystem-audits/implementation-plans/13-tier3-security-hardening.md` § T3-22.

## Why
Defense in depth. `validate_bind_safety` at `crates/roko-serve/src/lib.rs:641` already refuses public binds without auth, but a config edit could ship `auth.enabled = false` and `acknowledge_public_risk = true` together. Making the default `true` means a fresh user must opt out of auth, not opt in.
