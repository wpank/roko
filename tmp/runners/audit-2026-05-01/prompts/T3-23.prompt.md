# T3-23: Add tower-governor rate limiting

## Task
Add `tower-governor` rate limiting to `roko-serve`. Apply a global cap (100 req/sec/client, burst 200) and per-route caps for cost-amplifying endpoints.

## Runner Context
Runner audit-2026-05-01, group T3. No dependencies. Wave 1.

## Source plan
`tmp/subsystem-audits/implementation-plans/13-tier3-security-hardening.md` § T3-23.

## Why
A single client can saturate inference cost or fill workspace disk by spamming `POST /api/inference/complete` or `POST /api/agents/register`. No rate limit exists today.

Identify clients by `X-API-Key` when present (so legitimate API users get individual quotas), fall back to peer IP for unauthenticated requests.

### 2. Add helpers to `crates/roko-serve/src/routes/middleware.rs`

```rust
use std::sync::Arc;
use tower_governor::{governor::GovernorConfigBuilder, key_extractor::KeyExtractor, GovernorLayer};
use governor::clock::DefaultClock;
use axum::http::HeaderValue;

/// Identify rate-limit clients by X-API-Key when present, else by peer IP.
#[derive(Clone, Default)]
pub struct ApiKeyOrPeerIp;

impl KeyExtractor for ApiKeyOrPeerIp {
    type Key = String;

    fn extract<B>(&self, req: &axum::http::Request<B>) -> Result<Self::Key, tower_governor::GovernorError> {
        if let Some(key) = req.headers().get("x-api-key").and_then(|h| h.to_str().ok()) {
            return Ok(format!("apikey:{}", key));
        }
        // axum's ConnectInfo extension carries peer addr if the router was configured for it.
        if let Some(addr) = req.extensions().get::<axum::extract::ConnectInfo<std::net::SocketAddr>>() {
            return Ok(format!("ip:{}", addr.ip()));
        }
        Ok("anon".into())
    }
}

/// Global rate limiter: 100 req/sec/client with burst 200.
pub fn global_rate_limit_layer() -> GovernorLayer<ApiKeyOrPeerIp, DefaultClock> {
    let cfg = Arc::new(
        GovernorConfigBuilder::default()
            .key_extractor(ApiKeyOrPeerIp)
            .per_second(100)
            .burst_size(200)
            .finish()
            .expect("valid global governor config"),
    );
    GovernorLayer { config: cfg }
}

/// Per-route rate limiter for expensive endpoints.
pub fn expensive_rate_limit_layer(per_minute: u32) -> GovernorLayer<ApiKeyOrPeerIp, DefaultClock> {
    let per_second = ((per_minute as u64) / 60).max(1);
    let burst = ((per_minute / 10).max(2)) as u32;
    let cfg = Arc::new(
        GovernorConfigBuilder::default()
            .key_extractor(ApiKeyOrPeerIp)
            .per_second(per_second)
            .burst_size(burst)
            .finish()
            .expect("valid per-route governor config"),
    );
    GovernorLayer { config: cfg }
}
```

### 3. Apply global layer in `crates/roko-serve/src/routes/mod.rs`

Find the existing layer chain near the bottom of `build_router` (around line 183, where `RequestBodyLimitLayer` lives):

```rust
router
    .layer(RequestBodyLimitLayer::new(32 * 1024 * 1024))
    .layer(TraceLayer::new_for_http())
    .layer(cors)
    .with_state(state)
```

Add the global rate limit:

```rust
router
    .layer(middleware::global_rate_limit_layer())
    .layer(RequestBodyLimitLayer::new(32 * 1024 * 1024))
    .layer(TraceLayer::new_for_http())
    .layer(cors)
    .with_state(state)
```

(Layer order is bottom-up: outer-most first. Putting rate limit before body limit is fine; both run before route handlers.)

## Do NOT

- Do NOT use `tower::limit::RateLimitLayer` (global token bucket; doesn't identify clients).
- Do NOT apply per-route caps to read-only `GET` endpoints (no cost amplification).
- Do NOT lower the global cap below 100 req/sec — at that point legitimate UI traffic suffers.
- Do NOT add per-IP-only extraction; behind a load balancer all clients share one IP.
- Do NOT silence rate limit errors — let them return 429.
- Do NOT add rate limits to `/health` (top-level) or `/api/health`.
- Do NOT make rate limits configurable from `roko.toml` in this batch (separate task if desired).
