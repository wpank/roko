# T3-27: Fix path traversal + TOML injection in agent creation

## Task
Fix two security issues in `POST /api/agents/create` (`crates/roko-serve/src/routes/agents.rs:605`):

(a) **Path traversal**: `req.name` is unvalidated and used directly in `state.workdir.join(".roko").join("agents").join(&req.name)`. `name = "../../../etc"` writes arbitrary paths.

(b) **TOML injection**: the manifest is `format!`-built with `toml_quote(prompt)`. While `toml_quote` escapes `"`, it doesn't prevent table injection (a `[malicious]\n` line in the prompt).

## Exact changes

### 1. Add `validate_workspace_dir_segment` in `crates/roko-serve/src/error.rs`

Add (or extend) a validator alongside `validate_path_segment`:

```rust
/// Validate a name component for use as a directory name inside the workspace.
/// Rejects: empty, `.`, `..`, leading dot, slashes, backslashes, control characters.
pub fn validate_workspace_dir_segment(name: &str, what: &str) -> Result<(), ApiError> {
    if name.is_empty()
        || name == "."
        || name == ".."
        || name.starts_with('.')
        || name.contains('/')
        || name.contains('\\')
        || name.chars().any(|c| c.is_control())
    {
        return Err(ApiError::bad_request(format!(
            "invalid {what}: {:?} (only alphanumeric, dashes, underscores allowed)",
            name
        )));
    }
    Ok(())
}
```

### 2. Define `validate_domain_key` in `crates/roko-serve/src/routes/agents.rs`

The TOML key `[core.domain.{domain}]` is a path segment too; restrict it:

```rust
fn validate_domain_key(s: &str) -> Result<(), ApiError> {
    let valid = !s.is_empty()
        && s.len() <= 32
        && s.chars().next().is_some_and(|c| c.is_ascii_lowercase())
        && s.chars().all(|c| c.is_ascii_lowercase() || c.is_ascii_digit() || c == '_' || c == '-');
    if !valid {
        return Err(ApiError::bad_request(format!(
            "invalid domain {:?} (must match [a-z][a-z0-9_-]{{0,31}})",
            s
        )));
    }
    Ok(())
}
```

### 3. Define structured manifest types

use std::collections::BTreeMap;

}

    prompt: String,

(`BTreeMap<String, BTreeMap<...>>` produces `[core.domain.<key>]` as a TOML table; the inner empty map renders the body cleanly.)

### 4. Rewrite `create_agent`

Replace the entire body up through `tokio::fs::write(&manifest_path, ...)`:

```rust
async fn create_agent(
    State(state): State<Arc<AppState>>,
    ValidJson(req): ValidJson<CreateAgentRequest>,
) -> Result<impl IntoResponse, ApiError> {
    crate::error::validate_workspace_dir_segment(&req.name, "agent name")?;
    validate_domain_key(&req.domain)?;

    let agents_dir = state.workdir.join(".roko").join("agents").join(&req.name);
    let manifest_path = agents_dir.join("manifest.toml");

    if manifest_path.exists() {
        return Err(ApiError::conflict(format!(
            "agent '{}' already exists at {}",
            req.name,
            manifest_path.display()
        )));
    }

    let prompt = req
        .prompt
        .as_deref()
        .unwrap_or("You are a helpful autonomous agent.")
        .to_string();

    let mut domain = BTreeMap::new();
    domain.insert(req.domain.clone(), BTreeMap::new());

    let manifest = AgentManifest {
        schema_version: 1,
        core: AgentCoreManifest {
            prompt,
            mode: "self_hosted".to_string(),
            domain,
        },
    };

    let manifest_toml = toml::to_string_pretty(&manifest)
        .map_err(|e| ApiError::internal(format!("serialize manifest: {e}")))?;

    tokio::fs::create_dir_all(&agents_dir)
        .await
        .map_err(|e| ApiError::internal(format!("create agent dir: {e}")))?;

    // Defense in depth: after creation, canonicalize and assert containment.
    let workspace_root = state
        .workdir
        .canonicalize()
        .map_err(|e| ApiError::internal(format!("canonicalize workspace: {e}")))?;
    let canonical_dir = agents_dir
        .canonicalize()
        .map_err(|e| ApiError::internal(format!("canonicalize agent dir: {e}")))?;
    if !canonical_dir.starts_with(&workspace_root) {
        // Best-effort cleanup
        let _ = tokio::fs::remove_dir_all(&agents_dir).await;
        return Err(ApiError::bad_request("agent path escaped workspace"));
    }

    tokio::fs::write(&manifest_path, &manifest_toml)
        .await
        .map_err(|e| ApiError::internal(format!("write manifest: {e}")))?;

    // ... rest of the existing function (capabilities, registration, chain dual-write) unchanged
}
```

## Acceptance Criteria

- `validate_workspace_dir_segment` rejects path-traversal patterns.
- `validate_domain_key` restricts TOML key to safe charset.
- Manifest is built via `toml::to_string_pretty(&AgentManifest)`.
- Post-create canonicalize containment check rejects escaped paths and cleans up.
- 3 regression tests added (traversal, invalid domain, table injection).
- `toml_quote` either deprecated or deleted.

## Do NOT

- Do NOT canonicalize `agents_dir` BEFORE `create_dir_all` — it doesn't exist yet at that point.
- Do NOT use `Path::strip_prefix` for the containment check — it doesn't resolve symlinks.
- Do NOT loosen `validate_workspace_dir_segment` to admit unusual but "common" agent names.
- Do NOT keep the `format!`-based manifest "as a fallback."
- Do NOT widen `validate_domain_key` to accept dots, slashes, or uppercase.
- Do NOT skip the cleanup `remove_dir_all` on containment failure (avoid leaving junk).
- Do NOT add a `--unsafe` flag to bypass validation.
