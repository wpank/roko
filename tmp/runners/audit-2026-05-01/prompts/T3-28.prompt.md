# T3-28: Restrict CORS allow_methods + allow_headers to explicit lists

## Task
Replace `allow_methods(Any).allow_headers(Any)` in `cors_layer` (and the duplicate `build_cors_layer`) with explicit lists of methods and headers we actually use.

## Runner Context
Runner audit-2026-05-01, group T3. No dependencies. Wave 1.

## Source plan
`tmp/subsystem-audits/implementation-plans/13-tier3-security-hardening.md` § T3-28.

## Why
`allow_methods(Any)` accepts unusual verbs (TRACE, CONNECT) we don't intend to support. `allow_headers(Any)` accepts arbitrary header names that the server never reads. Restricting both to known-good lists tightens the cross-origin surface.

Rewrite `cors_layer`:

```rust
pub fn cors_layer(cors_origins: &[String], unsafe_public: bool) -> CorsLayer {
    let base = CorsLayer::new()
        .allow_methods(ALLOWED_METHODS.to_vec())
        .allow_headers(allowed_headers());

    if !cors_origins.is_empty() {
        let allowed: Vec<axum::http::HeaderValue> = cors_origins
            .iter()
            .filter_map(|o| o.parse().ok())
            .collect();
        return base.allow_origin(allowed);
    }

    if unsafe_public {
        if UNSAFE_PUBLIC_CORS_WARNING.set(()).is_ok() {
            tracing::warn!(
                "CORS is unrestricted (allow *) because server.unsafe_public_cors = true and no \
                 cors_origins are configured. Set cors_origins to limit access."
            );
        }
        // Even for permissive, restrict methods/headers — we never accept TRACE/CONNECT.
        return base.allow_origin(AllowOrigin::any());
    }

    base.allow_origin(AllowOrigin::predicate(
        |origin: &axum::http::HeaderValue, _parts: &axum::http::request::Parts| {
            match origin.to_str() {
                Ok(origin) => is_local_origin(origin),
                Err(_) => false,
            }
        },
    ))
}
```

## Acceptance Criteria

- `cors_layer` returns a `CorsLayer` with explicit method + header lists.
- `build_cors_layer` (if it survives) uses the same lists.
- `CorsLayer::permissive()` is no longer used (replaced with explicit method/header allowlist + `AllowOrigin::any()` only when `unsafe_public_cors = true`).
- Two regression tests for preflight rejection of TRACE method and a custom header.

## Do NOT

- Do NOT include `TRACE`, `CONNECT`, or other unusual methods in `ALLOWED_METHODS`.
- Do NOT include wildcard headers in `allowed_headers`.
- Do NOT remove `OPTIONS` from `ALLOWED_METHODS` (preflight needs it).
- Do NOT add `Access-Control-Allow-Credentials: true` — out of scope.
- Do NOT change the loopback predicate (`is_local_origin`).
- Do NOT add origins to the allowlist (`cors_origins` is config-driven).
- Do NOT call `CorsLayer::permissive()` — it bypasses our restrictions.
