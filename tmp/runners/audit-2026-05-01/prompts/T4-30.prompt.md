### 6. Update `RoutingObservationSink::on_event`

`crates/roko-cli/src/runtime_feedback/routing.rs:61-82`:

```rust
async fn on_event(&self, event: &FeedbackEvent) -> Result<(), anyhow::Error> {
    let FeedbackEvent::TaskCompleted {
        outcome,
        model_source,
        succeeded,
        ..
    } = event
    else {
        return Ok(());
    };

    if matches!(model_source, ModelChoiceSource::Override) {
        // Operator overrides bypass router learning; record dampened.
        self.router.record_override_outcome(&outcome.model, *succeeded);
        return Ok(());
    }

    if let Some(ctx) = outcome.routing_context.as_ref() {
        // Real contextual update.
        self.router.observe_multi_objective(
            ctx,
            crate::dispatch::routing_outcome_from(outcome, *succeeded),
        );
    } else {
        // Confidence-only fallback when context unavailable.
        self.router
            .record_confidence_outcome(&outcome.model, *succeeded);
    }
    Ok(())
}
```

The `routing_outcome_from` helper builds a `RoutingOutcome` (whatever the router expects: success bool, latency, cost, quality). If the router accepts a simpler `(outcome, succeeded)` tuple, use that.
