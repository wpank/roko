# T4-31a: Anthropic provider — preserve UsageObservation

## Task
Migrate the Anthropic provider's usage parsing to `UsageObservation` so absent usage stays `None` and `Some(0)` stays zero.

## Runner Context
Runner audit-2026-05-01, group T4. No dependencies. Wave 1. Sister batches: T4-31b/c/d/e (other providers).

## Source plan
`tmp/subsystem-audits/implementation-plans/14-tier4-feedback-loops.md` § T4-31.

## Why
Today the Anthropic adapter converts missing fields to `0`, poisoning cost telemetry. `UsageObservation` (in `crates/roko-core/src/usage.rs`) has `Option<u64>` fields that distinguish "absent" from "zero." Preserve the distinction.

## Reference template

`crates/roko-agent/src/translate/openai.rs` is the **canonical example** that already uses `UsageObservation` correctly. Read it before changing anything; follow the same pattern.

## Anthropic usage shapes

Non-streaming response:

```json
{
  "id": "...",
  "content": [{"type": "text", "text": "..."}],
  "stop_reason": "end_turn",
  "usage": {
    "input_tokens": 100,
    "output_tokens": 50
  }
}
```

Streaming `message_delta` event:

```json
{"type": "message_delta", "delta": {"stop_reason": "end_turn"}, "usage": {"output_tokens": 50}}
```

Note: streaming `message_start` may carry `input_tokens`; `message_delta` carries `output_tokens`. Combining them needs a state machine — preserve the existing logic, only change how `Usage` becomes `UsageObservation`.

## Exact changes

### 1. Replace `Usage` with `UsageObservation` at parser exits

For non-stream `parse_response` in `anthropic.rs`:

Before:

```rust
let usage = response.usage.map(|u| Usage {
    input_tokens: u.input_tokens.unwrap_or(0),
    output_tokens: u.output_tokens.unwrap_or(0),
}).unwrap_or_default();
```

After:

```rust
use roko_core::usage::UsageObservation;

let usage = response.usage.map(|u| UsageObservation {
    input_tokens: u.input_tokens,    // Option<u64> stays Option<u64>
    output_tokens: u.output_tokens,
    total_tokens: match (u.input_tokens, u.output_tokens) {
        (Some(i), Some(o)) => Some(i + o),
        _ => None,
    },
    cost_usd: None,                  // computed downstream from token counts
});  // overall: Option<UsageObservation>; absent block → None
```

### 2. Streaming: combine `message_start` + `message_delta` into Optional fields

In `anthropic_stream.rs`, find the state machine that combines per-event partial usage into a final usage. Replace its accumulator type with `UsageObservation`:

```rust
// Initialize accumulator
let mut accumulated = UsageObservation::default();   // all None

// On message_start with usage:
if let Some(u) = event_msg_start_usage {
    accumulated.input_tokens = u.input_tokens;
    accumulated.output_tokens = u.output_tokens.or(accumulated.output_tokens);
}

// On message_delta with usage:
if let Some(u) = event_msg_delta_usage {
    if let Some(o) = u.output_tokens { accumulated.output_tokens = Some(o); }
    if let Some(i) = u.input_tokens  { accumulated.input_tokens  = Some(i); }
}

// At final completion, compute total if both halves present:
accumulated.total_tokens = match (accumulated.input_tokens, accumulated.output_tokens) {
    (Some(i), Some(o)) => Some(i + o),
    _ => None,
};
```

Use a type that carries `Option<u64>` for the per-event partial. If the existing intermediate type uses `u64`, replace with `Option<u64>` or wrap with `Some(...)` only at known sources.

### 3. Update `ModelCallResponse` consumer

`ModelCallResponse` (in `roko-agent/src/model_call_service.rs` or similar) likely already accepts `Option<UsageObservation>`. If it accepts `Usage`, that's `Option<UsageObservation>`. Make sure the Anthropic adapter returns the new type.

If a compatibility shim wraps the old `Usage` type, **leave the shim** — other adapters still produce `Usage`. T4-31b through T4-31e migrate them; only after all providers migrate can the shim be deleted.
