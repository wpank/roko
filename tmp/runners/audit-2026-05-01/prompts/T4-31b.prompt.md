# T4-31b: Ollama provider — preserve UsageObservation

## Task
Migrate the Ollama provider's usage parsing to `UsageObservation`.

## Runner Context
Runner audit-2026-05-01, group T4. No dependencies. Wave 1.

## Source plan
`tmp/subsystem-audits/implementation-plans/14-tier4-feedback-loops.md` § T4-31.

## Why
Ollama returns its own field names (`prompt_eval_count`, `eval_count`) that map to input/output tokens. Today the adapter probably collapses to the legacy `Usage` with `0` defaults; switch to `UsageObservation` preserving `Option`.

## Reference template

`crates/roko-agent/src/translate/openai.rs` and the (now-migrated) `crates/roko-agent/src/providers/anthropic.rs` (T4-31a).

## Read first

```bash
ls crates/roko-agent/src/providers/ollama*.rs
rg 'prompt_eval_count|eval_count|fn parse' crates/roko-agent/src/providers/ollama*.rs -n
```

Identify the parser entry point. Ollama responses look like:

```json
{
  "model": "llama3.2",
  "created_at": "...",
  "message": {"role": "assistant", "content": "..."},
  "done": true,
  "prompt_eval_count": 100,
  "eval_count": 50,
  "total_duration": 12345
}
```

`prompt_eval_count` → input tokens. `eval_count` → output tokens. Either may be absent in non-final stream events.

## Exact changes

### 1. Replace `Usage` construction with `UsageObservation`

For non-stream (`done: true` event) and final stream event:

```rust
use roko_core::usage::UsageObservation;

let usage = if response.prompt_eval_count.is_none() && response.eval_count.is_none() {
    None
} else {
    let input = response.prompt_eval_count;   // Option<u64>
    let output = response.eval_count;         // Option<u64>
    let total = match (input, output) {
        (Some(i), Some(o)) => Some(i + o),
        _ => None,
    };
    Some(UsageObservation {
        input_tokens: input,
        output_tokens: output,
        total_tokens: total,
        cost_usd: None,
    })
};
```

If the response struct currently has `prompt_eval_count: u64` (not `Option<u64>`), change the deserialization struct to `Option<u64>`:

```rust
#[derive(Deserialize)]
struct OllamaResponse {
    // ... other fields
    #[serde(default)]
    prompt_eval_count: Option<u64>,
    #[serde(default)]
    eval_count: Option<u64>,
}
```

### 2. Streaming: only the final event has both counts

For streaming, accumulate per event:

```rust
let mut accumulator = UsageObservation::default();   // all None
// per stream event:
if let Some(p) = event.prompt_eval_count {
    accumulator.input_tokens = Some(p);
}
if let Some(e) = event.eval_count {
    accumulator.output_tokens = Some(e);
}
// On `done: true`, finalize:
accumulator.total_tokens = match (accumulator.input_tokens, accumulator.output_tokens) {
    (Some(i), Some(o)) => Some(i + o),
    _ => None,
};
let final_usage = if accumulator.input_tokens.is_some() || accumulator.output_tokens.is_some() {
    Some(accumulator)
} else {
    None
};
```

### 3. Tests

In `crates/roko-agent/src/providers/ollama.rs` (or sibling `tests/`):

```rust
#[test]
fn ollama_parser_preserves_absent_eval_counts() {
    let body = r#"{
        "model": "llama3.2",
        "created_at": "2026-05-01T10:00:00Z",
        "message": {"role": "assistant", "content": "hi"},
        "done": true
    }"#;
    let parsed = parse_response(body).unwrap();
    assert!(parsed.usage.is_none());
}

#[test]
fn ollama_parser_distinguishes_zero_counts() {
    let body = r#"{
        "model": "llama3.2",
        "created_at": "2026-05-01T10:00:00Z",
        "message": {"role": "assistant", "content": "hi"},
        "done": true,
        "prompt_eval_count": 0,
        "eval_count": 0
    }"#;
    let parsed = parse_response(body).unwrap();
    let u = parsed.usage.expect("usage block present");
    assert_eq!(u.input_tokens, Some(0));
    assert_eq!(u.output_tokens, Some(0));
}
```
