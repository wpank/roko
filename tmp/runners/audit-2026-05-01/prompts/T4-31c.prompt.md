# T4-31c: Gemini provider — preserve UsageObservation

## Task
Migrate the Gemini provider's `usageMetadata` parsing to `UsageObservation`.

## Runner Context
Runner audit-2026-05-01, group T4. No dependencies. Wave 1.

## Source plan
`tmp/subsystem-audits/implementation-plans/14-tier4-feedback-loops.md` § T4-31.

## Why
Gemini returns `usageMetadata` with `promptTokenCount`, `candidatesTokenCount`, `totalTokenCount`. Cost is computed client-side. Today the adapter likely zero-fills missing fields; switch to `UsageObservation` so absent stays `None`.

## Read first

```bash
ls crates/roko-agent/src/providers/gemini*.rs
rg 'usageMetadata|promptTokenCount|candidatesTokenCount|totalTokenCount' crates/roko-agent/src/providers/gemini*.rs -n
```

Gemini response shape:

```json
{
  "candidates": [{"content": {"parts": [{"text": "..."}]}, "finishReason": "STOP"}],
  "usageMetadata": {
    "promptTokenCount": 100,
    "candidatesTokenCount": 50,
    "totalTokenCount": 150
  }
}
```

## Exact changes

### 1. Parser update

```rust
use roko_core::usage::UsageObservation;

let usage = response.usage_metadata.map(|m| UsageObservation {
    input_tokens: m.prompt_token_count,         // Option<u64>
    output_tokens: m.candidates_token_count,    // Option<u64>
    total_tokens: m.total_token_count,          // Option<u64>; trust API value, do not recompute
    cost_usd: None,
});
```

If the deserialization struct uses `u64` (not `Option<u64>`), change to `Option<u64>` with `#[serde(default)]`.

```rust
#[derive(Deserialize)]
#[serde(rename_all = "camelCase")]
struct GeminiUsageMetadata {
    #[serde(default)]
    prompt_token_count: Option<u64>,
    #[serde(default)]
    candidates_token_count: Option<u64>,
    #[serde(default)]
    total_token_count: Option<u64>,
}
```

Important: trust `totalTokenCount` from the API rather than recomputing as `input + output`. Some Gemini responses include cached / system tokens in `total` that aren't in the other two.

### 2. Streaming

Gemini streaming sends a single `usageMetadata` on the final chunk. Same logic as non-stream: `Option<UsageObservation>` from the final event.

### 3. Tests

```rust
#[test]
fn gemini_parser_preserves_absent_usage_metadata() {
    let body = r#"{
        "candidates": [{"content": {"parts": [{"text": "hi"}]}, "finishReason": "STOP"}]
    }"#;
    let parsed = parse_response(body).unwrap();
    assert!(parsed.usage.is_none());
}

#[test]
fn gemini_parser_distinguishes_zero_token_counts() {
    let body = r#"{
        "candidates": [{"content": {"parts": [{"text": ""}]}}],
        "usageMetadata": {"promptTokenCount": 0, "candidatesTokenCount": 0, "totalTokenCount": 0}
    }"#;
    let parsed = parse_response(body).unwrap();
    let u = parsed.usage.expect("usageMetadata block present");
    assert_eq!(u.input_tokens, Some(0));
    assert_eq!(u.output_tokens, Some(0));
    assert_eq!(u.total_tokens, Some(0));
}

#[test]
fn gemini_parser_preserves_total_when_input_or_output_missing() {
    let body = r#"{
        "candidates": [{"content": {"parts": [{"text": "hi"}]}}],
        "usageMetadata": {"totalTokenCount": 200}
    }"#;
    let parsed = parse_response(body).unwrap();
    let u = parsed.usage.expect("usageMetadata block present");
    assert_eq!(u.input_tokens, None);
    assert_eq!(u.output_tokens, None);
    assert_eq!(u.total_tokens, Some(200));   // do NOT compute as input+output
}
```

## Acceptance Criteria

- Gemini parser returns `Option<UsageObservation>` (None when `usageMetadata` absent).
- API-provided `totalTokenCount` preserved verbatim; not recomputed as `input + output`.
- 3 tests covering absent / zero / partial cases.

## Do NOT

- Do NOT recompute `total_tokens` as `input + output`. Use the API value.
- Do NOT bundle with another provider.
- Do NOT change Gemini request structure (T5-42b handles structured `contents`).
- Do NOT remove `cached_content_token_count` parsing if present (it's a separate signal; keep as-is or wire into `UsageObservation` only if there's a clear field).
