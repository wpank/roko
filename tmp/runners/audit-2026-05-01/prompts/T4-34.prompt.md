# T4-34: Make chat /model switch atomic

## Task
Resolve-then-commit. The `/model <slug>` command in the chat REPL must build all next-state values in temporaries and commit to `self.agent_session` only after every fallible step succeeds. Failed switch leaves all session fields unchanged.

## Runner Context
Runner audit-2026-05-01, group T4. No dependencies. Wave 1.

## Source plan

## Why
Today, `/model <bad-slug>` may mutate `agent_session.model` and the display-current-model field, then fail during auth or adapter construction. The user ends up with a torn session: display says one thing, dispatch uses another, history was clipped, etc.

## Exact changes

### 1. Refactor to resolve-then-commit

Replace the mutation-as-you-go pattern with:

```rust
async fn handle_model_command(&mut self, args: &str) -> Result<(), ChatError> {
    let slug = args.trim();
    if slug.is_empty() {
        return Err(ChatError::usage("/model <model-slug>"));
    }

    // === RESOLVE phase: every fallible step happens BEFORE any mutation ===

    let next_model = self
        .config
        .resolve_model(slug)
        .ok_or_else(|| ChatError::unknown_model(slug.to_string()))?;

    // Validate provider auth/capability via DispatchResolver.
    // (Plan 21 ACP-1 / S-acp-1 introduces typed validation; if not yet
    // landed, the existing "Unvalidated" path returns Ok(_) but should
    // upgrade gracefully when validation lands.)
    let plan = self.dispatch_resolver.resolve_for_chat(&next_model)?;
    self.dispatch_resolver.validate_for_call(&plan)?;

    let next_provider = self
        .config
        .resolve_provider(&next_model.provider)
        .ok_or_else(|| ChatError::unknown_provider(next_model.provider.clone()))?;

    let next_call_config = self.build_model_call_config(&next_model, &next_provider)?;
    let next_display = self.display_for_model(&next_model);
    // (Add any other fallible build steps as resolve-time temporaries.)

    // === COMMIT phase: from here, no fallible operations ===

    self.agent_session.model = next_model;
    self.agent_session.provider = next_provider.id.clone();
    self.agent_session.model_call_config = next_call_config;
    self.agent_session.display.set_model(next_display);
    // ... assign every other field captured during the audit

    Ok(())
}
```

Important: every field you write to in the commit phase **must** have a corresponding `let next_* = ...` in the resolve phase. If a field is mutated mid-handler today but you can't resolve a "next value" up front, that's a sign the handler is doing extra work that should split into another command.

If a field that's mutated today is **not** part of the model-switch contract (e.g. it's a counter / cache / unrelated state), audit whether it should remain in this handler. Often these are stale leftovers; remove them.

### 3. Tests

```rust
#[tokio::test]
async fn failed_model_switch_leaves_session_unchanged() {
    let mut session = test_chat_session();
    let original = session.snapshot();   // capture all model-related fields
    let result = session.handle_model_command("totally-fake-model").await;
    assert!(result.is_err());
    let after = session.snapshot();
    assert_eq!(after, original, "no field should change on failed switch");
}

#[tokio::test]
async fn successful_switch_updates_all_fields_atomically() {
    let mut session = test_chat_session(); // starts on default model
    session.handle_model_command("opus").await.unwrap();
    assert_eq!(session.agent_session.model.slug, "opus");
    // Display, model_call_config, provider all updated:
    assert_eq!(session.agent_session.display.current_model_slug(), "opus");
    assert_eq!(session.agent_session.model_call_config.model, "opus");
}

#[tokio::test]
async fn missing_auth_fails_without_partial_mutation() {
    let mut session = test_chat_session_with_no_anthropic_key();
    let original = session.snapshot();
    let r = session.handle_model_command("claude-opus").await;
    assert!(matches!(
        r,
        Err(ChatError::Validation(DispatchValidationError::MissingApiKey { .. }))
    ));
    assert_eq!(session.snapshot(), original);
}
```

If `session.snapshot()` doesn't exist, add a `pub(crate) fn snapshot(&self) -> ChatSessionSnapshot` that captures the audit-list fields. The snapshot is `PartialEq`/`Debug`.

### 4. Document the contract

Above `handle_model_command`:

```rust
/// Switch the active model atomically.
///
/// On failure (unknown model, missing auth, invalid config), no field of
/// `self.agent_session` is mutated. The error is returned to the caller;
/// the previous model remains in effect.
///
/// All resolve-time work happens in temporaries; the commit phase consists
/// only of infallible field assignments. If a future change needs to mutate
/// state mid-handler, factor that into a separate command instead.
async fn handle_model_command(&mut self, args: &str) -> Result<(), ChatError>
```

## Acceptance Criteria

- `handle_model_command` builds all next-state values in `let next_*` temporaries.
- All `self.agent_session.*` assignments are below the resolve block, with no intervening `?` or `await` that could fail.
- Snapshot helper exists for testing.
- 3 tests cover happy path, unknown model, missing auth.
- Doc comment documents the atomic contract.

## Do NOT

- Do NOT use a "best-effort rollback" pattern (mutate, then try to restore on error). Resolve-then-commit is the only correct shape.
- Do NOT skip auth resolution in fast-path "model already active" cases. Auth may have changed; revalidate.
- Do NOT touch ACP `/model` semantics in this batch (separate transport, separate session type).
- Do NOT touch `roko model` CLI subcommand (separate code path; out of scope).
- Do NOT introduce a `mutex.lock()` to "serialize" switches — single-threaded chat REPL handles one command at a time already.
- Do NOT change the dispatch validation logic; just consume it. (S-acp-1 supplies typed errors when it lands.)
- Do NOT log "switching model" until after the commit phase; otherwise a failed switch produces a misleading log.
