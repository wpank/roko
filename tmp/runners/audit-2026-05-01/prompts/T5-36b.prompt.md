## Exact changes

Same pattern as T5-36a:

1. Replace `reqwest::Client::new()` + manual POST with `state.model_call_service.call(req)` (or `.stream(...)` if streaming).
2. Build `ModelCallRequest` from the inbound `ResearchRequest`. Provider field defaults to `Some("perplexity")` if the route is hardcoded; otherwise let `model_call_service` resolve from the model slug.
3. Return the response with `text`, `usage`, `model`, and any research-specific fields (citations) already produced by the model adapter.
4. Test: stub dispatch returns expected text; assert response shape preserved.

If the route also handles citations / sources (Perplexity returns `citations` in its response), keep that adapter-specific extraction inside the route. The shared `ModelCallResponse` carries `text` and `usage`; provider-specific extras come from `ModelCallResponse::extra_metadata` or the streaming events.

If the shared dispatch path doesn't currently surface citations, log it: this batch focuses on the dispatch migration; citation plumbing is a follow-up. Keep the citations parse step inside the route by deserializing the raw provider body separately if needed.
