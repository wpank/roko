# T5-40d: RunLedger — resume / checkpoint via ledger

## Task
Add `RunLedgerEntry::Checkpoint { state, sequence, ts_ms }` and make resume read the latest checkpoint from the ledger instead of `executor.json`. After this slice, `executor.json` writing can be removed.

## Runner Context
Runner audit-2026-05-01, group T5. Depends on T5-40a. Wave 2.

## Source plan
`tmp/subsystem-audits/implementation-plans/24-runtime-ledger-migration.md` § Slice 4.

## Why
Today `--resume .roko/state/executor.json` requires the user to know the file path. After this slice, resume reads from `RunLedger`'s typed checkpoint entries; no manual flag.

## Exact changes

### 1. Add ledger variant

```rust
// crates/roko-runtime/src/run_ledger.rs

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct CheckpointState {
    pub plan: Plan,
    pub completed_tasks: Vec<TaskId>,
    pub current_task: Option<TaskId>,
    pub gate_baselines: HashMap<Rung, AdaptiveThreshold>,
    // Mirror whatever executor.json carries. Don't invent new fields.
}

RunLedgerEntry::Checkpoint {
    state: CheckpointState,
    sequence: u64,
    ts_ms: i64,
}

impl RunLedger {
    pub fn record_checkpoint(&mut self, state: CheckpointState, sequence: u64) {
        self.append(RunLedgerEntry::Checkpoint {
            state,
            sequence,
            ts_ms: chrono::Utc::now().timestamp_millis(),
        });
    }

    pub fn latest_checkpoint(&self) -> Option<&CheckpointState> {
        self.entries().rev().find_map(|e| match e {
            RunLedgerEntry::Checkpoint { state, .. } => Some(state),
            _ => None,
        })
    }
}
```

### 2. Periodic checkpoint write

In the runner / orchestrate loop, after each task completion:

```rust
let state = self.checkpoint_state();
self.run_ledger.lock().await.record_checkpoint(state, self.next_sequence());
```

`checkpoint_state()` builds a `CheckpointState` from current orchestrator state. `next_sequence()` is a monotonic counter.

### 3. Resume reads ledger

```rust
// crates/roko-cli/src/runner/resume.rs

pub fn resume_from_ledger(ledger_path: &Path) -> Result<RunLedger, ResumeError> {
    let ledger = RunLedger::load(ledger_path)?;
    let _ = ledger.latest_checkpoint().ok_or(ResumeError::NoCheckpoint)?;
    Ok(ledger)
}
```

The runner constructor:

```rust
pub fn from_ledger(ledger: RunLedger) -> Result<Self, ResumeError> {
    let cp = ledger.latest_checkpoint().ok_or(ResumeError::NoCheckpoint)?.clone();
    Ok(Self {
        plan: cp.plan,
        completed_tasks: cp.completed_tasks,
        // ... reconstruct from CheckpointState
        ledger: Arc::new(Mutex::new(ledger)),
    })
}
```

### 4. Drop `executor.json` writes

After verifying ledger-based resume works:

```bash
rg 'executor\.json' crates/roko-cli/src/runner/ -n
```

For each writer site (after the ledger checkpoint is recorded), remove the `executor.json` write.

For the reader (`--resume <path>`), keep backward compatibility for one release: if the user passes a path that's an `executor.json`, log a deprecation warning and read it; otherwise treat the path as a ledger path.

### 5. Tests

```rust
#[test]
fn resume_reconstructs_from_latest_checkpoint() {
    let dir = tempdir().unwrap();
    let path = dir.path().join("ledger.jsonl");
    let mut ledger = RunLedger::open(&path).unwrap();

    let state1 = test_checkpoint_state(0_completed);
    ledger.record_checkpoint(state1, 1);
    let state2 = test_checkpoint_state(1_completed);
    ledger.record_checkpoint(state2, 2);

    let restored = RunLedger::load(&path).unwrap();
    let cp = restored.latest_checkpoint().unwrap();
    assert_eq!(cp.completed_tasks.len(), 1);
}
```
