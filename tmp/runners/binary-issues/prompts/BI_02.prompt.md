# BI_02: Add command allowlist to /api/terminal/sessions

## Task
Reject `POST /api/terminal/sessions` requests whose `command` field starts with anything outside a strict allowlist (`bash`, `zsh`, `sh`, `fish`, `python`, `python3`). Return HTTP 400 with a structured error body when the program is not allowed; allow the existing default-shell branch (no `command` field) unchanged.

## Runner Context
Group **BI_SEC**, batch 2 of 56. Closes MASTER-INDEX § **S5.6**. Deps: none.

## Problem
`crates/roko-serve/src/terminal.rs` exposes `POST /api/terminal/sessions` (route registered at `terminal.rs:609-612`, handler at `terminal.rs:457-482`). The JSON body deserialises into `CreateSessionRequest` (`terminal.rs:58-66`), whose `command: Option<String>` is forwarded into `SessionManager::create_session_inner` (`terminal.rs:218-259`). At line 240-251 the command string is split on whitespace and the first token is fed straight into `CommandBuilder::new(program)`:

```240:255:crates/roko-serve/src/terminal.rs
let (mut cmd, command_label) = if let Some(command) = command {
    let mut parts = command.split_whitespace();
    let Some(program) = parts.next() else {
        // ... empty-command error ...
        return Err(anyhow::anyhow!("empty command"));
    };

    let mut cmd = CommandBuilder::new(program);
    for arg in parts {
        cmd.arg(arg);
    }
    (cmd, command.to_string())
} else {
    let shell = std::env::var("SHELL").unwrap_or_else(|_| "/bin/zsh".to_string());
    (CommandBuilder::new(shell.clone()), shell)
};
```

Anything an authenticated REST caller puts in `command` becomes a child process of the serve binary. Even with `serve.terminal_enabled = false` by default and `require_api_key` on non-loopback (`routes/mod.rs:136-212`), an authenticated operator (or a stolen API key) can spawn arbitrary binaries — `curl ... | sh`, `nc -e /bin/bash …`, anything in `$PATH`. Universal rule **S3** in `00-RULES.md` requires a strict allowlist for terminal commands; the current implementation is "anything goes".

The fix is mechanical: introduce a `pub const ALLOWED_TERMINAL_PROGRAMS: &[&str]` near the top of `terminal.rs`, validate the parsed `program` token against it inside `create_session_inner`, and return a typed error that the REST handler maps to `400 Bad Request` (instead of the current `500 Internal Server Error` from `anyhow::anyhow!`).

A future batch (out of write scope here — the serve config struct lives in `roko-core/src/config/serve.rs`) will promote the allowlist to a config-driven `serve.terminal.allowed_commands` extension list. For now we follow universal rule **A3**: "If it's not [a config field], leave the literal but pull it into a `pub const` near the top of the module so it is at least discoverable."

## Required Changes

### Step 1: Add the allowlist constant and a helper

In `crates/roko-serve/src/terminal.rs`, immediately after the existing `TERMINAL_DISABLED_*` constants at line 81-82, add:

```rust
/// Programs the REST API will accept as the first whitespace-separated
/// token of `CreateSessionRequest.command`. Anything else is rejected
/// with HTTP 400 (see [`CreateSessionError::ProgramNotAllowed`]).
///
/// A future batch will promote this list to `serve.terminal.allowed_commands`
/// (config-driven extension) per BI_02 follow-up.
pub const ALLOWED_TERMINAL_PROGRAMS: &[&str] =
    &["bash", "zsh", "sh", "fish", "python", "python3"];

/// Return `true` when `program` (the first whitespace-separated token of a
/// `command` request) appears in [`ALLOWED_TERMINAL_PROGRAMS`].
///
/// The match is on the basename only (e.g. `/usr/bin/bash` → `bash`) so
/// fully-qualified paths to allowlisted shells are accepted; everything
/// else (including `/usr/bin/curl` or `python2`) is rejected.
fn is_allowed_terminal_program(program: &str) -> bool {
    let basename = std::path::Path::new(program)
        .file_name()
        .and_then(|s| s.to_str())
        .unwrap_or(program);
    ALLOWED_TERMINAL_PROGRAMS.contains(&basename)
}
```

### Step 2: Add a typed error so the REST handler can return 400 vs 500

Above the `pub struct SessionManager { ... }` block (currently around line 176), add:

```rust
/// Errors that `SessionManager::create_session*` can return as a typed value.
///
/// The REST handler maps `ProgramNotAllowed` and `EmptyCommand` to 400 and
/// every other variant to 500.
#[derive(Debug, thiserror::Error)]
pub enum CreateSessionError {
    #[error("empty command")]
    EmptyCommand,
    #[error("program `{program}` is not in the terminal allowlist")]
    ProgramNotAllowed { program: String },
    #[error("{0}")]
    Other(#[from] anyhow::Error),
}
```

If `thiserror` is not already in scope for `roko-serve`, search for existing `use thiserror::Error;` lines in the crate (e.g. `roko-serve/src/lib.rs` or another module) and mirror that import. Do not modify `Cargo.toml` — `thiserror` is a workspace dep used elsewhere in the crate.

```rust
pub async fn create_session(
    State(state): State<Arc<AppState>>,
    Json(req): Json<CreateSessionRequest>,
) -> impl IntoResponse {
    match state.terminal_sessions.create_session(
        req.cols,
        req.rows,
        req.command.as_deref(),
        req.workdir.as_deref(),
    ) {
        Ok((id, _reader, _sess_gen)) => {
            let info = state
                .terminal_sessions
                .session_info
                .lock()
                .get(&id)
                .cloned();
            Json(serde_json::json!({ "id": id, "session": info })).into_response()
        }
        Err(err @ CreateSessionError::EmptyCommand)
        | Err(err @ CreateSessionError::ProgramNotAllowed { .. }) => (
            axum::http::StatusCode::BAD_REQUEST,
            Json(serde_json::json!({
                "error": err.to_string(),
                "allowed": ALLOWED_TERMINAL_PROGRAMS,
            })),
        )
            .into_response(),
        Err(err) => (
            axum::http::StatusCode::INTERNAL_SERVER_ERROR,
            Json(serde_json::json!({"error": err.to_string()})),
        )
            .into_response(),
    }
}
```
