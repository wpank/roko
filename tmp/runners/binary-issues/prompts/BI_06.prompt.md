# BI_06: cfg(test)-gate AgentContract::permissive

## Task
Wrap `AgentContract::permissive` in `#[cfg(test)]` so it cannot be called from production code. Both existing call sites are already inside `#[cfg(test)]` test modules, so no caller needs shimming.

## Runner Context
Group **BI_SEC**, batch 6 of 56. Closes MASTER-INDEX § **S5.10**. Deps: none.

## Problem
Universal rule **A6** in `00-RULES.md`: "Anything labeled 'permissive', 'test only', or 'for tests' must be `#[cfg(test)]`-gated, not just documented as such."

`AgentContract::permissive` (`crates/roko-agent/src/safety/contract.rs:84-96`) is a `pub fn` whose own doc comment says "This is retained for tests and adapter shims" — but the `pub` visibility means any production caller can build a contract with no invariants, no governance, no recovery, and no `allowed_tools` restriction. That is the broadest possible attack surface for the dispatcher.

```84:96:crates/roko-agent/src/safety/contract.rs
impl AgentContract {
    /// Build a permissive contract for `role`.
    ///
    /// This is retained for tests and adapter shims. New code should prefer
    /// either [`AgentContract::load_for_role`] or
    /// [`AgentContract::restricted`] so missing-role fallbacks fail closed.
    #[must_use]
    pub fn permissive(role: impl Into<String>) -> Self {
        Self {
            role: role.into(),
            ..Self::default()
        }
    }
```

A workspace-wide search confirms there are exactly **two** callers, both already inside `#[cfg(test)] mod tests` blocks:

- `crates/roko-agent/src/safety/contract.rs:712` — the unit test `permissive_contract_is_empty_except_for_role` inside `#[cfg(test)] mod tests` (the cfg attribute starts at line 699).
- `crates/roko-agent/src/safety/mod.rs:1094` — the test helper `permissive_layer()` inside `#[cfg(test)] mod tests` (the cfg attribute starts at line 1079).

Both already compile only when `cfg(test)` is active. The fix is therefore zero-risk: gate the function definition with `#[cfg(test)]` and keep its `pub` visibility (so cross-module test consumers like `safety/mod.rs::permissive_layer` continue to work — `cfg(test)` items in one module are visible to `cfg(test)` items in sibling modules within the same crate).

## Required Changes

### Step 1: Gate `AgentContract::permissive` with `#[cfg(test)]`

In `crates/roko-agent/src/safety/contract.rs`, change the function definition at lines 84-96 from:

```rust
impl AgentContract {
    /// Build a permissive contract for `role`.
    ///
    /// This is retained for tests and adapter shims. New code should prefer
    /// either [`AgentContract::load_for_role`] or
    /// [`AgentContract::restricted`] so missing-role fallbacks fail closed.

        Self {
            role: role.into(),
            ..Self::default()
        }
    }
```

to:

```rust
impl AgentContract {
    /// Build a permissive contract for `role`. **Test-only** (BI_06).
    ///
    /// `cfg(test)`-gated since BI_06 / MASTER-INDEX § S5.10. Production code
    /// must use [`AgentContract::load_for_role`] (loads the bundled YAML for
    /// the role) or [`AgentContract::restricted`] (deny-everything fallback)
    /// so missing-role fallbacks fail closed.
    #[cfg(test)]
    #[must_use]
    pub fn permissive(role: impl Into<String>) -> Self {
        Self {
            role: role.into(),
            ..Self::default()
        }
    }
```

The other functions in the same `impl AgentContract` block (`restricted`, `load_for_role`, etc.) stay as-is — only `permissive` gets gated.
