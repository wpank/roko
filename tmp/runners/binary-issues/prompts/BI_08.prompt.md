# BI_08: Auto-trigger EpisodeLogger::compact on session start / N episodes

## Task
`EpisodeLogger::compact` performs age- and size-based retention but has zero
production callers тАФ only test code reaches it. Add a `compact_on_append`
self-trigger inside `EpisodeLogger` that fires every N successful appends, and
expose a public helper so chat / workflow startup can run an opportunistic
compaction. The interval is configured via a `RetentionPolicy` field defaulting
to 100.

## Runner Context
Group **BI_PHN**, batch 8 of 56. Closes MASTER-INDEX ┬з **S8.1**. Deps: none.

## Problem
`EpisodeLogger::compact` is defined at
`crates/roko-learn/src/episode_logger.rs:1094-1172`. Its docs describe age-based
+ size-based pruning and atomic rename. A `RetentionPolicy` struct is declared
right after it (line ~1178). Today the function is called only from
`#[cfg(test)]` modules: `rg "logger\.compact|\.compact\(now" crates/roko-learn`
returns test-only matches, and `rg compact_on_append crates` returns nothing.

That makes `episodes.jsonl` grow without bound in production тАФ the very file
the runner / chat / dreams subsystems read on every session boot. The fix has
two parts:

1. Add a `compact_interval` field to `RetentionPolicy` (default 100). Add an
   internal append counter on `EpisodeLogger` and run `compact()` every
   `compact_interval` appends when a `RetentionPolicy` has been attached. This
   gives N-episode triggering inside the logger itself, where we already hold
   the file handle and the write gate.
2. Add a public `compact_if_due(&self, now, policy)` (or equivalent) that does
   a one-shot age-only compaction without bumping the counter. The CLI session
   path can call this on boot once `EpisodeLogger` is wired in (a separate
   batch тАФ keep the call-site change small here).

The `also_read` files (`workflow_engine.rs`, `chat_session.rs`) are listed so
the prompt author can see *why* this matters; this batch only edits
`episode_logger.rs`. Wiring boot-time invocations from the chat / workflow
paths is out of scope for this batch.

## Required Changes

### Step 1: Add `compact_interval` to `RetentionPolicy`

In `crates/roko-learn/src/episode_logger.rs`, extend `RetentionPolicy`:

```rust
#[derive(Debug, Clone)]
pub struct RetentionPolicy {
    pub max_age_days: u32,
    pub max_episodes: usize,
    /// Run [`EpisodeLogger::compact`] automatically once every
    /// `compact_interval` successful appends. `0` disables the auto-trigger
    /// (callers must invoke `compact` manually).
    pub compact_interval: u32,
}
```

Add a `pub const DEFAULT_COMPACT_INTERVAL: u32 = 100;` near the top of the
module (or inside an `impl RetentionPolicy` block) and use it in
`Default::default()` for `RetentionPolicy`. If `RetentionPolicy` does not yet
have a `Default` impl, do **not** add one as part of this batch тАФ instead pass
`100` literally where the struct is constructed.

### Step 2: Track append count and auto-trigger compaction

Add an `append_counter: AtomicU64` (or `tokio::sync::Mutex<u64>` if the
existing struct already uses `tokio::sync::Mutex`) to the private inner struct
of `EpisodeLogger`. Initialize to 0 in the constructor.

Find the function that performs a successful JSONL append (search for
`write_all` near a `serde_json::to_string` of an `Episode`). After the append
flushes successfully, increment the counter. If the logger has been configured
with a `RetentionPolicy` whose `compact_interval > 0` and the counter is
divisible by `compact_interval`, schedule a compaction.

Use a "best-effort" pattern: spawn `tokio::spawn(async move { let _ =
self_clone.compact(Utc::now(), &policy).await; })` and ignore the join handle
тАФ compaction failure must NOT fail the append. Log compaction errors at
`tracing::warn!`.

If the existing `EpisodeLogger` does not store a `RetentionPolicy`, add a
`with_retention_policy(self, policy: RetentionPolicy) -> Self` builder so the
caller can opt in. When no policy is attached, the auto-trigger is inert.

### Step 3: Add `compact_if_due` for opportunistic boot-time compaction

```rust
/// Run [`Self::compact`] only when the on-disk file size exceeds an
/// approximate threshold, used for one-shot startup compaction without
/// bumping the append counter.
///
/// # Errors
///
/// Same as [`Self::compact`].
pub async fn compact_if_due(
    &self,
    now: DateTime<Utc>,
    policy: &RetentionPolicy,
) -> Result<Option<CompactStats>, LoggerError> {
    // Cheap size probe тАФ if file is small, skip.
    let len = match tokio::fs::metadata(&self.inner.path).await {
        Ok(m) => m.len(),
        Err(_) => return Ok(None),
    };
    // Heuristic: 1 MiB worth of episodes ~ a few thousand lines.
    if len < 1_048_576 {
        return Ok(None);
    }
    Ok(Some(self.compact(now, policy).await?))
}
```

Place this method directly after `compact()` in the `impl EpisodeLogger`
block.

### Step 4: Update tests

`compact_interval: 0` (or the new `DEFAULT_COMPACT_INTERVAL` constant) so the
test continues to compile. Do not add new test functions in this batch.
