# BI_09: Wire DreamTriggerSink::with_runner and consume dream_triggers.jsonl

## Task
The `roko-dreams` crate ships `DreamRunner` and `DreamTrigger` but neither is
called from the active `plan run` path. Create a new `DreamTriggerSink` in
`crates/roko-runtime/src/runtime_feedback/dreams.rs` that records triggers to
`.roko/dream_triggers.jsonl` *and* (when a `DreamRunner` is attached) drains
queued triggers in a background tokio task once the workflow is idle. Wire it
into `commands/plan.rs` alongside the existing
`EpisodeSink` / `RoutingObservationSink` / `KnowledgeIngestionSink`.

- `crates/roko-dreams/src/runner.rs:721-771` — `DreamRunner` with
  `new(workdir, config)` and `with_model_caller(...)` builders, plus
  `consolidate_now` / `replay_insights` async entry points (search
  `pub async fn consolidate_now` and `pub fn replay_insights` to confirm
  signatures).
- `crates/roko-cli/src/runtime_feedback/mod.rs:122-200` — the
  `FeedbackSink` trait (`async fn on_event`, `fn interested`,
  `fn name`) and `FeedbackFacade::with_sink(...)` builder used by
  `plan.rs:372-390` to fan out runtime events.

The required wiring is therefore:

1. Define a new `DreamTriggerSink` that lives next to the other feedback sinks
   but in `roko-runtime` (per the batch scope) so the workflow engine and the
   CLI can both depend on it without crossing into `roko-cli`.
2. The sink JSONL-appends one line per qualifying `FeedbackEvent`
   (`PlanCompleted` and `IdleTick` are the two natural triggers; map them to
   `DreamTrigger::EpisodeCount` and `DreamTrigger::Idle` respectively).
3. Add a `with_runner(self, runner: Arc<DreamRunner>)` builder. When a runner
   is attached, the sink also pushes the trigger onto an in-memory
   `mpsc::UnboundedSender<DreamTrigger>` whose receiver is drained by a
   background tokio task that calls `runner.consolidate_now(...)` and
   *removes* the corresponding line from the JSONL.
4. Wire one instance of the sink into `commands/plan.rs` immediately after
   the existing `KnowledgeIngestionSink` registration so any plan run with
   `learning.dream_on_completion = true` actually consolidates dreams.

P3 in `00-RULES.md` is the rule we are honoring: a JSONL producer with no
in-process consumer is "half a feature".

### Step 1: Create `crates/roko-runtime/src/runtime_feedback/dreams.rs`

```rust
//! Dream-trigger sink for the workflow feedback fan-out.
//!
//! Writes one JSONL line per fired trigger to `.roko/dream_triggers.jsonl`,
//! and (when a `DreamRunner` is attached via `with_runner`) drains the
//! queue from a background tokio task so dreams actually consolidate
//! after qualifying events.

use std::path::{Path, PathBuf};
use std::sync::Arc;

use async_trait::async_trait;
use roko_dreams::{DreamRunner, DreamTrigger};
use serde::{Deserialize, Serialize};
use tokio::sync::mpsc;

use crate::runtime_feedback::{FeedbackEvent, FeedbackSink};

const JSONL_FILE: &str = "dream_triggers.jsonl";

/// One JSONL record written by [`DreamTriggerSink`].
#[derive(Debug, Clone, Serialize, Deserialize)]
struct TriggerRecord {
    ts: chrono::DateTime<chrono::Utc>,
    label: &'static str,
    plan_id: Option<String>,
    succeeded: Option<bool>,
}

#[derive(Debug)]
pub struct DreamTriggerSink {
    jsonl_path: PathBuf,
    queue: Option<mpsc::UnboundedSender<DreamTrigger>>,
}

impl DreamTriggerSink {
    /// Build a sink that appends to `<dir>/dream_triggers.jsonl`.
    #[must_use]
    pub fn at(dir: impl Into<PathBuf>) -> Self {
        let mut jsonl_path = dir.into();
        if jsonl_path.is_dir() || jsonl_path.extension().is_none() {
            jsonl_path = jsonl_path.join(JSONL_FILE);
        }
        Self {
            jsonl_path,
            queue: None,
        }
    }

    /// Attach a [`DreamRunner`]. After this call qualifying events not only
    /// land in the JSONL file but also fire `runner.consolidate_now(...)` on
    /// a background tokio task.
    #[must_use]
    pub fn with_runner(mut self, runner: Arc<DreamRunner>) -> Self {
        let (tx, mut rx) = mpsc::unbounded_channel::<DreamTrigger>();
        let runner = runner.clone();
        let jsonl_path = self.jsonl_path.clone();
        tokio::spawn(async move {
            while let Some(trigger) = rx.recv().await {
                if let Err(err) = runner.consolidate_now(&trigger).await {
                    tracing::warn!(
                        error = %err,
                        "DreamTriggerSink: consolidate_now failed"
                    );
                    continue;
                }
                if let Err(err) = remove_processed_trigger(&jsonl_path, &trigger).await {
                    tracing::warn!(
                        error = %err,
                        "DreamTriggerSink: prune jsonl failed"
                    );
                }
            }
        });
        self.queue = Some(tx);
        self
    }
}

#[async_trait]
impl FeedbackSink for DreamTriggerSink {
    fn name(&self) -> &'static str {
        "dream_triggers"
    }

    fn interested(&self, event: &FeedbackEvent) -> bool {
        matches!(
            event,
            FeedbackEvent::PlanCompleted { .. } | FeedbackEvent::IdleTick { .. }
        )
    }

    async fn on_event(&self, event: &FeedbackEvent) -> Result<(), anyhow::Error> {
        let (label, trigger, plan_id, succeeded) = classify(event);
        append_record(&self.jsonl_path, label, plan_id, succeeded).await?;
        if let Some(tx) = self.queue.as_ref() {
            // Best-effort: queue overflow is logged but not fatal.
            let _ = tx.send(trigger);
        }
        Ok(())
    }
}

fn classify(
    event: &FeedbackEvent,
) -> (&'static str, DreamTrigger, Option<String>, Option<bool>) {
    match event {
        FeedbackEvent::PlanCompleted {
            plan_id, succeeded, ..
        } => (
            "plan_completed",
            DreamTrigger::EpisodeCount,
            Some(plan_id.clone()),
            Some(*succeeded),
        ),
        FeedbackEvent::IdleTick { .. } => {
            ("idle_tick", DreamTrigger::Idle, None, None)
        }
        _ => ("unknown", DreamTrigger::Manual, None, None),
    }
}

async fn append_record(
    path: &Path,
    label: &'static str,
    plan_id: Option<String>,
    succeeded: Option<bool>,
) -> Result<(), anyhow::Error> { /* tokio fs append, JSON line */ unimplemented!() }

async fn remove_processed_trigger(
    path: &Path,
    trigger: &DreamTrigger,
) -> Result<(), anyhow::Error> { /* read-rewrite without first matching label */ unimplemented!() }
```

Replace the two `unimplemented!()` stubs with real implementations following
the same JSONL conventions used in `crates/roko-cli/src/runtime_feedback/episodes.rs`
(open with `OpenOptions::new().create(true).append(true)`, write
`serde_json::to_string(&record)?` plus `\n`, flush). For `remove_processed_trigger`,
read the whole file, drop the first line whose `label` matches the trigger
classification, atomic rename via `<path>.tmp` -> `<path>` (the cascade
snapshot saves at `crates/roko-learn/src/cascade_router.rs:1587-1602` are a
good shape to mirror).
