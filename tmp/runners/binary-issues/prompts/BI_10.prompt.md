# BI_10: Persist LinUCB arm matrices via CascadeSnapshot

## Task
The cascade router serializes per-model trial counts and stage transitions but
not the LinUCB arm matrices. Extend `CascadeSnapshot` with an optional
`linucb: Option<LinUCBSnapshot>` field, fold the arm matrices into the
cascade JSON written by `CascadeRouter::save`, and restore them in
`CascadeRouter::from_snapshot`. After this batch, restarting the runner
preserves both the per-model confidence stats and the LinUCB
`(A, b, observations, reward_stats, ewc)` state.

## Runner Context
Group **BI_PHN**, batch 10 of 56. Closes MASTER-INDEX § **S8.3**. Deps: none.

## Problem
Three independent files participate today:

- `crates/roko-learn/src/cascade/persistence.rs:11-25` — `CascadeSnapshot`
  has fields `model_slugs`, `role_table`, `confidence_stats`,
  `total_observations`, `stage_transitions`. **No** LinUCB arms.
- `crates/roko-learn/src/cascade_router.rs:1547-1602` —
  `CascadeRouter::snapshot_json` and `::save` build a `CascadeSnapshot`
  ignoring the `linucb` field that the cascade router owns at
  `CascadeRouter.linucb` (line 84).
- `crates/roko-learn/src/model_router.rs:1157-1183` —
  `LinUCBRouter::save(&self) -> std::io::Result<()>` and
  `LinUCBRouter::load(...)` exist with their own `RouterSnapshot` (line 669)
  containing `arms: Vec<ArmState>` (with `a_matrix`, `b_vector`,
  `observations`, `reward_stats`, `ewc`) and `total_observations`. The
  `save()` method has zero non-test callers
  (`rg -n 'linucb\.save\(' crates` returns empty).

P2 in `00-RULES.md`: every learning state must round-trip through disk on
shutdown AND load on startup. Today the cascade snapshot persists half (per-
model trials, stage transitions) and the LinUCB half is silently re-initialized
on every boot.

The fix folds the LinUCB arm matrices into the cascade snapshot so a single
JSON write/read round-trips both subsystems. We do **not** keep two parallel
persistence paths.

## Required Changes

### Step 1: Promote LinUCB arms to a publicly nameable snapshot

In `crates/roko-learn/src/model_router.rs`, change visibility of
`RouterSnapshot` from private `struct` to `pub(crate) struct LinUCBSnapshot`
(rename to clarify it's the LinUCB-specific shape and avoid collision with
the cascade snapshot). Update both call sites in `LinUCBRouter::save` and
`LinUCBRouter::load`. Add a public method:

```rust
impl LinUCBRouter {
    /// Build an in-memory snapshot of the router's arm matrices and
    /// observation counts, intended for embedding inside `CascadeSnapshot`.
    #[must_use]
    pub(crate) fn snapshot(&self) -> LinUCBSnapshot {
        let state = self.state.read();
        LinUCBSnapshot {
            arms: state.arms.clone(),
            total_observations: state.total_observations,
        }
    }

    /// Rehydrate the router's arm matrices and observation counts from a
    /// previously captured snapshot. Arms whose slugs do not appear in the
    /// snapshot keep their identity-matrix initial state.
    pub(crate) fn restore(&self, snapshot: LinUCBSnapshot) {
        let mut state = self.state.write();
        for arm in snapshot.arms {
            if let Some(existing) = state.arms.iter_mut().find(|a| a.slug == arm.slug) {
                *existing = arm;
            }
        }
        state.total_observations = snapshot.total_observations;
    }
}
```

`ArmState` is already `pub` and already `Serialize + Deserialize` (verify
with the `#[derive(Debug, Clone, Serialize, Deserialize)] pub struct
ArmState` at `model_router.rs:421-437`), so no type-level changes are needed
there.

### Step 2: Add `linucb: Option<LinUCBSnapshot>` to `CascadeSnapshot`

In `crates/roko-learn/src/cascade/persistence.rs`, add the field:

```rust
use crate::model_router::LinUCBSnapshot;

#[derive(Serialize, Deserialize)]
pub(crate) struct CascadeSnapshot {
    pub(crate) model_slugs: Vec<String>,
    #[serde(default)]
    pub(crate) role_table: HashMap<AgentRole, String>,
    pub(crate) confidence_stats: HashMap<String, PersistedModelStats>,
    #[serde(default)]
    pub(crate) total_observations: u64,
    #[serde(default)]
    pub(crate) stage_transitions: Vec<StageTransition>,
    /// LinUCB arm matrices captured at save time. `None` for snapshots
    /// written before BI_10; the cascade router falls back to its
    /// freshly-initialised LinUCB state in that case.
    #[serde(default)]
    pub(crate) linucb: Option<LinUCBSnapshot>,
}
```

`#[serde(default)]` makes the new field backward compatible — older
`cascade-router.json` files without `"linucb"` will still parse.

If `LinUCBSnapshot` cannot be made `pub(crate)` because the modules sit in
sibling directories with no `pub(crate)` visibility path (verify with
`rg -n 'pub(crate)' crates/roko-learn/src/lib.rs`), make it
`pub(in crate)` or `pub` — but keep the constructor private.

### Step 3: Capture and restore the snapshot in `CascadeRouter`

In `crates/roko-learn/src/cascade_router.rs`:

- Update `snapshot_json` (line ~1547) to set `linucb: Some(self.linucb.snapshot())`.
- Update `from_snapshot` (line ~1604) to destructure the new field and call
  `router.linucb.restore(snapshot)` only when `Some(snapshot)`. When `None`,
  log a single `tracing::debug!` ("cascade snapshot missing linucb arms;
  starting fresh") and continue.
- Update the `let CascadeSnapshot { ... } = snapshot;` destructure to include
  `linucb,`.
- Inside the `Self { linucb: LinUCBRouter::new(model_slugs.clone()), ... }`
  initializer (line ~152), the call order is: build the cascade router
  with a fresh LinUCB, then `restore` arms from the snapshot if present.

### Step 4: Drop dead `LinUCBRouter::save` invocations (audit only)

`LinUCBRouter::save` and `::load` stay in place — they are still useful for
ad-hoc tools. But since the cascade snapshot now folds the arm matrices in,
add a doc comment near `save`:

```rust
/// Persist router state to the configured path.
///
/// Note: in production the LinUCB arm matrices are persisted as part of
/// `CascadeSnapshot` (see `cascade_router::CascadeRouter::save`). This
/// standalone path is retained for diagnostic dumps and tests.
```

Do NOT delete `LinUCBRouter::save` / `::load` in this batch.
