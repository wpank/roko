# BI_12: Enforce cumulative MaxCostPerTurn across tool calls

## Task
The `MaxCostPerTurn` rule rejects only single tool calls whose declared cost
exceeds the budget. The TODO at
`crates/roko-agent/src/safety/contract.rs:481-482` calls out the missing
cumulative check. Sum estimated costs across all tool calls in the current
turn (using the existing `external_actions` log) and reject when the running
total would exceed `max`.

The single-call branch only fires when one tool call is itself bigger than
`max` — a rare case. The realistic scenario (twenty `model_call`s adding up
to $30 against a $20 budget) is silently allowed.

`ToolContext.external_actions` (`crates/roko-core/src/tool/handler.rs:33-49,
164`) is already the per-turn ledger; every executed call is appended via
`ctx.record_external_action(action)` at line 282. `ExternalAction.metadata`
is a `serde_json::Value` — which is exactly where prior turns store their
declared costs. The neighbouring `MaxToolCallsPerTurn` rule already uses
`ctx.external_actions.read().len()` (line 451) so we have the access pattern.

We also have the cost-estimate helper:
`estimated_cost_usd(call)` at `contract.rs:568-584` reads
`call.arguments.get("estimated_cost_usd")` (or `cost_usd`, or
`estimated_cost_usd_cents`). The matching field lives on
`ExternalAction.metadata`, so the cumulative path needs a sibling helper that
reads `metadata.get("estimated_cost_usd")` instead of `arguments`.

A separate `safety::risk` module also tracks `estimated_cost: f64` directly
on `ExternalAction`-shaped records (`crates/roko-agent/src/safety/risk.rs:315`)
— useful to confirm the field name we should be writing into `metadata` for
new actions.

## Required Changes

### Step 1: Add a cumulative-cost helper next to `estimated_cost_usd`

In `crates/roko-agent/src/safety/contract.rs`, immediately after the
existing `estimated_cost_usd(call: &ToolCall)` helper (around line 584), add
a sibling that reads from an `ExternalAction.metadata`:

```rust
fn external_action_cost_usd(action: &ExternalAction) -> f64 {
    let meta = &action.metadata;
    meta.get("estimated_cost_usd")
        .and_then(|v| v.as_f64())
        .or_else(|| meta.get("cost_usd").and_then(|v| v.as_f64()))
        .or_else(|| {
            meta.get("estimated_cost_usd_cents")
                .and_then(|v| v.as_u64())
                .and_then(|cents| u32::try_from(cents).ok())
                .map(|cents| f64::from(cents) / 100.0)
        })
        .unwrap_or(0.0)
}
```

Keep the existing `estimated_cost_usd(call)` helper unchanged — both are now
needed, one for the *incoming* call, one for *prior* calls in this turn.

### Step 2: Replace the TODO with a cumulative check

Inside the `Self::MaxCostPerTurn(max)` arm at `contract.rs:471-483`, replace
the body:

```rust
Self::MaxCostPerTurn(max) => {
    let incoming = estimated_cost_usd(call).unwrap_or(0.0);
    if incoming > *max {
        return Err(ContractViolation::new(
            role,
            "MaxCostPerTurn",
            format!("call cost {incoming:.4} > {max:.4}"),
        ));
    }
    let prior_total: f64 = ctx
        .external_actions
        .read()
        .iter()
        .map(external_action_cost_usd)
        .sum();
    let projected = prior_total + incoming;
    if projected > *max {
        return Err(ContractViolation::new(
            role,
            "MaxCostPerTurn",
            format!(
                "cumulative cost {projected:.4} (prior {prior_total:.4} + this {incoming:.4}) > {max:.4}"
            ),
        ));
    }
}
```

The two-step structure preserves the existing single-call diagnostic message
when the offender is one fat call, and provides a cumulative diagnostic when
the per-call cost is fine but the running sum is not.

        .expect_err("cumulative cost must block");

}
