# BI_14: Harden create_share empty-transcript edge cases

## Task
The `POST /api/runs/{id}/share` handler in
`crates/roko-serve/src/routes/shared_runs.rs:140-262` has three failure
shapes that today either return HTTP 500 or silently produce a stub. Tighten
the contract: missing payload, missing events file, and partial transcripts
must each return a typed error with the correct 4xx status, and any case
that proceeds to write a transcript must produce a non-empty stub
(at minimum `"no events recorded for this run"`) rather than an empty
shell.

## Runner Context
Group **BI_PHN**, batch 14 of 56. Closes MASTER-INDEX § **S8.8**. Deps: none.

## Problem
Three concrete edge cases live in `create_share` today:

1. **Missing payload + missing events file**
   `shared_runs.rs:180-198`:
   ```rust
   let transcript = match loaded {
       Some(loaded) => loaded.transcript,
       None => match transcript_from_runtime_events(&state, &id, &token) {
           Some(transcript) => transcript,
           None => {
               let Some(Json(request)) = payload else {
                   return StatusCode::NOT_FOUND.into_response();
               };
               // ...run_shared_workflow(...)
           }
       },
   };
   ```
   Returning a bare `StatusCode::NOT_FOUND` (no body) is hostile to the
   browser client at `demo/demo-app/src/pages/Share.tsx`. We should return a
   structured 404 with an `error` field naming the missing run.

2. **`run_shared_workflow` failure**
   At `shared_runs.rs:188-196` a workflow build / run failure is collapsed
   into:
   ```rust
   return (StatusCode::INTERNAL_SERVER_ERROR, Json(json!({"error": e})))
       .into_response();
   ```
   `run_shared_workflow` returns `Err("share request requires a non-empty
   prompt")` for malformed payloads (line 470). That is a 400, not a 500.
   We must inspect the error string (or, preferably, return a typed enum
   from `run_shared_workflow`) and map known cases to 4xx.

3. **Partial transcripts**
   `transcript_from_runtime_events` (line 321-457) requires `agent`, `role`,
   `prompt` to be `Some(...)` before constructing the transcript
   (`agent: agent?, role: role?, prompt: prompt?`, lines 440-442). If any of
   those is missing, it returns `None` and falls into the workflow path. But
   a run that emitted *only* a `WorkflowStarted` and nothing else genuinely
   has no agent / role / prompt — currently we build a brand-new transcript
   from scratch via `run_shared_workflow`, which double-charges the user.
   Instead we should build a stub transcript with placeholder values and a
   `notes` line.

## Required Changes

### Step 1: Introduce a typed error for `create_share`

At the top of `crates/roko-serve/src/routes/shared_runs.rs` (under the
existing `use` block), add:

```rust
#[derive(Debug)]
enum CreateShareError {
    MissingRun { id: String },
    EmptyPrompt,
    InvalidWorkflow(String),
    WorkflowFailed(String),
    ScrubFailed,
    Io(String),
}

impl CreateShareError {
    fn status(&self) -> StatusCode {
        match self {
            Self::MissingRun { .. } => StatusCode::NOT_FOUND,
            Self::EmptyPrompt | Self::InvalidWorkflow(_) => StatusCode::BAD_REQUEST,
            Self::WorkflowFailed(_) | Self::ScrubFailed | Self::Io(_) => {
                StatusCode::INTERNAL_SERVER_ERROR
            }
        }
    }

    fn body(&self) -> Value {
        match self {
            Self::MissingRun { id } => json!({
                "error": "run not found",
                "id": id,
            }),
            Self::EmptyPrompt => json!({
                "error": "share request requires a non-empty prompt",
            }),
            Self::InvalidWorkflow(name) => json!({
                "error": "invalid workflow",
                "workflow": name,
            }),
            Self::WorkflowFailed(detail) => json!({
                "error": "workflow execution failed",
                "detail": detail,
            }),
            Self::ScrubFailed => json!({
                "error": "failed to scrub share transcript",
            }),
            Self::Io(detail) => json!({
                "error": "filesystem error",
                "detail": detail,
            }),
        }
    }

    fn into_response(self) -> Response {
        (self.status(), Json(self.body())).into_response()
    }
}
```

### Step 2: Switch `run_shared_workflow` to typed errors

Change `run_shared_workflow`'s return type from `Result<RunTranscript, String>`

- `format!("build workflow services: {error}")` → `CreateShareError::WorkflowFailed(...)`

### Step 3: Replace fall-through with stub transcript for partial events

Replace `transcript_from_runtime_events`'s `None` return for the
"events file present but partial" case. Today it returns `None` if any of
`agent`, `role`, `prompt` is missing. Add a sibling helper:

```rust
fn stub_transcript_from_partial(
    state: &AppState,
    run_id: &str,
    token: &str,
) -> Option<RunTranscript> {
    let path = state.layout.root().join("runtime-events.jsonl");
    let data = std::fs::read_to_string(&path).ok()?;
    if data.trim().is_empty() {
        return None;
    }
    // The events file exists but no envelope matched run_id, or the matched
    // envelopes lack required fields. Build a stub so the share endpoint
    // returns *something* useful rather than 404.
    Some(RunTranscript {
        id: token.to_string(),
        agent: "unknown".into(),
        role: "unknown".into(),
        prompt: format!("no events recorded for run {run_id}"),
        success: false,
        gates: Vec::new(),
        output: Some(
            "This run produced no actionable events. The original execution may have been
cancelled before any agent started.".to_string(),
        ),
        cost_usd: None,
        input_tokens: None,
        output_tokens: None,
        model: None,
        duration_s: None,
        episode_id: Some(run_id.to_string()),
        transcript: Vec::new(),
        timestamp: chrono::Utc::now().to_rfc3339(),
    })
}
```

The exact field set must match the `RunTranscript` struct in the same file
— verify by reading the `RunTranscript` definition (search
`pub struct RunTranscript` in this crate) before writing.

### Step 4: Restructure the resolution chain in `create_share`

Replace the existing match (lines 180-199) with:

```rust
let transcript = if let Some(loaded) = loaded {
    loaded.transcript
} else if let Some(transcript) = transcript_from_runtime_events(&state, &id, &token) {
    transcript
} else if let Some(stub) = stub_transcript_from_partial(&state, &id, &token) {
    // Events file exists but did not yield a complete transcript — return
    // a stub so the share UI has something to render.
    stub
} else if let Some(Json(request)) = payload {
    match run_shared_workflow(&state, &token, request, Arc::clone(&workspace_config)).await {
        Ok(transcript) => transcript,
        Err(err) => return err.into_response(),
    }
} else {
    return CreateShareError::MissingRun { id: id.clone() }.into_response();
};
```
