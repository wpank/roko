# BI_15: Fix Share.tsx endpoint mismatch (/api/share -> /api/shared)

## Task
`Share.tsx` fetches `/api/share/${token}` but the server registers the route
at `/api/shared/{token}`. The mismatch silently returns 404 every time a
share link is opened. Change the URL to `/api/shared/${token}` and pin it
in a `SHARE_API_PATH` const at the top of the file so the next typo gets
caught at the source.

## Runner Context
Group **BI_PHN**, batch 15 of 56. Closes MASTER-INDEX § **S8.9**. Deps: none.

## Problem
Frontend (`demo/demo-app/src/pages/Share.tsx:103`):

```tsx
const r = await get<Receipt>(`/api/share/${token}`);
```

Server (`crates/roko-serve/src/routes/shared_runs.rs:874`):

```rust
.route("/api/shared/{token}", axum::routing::get(get_shared_run))
```

The "P4 Routes match clients" rule in `00-RULES.md` covers exactly this
class of bug. The fix is one-character (`share` → `shared`) plus a const
that makes the path discoverable.

There is also a sibling route at line ~862:

```rust
axum::Router::new().route("/runs/{id}/share", axum::routing::post(create_share))
```

That one is `POST /runs/{id}/share` (no `/api` prefix in the same file —
verify the prefix is composed elsewhere). It is the **create** route, not
the **get** route. `Share.tsx` only consumes the GET route, so this batch
does not need to touch any POST URL — but spelling matters: the GET path is
`/api/shared/...` and the const must reflect that.

## Current Code

`demo/demo-app/src/pages/Share.tsx:88-111`:

```tsx
export default function SharePage() {
  const { token } = useParams<{ token: string }>();
  const { get } = useLiveApi();
  const [receipt, setReceipt] = useState<Receipt | null>(null);
  const [error, setError] = useState(false);
  const [loaded, setLoaded] = useState(false);

  useEffect(() => {
    if (!token) {
      setLoaded(true);
      setError(true);
      return;
    }
    (async () => {
      try {
        const r = await get<Receipt>(`/api/share/${token}`);
        setReceipt(r);
      } catch {
        setError(true);
      } finally {
        setLoaded(true);
      }
    })();
  }, [token, get]);
```

## Required Changes

### Step 1: Add the path const

At the top of `demo/demo-app/src/pages/Share.tsx`, after the imports
(immediately before the `interface Receipt {` declaration on line 7), add:

```ts
/**
 * Server route at `crates/roko-serve/src/routes/shared_runs.rs:874`:
 *   `.route("/api/shared/{token}", axum::routing::get(get_shared_run))`
 *
 * Pinned here so the URL stays string-identical with the server side.
 */
const SHARE_API_PATH = '/api/shared';
```

### Step 2: Use the const in `useEffect`

Replace line 103:

```tsx
const r = await get<Receipt>(`/api/share/${token}`);
```

with:

```tsx
const r = await get<Receipt>(`${SHARE_API_PATH}/${token}`);
```

That is the only call site in this file. If a future component imports
`Share.tsx` and reuses the path, it can re-export the const — but do not
move the const to a shared module in this batch.

### Step 3: Sanity-check the rest of the demo app

`rg -n "/api/share[^d]" demo/demo-app/src` will reveal any sibling
`/api/share/` literals (without the trailing `d`). If any exist, fix them
in this batch only if they appear inside `Share.tsx`. Other files
(`SuiteSelector.tsx`, etc.) are out of scope; if a hit lands outside
`Share.tsx`, leave a `TODO(BI_15)` comment noting the orphan.
