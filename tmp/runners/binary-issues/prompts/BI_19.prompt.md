## Required Changes

### Step 1: Add a `TerminalEventForwarder` near the bottom of `chat_inline.rs`

Below `read_roko_toml` (around `chat_inline.rs:2317-2323`), add a small consumer that fans `RuntimeEvent`s into the terminal buffer via a tokio mpsc:

```rust
/// `EventConsumer` that forwards key WorkflowEngine lifecycle events
/// into a tokio mpsc channel. The chat event loop pumps the channel
/// each tick and renders into the TUI via `term.push_lines`.
struct TerminalEventForwarder {
    tx: tokio::sync::mpsc::UnboundedSender<String>,
}

impl roko_core::foundation::EventConsumer for TerminalEventForwarder {
    fn consume(&self, event: &roko_core::RuntimeEvent) {
        let line = match event {
            roko_core::RuntimeEvent::WorkflowStarted { run_id, template, .. } => {
                Some(format!("started {template} run {run_id}"))
            }
            roko_core::RuntimeEvent::PhaseTransition { from, to, .. } => {
                Some(format!("phase {from} → {to}"))
            }
            roko_core::RuntimeEvent::AgentSpawned { agent_id, role, .. } => {
                Some(format!("agent {role} ({agent_id})"))
            }
            roko_core::RuntimeEvent::WorkflowCompleted { outcome, .. } => {
                Some(format!("completed: {outcome:?}"))
            }
            _ => None,
        };
        if let Some(line) = line {
            // Best-effort: the receiver may have been dropped if the user
            // cancelled the run. We don't care; tracing already logs the event.
            let _ = self.tx.send(line);
        }
    }
}
```

If a different `RuntimeEvent` variant set is preferred, mirror exactly the variants `StateHubBridge` matches at `crates/roko-cli/src/run.rs:280-316` so we stay consistent with the SSE/WS surface.
