### Step 1: Add a small `cmd_plan_run_inline` helper

Near `read_roko_toml` (around line 2317) in `chat_inline.rs`, add a free function that resolves `<dir>` to a list of `(task_id, prompt)` pairs the way `cmd_plan_run` does, then defers to `crate::run::run_plan_with_workflow_engine_with_consumers` (BI_24's consumer-aware variant of the existing `run_plan_with_workflow_engine`):

```rust
/// Discover tasks under `<plans_dir>` and execute them inline through
/// `WorkflowEngine`, forwarding lifecycle events to the supplied consumer.
///
/// `plans_dir` may be a plan directory (`<wd>/plans/<id>`) or a parent
/// directory containing multiple plans; we mirror what `roko_cli::plan::
/// summarize_discovered_plans` does. Returns the engine's
/// `PlanWorkflowReport` summary on success.
async fn cmd_plan_run_inline(
    plans_dir: &std::path::Path,
    workdir: &std::path::Path,
    forwarder: std::sync::Arc<dyn roko_core::foundation::EventConsumer>,
) -> anyhow::Result<roko_cli::run::PlanWorkflowReport> {
    let plans = roko_cli::runner::plan_loader::load_plans(plans_dir)?;
    let mut tasks: Vec<(String, String)> = Vec::new();
    for plan in &plans {
        for task in &plan.tasks {
            tasks.push((
                format!("{}:{}", plan.id, task.id),
                task.prompt.clone(),
            ));
        }
    }
    if tasks.is_empty() {
        anyhow::bail!("no tasks found under {}", plans_dir.display());
    }
    crate::run::run_plan_with_workflow_engine_with_consumers(
        &tasks,
        workdir,
        "standard",
        Vec::new(),
        Vec::new(),
        vec![forwarder],
    )
    .await
}
```

If `roko_cli::runner::plan_loader::load_plans` is not in the import surface (`rg -n 'pub use runner::plan_loader|pub mod runner' crates/roko-cli/src/lib.rs`), use the qualified path directly. If `Plan { id, tasks }` does not have those exact field names, mirror the field layout from `cmd_plan_dry_run` (`commands/plan.rs:783-959`) — that function does the same enumeration and is a known-correct reference.

`run_plan_with_workflow_engine_with_consumers` is BI_24's responsibility to define; this batch only calls it.
