### Step 1: Add a small `append_idea_jsonl` helper in `chat_inline.rs`

Near the other free-function helpers (around `read_roko_toml` at line 2317), add:

```rust
/// Append an idea row to `.roko/prd/ideas.jsonl` (JSON-line per idea).
///
/// `ideas.md` is still written by `roko_cli::prd::cmd_idea` for human
/// readability; this JSONL feed is what tools / dashboards consume.
fn append_idea_jsonl(workdir: &std::path::Path, text: &str) -> anyhow::Result<String> {
    let id = uuid::Uuid::new_v4().to_string();
    let ts = chrono::Utc::now().to_rfc3339();
    let path = workdir.join(".roko").join("prd").join("ideas.jsonl");
    if let Some(parent) = path.parent() {
        std::fs::create_dir_all(parent)?;
    }
    let row = serde_json::json!({
        "id": id,
        "ts": ts,
        "text": text,
    });
    let line = serde_json::to_string(&row)?;
    use std::io::Write as _;
    let mut file = std::fs::OpenOptions::new()
        .create(true)
        .append(true)
        .open(&path)?;
    writeln!(file, "{line}")?;
    Ok(id)
}
```

`uuid`, `chrono`, and `serde_json` are already workspace deps for `roko-cli` (verify with `rg -n '^uuid|^chrono|^serde_json' crates/roko-cli/Cargo.toml`). If `uuid` is feature-gated and the active features don't include `v4`, swap to `uuid::Uuid::new_v4()` only after confirming the feature is on; otherwise generate a millis-since-epoch + counter ID instead. Do NOT add new dependencies in this batch.
