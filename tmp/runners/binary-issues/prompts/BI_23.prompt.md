## Problem
The CLI today is rule **C4** ("`--dry-run` means not-applied; a flag without that name doing dry-run behavior is a bug") in reverse: the flag exists, but the non-dry-run path also doesn't write. The function reads thresholds, deserializes them as a generic `serde_json::Value`, prints them, and returns:

```62:108:crates/roko-cli/src/commands/learn.rs

            }
        }
        /* "routing" / "budget" arms also read-and-print only */
        other => { /* error */ }
    }

        println!("(dry-run: no changes applied)");
    }

```

`AdaptiveThresholds` (the typed form of the JSON file) lives at `crates/roko-gate/src/adaptive_threshold.rs:256-286` and already has `load`, `load_or_new`, and `save` methods (atomic write via `.tmp` + rename). The fix is to (a) deserialize as the typed struct, not a `serde_json::Value`, (b) compute the *next* `AdaptiveThresholds` snapshot the same way the runtime would, (c) diff old vs. new, and (d) call `save(&path)` when `dry_run == false`.

What "the next snapshot" means: tonight's batch is **not** to re-derive thresholds from raw episode logs (that's a much bigger change). Instead, we close the gap that the user asked for: when the runtime has been auto-updating thresholds in memory and *would have written them on shutdown if persistence were already wired*, the CLI command lets the operator persist the current on-disk file (or, if the runtime has emitted a side file with proposed updates, merge that file). Concretely:

- Read `.roko/learn/gate-thresholds.json` into `AdaptiveThresholds` (typed). Call this **`current`**.
- Look for a **proposal file** at `.roko/learn/gate-thresholds.proposed.json`. If it exists, deserialize it as `AdaptiveThresholds` and call this **`next`**. Otherwise `next = current.clone()` (no-op apply, but still pretty-print + exit).
- Diff `current` vs. `next` (by rung number; show ema_pass_rate / cusum / count deltas).
- If `dry_run == true`: print the diff, print `(dry-run: no changes applied)`, exit 0.
- If `dry_run == false`:
  - If `current == next`: print "thresholds already in sync", exit 0.
  - Otherwise: print the diff. Unless a `--yes` flag is set (see Step 1), prompt the user `apply? [y/N]` on stdin. On confirm, call `next.save(&path)?`, delete the proposal file, print `applied (wrote N rungs to <path>)`, exit 0. On deny, print "skipped", exit 0.

This keeps the writing surface minimal (one `.save()` call) and makes `--dry-run` honest: it diffs and prints, never writes.

### Step 3: Add the `diff_adaptive_thresholds` helper

Beneath `cmd_tune` in the same file, add:

```rust
/// Render a per-rung diff between two `AdaptiveThresholds` snapshots.
/// Returns one human-readable line per rung that changed.
fn diff_adaptive_thresholds(
    before: &roko_gate::adaptive_threshold::AdaptiveThresholds,
    after: &roko_gate::adaptive_threshold::AdaptiveThresholds,
) -> Vec<String> {
    let before_json = serde_json::to_value(before).unwrap_or(serde_json::Value::Null);
    let after_json = serde_json::to_value(after).unwrap_or(serde_json::Value::Null);
    let before_rungs = before_json
        .get("rungs")
        .and_then(serde_json::Value::as_object)
        .cloned()
        .unwrap_or_default();
    let after_rungs = after_json
        .get("rungs")
        .and_then(serde_json::Value::as_object)
        .cloned()
        .unwrap_or_default();
    let mut keys: std::collections::BTreeSet<String> = before_rungs.keys().cloned().collect();
    keys.extend(after_rungs.keys().cloned());
    let mut lines = Vec::new();
    for key in keys {
        let b = before_rungs.get(&key);
        let a = after_rungs.get(&key);
        if b == a {
            continue;
        }
        let extract_ema = |v: Option<&serde_json::Value>| -> String {
            v.and_then(|v| v.get("ema_pass_rate"))
                .and_then(serde_json::Value::as_f64)
                .map_or_else(|| "—".to_string(), |f| format!("{f:.3}"))
        };
        lines.push(format!(
            "rung {key}: ema {} → {}",
            extract_ema(b),
            extract_ema(a),
        ));
    }
    lines
}
```

The JSON round-trip above is a safe default that matches the file format on disk.
