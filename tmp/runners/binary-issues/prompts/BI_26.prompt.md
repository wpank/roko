# BI_26: Add 3s timeout to claude --version probe in auth_detect

## Task
Wrap the synchronous `claude --version` probe in `auth_detect.rs` with a 3-second timeout so a hanging or slow `claude` binary cannot stall `roko` startup. On timeout (or any I/O error), treat the probe as "not found" and continue to the next auth check.

## Runner Context
Group **BI_SUB**, batch 26 of 56. Closes MASTER-INDEX § **S9.1**. Deps: none.

## Problem
`crates/roko-cli/src/auth_detect.rs:69-75` runs `Command::new("claude").arg("--version").output()` on the calling thread with **no timeout and no async**. If the user has a misconfigured shim, a stuck npm wrapper, or a Node process that pegs CPU on startup, the entire `roko` CLI hangs before chat ever boots. Per `00-RULES.md` U1: every spawn has a timeout.

The probe is the very first thing `detect_auth()` does, so a 3s upper bound is cheap and bounds the worst-case startup latency. Other auth branches (env vars) are already fast.

pub fn detect_auth() -> AuthMethod {
    // 1. Claude CLI — lightweight probe via `claude --version`
    if let Ok(output) = Command::new("claude").arg("--version").output() {
        if output.status.success() {
            return AuthMethod::ClaudeCli;
        }

Note that `detect_auth` itself is `fn` (not `async fn`). The function is called from `unified.rs:34` (`async fn cmd_unified_chat`) and `unified.rs:99` (`async fn cmd_oneshot_inline`), so the simplest correct fix is to keep the timeout self-contained inside `detect_auth` using `tokio::task::spawn_blocking` + `tokio::time::timeout` invoked from the existing tokio runtime, **without** changing the function signature.

## Required Changes

### Step 1: Add a private helper that probes with a 3s timeout
At the top of `auth_detect.rs`, add the timeout constant after the existing `use std::process::Command;`:
```rust
use std::time::Duration;

/// Maximum time to wait for `claude --version` before giving up.
const CLAUDE_PROBE_TIMEOUT: Duration = Duration::from_secs(3);
```

Then add a private helper just above `pub fn detect_auth()`:
```rust
/// Probe whether the `claude` CLI is installed and responsive.
///
/// Runs `claude --version` with a 3-second timeout. Returns `true` only on
/// successful exit before the timeout fires; any error, non-zero exit, or
/// timeout is treated as "not available".
fn probe_claude_cli() -> bool {
    // We are called from an async tokio context (`cmd_unified_chat` /
    // `cmd_oneshot_inline`). Use `Handle::try_current` so unit tests that call
    // `detect_auth` outside a runtime still work — they fall back to the
    // sync probe with no extra overhead.
    if let Ok(handle) = tokio::runtime::Handle::try_current() {
        let result = handle.block_on(async {
            let probe = tokio::task::spawn_blocking(|| {
                Command::new("claude").arg("--version").output()
            });
            match tokio::time::timeout(CLAUDE_PROBE_TIMEOUT, probe).await {
                Ok(Ok(Ok(output))) => output.status.success(),
                _ => false,
            }
        });
        return result;
    }

    Command::new("claude")
        .arg("--version")
        .output()
        .map(|o| o.status.success())
        .unwrap_or(false)
}
```

### Step 2: Use the helper in `detect_auth`
Replace `crates/roko-cli/src/auth_detect.rs:70-75` with:
```rust
    // 1. Claude CLI — lightweight probe via `claude --version` (3s timeout)
    if probe_claude_cli() {
        return AuthMethod::ClaudeCli;
    }
```

### Step 3: Cover the timeout path with a unit test
Append to the existing `#[cfg(test)] mod tests { ... }` block at the bottom of the file:
```rust
    #[test]
    fn probe_claude_cli_returns_false_for_missing_binary() {
        // Override PATH so `claude` cannot be found.
        let orig = std::env::var_os("PATH");
        // SAFETY: tests in this file are not run in parallel with PATH-sensitive code.
        unsafe { std::env::set_var("PATH", "/nonexistent-roko-test-path") };
        let result = probe_claude_cli();
        if let Some(p) = orig {
            unsafe { std::env::set_var("PATH", p) };
        }
        assert!(!result, "missing binary must report not-available");
    }
```
