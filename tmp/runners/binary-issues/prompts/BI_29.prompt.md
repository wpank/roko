# BI_29: Store chain-watcher join handle and abort on shutdown

## Task
The chain-watcher subprocess is currently spawned via `tokio::spawn(...)` with the `JoinHandle` discarded — the task plus its child `roko-chain-watcher` process keep running after `state.shutdown().await` because nothing aborts them. Capture the handle in `AppState`, and on shutdown both `abort()` the supervising task and rely on `kill_on_drop`/explicit kill to bring the subprocess down.

## Runner Context
Group **BI_SUB**, batch 29 of 56. Closes MASTER-INDEX § **S9.6**. Deps: none.

## Problem
`crates/roko-serve/src/lib.rs:351-392` spawns the chain-watcher inside `start_background()`:
```rust
if let Some(rpc_url) = self.config.roko_config.chain.rpc_url.as_deref() {
    let rpc = rpc_url.to_string();
    let log_path = self.config.workdir.join(".roko").join("chain-watcher.log");
    tokio::spawn(async move {                       // <-- handle dropped
        let watcher = ...;
        match tokio::process::Command::new(&watcher)
            .arg("--rpc-url").arg(&rpc)
            // ...
            .stdout(stdout_target)
            .stderr(stderr_target)
            .status()                              // <-- await blocks until child exits
            .await
        { ... }
    });
}
```

Two leaks:
1. The supervising task is never aborted — `state.cancel.cancel()` does not reach this loop because it never observes the cancel token.
2. Even if the task is aborted, `Command::status()` does not auto-kill the child by default. We have to either use `kill_on_drop` or convert to `spawn` + explicit `child.kill()` on cancel.

`AppState` already owns `pub cancel: CancelToken` and `pub supervisor: Arc<ProcessSupervisor>` (`crates/roko-serve/src/state.rs:353,359`), and `shutdown()` is the central tear-down site (`state.rs:659-675`). The handle should live alongside other "spawned at start, aborted at shutdown" handles.

## Required Changes

### Step 1: Replace `Command::status()` with `Command::spawn()` + select on cancel
This is the only way to actually kill the subprocess. Update the spawn block in `crates/roko-serve/src/lib.rs:351-392` so the supervising task observes `state.cancel`:

```rust
if let Some(rpc_url) = self.config.roko_config.chain.rpc_url.as_deref() {
    let rpc = rpc_url.to_string();
    let log_path = self.config.workdir.join(".roko").join("chain-watcher.log");
    let cancel = state.cancel.clone();
    let handle = tokio::spawn(async move {
        let watcher = std::env::current_exe()
            .ok()
            .and_then(|p| p.parent().map(|d| d.join("roko-chain-watcher")))
            .unwrap_or_else(|| std::path::PathBuf::from("roko-chain-watcher"));

        let (stdout_target, stderr_target) = if let Ok(f) = std::fs::OpenOptions::new()
            .create(true)
            .append(true)
            .open(&log_path)
        {
            let f2 = f
                .try_clone()
                .unwrap_or_else(|_| std::fs::File::open("/dev/null").expect("/dev/null"));
            (std::process::Stdio::from(f), std::process::Stdio::from(f2))
        } else {
            (std::process::Stdio::null(), std::process::Stdio::null())
        };

        let mut child = match tokio::process::Command::new(&watcher)
            .arg("--rpc-url")
            .arg(&rpc)
            .env("ROKO_LOG", "warn")
            .stdout(stdout_target)
            .stderr(stderr_target)
            .kill_on_drop(true)
            .spawn()
        {
            Ok(c) => c,
            Err(e) => {
                tracing::debug!(error = %e, path = ?watcher, "chain-watcher not available");
                return;
            }
        };

        tokio::select! {
            biased;
            _ = cancel.cancelled() => {
                tracing::info!("chain-watcher: shutdown requested, killing subprocess");
                let _ = child.start_kill();
                let _ = child.wait().await;
            }
            res = child.wait() => match res {
                Ok(s) => tracing::info!(exit = %s, "chain-watcher exited"),
                Err(e) => tracing::warn!(error = %e, "chain-watcher wait failed"),
            },
        }
    });

    // Stash for shutdown abort.
    state.register_chain_watcher(handle);
}
```

### Step 2: Add the registration slot to `AppState`
Edit `crates/roko-serve/src/state.rs`. Find the `pub struct AppState { ... }` block (line 345) and add a new field next to `cancel`:
```rust
    /// Optional handle to the chain-watcher supervising task, stored so
    /// `shutdown()` can abort it. Set by `start_background` when
    /// `chain.rpc_url` is configured.
    pub chain_watcher_handle: tokio::sync::Mutex<Option<tokio::task::JoinHandle<()>>>,
```

Initialize it to `Mutex::new(None)` in the `AppState::new` (or equivalent constructor) — use `tokio::sync::Mutex::new(None)`.

Add a small helper on `impl AppState`:
```rust
/// Stash the chain-watcher join handle so it can be aborted on shutdown.
pub async fn register_chain_watcher(&self, handle: tokio::task::JoinHandle<()>) {
    let mut slot = self.chain_watcher_handle.lock().await;
    if let Some(prev) = slot.replace(handle) {
        prev.abort();
    }
}
```

If the `state` variable in `lib.rs:351-392` is an `Arc<AppState>`, the call becomes `state.register_chain_watcher(handle).await;` — but the spawn is synchronous code outside an `async` block. Resolve this by wrapping the registration in a small async block immediately after the `tokio::spawn`:
```rust
{
    let state = Arc::clone(&state);
    tokio::spawn(async move {
        state.register_chain_watcher(handle).await;
    });
}
```

(Alternative: change `chain_watcher_handle` to `std::sync::Mutex` if you prefer sync registration. Either is acceptable; keep one and document the choice in a one-line comment.)

### Step 3: Abort the handle in `AppState::shutdown`
Edit `crates/roko-serve/src/state.rs:659-675`. Inside `pub async fn shutdown(&self)`, after `self.cancel.cancel();`, add:

Wait — `handle.abort()` consumes nothing; `handle` is still owned. Replace the snippet above with the simpler:
```rust

            handle.abort();

        }
```
