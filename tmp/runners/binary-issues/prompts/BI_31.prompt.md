### Step 3: Guard `main.rs:2314-2343`

Replace the `eprintln!`s with `tracing::error!` so logging policy decides where they go. Severities are clear (these are user-visible failures that abort with exit 1):
```rust
if !snapshot.exists() {
    tracing::error!(path = %snapshot.display(), "no resume snapshot found");
    if !roko_cli::inline::should_use_inline() {
        eprintln!("no snapshot found at {}", snapshot.display());
        eprintln!("hint: run `roko plan run <dir>` first to create a checkpoint");
    }
    return Ok(1);
}
```

```rust
let plan_dir = resolve_plans_dir(&workdir, None);
if !plan_dir.exists() {
    let canonical = workdir.join(".roko").join("plans");
    let fallback = workdir.join("plans");
    tracing::error!(
        canonical = %canonical.display(),
        fallback = %fallback.display(),
        "no plans directory found",
    );
    if !roko_cli::inline::should_use_inline() {
        eprintln!(
            "error: no plans directory found. Checked:\n  canonical: {}\n  fallback: {}",
            canonical.display(),
            fallback.display(),
        );
    }
    return Ok(1);
}
```

The pattern: emit `tracing::error!` always (so logs/test runners see it) and a human `eprintln!` only when NOT in inline TUI mode.
