# BI_32: Log JSONL write/flush errors at warn instead of let _ =

## Task
The single-line `let _ = self.write_event(event);` in `JsonlLogger::consume` swallows every IO failure — disk full, permission denied, broken parent dir — silently. Replace it with an `if let Err(e) = ...` arm that emits `tracing::warn!` carrying the file path and the underlying error so operators have a chance to notice.

## Runner Context
Group **BI_ERR**, batch 32 of 56. Closes MASTER-INDEX § **S4.2**. Deps: none.

## Problem
`crates/roko-runtime/src/jsonl_logger.rs:89-93`:
```rust
impl EventConsumer for JsonlLogger {
    fn consume(&self, event: &RuntimeEvent) {
        let _ = self.write_event(event);
    }
}
```

`00-RULES.md` A2: *"`let _ = foo()` is allowed only when followed by an inline comment that justifies why the failure mode is harmless. Otherwise log at minimum `tracing::warn!`."*

JSONL is the canonical replay log; a missing event means a stale debug session. The fix is trivial — log at `warn` and continue.

`write_event` already does both write AND flush internally (`crates/roko-runtime/src/jsonl_logger.rs:62-86`):
```rust
fn write_event(&self, event: &RuntimeEvent) -> std::io::Result<()> {
    self.ensure_writer()?;
    // ...
    if let Some(ref mut w) = *writer {
        writeln!(w, "{json}")?;
        w.flush()?;
    }
    Ok(())
}
```

So one log call covers both failure modes — no need to split write vs. flush in the caller.

## Required Changes

### Step 1: Replace `let _ =` with a logging branch

Edit `crates/roko-runtime/src/jsonl_logger.rs:89-93` to:
```rust
impl EventConsumer for JsonlLogger {
    fn consume(&self, event: &RuntimeEvent) {
        if let Err(e) = self.write_event(event) {
            tracing::warn!(
                target: "jsonl",
                path = %self.path.display(),
                error = %e,
                "jsonl write failed",
            );
        }
    }
}
```

### Step 2: (Optional but recommended) split write and flush warnings inside `write_event`

The current `write_event` returns one `io::Result` for both `writeln!` and `flush()`. If you want to differentiate, restructure the inner block:
```rust
fn write_event(&self, event: &RuntimeEvent) -> std::io::Result<()> {
    self.ensure_writer()?;

    let envelope = RuntimeEventEnvelope::new(
        event.run_id(),
        self.seq.fetch_add(1, Ordering::Relaxed),
        "jsonl_logger",
        event.clone(),
    );

    let json = serde_json::to_string(&envelope)
        .map_err(|err| std::io::Error::new(std::io::ErrorKind::InvalidData, err))?;

    let mut writer = self
        .writer
        .lock()
        .unwrap_or_else(std::sync::PoisonError::into_inner);
    if let Some(ref mut w) = *writer {
        if let Err(e) = writeln!(w, "{json}") {
            tracing::warn!(
                target: "jsonl",
                path = %self.path.display(),
                error = %e,
                "jsonl line write failed",
            );
            return Err(e);
        }
        if let Err(e) = w.flush() {
            tracing::warn!(
                target: "jsonl",
                path = %self.path.display(),
                error = %e,
                "jsonl flush failed",
            );
            return Err(e);
        }
    }
    drop(writer);

    Ok(())
}
```

(If you take this option, the `EventConsumer::consume` impl still needs the warn so the `ensure_writer()?` path is also covered.)
