# BI_35: Return PTY send_input errors to WS client

## Task
Two `let _ = state.terminal_sessions.send_input(...)` calls in the WebSocket bridge swallow PTY send errors. Replace each with a real branch that converts the error into a structured WS error frame the frontend can render (`{"type":"error","message":"..."}`), so the user sees "session not found" instead of typing into a void.

## Runner Context
Group **BI_ERR**, batch 35 of 56. Closes MASTER-INDEX § **S4.7**. Deps: none.

## Problem
`crates/roko-serve/src/terminal.rs:589-593`:
```rust
// Regular input
let _ = state.terminal_sessions.send_input(&id, text.as_bytes());
}
Some(Ok(Message::Binary(data))) => {
    let _ = state.terminal_sessions.send_input(&id, &data);

```

When the PTY session has expired or the manager has dropped it (e.g. after BI_03 enforces a TTL), the next keystroke disappears with no feedback. The frontend has no way to display "your session ended; reconnect" because the failure is invisible.

`00-RULES.md` A2: silent error swallowing is forbidden. Either log + recover, OR (better here) propagate to the client so the UI can react.

## Required Changes

### Step 1: Add a small helper to send a JSON error frame

Above `async fn handle_ws` (around line 539), add:
```rust
/// Send a structured `{"type":"error","message":"..."}` text frame to the
/// client. Non-fatal: returns `Err(())` if the WebSocket is already closed,
/// so the bridge loop can `break` cleanly.
async fn send_ws_error(
    sink: &mut futures::stream::SplitSink<WebSocket, Message>,
    message: impl Into<String>,
) -> Result<(), ()> {
    use futures::SinkExt;
    let body = serde_json::json!({
        "type": "error",
        "message": message.into(),
    });
    sink.send(Message::Text(body.to_string().into()))
        .await
        .map_err(|_| ())
}
```

(If `futures::SinkExt` is already imported at the top of the file, drop the inline `use`.)

### Step 2: Replace the swallowed `send_input` calls

Inside `handle_ws`, update the text and binary input branches:
