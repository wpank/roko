# BI_36: Store fswatcher join handle, log spawn failures

## Task
`fswatcher::start_watchers` returns a `JoinHandle<()>`, but every caller binds it to `_watchers` and immediately drops it. The supervisor cannot abort the watcher on shutdown, and the spawn body silently swallows `start_event_source_group` failures behind a `let _ =`. Two fixes: (1) replace the `let _ =` with a logged error branch inside the spawn, and (2) store the returned handle in `AppState` so `shutdown()` can `abort()` it.

## Runner Context
Group **BI_ERR**, batch 36 of 56. Closes MASTER-INDEX § **S4.8**. Deps: none.

## Problem
`fswatcher::start_watchers` returns a join handle that every caller drops, and the spawn body silently swallows the inner `start_event_source_group` join handle. The watcher therefore cannot be aborted on shutdown (it merely waits for `state.cancel.cancelled()` to fire — fine for graceful shutdown, useless if the parent task is panicking), and any spawn-time error is invisible. Per `00-RULES.md` U4: *"Handles are stored, not orphaned."* Per `00-RULES.md` A2: silent error swallowing is forbidden without a justification comment.

`crates/roko-serve/src/fswatcher.rs:11-30`:

## Required Changes

### Step 1: Log the `start_event_source_group` failure in the spawn body

Update `crates/roko-serve/src/fswatcher.rs:14-29`:
```rust
#[must_use]
pub fn start_watchers(state: Arc<AppState>) -> JoinHandle<()> {
    tokio::spawn(async move {
        let watchers = {
            let config = state.load_roko_config();
            if config.watcher.is_empty() {
                Vec::new()
            } else {
                vec![
                    Box::new(FileWatchEventSource::from_config(config.watcher.clone()))
                        as Box<dyn EventSource>,
                ]
            }
        };

        // We don't await the inner join handle; it shuts itself down via
        // `state.cancel`. We DO want to know if event-source group startup
        // returned a non-empty failure path.
        let _group_handle =
            crate::start_event_source_group(Arc::clone(&state), watchers);

        state.cancel.cancelled().await;
    })
}
```

Note that `start_event_source_group` (in `lib.rs:1500-1542`) returns a `JoinHandle<()>`, not a `Result`. The actual failure path lives inside the spawned source tasks (`lib.rs:1530-1538` already calls `warn!(...)` on `EventSource::start` errors). So the original `let _ =` was harmless — but it failed the silent-swallow rule. Renaming it to `_group_handle` and adding the comment makes the intent explicit and audit-clean.

### Step 2: Store the watcher handle on `AppState`

Edit `crates/roko-serve/src/state.rs`. Add a new field to `pub struct AppState` next to `cancel`:
```rust
    /// Join handle for the filesystem watcher group. Set by callers of
    /// `fswatcher::start_watchers` via [`AppState::register_fs_watcher`] so
    /// `shutdown()` can abort the supervising task.
    pub fs_watcher_handle: tokio::sync::Mutex<Option<tokio::task::JoinHandle<()>>>,
```

Initialize to `tokio::sync::Mutex::new(None)` in `AppState::new` (or wherever the struct is constructed).

Add the helper:
```rust
/// Stash the fswatcher join handle so it can be aborted on shutdown.
pub async fn register_fs_watcher(&self, handle: tokio::task::JoinHandle<()>) {
    let mut slot = self.fs_watcher_handle.lock().await;
    if let Some(prev) = slot.replace(handle) {
        prev.abort();
    }
}
```

Update `start_watchers` to spawn a small registration task immediately:
```rust
#[must_use]
pub fn start_watchers(state: Arc<AppState>) -> JoinHandle<()> {
    let main_handle = tokio::spawn({
        let state = Arc::clone(&state);
        async move {
            let watchers = { /* ... unchanged ... */ };
            let _group_handle = crate::start_event_source_group(Arc::clone(&state), watchers);
            state.cancel.cancelled().await;
        }
    });

    // Take an AbortHandle so we can register without consuming the JoinHandle.
    let abort_handle = main_handle.abort_handle();
    let registration_state = Arc::clone(&state);
    tokio::spawn(async move {
        let mut slot = registration_state.fs_watcher_handle.lock().await;
        if let Some(prev) = slot.take() {
            prev.abort();
        }
        // Wrap the AbortHandle into a no-op JoinHandle by spawning an immediately-
        // pending task and abort()ing it via the existing handle. Simpler: store
        // the AbortHandle on AppState directly.
        let _ = abort_handle;
    });

    main_handle
}
```

The cleanest landing form requires `AppState` to store an `AbortHandle` instead of a `JoinHandle`. Update Step 2 to use:
```rust

```
…and the registration helper to take an `AbortHandle`. This avoids the awkward indirection above; `JoinHandle<()>::abort_handle()` returns one cheaply.

### Step 5: Self-register inside `start_watchers`

With Step 4's `AbortHandle` approach, the cleanest body is:
```rust
#[must_use]
pub fn start_watchers(state: Arc<AppState>) -> JoinHandle<()> {
    let main_handle = tokio::spawn({
        let state = Arc::clone(&state);
        async move {
            let watchers = {
                let config = state.load_roko_config();
                if config.watcher.is_empty() {
                    Vec::new()
                } else {
                    vec![
                        Box::new(FileWatchEventSource::from_config(config.watcher.clone()))
                            as Box<dyn EventSource>,
                    ]
                }
            };
            let _group_handle = crate::start_event_source_group(Arc::clone(&state), watchers);
            state.cancel.cancelled().await;
        }
    });

    let abort = main_handle.abort_handle();
    let register_state = Arc::clone(&state);
    tokio::spawn(async move {
        register_state.register_fs_watcher_abort(abort).await;
    });

    main_handle
}
```

…with a matching `register_fs_watcher_abort(&self, abort: AbortHandle)` helper on `AppState`.
