# BI_37: Fix double REST event delivery FIXME on EventBus

## Task
The REST event delivery loop publishes events to `EventBus` AND those same events come back via `StateHub→EventBus` bridge, so SSE/WS clients receive every REST event twice. Fix by routing each event through exactly one channel: keep the REST→`EventBus` publish, suppress its echo when it returns through `start_orchestrator_event_bridge`. Add a regression test that uses a mock `EventBus` + mock handler and asserts `received_count == 1`.

### Step 1: Add a process-static "echo suppression" set

At module scope in `crates/roko-serve/src/lib.rs` (top-of-file, with the other `static`/`const` items):
```rust
use std::collections::VecDeque;
use std::sync::Mutex as StdMutex;

/// Recent dashboard events that originated in `start_state_hub_bridge`.
/// Used by `start_orchestrator_event_bridge` to suppress the echo so REST
/// events are delivered exactly once.
///
/// Entries are dashboard event identity strings (`format!("{:?}", event)`)
/// with a short TTL — a 1024-slot ring is plenty given event rate caps.
static REST_ECHO_SUPPRESS: StdMutex<VecDeque<(u64, String)>> = StdMutex::new(VecDeque::new());

/// How many recent REST events to remember. Sized so a normal event burst
/// (multi-step plan run) cannot push older entries out before the orchestrator
/// bridge sees them.
const REST_ECHO_SUPPRESS_CAP: usize = 1024;

/// Mark a dashboard event as REST-originated so the orchestrator bridge can
/// drop the echo.
fn mark_rest_origin(event: &roko_core::DashboardEvent) {
    let key = format!("{event:?}");
    let now = current_ts_millis();
    let mut buf = REST_ECHO_SUPPRESS.lock().expect("REST_ECHO_SUPPRESS poisoned");
    buf.push_back((now, key));
    while buf.len() > REST_ECHO_SUPPRESS_CAP {
        buf.pop_front();
    }
}

/// Returns true if `event` was previously marked by [`mark_rest_origin`].
/// On hit, removes the entry so an orchestrator-originated duplicate of the
/// same payload still passes through (single suppression per echo).
fn take_rest_origin(event: &roko_core::DashboardEvent) -> bool {
    let key = format!("{event:?}");
    let mut buf = REST_ECHO_SUPPRESS.lock().expect("REST_ECHO_SUPPRESS poisoned");
    if let Some(pos) = buf.iter().position(|(_, k)| k == &key) {
        buf.remove(pos);
        true
    } else {
        false
    }
}

/// Local copy of the EventBus timestamp helper to avoid importing the private one.
fn current_ts_millis() -> u64 {
    std::time::SystemTime::now()
        .duration_since(std::time::UNIX_EPOCH)
        .map(|d| u64::try_from(d.as_millis()).unwrap_or(u64::MAX))
        .unwrap_or(0)
}
```

### Step 3: Suppress echoes in `start_orchestrator_event_bridge`

```rust
pub fn start_orchestrator_event_bridge(state: Arc<AppState>) -> JoinHandle<()> {
    let mut rx = state.state_hub.subscribe_events();
    let bus = state.event_bus.clone();
    tokio::spawn(async move {
        loop {
            match rx.recv().await {
                Ok(envelope) => {
                    // Drop the echo of any event we ourselves pushed via
                    // start_state_hub_bridge. This breaks the REST→EventBus
                    // ↔ StateHub bidirectional loop while preserving genuine
                    // orchestrator-originated events.
                    if take_rest_origin(&envelope.payload) {
                        continue;
                    }
                    if let Some(server_event) = dashboard_event_to_server(&envelope.payload) {
                        bus.publish(server_event);
                    }
                }
                Err(tokio::sync::broadcast::error::RecvError::Lagged(n)) => {
                    warn!(n, "orchestrator bridge lagged behind state hub");
                }
                Err(tokio::sync::broadcast::error::RecvError::Closed) => break,
            }
        }
    })
}
```

### Step 5: Regression test

At the bottom of `crates/roko-serve/src/lib.rs` (or in a new `#[cfg(test)] mod bridge_loop_tests { ... }` block near the existing tests), add:
```rust
#[cfg(test)]
mod bridge_loop_tests {
    use super::*;
    use roko_core::DashboardEvent;
    use std::sync::atomic::{AtomicUsize, Ordering};
    use std::sync::Arc;

    /// Reduced state harness: just an EventBus<ServerEvent> + a SharedStateHub
    /// + the two bridges. Asserts that one REST publish produces exactly one
    /// downstream EventBus delivery to a fresh subscriber.
    #[tokio::test]
    async fn rest_event_delivered_exactly_once() {
        // Build minimal state needed by the bridges. If `AppState::test_only_minimal()`
        // exists, use it; otherwise build pieces directly:
        let state = Arc::new(AppState::test_only_minimal());

        // Subscribe BEFORE bridges start so we observe every published event.
        let mut subscriber = state.event_bus.subscribe();

        let _bridge_in = start_state_hub_bridge(Arc::clone(&state));
        let _bridge_out = start_orchestrator_event_bridge(Arc::clone(&state));

        // Simulate a REST endpoint emitting on the bus.
        state.event_bus.publish(ServerEvent::PlanStarted {
            plan_id: "p1".to_string(),
        });

        // Drain delivered events for ~150ms.
        let received = Arc::new(AtomicUsize::new(0));
        let received_for_task = Arc::clone(&received);
        let drain = tokio::spawn(async move {
            let timeout = tokio::time::sleep(std::time::Duration::from_millis(150));
            tokio::pin!(timeout);
            loop {
                tokio::select! {
                    _ = &mut timeout => break,
                    msg = subscriber.recv() => {
                        match msg {
                            Ok(envelope) => {
                                if matches!(envelope.payload, ServerEvent::PlanStarted { ref plan_id } if plan_id == "p1") {
                                    received_for_task.fetch_add(1, Ordering::Relaxed);
                                }
                            }
                            Err(_) => break,
                        }
                    }
                }
            }
        });
        let _ = drain.await;

        assert_eq!(
            received.load(Ordering::Relaxed),
            1,
            "REST-originated event must be delivered exactly once after bridge loop fix",
        );
    }
}
```

If `AppState::test_only_minimal()` does not exist, the test should build the EventBus and StateHub directly and call the bridge functions with whatever signatures fit; the *requirement* is that the test asserts `received_count == 1` for a single REST publish with both bridges live.
