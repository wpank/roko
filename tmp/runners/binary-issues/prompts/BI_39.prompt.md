## Problem
`crates/roko-agent/src/claude_agent.rs:29` defines `pub const DEFAULT_BASE_URL: &str = "https://api.anthropic.com"`, and `ClaudeAgent::new` (line 198) hardcodes that constant into `self.base_url`. The override path is `with_base_url(...)` (line 228), which means every caller that wants to honor `ProviderConfig::base_url` has to manually construct the agent and then call `with_base_url`. One existing caller — `crates/roko-agent/src/provider/anthropic_api.rs:56-57` — does exactly that:

```rust
let mut agent = ClaudeAgent::new(api_key, model.slug.clone())
    .with_base_url(Self::base_url(provider))
```

But two adjacent paths skip the `with_base_url` call:

- `crates/roko-agent/src/provider/anthropic_api/tool_loop.rs:243` constructs an inline payload with `base_url: DEFAULT_BASE_URL.to_string()` and never consults `ProviderConfig`.
- The synthesized `ProviderConfig` at `crates/roko-agent/src/provider/mod.rs:208-219` (the fallback when neither model nor provider config is registered) sets `base_url: None`, so the consumer must remember to fall back to the const.

The result is that "wire the base URL through provider config" depends on every adapter author remembering the manual chain. Universal rule **H2** in `00-RULES.md` is the opposite: a `pub const DEFAULT_FOO` is fine *as a fallback*, but the function that needs the value must accept an override and resolve it once.

The fix is a focused constructor — `ClaudeAgent::from_provider` — that does the resolve-or-fallback in one place. Adapters then have one obvious call to make. The `tool_loop.rs` path is **out of scope here** (its scope file is not on the batch list); it will migrate in a follow-up batch once the constructor exists.

## Required Changes

### Step 1: Add `ClaudeAgent::from_provider`

In `crates/roko-agent/src/claude_agent.rs`, **at the top of the existing `impl ClaudeAgent` block**, immediately after `pub fn new(...)` (currently lines 197-215), add:

```rust
    /// Construct a `ClaudeAgent` whose base URL and timeout come from a
    /// resolved [`ProviderConfig`].
    ///
    /// Behavior:
    /// - If `provider.base_url` is `Some(url)`, that URL is used (trailing
    ///   slashes stripped, identical to [`with_base_url`](Self::with_base_url)).
    /// - If `provider.base_url` is `None`, [`DEFAULT_BASE_URL`] is used as a
    ///   fallback. The constant remains the single hardcoded fallback for
    ///   the Anthropic Messages API; consumers should reach it through this
    ///   constructor rather than reading [`DEFAULT_BASE_URL`] directly.
    /// - If `provider.timeout_ms` is `Some(ms)`, that timeout is used; the
    ///   default of 120 s is preserved when the field is `None`.
    ///
    /// Other [`ClaudeAgent`] knobs (max tokens, system prompt, tools,
    /// extra headers, anthropic version) remain untouched and can be added
    /// via the existing builder methods. See BI_40 for the API-version
    /// counterpart of this constructor.
    #[must_use]
    pub fn from_provider(
        api_key: impl Into<String>,
        model: impl Into<String>,
        provider: &roko_core::config::schema::ProviderConfig,
    ) -> Self {
        let mut agent = Self::new(api_key, model);
        if let Some(base_url) = provider.base_url.as_deref() {
            agent = agent.with_base_url(base_url);
        }
        if let Some(ms) = provider.timeout_ms {
            agent = agent.with_timeout_ms(ms);
        }
        agent
    }
```

The fully qualified `roko_core::config::schema::ProviderConfig` path mirrors how `provider/mod.rs:50` and `provider/anthropic_api.rs:7` import it; `claude_agent.rs` does **not** currently import that type and an extra `use` line is unnecessary for a single reference. (If you prefer adding a `use roko_core::config::schema::ProviderConfig;` near the existing `use roko_core::{...};` block on line 21 for readability, that is equivalent and acceptable.)
