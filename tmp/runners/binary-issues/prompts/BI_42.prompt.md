### Step 2: Rewrite `naive_opus_cost`

Replace `crates/roko-cli/src/chat_inline.rs:4268-4271` with:

```rust
/// Compute naive opus-rate baseline cost for savings comparison.
///
/// Uses [`CostTable::lookup`] for the [`NAIVE_BASELINE_MODEL`] slug. When
/// the model is missing from the table (which should not happen because
/// `CostTable::with_defaults` seeds it), the per-million rates fall back
/// to the historical 15.0 input / 75.0 output values to keep call sites
/// from panicking. The fallback values are the same numbers that lived
/// inline in this function before BI_42 — the move centralizes ownership
/// without changing the baseline when the table is well-formed.
fn naive_opus_cost(table: &CostTable, input_tokens: u64, output_tokens: u64) -> f64 {
    const FALLBACK_INPUT_PER_M: f64 = 15.0;
    const FALLBACK_OUTPUT_PER_M: f64 = 75.0;

    let pricing = table.lookup(NAIVE_BASELINE_MODEL);
    let input_per_m = pricing
        .map(|p| p.input_per_m)
        .unwrap_or(FALLBACK_INPUT_PER_M);
    let output_per_m = pricing
        .map(|p| p.output_per_m)
        .unwrap_or(FALLBACK_OUTPUT_PER_M);

    (input_tokens as f64 * input_per_m / 1_000_000.0)
        + (output_tokens as f64 * output_per_m / 1_000_000.0)
}
```

Keep the function `fn` (private to the file). The two `FALLBACK_*` constants stay inline as last-resort fallbacks per `00-RULES.md` rule **H2** ("Hardcoded as fallback only").
