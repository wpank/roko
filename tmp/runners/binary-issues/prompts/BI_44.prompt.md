# BI_44: Perplexity URL/model through web_search config

## Task
Introduce a `WebSearchConfig` value type inside `crates/roko-std/src/tool/builtin/web_search.rs` that holds the runtime-overridable knobs (`provider`, `url`, `model`, `api_key_env`) and replace direct reads of `PERPLEXITY_API_URL` / `DEFAULT_MODEL` / the literal `"PERPLEXITY_API_KEY"` string with reads through that struct. The existing `pub const` values remain as fallbacks. Add an opt-in `Handler::with_config` constructor so callers (and a future `[tool.web_search]` config section) can override defaults without touching the tool code.

## Runner Context
Group **BI_HRD**, batch 44 of 56. Closes MASTER-INDEX § **S7.9**. Deps: none.

## Problem
`crates/roko-std/src/tool/builtin/web_search.rs:31-35` defines:

```rust
const PERPLEXITY_API_URL: &str = "https://api.perplexity.ai/chat/completions";
const DEFAULT_MODEL: &str = "sonar";
```

and the executor reads them directly:

- `web_search.rs:125` — `"model": DEFAULT_MODEL,` inside the JSON body for `call_perplexity`.
- `web_search.rs:162` — `client.post(PERPLEXITY_API_URL)` for the HTTP call.
- `web_search.rs:188` — `model: DEFAULT_MODEL.to_string(),` for the gateway path.
- `web_search.rs:332` — `std::env::var("PERPLEXITY_API_KEY")` literal env name.

Every operator who wants to point web search at a different provider (e.g. a self-hosted gateway, an OpenAI-compatible search endpoint, a non-default Perplexity model like `sonar-pro`) has to fork the source. Universal rule **H1** in `00-RULES.md`: hardcoded provider URLs / model slugs are owned by config; library modules read.

This batch:

1. Adds a `WebSearchConfig` struct local to `web_search.rs` with the four fields the user spec calls out (`provider`, `url`, `model`, `api_key_env`). Default values keep behavior identical.
2. Routes every literal read through that struct.
3. Adds a `Handler::with_config(...)` constructor so callers can pass in overrides.
4. Adds a forward-compatible reader stub for `[tool.web_search]` config (mirrors the BI_43 pattern).

The schema field on `RokoConfig` itself — `tool.web_search` — is **out of scope** for this batch (its scope is `crates/roko-core/src/config/mod.rs` as `also_read` only). The follow-up batch will add the schema; until then operators override via `Handler::with_config(...)` from the call sites that build the tool registry.

### Step 2: Add the `WebSearchConfig` struct

Immediately after the constants (before the `static WARNED_DIRECT_API_KEY_PATH` line), add:

```rust
/// Runtime configuration for [`Handler`].
///
/// All fields are `Option` so callers can override one knob without
/// committing to the others; the `None` branches fall back to the
/// `DEFAULT_*` / `PERPLEXITY_*` constants in this module so behavior is
/// unchanged when the struct is left at its default.
///
/// The intended long-term home of this struct is the
/// `[tool.web_search]` section of `roko.toml`; until that schema lands
/// (a follow-up batch in BI_HRD), callers can build the struct directly
/// and pass it to [`Handler::with_config`].
#[derive(Clone, Debug, Default, PartialEq, Eq)]
pub struct WebSearchConfig {
    /// Provider id (e.g. `"perplexity"`, `"openai"`, `"self-hosted"`).
    /// Currently informational; future batches will switch on this value
    /// to route through different adapters.
    pub provider: Option<String>,
    /// Override URL for the search-grounded chat/completions endpoint.
    pub url: Option<String>,
    /// Override model slug.
    pub model: Option<String>,
    /// Environment variable name to read the API key from.
    pub api_key_env: Option<String>,
}

impl WebSearchConfig {
    /// Provider id, falling back to [`DEFAULT_PROVIDER`].
    #[must_use]
    pub fn provider_or_default(&self) -> &str {
        self.provider.as_deref().unwrap_or(DEFAULT_PROVIDER)
    }

    /// Endpoint URL, falling back to [`PERPLEXITY_API_URL`].
    #[must_use]
    pub fn url_or_default(&self) -> &str {
        self.url.as_deref().unwrap_or(PERPLEXITY_API_URL)
    }

    /// Model slug, falling back to [`DEFAULT_MODEL`].
    #[must_use]
    pub fn model_or_default(&self) -> &str {
        self.model.as_deref().unwrap_or(DEFAULT_MODEL)
    }

    /// API-key environment variable name, falling back to
    /// [`DEFAULT_API_KEY_ENV`].
    #[must_use]
    pub fn api_key_env_or_default(&self) -> &str {
        self.api_key_env.as_deref().unwrap_or(DEFAULT_API_KEY_ENV)
    }
}

/// Forward-compatible reader for `[tool.web_search]` in `RokoConfig`.
///
/// Returns `None` while the schema is not declared. When the
/// `[tool.web_search]` section lands in a follow-up batch, replace this
/// stub with the real lookup; the rest of [`Handler`] needs no further
/// edits.
#[must_use]
pub fn web_search_config_from_env() -> Option<WebSearchConfig> {
    // BI_44 forward-compatibility hook. The roko.toml schema does not
    // currently declare a `[tool.web_search]` table; callers wire
    // overrides explicitly via `Handler::with_config` until then.
    None
}
```
