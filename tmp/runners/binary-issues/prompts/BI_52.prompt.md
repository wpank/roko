## Problem
`crates/roko-serve/src/routes/feeds.rs:127` calls `reg.get(&id).expect("just registered")` immediately after `reg.register(info)`. Per `00-RULES.md` rule **M3** (`expect("just registered")` is a panic — works until it doesn't) and rule **M2** (TOCTOU between register and get), an `expect` here means a single registry implementation bug crashes the whole `roko serve` worker thread instead of returning a clean 500 to the client.

The function already returns `Result<(StatusCode, Json<CreateFeedResponse>), ApiError>` and `ApiError::internal(msg)` exists at `crates/roko-serve/src/error.rs:69-76` (returns `StatusCode::INTERNAL_SERVER_ERROR`, `code: "internal_error"`, configurable `message`). `tracing` is already a dependency of `roko-serve` (used elsewhere in the crate); no new imports beyond `tracing::error` are required.
