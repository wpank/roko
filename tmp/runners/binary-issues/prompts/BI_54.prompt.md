## Task

## Problem
Per `MASTER-INDEX § S1.3` and the verification log in `context-pack/02-VERIFICATION-NOTES.md`, `ModelCallRequest` (`crates/roko-core/src/foundation.rs:22-65`) has no `tools` field. Tools currently flow only through `AgentOptions::tools: Option<String>` (`crates/roko-agent/src/provider/mod.rs:528`), which is the legacy Claude-CLI `--tools` CSV. That means:

- Non-Claude provider adapters can never see the tool list.
- Knowledge / observability sinks see `req.knowledge_ids` and `req.prompt_section_ids` but no `req.tools`.
- The chat path's `allowed_tools_csv` (`crates/roko-cli/src/chat_session.rs:322`) cannot be observed via `ModelCallRequest`.

The canonical tool type already exists: `roko_core::tool::ToolDef` is defined at `crates/roko-core/src/tool/def.rs:303-327` and re-exported from `crates/roko-core/src/lib.rs:291`. Per **A7** ("One trait, one definition"), this batch must use it — not introduce a parallel type.

This batch is the **typed-field plumbing only**. Wiring native (Anthropic / OpenAI / Gemini) tool serialization downstream is left for a follow-up batch in `BI_PRT`. The mechanical contract here is: chat session writes typed tools into the request → `ModelCallService` reads them and forwards to the existing `AgentOptions::tools` CSV (preserving Claude CLI behaviour exactly). Future batches will replace the CSV bridge with native per-provider serialization.

    fn model_call_request(&self, prompt: &str) -> ModelCallRequest {
        let mut messages = self.api_history.clone();
        messages.push(ChatMessage {
            role: MessageRole::User,
            content: prompt.to_string(),
        });

        // BI_54: Lift the legacy CSV allowlist into typed ToolDef entries so
        // ModelCallService and downstream observers see the chat path's tool
        // surface. We only know the names here; provider adapters that need
        // descriptions/schemas look the tool up in the dispatcher's
        // ToolRegistry. ToolCategory::Read + ToolPermission::ReadOnly is the
        // safe default — the dispatcher's registry overrides on lookup.
        let tools = if self.allowed_tools_csv.is_empty() {
            Vec::new()
        } else {

                .map(str::trim)

                .map(|name| {
                    ToolDef::new(name, "", ToolCategory::Read, ToolPermission::ReadOnly)
                })
                .collect()
        };

        ModelCallRequest {
            model: self.model_call_model_key(),
            system: (!self.system_prompt.is_empty()).then(|| self.system_prompt.clone()),
            messages,
            max_tokens: Some(4096),
            temperature: None,
            role: Some("chat".to_string()),
            caller: Some(caller::CLI.to_string()),
            run_id: None,
            prompt_section_ids: Vec::new(),
            knowledge_ids: Vec::new(),
            budget: None,
            budget_remaining: None,
            routing_hints: Vec::new(),
            tools,
            cache_policy: CachePolicy::Bypass,
        }

### Step 3: Forward `req.tools` to `AgentOptions::tools` in `model_call_service.rs`

In `crates/roko-agent/src/model_call_service.rs`, modify `build_agent_options` (lines 399-416) to read `req.tools` and project it to the legacy CSV expected by `AgentOptions::tools`:

```rust
fn build_agent_options(
    &self,
    req: &ModelCallRequest,
    system_prompt: Option<String>,
) -> AgentOptions {
    // BI_54: Bridge the new typed `req.tools` to the legacy
    // `AgentOptions::tools` CSV so the Claude CLI adapter keeps working.
    // Native HTTP adapters (Anthropic / OpenAI / Gemini) will consume
    // `req.tools` directly in a follow-up batch; until then this CSV is the
    // single observable signal for tool exposure.
    let tools_csv = if req.tools.is_empty() {
        None
    } else {
        let names: Vec<&str> = req
            .tools
            .iter()
            .map(|tool| tool.name.as_str())
            .filter(|name| !name.is_empty())
            .collect();
        (!names.is_empty()).then(|| names.join(","))
    };

    let mut options = AgentOptions {
        system_prompt,
        mcp_config: self.mcp_config.clone(),
        name: req.role.clone().unwrap_or_else(|| "model_call".to_string()),
        env: self.env.clone(),
        tools: tools_csv,
        ..AgentOptions::default()
    };
    options.mcp_config = self
        .mcp_config
        .clone()
        .or_else(|| self.config_agent_mcp_config());
    options
}
```

`AgentOptions::tools` is `Option<String>` (`crates/roko-agent/src/provider/mod.rs:528`). When `req.tools` is empty (the default), the CSV stays `None`, which preserves today's behaviour for every caller that does not set the field.
