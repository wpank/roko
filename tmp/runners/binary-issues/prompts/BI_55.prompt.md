# BI_55: Cache and reuse provider agent in ProviderCallCell::execute

## Task
In `crates/roko-agent/src/model_call_service.rs`, replace the per-attempt `create_agent_for_model(...)` call inside `ProviderCallCell::execute` (lines 1280-1283) with a process-local agent cache keyed by `(provider, model)`. The cache stores `Arc<dyn Agent>` (rebuilt only on cache miss) and is owned by `ProviderCallCell`. Subsequent calls for the same `(provider, model)` reuse the same agent. Cache invalidation is per-`ProviderCallCell` lifetime — since the cell is constructed fresh per `ModelCallService::call` (`model_call_service.rs:1511`), the natural invalidation seam already exists at the `ModelCallService` level for config changes.

## Problem
Per `MASTER-INDEX § S2.4`, every `ModelCallService::call` constructs a fresh agent in `ProviderCallCell::execute`:

```rust
for (attempt_index, attempt_model) in std::iter::once(model)
    .chain(fallback_models.iter().map(String::as_str))
    .enumerate()
{
    let mut attempt_options = options.clone();
    attempt_options.system_prompt = system_prompt.map(ToOwned::to_owned);

    let agent = create_agent_for_model(&self.config, attempt_model, attempt_options)
        .map_err(|err| CellError::Construction { message: err.to_string() })?;
    // ...
}
```

`create_agent_for_model` (`crates/roko-agent/src/provider/mod.rs:135-…`) walks the provider table, applies safety layers, builds HTTP clients, parses extra args, and constructs the right adapter. Doing this on every attempt — including identical retries — wastes 7-43 ms of CPU and prevents adapters from holding warm state (e.g. cached provider semaphores, prepared header maps) across calls.

The fix is a **per-cell** agent cache. `ProviderCallCell` is currently a tiny struct with `config` and `cost_table` fields; we add a `Mutex<HashMap<(String, String), Arc<dyn Agent>>>` cache keyed by `(provider, model)` and a `get_or_build` helper. On miss we call `create_agent_for_model` once and store the result; on hit we return a cheap `Arc::clone`. `parking_lot::Mutex` is already a workspace dep (`crates/roko-agent/Cargo.toml:31`) and BI_50 / BI_51 also pull it into this crate.

The `Agent` trait (`crates/roko-agent/src/agent.rs:142-165`) is `Send + Sync` and `agent.run(&prompt, &ctx).await` already takes `&self`, so wrapping in `Arc<dyn Agent>` and reusing across attempts is sound.

`ProviderCallCell` is constructed in `ModelCallService::call` at `model_call_service.rs:1511`:

```rust
let cell = ProviderCallCell::new(config, self.cost_table.clone());
```

Because the cell is created fresh per top-level `ModelCallService::call`, the cache is reused **within** a single user-facing call (across the primary model + fallback models, plus any retries the cell might issue). Cross-call reuse is **explicitly out of scope** for this batch — that requires moving the cache up to `ModelCallService` itself, which has different lifetime semantics around config reloads. A `// TODO(BI_PRT follow-up)` comment marks the seam.

### Step 2: Update `ProviderCallCell::new` and add `get_or_build_agent`

Replace the constructor at `model_call_service.rs:1248-1250`:

with:

```rust
impl ProviderCallCell {
    fn new(config: RokoConfig, cost_table: CostTable) -> Self {
        Self {
            config,
            cost_table,
            agent_cache: parking_lot::Mutex::new(HashMap::new()),
        }
    }

    /// Look up (provider, model) in the agent cache, building once on miss.
    ///
    /// Builds with the **caller-supplied options** on miss. Subsequent calls
    /// in the same cell ignore later `options` mutations (e.g. a different
    /// `system_prompt`) — see the comment in `execute` for why this is safe
    /// in this batch's scope.
    fn get_or_build_agent(
        &self,
        attempt_model: &str,
        attempt_options: AgentOptions,
    ) -> std::result::Result<Arc<dyn Agent>, CellError> {
        let provider_key = self
            .config
            .effective_models()
            .get(attempt_model)
            .map(|profile| profile.provider.clone())
            .unwrap_or_default();
        let cache_key = (provider_key, attempt_model.to_string());

        if let Some(existing) = self.agent_cache.lock().get(&cache_key).cloned() {
            return Ok(existing);
        }

        let built = create_agent_for_model(&self.config, attempt_model, attempt_options)
            .map_err(|err| CellError::Construction { message: err.to_string() })?;
        let shared: Arc<dyn Agent> = Arc::from(built);

        // Insert and return the just-built agent. If two attempts race to
        // build the same key, both end up with valid agents; the loser's
        // build is discarded but no incorrect state is shared.
        let mut guard = self.agent_cache.lock();
        guard
            .entry(cache_key)
            .or_insert_with(|| shared.clone());
        Ok(shared)
    }
}
```

The `Agent` trait already requires `Send + Sync` (verified at `crates/roko-agent/src/agent.rs:143`), so storing in `Arc` is sound.
