# BI_56: Stream HTTP DispatchMode::Http deltas

## Task
Bring HTTP `DispatchMode::Http` to streaming parity with `DispatchMode::Session`. Inside `crates/roko-cli/src/chat_inline.rs`, install a `streaming_event_rx` channel on the session for the HTTP arm of `dispatch_prompt` (`chat_inline.rs:1881-1900`), and rewrite `send_and_receive` (`chat_inline.rs:4312-4455`) to take that `event_tx` and emit `roko_agent::AgentRuntimeEvent::MessageDelta { text }` events as the assistant text becomes available ā€” both in the synchronous-JSON case and in the run-id polling loop. The TUI's existing event-loop drain (already wired for Session mode at `chat_inline.rs:1348-1374` and `1714-1736`) flips `Phase::Thinking ā†’ Phase::Streaming` and appends to `session.streaming` on the first delta, identical to today's Session-mode UX.

This batch keeps the same pattern as `crates/roko-agent/src/openai_compat_backend.rs::send_turn_streaming` ā€” a single producer that shovels chunks into an mpsc channel that the TUI drains per frame. We do **not** add a new server-side SSE endpoint: roko-serve's `/api/agents/{id}/message` and `/api/run/{run_id}/status` stay as they are, and we synthesise per-poll deltas from `output_text` length growth (the `status` endpoint already returns the in-progress text).

## Problem
`DispatchMode::Session` already streams (`chat_inline.rs:1614` consumes `streaming_event_rx`; the producer lives in `chat_session.rs::send_turn_streaming`). `DispatchMode::Http` does not ā€” the relevant code is `chat_inline.rs:4312-4455` (`send_and_receive`), called from `dispatch_prompt::DispatchMode::Http` at `chat_inline.rs:1892-1899`. Today's HTTP path:

1. POSTs `{base_url}/api/agents/{agent_id}/message` (or `{base_url}/message` for sidecar).
2. If the response includes `response: Option<String>`, returns it as a single complete payload.
3. Otherwise polls `/api/run/{run_id}/status` every 500 ms until `finished: true`, returning the final `output_text`.

The TUI sits in `Phase::Thinking` for the entire wall-clock duration ā€” no incremental tokens, no spinner movement beyond elapsed time. A long agent reply that takes 25 s feels broken.

The fix mirrors Session mode without requiring server-side changes: install the `streaming_event_rx` on the session in the HTTP arm, hand its `event_tx` clone to a `send_and_receive_streaming` variant, and emit `MessageDelta` events whenever new text becomes visible ā€” once for the synchronous response, or per-poll-delta for the polling loop. The existing event-loop drain (which already handles `MessageDelta`, `ToolCall`, `ToolOutput` for Session mode) requires no change.

```rust
/// Streaming variant of [`send_and_receive`] that emits assistant text deltas
/// to `event_tx` as they become available.
///
/// BI_56: Parity with Session mode (which streams via
/// `ChatAgentSession::send_turn_streaming`). The HTTP path has no SSE today,
/// so we synthesise deltas:
/// - Synchronous JSON response: emit one `MessageDelta` with the full text
///   before returning, so the TUI gets at least one delta to switch into
///   `Phase::Streaming` for the brief moment before the final result lands.
/// - run-id polling: each `/status` poll returns the in-progress
///   `output_text`; we emit the *new tail* (text beyond the last delta we
///   sent) as a `MessageDelta`. When `finished` flips to `true`, we emit
///   any final tail before returning.
///
/// `event_tx` is best-effort ā€” if the receiver has been dropped (e.g. the TUI
/// loop already moved on), every send is silently ignored.
async fn send_and_receive_streaming(
    client: &reqwest::Client,
    base_url: &str,
    agent_id: &str,
    message: &str,
    is_sidecar: bool,
    event_tx: tokio::sync::mpsc::Sender<roko_agent::AgentRuntimeEvent>,
) -> Result<HttpResponse> {
    let poll_base = base_url;
    let (url, body) = if is_sidecar {
        (
            format!("{}/message", base_url.trim_end_matches('/')),
            json!({ "prompt": message }),
        )
    } else {
        (
            format!(
                "{}/api/agents/{agent_id}/message",
                base_url.trim_end_matches('/')
            ),
            json!({ "message": message }),
        )
    };

    let response = client
        .post(&url)
        .json(&body)
        .timeout(Duration::from_secs(120))
        .send()
        .await
        .with_context(|| format!("POST {url} ā€” is `roko serve` running?"))?;

    if !response.status().is_success() {
        let status = response.status();
        let body_text = response.text().await.unwrap_or_default();
        let detail = serde_json::from_str::<serde_json::Value>(&body_text)
            .ok()
            .and_then(|v| v.get("error").and_then(|e| e.as_str()).map(String::from))
            .unwrap_or(body_text);
        bail!("{status}: {detail}");
    }

    #[derive(Deserialize)]
    struct Resp {
        #[serde(default)]
        response: Option<String>,
        #[serde(default)]
        run_id: Option<String>,
        #[serde(default)]
        model: Option<String>,
        #[serde(default)]
        input_tokens: Option<u64>,
        #[serde(default)]
        output_tokens: Option<u64>,
    }

    let resp: Resp = response.json().await.context("decode response")?;
    let resp_model = resp.model.unwrap_or_default();
    let resp_input = resp.input_tokens.unwrap_or(0);
    let resp_output = resp.output_tokens.unwrap_or(0);

    if let Some(text) = resp.response.filter(|s| !s.trim().is_empty()) {
        let clean = extract_clean_text(&text);

        // Emit one MessageDelta so the TUI flips to Phase::Streaming and the
        // user sees the response before the final response_rx delivers it.
        let _ = event_tx
            .send(roko_agent::AgentRuntimeEvent::MessageDelta { text: clean.clone() })
            .await;

        let input_tokens = if resp_input > 0 {
            resp_input
        } else {
            (message.len() as u64) / 4
        };
        let output_tokens = if resp_output > 0 {
            resp_output
        } else {
            (clean.len() as u64) / 4
        };
        let model = if resp_model.is_empty() {
            "http".to_string()
        } else {
            resp_model
        };
        return Ok(HttpResponse {
            text: clean,
            model,
            input_tokens,
            output_tokens,
        });
    }

    if let Some(run_id) = resp.run_id.filter(|s| !s.trim().is_empty()) {
        let status_url = format!(
            "{}/api/run/{run_id}/status",
            poll_base.trim_end_matches('/'),
        );

        // The slice of `output_text` we have already streamed to the TUI.
        // Each poll compares the latest server-side text to this prefix and
        // emits the unseen tail as a single MessageDelta.
        let mut streamed_prefix_len: usize = 0;

        loop {
            tokio::time::sleep(Duration::from_millis(500)).await;
            let status_resp = client
                .get(&status_url)
                .send()
                .await
                .context("poll status")?;

            #[derive(Deserialize)]
            struct StatusResp {
                #[serde(default)]
                finished: bool,
                #[serde(default)]
                output_text: Option<String>,
                #[serde(default)]
                error: Option<String>,
                #[serde(default)]
                model: Option<String>,
                #[serde(default)]
                input_tokens: Option<u64>,
                #[serde(default)]
                output_tokens: Option<u64>,
            }

            let status: StatusResp = status_resp.json().await.context("decode status")?;
            let in_progress = status.output_text.clone().unwrap_or_default();

            // Emit any new tail since the last poll. Indexing on a str byte
            // boundary is safe here because we always advance to the latest
            // `in_progress.len()` (a valid char boundary at the end of the
            // string), and `streamed_prefix_len` was set to a previous
            // `in_progress.len()` from the same JSON payload shape.
            if in_progress.len() > streamed_prefix_len
                && in_progress.is_char_boundary(streamed_prefix_len)
            {
                let new_tail = in_progress[streamed_prefix_len..].to_string();
                if !new_tail.is_empty() {
                    let _ = event_tx
                        .send(roko_agent::AgentRuntimeEvent::MessageDelta { text: new_tail })
                        .await;
                    streamed_prefix_len = in_progress.len();
                }
            }

            if status.finished {
                if let Some(err) = status.error.filter(|s| !s.trim().is_empty()) {
                    bail!("agent error: {err}");
                }
                let text = status.output_text.unwrap_or_default();
                let input_tokens = status
                    .input_tokens
                    .unwrap_or_else(|| (message.len() as u64) / 4);
                let output_tokens = status
                    .output_tokens
                    .unwrap_or_else(|| (text.len() as u64) / 4);
                let model = status.model.unwrap_or_else(|| "http".to_string());
                return Ok(HttpResponse {
                    text,
                    model,
                    input_tokens,
                    output_tokens,
                });
            }
        }
    }

    bail!("no response or run_id in reply");
}
```
