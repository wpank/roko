# A01: RuntimeEvent Serde Envelope

## Task

Add proper serde serialization to `RuntimeEvent` in `crates/roko-core/src/runtime_event.rs`.

Currently `RuntimeEvent` is a plain enum used only for in-memory dispatch and debug-string logging.
Make it a durable, versioned event envelope suitable for JSONL persistence and typed projection.

## Changes Required

1. Add these fields to a new `RuntimeEventEnvelope` wrapper (or extend the existing enum):
   - `run_id: String` — identifies the workflow run
   - `seq: u64` — monotonic sequence number within the run
   - `ts: chrono::DateTime<chrono::Utc>` — event timestamp (use `chrono` if available, or `std::time::SystemTime` serialized as ISO 8601)
   - `schema_version: u8` — starts at `1`, for forward compatibility
   - `source: String` — which component emitted the event (e.g., "workflow_engine", "effect_driver")
   - `payload: RuntimeEvent` — the existing enum variants

2. Derive `Serialize, Deserialize, Debug, Clone` on both the envelope and all `RuntimeEvent` variants.

3. Ensure every variant's fields are serializable. Replace any non-serializable types (like `Box<dyn Error>`) with `String` error messages.

4. Add a `RuntimeEventEnvelope::new(run_id, seq, source, payload)` constructor that sets `ts` to now and `schema_version` to 1.

## Acceptance Criteria

- `serde_json::to_string(&envelope)` produces valid JSON for every variant.
- `serde_json::from_str::<RuntimeEventEnvelope>(&json)` round-trips correctly.
- No `format!("{:?}", ...)` is used for event persistence anywhere in this file.
- Existing code that pattern-matches on `RuntimeEvent` variants still compiles.

## Do NOT

- Change the semantic meaning of existing variants.
- Add new event variants (that's for later batches).
- Modify any file outside the scope.
