# A03: Extend ModelCallRequest and FeedbackEvent Metadata

## Task

Extend the foundation `ModelCallRequest` and `FeedbackEvent` types with metadata fields needed
for full observability and learning loops.

## Changes Required

### In `crates/roko-core/src/foundation.rs`:

1. Add to `ModelCallRequest`:
   - `caller: Option<String>` — who initiated the call (e.g., "workflow_engine", "gateway")
   - `role: Option<String>` — agent role (e.g., "implementer", "reviewer")
   - `run_id: Option<String>` — workflow run identifier
   - `prompt_section_ids: Vec<String>` — which prompt sections were assembled
   - `knowledge_ids: Vec<String>` — which knowledge entries were included
   - `budget_remaining: Option<f64>` — remaining token/cost budget
   - `routing_hints: Vec<String>` — hints for model routing (e.g., "prefer-fast", "needs-tool-use")

2. Add to `FeedbackEvent` (or create it if only `FeedbackSink` exists):
   - `run_id: Option<String>`
   - `request_id: Option<String>` — correlates with the model call
   - `prompt_section_ids: Vec<String>` — which prompt sections produced this outcome
   - `knowledge_ids: Vec<String>` — which knowledge entries were relevant
   - `model: Option<String>` — which model was actually used
   - `provider: Option<String>` — which provider backend
   - `token_usage: Option<u64>` — tokens consumed
   - `cost: Option<f64>` — estimated cost

3. Ensure all new fields have `Default` values (empty vecs, `None`) so existing construction
   sites compile without modification. Use `#[serde(default)]` on all new fields.

## Acceptance Criteria

- `ModelCallRequest::default()` compiles (all new fields default to None/empty).
- `FeedbackEvent` can be serialized/deserialized with the new fields.
- `cargo check --workspace` passes — no existing code breaks.

## Do NOT

- Modify call sites to populate these fields yet (that's Wave C).
- Remove any existing fields.
