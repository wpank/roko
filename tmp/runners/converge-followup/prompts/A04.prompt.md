# A04: Replace Debug-String JSONL With Typed Event JSON

## Task

Replace the debug-string event logging in `JsonlLogger` and the debug-string parsing in
`RuntimeProjection` with proper serde JSON serialization/deserialization.

## Changes Required

### In `crates/roko-runtime/src/jsonl_logger.rs`:

1. Find the line that writes events as debug strings (approximately line 64-69):
   ```rust
   // BEFORE (bad):
   writeln!(file, "{event:?}")
   ```

2. Replace with typed JSON serialization:
   ```rust
   // AFTER (good):
   let json = serde_json::to_string(&envelope)?;
   writeln!(file, "{json}")
   ```

3. The logger should accept `RuntimeEventEnvelope` (from A01) and serialize it. If the logger
   currently takes `RuntimeEvent` directly, wrap it in an envelope before writing.

4. Add `use roko_core::runtime_event::RuntimeEventEnvelope;` (or whatever the envelope type
   from A01 is called).

### In `crates/roko-runtime/src/projection.rs`:

1. Find the debug-string parsing logic (approximately lines 161-181) that parses `Debug` output
   back into state.

2. Replace with `serde_json::from_str::<RuntimeEventEnvelope>(&line)` deserialization.

3. Update the projection to work with typed event envelopes. The projection should use
   pattern matching on `envelope.payload` to extract state.

## Acceptance Criteria

- No `format!("{:?}", event)` or `format!("{event:?}")` appears in event logging code.
- No string parsing of `Debug` output appears in projection code.
- Events written by `JsonlLogger` can be read back by `RuntimeProjection`.
- `cargo check -p roko-runtime` passes.

## Do NOT

- Change the projection's semantic behavior (what state it tracks).
- Add new projection queries.
