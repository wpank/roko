# A05: Contract Guard Tests

## Task

Add compile-time and test-time guards that prevent the contract anti-patterns from returning.

## Changes Required

### In `crates/roko-runtime/src/lib.rs` (or a new test module in the runtime crate):

1. Add a `#[cfg(test)]` module with these negative checks:

   a. **No duplicate foundation traits**: A test that greps `crates/roko-runtime/src/` for
      `pub trait AffectPolicy` and `pub trait DispatchModulation` and asserts they don't exist
      (since the canonical versions are in core).

   b. **No debug-string event serialization**: A test that reads `jsonl_logger.rs` source and
      asserts it does NOT contain `format!("{:?}"` or `"{event:?}"` patterns.

   c. **Event round-trip**: A test that creates a `RuntimeEventEnvelope` for each major variant,
      serializes to JSON, deserializes back, and asserts equality.

2. These tests use `include_str!` or `std::fs::read_to_string` to read source files at test
   time for the negative grep checks. This is intentional — it catches regressions at test time.

## Acceptance Criteria

- `cargo test -p roko-runtime` passes with these new tests.
- If someone re-introduces `format!("{:?}", event)` in the logger, the test fails.
- If someone defines a local `pub trait AffectPolicy` in runtime, the test fails.
- Event envelope round-trip works for at least: `WorkflowStarted`, `PhaseTransition`,
  `AgentSpawned`, `AgentCompleted`, `GateResult`, `WorkflowCompleted`.

## Do NOT

- Add tests for functionality that doesn't exist yet (that's Wave F).
- Modify the actual logger or projection code (that was A04).
