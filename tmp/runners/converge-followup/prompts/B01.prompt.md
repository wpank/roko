# B01: Agent Event Ownership

## Task

Make `EffectDriver` the sole owner of agent lifecycle events with real metadata.
Remove the empty/duplicate `AgentSpawned` events from `WorkflowEngine`.

## Problem

Currently both `WorkflowEngine` and `EffectDriver` emit `AgentSpawned` events:
- `WorkflowEngine` emits them at ~lines 146-182 with **empty** `agent_id` and `model`
- `EffectDriver` emits them at ~lines 165-189 also with empty fields

This creates duplicate, low-value lifecycle events. The driver is the right owner because it
knows the real model, request id, and provider result.

## Changes Required

### In `crates/roko-runtime/src/workflow_engine.rs`:

1. Remove the `AgentSpawned` event emissions at approximately lines 146-151, 155-160, 166-171,
   and 177-182. These are the pre-spawn events with empty metadata.

2. Keep any high-level workflow phase events (like `PhaseTransition`) that the engine legitimately
   owns. The engine should emit phase transitions; the driver should emit agent lifecycle.

### In `crates/roko-runtime/src/effect_driver.rs`:

1. Where `AgentSpawned` is emitted (~lines 165-170), fill in real values:
   - `agent_id`: generate a unique id (e.g., `format!("{}-{}", role, uuid::Uuid::new_v4())` or
     use a counter)
   - `model`: from the `ModelCallRequest` or resolved provider

2. After the agent completes, emit `AgentCompleted` with the real result metadata (tokens used,
   duration, model, success/failure).

3. Use the `RuntimeEventEnvelope` from A01 for all events. Set the `source` field to
   `"effect_driver"`.

## Acceptance Criteria

- `grep -n 'AgentSpawned.*String::new' crates/roko-runtime/` returns nothing.
- `WorkflowEngine` does NOT emit `AgentSpawned` or `AgentCompleted` events.
- `EffectDriver` emits exactly one `AgentSpawned` + one `AgentCompleted` per agent effect.
- All agent events have non-empty `agent_id` and `model` fields.

## Do NOT

- Change the engine's phase transition events.
- Add new event variants.
