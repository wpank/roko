# C01: Shared Service Factory

## Task

Create a single service construction path used by CLI, server, and ACP, replacing the
divergent construction in `run.rs:480-547` and `state.rs:479-482`.

## Problem

CLI constructs services one way in `run.rs`, server constructs them differently in `state.rs`.
This means different entry points have different service capabilities (e.g., server has no
feedback, CLI has no knowledge store in prompt assembly).

## Changes Required

### In `crates/roko-cli/src/run.rs`:

1. Extract the service construction at ~lines 480-547 into a `ServiceFactory` struct or a
   `build_services(config, roko_dir)` function that returns all services:
   - `ModelCallService`
   - `PromptAssemblyService`
   - `FeedbackService`
   - `GateService`
   - `EffectDriver` (or the components needed to build one)

2. The factory should accept a `ServiceConfig` or similar that controls:
   - Whether feedback is enabled (always yes for production, can be no-op for tests)
   - The roko directory path
   - The workspace config

3. Make the existing CLI v2 path use this factory.

### In `crates/roko-serve/src/state.rs`:

1. Replace the independent `ModelCallService` construction at ~lines 479-482 with a call to
   the same factory/builder.

2. If the factory is defined in `roko-cli`, consider whether it should live in a shared crate
   (like `roko-runtime` or `roko-core`). Alternatively, if moving it is too invasive, have
   `state.rs` import from `roko-cli` or duplicate the builder but ensure it matches.

3. The minimal path: define the factory as a public function in `roko-cli::run` and have the
   server import it. Or define a `ServiceBundle` struct in `roko-runtime`.

## Acceptance Criteria

- CLI and server construct services through the same builder/factory.
- Adding a new service to the factory automatically gives it to all entry points.
- `cargo check -p roko-cli -p roko-serve` passes.

## Do NOT

- Make services mandatory that aren't already (that's a separate concern).
- Refactor the entire server state — just align service construction.
