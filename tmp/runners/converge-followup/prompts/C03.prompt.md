# C03: Gateway Durability

## Task

Wire `GatewayEventWriter` into the live CLI and server model call paths so gateway events
are durably persisted. Make gateway stats read from durable state, not placeholders.

## Problem

- `GatewayEventWriter` exists in `gateway_events.rs` but is never instantiated in live paths.
- Server gateway stats contain placeholders at ~`gateway.rs:418-425`.
- Gateway calls at ~`gateway.rs:265-335` call `ModelCallService` but don't write durable events.

## Changes Required

### In `crates/roko-serve/src/routes/gateway.rs`:

1. After a successful model call at ~lines 265-335, write a durable gateway event using
   `GatewayEventWriter`. The event should include:
   - Request metadata (model, provider, caller)
   - Response metadata (tokens, cost, duration, success)
   - Cache hit/miss status

2. Replace placeholder stats at ~lines 418-425 with stats derived from reading durable
   gateway events (via `GatewayEventWriter::projection()` or a similar read path).

### In `crates/roko-agent/src/gateway_events.rs`:

1. Ensure `GatewayEventWriter` has a constructor that takes a file path
   (e.g., `.roko/learn/gateway.jsonl`) and appends events.

2. Ensure there's a projection/reader that can compute aggregate stats from the event log.

3. If these already exist but aren't called, just verify they work and move on.

## Acceptance Criteria

- Gateway model calls write durable events to `.roko/learn/gateway.jsonl` or equivalent.
- Gateway stats endpoint returns data from the durable event log, not hardcoded placeholders.
- `cargo check -p roko-serve -p roko-agent` passes.

## Do NOT

- Change the gateway routing or model call logic.
- Add new gateway endpoints.
