# C04: PromptAssemblyService Live Context

## Task

Wire knowledge store, episodes, playbooks, conventions, tool instructions, and section
effectiveness into the live `PromptAssemblyService` construction.

## Problem

Live CLI v2 constructs `PromptAssemblyService::new()` at `run.rs:526-529` with no knowledge
store, no episodes, no playbooks, no tool instructions, and no section effectiveness input.
The service has useful assembly layers at `prompt_assembly_service.rs:265-390`, but they
receive no data.

## Changes Required

### In `crates/roko-compose/src/prompt_assembly_service.rs`:

1. Review the existing builder/constructor. It likely accepts optional knowledge stores,
   episode logs, playbook queries, etc. If it only has `new()`, add a builder pattern or
   `with_*` methods:
   - `with_knowledge_store(store)` — for knowledge-informed prompt sections
   - `with_episodes(episodes_path)` — for episode-derived few-shot examples
   - `with_playbooks(playbooks)` — for role-specific playbook sections
   - `with_conventions(conventions)` — for project conventions
   - `with_section_effectiveness(scores)` — for pruning low-value prompt sections

2. In the assembly flow at ~lines 265-390, use these data sources when present:
   - Query knowledge store for relevant entries → inject as context
   - Query recent episodes for few-shot examples → inject as examples
   - Look up role-specific playbook → inject as role instructions
   - Include section effectiveness to prioritize/prune sections

3. When assembling, record which sections and knowledge ids were included, so they can be
   set on the `ModelCallRequest` (via `prompt_section_ids` and `knowledge_ids` from A03).

### In the service factory (from C01):

1. Construct `PromptAssemblyService` with the available data sources from `.roko/`:
   - Knowledge store: `.roko/knowledge/` or neuro store
   - Episodes: `.roko/episodes.jsonl`
   - Playbooks: from config or `.roko/playbooks/`

## Acceptance Criteria

- `PromptAssemblyService` constructed by the factory includes at least knowledge store access.
- Assembled prompts include knowledge-derived sections when knowledge is available.
- The service returns prompt section ids after assembly.
- `cargo check -p roko-compose` passes.

## Do NOT

- Create new knowledge stores or episode formats.
- Change the assembly layer ordering.
