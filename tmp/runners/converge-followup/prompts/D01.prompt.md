# D01: WorkflowRunReport

## Task

Make `WorkflowEngine` return a first-class `WorkflowRunReport` that drives CLI output,
JSON output, sharing, StateHub updates, and feedback.

## Problem

`run_with_workflow_engine` returns `anyhow::Result<()>` at `run.rs:565-571`. There's no
structured report object. CLI outputs a hardcoded model string at `run.rs:612`. Plan v2
returns `self.current_report()` at `orchestrate.rs:7542`, ignoring actual results.

## Changes Required

### In `crates/roko-runtime/src/workflow_engine.rs`:

1. Define a `WorkflowRunReport` struct:
   ```rust
   #[derive(Debug, Clone, Serialize, Deserialize)]
   pub struct WorkflowRunReport {
       pub run_id: String,
       pub success: bool,
       pub model: String,
       pub provider: Option<String>,
       pub prompt_summary: String,
       pub output: String,
       pub agent_turns: u32,
       pub token_usage: u64,
       pub cost: Option<f64>,
       pub duration_secs: f64,
       pub gates: Vec<GateOutcome>,
       pub events: Vec<RuntimeEventEnvelope>,
       pub checkpoint_path: Option<String>,
   }
   ```

2. Have `WorkflowEngine::run()` return `Result<WorkflowRunReport>` instead of `Result<()>`.
   Populate the report from the events emitted during the run.

### In `crates/roko-cli/src/run.rs`:

1. Update `run_with_workflow_engine` to return the report.
2. Use the report for CLI output instead of hardcoded strings.
3. Use the report for `write_shared_run` (if `--share` is requested).
4. The report should have enough data to construct a `RunReport` (the existing type)
   for backward compatibility.

## Acceptance Criteria

- `WorkflowEngine::run()` returns a structured report with real values.
- CLI output uses the report's model, output, duration, and gate results.
- No hardcoded model strings in CLI output.
- `cargo check -p roko-runtime -p roko-cli` passes.

## Do NOT

- Change the engine's internal execution flow.
- Remove the existing `RunReport` type (other code may use it).
