# D02: `roko run --share` Parity on V2

## Task

Make `--share` work on the default v2 path using `WorkflowRunReport`.

## Problem

`--share` is effectively a no-op on the default v2 path. `main.rs:1987-1993` passes
`serve || share`, starting the server branch, but `util.rs:237-272` returns from v2 before
the legacy share code at `util.rs:280-312`.

`write_shared_run` at `run.rs:92-126` expects a `RunReport`, but the v2 path previously
returned `()`.

## Changes Required

### In `crates/roko-cli/src/run.rs`:

1. After the v2 engine returns `WorkflowRunReport` (from D01), convert it to the format
   needed by `write_shared_run`.

2. If `--share` is requested, call `write_shared_run` with the report data. The shared
   run should include: real agent name/role, real prompt, real model, real output, real
   gate results, and accurate success status.

### In `crates/roko-cli/src/commands/util.rs`:

1. At ~lines 237-272, instead of returning early from v2, check if `--share` was requested
   and handle it before returning.

2. Pass the `WorkflowRunReport` data to the share flow.

### In `crates/roko-serve/src/routes/shared_runs.rs`:

1. Verify that `create_shared_run` at ~lines 70-125 can accept real transcript data.
   If it currently creates placeholders (empty agent, empty role, empty prompt), update
   it to use the data from the report.

## Acceptance Criteria

- `roko run --share "test prompt"` on the v2 path creates a shared run with real data.
- The shared transcript has: non-empty agent, real role, real prompt, real output, real model.
- `cargo check -p roko-cli -p roko-serve` passes.

## Do NOT

- Change the share URL format or server routes.
- Add new sharing features.
