# D03: Plan Execution on V2

## Task

Move plan execution out of the env-gated adapter inside legacy `orchestrate.rs` and into
the same v2 workflow entry point used by `roko run`.

## Problem

Plan v2 remains an env-gated adapter at `orchestrate.rs:7502-7509`. The adapter returns
`self.current_report()` at `orchestrate.rs:7542`, ignoring the actual `PlanWorkflowReport`.
This keeps plan execution dependent on the legacy 21K-line orchestrator.

## Changes Required

### In `crates/roko-cli/src/orchestrate.rs`:

1. Find the v2 plan adapter at ~lines 7502-7542. This is the code that calls the workflow
   engine for plan tasks.

2. Extract the plan-to-workflow translation logic into a standalone function that does NOT
   live inside the legacy orchestrator. For example, a `plan_v2::execute_plan_task` function
   in `run.rs` or a new small module.

3. The extracted function should:
   - Take a plan task definition
   - Construct a workflow config from it
   - Run through `WorkflowEngine` using the same service factory (C01)
   - Return `WorkflowRunReport` (D01)
   - Use the real report instead of `self.current_report()`

4. Update the plan execution entry point to call this new function instead of the env-gated
   adapter inside orchestrate.rs.

## Acceptance Criteria

- Plan task execution uses `WorkflowEngine` without going through the legacy orchestrator.
- Plan results come from `WorkflowRunReport`, not `current_report()`.
- The env gate (`ROKO_ENGINE_V2` or similar) is no longer needed for plan execution.
- `cargo check -p roko-cli` passes.

## Do NOT

- Rewrite the entire plan execution system.
- Delete orchestrate.rs (that's E01's job).
- Change plan discovery or DAG resolution.
