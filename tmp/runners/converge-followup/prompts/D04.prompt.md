# D04: Server Workflow Event Consumers

## Task

Register workflow engine event consumers with the server's SSE/StateHub infrastructure.

## Problem

Server SSE endpoint subscribes to `state.sse_adapter` at `routes/mod.rs:158-164`, but the
v2 workflow engine is not registered as a producer for that adapter. `adapters.rs:166-171`
still describes future route integration.

## Changes Required

### In `crates/roko-serve/src/routes/mod.rs`:

1. When a workflow is started through the server (background run, ACP session, etc.),
   register the engine's event stream with the SSE adapter.

2. The engine emits `RuntimeEventEnvelope` events. Map relevant events to SSE events:
   - `PhaseTransition` → SSE phase update
   - `AgentSpawned` / `AgentCompleted` → SSE agent lifecycle
   - `GateResult` → SSE gate result
   - `WorkflowCompleted` → SSE run complete

### In `crates/roko-serve/src/adapters.rs`:

1. Replace the "future" TODO comments with actual event subscription.

2. The `SseAdapter` should accept `RuntimeEventEnvelope` and convert to SSE data format.
   If it already has this capability, just wire it up.

3. The adapter should also feed `StateHub` so the TUI/dashboard reflects workflow progress.

## Acceptance Criteria

- A workflow run started via the server produces SSE events viewable by subscribers.
- `StateHub` receives workflow phase transitions.
- `cargo check -p roko-serve` passes.

## Do NOT

- Change the SSE protocol format.
- Add new SSE event types beyond mapping existing RuntimeEvent variants.
- Modify the TUI code.
