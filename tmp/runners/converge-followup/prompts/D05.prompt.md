# D05: ACP Session Convergence

## Task

Ensure ACP (Agent Communication Protocol) sessions use the same engine, service factory,
feedback, and event stream as CLI and server runs.

## Problem

Shared run creation at `shared_runs.rs:70-125` creates placeholder transcripts with empty
agent, role, prompt, no output, and `success: false`. ACP sessions should produce real
workflow results.

## Changes Required

### In `crates/roko-serve/src/routes/shared_runs.rs`:

1. When a shared run / ACP session is started, use the service factory (C01) to construct
   services and `WorkflowEngine` to run the workflow.

2. Replace the placeholder transcript at ~lines 70-125 with data from `WorkflowRunReport`:
   - `agent`: from the report's agent information
   - `role`: from the report's role
   - `prompt`: from the report's prompt summary
   - `output`: from the report's output
   - `model`: from the report's model
   - `success`: from the report's success flag
   - `transcript`: from the report's events

3. Ensure the ACP session emits events to the same SSE/StateHub consumers (D04).

4. Ensure feedback is recorded for ACP sessions the same way as CLI runs (B05).

## Acceptance Criteria

- ACP shared runs have non-empty agent, role, prompt, output, and model fields.
- ACP sessions record feedback events.
- ACP sessions emit events to SSE subscribers.
- `cargo check -p roko-serve` passes.

## Do NOT

- Change the ACP protocol or shared run URL format.
- Add new ACP features.
