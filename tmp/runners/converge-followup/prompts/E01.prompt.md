# E01: Real Legacy-Orchestrate Feature Boundary

## Task

Make `legacy-orchestrate` a real compile-time feature flag that gates module declarations
and exports of the legacy orchestration code.

## Problem

`crates/roko-cli/src/lib.rs:77` and `lib.rs:126` export `orchestrate` as a public module.
The `legacy-orchestrate` feature exists conceptually but `orchestrate.rs` is still always
compiled and exported. The 21K-line file adds to compile time and keeps all old code reachable.

## Changes Required

### In `crates/roko-cli/Cargo.toml`:

1. Add a proper feature flag:
   ```toml
   [features]
   default = ["legacy-orchestrate"]
   legacy-orchestrate = []
   ```

2. If this feature already exists, ensure it's wired correctly.

### In `crates/roko-cli/src/lib.rs`:

1. Gate the orchestrate module behind the feature:
   ```rust
   #[cfg(feature = "legacy-orchestrate")]
   pub mod orchestrate;
   ```

2. Gate any re-exports or type aliases that depend on orchestrate:
   ```rust
   #[cfg(feature = "legacy-orchestrate")]
   pub use orchestrate::Orchestrator;
   ```

3. Also gate any imports or modules that ONLY exist for legacy orchestration support.

4. Ensure that with `default-features = true`, everything compiles as before (backward
   compatible).

## Acceptance Criteria

- `cargo check -p roko-cli` passes (with default features = legacy on).
- `cargo check -p roko-cli --no-default-features` shows compile errors only for code that
   references orchestrate (this proves the gate works). NOTE: it doesn't need to pass clean
   yet — E03 handles that.
- The `legacy-orchestrate` feature is declared in `Cargo.toml`.
- `orchestrate.rs` is behind `#[cfg(feature = "legacy-orchestrate")]`.

## Do NOT

- Delete any code from orchestrate.rs.
- Fix the compile errors that appear when the feature is disabled (that's E02/E03).
- Remove `legacy-orchestrate` from default features.
