# E03: Legacy-Disabled Compile Check

## Task

Make `cargo check -p roko-cli --no-default-features` compile clean, proving the v2 path
works without any legacy code.

## Problem

After E01 and E02 gate the legacy modules, there will be remaining compile errors from
code that references orchestrate types, dispatch_direct functions, or run_once paths
without feature guards.

## Changes Required

### In `crates/roko-cli/Cargo.toml`:

1. Ensure all dependencies that are only needed for legacy are gated:
   ```toml
   some-legacy-dep = { version = "...", optional = true }

   [features]
   legacy-orchestrate = ["dep:some-legacy-dep"]
   ```

### In any file in `crates/roko-cli/src/`:

1. Find all references to gated items (orchestrate types, dispatch_direct, run_once) that
   appear outside feature gates.

2. Either:
   a. Gate the reference: `#[cfg(feature = "legacy-orchestrate")]`
   b. Replace with the v2 equivalent (e.g., replace `dispatch_agent` call with
      `ModelCallService` call)
   c. Remove if genuinely dead code

3. Common places to check:
   - `main.rs` — subcommand dispatch may reference orchestrate
   - `run.rs` — may have un-gated references to legacy functions
   - `lib.rs` — may re-export legacy types
   - `commands/*.rs` — may import legacy utilities

## Acceptance Criteria

- `cargo check -p roko-cli --no-default-features` compiles with zero errors.
- `cargo check -p roko-cli` (with defaults) also still compiles.
- The v2 path (WorkflowEngine + ModelCallService + ServiceFactory) is fully functional
  without the legacy feature.

## Do NOT

- Delete any legacy code — it's still behind the feature gate.
- Weaken the v2 path to avoid compilation (e.g., don't add stubs).
