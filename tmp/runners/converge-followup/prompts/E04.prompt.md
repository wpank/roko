# E04: Legacy-Enabled Compatibility

## Task

Verify that `cargo check -p roko-cli` (with default features, including `legacy-orchestrate`)
still compiles and the full test suite passes.

## Problem

After E01-E03, it's possible that feature-gating created subtle issues where the legacy code
no longer compiles with the new v2 types (e.g., `WorkflowRunReport` might have changed types
that the legacy adapter references).

## Changes Required

### In `crates/roko-cli/Cargo.toml`:

1. Verify `default = ["legacy-orchestrate"]` is set.

### In `crates/roko-cli/src/`:

1. Run `cargo check -p roko-cli` (with all features). Fix any errors.

2. Run `cargo clippy -p roko-cli -- -D warnings`. Fix any warnings.

3. Common issues to look for:
   - Legacy code referencing types whose shape changed in Waves A-D
   - Import paths that moved due to feature gating
   - Unused imports that only existed for legacy paths

4. Fix each issue minimally — either update the legacy code to use the new type shape
   or add a compatibility shim.

## Acceptance Criteria

- `cargo check -p roko-cli` passes (default features).
- `cargo check -p roko-cli --no-default-features` passes (from E03).
- `cargo clippy -p roko-cli -- -D warnings` passes.
- `cargo test -p roko-cli` passes.

## Do NOT

- Remove the legacy feature gate.
- Add new functionality to legacy code.
