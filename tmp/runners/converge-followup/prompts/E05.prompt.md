# E05: Delete Old Prompt Builders and Report Shims

## Task

Delete legacy prompt building code and hand-rolled report shims that are now superseded
by `PromptAssemblyService` and `WorkflowRunReport`.

## Problem

`crates/roko-cli/src/run.rs` still contains:
- Local prompt construction logic that bypasses `PromptAssemblyService`
- Hand-rolled report formatting that ignores `WorkflowRunReport`
- Adapter code that translates between old and new report types

These create confusion about which path is authoritative.

## Changes Required

### In `crates/roko-cli/src/run.rs`:

1. Find any prompt construction that:
   - Builds system prompts inline (string formatting, concatenation)
   - Doesn't use `PromptAssemblyService`
   - Is NOT behind `#[cfg(feature = "legacy-orchestrate")]`

2. Delete these and replace with calls to `PromptAssemblyService`.

3. Find any report formatting that:
   - Constructs `RunReport` or similar by hand instead of using `WorkflowRunReport`
   - Hardcodes model strings (like `"claude-sonnet-4-20250514"`)
   - Is NOT behind the legacy feature gate

4. Delete these and use `WorkflowRunReport` fields directly.

5. Remove any adapter/conversion code between old and new types that exists only because
   both paths were active simultaneously.

## Acceptance Criteria

- No inline prompt construction outside of `PromptAssemblyService` in un-gated code.
- No hardcoded model strings in CLI output.
- `cargo check -p roko-cli` passes.

## Do NOT

- Touch code behind `#[cfg(feature = "legacy-orchestrate")]`.
- Delete `PromptAssemblyService` or `WorkflowRunReport`.
- Remove the `RunReport` type if other crates depend on it.
