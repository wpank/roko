# F01: Default V2 Run Integration Test

## Task

Add an integration test that proves the default `roko run` v2 path works end-to-end with
a mock provider, real prompt assembly, real feedback, real event logging, and real projection.

## Changes Required

### In `crates/roko-runtime/src/workflow_engine.rs` (test module):

1. Add an integration test `test_v2_run_end_to_end`:

   a. Set up a mock `ModelCaller` that returns a fixed response.

   b. Construct services through the service factory (or manually with all components):
      - `PromptAssemblyService` with default config
      - `FeedbackService` writing to a temp directory
      - `GateService` with basic gate config
      - `EffectDriver` with the mock caller

   c. Create a `WorkflowEngine` with these services.

   d. Run the engine with a simple prompt.

   e. Assert the `WorkflowRunReport`:
      - `success == true`
      - `model` is non-empty (matches mock)
      - `output` is non-empty (from mock response)
      - `events` is non-empty (has at least WorkflowStarted and WorkflowCompleted)
      - `token_usage > 0` (from mock)

   f. Assert that event JSONL was written with typed JSON (not debug strings).

   g. Assert that feedback was recorded (check the feedback service or temp directory).

   h. Assert that `RuntimeProjection` can read back the events and reconstruct state.

## Acceptance Criteria

- Test runs with `cargo test -p roko-runtime -- test_v2_run_end_to_end`.
- Test uses a mock provider (no real LLM calls).
- Test verifies: report, events, feedback, and projection all work together.
- Test does NOT use debug-string event parsing.

## Do NOT

- Make real LLM API calls.
- Test CLI parsing or command handling (this is engine-level).
- Add tests for features not yet implemented.
