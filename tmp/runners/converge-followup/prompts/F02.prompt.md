# F02: Share Transcript Integration Test

## Task

Add an integration test proving `roko run --share` on the v2 path produces a complete
transcript with real data.

## Changes Required

### In `crates/roko-cli/src/run.rs` (test module):

1. Add a test `test_v2_share_produces_real_transcript`:

   a. Set up the v2 run path with a mock provider (similar to F01).

   b. Simulate the `--share` flag being set.

   c. Run the workflow and capture the shared run output.

   d. Assert the shared transcript contains:
      - Non-empty `agent` field (not empty string)
      - Real `role` (not empty)
      - Real `prompt` (the prompt that was assembled)
      - Real `model` (from the provider/report)
      - Real `output` (from the mock response)
      - Accurate `success` status
      - At least one gate result

2. If `write_shared_run` is hard to test directly, test the data that would be passed to it:
   - Construct the `WorkflowRunReport` from a mock run
   - Verify the report has all fields needed for a complete shared transcript
   - Verify the conversion from `WorkflowRunReport` to shared run format preserves data

## Acceptance Criteria

- Test proves shared transcript data is complete (no empty placeholders).
- Test uses mock provider (no real API calls).
- `cargo test -p roko-cli -- test_v2_share` passes.

## Do NOT

- Start an actual HTTP server for the test.
- Test the share URL generation or server routes.
