# F03: Resume Integration Test

## Task

Add an integration test that proves checkpoint → resume works correctly with proper event
sequence and final report.

## Changes Required

### In `crates/roko-runtime/src/workflow_engine.rs` (test module):

1. Add a test `test_checkpoint_resume_round_trip`:

   a. Set up a `WorkflowEngine` with a mock provider that completes in two phases:
      - First call succeeds → triggers checkpoint
      - Engine stops (simulating interruption)

   b. Create a real checkpoint file from the first run.

   c. Resume the engine from the checkpoint.

   d. Assert the resumed run:
      - Emits a `PhaseTransition` event for the recovered phase
      - The checkpoint path in events is non-empty (real file path)
      - Continues from the correct phase (not from the beginning)
      - Produces a valid `WorkflowRunReport`

   e. Assert the event sequence across both runs:
      - Run 1: `WorkflowStarted` → `PhaseTransition(impl)` → `AgentCompleted` → `StateCheckpointed(path)`
      - Run 2: `PhaseTransition(recovered)` → ... → `WorkflowCompleted`

   f. Assert that `RuntimeProjection` can reconstruct the full state from both event logs.

## Acceptance Criteria

- Test proves resume emits recovered phase transition with non-empty checkpoint path.
- Test proves the engine continues from the checkpoint (doesn't re-run completed phases).
- `cargo test -p roko-runtime -- test_checkpoint_resume` passes.

## Do NOT

- Test partial checkpoint corruption recovery (that's a separate concern).
- Make real LLM calls.
