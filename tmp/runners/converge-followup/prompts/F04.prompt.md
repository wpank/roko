# F04: Gateway Durability Test

## Task

Add an integration test proving the gateway route writes durable events, records feedback,
updates provider health, and produces real stats.

## Changes Required

### In `crates/roko-serve/src/routes/gateway.rs` (test module):

1. Add a test `test_gateway_writes_durable_events`:

   a. Set up a server state with:
      - Mock model provider
      - `GatewayEventWriter` writing to a temp directory
      - `FeedbackService` writing to a temp directory

   b. Send a model call request through the gateway route.

   c. Assert:
      - A durable gateway event was written to the temp directory
      - The event contains: model, provider, tokens, cost, duration, cache status
      - A feedback event was recorded
      - Provider health was updated

   d. Query the gateway stats endpoint.

   e. Assert stats are derived from the durable event (not placeholders):
      - Total calls > 0
      - Average latency > 0
      - Model distribution includes the model used

## Acceptance Criteria

- Test proves gateway calls produce durable events.
- Test proves stats come from real data.
- `cargo test -p roko-serve -- test_gateway_writes_durable` passes.

## Do NOT

- Start a real HTTP server (use test harness / handler call directly).
- Make real LLM API calls.
