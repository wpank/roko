# F05: Knowledge Loop Test

## Task

Add a test proving the knowledge loop works: knowledge entry included in prompt → usage
recorded → outcome scored → future request observes updated score.

## Changes Required

### In `crates/roko-learn/src/feedback_service.rs` (test module):

1. Add a test `test_knowledge_loop_scoring`:

   a. Set up:
      - A knowledge store with one test entry (id: "K001", topic: "testing")
      - A `PromptAssemblyService` with the knowledge store attached
      - A `FeedbackService` writing to a temp directory

   b. Step 1 — Prompt assembly:
      - Assemble a prompt for a task related to "testing"
      - Assert knowledge entry "K001" was included
      - Capture the returned `knowledge_ids`

   c. Step 2 — Record usage:
      - Record knowledge usage with the ids from step 1
      - Record a successful outcome for this model call

   d. Step 3 — Score update:
      - Flush feedback
      - Query the knowledge score for "K001"
      - Assert the score increased from its initial value

   e. Step 4 — Future influence:
      - Query section effectiveness from the feedback service
      - Assert the knowledge entry's effectiveness score reflects the positive outcome

## Acceptance Criteria

- Test proves knowledge inclusion → feedback recording → score update works end-to-end.
- Score changes are durable (survive flush/reload).
- `cargo test -p roko-learn -- test_knowledge_loop` passes.

## Do NOT

- Implement complex ML scoring (simple increment is fine).
- Change the knowledge store format.
- Make real LLM calls.
