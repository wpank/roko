# F06: Architecture Negative CI Checks

## Task

Add CI-level negative checks that fail on the specific anti-patterns that convergence
introduced, preventing them from returning.

## Changes Required

### In `scripts/layer_check.rs`:

1. Add these negative pattern checks to the existing layer check script:

   a. **No duplicate foundation traits**: Scan `crates/roko-runtime/src/` and
      `crates/roko-cli/src/` for `pub trait AffectPolicy`, `pub trait DispatchModulation`,
      `pub trait AffectContext`. Fail if found outside `crates/roko-core/`.

   b. **No debug-string event logging**: Scan `crates/roko-runtime/src/jsonl_logger.rs`
      for `format!("{:?}"` or `"{event:?}"` or `Debug` formatting in write calls. Fail if found.

   c. **No direct model subprocess dispatch**: Scan un-gated code (files NOT behind
      `#[cfg(feature = "legacy-orchestrate")]`) for `Command::new("claude")` or
      `Command::new("codex")`. Fail if found.

   d. **No no-op gates**: Scan `crates/roko-gate/src/gate_service.rs` for
      `passed: true` with `stub`, `noop`, or `always` nearby. Fail if found.

   e. **No empty event fields**: Scan `crates/roko-runtime/src/` for
      `agent_id: String::new()` or `model: String::new()` in event construction. Fail if found.

   f. **No path-based shared modules**: Scan for `#[path =` in `crates/roko-cli/src/lib.rs`
      and `crates/roko-serve/src/lib.rs` targeting the same module. Warn if found.

2. Each check should print a clear error message explaining what was found and why it's wrong.

3. Exit with non-zero status if any check fails.

## Acceptance Criteria

- `cargo run -p roko-cli --bin layer_check` (or however it's invoked) passes on clean code.
- Introducing any of the anti-patterns causes the check to fail with a clear message.
- The checks are fast (grep/regex, not compilation).

## Do NOT

- Rewrite the entire layer check script.
- Add checks for hypothetical future anti-patterns — only check for the ones documented.
