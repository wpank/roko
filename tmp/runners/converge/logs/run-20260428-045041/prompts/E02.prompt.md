**BAD** (actual code found in codebase):

After E01, `WorkflowConfig` has `#[derive(Serialize, Deserialize)]`. `Phase` and
`PipelineStateV2` do not yet have serde derives.

The `Phase::Halted { reason }` variant carries a `reason: String` field. The serde
representation should be a tagged enum: `{"Halted": {"reason": "..."}}`. The other
unit variants serialize as plain strings.

`WorkflowOutcome` (lines 16–30) must also be serializable so checkpoints can record
completed runs, though it is not part of `PipelineStateV2` itself:
```rust
#[derive(Debug, Clone, PartialEq, Eq)]
pub enum WorkflowOutcome {
    Success { commit_hash: Option<String> },
    Halted { reason: String },
    Cancelled,
}
```

### Task

#### 1. Add serde derives to `Phase` and `WorkflowOutcome`

```rust
#[derive(Debug, Clone, PartialEq, Eq, serde::Serialize, serde::Deserialize)]
pub enum Phase { ... }

#[derive(Debug, Clone, PartialEq, Eq, serde::Serialize, serde::Deserialize)]
pub enum WorkflowOutcome { ... }
```

No `#[serde(...)]` attributes needed — the default tagged representation is correct.

#### 2. Add serde derives to `PipelineStateV2`

```rust
#[derive(Debug, Clone, serde::Serialize, serde::Deserialize)]
pub struct PipelineStateV2 { ... }
```

All fields are either primitives, `String`, `Vec<String>`, or `Option<String>`, except
`config: WorkflowConfig` which gets its serde support from E01. No custom serialization
is needed.

#### 3. Add `PipelineStateV2::checkpoint`

```rust
/// Serialize the current pipeline state to a JSON string.
///
/// The resulting JSON can be passed to `from_checkpoint` to restore the exact
/// state — including phase, iteration count, and accumulated findings — so a
/// workflow can resume after a process restart.
///
/// Returns an error only if serde_json serialization fails (in practice, never
/// for this struct).
pub fn checkpoint(&self) -> Result<String, Box<dyn std::error::Error + Send + Sync>> {
    Ok(serde_json::to_string(self)?)
}
```

#### 4. Add `PipelineStateV2::from_checkpoint`

```rust
/// Restore a `PipelineStateV2` from a JSON checkpoint string produced by
/// [`checkpoint`].
///
/// After restoring, call [`step`] with the next input as if the workflow is
/// continuing from the saved phase. Terminal phases (`Complete`, `Halted`,
/// `Cancelled`) are valid checkpoint states — callers should check
/// `phase.is_terminal()` before resuming.
///
/// Returns an error if the JSON is malformed or missing required fields.
pub fn from_checkpoint(json: &str) -> Result<Self, Box<dyn std::error::Error + Send + Sync>> {
    Ok(serde_json::from_str(json)?)
}
```
