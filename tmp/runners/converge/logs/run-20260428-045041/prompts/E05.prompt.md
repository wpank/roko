### Task

Add a `save_checkpoint` method to `EffectDriver` that:

1. Serializes the given `PipelineStateV2` to JSON using `state.checkpoint()`.
2. Writes the JSON to disk atomically: write to `path.with_extension("tmp")` first,
   then `tokio::fs::rename` to `path`. This avoids corrupted checkpoints if the
   process is killed mid-write.
3. Emits a `RuntimeEvent::StateCheckpointed` event on success.
4. Returns `Ok(())` on success or a boxed error on failure.

```rust
/// Serialize `state` to JSON and write it atomically to `path`.
///
/// The write is atomic: the JSON is first written to `<path>.tmp`, then
/// renamed to `path`. If the parent directory does not exist, it is created.
///
/// On success, emits `RuntimeEvent::StateCheckpointed`.
pub async fn save_checkpoint(
    &self,
    state: &crate::pipeline_state::PipelineStateV2,
    path: &std::path::Path,
) -> Result<()> {
    // 1. Serialize
    let json = state.checkpoint()?;

    // 2. Ensure parent directory exists
    if let Some(parent) = path.parent() {
        tokio::fs::create_dir_all(parent).await?;
    }

    // 3. Write to .tmp file
    let tmp_path = path.with_extension("tmp");
    tokio::fs::write(&tmp_path, &json).await?;

    // 4. Atomic rename
    tokio::fs::rename(&tmp_path, path).await?;

    // 5. Emit event
    self.emit(RuntimeEvent::StateCheckpointed {
        run_id: self.run_id.clone(),
        path: path.display().to_string(),
    });

    Ok(())
}
```

The `Result<T>` alias in effect_driver.rs is:
```rust
pub type Result<T> = std::result::Result<T, Box<dyn std::error::Error + Send + Sync>>;
```

Use this alias — do not re-declare a new Result type.

Also add a `FeedbackRecorded` event emission inside `save_checkpoint` after the rename
succeeds, consistent with the pattern established in E04:

```rust
self.emit(RuntimeEvent::FeedbackRecorded {
    run_id: self.run_id.clone(),
    kind: "checkpoint".to_string(),
    summary: format!("state saved to {}", path.display()),
});
```

This can be emitted before or after the `StateCheckpointed` event — order doesn't matter.
