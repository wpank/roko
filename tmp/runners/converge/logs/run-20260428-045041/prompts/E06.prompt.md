### AP-1: Never shell out to claude CLI

**BAD** (actual code found in codebase):
```rust

    .arg("--print")
    .arg("--dangerously-skip-permissions")
```


**Why**: Shelling out bypasses cost tracking, feedback recording, provider health monitoring,

batch extends the event emission to cover agent spawns and gate runs so that consumers
receive a complete picture of execution.

#### 1. Emit `RuntimeEvent::WorkflowStarted` at the start of `run()`

This already happens — verify the call exists and uses `RuntimeEvent::WorkflowStarted`
(not the old `WorkflowEvent::WorkflowStarted`).

#### 2. Emit `RuntimeEvent::AgentSpawned` before each agent dispatch

In the `run()` loop, before each `driver.spawn_agent(...)` call, emit:

```rust
self.emit(RuntimeEvent::AgentSpawned {
    run_id: run_id.clone(),
    agent_id: String::new(),  // agent_id not known until driver assigns it
    role: role.to_string(),
    model: String::new(),     // model not known until ModelCaller routes
});
```

The driver's `spawn_agent` already emits its own `AgentSpawned` (with agent_id and model
filled in after the call). This engine-level emit is a pre-dispatch notification so
consumers can show a "spawning..." indicator before the potentially long model call.

Add this pre-dispatch emit before each of the four `spawn_agent` call sites:
- `SpawnStrategist` arm: role = `"strategist"`
- `SpawnImplementer` arm: role = `"implementer"`
- `SpawnAutoFixer` arm: role = `"autofix"`
