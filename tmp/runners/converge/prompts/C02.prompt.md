#### Function signature

/// `routing` is a human-readable routing decision string such as
/// `"cascade-router → claude-sonnet-4-6-20251001"` or `"direct"`.
pub fn print_identity(agent: &str, model: &str, routing: &str) {
    // ...
}
```

#### Implementation

1. Call `intro(agent)` — prints `◆  <agent>` in bold.
2. Print the model line using `bar`:
   - Left column `"model"` in dim, padded to 8 chars with spaces.
   - Right column: `model` in cyan.
   - Full line: `│  model    <cyan:model>`
3. Print the routing line using `bar`:
   - Left column `"routing"` in dim, padded to 8 chars.
   - Right column: routing string. If it contains `" → "`, color the part before `→` in
     magenta and the part after in cyan. Otherwise render plain.
   - Full line: `│  routing  <magenta:router> → <cyan:target>` (or plain if no arrow).

Example implementation sketch:
```rust
pub fn print_identity(agent: &str, model: &str, routing: &str) {
    intro(agent);
    bar(&format!("{}  {}", dim(&format!("{:<8}", "model")), cyan(model)));
    let routing_display = if let Some((before, after)) = routing.split_once(" → ") {
        format!("{} {} {}", magenta(before), "→", cyan(after))
    } else {
        routing.to_string()
    };
    bar(&format!("{}  {}", dim(&format!("{:<8}", "routing")), routing_display));
}
```
