#### Helper: token formatting

Add a private helper that formats token counts with a thin-space (`\u{202F}`) thousands
separator so large numbers are readable at a glance:

```rust
/// Format a token count with thin-space thousands grouping.
/// 1234567 → "1 234 567"
fn fmt_tokens(n: u64) -> String {
    let s = n.to_string();
    let mut out = String::with_capacity(s.len() + s.len() / 3);
    for (i, ch) in s.chars().rev().enumerate() {
        if i > 0 && i % 3 == 0 {
            out.push('\u{202F}'); // thin space
        }
        out.push(ch);
    }
    out.chars().rev().collect()
}
```

#### Implementation

```rust
pub fn print_cost_prediction(estimated_tokens: u64, estimated_cost_usd: f64) {
    step("Cost estimate", "");
    bar(&format!("~{} tokens", cyan(&fmt_tokens(estimated_tokens))));
    bar(&format!("~{} USD", yellow(&format!("${:.4}", estimated_cost_usd))));
}
```

The cost is formatted to 4 decimal places (e.g., `$0.0042`) because sub-cent costs are
common and meaningful for demos.
