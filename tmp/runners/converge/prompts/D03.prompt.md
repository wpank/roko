# Batch D03 — Wire AffectPolicy into WorkflowEngine

## Write Scope
- **MODIFY**: `crates/roko-runtime/src/effect_driver.rs`
- **MODIFY**: `crates/roko-runtime/src/workflow_engine.rs`

## Dependencies
- **D01** must be completed first (provides `AffectPolicy`, `AffectContext`, `DispatchModulation`)

## Task

Add an optional `AffectPolicy` to the WorkflowEngine's service bundle and call it at the
appropriate lifecycle points. When absent (`None`), behavior is unchanged — no affect calls
are made.

**Important**: roko-runtime currently cannot import from roko-core due to a manifest dependency
direction issue (roko-core depends on roko-runtime, creating a potential cycle). The effect
driver and workflow engine define their own local versions of the foundation traits
(`ModelCaller`, `PromptAssembler`, etc.). Follow the same pattern: define a **local**
`AffectPolicy` trait in `effect_driver.rs` that mirrors the one from roko-core, and use it from
`workflow_engine.rs`. The CLI adapter layer (run.rs, D04) will bridge between the two.

### 1. Add local AffectPolicy trait to `crates/roko-runtime/src/effect_driver.rs`

Add these types after the existing `GateRunner` trait definition (around line 217), before
the `RuntimeEvent` enum:

```rust
// -- AffectPolicy --

/// Affect context snapshot for pre-dispatch modulation.
#[derive(Debug, Clone)]
pub struct AffectContext {
    /// Current behavioral state label.
    pub behavioral_state: String,
    /// PAD vector: [Pleasure, Arousal, Dominance].
    pub pad: [f32; 3],
    /// Emotional tag label, if available.
    pub emotional_tag: Option<String>,
}

/// Dispatch modulation parameters from the affect policy.
#[derive(Debug, Clone)]
pub struct DispatchModulation {
    /// Tier bias: -1.0 (cheapest) to +1.0 (most capable).
    pub tier_bias: f32,
    /// Turn limit multiplier (1.0 = no change).
    pub turn_limit_factor: f32,
    /// Exploration rate [0.0, 1.0].
    pub exploration_rate: f32,
}

impl Default for DispatchModulation {
    fn default() -> Self {
        Self {
            tier_bias: 0.0,
            turn_limit_factor: 1.0,
            exploration_rate: 0.0,
        }
    }
}

/// Behavioral affect modulation policy for workflow dispatch.
///
/// When wired, the effect driver calls these methods at lifecycle points.
/// The trait uses `BoxFuture` for persist to match the local async pattern.
pub trait AffectPolicy: Send + Sync {
    /// Called before dispatching a task.
    fn pre_dispatch(&self, task_id: &str, role: &str) -> AffectContext;

    /// Called after task completion.
    fn on_task_outcome(&mut self, task_id: &str, succeeded: bool, tokens_used: u64, cost_usd: f64);

    /// Called after gate verdict.
    fn on_gate_result(&mut self, gate_name: &str, passed: bool, rung: u8, confidence: f64);

    /// Modulate dispatch parameters based on current affect state.
    fn modulate_dispatch(&self, role: &str, params: &mut DispatchModulation);

    /// Current behavioral state label for logging.
    fn behavioral_state_label(&self) -> String;

    /// Persist state to disk.
    fn persist(&self) -> BoxFuture<'_, Result<()>>;
}
```

### 3. Wire affect calls into `EffectDriver::spawn_agent`

In the `spawn_agent` method (line ~310), add affect hooks:

**Before** building the model call request, query the affect policy:
```rust
// Query affect policy for pre-dispatch context.
let mut modulation = DispatchModulation::default();
if let Some(ref affect) = self.services.affect_policy {
    if let Ok(policy) = affect.lock() {
        let _ctx = policy.pre_dispatch(&agent_id, role);
        policy.modulate_dispatch(role, &mut modulation);
    }
}
```

**After** receiving the model call response (in the `Ok` match arm that produces
`PipelineInput::AgentDone`), call `on_task_outcome`:
```rust
if let Some(ref affect) = self.services.affect_policy {
    if let Ok(mut policy) = affect.lock() {
        policy.on_task_outcome(
            &agent_id,
            true, // succeeded
            response.usage.total_tokens,
            response.usage.cost_usd,
        );
    }
}
```

**In the error/failure path** (`PipelineInput::AgentFailed`), call `on_task_outcome` with
`succeeded: false`.
