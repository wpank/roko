```rust
/// Create a git commit.
///
/// Returns `PipelineInput::CommitDone` when a commit is created, or a noop
/// hash when there is nothing to commit.
pub async fn commit(&self, message: &str) -> PipelineInput {
    let add_result = tokio::process::Command::new("git")
        .args(["add", "-A"])
        .current_dir(&self.workdir)
        .output()
        .await;

    if let Err(err) = add_result {
        return PipelineInput::AgentFailed {
            error: format!("git add failed: {err}"),
        };
    }

    let commit_result = tokio::process::Command::new("git")
        .args(["commit", "-m", message])
        .current_dir(&self.workdir)
        .output()
        .await;

    match commit_result {
        Ok(output) if output.status.success() => {
            let hash_output = tokio::process::Command::new("git")
                .args(["rev-parse", "--short", "HEAD"])
                .current_dir(&self.workdir)
                .output()
                .await;

            let hash = hash_output
                .ok()
                .and_then(|output| String::from_utf8(output.stdout).ok())
                .map_or_else(|| "unknown".to_string(), |hash| hash.trim().to_string());

            PipelineInput::CommitDone { hash }
        }
        Ok(output) => {
            let stderr = String::from_utf8_lossy(&output.stderr);
            if stderr.contains("nothing to commit") {
                PipelineInput::CommitDone {
                    hash: "noop".to_string(),
                }
            } else {
                PipelineInput::AgentFailed {
                    error: format!("git commit failed: {stderr}"),
                }
            }
        }
        Err(err) => PipelineInput::AgentFailed {
            error: format!("git commit failed: {err}"),
        },
    }
}
```

### Task

The `commit` method body already handles the three outcomes (commit created, nothing to
commit, error). The gaps are:

1. The `git add -A` error is propagated as `AgentFailed` but should be its own error type
   to distinguish staging failures from agent failures. However, `PipelineInput` has no
   `CommitFailed` variant — return `AgentFailed` as today (do not add new enum variants).

2. After a successful commit, emit a `FeedbackRecorded` event so observers know the commit
   happened without adding a new `RuntimeEvent` variant:

   ```rust
   self.emit(RuntimeEvent::FeedbackRecorded {
       run_id: self.run_id.clone(),
       kind: "commit".to_string(),
       summary: format!("committed {hash}: {}", truncate_message(message, 72)),
   });
   ```

3. The "nothing to commit" branch should also emit a feedback event:

   ```rust
   self.emit(RuntimeEvent::FeedbackRecorded {
       run_id: self.run_id.clone(),
       kind: "commit_noop".to_string(),
       summary: "nothing to commit, working tree clean".to_string(),
   });
   ```

4. Add a private `truncate_message` helper (if `truncate` is not already available in
   the module — check first):

   ```rust
   fn truncate_message(s: &str, max: usize) -> &str {
       if s.len() <= max {
           s
       } else {
           &s[..s.floor_char_boundary(max)]
       }
   }
   ```

   Note: `str::floor_char_boundary` is stable since Rust 1.73.

5. Ensure the `git add -A` failure branch also emits a `FeedbackRecorded` event with
   `kind = "commit_error"` before returning.

The rest of the method — the three-way match structure, the `rev-parse --short HEAD` for
hash extraction, the stderr check for "nothing to commit" — must remain exactly as-is.
