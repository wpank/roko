`crates/roko-runtime/src/workflow_engine.rs` — `WorkflowEngine::run()` loop (lines 115–212):
```rust
pub async fn run(&self, config: WorkflowRunConfig) -> Result<WorkflowResult> {
    let run_id = generate_run_id();
    let mut pipeline = PipelineStateV2::new(config.workflow.clone(), config.prompt.clone());
    let driver = EffectDriver::new(...);

    self.emit(RuntimeEvent::WorkflowStarted { ... });

    let mut output = pipeline.step(PipelineInput::Start);

    loop {
        let old_phase = pipeline.phase.label();

        let input = match &output {
            PipelineOutput::SpawnStrategist { prompt } => { ... }
            PipelineOutput::SpawnImplementer { prompt, context } => { ... }
            PipelineOutput::SpawnAutoFixer { error_output } => { ... }
            PipelineOutput::SpawnReviewer { diff_context } => { ... }
            PipelineOutput::RunGates => driver.run_gates(&config.enabled_gates).await,
            PipelineOutput::Commit => { ... }
            PipelineOutput::Done { outcome } => {
                self.emit(RuntimeEvent::WorkflowCompleted { ... });
                return Ok(WorkflowResult { run_id, outcome: outcome.clone(), iterations: pipeline.iteration });
            }
            PipelineOutput::Halt { reason } => {
                self.emit(RuntimeEvent::WorkflowCompleted { ... });
                return Ok(WorkflowResult { run_id, outcome, iterations: pipeline.iteration });
            }
        };

        output = pipeline.step(input);
        let new_phase = pipeline.phase.label();
        if old_phase != new_phase {
            self.emit(RuntimeEvent::PhaseTransition { ... });
        }
    }
}
```

The `WorkflowOutcome::Cancelled` variant (from `roko_core::runtime_event`):
```rust
pub enum WorkflowOutcome {
    Success { commit_hash: Option<String> },
    Halted { reason: String },
    Cancelled,
}
```

The `PipelineInput::UserCancel` variant causes the state machine to transition to
`Phase::Cancelled` and return `PipelineOutput::Done { outcome: WorkflowOutcome::Cancelled }`.

#### 2. Implementation of `run_with_cancel`

The body is identical to `run()` except:

a. At the **top of each loop iteration**, before the `match &output` block, check the token:

```rust
if token.is_cancelled() {
    let cancel_input = PipelineInput::UserCancel;
    let cancel_output = pipeline.step(cancel_input);
    // The state machine returns Done { Cancelled } — extract the outcome.
    if let PipelineOutput::Done { outcome } = cancel_output {
        self.emit(RuntimeEvent::WorkflowCompleted {
            run_id: run_id.clone(),
            outcome: outcome.clone(),
        });
        return Ok(WorkflowResult {
            run_id,
            outcome,
            iterations: pipeline.iteration,
        });
    }
    // Fallthrough: step returned something unexpected (shouldn't happen).
    // Continue normally to avoid an infinite loop.
}
```

b. Everything else in the loop is unchanged.

#### 3. Make `run()` delegate to `run_with_cancel`

Replace the body of `run()` with:

pub async fn run(&self, config: WorkflowRunConfig) -> Result<WorkflowResult> {
