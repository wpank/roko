## Batch F06: Replace local WorkflowEvent/WorkflowEventConsumer in workflow_engine.rs

### Write Scope
- **MODIFY**: `crates/roko-runtime/src/workflow_engine.rs` (delete WorkflowEvent and WorkflowEventConsumer, use RuntimeEvent + EventConsumer)
- **MODIFY**: `crates/roko-runtime/src/lib.rs` (update re-exports to remove WorkflowEvent/WorkflowEventConsumer)

### Dependencies
- F02 (roko-core dep added, foundation traits imported)
- F03 (RuntimeEvent comes from roko-core)
- F04 (EventConsumer comes from roko-core::foundation)

### DO NOT
- Remove `WorkflowEngine`, `WorkflowRunConfig`, or `WorkflowResult`
- Change the `WorkflowEngine::run()` return type or its semantics

```rust
/// Runtime lifecycle events emitted directly by the workflow engine.
#[derive(Debug, Clone)]
pub enum WorkflowEvent {
    /// Workflow execution has started.
    WorkflowStarted {
        run_id: String,
        template: String,
        prompt: String,
    },
    /// Pipeline phase changed.
    PhaseTransition {
        run_id: String,
        from: String,
        to: String,
    },
    /// Workflow execution has completed.
    WorkflowCompleted {
        run_id: String,
        outcome: WorkflowOutcome,
    },
}

/// Consume workflow lifecycle events for side effects such as logging or UI updates.
pub trait WorkflowEventConsumer: Send + Sync {
    /// Called synchronously when the workflow engine emits an event.
    fn consume(&self, event: &WorkflowEvent);
}
```
