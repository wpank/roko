# Batch G01 — Service contract: unified request/response types

## Write Scope

- `crates/roko-core/src/foundation.rs`

## Task

Extend the existing `ModelCallRequest` and `ModelCallResponse` in `foundation.rs` with
gateway-specific fields so that every model call surface (CLI, ACP, Serve, Research, Dreams)
carries caller identity, budget policy, and cache policy through a single contract.

Add the following types and extend the existing structs:

### New types

```rust
/// Who originated this model call.
#[derive(Debug, Clone, Copy, PartialEq, Eq, Hash)]
pub enum CallerIdentity {
    Cli,
    Acp,
    Serve,
    Research,
    Dreams,
    Test,
}

/// Cache behaviour for this request.
#[derive(Debug, Clone, Copy, PartialEq, Eq, Default)]
pub enum CachePolicy {
    /// Use the default L1 cache behaviour.
    #[default]
    Default,
    /// Skip cache lookup but still store the result.
    Bypass,
    /// Skip cache lookup AND discard any prior cached result for this key.
    ForceRefresh,
}

/// Token and cost budget for a single model call.
#[derive(Debug, Clone, Copy, PartialEq)]
pub struct TokenBudget {
    /// Maximum input tokens the gateway should accept.
    pub max_input: Option<u64>,
    /// Maximum output tokens requested from the provider.
    pub max_output: Option<u64>,
    /// Maximum cost in USD for this single call.
    pub max_cost_usd: Option<f64>,
}

/// Errors specific to the gateway pipeline (not provider errors).
#[derive(Debug, Clone, thiserror::Error)]
pub enum GatewayError {
    #[error("provider error: {0}")]
    ProviderError(String),
    #[error("budget exceeded: {detail}")]
    BudgetExceeded { detail: String },
    #[error("rate limited: retry after {retry_after_ms:?} ms")]
    RateLimited { retry_after_ms: Option<u64> },
    #[error("cache error: {0}")]
    CacheError(String),
    #[error("request cancelled")]
    Cancelled,
    #[error("convergence detected after {consecutive} identical outputs")]
    ConvergenceDetected { consecutive: u32 },
}
```

### Extended fields on `ModelCallRequest`

Add these three optional fields (preserving all existing fields and their positions):

```rust
pub caller: Option<CallerIdentity>,
pub budget: Option<TokenBudget>,
pub cache_policy: CachePolicy,
```

Make `CachePolicy` default to `CachePolicy::Default` via `#[derive(Default)]` so existing
construction sites that use `..Default::default()` or struct literals are unaffected.

### Extended fields on `ModelCallResponse`

Add one optional field:

```rust
/// Gateway request id, set by ModelCallService when the call flows through the gateway.
pub request_id: Option<String>,
```

### Implementation notes

- `CallerIdentity` must derive `Serialize, Deserialize` (add `serde` derives) because it
  will be written to `gateway.jsonl` by G02.
- `TokenBudget` must derive `Serialize, Deserialize, Default`.
- `GatewayError` must implement `From<GatewayError> for RokoError` so the gateway pipeline
  can return errors through the existing `Result<_, RokoError>` path. Add an `impl From`
  block; map to `RokoError::Other` with the display string.
- Do NOT move or rename the `ModelCaller` trait. It stays as-is.
- Update the existing `ModelCallRequest` struct (do not create a second request type).
- Update the two existing unit tests in `model_call_service.rs` that construct
  `ModelCallRequest` literals to include the three new fields with default values. Those
  tests live in `crates/roko-agent/src/model_call_service.rs`, not in foundation.rs, but
  they will fail to compile if you add non-defaultable fields.

## DO NOT

- Create a new `ModelCallRequest` type or rename the existing one.
- Remove or reorder existing fields on `ModelCallRequest` or `ModelCallResponse`.
- Add `serde` derives to `ModelCallRequest` or `ModelCallResponse` themselves (they are
  internal types, not serialized directly).
- Touch anything in `crates/roko-serve/`, `crates/roko-cli/`, or `crates/roko-agent/src/provider/`.
- Add dependencies to `roko-core/Cargo.toml` beyond what is already there (`thiserror` is
  already a dependency; `serde` is already a dependency).

## Done Criteria

1. `cargo check -p roko-core` passes with no errors.
2. `cargo check -p roko-agent` passes (model_call_service.rs compiles with the new fields).
3. `CallerIdentity`, `CachePolicy`, `TokenBudget`, and `GatewayError` are all public exports
   from `roko_core::foundation`.
4. `GatewayError` converts to `RokoError` via `From`.
5. No new `#[allow(unused)]` attributes were added.
