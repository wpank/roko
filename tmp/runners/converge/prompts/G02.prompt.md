# Batch G02 — Durable gateway event writer

## Write Scope

- `crates/roko-agent/src/gateway_events.rs` (new file)
- `crates/roko-agent/src/lib.rs` (add module declaration and re-export)

## Task

Create a `GatewayEventWriter` that logs every model call flowing through the gateway to
`.roko/learn/gateway.jsonl`. This is the durable source of truth for gateway statistics,
replacing the volatile in-memory counters currently in `roko-serve`.

### GatewayEvent struct

```rust
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct GatewayEvent {
    /// Unique request id assigned by the gateway.
    pub request_id: String,
    /// Who initiated this call.
    pub caller: String,
    /// Model slug that served the request.
    pub model: String,
    /// Input tokens consumed.
    pub input_tokens: u64,
    /// Output tokens generated.
    pub output_tokens: u64,
    /// Estimated cost in USD.
    pub cost_usd: f64,
    /// Wall-clock latency in milliseconds.
    pub latency_ms: u64,
    /// Whether a cache hit satisfied this request.
    pub cache_hit: bool,
    /// Error message if the call failed, None on success.
    pub error: Option<String>,
    /// ISO 8601 timestamp.
    pub timestamp: String,
}
```

### GatewayEventWriter

```rust
pub struct GatewayEventWriter {
    path: PathBuf,
}

impl GatewayEventWriter {
    /// Create a writer targeting the given JSONL file.
    pub fn new(path: impl Into<PathBuf>) -> Self;

    /// Create a writer at the conventional location: `{workdir}/.roko/learn/gateway.jsonl`.
    pub fn for_workdir(workdir: &Path) -> Self;

    /// Append a single gateway event as a JSON line.
    pub fn write(&self, event: &GatewayEvent) -> std::io::Result<()>;
}
```

The `write()` method must:
1. Serialize the event to a single JSON line (no pretty-printing).
2. Append to the file (create parent directories if needed, open with `append` + `create`).
3. Write atomically per line (single `write_all` call with trailing newline).

### GatewayProjection

```rust
pub struct GatewayProjection {
    events: Vec<GatewayEvent>,
}

impl GatewayProjection {
    /// Load all events from the given JSONL file.
    pub fn load(path: &Path) -> std::io::Result<Self>;

    /// Load from the conventional workdir location.
    pub fn for_workdir(workdir: &Path) -> std::io::Result<Self>;

    /// Find an event by request_id.
    pub fn by_request_id(&self, request_id: &str) -> Option<&GatewayEvent>;

    /// Aggregate stats grouped by caller.
    pub fn stats_by_caller(&self) -> HashMap<String, AggregateStats>;

    /// Aggregate stats grouped by model.
    pub fn stats_by_model(&self) -> HashMap<String, AggregateStats>;

    /// Total number of events.
    pub fn total_events(&self) -> usize;

    /// Total cost across all events.
    pub fn total_cost_usd(&self) -> f64;
}

#[derive(Debug, Clone, Default, Serialize, Deserialize)]
pub struct AggregateStats {
    pub count: u64,
    pub total_input_tokens: u64,
    pub total_output_tokens: u64,
    pub total_cost_usd: f64,
    pub cache_hits: u64,
    pub errors: u64,
}
```

### Tests

Add at least these tests in `gateway_events.rs`:

1. `write_and_read_roundtrip` — write 3 events, load via `GatewayProjection`, verify count and
   content match.
2. `by_request_id_finds_correct_event` — write 2 events with different request_ids, look up each.
3. `stats_by_model_aggregates_correctly` — write events for 2 different models, verify
   per-model token and cost sums.
4. `empty_file_loads_cleanly` — create an empty file, load it, verify zero events.

Use `tempfile::tempdir()` for all test file paths.

## DO NOT

- Add any async code. The writer is synchronous (same as EpisodeLogger).
- Use `BufWriter` with deferred flush. Each `write()` call must be immediately visible.
- Lock the file. Concurrent appends from different threads are acceptable because each write
  is a single atomic `write_all` of one line.
- Touch `model_call_service.rs`. That file is modified by G03, not here.
- Add new dependencies to `roko-agent/Cargo.toml` (serde, chrono, and tempfile are already
  available).

4. `GatewayProjection::load` gracefully handles missing files (returns empty projection or

5. The module and types are re-exported from `roko_agent`.
