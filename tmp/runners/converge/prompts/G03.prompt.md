# Batch G03 — ProviderCallCell: move provider execution into cell

## Write Scope

- `crates/roko-agent/src/model_call_service.rs`

## Task

Refactor `ModelCallService` to route provider execution through a `ProviderCallCell` that
wraps the existing `create_agent_for_model()` -> `Agent::run()` flow. This is the first
pipeline cell. It does not replace `ModelCallService`; it restructures its internals.

### ProviderCallCell

Create a struct inside `model_call_service.rs`:

```rust
/// Encapsulates a single provider execution attempt with fallback support.
struct ProviderCallCell {
    config: RokoConfig,
    cost_table: CostTable,
}

impl ProviderCallCell {
    fn new(config: RokoConfig, cost_table: CostTable) -> Self;

    /// Execute a model call through the provider layer.
    ///
    /// 1. Calls `create_agent_for_model()` to get a `Box<dyn Agent>`.
    /// 2. Runs `agent.run()` with the prompt.
    /// 3. Classifies the result as success or failure.
    /// 4. On retryable failure, tries the next model in `fallback_models` (if any).
    /// 5. Returns the response or the final error.
    async fn execute(
        &self,
        model: &str,
        system_prompt: Option<&str>,
        user_content: &str,
        options: AgentOptions,
        fallback_models: &[String],
    ) -> Result<CellOutput, CellError>;
}

struct CellOutput {
    content: String,
    model_used: String,
    usage: Usage,
    cost_usd: f64,
    latency_ms: u64,
    fallback_used: bool,
}

/// Classified cell-level error.
#[derive(Debug)]
enum CellError {
    /// Provider returned an error but another provider may work.
    Retryable { message: String },
    /// Permanent failure, do not retry.
    Terminal { message: String },
    /// Agent construction failed (missing credentials, bad config).
    Construction { message: String },
}
```

### Fallback chain logic

The `execute` method should:

1. Try the primary `model` first.
2. If the call fails with a retryable error (agent returns `!result.success` and the error
   message contains "rate limit", "timeout", "503", or "temporarily unavailable"), try each
   model in `fallback_models` in order.
3. If no fallback succeeds, return the last error.
4. Set `fallback_used = true` in `CellOutput` when a non-primary model served the request.

### Refactor ModelCallService::call()

The existing `ModelCaller::call()` implementation must delegate to `ProviderCallCell::execute()`
instead of inlining the `create_agent_for_model` / `agent.run()` flow directly.

The new flow in `ModelCallService::call()`:

```
1. resolve_model()
2. build AgentOptions from request fields
3. cell.execute(model, system, content, options, fallback_models)
4. emit RuntimeEvent (success or failure)
5. record_feedback()
6. construct and return ModelCallResponse
```

The cell's `config` and `cost_table` come from `ModelCallService`'s existing fields.

`fallback_models` should be an empty `Vec` for now (no config wiring yet). The cell must
still handle the empty-fallback case gracefully (just return the primary error).

impl ModelCaller for ModelCallService {

        let model = self.resolve_model(&req);

## DO NOT

- Change the `ModelCaller` trait signature. The public API stays the same.
- Remove the existing helper functions (`request_prompt`, `output_text`, `token_usage`).
  The cell can call them.
- Make `ProviderCallCell` public. It is an internal implementation detail.
- Add fallback model configuration (that comes later). Just accept `&[String]` and handle
  the empty-slice case.
- Touch `crates/roko-agent/src/provider/mod.rs`. The cell wraps `create_agent_for_model`;
  it does not modify it.
- Wire `GatewayEventWriter` here. That wiring happens in a later batch after G02 lands.
