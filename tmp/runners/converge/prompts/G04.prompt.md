# Batch G04 — HTTP gateway adapter

## Write Scope

- `crates/roko-serve/src/routes/gateway.rs`

## Task

Replace the route-local inference implementation in `routes/gateway.rs` with calls to
`ModelCallService`. The HTTP route becomes a thin adapter: it validates the request,
constructs a `ModelCallRequest`, calls `ModelCallService::call()`, and maps the response
back to JSON.

### Changes to `inference_complete`

Replace the current flow:

```
format_messages_as_prompt() -> state.runtime.run_once() -> estimate_tokens() -> compute_cost()
```

With:

```
build ModelCallRequest from CompletionRequest -> model_call_service.call() -> map ModelCallResponse to CompletionResponse
```

Specifically:

1. Obtain a `ModelCallService` from `AppState`. Add a field to `AppState`:
   ```rust
   pub model_call_service: Arc<ModelCallService>,
   ```
   Initialize it in `AppState::new()` using the config's `agent.default_model` and passing
   the `RokoConfig`.

2. Build a `ModelCallRequest`:
   ```rust
   let req = ModelCallRequest {
       model: body.model.clone().unwrap_or_default(),
       system: None, // extracted from messages below
       messages: body.messages.iter().map(|m| CoreChatMessage {
           role: match m.role.as_str() {
               "system" => CoreMessageRole::System,
               "assistant" => CoreMessageRole::Assistant,
               _ => CoreMessageRole::User,
           },
           content: m.content.clone(),
       }).collect(),
       max_tokens: body.max_tokens,
       temperature: body.temperature.map(|t| t as f32),
       role: body.agent_id.clone(),
       caller: Some(CallerIdentity::Serve),
       budget: None,
       cache_policy: CachePolicy::Default,
   };
   ```

3. Call `model_call_service.call(req).await`.

4. Map the `ModelCallResponse` to the existing `CompletionResponse` JSON shape. The
   `usage` field comes from `response.usage`; `cost_usd` from `response.usage.cost_usd`.

5. Keep the existing `CascadeRouter` model selection for the case where `body.model` is
   `None`. Move the router-selected model slug into `req.model` before calling the service.

6. Keep the `ServerEvent::AgentOutput` publication and the `provider_health` recording.
   These are adapter concerns, not gateway concerns, so they stay in the route.

### Changes to `batch_submit`

Replace the inner per-item loop that calls `state.runtime.run_once()` with a call to
`model_call_service.call()` for each item. The concurrent fan-out via
`buffer_unordered(BATCH_CONCURRENCY)` stays.

### Remove

- `format_messages_as_prompt()` function (no longer needed; messages go structured into
  `ModelCallRequest`).
- `estimate_tokens()` function (token counts come from `ModelCallResponse.usage`).
- `compute_cost()` function (cost comes from `ModelCallResponse.usage.cost_usd`).
- The hardcoded `cache_read_tokens: 0` in the response construction.

### Keep

- All existing route registrations (the paths and HTTP methods stay the same).
- `gateway_stats`, `gateway_models`, `batch_status` handlers (unchanged for now).
- The `RoutingHints` struct and `select_model_via_router` function.
- All request/response serde types.
- All existing tests. Update test assertions where the response shape changes (e.g., if
  `cache_read_tokens` is removed from `TokenUsage`, update the struct).

## DO NOT

- Change any route paths or HTTP methods.
- Remove or rename the `gateway_stats`, `gateway_models`, or `batch_status` handlers.
- Move `select_model_via_router` out of this file. It is adapter-level routing that belongs
  with the HTTP layer.
- Wire `GatewayEventWriter` in the route. The writer is wired inside `ModelCallService`
  (done in a later batch).
- Modify `crates/roko-serve/src/state.rs` beyond adding the `model_call_service` field
  to `AppState` and initializing it in `new()`.
- Break the existing test helpers (`test_state`, `NoOpRuntime`). If `ModelCallService`
  needs a mock, use the existing `ModelCallService::new("default".into())` with a default
  config (it will fail at provider construction, which the `NoOpRuntime`-based tests already
  handle).
