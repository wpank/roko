# Batch G05 — CLI runner adapter

## Write Scope

- `crates/roko-cli/src/run.rs`

## Task

Ensure the CLI `run` command's `WorkflowEngine` path uses `ModelCallService` for model calls
instead of calling `create_agent_for_model()` directly. The CLI becomes a `CallerIdentity::Cli`
surface through the unified gateway.

### Background

The `run.rs` file implements `roko run "<prompt>"`. It sets up a `WorkflowEngine` with
`EffectServices` that include a `RuntimeModelCaller`. The `RuntimeModelCaller` trait is
defined in `roko_runtime::effect_driver` and mirrors `roko_core::foundation::ModelCaller`.

There are two code paths:
1. The `WorkflowEngine` path (newer) -- uses `EffectServices` with a `RuntimeModelCaller`.
2. The direct `spawn_agent_scoped` path (older) -- constructs agents directly.

### What to change

1. In the section where `EffectServices` is constructed, verify that the `model_caller` field
   is backed by `ModelCallService`. If it currently wraps a direct `create_agent_for_model()`
   call, replace it with `ModelCallService`:

   ```rust
   let model_call_service = ModelCallService::new(config.default_model().to_string())
       .with_config(roko_config.clone())
       .with_run_id(run_id.clone());
   ```

   Then wrap it in an adapter that implements `RuntimeModelCaller` by delegating to
   `ModelCallService::call()`.

2. Set `CallerIdentity::Cli` on every `ModelCallRequest` constructed in this file.

3. If the direct `spawn_agent_scoped` path still exists and calls `create_agent_for_model()`
   independently, add a TODO comment noting it should be migrated in a future batch (do NOT
   remove the working code path yet).

### Adapter pattern

If `RuntimeModelCaller` and `roko_core::foundation::ModelCaller` are not trait-compatible
(different crates, different signatures), create a thin adapter struct:

```rust
struct CliModelCaller {
    inner: Arc<ModelCallService>,
}

#[async_trait]
impl RuntimeModelCaller for CliModelCaller {
    async fn call(&self, req: RuntimeModelCallRequest) -> RuntimeResult<RuntimeModelCallResponse> {
        let core_req = CoreModelCallRequest {
            model: req.model,
            system: req.system,
            messages: req.messages.into_iter().map(|m| CoreChatMessage {
                role: match m.role {
                    RuntimeMessageRole::System => CoreMessageRole::System,
                    RuntimeMessageRole::User => CoreMessageRole::User,
                    RuntimeMessageRole::Assistant => CoreMessageRole::Assistant,
                },
                content: m.content,
            }).collect(),
            max_tokens: req.max_tokens,
            temperature: req.temperature,
            role: req.role,
            caller: Some(CallerIdentity::Cli),
            budget: None,
            cache_policy: CachePolicy::Default,
        };
        let resp = self.inner.call(core_req).await
            .map_err(|e| anyhow::anyhow!("{e}"))?;
        Ok(RuntimeModelCallResponse {
            content: resp.content,
            model: resp.model,
            usage: RuntimeTokenUsage { ... },
            stop_reason: resp.stop_reason,
        })
    }
}
```

## DO NOT

- Remove the `spawn_agent_scoped` code path. It is still the production path for direct
  agent runs. Mark it with a TODO for future migration.
- Change the `WorkflowEngine` or `EffectServices` structs in `roko-runtime`. Only change
  what is passed into them from `run.rs`.
- Wire `GatewayEventWriter` here. Event writing is internal to `ModelCallService`.
- Break the existing `roko run "<prompt>"` command. It must still work end-to-end.
- Touch `crates/roko-runtime/` source files.
