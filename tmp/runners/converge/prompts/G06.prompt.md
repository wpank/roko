# Batch G06 — Domain caller migration (research, dreams, neuro)

## Write Scope

- `crates/roko-dreams/src/runner.rs`
- `crates/roko-neuro/src/episode_completion.rs`
- `crates/roko-std/src/tool/builtin/web_search.rs`

## Task

Migrate three domain callers from direct `create_agent_for_model()` / direct API key reads
to accept an `Arc<dyn ModelCaller>` and use it for inference. Each caller should set the
appropriate `CallerIdentity` on its requests.

### 1. Dreams runner (`crates/roko-dreams/src/runner.rs`)

**Current**: `DreamAgentConfig::build_agent()` calls `create_agent_for_model()` directly
and constructs a `DreamReviewAgent` wrapping `Box<dyn Agent>`.

**Target**: Add an optional `model_caller: Option<Arc<dyn ModelCaller>>` field to the
runner or `DreamAgentConfig`. When present, the dream review should call
`model_caller.call()` instead of constructing agents directly.

Approach:
1. Add a method `DreamAgentConfig::build_agent_via_gateway()` that takes
   `Arc<dyn ModelCaller>` and returns a `DreamReviewAgent` that delegates to it.
2. `DreamReviewAgent` already wraps `Box<dyn Agent>`. Add an alternative variant that
   wraps `Arc<dyn ModelCaller>` and implements the same `run()` signature by building a
   `ModelCallRequest` with `caller: Some(CallerIdentity::Dreams)`.
3. Keep the existing `build_agent()` as a fallback for callers that don't have a gateway
   reference yet. Add a `// TODO(gateway): remove direct construction once all callers
   provide a ModelCaller` comment.

```rust
use roko_core::foundation::{
    CallerIdentity, CachePolicy, ChatMessage, MessageRole,
    ModelCallRequest, ModelCaller,
};
```

### 2. Neuro episode completion (`crates/roko-neuro/src/episode_completion.rs`)

**Current**: `distill_episode()` reads `ANTHROPIC_API_KEY` from the environment and calls
`Distiller::with_claude(api_key)`.

**Target**: Accept an optional `Arc<dyn ModelCaller>` parameter. When present, create a
`Distiller` that delegates to the model caller instead of making direct API calls.

Approach:
1. Change `spawn_episode_distillation` to accept an optional
   `model_caller: Option<Arc<dyn ModelCaller>>`.
2. When `model_caller` is `Some`, skip the `ANTHROPIC_API_KEY` read and instead create a
   distiller that calls `model_caller.call()` with `caller: Some(CallerIdentity::Research)`.
3. When `model_caller` is `None`, keep the existing `ANTHROPIC_API_KEY` fallback with a
   `tracing::warn!` noting "distillation using direct API key; gateway not available".

```rust
pub fn spawn_episode_distillation(
    workdir: PathBuf,
    episode: Episode,
    model_caller: Option<Arc<dyn ModelCaller>>,
) {
```

If `Distiller` does not have a trait-based construction path, add a `// TODO(gateway):
refactor Distiller to accept ModelCaller` comment and keep the existing code as the
only working path. Do not break the existing functionality.

### 3. Web search tool (`crates/roko-std/src/tool/builtin/web_search.rs`)

**Current**: The `Handler::execute()` method reads `PERPLEXITY_API_KEY` from the environment
and makes a direct HTTP call to `https://api.perplexity.ai/chat/completions`.

**Target**: Accept a `ModelCaller` via `ToolContext` or a new field on `Handler`. When
present, call `model_caller.call()` with `caller: Some(CallerIdentity::Cli)` and
`model: "sonar"`.

Approach:
1. Add an `Arc<dyn ModelCaller>` field to `Handler` (or an `Option` for backward compat):
   ```rust
   pub struct Handler {
       model_caller: Option<Arc<dyn ModelCaller>>,
   }
   ```
2. When `model_caller` is `Some`, build a `ModelCallRequest` with the search query as the
   user message and call through the gateway.
3. When `model_caller` is `None`, fall back to the existing direct Perplexity HTTP call.
   Add a `tracing::warn!` on first use noting the direct API key path.
4. Keep all existing test paths working. Tests use the `None` path (or a mock).

If `ToolContext` does not carry a model caller and adding one would require changes across
too many files, use the `Option` field approach on `Handler` and add a `// TODO(gateway):
wire ModelCaller from runtime ToolContext` comment.

## DO NOT

- Remove the existing working code paths. Every change must be backward compatible with
  `model_caller: None` falling back to the current behavior.
- Add `roko-agent` as a dependency of `roko-std` or `roko-dreams` or `roko-neuro`. These
  crates depend on `roko-core` which owns the `ModelCaller` trait. That is sufficient.
- Change the `Distiller` API in `roko-neuro` beyond what is minimally needed. If the
  refactor is too large, add a TODO and keep the existing path.
- Break any existing tests in any of the three crates.
- Wire the `GatewayEventWriter` here. Event writing is inside `ModelCallService`, not in
  domain callers.

## Done Criteria

1. `cargo check -p roko-dreams` passes.
2. `cargo check -p roko-neuro` passes.
3. `cargo check -p roko-std` passes.
4. Each of the three callers has an `Option<Arc<dyn ModelCaller>>` path that sets the
   correct `CallerIdentity` on outgoing requests.
5. The existing fallback paths (direct API key / direct agent construction) still work
   when `model_caller` is `None`.
6. No new direct `std::env::var("*_API_KEY")` reads are introduced (existing ones stay
   in the fallback path).
