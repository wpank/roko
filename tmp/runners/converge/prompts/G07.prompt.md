# Batch G07 — Cache cell + budget cell

## Write Scope

- `crates/roko-agent/src/model_call_service.rs`

## Task

Add two pipeline cells to `ModelCallService` that run before `ProviderCallCell::execute()`:

1. **CacheCell** — L1 in-memory exact-match cache.
2. **BudgetCell** — token/cost budget enforcement.

Both cells are internal to `model_call_service.rs`. They transform or short-circuit the
request before it reaches the provider.

### CacheCell

```rust
use std::collections::HashMap;
use std::hash::{Hash, Hasher};
use std::sync::Mutex;

/// L1 in-memory response cache. Keyed by (model, messages, temperature).
struct CacheCell {
    /// Maximum number of cached entries.
    max_entries: usize,
    /// The cache store. Uses a simple HashMap with LRU eviction.
    entries: Mutex<HashMap<u64, CachedResponse>>,
    /// Insertion-order keys for LRU eviction.
    order: Mutex<Vec<u64>>,
}

struct CachedResponse {
    content: String,
    model: String,
    usage: TokenUsage,
    stop_reason: Option<String>,
}

impl CacheCell {
    fn new(max_entries: usize) -> Self;

    /// Compute a cache key from request fields.
    /// Hash of: model + sorted message contents + temperature (if set).
    fn cache_key(model: &str, messages: &[ChatMessage], temperature: Option<f32>) -> u64;

    /// Look up a cached response. Returns None on miss.
    fn lookup(&self, key: u64) -> Option<CachedResponse>;

    /// Store a successful response. Evicts the oldest entry if at capacity.
    fn store(&self, key: u64, response: CachedResponse);
}
```

**Cache policy integration**:
- `CachePolicy::Default` — check cache, store on miss.
- `CachePolicy::Bypass` — skip lookup, still store result.
- `CachePolicy::ForceRefresh` — skip lookup, evict existing entry for this key, store new result.

### BudgetCell

```rust
use std::sync::atomic::{AtomicU64, Ordering};

/// Tracks cumulative cost across calls within a workflow run and enforces per-call budgets.
struct BudgetCell {
    /// Cumulative cost in micro-USD (1e-6 USD) for atomic tracking.
    cumulative_cost_micro_usd: AtomicU64,
    /// Maximum cumulative cost in micro-USD, if set.
    max_cumulative_cost_micro_usd: Option<u64>,
}

impl BudgetCell {
    fn new(max_cumulative_cost_usd: Option<f64>) -> Self;

    /// Check whether a request's budget allows it to proceed.
    /// Returns Err(GatewayError::BudgetExceeded) if the cumulative cost has been exceeded
    /// or if the request's own TokenBudget.max_cost_usd would be exceeded.
    fn check(&self, budget: &Option<TokenBudget>) -> Result<(), GatewayError>;

    /// Record the cost of a completed call. Adds to cumulative total.
    fn record_cost(&self, cost_usd: f64);

    /// Current cumulative cost in USD.
    fn cumulative_cost_usd(&self) -> f64;
}
```

### Updated call() flow

```
1. resolve_model()
2. compute cache_key
3. if cache_policy != Bypass && cache_policy != ForceRefresh:
     check cache -> if hit, return cached response (set request_id, skip provider)
4. budget.check(req.budget) -> if exceeded, return GatewayError::BudgetExceeded
5. cell.execute() (ProviderCallCell from G03)
6. budget.record_cost(cost_usd)
7. cache.store(key, response) (unless cache_policy is ForceRefresh with eviction first)
8. emit events, record feedback
9. return response
```

## DO NOT

- Use an external LRU crate. Implement the simple HashMap + Vec approach. The cache is
  small (128 entries) and does not need to be performant under high concurrency.
- Make `CacheCell` or `BudgetCell` public. They are internal to `model_call_service.rs`.
- Add L2 semantic cache. That is explicitly deferred. Add a `// TODO(gateway): L2 semantic
  cache` comment near the cache implementation.
- Block on cache writes. The `store()` call is synchronous and fast.
- Change the `ModelCaller` trait signature.
- Touch files outside `model_call_service.rs`.
