# Batch G08 — Thinking cap + convergence detection

## Write Scope

- `crates/roko-agent/src/model_call_service.rs`

## Task

Add two more pipeline cells to `ModelCallService`:

1. **ThinkingCapCell** — limit extended thinking tokens to prevent runaway costs.
2. **ConvergenceDetectionCell** — detect agent output loops and short-circuit.

Both cells are internal to `model_call_service.rs`.

### ThinkingCapCell

```rust
/// Caps thinking/reasoning tokens for models that support extended thinking.
struct ThinkingCapCell {
    /// Default maximum thinking tokens when thinking is enabled.
    default_thinking_budget: u32,
}

impl ThinkingCapCell {
    fn new(default_thinking_budget: u32) -> Self;

    /// Apply thinking cap to a request, returning the effective thinking budget.
    ///
    /// Rules:
    /// - If the model does NOT support thinking (not claude-opus), return None.
    /// - If the request already has an explicit max_tokens < default, respect it.
    /// - Otherwise, cap thinking at default_thinking_budget.
    /// - Never force thinking ON. Only cap when it is already enabled.
    fn apply(&self, model: &str, max_tokens: Option<u32>) -> ThinkingCapResult;
}

struct ThinkingCapResult {
    /// Effective thinking budget (None = thinking not applicable).
    thinking_budget: Option<u32>,
    /// Whether the cap was applied (original was higher or unset).
    was_capped: bool,
    /// Original value before capping.
    original: Option<u32>,
}
```

**Model detection**: A model supports thinking if its slug contains "opus" or if it
matches known thinking-capable models. Use a simple string check:

```rust
fn supports_thinking(model: &str) -> bool {
    let lower = model.to_ascii_lowercase();
    lower.contains("opus")
        || lower.contains("o1")
        || lower.contains("o3")
        || lower.contains("o4")
        || lower.contains("deepseek-r1")
}
```

### ConvergenceDetectionCell

```rust
use std::collections::HashMap;
use std::sync::Mutex;

/// Detects when an agent produces near-identical outputs repeatedly.
struct ConvergenceDetectionCell {
    /// Maximum number of recent outputs to track per key.
    window_size: usize,
    /// Similarity threshold (0.0-1.0). Above this = "identical enough".
    similarity_threshold: f64,
    /// Number of consecutive similar outputs before triggering.
    consecutive_trigger: usize,
    /// Recent outputs keyed by (run_id, role).
    history: Mutex<HashMap<String, Vec<String>>>,
}

impl ConvergenceDetectionCell {
    fn new(window_size: usize, similarity_threshold: f64, consecutive_trigger: usize) -> Self;

    /// Check a new output against the history for this key.
    ///
    /// Returns Err(GatewayError::ConvergenceDetected) if the last
    /// `consecutive_trigger` outputs are all above `similarity_threshold`.
    fn check(
        &self,
        key: &str,
        new_output: &str,
    ) -> Result<(), GatewayError>;

    /// Record an output in the history for future checks.
    fn record(&self, key: &str, output: String);
}
```

**Similarity function**: Use normalized edit distance ratio (Levenshtein distance divided
by max length of the two strings). This is simple and sufficient:

```rust
fn similarity(a: &str, b: &str) -> f64 {
    if a.is_empty() && b.is_empty() {
        return 1.0;
    }
    let max_len = a.len().max(b.len());
    if max_len == 0 {
        return 1.0;
    }
    let distance = edit_distance(a, b);
    1.0 - (distance as f64 / max_len as f64)
}
```

Implement `edit_distance` as a standard Levenshtein distance with a two-row DP approach.
For efficiency, truncate both strings to the first 500 characters before comparing (agent
outputs can be very long; we only need the similarity signal, not exact distance).

### Updated call() flow

Insert into the pipeline (between budget check and provider call):

```
1. resolve_model()
2. cache check (from G07)
3. budget check (from G07)
4. thinking_cap.apply(model, max_tokens) -> adjust max_tokens on the AgentOptions
5. cell.execute() (ProviderCallCell)
6. convergence.check(key, output) -> if converged, return GatewayError::ConvergenceDetected
7. convergence.record(key, output)
8. budget.record_cost()
9. cache.store()
10. emit events, record feedback
11. return response
```

The convergence key should be `format!("{}:{}", self.run_id, role)` where `role` comes from
`req.role.unwrap_or("default")`.

## DO NOT

- Use an external edit distance crate. Implement the standard two-row DP Levenshtein.
- Make `ThinkingCapCell` or `ConvergenceDetectionCell` public.
- Modify the `ProviderCallCell` or `CacheCell` or `BudgetCell` (those are done in G03/G07).
- Add thinking enforcement to the provider layer. The cap is advisory — it adjusts the
  `max_tokens` option but does not truncate actual provider output.
- Block the entire `call()` method on convergence detection. The mutex lock should be
  short-lived (just a HashMap lookup and Vec push).
- Touch files outside `model_call_service.rs`.
