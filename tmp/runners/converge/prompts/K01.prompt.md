## Task

Add a new public method `route_with_knowledge()` to `CascadeRouter` that applies knowledge-informed score adjustments to the routing decision.

### 1. Add method signature

```rust
pub fn route_with_knowledge(
    &self,
    ctx: &RoutingContext,
    cfactor: Option<&CFactor>,
    agent_id: Option<&str>,
    knowledge: Option<&KnowledgeRoutingAdvice>,
) -> CascadeModel
```

### 2. Implementation

The method should:

1. Call the existing `route_with_cfactor(ctx, cfactor, agent_id)` to get the baseline `CascadeModel`.
2. If `knowledge` is `None` or `knowledge.has_signal` is `false`, return the baseline result unchanged.
3. When knowledge has signal, check if there is a hint for the selected primary model via `knowledge.hint_for(&result.primary)`.
4. Collect all candidate slugs from `self.model_slugs` and compute knowledge-adjusted scores for each:
   - Start with the LinUCB score (or 0.0 if unavailable).
   - Add the knowledge hint score (already clamped to +/-0.3 by the caller).
5. If a different model now has the highest adjusted score AND the delta exceeds 0.1 (to avoid noisy flips), swap the primary model.
6. Log the knowledge influence via `tracing::debug!`:
   ```
   tracing::debug!(
       primary = %result.primary,
       knowledge_hints = ?knowledge.hints.len(),
       swapped = %swapped,
       "knowledge-aware routing applied"
   );
   ```
7. Return the (possibly updated) `CascadeModel`.
