### 3. Add private helper to build routing advice

Add a private method that queries the knowledge store and builds `KnowledgeRoutingAdvice`:

```rust
fn build_knowledge_advice(
    &self,
    candidate_slugs: &[String],
    role: Option<&str>,
    task_hint: Option<&str>,
) -> Option<roko_learn::cascade_router::KnowledgeRoutingAdvice> {
    let store = self.knowledge_store.as_ref()?;

    let query = format!(
        "{} {} routing model",
        role.unwrap_or("default"),
        task_hint.unwrap_or("general")
    );

    let entries = match store.query(&query, 10) {
        Ok(entries) => entries,
        Err(err) => {
            tracing::debug!(error = %err, "knowledge store query failed for routing");
            return None;
        }
    };

    if entries.is_empty() {
        return None;
    }

    // Build hints per candidate model
    use roko_learn::cascade_router::{KnowledgeHint, KnowledgeRoutingAdvice};
    let mut hints = Vec::new();

    for slug in candidate_slugs {
        let slug_lower = slug.to_lowercase();
        let mut score = 0.0_f64;
        let mut supporting = 0_u32;

        for entry in &entries {
            let content_matches = entry.content.to_lowercase().contains(&slug_lower)
                || entry.source_model.as_deref().is_some_and(|sm| sm.eq_ignore_ascii_case(slug))
                || entry.tags.iter().any(|t| t.eq_ignore_ascii_case(slug));
            if !content_matches {
                continue;
            }
            supporting += 1;
            let weight = entry.confidence.clamp(0.0, 1.0);
            if entry.kind == roko_neuro::KnowledgeKind::AntiKnowledge {
                score -= weight * 0.15;
            } else {
                score += weight * 0.10;
            }
        }

        if supporting > 0 {
            hints.push(KnowledgeHint {
                model_slug: slug.clone(),
                score: score.clamp(-0.3, 0.3),
                supporting_entries: supporting,
                reason: format!("{supporting} knowledge entries for {slug}"),
            });
        }
    }

    let has_signal = !hints.is_empty();
    Some(KnowledgeRoutingAdvice { hints, has_signal })
}
```

### 4. Wire into the ModelCaller implementation

In the `ModelCaller::call()` implementation, if the service has a knowledge store:
1. Before model resolution, build knowledge advice using the request's model (if any) and the default model as candidates.
2. Log whether knowledge advice was produced.
3. Store the advice in a local variable for potential use by downstream routing.

This does NOT require changing the `ModelCaller` trait signature -- the advice is used internally within the `call()` method body. If no `CascadeRouter` is directly accessible from `ModelCallService`, just log the advice and store it in the response metadata for now (the full wiring through CascadeRouter happens at the orchestrator level).
