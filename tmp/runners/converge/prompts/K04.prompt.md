# Batch K04 — Record knowledge usage in episode metadata

## Write Scope
- **MODIFY**: `crates/roko-learn/src/feedback_service.rs`

## Task

Extend `FeedbackService` to record which knowledge entries influenced a routing or prompt decision, and score those entries based on the gate verdict.

### 1. Add a new FeedbackEvent variant for knowledge usage

Check if `FeedbackEvent` is defined in `roko-core` (`crates/roko-core/src/foundation/`). If so, add a new variant there. If `FeedbackEvent` is defined locally in roko-learn, add it there. The new variant:

```rust
FeedbackEvent::KnowledgeUsage {
    /// Run or episode ID.
    run_id: String,
    /// IDs of knowledge entries that influenced this decision.
    knowledge_ids: Vec<String>,
    /// Whether the gate passed (true) or failed (false).
    gate_passed: bool,
    /// The model that was selected (to track model-knowledge correlation).
    model_slug: String,
    /// Timestamp.
    timestamp: String,
}
```

If adding a variant to an enum in roko-core is too invasive, instead add a separate method on `FeedbackService` that accepts these fields directly (not through the trait).

### 2. Add `record_knowledge_usage()` method to FeedbackService

```rust
/// Record which knowledge entries influenced a routing/prompt decision
/// and whether the subsequent gate check passed.
pub fn record_knowledge_usage(
    &self,
    run_id: &str,
    knowledge_ids: Vec<String>,
    gate_passed: bool,
    model_slug: &str,
) -> Result<()>
```

Implementation:
1. If `knowledge_ids` is empty, return `Ok(())` immediately.
2. Serialize a JSON object with fields: `type: "knowledge_usage"`, `run_id`, `knowledge_ids`, `gate_passed`, `model_slug`, `timestamp` (ISO 8601).
3. Append it to the `knowledge-feedback.jsonl` file in `self.data_dir`.
4. Log via `tracing::debug!`.

### 3. Add `compute_knowledge_scores()` method

```rust
/// Read the knowledge feedback log and compute per-entry success rates.
///
/// Returns a map of knowledge_id -> (successes, total_uses).
pub fn compute_knowledge_scores(&self) -> Result<HashMap<String, (u32, u32)>>
```

Implementation:
1. Read `knowledge-feedback.jsonl` line by line.
2. Parse each line as JSON.
3. For each `knowledge_id` in each record, increment `total_uses` and conditionally increment `successes` if `gate_passed` is true.
4. Return the aggregated map.
