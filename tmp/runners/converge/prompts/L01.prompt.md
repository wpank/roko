## Write Scope
- **MODIFY**: All `crates/*/Cargo.toml` files (30 files)

## Task

Add a `[package.metadata.roko]` section to every crate's `Cargo.toml` with the correct layer assignment. This metadata will be consumed by the layer-check tool (L02) to enforce dependency direction.

### Layer Map

The 5-layer architecture:

| Layer | Crates |
|-------|--------|
| **L0** (primitives) | `roko-primitives`, `roko-runtime` |
| **L1** (core) | `roko-core` |
| **L2** (services) | `roko-std`, `roko-agent`, `roko-fs`, `roko-compose`, `roko-learn`, `roko-neuro`, `roko-dreams`, `roko-daimon`, `roko-plugin`, `roko-index`, `roko-lang-rust`, `roko-lang-typescript`, `roko-lang-go`, `roko-mcp-code`, `roko-mcp-github`, `roko-mcp-slack`, `roko-mcp-scripts`, `roko-mcp-stdio`, `roko-chain`, `roko-demo` |
| **L3** (orchestration) | `roko-orchestrator`, `roko-gate`, `roko-conductor` |
| **L4** (surfaces) | `roko-cli`, `roko-serve`, `roko-agent-server`, `roko-acp` |

### Classification Rationale for Ambiguous Crates

- `roko-index`: L2 -- it provides code-intelligence indexing services, depends on roko-core but not orchestration.
- `roko-lang-*`: L2 -- language support libraries used by the indexer and agents.
- `roko-mcp-*`: L2 -- MCP integration services, peer to roko-agent.
- `roko-chain`: L2 -- chain witness primitives, peer to other service crates.
- `roko-demo`: L2 -- demo utilities, depends on service crates.
- `roko-acp`: L4 -- ACP protocol surface, peer to roko-serve.

### Format

Add to each `Cargo.toml`, after the `[dependencies]` section (or at the end of the file if no better location):

```toml
[package.metadata.roko]
layer = 2
```

Use the correct layer number from the map above.

### Implementation

For each of the 30 crate Cargo.toml files:
1. Check if `[package.metadata.roko]` already exists (skip if so).
2. Append the section at the end of the file, preceded by a blank line.
3. Use the layer number from the map above.
