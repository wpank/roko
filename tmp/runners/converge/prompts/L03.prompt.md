# Batch L03 — Configure cargo-deny

## Write Scope
- **CREATE**: `deny.toml` (at workspace root: `/Users/will/dev/nunchi/roko/roko/deny.toml`)

## Task

Create a `deny.toml` configuration file for `cargo-deny` that enforces license compliance, source restrictions, and security advisories for the workspace.

### 1. Create `deny.toml`

```toml
# cargo-deny configuration for the roko workspace.
# Run: cargo deny check
# Install: cargo install cargo-deny

[graph]
# Only check workspace crates and their direct/transitive deps.
all-features = true

# ─── Advisories ──────────────────────────────────────────────────────────────

[advisories]
# Use the RustSec advisory database.
db-path = "~/.cargo/advisory-db"
db-urls = ["https://github.com/rustsec/advisory-db"]
# Treat unmaintained crates as warnings, not errors.
unmaintained = "warn"
# Treat yanked crates as warnings.
yanked = "warn"
# Treat advisories with no fix as warnings.
notice = "warn"
# Allow specific advisories that we've reviewed and accepted.
ignore = []

# ─── Licenses ────────────────────────────────────────────────────────────────

[licenses]
# Confidence threshold for license detection.
confidence-threshold = 0.8
# Allow common permissive OSS licenses.
allow = [
    "MIT",
    "Apache-2.0",
    "BSD-2-Clause",
    "BSD-3-Clause",
    "ISC",
    "Unicode-3.0",
    "Unicode-DFS-2016",
    "Zlib",
    "BSL-1.0",
    "CC0-1.0",
    "OpenSSL",
    "MPL-2.0",
]
# Deny copyleft licenses.
deny = [
    "GPL-2.0",
    "GPL-3.0",
    "AGPL-3.0",
]
# Allow dual-licensed crates where at least one license is acceptable.
# This handles the common "MIT OR Apache-2.0" pattern.

# Some crates have non-standard license expressions. Clarify them here.
[[licenses.clarify]]
name = "ring"
expression = "MIT AND ISC AND OpenSSL"
license-files = [{ path = "LICENSE", hash = 0xbd0eed23 }]

# ─── Bans ────────────────────────────────────────────────────────────────────

[bans]
# Warn on multiple versions of the same crate.
multiple-versions = "warn"
# Allow specific duplicate crates that are hard to consolidate.
# Add entries here as needed:
# allow = []
# Deny specific crates known to be problematic.
deny = []
# Allow wildcard dependencies in dev/build (not in normal deps).
wildcards = "allow"

# ─── Sources ─────────────────────────────────────────────────────────────────

[sources]
# Only allow crates from crates.io.
unknown-registry = "deny"
# Allow git sources (some deps may use git refs).
unknown-git = "warn"
# Explicitly allow crates.io.
allow-registry = ["https://github.com/rust-lang/crates.io-index"]
allow-git = []
```

### 2. Verify the configuration syntax

The file should be valid TOML. After creating it, verify with:
```bash
cargo deny check 2>&1 || true
```

The command may produce advisory warnings or license warnings for specific crates -- that is expected and acceptable. What matters is that the config file itself is parsed without errors.

### Notes on the `ring` license clarification

The `ring` crate uses a non-standard license that cargo-deny cannot auto-detect. The clarification block above may need its hash updated. If `cargo deny check` fails specifically on the ring license hash, update the hash value to match what cargo-deny reports, or remove the clarification block entirely and add `ring` to an exceptions list instead:

```toml
[[licenses.exceptions]]
allow = ["ISC", "MIT", "OpenSSL"]
name = "ring"
```
