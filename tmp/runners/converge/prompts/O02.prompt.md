## Task

Create `crates/roko-serve/src/routes/runs.rs`:

```
GET /api/dashboard/runs
```

Handler logic:
1. Build the path: `state.layout.root().join("runtime-events.jsonl")`
2. Call `RuntimeProjection::from_file(&path)` — returns `Result<HashMap<String, RunSummary>>`
3. On error, return an empty runs array (best-effort, the file may not exist yet)
4. Convert the `HashMap<String, RunSummary>` to a `Vec<Value>` sorted by `run_id`
5. Each entry should serialize as:
   ```json
   {
     "run_id": "run_1a2b3c",
     "template": "express",
     "prompt": "fix the auth bug",
     "current_phase": "implement",
     "phases_visited": ["plan", "implement"],
     "gates_passed": ["compile", "test"],
     "gates_failed": [],
     "agents_spawned": 2,
     "is_complete": true,
     "outcome": "workflow_completed"
   }
   ```
6. Return `Json(json!({ "runs": runs_vec }))`

In `mod.rs`:
- Add `mod runs;` alongside the other module declarations
- Add `.merge(runs::routes())` inside `build_router`, next to `.merge(run::routes())`
