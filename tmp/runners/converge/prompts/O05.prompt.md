### Task

Add `output_format` calls at key points in `run_with_workflow_engine` so the v2
engine path produces structured Clack-style output.

#### 1. Add import

At the top of the function body or module:
```rust
use crate::output_format;
```

#### 2. Print intro block before engine.run()

After building the `WorkflowRunConfig` but before calling `engine.run(config)`:

```rust
output_format::intro("roko run");
output_format::step("prompt", &output_format::dim(&truncate(prompt, 60)));
output_format::step("workflow", workflow_template);
output_format::step("model", "claude-sonnet-4-20250514");
output_format::divider();
output_format::bar("starting workflow...");
output_format::divider();
```

The model name can be hardcoded here (it matches the existing `ModelCallService::new`
argument). If the model is extracted to a variable elsewhere, use that instead.

#### 3. Print success block after engine.run() succeeds

Replace the existing `println!` lines with:

```rust
let result = engine.run(config).await
    .map_err(|error| anyhow!("workflow engine failed: {error}"))?;

match &result.outcome {
    WorkflowOutcome::Success { commit_hash } => {
        let hash_str = commit_hash
            .as_deref()
            .map(|h| format!(" ({})", &h[..7.min(h.len())]))
            .unwrap_or_default();
        output_format::success(&format!(
            "workflow completed ({} iteration{}){hash_str}",
            result.iterations,
            if result.iterations == 1 { "" } else { "s" },
        ));
    }
    WorkflowOutcome::Halted { reason } => {
        output_format::error(&format!("workflow halted: {reason}"));
    }
    WorkflowOutcome::Cancelled => {
        output_format::warning("workflow cancelled");
    }
}
output_format::end(&output_format::dim(&result.run_id));
```

#### 4. Remove old println! lines

Delete these two lines:
```rust
println!("Workflow complete: {} ({:?})", result.run_id, result.outcome);
println!("Iterations: {}", result.iterations);
```

They are replaced by the structured output above.
