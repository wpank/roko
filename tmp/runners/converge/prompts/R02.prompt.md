## Batch R02: Feature-gate orchestrate.rs behind legacy-orchestrate

### Write Scope
- **MODIFY**: `crates/roko-cli/src/lib.rs`
- **MODIFY**: `crates/roko-cli/src/orchestrate.rs` (only the module-level attribute)

### Dependencies
- R01 (legacy-orchestrate feature added to Cargo.toml)
- W04 (WorkflowEngine fully wired as the replacement path — ensures the default
  build has a working alternative to orchestrate.rs)

### DO NOT
- Modify any other file
- Delete or rename `orchestrate.rs` — it remains in the source tree, just gated
- Change any code inside `orchestrate.rs` — only add a `#[cfg]` attribute
- Gate `run.rs` or `run_with_workflow_engine` — those are the v2 path and must
  always be available
- Change `Cargo.toml` — the feature is already declared (R01)

### Existing Code Context

`crates/roko-cli/src/lib.rs` — the module declaration and re-exports:
```rust
pub mod orchestrate;
// ...
pub use orchestrate::{OrchestrationReport, PlanRunReport, PlanRunner};
```

`crates/roko-cli/src/orchestrate.rs` — module header:
```rust
//! Plan-level orchestration: read tasks, dispatch agents, run gates, persist.
//!
//! This is the original plan runner that drives `roko plan run`. It is being
//! superseded by the WorkflowEngine (event-driven architecture) but remains
//! as the production path until the v2 engine reaches full parity.
```

Other modules that reference `orchestrate`:
- `crates/roko-cli/src/plan.rs` — uses `PlanRunner` from orchestrate
- `crates/roko-cli/src/commands/util.rs` — calls `PlanRunner::new()` for `plan run`
- `crates/roko-cli/src/main.rs` — may import `PlanRunner` or `OrchestrationReport`

### Task

#### 1. Gate the `mod orchestrate` declaration in `lib.rs`

```rust
#[cfg(feature = "legacy-orchestrate")]
pub mod orchestrate;
```

#### 2. Gate the `pub use orchestrate::...` re-export in `lib.rs`

```rust
#[cfg(feature = "legacy-orchestrate")]
pub use orchestrate::{OrchestrationReport, PlanRunReport, PlanRunner};
```

#### 3. Gate any other `use orchestrate::...` imports in `lib.rs`

Search for any other lines that reference `orchestrate` in `lib.rs` and gate them
with `#[cfg(feature = "legacy-orchestrate")]`.

#### 4. Do NOT gate the module contents

Do not add `#[cfg(feature = "legacy-orchestrate")]` inside `orchestrate.rs` itself.
The gating at the `mod` declaration level is sufficient — when the feature is off,
the entire module is excluded from compilation.

### Done Criteria
```bash
# mod orchestrate is gated
grep -q '#\[cfg(feature = "legacy-orchestrate")\]' crates/roko-cli/src/lib.rs

# pub use orchestrate is gated
grep -B1 'pub use orchestrate' crates/roko-cli/src/lib.rs | grep -q 'cfg.*legacy-orchestrate'

# Compiles WITHOUT the feature (this may fail if other modules
# reference orchestrate — that's expected, R03/R04 will fix it)
cargo check -p roko-cli 2>&1 || echo "Expected: may fail until R03/R04"

# Compiles WITH the feature (everything works as before)
cargo check -p roko-cli --features legacy-orchestrate
```
