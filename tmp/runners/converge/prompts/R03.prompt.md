## Batch R03: Feature-gate dispatch_helpers + agent_spawn behind legacy

### Write Scope
- **MODIFY**: `crates/roko-cli/src/lib.rs`

### Dependencies
- R02 (orchestrate.rs gated behind legacy-orchestrate)

### DO NOT
- Modify any file other than `lib.rs`
- Delete or rename any module files
- Change code inside the gated modules themselves
- Gate modules that are used by both the legacy and v2 paths — only gate modules
  that are exclusively consumed by `orchestrate.rs`
- Change `Cargo.toml`

### Existing Code Context


Modules used ONLY by `orchestrate.rs` (verify by grepping for imports):

1. `dispatch_helpers` — helper functions for agent dispatch in the plan runner.
   Only imported by `orchestrate.rs`.
2. `gate_runner` — gate execution helpers for the plan runner.
   Only imported by `orchestrate.rs`.
3. `knowledge_helpers` — knowledge store queries for enrichment.
   Only imported by `orchestrate.rs`.
4. `learning_helpers` — learning feedback helpers.
   Only imported by `orchestrate.rs`.
5. `task_helpers` — task parsing and manipulation for plan execution.
   May also be used by `plan.rs` — check before gating.
6. `task_parser` — TOML task file parser.
   May also be used by `plan.rs` — check before gating.

**Important**: Before gating each module, verify it is NOT imported by any ungated
module. Run:
```bash
grep -rn 'use crate::dispatch_helpers' crates/roko-cli/src/ --include='*.rs' | grep -v orchestrate
grep -rn 'use crate::gate_runner' crates/roko-cli/src/ --include='*.rs' | grep -v orchestrate
grep -rn 'use crate::knowledge_helpers' crates/roko-cli/src/ --include='*.rs' | grep -v orchestrate
grep -rn 'use crate::learning_helpers' crates/roko-cli/src/ --include='*.rs' | grep -v orchestrate
```

Only gate modules where the grep returns zero non-orchestrate results.

### Task

#### 1. Identify orchestrate-only modules

Run the grep commands above to determine which modules are exclusively used by
`orchestrate.rs`. The expected result is that `dispatch_helpers`, `gate_runner`,
`knowledge_helpers`, and `learning_helpers` are orchestrate-only.

#### 2. Gate confirmed orchestrate-only modules

For each module that is confirmed to be used only by `orchestrate.rs`, add the
cfg gate:

```rust
#[cfg(feature = "legacy-orchestrate")]
pub(crate) mod dispatch_helpers;

#[cfg(feature = "legacy-orchestrate")]
pub(crate) mod gate_runner;

#[cfg(feature = "legacy-orchestrate")]
pub(crate) mod knowledge_helpers;

#[cfg(feature = "legacy-orchestrate")]
pub(crate) mod learning_helpers;
```

#### 3. Gate any associated re-exports

If any of these modules have `pub use` re-exports at the crate root, gate those
too:

```rust
#[cfg(feature = "legacy-orchestrate")]
pub use dispatch_helpers::SomeType;
```

#### 4. Do NOT gate modules shared with v2

If `task_helpers` or `task_parser` are also used by `plan.rs`, `plan_generate.rs`,
or other ungated modules, leave them ungated. The goal is to gate only the modules
that would cause compile errors when `orchestrate.rs` is absent.

#### 5. Do NOT gate `agent_spawn` or `agent_config`

These modules are used by `run.rs` (the v2 path) and possibly `chat.rs`, so they
must remain ungated regardless of whether `orchestrate.rs` also uses them.
