## Batch R04: Ensure cargo check passes without legacy-orchestrate

### Write Scope
- **MODIFY**: `crates/roko-cli/src/run.rs` (primary)
- **MODIFY**: `crates/roko-cli/src/lib.rs` (if needed for additional gates)
- **MODIFY**: any other `crates/roko-cli/src/*.rs` file that fails to compile

### Dependencies
- R03 (dispatch_helpers, gate_runner, etc. gated behind legacy-orchestrate)

### DO NOT
- Modify files outside `crates/roko-cli/src/`
- Add new Cargo.toml dependencies
- Remove functionality from the v2 path — only fix compilation issues
- Change `orchestrate.rs` contents
- Change `roko-runtime` or `roko-core` source files
- Delete any source files — only gate or provide alternatives

### Existing Code Context

After R02 and R03, the following modules are behind `#[cfg(feature = "legacy-orchestrate")]`:
- `orchestrate`
- `dispatch_helpers`
- `gate_runner`
- `knowledge_helpers`
- `learning_helpers`

When building WITHOUT the feature, any file that imports types from these modules
will fail to compile. Common patterns that need fixing:

1. **`lib.rs` re-exports**: `pub use orchestrate::{PlanRunner, ...}` — already
   gated by R02 but verify nothing was missed.

2. **`plan.rs`**: may import `PlanRunner` from `crate::orchestrate`. If so, gate
   the relevant code path with `#[cfg(feature = "legacy-orchestrate")]`.

3. **`commands/util.rs`**: the `plan run` handler calls `PlanRunner`. Gate the
   `plan run` match arm or the function body.

4. **`main.rs`**: may import `PlanRunner` or `OrchestrationReport` from the crate
   root. Gate those imports.

5. **`run.rs`**: should NOT reference orchestrate modules — it uses the v2
   WorkflowEngine. But verify.

### Task

#### 1. Run cargo check and collect errors

```bash
cargo check -p roko-cli 2>&1 | head -100
```

Read the errors carefully. Each error will be one of:
- "unresolved import `crate::orchestrate`" — the importing file needs a cfg gate
- "cannot find type `PlanRunner`" — the usage site needs a cfg gate
- "cannot find module `dispatch_helpers`" — same pattern

#### 2. Fix each error

For each error, apply the minimal fix:

**Pattern A — Import gate**: If a `use crate::orchestrate::...` appears in a file
that has both legacy and v2 code paths, gate just the import:

```rust
#[cfg(feature = "legacy-orchestrate")]
use crate::orchestrate::PlanRunner;
```

**Pattern B — Function gate**: If an entire function only exists for the legacy
path, gate the whole function:

```rust
#[cfg(feature = "legacy-orchestrate")]
pub async fn cmd_plan_run(...) -> Result<i32> {
    // ...
}
```

**Pattern C — Match arm gate**: If a match arm dispatches to the legacy path,
gate just that arm and provide a fallback:

```rust
Command::Plan { action: PlanAction::Run { dir } } => {
    #[cfg(feature = "legacy-orchestrate")]
    {
        // existing PlanRunner code
    }
    #[cfg(not(feature = "legacy-orchestrate"))]
    {
        eprintln!("plan run requires --features legacy-orchestrate");
        Ok(EXIT_AGENT_FAILURE)
    }
}
```

**Pattern D — Type re-export gate**: If a `pub use` at crate root references a
gated module, gate the re-export (should already be done by R02, but verify).

#### 3. Verify `run.rs` has no legacy dependencies

Confirm that `run.rs` does NOT import anything from `orchestrate`,
`dispatch_helpers`, `gate_runner`, `knowledge_helpers`, or `learning_helpers`.
If it does, either gate those imports or provide v2 alternatives.

#### 4. Provide stub error messages

For CLI subcommands that depend on the legacy path (like `plan run`), provide a
clear error message when the feature is not enabled:

```rust
anyhow::bail!(
    "`roko plan run` requires the legacy-orchestrate feature. \
     Build with: cargo build -p roko-cli --features legacy-orchestrate"
);
```

### Done Criteria
```bash
# Compiles cleanly WITHOUT the feature
cargo check -p roko-cli

# No errors, no unresolved imports
cargo check -p roko-cli 2>&1 | grep -c 'error\[' | grep -q '^0$'

# Still compiles WITH the feature
cargo check -p roko-cli --features legacy-orchestrate

# v2 path (run_with_workflow_engine) is always available
grep -q 'pub async fn run_with_workflow_engine' crates/roko-cli/src/run.rs

# Legacy path is available only with feature
cargo check -p roko-cli --features legacy-orchestrate 2>&1 | grep -v warning | grep -c error | grep -q '^0$'
```
