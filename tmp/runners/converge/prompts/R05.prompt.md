## Batch R05: Ensure cargo check passes with legacy-orchestrate

### Write Scope
- **MODIFY**: `crates/roko-cli/src/run.rs` (if needed)
- **MODIFY**: any `crates/roko-cli/src/*.rs` file that fails to compile

### Dependencies
- R04 (cargo check passes WITHOUT legacy-orchestrate)

### DO NOT
- Modify files outside `crates/roko-cli/src/`
- Add new Cargo.toml dependencies
- Remove or change the `#[cfg]` gates added in R02–R04
- Delete any source files
- Modify `roko-runtime` or `roko-core` source files
- Change `orchestrate.rs` contents (beyond fixing compile errors)

### Existing Code Context

After R04, `cargo check -p roko-cli` (without features) compiles cleanly. The
legacy modules (`orchestrate`, `dispatch_helpers`, `gate_runner`,
`knowledge_helpers`, `learning_helpers`) are excluded.

With `--features legacy-orchestrate`, all gated modules are included. This may
introduce NEW compilation errors because:

1. **Duplicate definitions**: If R04 added alternative implementations for the
   v2 path that conflict with legacy types when both are compiled.

2. **Conflicting trait impls**: If a trait impl was duplicated for the v2 path
   that also exists in the legacy path.

3. **Ambiguous imports**: If `use crate::SomeType` resolves to two different
   modules when the feature is enabled.

4. **Stale orchestrate.rs references**: `orchestrate.rs` may reference modules
   or types that were changed during the converge work and are no longer compatible.

### Task

#### 1. Run cargo check with the feature and collect errors

```bash
cargo check -p roko-cli --features legacy-orchestrate 2>&1 | head -100
```

#### 2. Fix compilation errors

Common fix patterns:

**Pattern A — Conditional alternative**: If R04 added a stub function for the
`not(feature)` case, make sure it is properly gated so it doesn't conflict:

```rust
#[cfg(feature = "legacy-orchestrate")]
pub async fn cmd_plan_run(...) -> Result<i32> {
    // real implementation using PlanRunner
}

#[cfg(not(feature = "legacy-orchestrate"))]
pub async fn cmd_plan_run(...) -> Result<i32> {
    anyhow::bail!("requires --features legacy-orchestrate");
}
```

Both branches must have exactly the same function signature.

**Pattern B — Feature-conditional re-exports**: If a type is re-exported from
both `orchestrate` (legacy) and `run` (v2), ensure there is no ambiguity:

```rust
#[cfg(feature = "legacy-orchestrate")]
pub use orchestrate::PlanRunReport;
```

**Pattern C — API drift in orchestrate.rs**: If `orchestrate.rs` references types
or functions that were renamed or removed during the converge work, update the
references to match the current API. Keep changes minimal — fix only what is
needed for compilation.

#### 3. Run clippy with the feature

After fixing compile errors, also run clippy:

```bash
cargo clippy -p roko-cli --features legacy-orchestrate --no-deps -- -D warnings 2>&1 | head -50
```

Fix any warnings that are introduced by the feature gate (unused imports, dead
code in the conditional blocks, etc.). Use `#[cfg_attr(...)]` if needed:

```rust
#[cfg_attr(not(feature = "legacy-orchestrate"), allow(unused_imports))]
use crate::task_parser::parse_tasks;
```

#### 4. Run tests with both configurations

```bash
cargo test -p roko-cli --lib 2>&1 | tail -5
cargo test -p roko-cli --lib --features legacy-orchestrate 2>&1 | tail -5
```

### Done Criteria
```bash
# Compiles cleanly WITH the feature
cargo check -p roko-cli --features legacy-orchestrate

# No errors
cargo check -p roko-cli --features legacy-orchestrate 2>&1 | grep -c 'error\[' | grep -q '^0$'

# Still compiles WITHOUT the feature (R04 not regressed)
cargo check -p roko-cli

# Clippy clean with feature
cargo clippy -p roko-cli --features legacy-orchestrate --no-deps -- -D warnings

# Clippy clean without feature
cargo clippy -p roko-cli --no-deps -- -D warnings

# Tests pass in both configurations
cargo test -p roko-cli --lib
cargo test -p roko-cli --lib --features legacy-orchestrate
```
