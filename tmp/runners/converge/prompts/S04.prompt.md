### Existing Code Context

`CostTable` (from `crates/roko-agent/src/task_runner.rs`):
```rust
#[derive(Debug, Clone, Default, PartialEq)]
pub struct CostTable {
    pub models: HashMap<String, ModelPricing>,
}

#[derive(Debug, Clone, PartialEq)]
pub struct ModelPricing {
    pub input_per_m: f64,   // USD per million input tokens
    pub output_per_m: f64,  // USD per million output tokens
    pub cache_read_per_m: f64,
    pub cache_write_per_m: f64,
}

impl CostTable {
    pub fn insert(&mut self, model_slug: impl Into<String>, pricing: ModelPricing) { ... }
    pub fn calculate(&self, model_slug: &str, usage: &Usage) -> f64 { ... }
}
```

`ModelCallService` already holds a `cost_table: CostTable` and uses it in `call`:
```rust
let calculated_cost = self.cost_table.calculate(&model, &result.usage);
let cost_usd = if calculated_cost > 0.0 {
    calculated_cost
} else {
    f64::from(result.usage.cost_usd)
};
let usage = token_usage(&result.usage, cost_usd);
```

### Task

1. Add a `CostEstimate` struct to `model_call_service.rs`:
   ```rust
   /// Predicted cost for a model call before it is executed.
   #[derive(Debug, Clone)]
   pub struct CostEstimate {
       /// Model that would be used.
       pub model: String,
       /// Estimated input token count (prompt length heuristic: 4 chars/token).
       pub estimated_input_tokens: u64,
       /// Worst-case output tokens (from max_tokens, or a default of 2048).
       pub max_output_tokens: u64,
       /// Predicted cost in USD, or 0.0 if the model has no pricing entry.
       pub predicted_cost_usd: f64,
   }
   ```

2. Add a `cost_predict(&self, req: &ModelCallRequest) -> CostEstimate` method on
   `ModelCallService`:
   - Resolve model using the same `resolve_model` logic
   - Estimate input tokens as `total_chars / 4` across system + all messages
   - Use `req.max_tokens.unwrap_or(2048)` for max output tokens
   - Call `self.cost_table.calculate(model, &usage_estimate)` to compute cost
   - Return `CostEstimate` with those values

3. Ensure the existing `call` method's cost calculation still follows this precedence (already
   in place, just verify):
   - `cost_table.calculate(model, usage)` is tried first (real measured cost)
   - Fall back to `f64::from(result.usage.cost_usd)` (provider-reported cost)
   - `TokenUsage.cost_usd` is always filled in

4. Add a `with_cost_table` builder method that accepts a pre-populated `CostTable`:
   ```rust
   pub fn with_cost_table(mut self, cost_table: CostTable) -> Self {
       self.cost_table = cost_table;
       self
   }
   ```
   (Check if this already exists; if so, leave it. The current file already has it.)

5. Add unit tests:
   - `cost_predict_returns_zero_for_unknown_model`: empty cost table, request with model
     `"unknown-model"`, assert `predicted_cost_usd == 0.0`
   - `cost_predict_estimates_cost_for_known_model`: insert `ModelPricing` with
     `input_per_m=3.0, output_per_m=15.0`, request 1000-char prompt, assert cost > 0.0
