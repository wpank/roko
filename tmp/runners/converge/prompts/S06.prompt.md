### Existing Code Context

`PromptSpec` (from `roko_core::foundation`):
```rust

    pub role: Option<String>,

    pub workdir: Option<PathBuf>,
    pub gate_feedback: Vec<String>,
    pub anti_patterns: Vec<String>,

```

Current `PromptAssemblyService::assemble` (the existing implementation — already uses
`SystemPromptBuilder`):
```rust
#[async_trait]
impl PromptAssembler for PromptAssemblyService {
    async fn assemble(&self, spec: PromptSpec) -> Result<String> {
        let role = resolve_role(spec.role.as_deref());
        let identity = role_identity_for(role);

        let mut builder = SystemPromptBuilder::new(identity);

        if let Some(conventions) = conventions_for_spec(&spec, self.default_conventions.as_deref()) {
            builder = builder.with_conventions(conventions);
        }

        if let Some(task) = spec.task {
            builder = builder.with_task(task);
        }

        for feedback in spec.gate_feedback {
            builder = builder.with_gate_feedback_text(feedback);
        }

        if !spec.anti_patterns.is_empty() {
            builder = builder.with_anti_patterns(spec.anti_patterns);
        }

        Ok(builder.build())
    }
}
```

`SystemPromptBuilder` available builder methods (from `system_prompt_builder.rs`):
```rust
impl SystemPromptBuilder {
    pub fn new(role_identity: impl Into<String>) -> Self
    pub fn with_conventions(mut self, conventions: impl Into<String>) -> Self   // layer 2
    pub fn with_domain(mut self, domain: impl Into<String>) -> Self            // layer 3
    pub fn with_context(mut self, context: impl Into<String>) -> Self          // layer 3b
    pub fn with_task(mut self, task: impl Into<String>) -> Self                // layer 4
    pub fn with_gate_feedback_text(mut self, feedback: impl Into<String>) -> Self  // layer 4b
    pub fn with_tools(mut self, tools: impl Into<String>) -> Self              // layer 5
    pub fn with_relevant_skills(mut self, skills: Vec<Skill>) -> Self         // layer 6
    pub fn with_relevant_playbooks(mut self, playbooks: Vec<Playbook>) -> Self // layer 6
    pub fn with_anti_patterns(mut self, patterns: Vec<String>) -> Self        // layer 7
    pub fn with_affect_state(mut self, state: PadState) -> Self               // layer 8
    pub fn with_cache_markers(mut self, enabled: bool) -> Self
    pub fn with_token_budget(mut self, budget: usize) -> Self
    pub fn build(self) -> String
}
```

`PromptSpec` is currently missing fields for:
- `domain` (layer 3)
- `context` (layer 3b)
- `tools` (layer 5)
- `token_budget` (for capping)
- `workspace_map` (a text snapshot of key project files)

These fields do NOT exist on `PromptSpec` yet. We cannot add them without changing
`roko_core::foundation`, which is out of scope for this prompt. Instead, detect them from
the workdir and service-level configuration.

The `collect_source_context` and `detect_workdir_conventions` helpers already scan workdir
for Rust source context. The service-level `with_conventions(String)` overrides those.

### Task

1. Add new optional fields to `PromptAssemblyService` (NOT to `PromptSpec`):
   ```rust
   pub struct PromptAssemblyService {
       default_conventions: Option<String>,
       // NEW:
       domain_context: Option<String>,       // layer 3 static domain knowledge
       tool_instructions: Option<String>,    // layer 5 tool description
       token_budget: Option<usize>,          // token cap passed to builder
   }
   ```

2. Add corresponding builder methods:
   ```rust
   pub fn with_domain_context(mut self, domain: String) -> Self
   pub fn with_tool_instructions(mut self, tools: String) -> Self
   pub fn with_token_budget(mut self, budget: usize) -> Self
   ```

3. In `assemble`, wire the new layers into the builder:
   ```rust
   if let Some(domain) = &self.domain_context {
       builder = builder.with_domain(domain.clone());
   }
   if let Some(tools) = &self.tool_instructions {
       builder = builder.with_tools(tools.clone());
   }
   if let Some(budget) = self.token_budget {
       builder = builder.with_token_budget(budget);
   }
   ```

4. Enable cache markers by default (they are cache-tier boundary annotations):
   ```rust
   builder = builder.with_cache_markers(true);
   ```

5. Build a `workspace_map` string from the workdir file listing (already computed in
   `collect_source_context`) and inject it via `builder.with_context(workspace_map)`.
   Limit the workspace map to 200 lines max to avoid token blowout.

6. Add a unit test `assembly_includes_domain_context` that builds a service with
   `with_domain_context("DeFi context: Uniswap v4 hooks")`, assembles a prompt, and asserts
   the result contains the domain string.

7. Add a unit test `assembly_includes_tool_instructions` that checks layer 5 appears in the
   assembled output.
