The `SystemPromptBuilder::with_domain` is layer 3 â€” relevant knowledge facts should be
appended to the domain context section rather than a separate section, to stay within the
9-layer structure.

Knowledge query topic: derive from `spec.task` if present (first 200 chars), otherwise
use `spec.role` as a fallback. Do not query if neither is set.

Max entries to inject: 5 (keep prompts focused). Only inject entries with
`confidence >= 0.5` and `tier != Transient` (stable knowledge only).

Formatting pattern for injected knowledge:
```
## Relevant Knowledge

- [Fact] Uniswap v4 pools use hooks for custom logic.
- [Pattern] PoolManager.unlock() is the entry point for all operations.
```

### Task

1. Check `crates/roko-compose/Cargo.toml` for an existing `roko-neuro` dependency. Add it
   if missing:
   ```toml
   roko-neuro = { path = "../roko-neuro" }
   ```

2. Add a `knowledge_store: Option<Arc<roko_neuro::KnowledgeStore>>` field to
   `PromptAssemblyService`.

3. Add a builder method:
   ```rust
   pub fn with_knowledge_store(mut self, store: Arc<roko_neuro::KnowledgeStore>) -> Self {
       self.knowledge_store = Some(store);
       self
   }
   ```

4. In `assemble`, after the task is resolved, query the knowledge store if configured:
   ```rust
   if let Some(store) = &self.knowledge_store {
       let topic = spec.task.as_deref()
           .map(|t| &t[..t.len().min(200)])
           .or(spec.role.as_deref())
           .unwrap_or("general");
       let entries = store.query(topic, 5);
       let facts: Vec<&KnowledgeEntry> = entries.iter()
           .filter(|e| e.confidence >= 0.5 && !matches!(e.tier, KnowledgeTier::Transient))
           .collect();
       if !facts.is_empty() {
           let knowledge_text = format_knowledge_section(&facts);
           // Append to domain context (or inject as a separate domain call)
           let combined = match &self.domain_context {
               Some(existing) => format!("{existing}\n\n{knowledge_text}"),
               None => knowledge_text,
           };
           builder = builder.with_domain(combined);
       }
   }
   ```

5. Add a helper `format_knowledge_section(entries: &[&KnowledgeEntry]) -> String` that
   formats each entry as `- [{kind}] {content}`.

6. Add a unit test `knowledge_store_facts_injected_into_domain_layer`:
   - Create a temp `KnowledgeStore`, insert one high-confidence `Fact` entry
   - Build `PromptAssemblyService` with `with_knowledge_store(store)`
   - Assemble a prompt with `task = Some("Fix the rate limiter".into())`
   - Assert the assembled prompt contains the fact content

7. Add a unit test `knowledge_store_skips_low_confidence_entries`:
   - Insert an entry with `confidence = 0.1`
   - Assert it does NOT appear in the assembled prompt
