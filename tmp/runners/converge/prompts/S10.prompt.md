Current `FeedbackSink` impl:
```rust
#[async_trait]
impl FeedbackSink for FeedbackService {
    async fn record(&self, event: FeedbackEvent) -> Result<()> {
        let should_flush = {
            let mut buf = self.buffer.lock()
                .map_err(|error| RokoError::Invalid(format!("lock poisoned: {error}")))?;
            buf.push(event);
            buf.len() >= self.buffer_capacity
        };

        if should_flush {
            self.flush()?;
        }

        Ok(())
    }
}
```

### Task

1. Add a new optional field to `FeedbackService` for episode recording:
   ```rust
   pub struct FeedbackService {
       data_dir: PathBuf,
       buffer: Mutex<Vec<FeedbackEvent>>,
       buffer_capacity: usize,
       // NEW:
       episode_logger: Option<EpisodeLogger>,
   }
   ```

2. Add a builder method:
   ```rust
   /// Attach an episode logger to record workflow outcomes as episodes.
   ///
   /// When a `WorkflowComplete` event is flushed, the service will also
   /// append an `Episode` record to the logger's JSONL file.
   pub fn with_episode_logger(mut self, logger: EpisodeLogger) -> Self {
       self.episode_logger = Some(logger);
       self
   }
   ```

3. Add a convenience constructor that creates the logger from the standard path:
   ```rust
   /// Create a service from the `.roko` directory with episode recording enabled.
   pub fn from_roko_dir_with_episodes(roko_dir: &Path) -> Self {
       let episodes_path = roko_dir.join("episodes.jsonl");
       let logger = EpisodeLogger::new(episodes_path);
       Self::from_roko_dir(roko_dir).with_episode_logger(logger)
   }
   ```

4. Modify `flush()` to also write episodes. After writing the efficiency JSONL events,
   iterate over the flushed events and for each `WorkflowComplete`, build an `Episode`
   and append it via the logger:
   ```rust
   if let Some(ref logger) = self.episode_logger {
       for event in &events {
           if let FeedbackEvent::WorkflowComplete {
               run_id, outcome, total_cost_usd, total_tokens, duration_ms,
           } = event {
               let episode = build_episode_from_workflow(
                   run_id, outcome, *total_cost_usd, *total_tokens, *duration_ms,
               );
               // Best-effort: log but don't fail the flush
               if let Err(err) = logger.append(&episode).await {
                   tracing::warn!("failed to append episode: {err}");
               }
           }
       }
   }
   ```

5. Because `flush()` is currently synchronous (`pub fn flush(&self) -> Result<()>`) but
   `EpisodeLogger::append` is async, you need to handle this carefully. Two options:
   - **Option A (preferred)**: Keep `flush()` synchronous. Use `tokio::runtime::Handle::try_current()`
     to detect if we are inside a tokio runtime, and if so spawn a blocking task. If not,
     skip episode recording in the synchronous path.
   - **Option B**: Add a separate `async fn flush_async(&self) -> Result<()>` method that
     does the full flush including episodes, and have the sync `flush()` skip episode recording
     (it is called from `Drop` where async is not available).

   Choose Option B. Keep the sync `flush()` unchanged (it already works for efficiency JSONL).
   Add `flush_async()` that calls the sync flush first, then appends episodes.

6. Update the `FeedbackSink::record` impl to call `flush_async()` instead of `flush()`
   when the buffer is full. Since `record` is already async, this works naturally:
   ```rust
   async fn record(&self, event: FeedbackEvent) -> Result<()> {
       let should_flush = { /* ... same lock logic ... */ };
       if should_flush {
           self.flush_async().await?;
       }
       Ok(())
   }
   ```

7. Write `build_episode_from_workflow` as a private helper:
   ```rust
   fn build_episode_from_workflow(

           // ... default the rest

       }
   }
   ```
