
Express happy path: `Pending -> Implementing -> Gating -> Committing -> Complete`.
The `EffectDriver` calls `model_caller.call()` (for the implementer agent) and

`prompt_assembler.assemble()` before every model call and `feedback_sink.record()`
after.

## Task

Implement two tests inside the existing `mod tests` block:

### `workflow_engine_express_completes_with_success`

1. Define four minimal mock structs in the test module. Each implements the
   corresponding service trait by returning a fixed canned value:
   - `MockModelCaller` — returns `ModelCallResponse { content: "done".into(), model:
     "mock".into(), usage: TokenUsage::default(), stop_reason: None }`.
   - `MockPromptAssembler` — returns `Ok("system prompt".to_string())`.
   - `MockFeedbackSink` — returns `Ok(())`.
   - `MockGateRunner` — returns `GateReport { verdicts: vec![] }` (empty verdicts
     means `all_passed()` returns `true`).
2. Build `EffectServices` from the four mocks (each wrapped in `Arc::new`).
3. Build a `WorkflowRunConfig` with `workflow: WorkflowConfig::express()`,
   `prompt: "fix the bug".into()`, `workdir: PathBuf::from(".")`,
   `enabled_gates: vec![]`, `commit_prefix: None`.
4. Create a `WorkflowEngine` and call `.run(config).await`.
5. Assert:
   - `result.is_ok()`.
   - `result.outcome` matches `WorkflowOutcome::Success { .. }`.
   - `result.iterations == 1`.
   - `result.run_id.starts_with("run_")`.

### `workflow_engine_express_emits_lifecycle_events`

Same setup as above, but attach a `WorkflowEventConsumer` implementation that
collects emitted events into a `Arc<Mutex<Vec<WorkflowEvent>>>` (use
`parking_lot::Mutex` or `std::sync::Mutex`).

After `engine.run()` completes assert that the collected event list:
- Contains exactly one `WorkflowStarted` event with `template == "express"`.
- Contains at least one `PhaseTransition` event.
- Contains exactly one `WorkflowCompleted` event whose `outcome` is
  `WorkflowOutcome::Success { .. }`.
