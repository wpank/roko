## Task

Implement three tests inside the existing `mod tests` block:

### `checkpoint_preserves_phase_and_iteration`

Drive a `WorkflowConfig::express()` state machine from `Start` through
`AgentCompleted` (so it reaches `Gating`). Clone the state at that point.
Assert:
- `checkpoint.phase == Phase::Gating`.
- `checkpoint.iteration == 1`.
- `checkpoint.original_prompt == "fix bug"`.

### `resume_from_gating_checkpoint_skips_implement`

Using the checkpoint from above (or re-derive it), call `resumed.step(GatesPassed)`
and assert it returns `PipelineOutput::Commit` without re-running the implement
phase. Then call `resumed.step(CommitDone { hash: "abc123".into() })` and assert:
- Return value matches `PipelineOutput::Done { outcome: WorkflowOutcome::Success { commit_hash: Some("abc123".into()) } }`.
- `resumed.phase == Phase::Complete`.

### `resume_does_not_affect_original`

Drive original state machine to `Gating`. Clone it as `checkpoint`. Drive the
**original** to `Complete` via `GatesPassed` -> `CommitDone`. Drive the
**checkpoint** independently through a different path: inject `GateFailed` to
trigger autofix, then bring it to completion. Assert at the end that
`original.phase == Phase::Complete` and `checkpoint.phase == Phase::Complete`
but that `checkpoint.autofix_attempts > 0` while `original.autofix_attempts == 0`.

This confirms the two instances are fully independent after the clone.
