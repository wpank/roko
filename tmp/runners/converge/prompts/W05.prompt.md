#### 4. Verify the AcpWorkflowEventConsumer bridge is correct

The `AcpWorkflowEventConsumer` in `runner.rs` bridges `WorkflowEvent` variants
to `CognitiveEvent` variants via the `AcpAdapter`. Verify these mappings exist
and are correct:

| WorkflowEvent | CognitiveEvent |
|---|---|
| `WorkflowStarted` | `PlanUpdate` (with initial plan entries) |
| `PhaseTransition` | `PlanUpdate` (updated plan status) |
| `AgentOutput { chunk }` | `TokenChunk(chunk)` |
| `AgentCompleted` | ignored (completion comes from `WorkflowCompleted`) |
| `GateStarted` | `ToolCallStart` (gate as tool) |
| `GatePassed` / `GateFailed` | `ToolCallComplete` |
| `WorkflowCompleted` | `Complete` |

If any mappings are missing, add them to the `WorkflowEventConsumer::consume`
implementation on `AcpWorkflowEventConsumer`.
