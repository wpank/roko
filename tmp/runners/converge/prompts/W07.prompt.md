## Batch W07: Wire roko-serve background tasks to WorkflowEngine events

### Write Scope
- **MODIFY**: `crates/roko-serve/src/lib.rs`

### Dependencies
- E06 (WorkflowEngine progress callback and event emission wired)

### DO NOT
- Modify `routes/run.rs`, `adapters.rs`, `state.rs`, or `routes/mod.rs`
- Modify any files in `crates/roko-runtime/`
- Add new Cargo.toml dependencies
- Remove `start_state_hub_bridge` or `start_state_snapshot_saver`
- Change `start_server_background` or `ServerBuilder::start_background` signatures
- Break the existing SSE/WebSocket event streams

### Existing Code Context

`lib.rs:run_server_with_state` (lines 429-449) starts background components:
`config_watcher`, `prd_publish_subscriber`, `state_hub_bridge`, `state_saver`,
`job_runner`, then builds the router and binds the listener.

`AppState` has `state_hub: SharedStateHub`, `event_bus`, `workdir`, `runtime`, `active_runs`.

`EventConsumer` trait: `fn consume(&self, event: &RuntimeEvent)` (sync, Send+Sync).

`RuntimeEvent` has 12 variants: `WorkflowStarted`, `PhaseTransition`, `AgentSpawned`,
`AgentOutput`, `AgentCompleted`, `AgentFailed`, `GateStarted`, `GatePassed`,
`GateFailed`, `FeedbackRecorded`, `StateCheckpointed`, `WorkflowCompleted`.

### Task

#### 1. Add `DashboardEventBridge` implementing `EventConsumer`

```rust
/// Bridges WorkflowEngine RuntimeEvents to SharedStateHub as DashboardEvents.
struct DashboardEventBridge {
    state_hub: SharedStateHub,
}

impl DashboardEventBridge {
    fn new(state_hub: SharedStateHub) -> Self {
        Self { state_hub }
    }
}
```

Implement `EventConsumer` with these mappings:

| RuntimeEvent | DashboardEvent |
|---|---|
| `WorkflowStarted { run_id, prompt }` | `PlanStarted` + `TaskStarted` |
| `AgentSpawned { agent_id, role, model }` | `AgentSpawned` |
| `AgentOutput { agent_id, chunk }` | `AgentOutput` |
| `GatePassed { run_id, gate_name, duration_ms }` | `EventLogEntry` (PASS) |
| `GateFailed { run_id, gate_name, duration_ms }` | `EventLogEntry` (FAIL) |
| `WorkflowCompleted { run_id, outcome }` | `TaskCompleted` |
| All others | Ignored (empty vec) |

Use `state_hub.publish_batch(events)` for non-empty event lists.
Truncate `run_id` to 8 chars for `plan_id`: `format!("wf-{}", &run_id[..8.min(run_id.len())])`.

If `DashboardEvent::GateCompleted` exists, use it. Otherwise use `EventLogEntry`:
```rust
DashboardEvent::EventLogEntry {
    timestamp_ms: now_millis(),
    event_type: "gate_result".into(),
    plan_id: short_id,
    task_id: gate_name.clone(),
    message: format!("{} {} ({}ms)", if passed { "PASS" } else { "FAIL" }, gate_name, duration_ms),
}
```

#### 2. Export a constructor for routes layer

```rust
/// Create a DashboardEventBridge for attaching to WorkflowEngine instances.
pub fn dashboard_event_bridge(state: &Arc<AppState>) -> Arc<dyn EventConsumer> {
    Arc::new(DashboardEventBridge::new(state.state_hub.clone()))
}
```

Export from crate root so `routes/run.rs` can call it (W08 will use this).

#### 3. No background task needed

The bridge is synchronous (`EventConsumer::consume` is sync). It is attached
to `WorkflowEngine` instances by the routes layer, not started as a standalone
task. No changes to `run_server_with_state` are required beyond ensuring the
export is accessible.
