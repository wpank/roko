## Task
Record CascadeRouter observations from serve gateway inference completions

## Runner Context
You are working in runner `mega-parity`, batch D8_C03.
This batch is part of Demo Enhancement D8C — Backend reliability wiring.

**Depends on:** D8_C01 (CascadeRouter must be loaded into AppState at startup)

## Problem
The gateway `POST /api/inference/complete` handler dispatches LLM requests and records per-model token/cost counters (B1), but never feeds outcomes back to the `CascadeRouter`. After D8_C01 loads the router, it sits idle during serve — no new observations are recorded. This means the bandit cannot learn from gateway traffic, and the UCB model selection policy stagnates. The `record_observation()` method on `CascadeRouter` needs to be called with model slug, success, and a reward signal after each inference completes.

## Architecture Contract
- `state.cascade_router` is `RwLock<Option<CascadeRouter>>` — acquire a write lock to record observations.
- `CascadeRouter::record_observation()` requires a `RoutingContext`, model slug, reward (f64), and success (bool). The RoutingContext is already built inside `select_model_via_router()` but not persisted — you need to build one from the request hints.
- Recording observations should be fire-and-forget: errors logged, never propagated to the HTTP response.
- Do NOT block the inference response on observation recording.

    State(state): State<Arc<AppState>>,
    ValidJson(body): ValidJson<CompletionRequest>,
) -> Result<impl IntoResponse, ApiError> {
    let request_id = uuid::Uuid::new_v4().to_string();
    let config = state.load_roko_config();

    let hints = RoutingHints { /* ... from body ... */ };
    let model_slug = if let Some(ref requested) = body.model {
        let resolved = resolve_model(&config, requested);
        resolved.slug
    } else {
        select_model_via_router(&state, &hints).await
    };

    let req = model_call_request(model_slug.clone(), &body.messages, ...);
    let model_response = state.model_call_service.call(req).await.map_err(|err| {
        state.provider_health.record_failure(&model_slug);
        ApiError::internal(format!("inference dispatch failed: {err}"))

    state.provider_health.record_success(&model_slug);
    let served_model = model_response.model.clone();
    let input_tokens = model_response.usage.input_tokens;
    let output_tokens = model_response.usage.output_tokens;
    let cost_usd = model_response.usage.cost_usd;

    // B1: accumulate per-model token + cost counters.
    let counters = state.gateway_counters_for(&served_model).await;
    counters.record(input_tokens, output_tokens, cost_usd);

    // Publish event for SSE/WS consumers.
    state.event_bus.publish(ServerEvent::AgentOutput { ... });

    let response = completion_response_from_model_call(request_id, model_response);
    Ok(Json(response))
}

## Changes Required

### Step 1 — Extract a helper to build RoutingContext from RoutingHints

The `select_model_via_router` function already builds a `RoutingContext` from `RoutingHints`. Extract the RoutingContext construction into a reusable helper so both model selection and observation recording can use it.

**File:** `crates/roko-serve/src/routes/gateway.rs`

Add a new function near `select_model_via_router`:
```rust
/// Build a [`RoutingContext`] from caller-provided [`RoutingHints`] and live state.
fn routing_context_from_hints(hints: &RoutingHints, active_agents: u32) -> RoutingContext {
    let task_category = hints
        .task_category
        .as_deref()
        .map(parse_task_category)
        .unwrap_or(TaskCategory::Implementation);
    let complexity = hints
        .complexity
        .as_deref()
        .map(parse_complexity)
        .unwrap_or(TaskComplexityBand::Standard);
    let role = hints
        .role
        .as_deref()
        .map(parse_agent_role)
        .unwrap_or(AgentRole::Implementer);
    let iteration = hints.iteration.unwrap_or(1);
    let has_prior_failure = hints.has_prior_failure.unwrap_or(iteration > 1);

    RoutingContext {
        task_category,
        complexity,
        iteration,
        role,
        crate_familiarity: 0.5,
        has_prior_failure,
        conductor_load: 0.0,
        active_agents,
        ready_queue_depth: 0,
        max_queue_wait_hours: 0.0,
        daimon_policy: roko_core::DaimonPolicy::default(),
    }
}
```

Update `select_model_via_router` to call this helper instead of duplicating the logic.

### Step 3 — Record failure observation on inference error

In the error path (where `provider_health.record_failure` is called), record a failure observation:

Find this block (~line 302-304):
```rust
    let model_response = state.model_call_service.call(req).await.map_err(|err| {
        state.provider_health.record_failure(&model_slug);
        ApiError::internal(format!("inference dispatch failed: {err}"))
    })?;
```

The `map_err` closure cannot be async, so move the failure observation to a separate check. Instead, wrap the entire dispatch in a match:

```rust
    let model_response = match state.model_call_service.call(req).await {
        Ok(resp) => resp,
        Err(err) => {
            state.provider_health.record_failure(&model_slug);
            // Record failure observation for CascadeRouter.
            let active_agents = state.operations.read().await.len() as u32;
            let routing_ctx = routing_context_from_hints(&hints, active_agents);
            if let Some(ref router) = *state.cascade_router.read().await {
                router.record_observation(&routing_ctx, &model_slug, 0.0, false);
            }
            return Err(ApiError::internal(format!("inference dispatch failed: {err}")));
        }
    };
```
