## Task

Implement the Minimal strategy: skip playbooks, skip knowledge, skip replan.

## Runner Context

You are working in runner `mega-parity`, batch D8_F07.
Part of D8F — Bench Self-Learning Pipeline.

**Depends on:** D8_F06 (which adds `BenchStrategy` enum to `run_once_with_config`)

## Problem

After D8_F06 lands, `run_once()` receives a `BenchStrategy` but ignores it. The Minimal strategy is the baseline — it should explicitly skip all enrichment features so we can measure the raw model performance.

Currently, the `run_once` function (behind `#[cfg(feature = "legacy-orchestrate")]`) and the `run_with_workflow_engine` path both use whatever enrichment is configured globally. For bench runs, Minimal strategy must suppress:

1. Playbook injection into system prompts
2. Knowledge store queries (strategy fragments, anti-patterns)
3. Gate failure replan (no second chance)

## Changes Required

### Step 1 — Add strategy field to orchestration context

In `run.rs`, add an optional strategy field to the function that `serve_runtime.rs` calls. This could be:
- A new parameter on `run_once`: `strategy: Option<BenchStrategy>`
- Or a new wrapper: `run_once_with_strategy`

The strategy needs to flow down to where enrichment decisions are made.

### Step 2 — Gate enrichment behind strategy checks

Find the code paths where playbooks, knowledge, and replan are invoked. Guard them:

```rust
let skip_enrichment = strategy == Some(BenchStrategy::Minimal);

// Before playbook injection:
if !skip_enrichment {
    // ... existing playbook code ...
}

// Before knowledge query:
if !skip_enrichment {
    // ... existing knowledge code ...
}

// Before replan on gate failure:
if !skip_enrichment {
    // ... existing replan code ...
}
```

### Step 3 — Wire from serve_runtime

In `RokoCliRuntime::run_once_with_config`, pass `overrides.strategy` through to the run function.
