## Problem

The Neuro-Augmented strategy adds the knowledge store on top of Context-Enriched. It enables:
- **Playbooks** (same as Context-Enriched)
- **Skills** (same as Context-Enriched)
- **Strategy fragments**: Query `KnowledgeStore` for `StrategyFragment` and `Heuristic` entries, inject via `build_strategy_fragment_context()`
- **Anti-patterns**: Query `KnowledgeStore` for `AntiKnowledge` entries via `query_anti_knowledge_patterns()`, inject as warnings

It does NOT enable:
- Gate failure replan (that's Full Cascade only)

This strategy shows how the neuro knowledge store (durable learnings) improves over plain playbooks.

## Changes Required

### Step 1 — Add Neuro-Augmented branch to strategy dispatch

In `run.rs`, extend the strategy match:

```rust
Some(BenchStrategy::NeuroAugmented) => {
    // Enable playbooks + skills (same as ContextEnriched)
    // Enable knowledge store queries
    // Skip replan
}
```

### Step 2 — Knowledge store injection

Use the existing helpers from `knowledge_helpers.rs`:

1. **Strategy fragments**: Call `build_strategy_fragment_context(&knowledge_store, role, task_def, task_text, current_model)` — returns `Option<String>` markdown block to inject into the system prompt.

2. **Anti-patterns**: Call `query_anti_knowledge_patterns(&knowledge_store, task_text, 5)` — returns `Vec<String>` of anti-pattern descriptions. Render these as a "## Common Mistakes to Avoid" section.

### Step 3 — Combine with playbooks

The Neuro-Augmented prompt should contain:
1. Base system prompt (layers 1-5)
2. Layer 6: Playbook techniques (from PlaybookStore)
3. Strategy fragments (from KnowledgeStore)
4. Anti-pattern warnings (from KnowledgeStore)
5. Task prompt
