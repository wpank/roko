## Task

Implement the Full Cascade strategy: all enrichment + replan on gate failure.

## Runner Context

You are working in runner `mega-parity`, batch D8_F10.
Part of D8F — Bench Self-Learning Pipeline.

**Depends on:** D8_F06 (which adds `BenchStrategy` enum)

## Problem

The Full Cascade strategy enables everything the system can do:
- **Playbooks** (query + inject)
- **Skills** (render + inject)
- **Strategy fragments** (knowledge store query + inject)
- **Anti-patterns** (knowledge store query + inject as warnings)
- **Gate failure replan**: When a gate fails, use `build_gate_failure_plan_revision()` to generate a revised prompt and retry the task once

This is the maximum-enrichment strategy. It should produce the highest pass rate but also the highest token cost.

## Changes Required

### Step 1 — Add Full Cascade branch to strategy dispatch

In `run.rs`, extend the strategy match:

```rust
Some(BenchStrategy::FullCascade) => {
    // Enable all enrichment (same as NeuroAugmented)
    // Enable replan on gate failure
}
```

### Step 2 — Replan on gate failure

The replan logic already exists in `orchestrate.rs` as `build_gate_failure_plan_revision()`. For the bench path:

1. After running the task and checking gates, if any gate fails:
2. Collect gate failure details (which gate, what error)
3. Build a revised prompt that includes the failure context:
   ```
   The previous attempt failed gate checks:
   - {gate_name}: {failure_detail}

   Please fix the issues and try again.

   Original task: {original_prompt}
   ```
4. Re-run the task with the revised prompt (single retry, not infinite)
5. Use the retry result as the final result

### Step 3 — Track retry in result

If a retry happened, the `BenchTaskResult` should reflect:
- The retry's success/failure status
- Combined token usage (original + retry)
- Combined duration
