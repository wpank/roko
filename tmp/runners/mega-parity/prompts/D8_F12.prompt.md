## Task

Wire playbook extraction into the bench post-task hook: after a successful bench task, extract and save a playbook.

## Runner Context

You are working in runner `mega-parity`, batch D8_F12.
Part of D8F — Bench Self-Learning Pipeline.

**Depends on:** D8_F11 (extract_playbook_from_episode), D8_F06 (BenchStrategy enum)

## Problem

D8_F11 adds `extract_playbook_from_episode()` to `PlaybookStore`, but nothing calls it. The bench execution loop in `routes/bench.rs` → `execute_bench_run()` → `run_once_with_config()` needs a post-task hook that:

1. Checks if the task succeeded
2. Checks if the strategy is not Minimal (Minimal skips learning)
3. Extracts the tool-call sequence from the run result
4. Calls `extract_playbook_from_episode()` to create a playbook
5. Saves it to the `PlaybookStore`

## Changes Required

### Step 1 — Accept PlaybookStore in run_once path

The `run_once_with_config` in `serve_runtime.rs` needs access to a `PlaybookStore`. Either:
- Add it to `RokoCliRuntime` struct
- Pass it through `BenchConfigOverrides` (not ideal)
- Have the caller in `routes/bench.rs` do the extraction after getting `RunResult`

The cleanest approach: add `PlaybookStore` as a field on `RokoCliRuntime` (initialized from the workdir `.roko/learn/playbooks/` path), and have `run_once_with_config` do the extraction internally when strategy permits.

### Step 2 — Post-task extraction in serve_runtime

After `run_once` returns successfully, and if strategy is not `Minimal`:

```rust
if result.success && strategy != BenchStrategy::Minimal {
    // Extract tool calls from the episode/output
    let tool_calls = extract_tool_calls_from_output(&result);
    if let Some(playbook) = extract_playbook_from_episode(task_id, prompt, &tool_calls) {
        if let Err(e) = playbook_store.save_or_merge(&playbook).await {
            tracing::warn!("Failed to save playbook: {e}");
        }
    }
}
```

### Step 3 — Extract tool calls from RunResult

The `RunResult` has `output_text: Option<String>` which may contain tool call traces. Parse this to extract `(tool_name, description)` pairs. If the output format doesn't contain structured tool calls, use the episode log file (`.roko/episodes.jsonl`) — read the most recent entry.
