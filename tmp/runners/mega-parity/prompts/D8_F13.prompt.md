Update playbook success/failure counters after each bench task.

## Problem

When playbooks are injected into an agent's prompt (ContextEnriched, NeuroAugmented, FullCascade strategies), we need to track whether the advice helped. After each bench task:
- If the task **passed** and playbooks were injected → increment `success_count` on those playbooks
- If the task **failed** and playbooks were injected → increment `failure_count` on those playbooks

This feedback loop makes `PlaybookStore.query()` ranking smarter over time — playbooks with high success rates get recommended more.

## Existing API

`PlaybookStore` already has:
```rust
pub async fn record_outcome(&self, id: &str, success: bool) -> io::Result<bool>
```

This increments `success_count` or `failure_count` on the playbook with the given ID.

## Changes Required

### Step 1 — Track which playbooks were injected

In `run.rs`, when a strategy injects playbooks into the prompt, collect their IDs:

```rust
let injected_playbook_ids: Vec<String> = queried_playbooks.iter().map(|p| p.id.clone()).collect();
```

### Step 2 — Record outcomes after task completion

After the task result is known:

```rust
for pb_id in &injected_playbook_ids {
    if let Err(e) = playbook_store.record_outcome(pb_id, task_passed).await {
        tracing::warn!("Failed to record playbook outcome for {pb_id}: {e}");
    }
}
```

### Step 3 — Only record when playbooks were actually used

Don't record outcomes for:
- Minimal strategy (no playbooks injected)
- Runs where no relevant playbooks were found (empty query result)
