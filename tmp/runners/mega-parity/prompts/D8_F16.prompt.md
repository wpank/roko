## Task

Wire anti-pattern extraction into the bench post-task hook: after a failed bench task, extract and save an anti-pattern.

## Runner Context

You are working in runner `mega-parity`, batch D8_F16.
Part of D8F — Bench Self-Learning Pipeline.

**Depends on:** D8_F15 (extract_anti_pattern_from_failure), D8_F06 (BenchStrategy enum)

## Problem

D8_F15 adds `extract_anti_pattern_from_failure()` to the knowledge store, but nothing calls it. The bench execution path needs a post-task hook that:

1. Checks if the task failed
2. Checks if the strategy is not Minimal (Minimal skips learning)
3. Extracts gate failure details from the run result
4. Calls `extract_anti_pattern_from_failure()` to create an anti-pattern entry
5. Saves it to the `KnowledgeStore`

## Changes Required

### Step 1 — Accept KnowledgeStore in run path

Similar to D8_F12 for PlaybookStore, the `RokoCliRuntime` needs access to a `KnowledgeStore`. Add it as a field, initialized from `.roko/neuro/` path.

### Step 2 — Post-task extraction in serve_runtime

After `run_once` returns with failure, and if strategy is not `Minimal`:

```rust
if !result.success && strategy != BenchStrategy::Minimal {
    // Determine which gate failed (from run result or episode log)
    let gate_name = determine_failed_gate(&result);
    let gate_error = result.output_text.as_deref().unwrap_or("unknown error");

    let entry = extract_anti_pattern_from_failure(
        task_id,
        prompt,
        &gate_name,
        gate_error,
        result.output_text.as_deref(),
    );

    if let Err(e) = knowledge_store.add(entry) {
        tracing::warn!("Failed to save anti-pattern: {e}");
    }
}
```

### Step 3 — Determine failed gate

The `RunResult` currently only has `success: bool` and `output_text`. To know which gate failed, check:
- The output text for gate failure markers (e.g., "Gate 'compile' failed")
- Or read the most recent episode from `.roko/episodes.jsonl`
- Or use "unknown" as fallback gate name
