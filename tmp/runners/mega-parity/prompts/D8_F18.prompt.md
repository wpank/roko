## Task

Pass `PlaybookStore` and `KnowledgeStore` through `CliRuntime` into `run_once` so bench runs can read and write learning state.

## Runner Context

You are working in runner `mega-parity`, batch D8_F18.
Part of D8F — Bench Self-Learning Pipeline.

**Depends on:** D8_F06 (BenchStrategy enum)

## Problem

The bench execution loop calls `runtime.run_once_with_config()` which calls `run_once()`. The strategy implementations (D8_F07-F10) need `PlaybookStore` and `KnowledgeStore` to query playbooks and anti-patterns. But `run_once` currently has no access to these stores.

The stores need to flow from the application state (where they're initialized at startup) through the `CliRuntime` trait into the actual run function.

## Changes Required

### Step 1 — Add stores to RokoCliRuntime

In `serve_runtime.rs`, add fields to `RokoCliRuntime`:

```rust
pub struct RokoCliRuntime {
    config: Config,
    repo_registry: RepoRegistry,
    playbook_store: Option<Arc<PlaybookStore>>,    // NEW
    knowledge_store: Option<Arc<KnowledgeStore>>,  // NEW
}
```

Initialize them in the constructor from the workdir:
```rust
impl RokoCliRuntime {
    pub fn new(config: Config, repo_registry: RepoRegistry) -> Self {
        let workdir = config.workdir();
        let playbook_store = PlaybookStore::new(workdir.join(".roko/learn/playbooks"));
        let knowledge_store = KnowledgeStore::new(workdir.join(".roko/neuro/knowledge.jsonl"));
        Self {
            config, repo_registry,
            playbook_store: Some(Arc::new(playbook_store)),
            knowledge_store: Some(Arc::new(knowledge_store)),
        }
    }
}
```

### Step 2 — Pass stores through run_once_with_config

In `run_once_with_config`, pass the stores to `run_once`:

```rust
async fn run_once_with_config(&self, workdir: &Path, prompt: &str, overrides: &BenchConfigOverrides) -> Result<RunResult> {
    // ... existing config cloning ...
    run_once_with_stores(
        workdir, &config, prompt, None,
        overrides.strategy,
        self.playbook_store.clone(),
        self.knowledge_store.clone(),
    ).await
}
```

### Step 3 — Update run_once to accept stores

In `run.rs`, create a new entry point (or extend `run_once`) that accepts optional stores:

```rust
pub async fn run_once_with_stores(
    workdir: &Path,
    config: &Config,
    prompt_text: &str,
    external_hub: Option<&StateHub>,
    strategy: BenchStrategy,
    playbook_store: Option<Arc<PlaybookStore>>,
    knowledge_store: Option<Arc<KnowledgeStore>>,
) -> Result<RunReport>
```

The existing `run_once` should delegate to this with `None` stores and default strategy.
