## Task

Emit `BenchLearningEvent` SSE events with `playbooks_created` and `anti_patterns_created` counts after each bench task.

## Runner Context

You are working in runner `mega-parity`, batch D8_F19.
Part of D8F — Bench Self-Learning Pipeline.

**Depends on:** D8_F12 (playbook extraction wired), D8_F16 (anti-pattern extraction wired)

## Problem

After D8_F12 and D8_F16, playbooks and anti-patterns are being created during bench runs, but the frontend has no way to know about them. We need a new SSE event type that tells the UI what learning happened after each task.

## Existing SSE Infrastructure

`ServerEvent` enum in `events.rs` already has bench events:
```rust
ServerEvent::BenchTaskCompleted { run_id, task_id, passed, duration_ms, cost_usd }
```

The SSE endpoint at `GET /bench/events` broadcasts all `ServerEvent`s to connected clients.

## Changes Required

### Step 1 — Add BenchLearningEvent variant to ServerEvent

In `events.rs`:

```rust
ServerEvent::BenchLearningEvent {
    run_id: String,
    task_id: String,
    playbooks_created: u32,
    anti_patterns_created: u32,
    total_playbooks: u32,
    total_anti_patterns: u32,
}
```

Add serialization in the `to_sse_data` method (or wherever ServerEvent is serialized):
```rust
ServerEvent::BenchLearningEvent { run_id, task_id, playbooks_created, anti_patterns_created, total_playbooks, total_anti_patterns } => {
    serde_json::json!({
        "type": "BenchLearningEvent",
        "run_id": run_id,
        "task_id": task_id,
        "playbooks_created": playbooks_created,
        "anti_patterns_created": anti_patterns_created,
        "total_playbooks": total_playbooks,
        "total_anti_patterns": total_anti_patterns,
    })
}
```

### Step 2 — Emit after post-task learning hooks

In `routes/bench.rs`, after the post-task playbook/anti-pattern extraction (D8_F12/D8_F16), emit the event:

```rust
// After task completion and learning extraction:
let playbooks_created = if new_playbook_saved { 1 } else { 0 };
let anti_patterns_created = if new_anti_pattern_saved { 1 } else { 0 };

state.event_bus.publish(ServerEvent::BenchLearningEvent {
    run_id: run_id.clone(),
    task_id: task.id.clone(),
    playbooks_created,
    anti_patterns_created,
    total_playbooks: playbook_store.list().await.map(|v| v.len() as u32).unwrap_or(0),
    total_anti_patterns: knowledge_store.stats().map(|s| s.anti_knowledge_count as u32).unwrap_or(0),
});
```

### Step 3 — Count tracking

Track cumulative counts within the bench run loop so the totals are accurate. The `playbook_store.list()` and `knowledge_store.stats()` calls give the global totals.
