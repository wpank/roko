## Task

Add `learning_summary` to `BenchRunSummary`: aggregate learning metrics at the end of a bench run.

## Runner Context

You are working in runner `mega-parity`, batch D8_F20.
Part of D8F — Bench Self-Learning Pipeline.

**Depends on:** D8_F19 (BenchLearningEvent SSE)

## Problem

After a bench run completes, the `BenchRunSummary` shows pass rate, cost, and duration — but not what learning happened. The frontend needs aggregate learning stats to show on the results tab and learning curve chart.

## Changes Required

### Step 1 — Add BenchLearningSummary struct

In `bench.rs`:

```rust
#[derive(Debug, Clone, Serialize, Deserialize, Default)]
pub struct BenchLearningSummary {
    pub playbooks_created: u32,
    pub playbooks_used: u32,
    pub anti_patterns_created: u32,
    pub anti_patterns_used: u32,
    pub knowledge_entries_queried: u32,
    pub strategy: String,
}
```

### Step 2 — Add to BenchRunSummary

```rust
pub struct BenchRunSummary {
    pub total_tasks: usize,
    pub passed: usize,
    pub failed: usize,
    pub pass_rate: f64,
    pub total_duration_ms: u64,
    pub total_cost_usd: f64,
    pub total_input_tokens: u64,
    pub total_output_tokens: u64,
    pub learning: Option<BenchLearningSummary>,  // NEW
}
```

### Step 3 — Populate in execute_bench_run

In `routes/bench.rs`, track learning counts throughout the run loop:

```rust
let mut learning = BenchLearningSummary {
    strategy: format!("{:?}", overrides.strategy),
    ..Default::default()
};

// During task loop:
// After playbook injection: learning.playbooks_used += injected_count;
// After playbook creation: learning.playbooks_created += 1;
// After anti-pattern creation: learning.anti_patterns_created += 1;
// After knowledge query: learning.knowledge_entries_queried += queried_count;

// After all tasks:
summary.learning = Some(learning);
```

### Step 4 — Include in BenchRunCompleted SSE

Update the `BenchRunCompleted` event to include learning summary:

```rust
ServerEvent::BenchRunCompleted {
    run_id, suite_id, pass_rate, total_cost_usd, total_duration_ms,
    learning: Option<BenchLearningSummary>,  // NEW
}
```

### Step 5 — Persist with run

The `BenchRun.summary` already stores `BenchRunSummary`. Since we added `learning` to it, this is automatically persisted when `save_bench_run` is called.
