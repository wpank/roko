### Step 2 — Handle in useBenchSSE

The `useBenchSSE` hook already parses all SSE messages as JSON and stores them. Since it uses a generic approach (parse JSON, push to events array), it should already handle the new event type without changes. Verify this is the case.

### Step 3 — Process in useBench

In `useBench.ts`, add a case in the SSE event processing switch:

```typescript
case 'BenchLearningEvent': {
  const evt = lastEvent as BenchLearningEvent;
  // Store learning state for the active run
  setLearningState(prev => ({
    ...prev,
    playbooks_created: (prev?.playbooks_created ?? 0) + evt.playbooks_created,
    anti_patterns_created: (prev?.anti_patterns_created ?? 0) + evt.anti_patterns_created,
    total_playbooks: evt.total_playbooks,
    total_anti_patterns: evt.total_anti_patterns,
  }));
  break;
}
```

### Step 4 — Add learning state to useBench return

```typescript
interface LearningState {
  playbooks_created: number;
  anti_patterns_created: number;
  total_playbooks: number;
  total_anti_patterns: number;
}

// In useBench:
const [learningState, setLearningState] = useState<LearningState | null>(null);

// Return it:
return { ...existing, learningState };
```
