## Task

Make roko init emit schema v2 with providers

## Runner Context

## Problem

`roko init` currently emits a v1 config template that lacks the `[providers.*]` and `[models.*]` blocks required by the v2 schema. Users must manually edit the generated `roko.toml` or run `config migrate` before any agent command works with provider routing. This is the first thing every new user hits.

## Architecture Contract

- `roko init` must produce a config that works out of the box with the v2 provider/model resolution pipeline
- No changes to config parsing — only the template output

## Changes Required

- Set `schema_version = 2` in the generated template
- Add `[providers.claude_cli]` section with `kind = "claude_cli"` and `command = "claude"`
- Add `[models.claude-sonnet-4-6]` section mapping to `claude_cli` provider
- If `claude` is not on PATH, write the provider block commented out with a hint
- Remove any bare `[agent]` v1 section from the template
- Keep all other init behavior (directory creation, .gitignore, etc.) unchanged
