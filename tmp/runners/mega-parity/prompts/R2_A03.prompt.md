## Problem

`roko config migrate` currently requires interactive confirmation, making it unusable in CI pipelines, scripts, and automated agent sessions. Agents that detect a v1 config cannot programmatically migrate it.

## Architecture Contract

- `--yes`/`-y` flag provides non-interactive confirmation
- Without the flag, existing interactive behavior is preserved
- Dry-run mode is unaffected

## Changes Required

- Add `--yes` / `-y` flag to the `config migrate` subcommand
- When `-y` is passed: apply migration and print result without prompting
- When `-y` is NOT passed: keep existing interactive confirmation prompt
- `--dry-run` continues to work independently of `-y` (dry-run always prints without applying)
- Print the same output regardless of whether `-y` was used
