## What to Add

Add a private helper function **after** `dispatch_subcommand` (so it doesn't disrupt the existing function structure):

```rust
/// Emit a one-line warning to stderr when the workspace `roko.toml` uses the
/// v1 schema (no `[providers.*]` section).  This is a non-blocking advisory.
fn maybe_warn_v1_config(cli: &Cli) {
    if cli.quiet {
        return;
    }
    let workdir = resolve_workdir(cli);
    let config_path = workdir.join("roko.toml");
    if !config_path.is_file() {
        return;
    }
    let Ok(text) = std::fs::read_to_string(&config_path) else {
        return;
    };
    let Ok(config) = roko_core::config::schema::RokoConfig::from_toml(&text) else {
        return;
    };
    if config.is_stale() {
        eprintln!(
            "warning: roko.toml uses v1 schema — run `roko config migrate` for provider support"
        );
    }
}
```

## Key Facts About `is_stale()`

`RokoConfig::is_stale()` is defined in `crates/roko-core/src/config/schema.rs` (line 196):
```rust
pub const fn is_stale(&self) -> bool {
    self.schema_version < CURRENT_SCHEMA_VERSION
}
```
Where `CURRENT_SCHEMA_VERSION = 2`. A v1 config has `schema_version` absent (defaults to 1 via `default_schema_version()` which returns `CURRENT_SCHEMA_VERSION`, **but** `config_version` defaults to 1 via `default_config_version()`). A config with no providers but parsed cleanly will have `schema_version = 2` (the default) unless the TOML file explicitly sets `schema_version = 1`.

Therefore `is_stale()` alone may not catch all v1 configs. Use a combined check:
```rust
if config.is_stale() || config.providers.is_empty() {
```

This catches both:
- Files that explicitly set `schema_version = 1`
- Files generated by the old init (no `[providers.*]` table at all)

Update `maybe_warn_v1_config` to use:
```rust
if config.is_stale() || config.providers.is_empty() {
    eprintln!(
        "warning: roko.toml uses v1 schema — run `roko config migrate` for provider support"
    );
}
```
