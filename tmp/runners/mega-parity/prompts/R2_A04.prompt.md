## Runner Context

## Problem

Users with v1 configs get confusing errors deep in provider resolution instead of an early, actionable warning. Commands that need providers silently fall back or fail with opaque messages.

## Architecture Contract

- Early warning, not a blocker — commands still execute
- Warning goes to stderr, printed once per invocation
- Only on commands that actually use providers (run, prd, plan)

## Changes Required

- Before `run`, `prd`, and `plan` commands, check `schema_version` from the loaded config
- If v1 or missing providers section: print a one-line warning to stderr (e.g., "warning: roko.toml uses v1 schema — run `roko config migrate` for provider support")
- Do not block execution — the warning is advisory
- Print the warning at most once per invocation (not per subcommand)
- Do not add the check to non-provider commands (status, dashboard, config, etc.)
