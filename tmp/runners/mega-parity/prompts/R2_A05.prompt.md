## Task

Default gates use real programs, not true

## Runner Context

You are working in runner `mega-parity`, batch R2_A05.
This batch is part of Runner 2: execution-contract — Make CLI execution contracts truthful enough that demo scenarios and agent sessions can rely on them.

## Problem

The init template currently emits `program = "true"` for gate commands, which means every gate always passes. This defeats the purpose of gates — users think they have validation but actually have none. Demo scenarios that rely on gates get false confidence.

## Architecture Contract

- Gate verdicts distinguish pass/fail/skipped
- Gates must use real programs that actually validate code

## Changes Required

- For `--profile rust`: emit gates with `program = "cargo"` and appropriate args:
  - compile gate: `cargo check`
  - test gate: `cargo test`
  - lint gate: `cargo clippy`
- For `--profile typescript`: emit gates with real programs:
  - compile gate: `npx tsc --noEmit`
  - test gate: `npm test`
- For unknown/no profile: emit empty gates section with a TOML comment explaining how to configure
- NEVER emit `program = "true"` in any generated config
