## Problem

`roko config validate` currently has two honest-counting gaps:

1. **v1 schema is not a warning.** When `schema_version < 2` or `providers` is empty, the command prints "Phase 2: Schema validation ✓" and reports "0 warnings". It does not emit a schema warning.

2. **Warning count may be wrong in future code.** The `SemanticValidationReport::warning_count()` method (line 969) only counts `self.warnings.len()`. It does not count schema-level warnings separately. The final output line `"Result: N warnings, N errors"` must equal the actual number of `⚠` lines printed.

The `SemanticValidationReport` struct (lines 952-958) and `cmd_validate` function (lines 306-368) are the targets.

## Architecture Contract

- State views agree — if warnings are printed, the count must match
- Schema warnings (v1 schema, missing providers) must increment the warning counter
- Each category counted separately in summary
- v1 schema is a warning, not an error

pub async fn cmd_validate(workdir: &Path) -> Result<()> {

            println!("  ✗ {err}");

        }
    };

    println!("Phase 3: Semantic validation:");
    print_semantic_result(
        "All model providers exist in [providers.*]",
        &report.provider_reference_errors,
    );
    print_semantic_result(
        "Fallback chain models exist",
        &report.fallback_reference_errors,
    );
    print_semantic_result("Tier model keys exist", &report.tier_model_errors);
    print_semantic_result("API key env vars are set", &report.api_key_errors);
    for warning in &report.warnings {
        println!("  ⚠ {warning}");
    }

    println!();
    println!(
        "Result: {} warnings, {} errors",
        report.warning_count(),
        report.error_count()

        Ok(())
    } else {

pub async fn cmd_validate(workdir: &Path) -> Result<()> {
    let paths = resolve_paths(workdir);
    let config_path = validate_config_path(&paths, workdir)?;
    let text = fs::read_to_string(&config_path)

    if let Err(err) = toml::from_str::<toml::Value>(&text) {

    print_phase_status("Phase 1: TOML syntax", true);

    let config = match RokoConfig::from_toml(&text) {

## Important: `is_stale()` Note

`RokoConfig::is_stale()` returns `self.schema_version < CURRENT_SCHEMA_VERSION` (where `CURRENT_SCHEMA_VERSION = 2`). A config with no `[providers.*]` but `schema_version = 2` (e.g., generated by old init) will have `is_stale() == false` but `config.providers.is_empty() == true`. The combined check `config.is_stale() || config.providers.is_empty()` catches both cases.
