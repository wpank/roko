## Problem

Model resolution is scattered across 8+ ad-hoc code paths. There is no single struct or function that encapsulates the resolution precedence. Each command reimplements its own logic, leading to inconsistent behavior (e.g., `--model` silently dropped in some paths).

## Architecture Contract

- One `EffectiveModelSelection` module, one `resolve_effective_model()` function
- CLI --model is a hard override
- Precedence: CLI > task > role > cascade > default > builtin

- Define `EffectiveModelSelection` struct:
  ```rust
  pub struct EffectiveModelSelection {

      pub effective_model_key: String,
  ```


- Implement `resolve_effective_model()` function that:
  - Takes: cli_model (Option), task_hint (Option), role (Option), cascade_router (Option ref), config (ref)
  - Returns: Result<EffectiveModelSelection, Error>
  - Applies precedence: CLI > task > role > cascade > default > builtin
  - Validates that the resolved provider exists and is configured
- Register the module in `crates/roko-cli/src/lib.rs`
- Write tests:
  - CLI override wins over all others
  - Task hint wins when no CLI override
  - Role default used as fallback
  - Cascade router consulted when no explicit selection
  - Config default used when cascade returns nothing
  - Builtin fallback when config has no default
  - CLI override with unavailable provider returns error
  - Unknown model slug returns error
