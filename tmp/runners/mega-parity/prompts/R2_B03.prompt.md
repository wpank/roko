## Problem

## Architecture Contract

- One `EffectiveModelSelection` module, one `resolve_effective_model()` function
- CLI --model is a hard override
- Unavailable provider is an error, not a silent fallback

## Changes Required

- Pass the resolved model key and provider to the dispatch/workflow call
