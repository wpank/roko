## Problem

`roko run` has two ad-hoc model resolution sub-paths that ignore the `--model` flag or reload
config from disk unnecessarily:

1. **v2/WorkflowEngine path** (`run_with_workflow_engine_with_hub` at run.rs:592): The function
   `build_workflow_effect_services(workdir, cli_config)` at run.rs:480 conditionally reloads config
   from disk if `cli_config` is `None` (line 493-496), and computes `display_model` by calling
   `load_layered` a second time at run.rs:619-626 — two redundant disk reads.
2. **`LearningRuntime` double-open**: `append_episode_log` at run.rs:1762 opens a second
   `LearningRuntime` even though one may already be open from prior in the call stack.

This batch fixes the v2 path only. The legacy `run_once` path is handled in B05.
