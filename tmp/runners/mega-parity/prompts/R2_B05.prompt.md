## Problem

`plan run` dispatches agents per-task and each task can have a `model_hint` in the TOML. But the CLI `--model` flag should override all task hints. Currently the override doesn't propagate correctly. Similarly, the legacy `roko run` path has its own resolution that doesn't use `resolve_effective_model()`.

## Architecture Contract

- CLI --model is a hard override — it beats task-level model_hint
- One `resolve_effective_model()` function used everywhere

## Changes Required

- In `orchestrate.rs` per-task dispatch: call `resolve_effective_model()` with cli_model and task.model_hint
- CLI `--model` overrides task `model_hint` (precedence: CLI > task > role > cascade > default)
- In `run.rs` legacy path: call `resolve_effective_model()` with cli_model
- `--model X` on legacy run uses X as the model
