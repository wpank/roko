## Problem

`gate_for_name("shell")` is not recognized as a gate name. The `"shell"` string is not in the match arms of either `rung_for_name` or `gate_for_name` in `gate_service.rs`. Shell gates configured in `roko.toml` with a `"shell"` type are dispatched as unknown gates, causing a failure verdict instead of actually running the configured program.
