## Task

Add skipped/not_wired to gate verdicts

## Problem

Gate verdicts currently only distinguish pass and fail. Stub gates (not yet implemented) and gates skipped due to configuration return "pass" verdicts, inflating the pass rate and giving false confidence. There is no way to distinguish "gate ran and passed" from "gate was skipped."

## Architecture Contract

- Gate verdicts distinguish pass/fail/skipped
- Skipped gates do not count as passes

## Changes Required

- Check if `GateVerdict` derives `Serialize, Deserialize`. Currently it only derives `Debug, Clone`. You must add `serde::Serialize, serde::Deserialize` to the derive list before adding `#[serde(default)]` attributes. The current definition in `crates/roko-core/src/foundation.rs` is:
  ```rust
  #[derive(Debug, Clone)]
  pub struct GateVerdict {
      pub gate_name: String,
      pub passed: bool,
      pub output: String,
      pub duration_ms: u64,
  }
  ```
- Add `serde::Serialize, serde::Deserialize` to the derive macro for `GateVerdict`
- Add to `GateVerdict` struct:
  - `pub skipped: bool` with `#[serde(default)]`
  - `pub skip_reason: Option<String>` with `#[serde(default)]`
- Stub/unimplemented gates set `skipped = true` and `skip_reason = Some("not implemented")`
- Existing real gates remain unchanged (skipped defaults to false via serde)
- In `gate_service.rs`: gates that cannot run (missing program, unknown type) return skipped verdict instead of pass
- Search for any other `GateVerdict` types in the workspace (e.g., in roko-learn) and ensure consistency. Run: `grep -rn 'GateVerdict' crates/ --include='*.rs' | grep -v target/`
