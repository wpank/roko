## Runner Context

## Problem

After C04 adds `skipped` to gate verdicts, the learning system and dashboard still count skipped gates as passes. This inflates pass rates, corrupts adaptive thresholds, and gives misleading efficiency metrics. The summary line says "5/5 passed" when 3 were skipped.

## Architecture Contract

- Gate verdicts distinguish pass/fail/skipped
- Learn paths match write paths
- State views agree

## Changes Required

- In `orchestrate.rs` gate summary:
  - `pass_rate = passed / (passed + failed)` — exclude skipped from both numerator and denominator
  - Summary format: `"X passed, Y failed, Z skipped"`
  - Efficiency events include `gates_skipped` count
- In `runtime_feedback.rs` adaptive thresholds:
  - Only update thresholds from real (non-skipped) gate results
  - Skipped gates do not contribute to EMA
- Dashboard/status displays the three-way breakdown
