## Problem

When a workflow halts (gate failure, agent error, timeout), the CLI process exits with code 0 (success). This means CI pipelines, scripts, and automated agents cannot detect failures by checking the exit code. A halted workflow looks identical to a successful one.

Specifically, `run_with_workflow_engine_with_hub` in
`/Users/will/dev/nunchi/roko/roko/crates/roko-cli/src/run.rs` returns `Ok(())` even
when `result.outcome` is `WorkflowOutcome::Halted { reason }` (lines 664–682). The
caller in `cmd_run` (in
`/Users/will/dev/nunchi/roko/roko/crates/roko-cli/src/commands/util.rs` lines 268–276)
maps `Ok(())` to `EXIT_SUCCESS` and `Err(_)` to `EXIT_AGENT_FAILURE`. So a halt that
doesn't throw an error propagates as exit code 0.

## Architecture Contract

- Workflow failures exit nonzero
- Success exits zero
- Exit codes are a reliable signal for automation
