## Problem

When a workflow halts (gate failure, agent error, timeout), the CLI process exits with code 0 (success). This means CI pipelines, scripts, and automated agents cannot detect failures by checking the exit code. A halted workflow looks identical to a successful one.

## Architecture Contract

- Workflow failures exit nonzero
- Success exits zero
- Exit codes are a reliable signal for automation

## Changes Required

- In `run.rs`: when workflow result is `Halted` or equivalent failure state, exit with code 1
- In `main.rs`: propagate nonzero exit codes from run/plan subcommands
- Success (all gates passed, task completed) exits with code 0
- Partial completion (some tasks done, some failed) exits with code 1
- Print the halt reason to stderr before exiting
