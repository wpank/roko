## Problem

When a plan run gets into a corrupted or confusing state, users have no clean way to start over. They must manually delete state files and hope they got all of them. Stale state from previous runs causes tasks to be skipped or status to be wrong.

## Architecture Contract

- State views agree
- Users can reset to a known-good state

## Changes Required

- Add `--fresh` flag to `plan run` command
- When `--fresh` is passed:
  - Archive the old state file (rename to `executor.json.bak.<timestamp>`)
  - Start with clean state (no completed tasks, no resume data)
- Without `--fresh`: existing behavior (resume from state if present)
- If state appears drifted (state references tasks not in current plan): print hint suggesting `--fresh`
