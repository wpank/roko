## Problem

`roko status`, `roko plan list`, and the internal run state can report different progress for the same plan. If the state file says 1/3 tasks complete, `status` might say "idle" and `plan list` might show all tasks as pending. Users and agents get contradictory information.

## Architecture Contract

- State views agree
- One canonical state, multiple views

## Changes Required

- Identify the single canonical state source (`.roko/state/executor.json`)
- `roko status`: read from canonical state, report task progress matching the state file
- `roko plan list`: read from canonical state, show per-plan completion matching the state file
- If state says 1/3 complete: both commands agree on 1/3
- If no state file exists: both report "no run state found" (not "idle" or "0/0")
- If state references a plan that doesn't exist: report "state references missing plan: <path>"
