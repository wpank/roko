## Problem

`roko learn all` (and other learn subcommands) read from hardcoded paths that may not match where `orchestrate.rs` actually writes learning data. Users run `roko learn all` after a plan run and see "no data" even though data was written — just to a different path.

**Current state after reading learn.rs:**

- `print_learn_efficiency` (line 228) calls `read_project_efficiency_events(workdir)` which reads
  `<workdir>/.roko/learn/efficiency.jsonl`. Orchestrate.rs writes to the same path. **MATCH.**
- `print_learn_episodes` (line 290) calls `read_project_episodes_lossy(workdir)` which checks three
  paths and returns empty if none found; error message is just `"Episodes: none"` with no path shown.
- `print_learn_router` (line 125) reads `workdir.join(".roko/learn/cascade-router.json")`.
  Orchestrate.rs writes to the same path. **MATCH.**
- When `print_learn_efficiency` fails (line 232), it prints `"Efficiency log: empty"` — no path shown.
- When `print_learn_episodes` fails (line 294), it prints `"Episodes: none"` — no path shown.
- When `print_learn_router` finds no file (line 126), it prints `"Cascade router: not initialized"` — no path shown.

**Fix needed:** When a data file is missing, print `"No data at <exact-path>"` (with the actual absolute path). When a file exists, print summary with count.
