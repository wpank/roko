## Problem

When the Claude CLI agent runs, raw JSON stream lines from the subprocess are dumped directly to the terminal. This makes the output unreadable — hundreds of `{"type":"content_block_delta","delta":{"text":"..."}}` lines mixed with actual content. Users and demo scenarios see garbage instead of agent responses.

## Architecture Contract

- CLI output is human-readable by default
- Debug output is available via --verbose or ROKO_DEBUG=1

## Changes Required

- Raw JSON lines from the Claude CLI subprocess go to a debug channel, not stdout/stderr
- `--verbose` flag or `ROKO_DEBUG=1` env var: raw lines go to stderr (for debugging)
- Normal mode: only the final assembled text is printed to stdout
- Tool call/result summaries are printed as brief lines (e.g., "tool: read_file path=/foo.rs")
- Streaming progress (if implemented): show a spinner or brief status, not raw JSON
