## Problem

When the Claude CLI agent runs, raw JSON stream lines from the subprocess are dumped directly to the terminal via `eprintln!`. This makes the output unreadable — hundreds of `{"type":"content_block_delta","delta":{"text":"..."}}` lines mixed with actual content. Users and demo scenarios see garbage instead of agent responses.

Looking at the **current** `stdout_handle` task in
`/Users/will/dev/nunchi/roko/roko/crates/roko-agent/src/claude_cli_agent.rs` lines 455–540:

```rust
// line 496 — raw tool calls are printed to stderr:
eprintln!("[{stdout_name}] tool: {name}");
// line 511 — same for content_block_start:
eprintln!("[{stdout_name}] tool: {name}");
// line 514 — text block start:
eprintln!("[{stdout_name}] generating text...");
// line 533 — result summary:
eprintln!("[{stdout_name}] result received ({summary})");
```

These `eprintln!` calls are ALREADY correctly going to stderr (not stdout), so the
actual raw stream-json lines are **not** being echoed. The problem described in the
issue is that the build command at line 307 includes `--verbose` which causes the
Claude CLI to print verbose status messages to its own stderr, and those get echoed
at `stderr_handle` lines 546–562:

```rust
// lines 553-558: every stderr line from claude subprocess is echoed to our stderr
while let Ok(Some(line)) = lines.next_line().await {
    if !line.trim().is_empty() {
        stderr_activity.store(true, std::sync::atomic::Ordering::Relaxed);
        eprintln!("[{agent_name}] {line}");  // <- echoes ALL claude stderr
    }
    ...
}
```

The `--verbose` flag at line 307:
```rust
cmd.arg("--print")
    .arg("--verbose")           // <- causes extra status lines in claude's stderr
    .arg("--output-format")
    .arg("stream-json")
```
