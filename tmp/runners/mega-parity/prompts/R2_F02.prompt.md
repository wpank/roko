## Task

Server unmatched /api/* returns JSON 404

## Runner Context

You are working in runner `mega-parity`, batch R2_F02.
This batch is part of Runner 2: execution-contract — Make CLI execution contracts truthful enough that demo scenarios and agent sessions can rely on them.

## Problem

Unmatched `/api/*` routes return the SPA HTML fallback instead of a JSON 404. API clients (demo app, agents, curl) that hit a typo'd or non-existent API endpoint get HTML back and parse-fail instead of getting a clean JSON error they can handle programmatically.

## Architecture Contract

- API routes return JSON, always
- SPA catch-all only applies to non-API, non-WS paths

## Changes Required

- Add a fallback handler for `/api/*` that returns JSON 404:
  ```json
  {"error": "not_found", "message": "No route matches /api/..."}
  ```
  with HTTP status 404
- SPA catch-all (`index.html` fallback) only applies to routes that do NOT start with `/api/` or `/ws/`
- `/ws/*` routes are also excluded from SPA catch-all
- Ordering: specific API routes > API 404 fallback > WS routes > SPA catch-all
