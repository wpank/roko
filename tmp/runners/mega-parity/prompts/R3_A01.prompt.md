## Task

## Runner Context

You are working in runner `mega-parity`, batch R3_A01.
This batch is part of Runner 3: agent-session-parity — Make interactive roko and one-shot use a real agent session via existing adapters.

## Problem

Interactive `roko` and one-shot `roko "prompt"` bypass the adapter layer entirely. They call `dispatch_direct::dispatch_claude_cli` which spawns a raw `Command::new("claude")` without system_prompt, tools, MCP, resume, effort, or safety settings. Meanwhile, `ClaudeCliAgent` in roko-agent already handles all of these correctly. We need a session struct that bridges the CLI entrypoints to the existing adapter.

## Architecture Contract

- One `ChatAgentSession` struct owns all session state
- Constructor resolves system prompt, tools, MCP config at creation time
- Fields align 1:1 with `ClaudeCliAgent` builder methods
- Uses `EffectiveModelSelection` from R2_B02 for model identity
- No send_turn implementation in this batch (that is R3_B01)

### Struct definition

```rust
use std::path::PathBuf;
use std::time::Duration;

use roko_core::foundation::ChatMessage;

/// Unified agent session for interactive and one-shot CLI modes.
///
/// Delegates to `ClaudeCliAgent` for Claude CLI turns and to provider
/// adapters for API turns, instead of duplicating command construction.
pub struct ChatAgentSession {
    /// Working directory for the agent.
    pub workdir: PathBuf,
    /// Resolved model identity (provider + slug + source).
    pub model_selection: EffectiveModelSelection,
    /// Reasoning effort level: "low", "medium", "high", "max".
    pub effort: String,
    /// System prompt built by SystemPromptBuilder.
    pub system_prompt: String,
    /// Tool allowlist as comma-separated names for --tools flag.
    pub allowed_tools_csv: String,
    /// Path to MCP config file, if discovered.
    pub mcp_config: Option<PathBuf>,
    /// Session ID from previous turn (for --resume).
    pub session_id: Option<String>,
    /// API message history for non-CLI providers.
    pub api_history: Vec<ChatMessage>,
    /// Shared HTTP client for API providers.
    pub http_client: reqwest::Client,
    /// Path to Claude CLI settings JSON file.
    pub settings_json: Option<PathBuf>,
    /// Per-turn timeout.
    pub timeout: Option<Duration>,
}
```

### Constructor

```rust
impl ChatAgentSession {
    /// Create a new session from CLI config and working directory.
    ///
    /// Resolves system prompt via `SystemPromptBuilder`, tool policy from
    /// safety contracts, and MCP config from discovery paths. Creates one
    /// shared `reqwest::Client`.
    pub fn new(
        config: &crate::config::Config,
        workdir: PathBuf,
        model_selection: EffectiveModelSelection,
    ) -> Result<Self> {
        // 1. Build system prompt via SystemPromptBuilder (placeholder — R3_A02 fills this in)
        let system_prompt = String::new();

        // 2. Resolve tool policy (placeholder — R3_A03 fills this in)
        let allowed_tools_csv = String::new();

        // 3. Discover MCP config (placeholder — R3_A04 fills this in)
        let mcp_config: Option<PathBuf> = None;

        // 4. Read effort from config (String with serde default, not Option)
        let effort = config.agent.effort.clone();

        // 5. Read timeout from config
        let timeout = if config.agent.timeout_ms > 0 {
            Some(Duration::from_millis(config.agent.timeout_ms))
        } else {
            None
        };

        Ok(Self {
            workdir,
            model_selection,
            effort,
            system_prompt,
            allowed_tools_csv,
            mcp_config,
            session_id: None,
            api_history: Vec::new(),
            http_client: reqwest::Client::new(),
            settings_json: None,
            timeout,
        })
    }
}
```
