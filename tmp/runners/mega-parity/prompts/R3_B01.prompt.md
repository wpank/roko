## Problem

`ChatAgentSession` (from R3_A01-A04) has all session state but no way to send a turn. Currently, `chat_inline.rs` calls `dispatch_direct::dispatch_prompt` which spawns a raw `Command::new("claude")` missing system_prompt, tools, MCP, resume, effort, and safety settings. `ClaudeCliAgent` already handles all of these correctly via its builder API and `Agent::run` trait method. We need `send_turn` to build a properly configured `ClaudeCliAgent` per turn and delegate to it.

## Architecture Contract

- `send_turn` builds a fresh `ClaudeCliAgent` each call (because model/effort/system_prompt can change via slash commands between turns)
- Passes ALL fields: workdir, model, effort, system_prompt, tools, mcp, resume (session_id), timeout
- Calls `ClaudeCliAgent`'s `Agent::run` method
- Returns `TurnResult` with text, model, tokens, tool_calls, session_id, duration
- No streaming in this batch (that is R3_C*)
- No API provider support (that is R3_E01)

## Performance Note

R3_G01 (later batch) will make `ReqwestPoster::new()` reuse a shared `reqwest::Client` with connection pooling. In this batch, building a fresh `ClaudeCliAgent` per turn is correct (because model/effort can change between turns). The shared client optimization happens transparently in the HTTP layer, not in this session code.

```rust
impl ChatAgentSession {
    /// Build a `ClaudeCliAgent` with current session state.
    ///
    /// Called from `send_turn`. Extracted as a helper so R3_B05 tests
    /// can inspect the constructed agent without running it.
    pub fn build_agent(&self) -> Result<ClaudeCliAgent> {
        // Resolve the `claude` binary path.
        // TODO(R3_E01): resolve from config.agent.command
        let program = "claude";

        let mut agent = ClaudeCliAgent::new(program, &self.workdir, &self.model)
            .with_effort(&self.effort)
            .with_bare_mode(false); // chat mode, not bare

        // Attach system prompt when present.
        if !self.system_prompt.is_empty() {
            agent = agent.with_system_prompt(&self.system_prompt);
        }

        // Attach tool allowlist when present.
        if !self.allowed_tools_csv.is_empty() {
            agent = agent.with_tools(&self.allowed_tools_csv);
        }

        // Attach MCP config when present.
        if let Some(ref mcp_path) = self.mcp_config {
            agent = agent.with_mcp_config(mcp_path);
        }

        // Pass --resume when we have a session_id from a prior turn.
        if let Some(ref sid) = self.session_id {
            agent = agent.with_resume(sid);
        }

        // Override timeout when configured.
        if let Some(t) = self.timeout {
            agent = agent.with_timeout_ms(t.as_millis() as u64);
        }

        Ok(agent)
    }

    /// Build the input engram for a user prompt.
    pub fn build_engram(&self, prompt: &str) -> Engram {
        // Use the builder API, NOT a struct literal (Engram has many fields
        // with defaults that the builder fills in).
        Engram::builder(Kind::Prompt)
            .body(Body::text(prompt))
            .build()
    }

    /// Send a single turn to the agent and return the result.
    ///
    /// Builds a fresh `ClaudeCliAgent` with the current session state,
    /// delegates via `Agent::run`, extracts usage from the result, and
    /// returns a unified `TurnResult`.
    ///
    /// **Session_id note**: `ClaudeCliAgent::run` does NOT extract `session_id`
    /// from the stream-json `result` event into `AgentResult`. Tags on the
    /// output Engram are only `"agent"` and `"model"`. Session_id must be
    /// captured from the raw stream (R3_C01-C03). For this non-streaming path,
    /// `TurnResult.session_id` is always `None`.
    ///
    /// **Token note**: `ClaudeCliAgent::run` populates only `usage.wall_ms`;
    /// `input_tokens` and `output_tokens` are left at 0. Token counts require
    /// the streaming path (R3_C03 parses the `result` event).
    pub async fn send_turn(&mut self, prompt: &str) -> Result<TurnResult> {
        let start = Instant::now();

        let agent = self.build_agent()?;
        let engram = self.build_engram(prompt);
        let ctx = Context::default();

        // Delegate to the Agent trait method (NOT act, NOT a raw subprocess).
        let result = agent.run(&engram, &ctx).await;

        // Extract text from the output Engram's Body.
        // body.as_text() returns Result<&str>, not Option<&str>.
        // The body is Body::Text when the agent succeeds.
        let text = result.output.body.as_text()
            .ok()
            .map(str::to_string)
            .unwrap_or_default();

        // Session_id: NOT available from ClaudeCliAgent::run.
        // Output Engram only has tags "agent" and "model".
        // result.output.tag("session_id") will always return None here.
        // The streaming path (R3_C01-C03) captures session_id from the
        // raw stream-json "system" and "result" events.
        let new_session_id: Option<String> = None;

        // Extract usage: input_tokens and output_tokens are u32 in Usage.
        // Cast to u64 for TurnResult (wider type for downstream consumers).
        // NOTE: ClaudeCliAgent sets these to 0 — only wall_ms is populated.
        let input_tokens = u64::from(result.usage.input_tokens);
        let output_tokens = u64::from(result.usage.output_tokens);

        Ok(TurnResult {
            text,
            model: self.model.clone(),
            input_tokens,
            output_tokens,
            tool_calls: Vec::new(), // R3_B03 fills this in
            session_id: new_session_id,
            duration: start.elapsed(),
            cancelled: false,
        })
    }
}
```
