## Problem

Claude CLI emits a `session_id` in two events: the `system` init event and the terminal `result` event. This enables multi-turn conversation via `--resume`. Currently, `dispatch_direct.rs::dispatch_claude_cli` extracts the session_id at line 206 (`session_id = event.get("session_id")`), but `ClaudeCliAgent::run` does NOT expose it through `AgentResult` — the output Engram only carries `"agent"` and `"model"` tags.

This batch ensures that:
1. The non-streaming `send_turn` path is correctly documented as "session_id = None" (it cannot capture session_id without the raw stream)
2. A `send_turn_oneshot` variant exists that explicitly disables resume
3. Debug tracing is in place for when session_id IS captured (will be used by R3_C03)

## Architecture Contract

- Non-streaming `send_turn` (R3_B01): `TurnResult.session_id` is always `None` because `ClaudeCliAgent::run` does not surface session_id through `AgentResult`
- Streaming `send_turn_streaming` (R3_C03): captures session_id from the raw `system` event (field: `session_id: String`) and `result` event (field: `session_id: String`)
- `self.session_id` is passed as `--resume` on the next turn via `with_resume()` in `build_agent()`
- `/reset` clears session_id (already done in R3_A05)
- One-shot mode: session_id never set (no `--resume` on any turn)
- Absent session_id: continue without `--resume` (first turn behavior)

### Step 1: Add a helper for updating session_id

Add this function at module level (not inside an impl block):

```rust
/// Update self.session_id and emit a debug trace.
///
/// Called by send_turn_streaming (R3_C03) when it captures a session_id
/// from the raw stream. The non-streaming send_turn path cannot capture
/// session_id (ClaudeCliAgent::run does not surface it through AgentResult).
fn apply_session_id(session_id: &mut Option<String>, new_id: Option<String>) {
    if let Some(ref sid) = new_id {
        tracing::debug!(session_id = %sid, "captured session_id for resume");
        *session_id = new_id;
    }
}
```

### Step 3: Add `send_turn_oneshot`

Add this method to the `impl ChatAgentSession` block:

```rust
/// Send a single turn in one-shot mode.
///
/// One-shot mode explicitly suppresses `--resume` so each call starts
/// a fresh conversation regardless of any stored session_id.
/// The session_id is NOT updated after this call — one-shot turns
/// are completely isolated from each other.
pub async fn send_turn_oneshot(&mut self, prompt: &str) -> Result<TurnResult> {
    // Temporarily clear session_id so build_agent() omits --resume.
    let saved_session_id = self.session_id.take();

    let result = self.send_turn(prompt).await;

    // Restore the previous session_id (one-shot does not accumulate state).
    // If the caller wants to discard state permanently, use /reset instead.
    self.session_id = saved_session_id;

    result
}
```
