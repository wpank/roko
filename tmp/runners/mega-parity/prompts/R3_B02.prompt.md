## Problem

Claude CLI emits a `session_id` in its `Result` event that enables multi-turn conversation via `--resume`. Currently, `dispatch_direct.rs` extracts this ID (line 166) but `chat_inline.rs` never passes it back on the next call, so every turn is a fresh conversation. R3_B01's `send_turn` stubs session_id extraction from `AgentResult::context`, but the actual extraction depends on how `ClaudeCliAgent` populates the result. This batch ensures the session_id lifecycle is correct end-to-end.

## Architecture Contract

- Session ID extracted from `ClaudeCliAgent`'s result (however it surfaces — context map, dedicated field, or parsed from output)
- Stored in `self.session_id` after each successful turn
- Passed as `--resume <session_id>` on next turn via `with_resume`
- `/reset` clears session_id (already done in R3_A05)
- One-shot mode: session_id never set (no `--resume` on single turn)
- Absent session_id: continue without `--resume` (first turn behavior)
