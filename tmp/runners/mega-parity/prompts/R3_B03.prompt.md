## Problem

`TurnResult.tool_calls` (from R3_B01) is always empty. The caller (chat_inline or one-shot) needs to know what tools were used during a turn to display them. Currently `dispatch_direct.rs` captures `ToolOutput` from stream-json `tool` events, but `ClaudeCliAgent` uses `AgentResult` which may surface tool information differently. This batch ensures tool call summaries are captured from the agent result.

## Architecture Contract

- `TurnResult.tool_calls: Vec<ToolCallSummary>` populated after each turn
- `ToolCallSummary` has: name, abbreviated input (first 200 chars), success status
- Extracted from `AgentResult` context or parsed from agent output
- Caller renders tool output (session does not print)
- Do not break stream parsing if tool data is absent

Add helper:

```rust
/// Extract tool call summaries from an AgentResult.
///
/// Checks the result's context map for tool execution records.
/// Falls back to parsing the text output for tool markers.
fn extract_tool_calls(result: &AgentResult) -> Vec<ToolCallSummary> {
    let mut calls = Vec::new();

    // Path 1: Check AgentResult.trace for tool-related Engrams.
    // AgentResult has `output: Engram` and `trace: Vec<Engram>` -- NOT a `context` map.
    // ClaudeCliAgent populates `trace` from stderr. Tool calls may appear there.
    // For structured tool data, parse from the output text (stream-json events).
    // This is a best-effort extraction; the streaming path (R3_C*) is more reliable.
    //
    // Fallback: parse tool events from the raw output if stream-json format.
    let _tools_value: Option<&serde_json::Value> = None; // placeholder
    if let Some(tools_value) = _tools_value {
        if let Some(tools_arr) = tools_value.as_array() {
            for tool in tools_arr {
                let name = tool.get("name")
                    .and_then(|v| v.as_str())
                    .unwrap_or("unknown")
                    .to_string();
                let input = tool.get("input")
                    .and_then(|v| v.as_str())
                    .unwrap_or("");
                let input_abbrev = if input.len() > 200 {
                    format!("{}...", &input[..200])
                } else {
                    input.to_string()
                };
                let success = tool.get("success")
                    .and_then(|v| v.as_bool())
                    .unwrap_or(true);
                calls.push(ToolCallSummary { name, input_abbrev, success });
            }
        }
    }

    // Path 2: Parse from text output if no structured data
    // (Claude CLI tool events appear as {"type":"tool",...} in stream-json)
    // This is best-effort — structured extraction is preferred.

    calls
}
```
