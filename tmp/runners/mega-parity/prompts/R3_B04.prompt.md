## Problem

When the user presses Ctrl-C during a turn, or a timeout expires, the agent subprocess must be killed cleanly. `ClaudeCliAgent` already handles its own internal timeout (default 120s) via `tokio::time::timeout` (line 582 of `claude_cli_agent.rs`). However, the session layer needs its own cancellation wrapper so: (a) the default is configurable per session, (b) Ctrl-C from the user cancels the turn without exiting the chat process, and (c) partial results are returned gracefully.

## Architecture Contract

- `send_turn` wraps `Agent::run` with `tokio::select!` racing against timeout and Ctrl-C
- Session timeout default: 300 seconds (different from ClaudeCliAgent's internal 120s default)
- Ctrl-C during a turn returns `TurnResult { cancelled: true, ... }` with empty text
- Cancelled turn does NOT exit the chat process (session loop continues)
- The `ClaudeCliAgent::run` method handles subprocess kill internally (via `kill_tree`) when its own internal timeout fires; the session layer does NOT need to call `kill_tree` directly
- `TurnResult::cancelled(duration)` constructor exists

### Step 3: Wrap `Agent::run` in `send_turn` with cancellation

Replace the existing `agent.run(&engram, &ctx).await` call in `send_turn` with:

```rust
// Race agent execution against timeout and Ctrl-C.
// ClaudeCliAgent uses kill_on_drop(true) so the subprocess is killed
// when the future is dropped (either by timeout or Ctrl-C cancellation).
let timeout_duration = self.timeout.unwrap_or(Duration::from_secs(300));

let agent_result = tokio::select! {
    // Branch 1: Normal completion.
    result = agent.run(&engram, &ctx) => result,

    // Branch 2: Session-level timeout fires before the agent's internal timeout.
    // ClaudeCliAgent has its own 120s internal timeout; this is the session's
    // configurable outer bound (default 300s).
    _ = tokio::time::sleep(timeout_duration) => {
        tracing::warn!(
            timeout_secs = timeout_duration.as_secs(),
            "turn timed out at session level"
        );
        return Ok(TurnResult::cancelled(start.elapsed()));
    }

    // Branch 3: Ctrl-C pressed by user.
    // The subprocess is killed via kill_on_drop when the agent future is dropped.
    _ = signal::ctrl_c() => {
        tracing::info!("turn cancelled by Ctrl-C");
        return Ok(TurnResult::cancelled(start.elapsed()));
    }
};

// If the agent's INTERNAL timeout fired (120s by default), it returns
// AgentResult { success: false, output: Engram("timed out after 120000 ms") }.
// Treat internal timeout as a failed turn (not cancelled).
if !agent_result.success {
    let error_text = agent_result.output.body.as_text()
        .ok()
        .map(str::to_string)
        .unwrap_or_else(|| "agent failed".to_string());
    return Ok(TurnResult {
        text: error_text,
        model: self.model.clone(),
        input_tokens: 0,
        output_tokens: 0,
        tool_calls: Vec::new(),
        session_id: None,
        duration: start.elapsed(),
        cancelled: false, // failed, not cancelled
    });
}

let result = agent_result;
```

Then continue with the existing text/session_id/token extraction from R3_B01.

    /// Convert an AgentResult into a TurnResult.

    /// Session_id is always None for the non-streaming path (see R3_B02).
