## Problem

`AgentRuntimeEvent::MessageDelta` events from R3_C01's `send_turn_streaming` need to reach the user's terminal immediately, not after the full response. Tool events should be displayed on stderr as status indicators. The session must support a `--no-stream` fallback that collects all text before printing.

## Architecture Contract

- `MessageDelta` events written to stdout immediately (no buffering, immediate flush)
- Tool events written to stderr as `[ToolName] running...` / `[ToolName] done` / `[ToolName] failed`
- `--no-stream` fallback: collect all MessageDelta text, print once at end
- Final newline after response completes
- No TUI rendering (that is chat_inline's job)
- No terminal capability detection (just write bytes)
- Functions are free functions, NOT methods on ChatAgentSession

### Step 2: Add `render_stream_event`

```rust
/// Render an `AgentRuntimeEvent` to stdout/stderr for plain terminal display.
///
/// This is the non-TUI rendering path — used when stdout is a pipe,
/// in one-shot mode, or when `--no-stream` is active.
/// TUI rendering is handled by chat_inline.
///
/// - `MessageDelta` → stdout (immediate flush, no newline)
/// - `ToolCall` → stderr as `[ToolName] running...`
/// - `ToolOutput` → stderr as `[ToolName] done` (success)
/// - `Error` → stderr
/// - Other variants → ignored (consumed internally by session)
pub fn render_stream_event(event: &AgentRuntimeEvent) {
    match event {
        AgentRuntimeEvent::MessageDelta { text } => {
            // Write to stdout immediately without buffering.
            // Do NOT add a newline — deltas are concatenated.
            let stdout = io::stdout();
            let mut handle = stdout.lock();
            let _ = handle.write_all(text.as_bytes());
            let _ = handle.flush();
        }
        AgentRuntimeEvent::ToolCall { name, .. } => {
            eprintln!("[{name}] running...");
        }
        AgentRuntimeEvent::ToolOutput { id, .. } => {
            // We don't have the tool name here — only the id.
            // Use the id as a fallback display name.
            eprintln!("[tool:{id}] done");
        }
        AgentRuntimeEvent::TurnCompleted { is_error, .. } => {
            if *is_error {
                eprintln!("[error] turn completed with error flag");
            }
            // Normal completion — no output (caller handles final newline)
        }
        AgentRuntimeEvent::Error { message } => {
            eprintln!("[error] {message}");
        }
        // SystemInit, TokenUsage, Started, Exited — consumed internally, not rendered
        _ => {}
    }
}

/// Print a final newline after streaming completes.
///
/// Call this after all `MessageDelta` events have been rendered.
/// Without this, the terminal prompt appears on the same line as
/// the last token of the response.
pub fn render_stream_end() {
    println!();
}

/// Render all collected text at once (--no-stream fallback).
///
/// Used when streaming is disabled — collects all MessageDelta text
/// into a single string and prints it with a trailing newline.
pub fn render_collected(text: &str) {
    println!("{text}");
}
```

### Step 3: Wire into `send_turn_streaming`

The `send_turn_streaming` method from R3_C01 already sends events to the `tx` channel.
The caller (chat_inline or one-shot) decides whether to call `render_stream_event`.

However, for the plain terminal path (when no TUI is active), we need a convenience
wrapper that renders inline:

```rust
impl ChatAgentSession {
    /// Send a streaming turn, rendering events to stdout/stderr directly.
    ///
    /// This is the plain terminal streaming path — no TUI, no channel consumer.
    /// Use `send_turn_streaming` with a channel for TUI integration (chat_inline).
    pub async fn send_turn_streaming_inline(&mut self, prompt: &str) -> Result<TurnResult> {
        let (tx, mut rx) = tokio::sync::mpsc::channel::<AgentRuntimeEvent>(256);

        // Spawn a task to render events as they arrive.
        let render_handle = tokio::spawn(async move {
            while let Some(event) = rx.recv().await {
                render_stream_event(&event);
            }
        });

        let result = self.send_turn_streaming(prompt, tx).await;

        // Wait for the render task to finish.
        let _ = render_handle.await;

        // Print final newline after the last text delta.
        render_stream_end();

        result
    }
}
```
