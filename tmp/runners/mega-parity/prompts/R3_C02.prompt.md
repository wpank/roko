## Problem

`StreamEvent::TextDelta` events from R3_C01 need to reach the user's terminal immediately, not after the full response. Tool events should be displayed on stderr as status indicators. The session must support a `--no-stream` fallback that collects all text before printing.

## Architecture Contract

- `TextDelta` events written to stdout immediately (no buffering sentences)
- Tool events written to stderr as `[ToolName] started...` / `[ToolName] done`
- `--no-stream` fallback: collect all deltas, print once at end
- Final newline after response completes
- No TUI rendering (that is chat_inline's job)
- No terminal capability detection (just write bytes)

```rust
use std::io::{self, Write};

/// Render stream events to stdout/stderr.
///
/// TextDelta → stdout (immediate flush)
/// ToolStart/ToolEnd → stderr
/// Result → (consumed internally, not rendered)
/// Error → stderr
pub fn render_stream_event(event: &StreamEvent) {
    match event {
        StreamEvent::TextDelta(text) => {
            let stdout = io::stdout();
            let mut handle = stdout.lock();
            let _ = handle.write_all(text.as_bytes());
            let _ = handle.flush();
        }
        StreamEvent::ToolStart { name } => {
            eprintln!("[{name}] running...");
        }
        StreamEvent::ToolEnd { name, success } => {
            if *success {
                eprintln!("[{name}] done");
            } else {
                eprintln!("[{name}] failed");
            }
        }
        StreamEvent::Error(msg) => {
            eprintln!("[error] {msg}");
        }
        StreamEvent::Result { .. } => {
            // Consumed by session internally, not rendered
        }
    }
}

/// Render all collected text at once (--no-stream fallback).
pub fn render_collected(text: &str) {
    println!("{text}");
}
```
