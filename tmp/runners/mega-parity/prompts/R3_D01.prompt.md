## Task

Route interactive roko through ChatAgentSession

## Runner Context

You are working in runner `mega-parity`, batch R3_D01.
This batch is part of Runner 3: agent-session-parity — Make interactive roko and one-shot use a real agent session via existing adapters.

## Problem

`roko` with no arguments currently calls `unified.rs::cmd_unified_chat` which calls `chat_inline::run_unified_inline(&auth)`. Inside `chat_inline.rs`, user input is dispatched via `dispatch_direct::dispatch_prompt` — a raw subprocess spawn that lacks system prompt, tools, MCP, resume, effort, and safety settings. After R3_A01 through R3_C04, `ChatAgentSession` provides all of these. This batch wires the interactive REPL to use `ChatAgentSession` instead.

## Architecture Contract

- No-args `roko` creates a `ChatAgentSession`
- REPL loop: read input -> check slash commands -> send_turn_streaming -> render
- Session state persists across turns (model, effort, system_prompt, session_id)
- `/exit`, `/quit`, Ctrl-D exits the REPL
- Ctrl-C cancels the current turn but does NOT exit the REPL
- Do NOT remove `dispatch_direct.rs` yet (R3_D03 deprecates it)
- Do NOT change `roko run` behavior
