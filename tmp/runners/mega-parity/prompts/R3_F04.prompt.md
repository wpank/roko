## Problem

ACP pipeline phases run with `--dangerously-skip-permissions` (see runner.rs `run_claude_cli_via_agent` after R3_F01). In production ACP editors, agents send a permission request to the editor before sensitive operations, and the editor shows a dialog. Roko should do the same.

The ACP protocol supports bidirectional communication. The ACP spec defines `session/request_permission` as a server→client request. Our `StdioTransport` already supports this: `send_request()` sends a JSON-RPC request from server to client and waits for the matching response.

Additionally, "Always Allow" decisions should be persisted — if a user always allows file edits in a workspace, future sessions pre-grant that permission.

```rust
/// Action types that require user permission before proceeding.
#[derive(Debug, Clone, PartialEq, Eq, Hash, serde::Serialize, serde::Deserialize)]
#[serde(rename_all = "snake_case")]
pub enum PermissionAction {
    /// Writing or editing a file.
    FileEdit,
    /// Creating a new file.
    FileCreate,
    /// Deleting a file.
    FileDelete,
    /// Running a terminal command.
    TerminalCommand,
    /// Making a network request.
    NetworkRequest,
    /// Running a git operation.
    GitOperation,
}

/// User decision in response to a permission request.
#[derive(Debug, Clone, PartialEq, Serialize, Deserialize)]
#[serde(rename_all = "snake_case")]
pub enum PermissionDecision {
    /// Allow this single occurrence.
    Allow,
    /// Allow this and all future occurrences in this session/workspace.
    AlwaysAllow,
    /// Reject — do not perform the action.
    Reject,
}

/// Parameters for the `session/request_permission` server→client request.
#[derive(Debug, Clone, Serialize, Deserialize)]
#[serde(rename_all = "camelCase")]
pub struct RequestPermissionParams {
    /// The session this request is for.
    pub session_id: String,
    /// Human-readable title for the permission dialog.
    pub title: String,
    /// Detailed description of the action being requested.
    pub detail: String,
    /// The action type being requested.
    pub action: PermissionAction,
}

/// Response from the editor to a `session/request_permission` request.
#[derive(Debug, Clone, Serialize, Deserialize)]
#[serde(rename_all = "camelCase")]
pub struct PermissionResponse {
    /// The user's decision.
    pub decision: PermissionDecision,
}
```

```rust
/// Load workspace-level trust from `.roko/trust/permissions.json`.
pub fn load_workspace_trust(workdir: &std::path::Path) -> std::collections::HashSet<crate::types::PermissionAction> {
    let path = workdir.join(".roko/trust/permissions.json");
    std::fs::read_to_string(&path)
        .ok()
        .and_then(|data| serde_json::from_str(&data).ok())
        .unwrap_or_default()
}

/// Persist workspace-level trust to `.roko/trust/permissions.json`.
pub fn save_workspace_trust(
    workdir: &std::path::Path,
    trust: &std::collections::HashSet<crate::types::PermissionAction>,
) {
    let path = workdir.join(".roko/trust/permissions.json");
    if let Some(parent) = path.parent() {
        let _ = std::fs::create_dir_all(parent);
    }
    if let Ok(data) = serde_json::to_string_pretty(trust) {
        let _ = std::fs::write(&path, data);
    }
}
```

### Step 6: Add `request_permission()` function to `bridge_events.rs`

Add this free function in `crates/roko-acp/src/bridge_events.rs`, before `handle_session_prompt`:

```rust
/// Send a `session/request_permission` request to the editor and wait for the decision.
///
/// Returns `PermissionDecision::Allow` if the action is already pre-granted.
/// Returns `PermissionDecision::Reject` on timeout (5 minutes) or error (SP-1: safe default).
///
/// If the user chooses "AlwaysAllow", the decision is remembered on the session
/// and persisted to `.roko/trust/permissions.json`.
pub async fn request_permission<R, W>(
    transport: &mut StdioTransport<R, W>,
    session: &mut AcpSession,
    workdir: &std::path::Path,
    action: crate::types::PermissionAction,
    title: &str,
    detail: &str,
) -> crate::types::PermissionDecision
where
    R: tokio::io::AsyncRead + Unpin,
    W: tokio::io::AsyncWrite + Unpin,
{
    use crate::types::{PermissionDecision, RequestPermissionParams};

    // 1. Check pre-granted (session or workspace trust).
    if session.is_pre_granted(&action) {
        tracing::debug!(
            session_id = %session.session_id,
            action = ?action,
            "permission pre-granted (always-allow)"
        );
        return PermissionDecision::Allow;
    }

    // 2. Build and send the request to the editor.
    let params = serde_json::to_value(RequestPermissionParams {
        session_id: session.session_id.clone(),
        title: title.to_string(),
        detail: detail.to_string(),
        action: action.clone(),
    })
    .unwrap_or(serde_json::Value::Null);

    let response_future = transport.send_request("session/request_permission", params);

    // 3. Wait up to 5 minutes for the user's decision.
    let response = tokio::time::timeout(
        std::time::Duration::from_secs(300),
        response_future,
    )
    .await;

    match response {
        Ok(Ok(json_response)) => {
            // Parse the response result into PermissionResponse.
            let decision = json_response
                .result
                .as_ref()
                .and_then(|v| serde_json::from_value::<crate::types::PermissionResponse>(v.clone()).ok())
                .map(|r| r.decision)
                .unwrap_or_else(|| {
                    tracing::warn!(
                        session_id = %session.session_id,
                        "permission response could not be parsed; defaulting to Reject (SP-1)"
                    );
                    PermissionDecision::Reject
                });

            // 4. If AlwaysAllow, remember for this session and persist to disk.
            if matches!(decision, PermissionDecision::AlwaysAllow) {
                session.grant_always_allow(action.clone());
                crate::session::AcpSession::save_workspace_trust(workdir, &session.always_allowed);
                tracing::info!(
                    session_id = %session.session_id,
                    action = ?action,
                    "permission permanently granted (always-allow persisted)"
                );
            }

            decision
        }
        Ok(Err(transport_err)) => {
            tracing::warn!(
                session_id = %session.session_id,
                error = %transport_err,
                "permission request transport error; defaulting to Reject (SP-1)"
            );
            PermissionDecision::Reject
        }
        Err(_timeout) => {
            tracing::warn!(
                session_id = %session.session_id,
                "permission request timed out after 5min; defaulting to Reject (SP-1)"
            );
            PermissionDecision::Reject
        }
    }
}
```

- Use `transport.supports_server_to_client_requests()` — this method does NOT exist; `StdioTransport` unconditionally supports `send_request()`

- Store `always_allowed` as a global singleton — it is per-session (`#[serde(skip)]`) with workspace persistence as a separate file
- Ask permission again for already-granted actions — check `session.is_pre_granted()` first
- Panic if `serde_json::to_value()` fails — use `unwrap_or(serde_json::Value::Null)` and log
- Re-implement bidirectional RPC — `StdioTransport::send_request()` + `handle_incoming_response()` already handle it
- Request permission for the pipeline path — the workflow engine manages this at the tool level; only the single-agent path (workflow="none") needs an upfront gate
