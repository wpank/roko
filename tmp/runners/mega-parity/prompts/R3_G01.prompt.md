## Architecture Contract

- One shared `reqwest::Client` per process, with connection pooling enabled
- All `ReqwestPoster::new()` calls reuse the shared client
- Test code can still inject a custom client via `with_client()` or `with_poster()`
- No behavioral change for callers of `HttpPoster` trait
- The function `shared_http_client()` is publicly exported from `crate::provider` for use by other crates (e.g., future `ChatAgentSession` API provider support)

### Step 1: Add `shared_http_client()` to `crates/roko-agent/src/provider/mod.rs`

Add the following after the existing imports block (after line 59), before the static adapter declarations (before line 77):

```rust
use std::sync::LazyLock;
use std::time::Duration;

/// Process-wide shared HTTP client with connection pooling.
///
/// Using a single `reqwest::Client` ensures TCP+TLS connections are reused
/// across all agent instances to the same API endpoint, eliminating redundant
/// handshakes on every new backend construction.
///
/// Pool settings:
/// - Up to 10 idle connections per host
/// - Idle connections closed after 90 seconds
/// - TCP keepalive every 30 seconds
/// - Connect timeout: 10 seconds
static SHARED_HTTP_CLIENT: LazyLock<reqwest::Client> = LazyLock::new(|| {
    reqwest::Client::builder()
        .pool_max_idle_per_host(10)
        .pool_idle_timeout(Duration::from_secs(90))
        .tcp_keepalive(Duration::from_secs(30))
        .connect_timeout(Duration::from_secs(10))
        .build()
        .expect("failed to build shared HTTP client")
});

/// Return the process-wide shared HTTP client.
///
/// The client maintains a connection pool that reuses TCP+TLS connections
/// across requests to the same host. All `ReqwestPoster::new()` calls use
/// this client.
///
/// Future API provider support in `ChatAgentSession` should also use this
/// client — do not create a new `reqwest::Client` per session turn.
#[must_use]
pub fn shared_http_client() -> reqwest::Client {
    SHARED_HTTP_CLIENT.clone()
}
```

**Exact insertion point**: After line 59 (`use tokio::sync::{OwnedSemaphorePermit, Semaphore};`) and before line 61 (the submodule declarations `pub mod anthropic_api;`).

Note: `std::sync::LazyLock` is stable since Rust 1.80. The workspace requires 1.91+, so this is safe.
