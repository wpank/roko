## Task

Pre-compute provider headers at agent construction

## Runner Context

You are working in runner `mega-parity`, batch R3_G03.
This batch is part of the performance optimization group: Provider Infrastructure — eliminate redundant header allocation by pre-computing HTTP headers once at `OpenAiCompatLlmBackend` construction time.

## Problem

`OpenAiCompatLlmBackend::headers()` at `openai_compat_backend.rs:150` allocates a new `Vec<(String, String)>`, clones all header strings, and formats the Authorization header via `format!("Bearer {}", self.api_key)` on every HTTP request. For multi-turn conversations with 5+ requests per agent, this is 5+ redundant allocations of identical data.

The current implementation:
```rust
fn headers(&self) -> Vec<(String, String)> {
    let mut headers = vec![("Content-Type".to_string(), "application/json".to_string())];
    if !self.api_key.is_empty() {
        headers.push((
            "Authorization".to_string(),
            format!("Bearer {}", self.api_key),
        ));
    }
    headers.extend(self.extra_headers.iter().cloned());
    headers
}
```

All inputs (`api_key`, `extra_headers`) are immutable after construction. The headers never change.

## Architecture Contract

- Headers are computed once at construction, not per request
- The `headers()` method returns a reference or cheap clone
- The `with_extra_headers()` builder method still works (it pre-computes during build)
- No behavioral change for callers of `headers()`
