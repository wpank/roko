## Task
Define RepoContextPack struct

## Runner Context
You are working in runner `mega-parity`, batch R4_A01.
This batch is part of Runner 4: plan-grounding — Ground PRD/plan generation in the real repository and reject invalid artifacts.

## Problem
PRD and plan generation prompts receive no structured information about the repository they target. Agents hallucinate crate names, reference nonexistent files, and propose greenfield rewrites of existing code. We need a bounded, serializable struct that carries repository context into generation prompts.

## Architecture Contract
- `RepoContextPack` carries bounded repo context for generation prompts
- Lives in `crates/roko-cli/src/repo_context.rs` (new file)
- Must be serializable to JSON (for sidecar persistence) and to bounded markdown (for prompt injection)
- `to_prompt_section()` output must be <=32000 chars, truncating large workspaces gracefully
- No dependency on code-intel index, no file contents, no generic utilities

## Changes Required

Create `crates/roko-cli/src/repo_context.rs`:

```rust
use std::path::PathBuf;
use serde::{Serialize, Deserialize};

/// Bounded repository context for grounding PRD/plan generation prompts.
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct RepoContextPack {
    /// Workspace root directory (absolute path).
    pub root: PathBuf,
    /// Detected project kind.
    pub project_kind: ProjectKind,
    /// Workspace member names (crate names for Rust, package names for TS/Go).
    pub workspace_members: Vec<String>,
    /// Key files relevant to the feature being generated (max 20).
    pub key_files: Vec<PathBuf>,
    /// Symbol matches from grep-like search (max 30).
    pub matching_symbols: Vec<SymbolHit>,
    /// Related PRD paths found in .roko/prd/ (max 5).
    pub related_prds: Vec<PathBuf>,
    /// Related plan paths found in .roko/plans/ (max 5).
    pub related_plans: Vec<PathBuf>,
    /// Names that already exist and must not be re-created (workspace members + known crates).
    pub do_not_create: Vec<String>,
    /// True when feature keywords match workspace members AND real source exists.
    pub context_root_verified: bool,
}

/// A single symbol match from repository search.
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct SymbolHit {
    /// File path relative to workspace root.
    pub file: PathBuf,
    /// Line number (1-indexed).
    pub line: u32,
    /// Matched line text (trimmed, max 200 chars).
    pub text: String,
}

/// Detected project kind based on manifest files.
#[derive(Debug, Clone, Serialize, Deserialize, PartialEq, Eq)]
pub enum ProjectKind {
    Rust,
    TypeScript,
    Go,
    Python,
    Mixed,
    Unknown,
}

impl RepoContextPack {
    /// Render as a markdown section suitable for prompt injection.
    /// Output is bounded to <=32000 characters, truncating gracefully.
    pub fn to_prompt_section(&self) -> String {
        // Build markdown with:
        // ## Repository Context
        // **Project**: {kind} workspace at {root}
        // **Workspace members** ({count}): {list, truncate if >50}
        // **Key files**: {list}
        // **Symbol matches**: {list}
        // **Related PRDs**: {list}
        // **Related plans**: {list}
        // **Do NOT create these** (they already exist): {list}
        // **Context verification**: {verified/UNVERIFIED WARNING}
        //
        // Truncation strategy:
        // 1. Truncate symbol text to 120 chars
        // 2. Drop symbols past 15
        // 3. Drop key_files past 10
        // 4. Truncate workspace members list with "... and N more"
        // 5. Final hard truncate at 32000 chars with "[truncated]"
        todo!()
    }
}
```
