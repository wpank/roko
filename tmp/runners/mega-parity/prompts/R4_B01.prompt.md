## Problem
The `prd draft new` command generates PRDs without any knowledge of the target repository. The agent prompt receives the user's description and system prompt but nothing about existing crates, files, or workspace structure. This causes generated PRDs to reference nonexistent code, propose redundant crates, and miss integration opportunities.

## Architecture Contract
- `RepoContextPack` carries bounded repo context for generation prompts
- Before agent execution in `prd draft new`, call `build_repo_context()`
- Append `to_prompt_section()` output to the user prompt (not the system prompt)
- Include a visible warning if `context_root_verified == false`
- The context injection is additive — it does not replace existing prompt content

Modify `crates/roko-cli/src/prd.rs` in the `PrdCmd::Draft` / `DraftCmd::New` handler:

```rust
// Before the agent call (find the point where the prompt is assembled):

// 4. Log context verification status

}

```

Add a keyword extraction helper:

```rust
/// Extract feature keywords from PRD slug and optional description.
/// Splits on hyphens, underscores, spaces. Filters short words (<3 chars).
/// Returns up to 10 keywords.
fn extract_keywords_from_slug_and_description(slug: &str, description: &str) -> Vec<String> {
    let mut words: Vec<String> = slug
        .split(|c: char| c == '-' || c == '_' || c.is_whitespace())
        .chain(description.split(|c: char| c == '-' || c == '_' || c.is_whitespace()))
        .filter(|w| w.len() >= 3)
        .map(|w| w.to_lowercase())
        .collect();
    words.sort();
    words.dedup();
    words.truncate(10);
    words
}
```
