## Task
Persist context pack and validation report sidecars

## Runner Context
You are working in runner `mega-parity`, batch R4_B04.
This batch is part of Runner 4: plan-grounding — Ground PRD/plan generation in the real repository and reject invalid artifacts.

## Problem
The `RepoContextPack` and `ArtifactValidationReport` need to be persisted as JSON sidecar files alongside the PRD they describe. This provides an audit trail, enables subsequent plan generation to reference the context, and allows `plan regenerate` to include validation errors in its prompt.

## Architecture Contract
- Sidecar files live next to the PRD: `<slug>.context.json` and `<slug>.validation.json`
- Both are JSON, human-readable, with timestamp
- Writing is non-blocking: failure to write sidecar does not block PRD generation
- These files are informational — no downstream process depends on them for correctness

## Changes Required

Add to `crates/roko-cli/src/prd.rs`:

```rust
/// Persist the RepoContextPack as a JSON sidecar alongside the PRD.
fn persist_context_sidecar(
    prd_dir: &Path,
    slug: &str,
    context: &RepoContextPack,
    keywords: &[String],
) {
    let sidecar_path = prd_dir.join(format!("{}.context.json", slug));

    #[derive(Serialize)]
    struct ContextSidecar<'a> {
        slug: &'a str,
        keywords: &'a [String],
        timestamp: String,
        context: &'a RepoContextPack,
    }

    let sidecar = ContextSidecar {
        slug,
        keywords,
        timestamp: chrono::Utc::now().to_rfc3339(),
        context,
    };

    match serde_json::to_string_pretty(&sidecar) {
        Ok(json) => {
            if let Err(e) = std::fs::write(&sidecar_path, json) {
                eprintln!("WARNING: Failed to write context sidecar {}: {}", sidecar_path.display(), e);
            }
        }
        Err(e) => {
            eprintln!("WARNING: Failed to serialize context sidecar: {}", e);
        }
    }
}

/// Persist the ArtifactValidationReport as a JSON sidecar alongside the PRD.
fn persist_validation_sidecar(
    prd_dir: &Path,
    slug: &str,
    report: &ArtifactValidationReport,
) {
    let sidecar_path = prd_dir.join(format!("{}.validation.json", slug));

    match serde_json::to_string_pretty(&report) {
        Ok(json) => {
            if let Err(e) = std::fs::write(&sidecar_path, json) {
                eprintln!("WARNING: Failed to write validation sidecar {}: {}", sidecar_path.display(), e);
            }
        }
        Err(e) => {
            eprintln!("WARNING: Failed to serialize validation sidecar: {}", e);
        }
    }
}
```

Wire these calls after PRD generation and validation:

```rust
// After validate_prd_grounding() returns the report:
persist_context_sidecar(&prd_drafts_dir, &slug, &repo_context, &feature_keywords);
persist_validation_sidecar(&prd_drafts_dir, &slug, &validation_report);

// Print summary
println!("Context sidecar: {}.context.json", slug);
println!("Validation sidecar: {}.validation.json", slug);
if validation_report.artifact_valid {
    println!("Artifact validation: PASSED ({} warnings)",
        validation_report.issues.iter().filter(|i| i.severity == Severity::Warning).count());
} else {
    println!("Artifact validation: FAILED ({} errors, {} warnings)",
        validation_report.issues.iter().filter(|i| i.severity == Severity::Error).count(),
        validation_report.issues.iter().filter(|i| i.severity == Severity::Warning).count());
}
```
