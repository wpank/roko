## Task
Require role field in generated tasks

## Runner Context
You are working in runner `mega-parity`, batch R4_C01.
This batch is part of Runner 4: plan-grounding — Ground PRD/plan generation in the real repository and reject invalid artifacts.

## Problem
Generated `tasks.toml` files sometimes omit the `role` field, which means tasks execute with a default/generic agent configuration instead of a role-specific system prompt. The plan generator prompt should require `role` per task, and validation should reject tasks with missing roles. This applies to both `plan validate` and pre-execution validation.

## Architecture Contract
- Generated artifacts validated before acceptance
- The plan generator prompt must specify that every `[[task]]` entry needs a `role` field
- `plan validate` catches missing roles as an error
- Pre-execution validation in orchestrate.rs catches missing roles
- Valid roles: known role names from the template system (implementer, reviewer, researcher, etc.)

### 2. Add role validation to plan validate

Modify `crates/roko-cli/src/prd.rs` (or wherever plan validation lives):

```rust
const KNOWN_ROLES: &[&str] = &[
    "implementer", "reviewer", "researcher", "architect",
    "tester", "debugger", "planner", "operator",
];

/// Validate that all tasks in a plan have a role field with a known value.
fn validate_task_roles(tasks: &[Task]) -> Vec<ValidationIssue> {
    let mut issues = Vec::new();
    for task in tasks {
        match &task.role {
            None => {
                issues.push(ValidationIssue {
                    severity: Severity::Error,
                    category: "missing_role".to_string(),
                    message: format!("Task '{}' has no role field", task.id),
                    location: Some(task.id.clone()),
                });
            }
            Some(role) if !KNOWN_ROLES.contains(&role.as_str()) => {
                issues.push(ValidationIssue {
                    severity: Severity::Warning,
                    category: "unknown_role".to_string(),
                    message: format!(
                        "Task '{}' has unknown role '{}' (known: {})",
                        task.id, role, KNOWN_ROLES.join(", ")
                    ),
                    location: Some(task.id.clone()),
                });
            }
            _ => {} // Valid role
        }
    }
    issues
}
```

### 3. Wire into plan validate command

Call `validate_task_roles()` from the `PlanCmd::Validate` handler and from pre-execution checks.
