## Task
Reject greenfield duplicates

## Runner Context
You are working in runner `mega-parity`, batch R4_C04.
This batch is part of Runner 4: plan-grounding — Ground PRD/plan generation in the real repository and reject invalid artifacts.

## Problem
The most damaging generation failure is when an agent produces a plan that treats an existing workspace as greenfield — proposing to "create crate roko-compose" when it already has 5000 lines of code, or claiming "no Rust crates exist yet" in a workspace with 18 crates. These artifacts must be detected and rejected before execution or learning.

## Architecture Contract
- Generated artifacts validated before acceptance
- No positive learning from failed artifacts
- Scan generated plans for "create crate" / "new crate" patterns
- Check proposals against `workspace_members` from `RepoContextPack`
- Existing crate proposed as new: ERROR
- New crate without `allow_new_crates` config: ERROR
- Banned phrases in existing workspace: "greenfield", "no Rust crates exist yet", "starting from scratch"

## Changes Required

Add to `crates/roko-cli/src/prd.rs` (or plan validation module):

```rust
/// Banned phrases that indicate ungrounded generation in an existing workspace.
const GREENFIELD_PHRASES: &[&str] = &[
    "no rust crates exist",
    "no existing crates",
    "starting from scratch",
    "greenfield project",
    "greenfield implementation",
    "new project from scratch",
    "no existing code",
    "empty workspace",
];

/// Validate that a generated plan does not propose greenfield work in an existing workspace.
fn validate_no_greenfield_duplicates(
    tasks: &[Task],
    context: &RepoContextPack,
    plan_content: &str,
) -> Vec<ValidationIssue> {
    let mut issues = Vec::new();

    // Skip all checks if workspace is actually empty
    if context.workspace_members.is_empty() {
        return issues;
    }

    // 1. Check for "create crate" / "new crate" proposals that duplicate existing
    for task in tasks {
        let prompt_lower = task.prompt.to_lowercase();

        for pattern in &["create crate ", "new crate ", "add crate "] {
            if let Some(idx) = prompt_lower.find(pattern) {
                let after = &task.prompt[idx + pattern.len()..];
                let proposed_name = after
                    .split(|c: char| c.is_whitespace() || c == '`' || c == '"' || c == '\'')
                    .next()
                    .unwrap_or("")
                    .trim_matches(|c: char| !c.is_alphanumeric() && c != '-' && c != '_');

                if !proposed_name.is_empty() {
                    if context.workspace_members.iter().any(|m| {
                        m.to_lowercase() == proposed_name.to_lowercase()
                    }) {
                        issues.push(ValidationIssue {
                            severity: Severity::Error,
                            category: "greenfield_duplicate".to_string(),
                            message: format!(
                                "Task '{}' proposes creating crate '{}' which already exists in workspace",
                                task.id, proposed_name
                            ),
                            location: Some(task.id.clone()),
                        });
                    }
                }
            }
        }
    }

    // 2. Check for banned greenfield phrases in the full plan content
    let content_lower = plan_content.to_lowercase();
    for phrase in GREENFIELD_PHRASES {
        if content_lower.contains(phrase) {
            issues.push(ValidationIssue {
                severity: Severity::Error,
                category: "greenfield_claim".to_string(),
                message: format!(
                    "Plan contains greenfield claim '{}' but workspace has {} existing members",
                    phrase, context.workspace_members.len()
                ),
                location: None,
            });
        }
    }

    // 3. Check for new crate proposals without allow_new_crates config
    // (This is a softer check — new crates are sometimes legitimate)
    // TODO: Read allow_new_crates from roko.toml config
    // For now, just warn on any new crate proposal in existing workspace
    for task in tasks {
        let prompt_lower = task.prompt.to_lowercase();
        if (prompt_lower.contains("create crate") || prompt_lower.contains("new crate"))
            && !context.workspace_members.is_empty()
        {
            // Check if the proposed name is NOT in workspace_members (legitimate new crate)
            // but also NOT in do_not_create (might be a directory name)
            // This is a warning, not an error, since new crates can be legitimate
            // The error case (duplicate) is already caught above
        }
    }

    issues
}
```

## Do NOT
- Ban new crate creation in empty workspaces (that's legitimate greenfield)
- Require exact name match (case-insensitive comparison)
- Block in a blank project (checks only apply when workspace_members is non-empty)
- Add NLP or ML-based detection (simple string matching is sufficient)
- Remove the ability to create legitimately new crates

## Evidence
- A dogfood run generated a plan proposing "create crate roko-compose" when roko-compose already had 5000 LOC
- Another run claimed "no Rust crates exist yet" in a workspace with 18 crates
- The `do_not_create` list in RepoContextPack captures both workspace members and directory names
- Empty workspace check prevents false positives for actual greenfield projects
