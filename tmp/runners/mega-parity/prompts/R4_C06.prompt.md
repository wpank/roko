## Task
Separate artifact validation from process status

## Runner Context
You are working in runner `mega-parity`, batch R4_C06.
This batch is part of Runner 4: plan-grounding — Ground PRD/plan generation in the real repository and reject invalid artifacts.

## Problem
Currently, the system conflates "the agent ran successfully" with "the artifact it produced is good." A PRD agent that exits cleanly but produces a greenfield-duplicate plan is recorded as a success, feeds positive signals to the cascade router, and seeds the knowledge store. We need to separate process success from artifact validity and only emit positive learning when BOTH are true.

## Architecture Contract
- `ArtifactValidationReport` separates process success from artifact quality
- Record TWO outcomes: `process_success` (agent ran) + `artifact_valid` (content passes grounding)
- Signal emission only when BOTH are true
- Failed artifact with successful process: `partial_success` status
- No positive signals, router rewards, or knowledge seeds for failed artifacts

### 1. Add artifact validation outcome to PRD flow

Modify `crates/roko-cli/src/prd.rs`:

First, import `ArtifactValidationReport` from its canonical location in `repo_context.rs`
(defined by R4_Z03). **Do NOT define a local copy** of `ArtifactValidationReport` here:

Then add the `GenerationOutcome` wrapper:

```rust
/// Outcome of a generation operation, distinguishing process from artifact.
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct GenerationOutcome {
    /// Whether the agent process completed without error.
    pub process_success: bool,
    /// Whether the generated artifact passes grounding validation.
    pub artifact_valid: bool,
    /// Validation report (if validation ran).
    pub validation_report: Option<ArtifactValidationReport>,
}

impl GenerationOutcome {
    /// True only when both process succeeded AND artifact is valid.
    pub fn fully_successful(&self) -> bool {
        self.process_success && self.artifact_valid
    }

    /// Status string for display and logging.
    pub fn status_label(&self) -> &'static str {
        match (self.process_success, self.artifact_valid) {
            (true, true) => "success",
            (true, false) => "partial_success",
            (false, _) => "failure",
        }
    }
}
```
