## Task
Withhold knowledge seeds on artifact failure

## Runner Context
You are working in runner `mega-parity`, batch R4_D01.
This batch is part of Runner 4: plan-grounding — Ground PRD/plan generation in the real repository and reject invalid artifacts.

## Problem
When a generated artifact fails validation (e.g., proposes duplicate crates, uses banned greenfield phrases), the system currently still stores knowledge seeds from the generation. This poisons the knowledge store with incorrect patterns — future generations that retrieve these seeds will be more likely to repeat the same mistakes. Seeds must be withheld when artifact validation fails.

## Architecture Contract
- No positive learning from failed artifacts
- `KnowledgeSeedRecord::from_successful_episode` already guards on `episode.success`; the new check guards on **artifact validity** separately from process success
- `artifact_valid == false` → skip seed storage with a tracing log
- Cascade router: skip positive observation for failed artifacts
- Do NOT delete existing seeds or add negative seeds
- Episode/attempt always recorded regardless of validity

