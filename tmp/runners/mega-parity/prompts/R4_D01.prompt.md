## Task
Withhold knowledge seeds on artifact failure

## Runner Context
You are working in runner `mega-parity`, batch R4_D01.
This batch is part of Runner 4: plan-grounding — Ground PRD/plan generation in the real repository and reject invalid artifacts.

## Problem
When a generated artifact fails validation (e.g., proposes duplicate crates, uses banned greenfield phrases), the system currently still stores knowledge seeds from the generation. This poisons the knowledge store with incorrect patterns — future generations that retrieve these seeds will be more likely to repeat the same mistakes. Seeds must be withheld when `artifact_valid == false`.

## Architecture Contract
- No positive learning from failed artifacts
- Check `artifact_valid` before writing knowledge seeds
- `artifact_valid == false` → skip seed storage with a log message
- Cascade router: skip positive observation for failed artifacts
- Do NOT delete existing seeds or add negative seeds

## Changes Required

### 1. Gate knowledge seed storage

Modify `crates/roko-cli/src/prd.rs` (or wherever seeds are stored after generation):

```rust
// Find the point where knowledge seeds are stored after PRD/plan generation.
// Add the artifact_valid check:

if outcome.fully_successful() {
    // Existing seed storage logic
    if let Err(e) = store_knowledge_seeds(&slug, &prd_content, &context) {

} else {
```

### 2. Gate in runtime_feedback.rs

Modify `crates/roko-learn/src/runtime_feedback.rs`:

```rust
// In the feedback recording path, add artifact validity check:

/// Record feedback from a generation task, respecting artifact validity.
pub fn record_generation_feedback(
    &mut self,
    task_id: &str,
    model: &str,
    process_success: bool,
    artifact_valid: bool,
) {
    // Always log the attempt for diagnostics
    tracing::debug!(
        task_id = %task_id,
        model = %model,
        process_success = process_success,
        artifact_valid = artifact_valid,
        "Recording generation feedback"
    );

    if process_success && artifact_valid {
        // Full success: record positive feedback, store seeds
        self.record_positive(task_id, model);
    } else if process_success && !artifact_valid {
        // Partial success: process OK but artifact bad
        // Do NOT record positive feedback
        // Do NOT store seeds
        // Do NOT feed cascade router positive observation
        tracing::info!(
            task_id = %task_id,
            model = %model,
            "Skipping positive feedback: artifact validation failed"
        );
    } else {
        // Process failure: record negative feedback as usual
        self.record_negative(task_id, model);
    }
}
```

### 3. Gate cascade router observation

```rust
// In the cascade router observation path:

if process_success && artifact_valid {

    // Do NOT observe negative either — absence of positive is sufficient
    tracing::debug!(
        model = %model,
        "Skipping cascade router positive observation: artifact_valid={}",
        artifact_valid
```
