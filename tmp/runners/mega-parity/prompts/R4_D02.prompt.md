## Task
Withhold cascade router rewards on artifact failure

## Runner Context
You are working in runner `mega-parity`, batch R4_D02.
This batch is part of Runner 4: plan-grounding — Ground PRD/plan generation in the real repository and reject invalid artifacts.

## Problem
The cascade router learns model effectiveness from task outcomes. Currently, any task where the agent process completes and result.success is true receives a positive observation, regardless of whether the generated artifact is valid. This teaches the router to prefer models that run successfully even when they produce garbage output (duplicate crate proposals, greenfield plans in existing workspaces). The router must only receive positive observations when process success AND artifact validity both hold for artifact-producing tasks.

## Architecture Contract
- No positive learning from failed artifacts
- Artifact tasks (PRD generation, plan generation): require `artifact_valid` in task tracker additionally
- Non-artifact tasks (implementation, test, review): `result.success` is sufficient (existing behavior unchanged)
- Do NOT feed negative observations for skipped rewards (silence, not punishment)
- The fix lives in `orchestrate.rs` where `observe_multi_objective` is called — NOT in cascade_router.rs
