## Task
Withhold cascade router rewards on artifact failure

## Runner Context
You are working in runner `mega-parity`, batch R4_D02.
This batch is part of Runner 4: plan-grounding — Ground PRD/plan generation in the real repository and reject invalid artifacts.

## Problem
The cascade router learns model effectiveness from task outcomes. Currently, any task where the agent process completes and gates pass receives a positive observation, regardless of whether the generated artifact is valid. This teaches the router to prefer models that run successfully even when they produce garbage. The router must only receive positive observations when process success AND gate pass AND artifact validity all hold.

## Architecture Contract
- No positive learning from failed artifacts
- `ArtifactValidationReport` separates process success from artifact quality
- Cascade router `observe(positive)` only when: process + gates + artifact all passed
- Non-artifact tasks (implementation, test, review): gate pass is sufficient
- Artifact tasks (PRD generation, plan generation): require `artifact_valid` additionally
- Do NOT feed negative observations for skipped rewards (silence, not punishment)

## Changes Required

### 1. Distinguish artifact tasks from non-artifact tasks

Modify `crates/roko-cli/src/orchestrate.rs`:

```rust
/// Determine if a task produces an artifact that requires validation.
fn is_artifact_producing_task(task: &Task) -> bool {
    // Tasks whose primary output is a document/plan that needs grounding validation
    matches!(
        task.role.as_deref(),
        Some("planner") | Some("architect")
    ) || task.prompt.to_lowercase().contains("generate prd")
      || task.prompt.to_lowercase().contains("generate plan")
      || task.prompt.to_lowercase().contains("create prd")
      || task.prompt.to_lowercase().contains("draft prd")
}
```
