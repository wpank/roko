## The Contract

### `RepoContextPack` — lives in `crates/roko-cli/src/repo_context.rs`

```rust
use std::path::PathBuf;
use serde::{Serialize, Deserialize};

/// Bounded repository context for grounding PRD/plan generation prompts.
/// Serializes to JSON for sidecar persistence.
/// `to_prompt_section()` output bounded to <=32000 chars.
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct RepoContextPack {
    /// Workspace root directory (absolute path).
    pub root: PathBuf,
    /// Detected project kind (from manifest files at root).
    pub project_kind: ProjectKind,
    /// Workspace member names: crate names for Rust, package names for TS/Go.
    /// Max ~200 entries (typical Rust workspace has 18–50).
    pub workspace_members: Vec<String>,
    /// Key files relevant to the feature being generated (max 20).
    /// Paths are relative to workspace root.
    pub key_files: Vec<PathBuf>,
    /// Symbol matches from content grep (max 30).
    pub matching_symbols: Vec<SymbolHit>,
    /// Related PRD paths found in .roko/prd/drafts/ (max 5).
    /// Paths are relative to workspace root.
    pub related_prds: Vec<PathBuf>,
    /// Related plan paths found in plans/ or .roko/plans/ (max 5).
    /// Paths are relative to workspace root.
    pub related_plans: Vec<PathBuf>,
    /// Names that already exist and must not be re-created.
    /// Populated from workspace_members + directory names under crates/, packages/, libs/.
    pub do_not_create: Vec<String>,
    /// True when at least one feature keyword matches a workspace member
    /// AND real source files exist for that member on disk.
    pub context_root_verified: bool,
}

/// A single symbol match from repository search.
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct SymbolHit {
    /// File path relative to workspace root.
    pub file: PathBuf,
    /// Line number (1-indexed).
    pub line: u32,
    /// Matched line text (trimmed, max 200 chars).
    pub text: String,
}

/// Detected project kind based on manifest files at workspace root.
#[derive(Debug, Clone, Serialize, Deserialize, PartialEq, Eq)]
pub enum ProjectKind {
    Rust,
    TypeScript,
    Go,
    Python,
    Mixed,
    Unknown,
}
```

**Size bounds:**
- `workspace_members`: up to 200 entries, each truncated to 50 chars in prompt
- `key_files`: up to 20 paths
- `matching_symbols`: up to 30 entries, each `text` max 200 chars
- `related_prds`, `related_plans`: up to 5 each
- `to_prompt_section()` hard-truncated at 32000 chars

**Construction API:**
```rust
pub async fn build_repo_context(
    workdir: &Path,
    feature_keywords: &[&str],
) -> Result<RepoContextPack>;
```
Time budget: 10 seconds. Returns partial results on timeout. Never errors on missing files.

### `ArtifactValidationReport` — lives in `crates/roko-cli/src/repo_context.rs` (CANONICAL)

**IMPORTANT:** This is the single canonical definition. R4_B03 (`commands/prd.rs`) and
R4_C06 (`prd.rs`) must import from `crate::repo_context::ArtifactValidationReport`,
NOT define their own copies. Ensure `repo_context.rs` makes these types `pub`.

```rust
/// Validation report for a generated PRD or plan artifact.
/// Persisted as a JSON sidecar next to the artifact file.
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct ArtifactValidationReport {
    /// Path of the validated artifact (tasks.toml or <slug>.md).
    pub artifact_path: PathBuf,
    /// Kind of artifact.
    pub artifact_kind: ArtifactKind,
    /// True when the agent process exited 0 (ran without error).
    pub process_success: bool,
    /// True when content passes all ERROR-level grounding checks.
    /// false when any ValidationIssue with severity=Error is present.
    pub artifact_valid: bool,
    /// All detected issues (errors + warnings).
    pub issues: Vec<ValidationIssue>,
    /// UTC timestamp when validation ran.
    pub timestamp: chrono::DateTime<chrono::Utc>,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct ValidationIssue {
    pub severity: Severity,
    pub category: ValidationCategory,
    pub message: String,
    /// Task id or PRD section name where the issue was found. None = whole artifact.
    pub location: Option<String>,
}

/// NOTE: The existing `Severity` enum in `plan_validate.rs` (line 13) has only
/// `Error` and `Warning`. If adding `Info`, it must be added to the existing enum.
/// Otherwise, re-use the existing two-variant `Severity` and map informational
/// messages to `Warning` severity.
#[derive(Debug, Clone, Serialize, Deserialize, PartialEq, Eq, PartialOrd, Ord)]
pub enum Severity {
    Error,    // artifact_valid = false; blocks positive learning signals
    Warning,  // artifact_valid = true; logged but does not block
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub enum ArtifactKind {
    Prd,
    Plan,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub enum ValidationCategory {
    MissingRole,           // task has no role field
    InvalidModelAlias,     // model/model_hint not in workspace
    DuplicateCrate,        // proposed crate already exists
    MissingFile,           // context.read_files path does not exist on disk
    UnverifiedContext,     // context_root_verified == false
    EmptyContent,          // PRD has no substantive body
    MissingFrontmatter,    // PRD lacks YAML frontmatter
    UnknownDependency,     // depends_on references nonexistent task
}
```

**Decision rules for `artifact_valid`:**
- `artifact_valid = true` when: no `Severity::Error` issues present
- `artifact_valid = false` when: any `Severity::Error` issue present
- `process_success = false` does NOT automatically set `artifact_valid = false` — the process may have still written a valid file

**ERROR-level categories (artifact_valid = false):**
- `DuplicateCrate`: proposed crate name matches `do_not_create` list
- `EmptyContent`: PRD body has no substantive content after frontmatter
- `MissingFrontmatter`: PRD lacks `---` frontmatter block

**WARNING-level categories (artifact_valid = true):**
- `MissingRole`: task `role` field is empty (already caught by PLAN_003, but grounding adds context)
- `InvalidModelAlias`: `model_hint` not in known model list
- `MissingFile`: `context.read_files[*].path` does not exist on disk
- `UnverifiedContext`: `context_root_verified == false` in the `RepoContextPack`
- `UnknownDependency`: structural (already caught by PLAN_005)

**Sidecar path convention:**
- For `tasks.toml`: sidecar is `tasks.validation.json` in same directory
- For `<slug>.md`: sidecar is `<slug>.validation.json` in same directory
