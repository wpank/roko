## Problem
The Claude CLI emits a stream-json `result` event containing `total_cost_usd`, `num_turns`, and a `usage` block (input_tokens, output_tokens, cache_creation_input_tokens, cache_read_input_tokens). The current `ClaudeCliAgent::run()` implementation parses stdout into text but never extracts these values into `AgentResult.usage`, so all usage fields remain zero.

## Architecture Contract
- Absent fields = None, NOT zero
- All usage fields are Option types
- Model from the `system` init event (not the result event)
- If no result event is received, leave usage fields at their defaults

## Changes Required

### Step 1: Add a helper method to extract usage from the raw stdout

Add the following private method to `impl ClaudeCliAgent` in `claude_cli_agent.rs`, after the existing `parse_stream_events` method (insert after line 293):

```rust
/// Scan the raw stdout for the terminal `result` event and extract usage data.
///
/// Returns `None` if no `result` event was found in the stream. When found,
/// returns a tuple of:
/// - `Option<f64>` — `total_cost_usd` (None if absent in the event)
/// - `Option<u64>` — `input_tokens` from the usage block (None if block absent)
/// - `Option<u64>` — `output_tokens` from the usage block (None if block absent)
/// - `Option<u64>` — `cache_creation_input_tokens` (None if block absent)
/// - `Option<u64>` — `cache_read_input_tokens` (None if block absent)
fn extract_result_usage(
    stdout: &str,
) -> Option<(
    Option<f64>,  // total_cost_usd
    Option<u64>,  // input_tokens
    Option<u64>,  // output_tokens
    Option<u64>,  // cache_creation_tokens
    Option<u64>,  // cache_read_tokens
)> {
    use crate::provider::claude_cli::{ClaudeResultEvent, ClaudeStreamEvent};

    for line in stdout.lines().map(str::trim).filter(|l| !l.is_empty()) {
        if let Ok(ClaudeStreamEvent::Result(res)) = serde_json::from_str::<ClaudeStreamEvent>(line)
        {
            let cost = res.total_cost_usd; // Option<f64>, None when absent
            let (inp, out, cache_create, cache_read) = match res.usage {
                Some(u) => (
                    Some(u.input_tokens),
                    Some(u.output_tokens),
                    Some(u.cache_creation_input_tokens),
                    Some(u.cache_read_input_tokens),
                ),
                None => (None, None, None, None),
            };
            return Some((cost, inp, out, cache_create, cache_read));
        }
    }
    None
}
```

```rust
// Extract usage from the stream-json result event, if present.
// Absent fields stay None rather than becoming zero.
let (cost_usd, input_tokens, output_tokens, cache_create, cache_read) =
    Self::extract_result_usage(&stdout)
        .map(|(cost, inp, out, cc, cr)| {
            (
                cost.unwrap_or(0.0) as f32,
                inp.unwrap_or(0) as u32,
                out.unwrap_or(0) as u32,
                cc.unwrap_or(0) as u32,
                cr.unwrap_or(0) as u32,
            )
        })
        .unwrap_or((0.0, 0, 0, 0, 0));

AgentResult::ok(output_signal)
    .with_trace(self.stderr_trace(&stderr))
    .with_usage(Usage {
        wall_ms,
        input_tokens,
        output_tokens,
        cache_read_tokens: cache_read,
        cache_create_tokens: cache_create,
        cost_usd,
    })
```

**Explanation of the conversion**: `roko_core::chat_types::Usage` uses `u32`/`f32`. The stream emits `u64`/`f64`. Cast with `as u32` / `as f32`. Values from the Anthropic API never exceed u32 range in practice. If `extract_result_usage` returns `None` (no result event was found, e.g. timeout before result was emitted), all fields default to 0 — the same behavior as before.

```rust
#[test]
fn extract_result_usage_parses_result_event() {
    let stdout = r#"{"type":"assistant","subtype":"message","message":{"content":[{"type":"text","text":"done"}],"usage":null}}
{"type":"result","session_id":"s1","total_cost_usd":0.0042,"num_turns":1,"is_error":false,"usage":{"input_tokens":100,"output_tokens":50,"cache_creation_input_tokens":0,"cache_read_input_tokens":10}}"#;
```
