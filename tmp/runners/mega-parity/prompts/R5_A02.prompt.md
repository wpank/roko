## Problem
There is no canonical type for usage observations. Different sites use ad-hoc fields, zero for unknown, and string placeholders for missing models. A single well-defined type is needed.

## Architecture Contract
- UsageObservation: unknown != zero
- All numeric fields are Option — None means "not reported"
- UsageSource enum tracks provenance
- serde(default) for backward compatibility
- Default impl: all None + Unknown source

## Changes Required
1. Create or extend `crates/roko-agent/src/usage.rs` with:
```rust
#[derive(Debug, Clone, Default, Serialize, Deserialize)]
pub struct UsageObservation {
    pub input_tokens: Option<u64>,
    pub output_tokens: Option<u64>,
    pub cache_creation_tokens: Option<u64>,
    pub cache_read_tokens: Option<u64>,
    pub cost_usd: Option<f64>,
    #[serde(default)]
    pub source: UsageSource,
    pub model: Option<String>,
    pub wall_ms: u64,
}

#[derive(Debug, Clone, Default, Serialize, Deserialize)]
pub enum UsageSource {
    ProviderReported,
    Estimated,
    #[default]
    Unknown,
}
```
2. Add `pub usage: UsageObservation` to `AgentResult` (or update existing usage field)
3. Ensure `#[serde(default)]` on UsageObservation so old JSONL deserializes
4. Export from `roko-agent` crate root
