## Problem
The efficiency logger in orchestrate.rs writes usage data but treats absent values as zero, making it impossible to distinguish "free" from "unknown cost." The logger needs to use UsageObservation and preserve None semantics.

## Architecture Contract
- Some(value) → write the value
- None → write null in JSON
- Preserve existing fields with serde(default)
- One efficiency event per dispatch attempt

## Changes Required
1. In `orchestrate.rs`, find the efficiency event writer
2. Replace ad-hoc usage fields with UsageObservation from AgentResult
3. When writing to `.roko/learn/efficiency.jsonl`:
   - `Some(42)` → `"input_tokens": 42`
   - `None` → `"input_tokens": null`
   - Never `"input_tokens": 0` when the value is unknown
4. Preserve wall_ms (always present)
5. Preserve existing non-usage fields (task_id, timestamp, etc.)
6. Ensure serde(default) so old entries still deserialize
