## Problem
The efficiency logger in `orchestrate.rs` writes `AgentEfficiencyEvent` structs. It sources `input_tokens`, `output_tokens`, and `cost_usd` from `result.usage`, which is currently always `Usage::zero()` because the stream-json result event was never parsed (fixed by R5_A01). After R5_A01, `result.usage` will carry real values. This batch ensures the efficiency event writes those real values rather than always writing zero.

Additionally, the failure efficiency event always writes hardcoded zeros. After R5_A01, failed tasks that did produce partial output still have no result event to parse (since the agent exited non-zero), so their usage zeros are correct — but this should be made explicit.

## Architecture Contract
- `Some(value)` → write the value to the efficiency event field
- When `AgentResult.usage` is non-zero (populated by R5_A01), use those values
- `wall_ms` always present
- One efficiency event per dispatch attempt
