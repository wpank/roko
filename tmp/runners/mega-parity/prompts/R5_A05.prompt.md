## Task
Display "unknown" instead of "$0.00" for null cost

## Runner Context
You are working in runner `mega-parity`, batch R5_A05.
This batch is part of Runner 5: telemetry-learning — Make cost, usage, episodes, learning, and cascade router feedback truthful enough that dashboards show real data and self-improvement actually works.

## Problem
When cost is unknown (None), the UI displays "$0.00" which is misleading — it implies the operation was free rather than unmetered. All display paths need to distinguish None from Some(0.0).

## Architecture Contract
- None → "unknown" or "-" in display
- Some(0.0) → "$0.00" (genuinely free)
- Some(1.42) → "$1.42"
- JSON API: None → null, Some(0.0) → 0.0, Some(1.42) → 1.42

## Changes Required
1. In `status.rs` (CLI status output): display None cost as "unknown" or "-"
2. In `learn.rs` (CLI learn output): display None cost as "unknown" or "-"
3. In serve routes (API responses): return null for None, 0.0 for Some(0.0)
4. Consistent format: CLI uses human-readable, API uses JSON null
5. Apply same rule to token counts: None → "-", Some(0) → "0"
