## Problem
Multiple cost events are emitted for a single dispatch attempt, inflating totals. Gate failures may also generate cost events, double-counting the attempt.

## Architecture Contract
- ONE cost event per dispatch completion
- Gate result is a SEPARATE event type
- No cost duplication for gate failure
- Gate event has no cost_usd field

## Changes Required
1. In `orchestrate.rs`, consolidate to ONE cost event per dispatch completion:
   - Fields: model, provider, usage (UsageObservation), wall_ms, task_id, outcome="dispatch_complete"
   - Emitted once when agent dispatch returns, regardless of gate outcome
2. Gate result is a SEPARATE event:
   - Fields: task_id, gate_outcome, gate_details, attempt_id (links to cost event)
   - No cost_usd field
3. Remove duplicate emission sites identified in R5_B01
4. Ensure gate failure + retry = two cost events (one per attempt), not three+
