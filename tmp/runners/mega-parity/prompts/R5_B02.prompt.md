## Problem
Gate failure + replan emits TWO zero-cost failure events for ONE dispatch attempt:
1. `record_task_failure` (~line 12655) calls `emit_failure_efficiency_event` ← CORRECT
2. `attempt_replan` (~line 11219) also calls `emit_failure_efficiency_event` ← DUPLICATE

Additionally, failure events have `cost_usd: 0.0` with no link to the preceding dispatch cost
event, making it impossible to join cost data with gate outcomes by attempt.

## Architecture Contract
- ONE efficiency event per dispatch attempt
- `attempt_replan` must NOT emit an efficiency event
- All events for the same attempt share `attempt_id`
