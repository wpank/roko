## Problem
The cascade router's `observe_multi_objective` method is already called on success. However:
1. The failure path calls `observe_cascade_router` (which uses reward=0.0) but only when
   `selected_model` is Some — this condition can be None for some failure paths.
2. Artifact validation failures currently send a 0.0 reward (treated same as model failure).
3. The router is persisted at shutdown only (via `save_cascade_router` in the shutdown sequence),
   not after each observation.

## Architecture Contract
- Success + gates passed → `observe_multi_objective(context_vec, model_idx, 1.0, ...)` — already done
- Failed dispatch → `observe(context_vec, model_idx, 0.0)` — partially done
- Artifact failure → no observation (ambiguous signal) — NOT yet enforced
- Persist after observation (not just at shutdown)
