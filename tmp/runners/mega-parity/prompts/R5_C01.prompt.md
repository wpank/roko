## Problem
The cascade router has an observe() method but dispatch outcomes are not fed into it. The router cannot learn which models work for which domains because it never receives feedback.

## Architecture Contract
- Success + gates passed → observe(model, domain, true)
- Failed → observe(model, domain, false)
- Artifact failure → no observation (ambiguous signal)
- task_domain derived from role/category
- Persist after observation

## Changes Required
1. In `orchestrate.rs`, after dispatch + gate evaluation:
   - If dispatch succeeded AND gates passed: `cascade_router.observe(model, domain, true)`
   - If dispatch failed: `cascade_router.observe(model, domain, false)`
   - If artifact validation failed (not a model failure): skip observation
2. Derive `domain` from the task's role or category field
3. Call `cascade_router.persist()` after observation
4. Model comes from EffectiveModelSelection (the actual model used)
