## Task
Exclude failed artifact validation from positive learning

## Runner Context
You are working in runner `mega-parity`, batch R5_C04.

## Problem
The learning system may count stub gate passes and artifact validation failures as positive learning signals. A task that "passes" because gates were stubbed is not evidence of model success.

## Architecture Contract
- Check artifact_valid + real gate pass (not stubs) before positive learning
- Process + real gates + artifact → positive
- Log when positive learning is withheld

if result.success {
    use roko_core::TaskComplexityBand;
    use roko_core::config::schema::RewardWeights;
    use roko_learn::model_router::CONTEXT_DIM;

    // Guard: exclude stub-gate passes and artifact validation failures.
    // NOTE: last_artifact_valid is added by Step 0 above.
    let artifact_valid = self

    let has_stub_gates = self

        let model = self.effective_model();
