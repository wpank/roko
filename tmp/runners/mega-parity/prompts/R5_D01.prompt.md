## Problem
The string "unknown-model" appears in multiple places as a placeholder when the actual model is not known. This looks like a real model name in logs, dashboards, and learning data. Unknown should be represented as None (null), not a magic string.

## Architecture Contract
- Unknown model → None (null in JSON)
- Never a string that looks like a model name when model is unknown
- Model from EffectiveModelSelection when available

## Changes Required
1. Find all "unknown-model" string occurrences in orchestrate.rs
2. Replace each with the resolved model from EffectiveModelSelection
3. If model selection is not available at that point, use None (Option<String>)
4. Update any type that uses String for model to use Option<String> if needed
5. Verify no "unknown-model" strings remain in the codebase

**Note**: This batch depends on R2_B02's `EffectiveModelSelection`. Use `model_selection.effective_model_key` wherever model was previously hardcoded as `"unknown-model"`. When model is genuinely unresolvable at log time, use `None` (serialize as JSON `null`), never a string placeholder.
