## Problem
When a user sends a prompt, roko should check its knowledge store (roko-neuro) for relevant playbooks, past episodes, and domain knowledge BEFORE dispatching to the agent. This serves two purposes:
1. Show the user what roko already knows (as a visible card in the editor)
2. Inject that knowledge into the agent's system prompt for better results

Currently ACP queries nothing — the agent starts from zero every time, even if roko has solved the exact same problem before. The orchestrate.rs reference implementation queries playbooks at dispatch time (line ~14400), but ACP doesn't.

## Architecture Contract
- **TL-5:** Learning reads what execution writes — knowledge store was populated by past episodes/dreams, now dispatch must read it back
- **ACP-3:** No silent learning gaps — if knowledge exists but isn't surfaced, learning is pointless
- **ACP-1:** One dispatch path — knowledge injection happens in the same place as SystemPromptBuilder

### 2. Create knowledge query service

In `bridge_events.rs` (or a new `knowledge.rs` module):

```rust
use roko_neuro::{KnowledgeStore, KnowledgeQueryHit, KnowledgeKind};
use roko_learn::playbooks::{PlaybookStore, QueryContext, Playbook};

/// Query knowledge + playbooks for a prompt. Returns context text for system prompt injection.
/// Never fails — returns empty on any error.
async fn query_dispatch_knowledge(
    prompt: &str,
    workdir: &Path,
    role: &str,
) -> (Vec<KnowledgeQueryHit>, Vec<Playbook>) {
    let mut hits = Vec::new();
    let mut playbooks = Vec::new();

    // 1. Query neuro knowledge store
    let neuro_path = workdir.join(".roko/neuro");
    if neuro_path.exists() {
        match KnowledgeStore::open(&neuro_path) {
            Ok(store) => {
                match store.query_hits(prompt, 5) {
                    Ok(h) => hits = h,
                    Err(e) => tracing::warn!("knowledge query failed: {e}"),
                }
            }
            Err(e) => tracing::debug!("neuro store not available: {e}"),
        }
    }

    // 2. Query playbook store
    let playbook_path = workdir.join(".roko/learn/playbooks");
    if playbook_path.exists() {
        let store = PlaybookStore::new(&playbook_path);
        match store.relevant(prompt, 3).await {
            Ok(pb) => playbooks = pb,
            Err(e) => tracing::warn!("playbook query failed: {e}"),
        }
    }

    (hits, playbooks)
}
```

### 3. Emit knowledge card to editor

```rust
/// Emit a ToolCall card showing what roko knows about this prompt.
async fn emit_knowledge_card(
    hits: &[KnowledgeQueryHit],
    playbooks: &[Playbook],
    event_sender: &mpsc::Sender<CognitiveEvent>,
) {
    if hits.is_empty() && playbooks.is_empty() {
        return; // No card for empty results
    }

    let mut lines = Vec::new();

    if !playbooks.is_empty() {
        lines.push("**Playbooks:**".to_string());
        for pb in playbooks {
            let rate = pb.success_rate()
                .map(|r| format!(" ({:.0}% success)", r * 100.0))
                .unwrap_or_default();
            lines.push(format!("  - {}{}", pb.goal, rate));
        }
    }

    if !hits.is_empty() {
        lines.push("**Knowledge:**".to_string());
        for hit in hits.iter().take(5) {
            let tier_label = match hit.entry.tier {
                KnowledgeTier::Persistent => "P",
                KnowledgeTier::Consolidated => "C",
                KnowledgeTier::Working => "W",
                KnowledgeTier::Transient => "T",
            };
            // Truncate content to first 80 chars
            let summary: String = hit.entry.content.chars().take(80).collect();
            lines.push(format!("  - [{}] {:.2} — {}", tier_label, hit.total_score, summary));
        }
    }

    let card_text = lines.join("\n");
    let total = hits.len() + playbooks.len();

    let tool_call_id = "knowledge-query".to_string();
    let _ = event_sender.send(CognitiveEvent::ToolCallStart {
        tool_call_id: tool_call_id.clone(),
        title: format!("Prior knowledge — {} hit{}", total, if total == 1 { "" } else { "s" }),
        kind: ToolCallKind::Other,
    }).await;

    let _ = event_sender.send(CognitiveEvent::ToolCallComplete {
        tool_call_id,
        status: ToolCallStatus::Completed,
        content: vec![ContentBlock::Text { text: card_text }],
    }).await;
}
```

### 4. Build context text for system prompt injection

```rust
/// Convert knowledge hits + playbooks into text for system prompt layer.
fn build_knowledge_context(hits: &[KnowledgeQueryHit], playbooks: &[Playbook]) -> String {
    if hits.is_empty() && playbooks.is_empty() {
        return String::new();
    }

    let mut parts = Vec::new();

    if !playbooks.is_empty() {
        parts.push("## Relevant playbooks from past tasks:".to_string());
        for pb in playbooks {
            let steps_text = pb.steps.iter()
                .map(|s| format!("  {}. {}", s.index, s.description))
                .collect::<Vec<_>>()
                .join("\n");
            parts.push(format!("### {}\nGoal: {}\n{}", pb.name, pb.goal, steps_text));
        }
    }

    if !hits.is_empty() {
        parts.push("## Relevant knowledge:".to_string());
        for hit in hits.iter().take(5) {
            let kind_label = match hit.entry.kind {
                KnowledgeKind::Heuristic => "heuristic",
                KnowledgeKind::Warning => "warning",
                KnowledgeKind::AntiKnowledge => "anti-pattern",
                KnowledgeKind::StrategyFragment => "strategy",
                KnowledgeKind::CausalLink => "causal",
                KnowledgeKind::Insight => "insight",
            };
            parts.push(format!("- [{}] {}", kind_label, hit.entry.content));
        }
    }

    parts.join("\n\n")
}
```

## Do NOT
- Query knowledge store synchronously — use async (store.query_hits is sync but wrap in spawn_blocking if needed)
- Block dispatch if store query fails — ALWAYS proceed with empty results
- Include full knowledge content in the card — truncate to 80-char summaries
- Return more than 5 knowledge hits or 3 playbooks (noise → signal ratio drops)
- Query on every pipeline micro-step — only on phase-level dispatches (strategist, implementer, reviewer)
- Create a new PlaybookStore instance per query — if possible, store in session/run state
- Emit card when there are zero hits — check emptiness before emitting
- Use `KnowledgeStore::query()` (returns entries) — use `query_hits()` (returns scored results with breakdown)
- Duplicate the orchestrate.rs pattern of building QueryContext with task_def — ACP doesn't have TaskDef; use `relevant()` with raw prompt text instead
