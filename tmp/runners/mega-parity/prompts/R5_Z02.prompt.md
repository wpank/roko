## Problem
There is no single canonical type for usage/cost observations. Different sites use different field names, different defaults (zero vs None), and different source indicators. This causes "$0.00" to appear when cost is unknown, and "unknown-model" strings to propagate through the system.

## Architecture Contract
- UsageObservation type: unknown != zero
- None means "not reported", Some(0) means "reported as zero"
- One cost event per attempt
- Model known before logging

## Changes Required
This is a context-only design batch. No code changes. Depends on R5_Z01 audit.

1. Define exact UsageObservation struct with field types and semantics
2. Define UsageSource enum: ProviderReported, Estimated, Unknown
3. Define migration path from current efficiency.jsonl fields to UsageObservation
4. Define where UsageObservation lives (which crate, which module)
5. Define display rules: None → "unknown"/"-", Some(0) → "$0.00"/"0", Some(n) → formatted value
6. Define JSON serialization rules: None → null, Some(0) → 0, Some(n) → n
7. Define Default impl: all None + UsageSource::Unknown
