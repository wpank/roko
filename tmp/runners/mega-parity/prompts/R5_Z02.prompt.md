## Problem
There is no single canonical type for usage/cost observations. Different sites use different field names, different defaults (zero vs None), and different source indicators. This causes "$0.00" to appear when cost is unknown, and "unknown-model" strings to propagate through the system.

## Architecture Contract
- UsageObservation type: unknown != zero
- None means "not reported", Some(0) means "reported as zero"
- One cost event per attempt
- Model known before logging

### Designed UsageObservation contract

**Placement**: `crates/roko-agent/src/usage.rs`

Rationale: `roko-agent` is the crate that receives stream-json results and constructs `AgentResult`. Adding `UsageObservation` here keeps the full observation (including the cost and model fields that come from the result event) in the same place as `AgentResult`. Downstream crates (`roko-cli`, `roko-learn`) already depend on `roko-agent`.

Do NOT add to `roko-core` — it has no dependency on agent-level streaming details and should stay thin.

**Exact struct definition**:
```rust
/// Observed token and cost metrics from a single agent invocation.
///
/// All numeric fields are `Option<T>`. `None` means the value was not
/// reported by the provider (distinct from `Some(0)` which means the
/// provider reported zero).
///
/// # Serde rules
/// - `None` → serialized as JSON `null` (via `#[serde(skip_serializing_if = "Option::is_none")]`)
/// - `Some(0)` → serialized as JSON `0`
/// - `Some(n)` → serialized as JSON `n`
/// - Old JSONL lines without these fields deserialize to `None` (via `#[serde(default)]`)
#[derive(Debug, Clone, Default, Serialize, Deserialize)]
pub struct UsageObservation {
    /// Prompt / input tokens billed by the provider.
    #[serde(default, skip_serializing_if = "Option::is_none")]
    pub input_tokens: Option<u64>,
    /// Completion / output tokens billed by the provider.
    #[serde(default, skip_serializing_if = "Option::is_none")]
    pub output_tokens: Option<u64>,
    /// Tokens written to the provider-side prompt cache.
    #[serde(default, skip_serializing_if = "Option::is_none")]
    pub cache_creation_tokens: Option<u64>,
    /// Tokens served from the provider-side prompt cache.
    #[serde(default, skip_serializing_if = "Option::is_none")]
    pub cache_read_tokens: Option<u64>,
    /// Actual USD cost after cache discounts.
    #[serde(default, skip_serializing_if = "Option::is_none")]
    pub cost_usd: Option<f64>,
    /// Provenance of the observation.
    #[serde(default)]
    pub source: UsageSource,
    /// Model that produced these tokens, if known.
    #[serde(default, skip_serializing_if = "Option::is_none")]
    pub model: Option<String>,
    /// Wall-clock duration in milliseconds. Always measurable (we control the timer).
    pub wall_ms: u64,
}
```

**UsageSource enum**:
```rust
/// Indicates how the usage data was obtained.
#[derive(Debug, Clone, Default, Serialize, Deserialize, PartialEq, Eq)]
#[serde(rename_all = "snake_case")]
pub enum UsageSource {
    /// Provider reported usage directly in the stream-json result event.
    ProviderReported,
    /// Usage was estimated from model price tables (no provider report).
    Estimated,
    /// Source is unknown; treat all fields as potentially absent.
    #[default]
    Unknown,
}
```

**Default impl**: Derived via `#[derive(Default)]`. Result:
```
UsageObservation {
    input_tokens: None,
    output_tokens: None,
    cache_creation_tokens: None,
    cache_read_tokens: None,
    cost_usd: None,
    source: UsageSource::Unknown,
    model: None,
    wall_ms: 0,
}
```

**`wall_ms` is non-Option**: We always measure wall time with `Instant::now()`. `Some(wall_ms)` is therefore always available; making it `Option` would add noise with no signal.

### Display rules

| Field state | CLI display | JSON API |
|-------------|------------|---------|
| `None` | `-` or `unknown` | `null` |
| `Some(0)` or `Some(0.0)` | `0` / `$0.00` | `0` / `0.0` |
| `Some(n)` | formatted value | `n` |

Examples:
```
cost_usd: None        → CLI: "unknown"   JSON: null
cost_usd: Some(0.0)   → CLI: "$0.00"    JSON: 0.0
cost_usd: Some(1.42)  → CLI: "$1.42"    JSON: 1.42

input_tokens: None    → CLI: "-"        JSON: null
input_tokens: Some(0) → CLI: "0"        JSON: 0
input_tokens: Some(42)→ CLI: "42"       JSON: 42
```

### How UsageObservation attaches to AgentResult

Add a new field `pub usage_obs: Option<UsageObservation>` alongside the existing `pub usage: Usage` in `AgentResult`. Use `#[serde(default)]` so existing serialized data continues to deserialize.

Do NOT remove or rename `pub usage: Usage` — it is referenced in many call sites.

### Migration from efficiency.jsonl

`AgentEfficiencyEvent` (in `roko-learn/src/efficiency.rs`) uses non-Option `f64`/`u64`. These fields continue to exist unchanged for backward compatibility. When R5_A03 threads `UsageObservation` through the efficiency logger:
- `Some(v)` → write `v` to the non-Option field
- `None` → leave the field at its existing default (0 / 0.0)

This is the minimum-disruption path. The field semantics on disk do NOT change for efficiency.jsonl because `AgentEfficiencyEvent` is not being modified in R5_A02.

Future work (out of scope for this runner): Add `Option<f64> cost_usd_obs` alongside `cost_usd: f64` in `AgentEfficiencyEvent` with the proper null serialization.

### Backward compatibility for existing JSONL files

All `UsageObservation` fields except `wall_ms` use `#[serde(default)]`, so any old JSONL line that was written before this type existed will deserialize with all Option fields as `None`. This is correct — they are genuinely unknown.
