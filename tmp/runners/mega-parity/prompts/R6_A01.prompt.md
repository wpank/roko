## Problem
The default bind is already `127.0.0.1` (safe -- see `default_bind()` in `crates/roko-core/src/config/serve.rs`). However: (1) the `PORT` env var override in `crates/roko-serve/src/lib.rs` (`start_background()`, around line 226) forces `0.0.0.0:{port}` without checking auth -- this is the Railway/cloud deploy path and bypasses the safe default, (2) there is no runtime guard preventing a non-loopback bind when auth is disabled, (3) there is no `--unsafe-public` escape hatch. Focus changes on adding the guard logic, not changing the default.

## Architecture Contract
- Default is safe
- 127.0.0.1:6677 by default
- Public bind requires auth_enabled OR --unsafe-public
- Loopback always works without auth

## Changes Required
1. In serve config (`config/serve.rs` or equivalent): default bind address is `127.0.0.1:6677`
2. In serve startup (`lib.rs` or `runtime.rs`):
   - If bind address is loopback (127.0.0.1 or ::1): allow without auth
   - If bind address is non-loopback AND auth not enabled: error with message
   - Error message: "Non-local bind requires `auth_enabled = true` in config or `--unsafe-public` flag"
3. Loopback detection: check if resolved address is loopback
4. Port remains configurable
