## Task
Require explicit flag for public bind

## Runner Context
You are working in runner `mega-parity`, batch R6_A02.
This batch is part of Runner 6: security-posture — Make server, terminal, CORS, and share behavior safe enough for real use.

## Problem
Users can accidentally expose the server to the network by configuring a non-loopback bind address. An explicit opt-in flag prevents accidental exposure.

## Architecture Contract
- Non-loopback + no auth → requires --unsafe-public
- Flag name includes "unsafe"
- Print warning when used
- Config alternative: acknowledge_public_risk

## Changes Required
1. In serve command handler: add `--unsafe-public` CLI flag
2. When non-loopback bind + no auth:
   - Without `--unsafe-public`: error with message explaining options
   - With `--unsafe-public`: allow but print warning to stderr
3. Config alternative: `serve.acknowledge_public_risk = true` in roko.toml
4. Warning message: "WARNING: Server bound to {addr} without authentication. All routes are accessible to the network."
5. The flag is not required if auth is enabled
