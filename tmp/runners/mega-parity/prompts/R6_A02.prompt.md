## Task
Require explicit flag for public bind

## Runner Context
You are working in runner `mega-parity`, batch R6_A02.
This batch is part of Runner 6: security-posture — Make server, terminal, CORS, and share behavior safe enough for real use.

## Problem
Users can accidentally expose the server to the network by configuring a non-loopback bind address (or via `PORT` env var). The guard added in R6_A01 rejects non-loopback+no-auth combinations with an error. This batch adds the `--unsafe-public` CLI flag that allows proceeding with a warning, and wires `acknowledge_public_risk` config as the alternative.

Depends on R6_A01 (which adds `acknowledge_public_risk` to `ServeConfig`).

## Architecture Contract
- Non-loopback + no auth → requires --unsafe-public
- Flag name includes "unsafe"
- Print warning when used
- Config alternative: acknowledge_public_risk
