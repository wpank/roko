## Task
Disable background serve for no-args roko

## Runner Context
You are working in runner `mega-parity`, batch R6_A03.
This batch is part of Runner 6: security-posture — Make server, terminal, CORS, and share behavior safe enough for real use.

## Problem
Running `roko` with no arguments may auto-start the HTTP server in the background, silently exposing routes. Background serve should be opt-in.

## Architecture Contract
- Default is safe
- Background serve gated behind config
- Explicit `roko serve` always works

## Changes Required
1. In `unified.rs` (or wherever no-args behavior is handled):
   - Gate background serve behind `config.serve.auto_start` (default: false)
   - If auto_start is false: do not start background server
   - Print tip: "Tip: run `roko serve` to start the HTTP control plane"
2. Explicit `roko serve` command always works regardless of auto_start
3. Depends on R2_D01 (unified entry point) — if that doesn't exist yet, gate in the current no-args handler
