## Task
Disable background serve for no-args roko

## Runner Context
You are working in runner `mega-parity`, batch R6_A03.
This batch is part of Runner 6: security-posture — Make server, terminal, CORS, and share behavior safe enough for real use.

## Problem
Running `roko` with no arguments auto-starts the HTTP server in the background, silently
exposing ~85 routes (including terminal, before R6_B01 fixes it). Background serve should be
opt-in via config: `serve.auto_start = true`. The default must be `false` (safe).

## Architecture Contract
- Default is safe: no background server unless explicitly enabled
- Background serve gated behind `serve.auto_start` config (default: false)
- Explicit `roko serve` command always works regardless of `auto_start`
- `--no-serve` flag still suppresses when `auto_start = true`
- Tip message printed to stderr when serve is skipped (unless `--quiet`)

```rust
/// Read `serve.auto_start` from `roko.toml` (or defaults if absent).
///
/// Returns `false` when the file is missing or unparseable — safe default.
fn load_auto_start_config(workdir: &std::path::Path) -> bool {
    let path = workdir.join("roko.toml");
    if !path.exists() {
        return false;
    }
    let Ok(text) = std::fs::read_to_string(&path) else {
        return false;
    };
    let Ok(config) = toml::from_str::<roko_core::config::schema::RokoConfig>(&text) else {
        return false;
    };
    config.serve.auto_start
}
```
