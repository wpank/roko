## Task
Gate terminal routes behind auth/flag

## Runner Context
You are working in runner `mega-parity`, batch R6_B01.
This batch is part of Runner 6: security-posture — Make server, terminal, CORS, and share behavior safe enough for real use.

## Problem
Terminal routes allow arbitrary command execution on the server. They should be disabled by default and require explicit opt-in, with additional auth requirements for non-local access.

## Architecture Contract
- Terminal requires auth
- Disabled by default
- Non-loopback requires auth even when enabled
- Loopback + enabled → allow

## Changes Required
1. In `routes/mod.rs`: terminal routes only registered if terminal_enabled=true
2. Configuration: `serve.terminal_enabled` (default: false) or `--enable-terminal` flag
3. When disabled: terminal routes return 403 JSON: `{"error": "Terminal disabled", "hint": "Set serve.terminal_enabled=true or use --enable-terminal"}`
4. When enabled + non-loopback: require auth (from R6_B02)
5. When enabled + loopback: allow without auth
