### Step 4 — Add `terminal_disabled_routes()` to `routes/mod.rs`

Add this function in `routes/mod.rs` (can go after the `top_level_health` function, around line 160):

```rust
/// Returns stub routes that respond with 403 when terminal is disabled.
fn terminal_disabled_routes() -> axum::Router<Arc<AppState>> {
    use axum::http::StatusCode;
    use axum::routing::{delete, get, post};

    let handler = || async {
        (
            StatusCode::FORBIDDEN,
            axum::Json(serde_json::json!({
                "error": "Terminal disabled",
                "hint": "Set serve.terminal_enabled=true in roko.toml or use --enable-terminal"
            })),
        )
    };

    axum::Router::new()
        .route("/api/terminal/sessions", post(handler))
        .route("/api/terminal/sessions", get(handler))
        .route("/api/terminal/sessions/{id}", delete(handler))
        .route("/api/terminal/sessions/{id}/input", post(handler))
        .route("/ws/terminal/{id}", get(handler))
}
```

Note: The handler closure must be a `Fn() -> impl Future` that axum can use. Since all terminal routes need the same 403 body, a single handler works for all methods/paths.

Axum requires that handlers used with different routing methods have compatible types. Use a named async function if the closure causes type issues:

```rust
async fn terminal_disabled() -> impl axum::response::IntoResponse {
    (
        axum::http::StatusCode::FORBIDDEN,
        axum::Json(serde_json::json!({
            "error": "Terminal disabled",
            "hint": "Set serve.terminal_enabled=true in roko.toml or use --enable-terminal"
        })),
    )
}

fn terminal_disabled_routes() -> axum::Router<Arc<AppState>> {
    use axum::routing::{delete, get, post};
    axum::Router::new()
        .route("/api/terminal/sessions", post(terminal_disabled))
        .route("/api/terminal/sessions", get(terminal_disabled))
        .route("/api/terminal/sessions/{id}", delete(terminal_disabled))
        .route("/api/terminal/sessions/{id}/input", post(terminal_disabled))
        .route("/ws/terminal/{id}", get(terminal_disabled))
}
```
