## Task
Add bearer token auth for non-local access

## Runner Context
You are working in runner `mega-parity`, batch R6_B02.
This batch is part of Runner 6: security-posture — Make server, terminal, CORS, and share behavior safe enough for real use.

## Problem
Non-local access to the server has no authentication. A simple bearer token mechanism provides basic access control without complex auth infrastructure.

## Architecture Contract
- Simple bearer token, not OAuth/sessions/cookies
- Token generated and stored in ~/.roko/serve-token
- Loopback always allowed without token
- /health always allowed without token

## Changes Required
1. Token generation: on first serve with auth_enabled, generate random token → `~/.roko/serve-token`
2. Print token on startup: "Auth token: {token} (stored at ~/.roko/serve-token)"
3. Middleware in `middleware.rs`:
   - Non-loopback + auth_enabled → check `Authorization: Bearer {token}` header
   - Invalid/missing token → 401 JSON: `{"error": "Unauthorized", "hint": "Provide Bearer token from ~/.roko/serve-token"}`
   - Loopback → always allowed (no token check)
   - `/health` → always allowed (no token check)
4. Config: `serve.auth_enabled` (default: false, required for non-loopback per R6_A01)
