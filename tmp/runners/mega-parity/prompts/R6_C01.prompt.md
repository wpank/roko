## Task
Restrict CORS to local origins by default

## Runner Context
You are working in runner `mega-parity`, batch R6_C01.
This batch is part of Runner 6: security-posture — Make server, terminal, CORS, and share behavior safe enough for real use.

## Problem
CORS may allow all origins (*), enabling any website to make requests to the local server. Default should restrict to local origins only.

## Architecture Contract
- Default: localhost:*, 127.0.0.1:*
- Config cors_origins for custom
- unsafe-public without cors config → warn + allow *
- Local dev must not break

## Changes Required
1. In middleware: default CORS allowed origins = `localhost:*` and `127.0.0.1:*` (any port)
2. Config: `serve.cors_origins` list for custom origins
3. When cors_origins configured: use those instead of defaults
4. When unsafe-public flag + no cors_origins: warn and allow * (backward compat)
5. When unsafe-public + cors_origins: use configured origins
6. /health preflight always allowed
