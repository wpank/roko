Share expiration

## Runner Context

## Problem
Shared runs persist indefinitely. Old shares with potentially sensitive data should expire after a reasonable period.

## Architecture Contract
- Default TTL: 7 days (configurable)
- Expired → 410 Gone
- --no-expire for permanent
- Backward compatible with existing shares

## Changes Required
1. In `shared_runs.rs` and/or `share.rs`:
   - Add `expires_at: Option<DateTime<Utc>>` to share metadata
   - Default: created_at + 7 days
   - Config: `serve.share_ttl_days` (default: 7)
2. When accessing expired share: return 410 Gone with JSON body
3. `--no-expire` flag: sets expires_at to None (permanent)
4. Existing shares without expires_at: treat as non-expiring (backward compat)
5. Optional: periodic cleanup of expired shares (lazy deletion on access is sufficient)
