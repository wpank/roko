## Runner Context
You are working in runner `mega-parity`, batch R6_Z01.

## Problem
The HTTP server binds to addresses, exposes terminal access, serves shared runs, and allows CORS without a documented security model. Before fixing anything, we need a complete map of the current surface.

## Architecture Contract
- Default is safe
- Terminal requires auth
- No wildcards for public
- Explicit over implicit

## Changes Required
This is a context-only audit batch. No code changes.

1. Read `routes/mod.rs` and catalog every route with:
   - Path, method, handler
   - Security classification: public, auth-required, localhost-only, terminal
   - Whether it reads/writes data
2. Read `terminal.rs` and document:
   - How terminal sessions are created/managed
   - What access controls exist
   - Lifecycle (start, stop, cleanup)
3. Read `config/serve.rs` (or equivalent) and document:
   - Bind address defaults
   - Auth configuration
   - CORS configuration
4. Read `middleware.rs` and document:
   - What middleware exists
   - CORS headers set
   - Auth checking
5. Read `share.rs` and `shared_runs.rs` and document:
   - What data is shared
   - What scrubbing exists
   - Expiration behavior
6. Check for background serve (auto-start on other commands)
