### Area 4 — CORS behavior

File: `crates/roko-serve/src/routes/middleware.rs`

**`cors_layer` function** (lines 426–437):
```rust
pub fn cors_layer(cors_origins: &[String]) -> CorsLayer {
    if cors_origins.is_empty() {
        CorsLayer::permissive()     // ← wildcard * when no origins configured
    } else {
        let allowed: Vec<axum::http::HeaderValue> =
            cors_origins.iter().filter_map(|o| o.parse().ok()).collect();
        CorsLayer::new()
            .allow_origin(allowed)
            .allow_methods(Any)
            .allow_headers(Any)
    }
}
```

**Call site** in `build_router` (`routes/mod.rs` line 69):
```rust
let cors = middleware::cors_layer(&self.config.roko_config.server.cors_origins);
```

**`server.cors_origins`** in `ServerConfig` (`serve.rs` line 163):
```rust
pub cors_origins: Vec<String>,  // default: empty Vec (line 183)
```

**Result:** By default, `cors_origins` is empty → `CorsLayer::permissive()` → `Access-Control-Allow-Origin: *`. Any website can make cross-origin requests to the local server.
