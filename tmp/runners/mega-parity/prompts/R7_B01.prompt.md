## Task
Rich tool call display in terminal

## Runner Context
You are working in runner `mega-parity`, batch R7_B01.
This batch is part of Runner 7: mori-polish — Complete remaining Mori-like UX polish after core contracts stable.

## Problem
Tool calls during chat sessions are either invisible or shown as raw JSON. Users need concise visual feedback when tools are invoked and complete.

## Architecture Contract
- Polish on top of contracts
- Truth before appearance
- Terminal capability detection

## Implementation Note
`chat_session.rs` is created by R3_A01 and extended through R3's batches. This batch adds tool rendering on top of the existing `ToolCallSummary` type from R3_B03. Check the `TurnResult` struct for the available tool call data.

## Changes Required
1. In `chat_session.rs` or new `tool_render.rs`:
   - Tool start: print line like `> [Read] src/main.rs`
   - Tool end success: print line like `< [Read] 47 lines`
   - Tool end error: print line like `x [Read] file not found`
2. Use Unicode indicators when terminal supports it:
   - Start: `▸` (or `>` fallback)
   - Success: `✓` (or `+` fallback)
   - Error: `✗` (or `x` fallback)
3. Colors when supported: tool name in cyan, success in green, error in red
4. Depends on R3_B03 (tool dispatch) and R7_A04 (slash commands complete)
