## Task

## Runner Context

## Problem
Users have no visibility into cost and token usage during interactive chat. Each turn should show a brief summary so users can monitor spending.

## Architecture Contract
- Truth before appearance
- Unknown = "unknown", not "$0.00"
- Suppressible with --quiet

## Changes Required
1. In `chat_session.rs`: after each assistant turn, print to stderr:
   - Format: `[model via provider | N tokens | $X.XX | Xs]`
   - Example: `[claude-sonnet-4-20250514 via claude-cli | 1,247 tokens | $0.03 | 2.1s]`
2. Handle unknown usage:
   - Unknown cost: `[model via provider | usage: unknown | 2.1s]`
   - Unknown everything: `[usage: unknown | 2.1s]`
3. Zero cost: `[model via provider | 0 tokens | free | 0.3s]`
4. Configurable: `chat.show_usage` (default: true)
5. Suppressible: `--quiet` flag hides it
6. Depends on R7_B01 (tool display complete)
