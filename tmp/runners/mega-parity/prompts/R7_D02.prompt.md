## Problem
The direct-provider chat path added in R7_D01 calls `agent.run(&engram, &ctx).await` which blocks
until the full response is ready. This makes the chat appear frozen during generation. The adapters
support streaming via `agent.supports_streaming()` and returning incremental output, but the REPL
loop in R7_D01 ignores this.

## Current State

### `Agent` trait (crates/roko-agent/src/agent.rs, approximately line 148)
```rust
pub trait Agent: Send + Sync {
    async fn run(&self, input: &Engram, ctx: &Context) -> AgentResult;
    fn name(&self) -> &str;
    fn backend_id(&self) -> &'static str { "unknown" }
    fn supports_streaming(&self) -> bool { false }
}
```

The `supports_streaming()` method is available on every `Box<dyn Agent>`. Adapters that use
streaming internally override it to return `true`. The `run()` method itself still returns
a complete `AgentResult` — streaming happens inside the adapter.

The fix is to check `agent.supports_streaming()` before each turn and, if true, print a progressive

## Codebase Reality Check

### Agent trait streaming
File: `/Users/will/dev/nunchi/roko/roko/crates/roko-agent/src/agent.rs` (line 148):
```rust
pub trait Agent: Send + Sync {
    async fn run(&self, input: &Engram, ctx: &Context) -> AgentResult;
    fn name(&self) -> &str;
    fn backend_id(&self) -> &'static str { "unknown" }
    fn supports_streaming(&self) -> bool { false }
}
```

Key fact: The `Agent::run` method is the unified entry point. Adapters that "support streaming"
(e.g. `ClaudeAgent::supports_streaming()` returns `true` for the direct API path) will stream
internally but still return a complete `AgentResult` from `run`. The `supports_streaming()`
flag only indicates whether the adapter uses streaming internally — it does NOT change the
`run()` return type.

### What this means for the implementation
The adapters do not expose a separate streaming API at the `Agent` trait level. The `run()` method
is the only call site. Therefore, "streaming support" in the REPL means:
- Print a `[waiting...]` indicator BEFORE calling `run()` when `agent.supports_streaming()` is true
- While `run()` executes, keep the indicator on screen (tokio `select!` is NOT needed — just print before, clear after)
- Print the full response after `run()` returns, replacing the indicator

For non-streaming agents (`supports_streaming()` returns `false`):
- Print `[...]` before calling `run()` as a generic waiting indicator
- Print the response after `run()` returns

        let ctx = Context::now();

        // `[streaming...]` means the adapter uses HTTP streaming internally.
        // `[waiting...]`   means the adapter makes a blocking round-trip.
        // Either way, `run()` only returns when the full response is ready —

        } else {
