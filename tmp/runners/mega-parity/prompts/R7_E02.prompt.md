Session history persistence (basic)

## Problem
Chat sessions disappear when closed. Users cannot review past sessions or resume conversations. A basic persistence layer saves session summaries.

## Architecture Contract
- Summaries only, not full transcripts
- Non-blocking persistence
- Backward compatible with --resume

## Changes Required
1. In `chat_session.rs`:
   - On session end: write summary to `.roko/sessions/<timestamp>.json`
   - Summary includes: session_id, start_time, end_time, model, provider, turn_count, summary (first/last message), total_cost, total_tokens
   - Write is non-blocking (don't delay exit)
2. Add `roko history` command:
   - Lists last 20 sessions with: timestamp, model, turns, cost
   - `roko history <id>` shows detail for one session
3. Depends on R7_D02 (all chat features complete)

## Write Scope (files you may modify)
- `crates/roko-cli/src/chat_session.rs` (or equivalent)
- `crates/roko-cli/src/` — history command handler

## Read-Only Context (do not modify these)
