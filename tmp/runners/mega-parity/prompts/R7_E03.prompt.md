## Problem
MCP server configuration is manual and opaque. Users need CLI commands to list, test, and add MCP servers without editing config files directly.

## Architecture Contract
- Use existing MCP config format
- No MCP protocol handling
- No lifecycle management

## Changes Required
1. In `config_cmd.rs` (or config command handler):
   - `roko config mcp list`: list configured MCP servers with name, command, status
   - `roko config mcp test <name>`: test if MCP server starts successfully (spawn, check, kill)
   - `roko config mcp add <name> <command>`: add MCP server to roko.toml config
2. Uses existing MCP config format from roko.toml
3. `list` reads from config, `test` spawns briefly, `add` writes to config
