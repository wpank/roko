# AC_03: Wire ContextManager into ACP bridge events

## Task
Wire the `ContextManager` from AC_02 into the ACP bridge events dispatch path so that knowledge, playbooks, and gate feedback are assembled through the priority queue.

## Runner Context
Runner AC_A (ACP: Session Improvements), batch 3 of 3. Depends on AC_02.

## Problem
The bridge events dispatch path assembles context by concatenating strings directly. It should use `ContextManager` to prioritize and budget-constrain context sections.

## Current Code

**bridge_events.rs** -- `crates/roko-acp/src/bridge_events.rs`:
Contains the dispatch path that calls `query_dispatch_knowledge()`, then `append_context()` or `prepend_context()` to add knowledge to the prompt.

**ContextManager** -- from AC_02 (`crates/roko-acp/src/context_manager.rs`):
Has `add_section()` and `assemble()` methods.

## Exact Changes

### Step 1: Create context manager in dispatch path

In the bridge events dispatch function, after querying dispatch knowledge:

```rust
use crate::context_manager::ContextManager;

// After knowledge query:
let mut ctx_mgr = ContextManager::new(Some(budget_tokens));

// Add knowledge hits (priority 80)
if !knowledge.hits.is_empty() {
    ctx_mgr.add_section("Prior Knowledge", knowledge.context_text(), 80);
}

// Add playbooks (priority 90 -- highest, most actionable)
if !knowledge.playbooks.is_empty() {
    let playbook_text = format_playbooks(&knowledge.playbooks);
    ctx_mgr.add_section("Relevant Playbooks", playbook_text, 90);
}

// Add gate feedback (priority 95 -- critical for iteration)
if let Some(ref feedback) = gate_feedback_text {
    ctx_mgr.add_section("Gate Feedback", feedback.clone(), 95);
}

// Add conventions (priority 70)
if let Some(ref conventions) = conventions_text {
    ctx_mgr.add_section("Project Conventions", conventions.clone(), 70);
}

// Assemble within budget
let (context_text, included_sections) = ctx_mgr.assemble();
tracing::debug!(
    sections = ?included_sections,
    total_tokens = ctx_mgr.total_estimated_tokens(),
    "assembled context via ContextManager"
);
```

### Step 2: Replace direct concatenation

Replace existing `append_context()` / `prepend_context()` calls with the assembled `context_text` from the context manager.

### Step 3: Log section inclusion

```rust
if !included_sections.is_empty() {
    tracing::info!(
        count = included_sections.len(),
        sections = ?included_sections,
        "context sections included in dispatch"
    );
}
```

## Acceptance Criteria
- Context assembly uses ContextManager
- Gate feedback has highest priority (95), then playbooks (90), knowledge (80), conventions (70)
- Budget enforcement prevents context overflow
- Included sections logged for debugging
- Existing dispatch behavior preserved (same output when budget is unlimited)

## Do NOT
- Modify ContextManager itself
- Change knowledge query functions
- Remove the existing append_context/prepend_context functions (keep for backward compat)
