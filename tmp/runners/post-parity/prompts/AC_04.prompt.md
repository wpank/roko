# AC_04: Add ParallelExecution phase to pipeline

## Task
Add a `ParallelExecution` phase to the ACP pipeline state machine that allows multiple agents to run concurrently within a single workflow.

## Runner Context
Runner AC_B (ACP: Parallel Agents), batch 1 of 3. No dependencies.

## Problem
The current pipeline is strictly sequential: Strategizing -> Implementing -> Gating -> Reviewing -> Committing. There is no way to run multiple agents in parallel (e.g., two implementers working on different subtasks, or a researcher and an implementer simultaneously).

### Step 4: Update state machine transitions

Handle `ParallelStart`, `SubtaskCompleted`, `SubtaskFailed` events in the state machine:

```rust
// In the state machine's handle_event:
(PipelinePhase::Strategizing, PipelineEvent::ParallelStart { subtask_count }) => {
    self.phase = PipelinePhase::ParallelExecution {
        active_count: subtask_count,
        total_subtasks: subtask_count,
        completed_subtasks: 0,
    };
    PipelineAction::SpawnParallel { subtasks: /* from strategy */ }
}

(PipelinePhase::ParallelExecution { active_count, total_subtasks, completed_subtasks },
 PipelineEvent::SubtaskCompleted { .. }) => {
    let completed = completed_subtasks + 1;
    if completed >= *total_subtasks {
        // All done, move to gating
        self.phase = PipelinePhase::Gating;
        PipelineAction::RunGates
    } else {
        self.phase = PipelinePhase::ParallelExecution {
            active_count: active_count - 1,
            total_subtasks: *total_subtasks,
            completed_subtasks: completed,
        };
        // Continue waiting
        PipelineAction::Noop
    }
}
```
