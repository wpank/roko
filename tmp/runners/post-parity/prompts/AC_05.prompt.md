# AC_05: Implement parallel agent spawning in runner

## Task
Implement the `SpawnParallel` action handler in the ACP runner, spawning multiple agents concurrently via `tokio::JoinSet` and collecting their results.

## Runner Context
Runner AC_B (ACP: Parallel Agents), batch 2 of 3. Depends on AC_04.

## Problem
The pipeline state machine emits `SpawnParallel` actions but the runner does not handle them. Each `ParallelSubtask` needs to be dispatched as a separate agent, with results collected and fed back to the state machine.

## Current Code

**Runner** -- `crates/roko-acp/src/runner.rs`:
Has `run_with_workflow_engine()` that drives the pipeline. Handles `SpawnImplementer`, `SpawnStrategist`, etc. but not `SpawnParallel`.

## Exact Changes

### Step 1: Add parallel spawn handler

```rust
PipelineAction::SpawnParallel { subtasks } => {
    let mut join_set = tokio::task::JoinSet::new();

    for subtask in subtasks {
        let workdir = workdir.to_path_buf();
        let model_slug = config.model_slug.clone();
        // Clone what each task needs
        let subtask_id = subtask.id.clone();
        let role = subtask.role.clone();
        let prompt = subtask.prompt.clone();
        let context = subtask.context.clone();

        join_set.spawn(async move {
            tracing::info!(
                subtask_id = %subtask_id,
                role = %role,
                "spawning parallel subtask"
            );

            // Use ClaudeCliAgent for each subtask
            let result = spawn_subtask_agent(
                &workdir,
                &role,
                &prompt,
                context.as_deref(),
                &model_slug,
            ).await;

            (subtask_id, result)
        });
    }

    // Collect results
    while let Some(result) = join_set.join_next().await {
        match result {
            Ok((subtask_id, Ok(output))) => {
                tracing::info!(subtask_id = %subtask_id, "subtask completed");
                // Feed SubtaskCompleted event
                let _event = PipelineEvent::SubtaskCompleted {
                    subtask_id,
                    output,
                };
                // Feed into state machine...
            }
            Ok((subtask_id, Err(error))) => {
                tracing::error!(subtask_id = %subtask_id, %error, "subtask failed");
                let _event = PipelineEvent::SubtaskFailed {
                    subtask_id,
                    error,
                };
            }
            Err(join_err) => {
                tracing::error!(%join_err, "subtask panicked");
            }
        }
    }
}
```

### Step 2: Add subtask agent spawn helper

```rust
async fn spawn_subtask_agent(
    workdir: &Path,
    role: &str,
    prompt: &str,
    context: Option<&str>,
    model_slug: &str,
) -> Result<String, String> {
    let full_prompt = match context {
        Some(ctx) => format!("{prompt}\n\n## Additional Context\n\n{ctx}"),
        None => prompt.to_string(),
    };

    // Build the agent command
    let mut cmd = Command::new(CLAUDE_CLI_BIN);
    cmd.arg("--print")
        .arg("--model").arg(model_slug)
        .arg("--max-turns").arg("10")
        .arg("--dangerously-skip-permissions")
        .current_dir(workdir);

    cmd.stdin(std::process::Stdio::piped());
    cmd.stdout(std::process::Stdio::piped());
    cmd.stderr(std::process::Stdio::piped());

    let mut child = cmd.spawn()
        .map_err(|e| format!("spawn {role}: {e}"))?;

    if let Some(ref mut stdin) = child.stdin {
        use tokio::io::AsyncWriteExt;
        stdin.write_all(full_prompt.as_bytes()).await
            .map_err(|e| format!("write prompt: {e}"))?;
        stdin.shutdown().await.map_err(|e| format!("close stdin: {e}"))?;
    }

    let output = child.wait_with_output().await
        .map_err(|e| format!("wait {role}: {e}"))?;

    if output.status.success() {
        Ok(String::from_utf8_lossy(&output.stdout).to_string())
    } else {
        Err(String::from_utf8_lossy(&output.stderr).to_string())
    }
}
```

- Results collected and fed back as SubtaskCompleted/SubtaskFailed events
- Panic in one subtask does not crash others
- All subtask agents shut down when pipeline is cancelled
