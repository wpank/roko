# AC_06: Add workflow templates (Research, Review, Docs)

## Task
Add predefined workflow templates to the ACP pipeline that configure different phase sequences for research, code review, and documentation tasks.

## Runner Context
Runner AC_B (ACP: Parallel Agents), batch 3 of 3. No dependencies.

## Problem
The pipeline currently supports one workflow: strategize -> implement -> gate -> review -> commit. Different task types need different workflows: research tasks skip implementation/gating, review tasks skip strategizing, documentation tasks skip testing gates.

## Current Code

**WorkflowTemplate** -- `crates/roko-acp/src/pipeline.rs`:
Check if a `WorkflowTemplate` enum or struct exists. If not, define one.

## Exact Changes

### Step 1: Define or extend WorkflowTemplate

```rust
/// Predefined workflow templates.
#[derive(Debug, Clone, PartialEq, Eq, Serialize, Deserialize)]
#[serde(rename_all = "snake_case")]
pub enum WorkflowTemplate {
    /// Full pipeline: strategize -> implement -> gate -> review -> commit.
    Standard,
    /// Quick pipeline: implement -> gate -> commit (no strategy, no review).
    Express,
    /// Research-only: strategize -> research -> summarize (no impl, no gates).
    Research,
    /// Code review: review existing changes -> report findings.
    Review,
    /// Documentation: implement (docs only) -> fmt gate -> commit.
    Documentation,
    /// Parallel: strategize -> parallel subtasks -> gate -> commit.
    Parallel,
}

impl WorkflowTemplate {
    /// Return the phase sequence for this template.
    pub fn phases(&self) -> Vec<PipelinePhase> {
        match self {
            Self::Standard => vec![
                PipelinePhase::Strategizing,
                PipelinePhase::Implementing,
                PipelinePhase::Gating,
                PipelinePhase::Reviewing,
                PipelinePhase::Committing,
            ],
            Self::Express => vec![
                PipelinePhase::Implementing,
                PipelinePhase::Gating,
                PipelinePhase::Committing,
            ],
            Self::Research => vec![
                PipelinePhase::Strategizing,
                PipelinePhase::Implementing, // research agent
                PipelinePhase::Complete,
            ],
            Self::Review => vec![
                PipelinePhase::Reviewing,
                PipelinePhase::Complete,
            ],
            Self::Documentation => vec![
                PipelinePhase::Implementing,
                PipelinePhase::Gating,
                PipelinePhase::Committing,
            ],
            Self::Parallel => vec![
                PipelinePhase::Strategizing,
                PipelinePhase::Implementing, // placeholder for parallel
                PipelinePhase::Gating,
                PipelinePhase::Committing,
            ],
        }
    }

    /// Which gates to run for this template.
    pub fn enabled_gates(&self) -> Vec<&'static str> {
        match self {
            Self::Standard => vec!["compile", "clippy", "test", "diff"],
            Self::Express => vec!["compile", "test"],
            Self::Research => vec![],
            Self::Review => vec![],
            Self::Documentation => vec!["fmt"],
            Self::Parallel => vec!["compile", "clippy", "test", "diff"],
        }
    }

    /// Agent role for the implementation phase.
    pub fn implementer_role(&self) -> &'static str {
        match self {
            Self::Research => "researcher",
            Self::Documentation => "documenter",
            _ => "implementer",
        }
    }

    /// Whether this template requires a strategist phase.
    pub fn needs_strategy(&self) -> bool {
        matches!(self, Self::Standard | Self::Research | Self::Parallel)
    }

    /// Whether this template requires a review phase.
    pub fn needs_review(&self) -> bool {
        matches!(self, Self::Standard | Self::Review)
    }
}

impl Default for WorkflowTemplate {
    fn default() -> Self {
        Self::Standard
    }
}
```

### Step 2: Add auto-detection

```rust
impl WorkflowTemplate {
    /// Auto-detect the best template for a given prompt.
    pub fn auto_detect(prompt: &str) -> Self {
        let lower = prompt.to_lowercase();
        if lower.contains("research") || lower.contains("investigate") || lower.contains("analyze") {
            Self::Research
        } else if lower.contains("review") || lower.contains("audit") {
            Self::Review
        } else if lower.contains("document") || lower.contains("readme") || lower.contains("docs") {
            Self::Documentation
        } else {
            Self::Standard
        }
    }
}
```
