# AD_01: Construct DreamRunner from ACP workdir + load latest report

## Task
At ACP server startup (or first `session/new`), construct a `DreamRunner` for the ACP workdir and call `latest_report()` to retrieve the most recent persisted dream consolidation. Cache the result in the `SessionManager` so AD_02 can render it without re-scanning the filesystem on every new session.

## Runner Context
Runner AD_A (ACP: Dream Journal at Session Start), batch 1 of 4. No dependencies.

This batch closes the deferred `R7_F08` "Dream Journal" workflow from `tmp/solutions/acp/08-NOVEL-WORKFLOWS.md` (Workflow 3) and `09-NOVEL-BATCHES.md`.

## Problem
`roko-dreams` is already a dependency of `roko-acp` (verified in `crates/roko-acp/Cargo.toml:27`), and `DreamRunner::latest_report()` exists at `crates/roko-dreams/src/runner.rs:801`. Dream consolidation runs in the background via `post-parity(PK_04)` (commit `bd02c588`). But ACP never queries the report and never tells the editor what was learned overnight.

### Step 2: Load on construction

Modify `SessionManager::new`:

```rust
impl SessionManager {
    pub fn new(workdir: PathBuf, roko_config: roko_core::config::schema::RokoConfig) -> Self {
        let pending_dream_report = load_pending_dream_report(&workdir);
        Self {
            sessions: HashMap::new(),
            workdir,
            roko_config,
            pending_dream_report,
        }
    }
}

/// Best-effort dream report load. Returns None on any error.
fn load_pending_dream_report(workdir: &Path) -> Option<roko_dreams::DreamReport> {
    use roko_dreams::{DreamLoopConfig, DreamRunner};
    let runner = DreamRunner::new(workdir.to_path_buf(), DreamLoopConfig::default());
    match runner.latest_report() {
        Ok(report) => report,
        Err(error) => {
            tracing::debug!(
                error = %error,
                workdir = %workdir.display(),
                "no dream report available at ACP startup"
            );
            None
        }
    }
}
```

## Acceptance Criteria
- `SessionManager.pending_dream_report: Option<DreamReport>` field
- Field is `#[serde(skip)]` (not part of session JSON)
- Loaded eagerly in `SessionManager::new`
- Returns `None` (not panic, not Err) when:
  - `.roko/dreams/` is missing
  - the dreams crate fails to deserialize a report
  - any I/O error occurs
- Tracing at `debug!` level for misses
- New unit test verifies missing-dir path
