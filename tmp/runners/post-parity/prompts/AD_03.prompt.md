# AD_03: Track presented dream report id and skip duplicates

## Task
Persist the `dream_card_id` of the most recently presented dream report in the workdir (`.roko/sessions/last-dream-shown.txt`) so the same report is not shown twice across `session/new` calls within the same process AND across process restarts.

## Runner Context
Runner AD_A (ACP: Dream Journal at Session Start), batch 3 of 4. Depends on AD_02.

## Problem
After AD_01+AD_02, every `session/new` while a report exists will show the same card. The mock requires "show once, then skip until the next consolidation runs". Current cache lives only in `SessionManager.pending_dream_report` which:
- is per-process (lost on restart)
- is shared across `session/new` calls (everyone in this process sees it)

We need cross-process / cross-session memory.

## Existing Anchor Points

`.roko/sessions/` already exists and holds session JSON (`crates/roko-acp/src/session.rs::persist_session`). A sibling file `.roko/sessions/last-dream-shown.txt` is the simplest persistence — single line of text, atomic write.

`crates/roko-acp/src/dream_card.rs::dream_card_id` (added by AD_02) returns `dream-<unix-ts>`.

## Exact Changes

### Step 1: Helper module for the marker

Append to `crates/roko-acp/src/dream_card.rs`:

```rust
use std::path::Path;

/// Path to the "last shown" marker. The file contains a single line: the
/// dream_card_id that was last presented.
pub fn last_shown_path(workdir: &Path) -> std::path::PathBuf {
    workdir.join(".roko").join("sessions").join("last-dream-shown.txt")
}

/// Read the previously-presented dream id, if any.
pub fn read_last_shown(workdir: &Path) -> Option<String> {
    let path = last_shown_path(workdir);
    let raw = std::fs::read_to_string(&path).ok()?;
    let trimmed = raw.trim().to_string();
    if trimmed.is_empty() { None } else { Some(trimmed) }
}

/// Atomically write the dream id we just presented.
pub fn write_last_shown(workdir: &Path, id: &str) -> std::io::Result<()> {
    let path = last_shown_path(workdir);
    if let Some(parent) = path.parent() {
        std::fs::create_dir_all(parent)?;
    }
    let tmp = path.with_extension("tmp");
    std::fs::write(&tmp, id)?;
    std::fs::rename(&tmp, &path)?;
    Ok(())
}
```

### Step 2: Skip duplicate presentation

Modify the `session/new` arm in `crates/roko-acp/src/handler.rs` (added by AD_02):

```rust
if let Some(report) = pending_report {
    let id = crate::dream_card::dream_card_id(&report);
    let already_shown = crate::dream_card::read_last_shown(&sessions_workdir).as_deref() == Some(&id);
    if !already_shown {
        emit_dream_card(transport, &session_id, &report).await?;
        if let Err(err) = crate::dream_card::write_last_shown(&sessions_workdir, &id) {
            tracing::warn!(error = %err, "failed to persist dream-shown marker");
        }
    } else {
        tracing::debug!(id = %id, "skipping already-presented dream report");
    }
}
```

`sessions_workdir` should be cloned from `sessions` (or its read guard) before any `await` that needs `&mut sessions`.

### Step 3: Add the cache invalidation hook

If a new dream consolidation produces a NEW report, the old marker becomes stale. The marker file is keyed on the report's `completed_at` timestamp, so a new report (later timestamp) automatically gets a different id and the duplicate-check passes correctly. No invalidation needed.

But: an in-flight ACP process that already loaded the OLD report into `pending_dream_report` will keep showing the old one until restart. Optional follow-up (file in tracker): add a file watcher that reloads when `.roko/dreams/` changes.

### Step 4: Unit tests

In `crates/roko-acp/src/dream_card.rs` `#[cfg(test)]`:

```rust
#[test]
fn last_shown_round_trip() {
    let tmp = tempfile::tempdir().unwrap();
    let id = "dream-1745923200";

    assert!(read_last_shown(tmp.path()).is_none());

    write_last_shown(tmp.path(), id).unwrap();
    assert_eq!(read_last_shown(tmp.path()).as_deref(), Some(id));

    // Idempotent overwrite
    write_last_shown(tmp.path(), id).unwrap();
    assert_eq!(read_last_shown(tmp.path()).as_deref(), Some(id));
}

#[test]
fn last_shown_atomic_write_no_partial() {
    let tmp = tempfile::tempdir().unwrap();
    write_last_shown(tmp.path(), "dream-1").unwrap();
    write_last_shown(tmp.path(), "dream-2").unwrap();
    assert_eq!(read_last_shown(tmp.path()).as_deref(), Some("dream-2"));
    // .tmp file should not linger
    let tmp_file = last_shown_path(tmp.path()).with_extension("tmp");
    assert!(!tmp_file.exists());
}
```

- `last_shown_path`, `read_last_shown`, `write_last_shown` exported from `dream_card`
- Atomic write via `.tmp` rename
- Returns `None` when file is missing OR empty (whitespace-only treated as empty)
- handler emits dream card only when `read_last_shown != Some(current_id)`
- Marker file path: `<workdir>/.roko/sessions/last-dream-shown.txt`
- Unit tests pass for round-trip + atomicity
