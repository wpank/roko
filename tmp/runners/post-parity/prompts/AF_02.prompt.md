# AF_02: Add AffectSnapshot type + serialize PAD vector for ACP emission

## Task
Define an `AffectSnapshot` payload type in `crates/roko-acp/src/types.rs` that carries the daimon's PAD vector, behavioral state, emotional tag, and confidence. Add a helper that constructs it from `AffectContext` (foundation type) and a deterministic markdown renderer used by AF_03 to emit the "Mood Ring" card.

## Runner Context
Runner AF_A (ACP: Affect / Mood Ring Surface), batch 2 of 5. Depends on AF_01.

## Problem
The runtime computes `AffectContext { behavioral_state, pad: [f32; 3], emotional_tag }` (`crates/roko-core/src/foundation.rs:496-502`), but ACP has no type for marshaling that into `session/update` notifications. AF_03 needs a stable struct + Display/Markdown impl so the editor can render a consistent "💛 Agent state: …" card.

### Step 1: Add types to `crates/roko-acp/src/types.rs`

Add at the end of the file, before any `#[cfg(test)]` block:

```rust
// =============================================================================
// AF_02: Affect snapshot payload for session/update emission.
// =============================================================================

/// Snapshot of the agent's affect state at a single instant.
///
/// Produced by `AffectPolicy::pre_dispatch` / `behavioral_state` and emitted
/// to the ACP client as part of an `AgentMessageChunk` so the editor can
/// render a "Mood Ring" indicator.
#[derive(Debug, Clone, Serialize, Deserialize, PartialEq)]
#[serde(rename_all = "camelCase")]
pub struct AffectSnapshot {
    /// PAD triple: pleasure, arousal, dominance, each in `[-1.0, 1.0]`.
    pub pad: [f32; 3],
    /// Discrete behavioral classification.
    pub behavioral_state: AcpBehavioralState,
    /// Optional human-readable trigger label (e.g. "gate_failure").
    #[serde(default, skip_serializing_if = "Option::is_none")]
    pub emotional_tag: Option<String>,
    /// Affect confidence in `[0.0, 1.0]` (defaults to 0.5 when unknown).
    #[serde(default = "default_confidence")]
    pub confidence: f32,
    /// Phase that produced this snapshot (e.g. "implementing", "gating").
    #[serde(default, skip_serializing_if = "Option::is_none")]
    pub phase: Option<String>,
    /// ISO-8601 timestamp.
    pub timestamp: String,
}

fn default_confidence() -> f32 { 0.5 }

/// Mirrors `roko_core::BehavioralState` but is the ACP-wire type.
#[derive(Debug, Clone, Copy, Serialize, Deserialize, PartialEq, Eq)]
#[serde(rename_all = "snake_case")]
pub enum AcpBehavioralState {
    Engaged,
    Struggling,
    Coasting,
    Exploring,
    Focused,
    Resting,
}

impl From<roko_core::BehavioralState> for AcpBehavioralState {
    fn from(state: roko_core::BehavioralState) -> Self {
        use roko_core::BehavioralState as B;
        match state {
            B::Engaged => Self::Engaged,
            B::Struggling => Self::Struggling,
            B::Coasting => Self::Coasting,
            B::Exploring => Self::Exploring,
            B::Focused => Self::Focused,
            B::Resting => Self::Resting,
        }
    }
}

impl AcpBehavioralState {
    pub fn glyph(&self) -> &'static str {
        match self {
            Self::Engaged => "💛",
            Self::Struggling => "🟠",
            Self::Coasting => "💚",
            Self::Exploring => "🔵",
            Self::Focused => "🎯",
            Self::Resting => "🌙",
        }
    }
    pub fn label(&self) -> &'static str {
        match self {
            Self::Engaged => "engaged",
            Self::Struggling => "struggling",
            Self::Coasting => "coasting",
            Self::Exploring => "exploring",
            Self::Focused => "focused",
            Self::Resting => "resting",
        }
    }
}

impl AffectSnapshot {
    pub fn from_context(
        ctx: &roko_core::foundation::AffectContext,
        phase: Option<String>,
        confidence: f32,
    ) -> Self {
        Self {
            pad: ctx.pad,
            behavioral_state: ctx.behavioral_state.into(),
            emotional_tag: ctx.emotional_tag.clone(),
            confidence: confidence.clamp(0.0, 1.0),
            phase,
            timestamp: chrono::Utc::now().to_rfc3339(),
        }
    }

    /// Render as compact markdown for `AgentMessageChunk` content.
    pub fn render_markdown(&self) -> String {
        let [p, a, d] = self.pad;
        let glyph = self.behavioral_state.glyph();
        let label = self.behavioral_state.label();
        let mut out = format!(
            "{glyph} **Agent state: {label}**  \n_pleasure: {p:+.2}, arousal: {a:+.2}, dominance: {d:+.2}_"
        );
        if let Some(tag) = &self.emotional_tag {
            out.push_str(&format!("  \n_trigger: {tag}_"));
        }
        if let Some(phase) = &self.phase {
            out.push_str(&format!("  \n_phase: {phase}_"));
        }
        out
    }
}
```

### Step 3: Unit test

Add to the `tests` module at the end of `types.rs` (or create one):

```rust
#[cfg(test)]
mod af_02_tests {
    use super::*;
    use roko_core::BehavioralState;
    use roko_core::foundation::AffectContext;

    #[test]
    fn snapshot_round_trips_json() {
        let ctx = AffectContext {
            behavioral_state: BehavioralState::Struggling,
            pad: [-0.4, 0.6, -0.3],
            emotional_tag: Some("gate_failure".into()),
        };
        let snap = AffectSnapshot::from_context(&ctx, Some("gating".into()), 0.7);
        let json = serde_json::to_string(&snap).unwrap();
        let back: AffectSnapshot = serde_json::from_str(&json).unwrap();
        assert_eq!(back, snap);
    }

    #[test]
    fn markdown_includes_glyph_and_pad() {
        let ctx = AffectContext {
            behavioral_state: BehavioralState::Struggling,
            pad: [-0.4, 0.6, -0.3],
            emotional_tag: None,
        };
        let snap = AffectSnapshot::from_context(&ctx, None, 0.5);
        let md = snap.render_markdown();
        assert!(md.contains("🟠"));
        assert!(md.contains("struggling"));
        assert!(md.contains("-0.40"));
    }
}
```
