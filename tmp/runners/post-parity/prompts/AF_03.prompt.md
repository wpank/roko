# AF_03: Emit affect card AgentMessageChunk after gate/phase transitions

## Task
Wire the `AffectSnapshot` from AF_02 into the ACP runner so a "Mood Ring" `AgentMessageChunk` is emitted after each pipeline phase transition (and after every gate result) when `affect_mode` is `observe` or `modulate`.

## Runner Context
Runner AF_A (ACP: Affect / Mood Ring Surface), batch 3 of 5. Depends on AF_02 (which depends on AF_01).

## Problem
With AF_01 the `WorkflowEngine` path now runs with `affect_enabled = true` and the `EffectDriver` already calls `AffectPolicy::on_task_outcome` and `on_gate_result` (verified via `converge-followup(B02)`). However:

1. The legacy `run_workflow_pipeline` path (`crates/roko-acp/src/runner.rs:892`) is the one used today for ACP and does NOT go through `EffectDriver`. It has its own `WorkflowRun` state and emits `phase_badge` + `emit_narrative` calls but never queries the daimon.
2. No ACP `session/update` carries the affect state — the editor cannot render a mood indicator.

This batch fixes both: it adds a daimon hook to the legacy pipeline AND emits the snapshot as an `AgentMessageChunk`.

## Exact Changes

### Step 1: Add affect helper

Add inside `crates/roko-acp/src/runner.rs`, near `emit_narrative`:

```rust
use roko_core::foundation::{AffectPolicy, AffectContext};
use roko_daimon::DaimonPolicy;
use std::sync::Arc;
use tokio::sync::Mutex as TokioMutex;
use crate::types::{AffectSnapshot, ContentBlock, SessionUpdate};
use crate::session::CognitiveEvent;

/// Optional affect handle threaded through the pipeline.
///
/// `None` when `affect_mode == "off"`. When present, every phase
/// transition queries it and emits a Mood Ring chunk.
pub(crate) type SharedAffect = Arc<TokioMutex<DaimonPolicy>>;

pub(crate) fn build_affect(workdir: &Path, mode: &str) -> Option<SharedAffect> {
    if matches!(mode, "observe" | "modulate") {
        let state_path = workdir.join(".roko").join("daimon-state.json");
        Some(Arc::new(TokioMutex::new(DaimonPolicy::new(state_path))))
    } else {
        None
    }
}

/// Emit a Mood Ring chunk if `affect` is enabled.
async fn emit_affect_card(
    affect: &Option<SharedAffect>,
    phase: &str,
    sender: &mpsc::Sender<CognitiveEvent>,
) {
    let Some(handle) = affect.as_ref() else { return };
    let snapshot = {
        let guard = handle.lock().await;
        let ctx = AffectPolicy::pre_dispatch(&*guard, phase, phase);
        AffectSnapshot::from_context(&ctx, Some(phase.to_string()), 0.7)
    };
    let md = snapshot.render_markdown();
    let event = CognitiveEvent::SessionUpdate(SessionUpdate::AgentMessageChunk {
        content: ContentBlock::Text { text: format!("\n{md}\n") },
        _meta: Some(serde_json::json!({ "kind": "affect_snapshot", "phase": phase })),
    });
    let _ = sender.send(event).await;
}

/// Hook called after a gate verdict — feeds outcome to the policy.
async fn record_gate_to_affect(
    affect: &Option<SharedAffect>,
    gate_name: &str,
    passed: bool,
    rung: u8,
    confidence: f64,
) {
    let Some(handle) = affect.as_ref() else { return };
    let mut guard = handle.lock().await;
    AffectPolicy::on_gate_result(&mut *guard, gate_name, passed, rung, confidence);
}

/// Hook called after a pipeline phase finishes — feeds task outcome.
async fn record_task_to_affect(
    affect: &Option<SharedAffect>,
    task_id: &str,
    succeeded: bool,
    tokens_used: u64,
    cost_usd: f64,
) {
    let Some(handle) = affect.as_ref() else { return };
    let mut guard = handle.lock().await;
    AffectPolicy::on_task_outcome(&mut *guard, task_id, succeeded, tokens_used, cost_usd);
}

/// Persist daimon state at the end of a pipeline run.
async fn persist_affect(affect: &Option<SharedAffect>) {
    let Some(handle) = affect.as_ref() else { return };
    let guard = handle.lock().await;
    if let Err(e) = AffectPolicy::persist(&*guard).await {
        tracing::warn!(error = %e, "failed to persist daimon state");
    }
}
```

### Step 6: ACP-level emission opt-in
