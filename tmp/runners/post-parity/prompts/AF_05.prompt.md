# AF_05: Emit ConfigOptionUpdate for routing/scope changes after escalation

## Task
After AF_04 auto-escalates the model, surface that change in the editor by emitting a `ConfigOptionUpdate` for the `model.tier` (or equivalent) config option, plus a follow-up `AgentMessageChunk` explaining why. This is the "⚡ Auto-escalating: sonnet → opus" line in the Workflow 2 mock.

## Runner Context
Runner AF_B (ACP: Affect-Driven Modulation), batch 2 of 2. Depends on AF_04.

## Problem
AF_04 mutates `session.auto_escalated_tier` and changes the resolved model in-place, but the editor's status bar and routing widget don't know — they only saw the old value at `session/new`. Without a `ConfigOptionUpdate`, the user sees a stale "claude-sonnet-4.7" while the actual dispatch goes to opus. Plus, there's no narrative explaining the escalation.

## Exact Changes

### Step 1: Add a small emitter helper

In `crates/roko-acp/src/bridge_events.rs`, near the AF_04 escalation block:

```rust
async fn emit_escalation_update(
    session_id: &str,
    from_slug: &str,
    to_slug: &str,
    sender: &mpsc::Sender<CognitiveEvent>,
) {
    // 1) ConfigOptionUpdate so the status-bar refreshes.
    let cfg_update = SessionUpdate::ConfigOptionUpdate {
        option_id: "model.active".to_owned(),
        value: serde_json::json!(to_slug),
        label: Some(format!("Auto-escalated: {to_slug}")),
    };
    let _ = sender
        .send(CognitiveEvent::SessionUpdate(cfg_update))
        .await;

    // 2) AgentMessageChunk narrating the change.
    let narrative = format!(
        "\n⚡ **Auto-escalating: `{from_slug}` → `{to_slug}`** \
         _(daimon detected persistent struggle; routing for the next dispatch is upgraded)_\n"
    );
    let chunk = SessionUpdate::AgentMessageChunk {
        content: ContentBlock::Text { text: narrative },
        _meta: Some(serde_json::json!({
            "kind": "auto_escalation",
            "from": from_slug,
            "to": to_slug,
        })),
    };
    let _ = sender.send(CognitiveEvent::SessionUpdate(chunk)).await;
}
```

### Step 2: Call from AF_04 escalation site

Replace the `tracing::info!(...)` block in AF_04 with:

```rust
if let Some(escalated) = escalate_via_cascade(&resolved.slug, workdir) {
    if escalated.slug != resolved.slug {
        tracing::info!(
            from = %resolved.slug,
            to = %escalated.slug,
            "AF_05 auto-escalating model"
        );
        emit_escalation_update(
            &session.session_id,
            &resolved.slug,
            &escalated.slug,
            &event_sender,
        ).await;
        session.auto_escalated_tier = Some(escalated.slug.clone());
        resolved = escalated;
    }
}
```

### Step 3: Emit revert when daimon recovers

In `runner.rs` where `consecutive_struggling` resets to 0 (AF_04 Step 2), if `auto_escalated_tier.is_some()`:

```rust

        label: Some("Recovered — routing back to default".to_owned()),
    };
    let _ = event_sender.send(CognitiveEvent::SessionUpdate(cfg_update)).await;
}
```
