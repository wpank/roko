# AS_01: Wrap SessionManager in Arc<tokio::sync::RwLock> at handler call site

## Task
Wrap the ACP `SessionManager` in `Arc<tokio::sync::RwLock<SessionManager>>` at the call site in `handler.rs` so that future concurrent transports (daemon mode, multi-editor websocket multiplex) can safely share session state without splitting state across tasks.

## Runner Context
Runner AS_A (ACP: Session Manager Concurrency), batch 1 of 1. No dependencies.

This batch closes the unresolved item from `R7_F03` (delivered in commit `199587a9`). That batch addressed slash-command thread-safety concerns but the `SessionManager` itself is still single-threaded — the in-source comment in `crates/roko-acp/src/session.rs:663-668` calls this out as deferred.

## Problem

Current state (verified 2026-05-01):

```663:676:crates/roko-acp/src/session.rs
/// In-memory store for ACP sessions.
///
/// The ACP stdio handler currently owns this manager on a single request
/// loop. If ACP gains concurrent transports, wrap it at the call site in
/// `Arc<tokio::sync::RwLock<SessionManager>>` instead of splitting session
/// state across tasks here.
#[derive(Debug, Clone)]
pub struct SessionManager {
    sessions: HashMap<String, AcpSession>,
    /// Working directory inherited from AcpConfig.
    pub workdir: PathBuf,
    /// Loaded roko.toml configuration.
    pub roko_config: roko_core::config::schema::RokoConfig,
}
```

The handler holds `let mut sessions = SessionManager::new(...)` directly and passes `&mut sessions` into every dispatch (`handler.rs:52`, `handler.rs:95`, `handler.rs:273`, `handler.rs:304`). This works for stdio but blocks any concurrent dispatcher.

## Exact Changes

### Step 1: Introduce a wrapper alias

Add at the top of `handler.rs`, after the existing imports:

```rust
use std::sync::Arc;
use tokio::sync::RwLock;

/// Shared session-manager handle. Cheap to clone; multiplexed handlers
/// can hold their own clone and still see consistent state.
pub type SharedSessionManager = Arc<RwLock<SessionManager>>;
```

### Step 2: Build the shared manager in `run_acp_server_with_transport`

Replace the local mutable binding:

```rust
// BEFORE (handler.rs:52):
let mut sessions = SessionManager::new(config.workdir.clone(), roko_config);
sessions.gc_old_sessions(chrono::Duration::days(7));

// AFTER:
let sessions: SharedSessionManager = Arc::new(RwLock::new(SessionManager::new(
    config.workdir.clone(),
    roko_config,
)));
sessions.write().await.gc_old_sessions(chrono::Duration::days(7));
```

### Step 3: Update internal helpers

Convert `&mut SessionManager` to `&SharedSessionManager` and acquire the right lock guard at each call site:

```rust
// handle_request signature
async fn handle_request<R, W>(
    transport: &mut StdioTransport<R, W>,
    sessions: &SharedSessionManager,
    request: JsonRpcRequest,
) -> Result<()>
where R: AsyncRead + Unpin, W: AsyncWrite + Unpin,
```

Inside each method arm:

| Method | Lock |
|---|---|
| `session/new` | `sessions.write().await` (mutates `sessions: HashMap`) |
| `session/list` | `sessions.read().await` (read-only) |
| `session/load` | `sessions.write().await` (mutates `sessions`) |
| `session/prompt` | clone the workdir+config from a read guard, drop, then take a write guard for mutable access |
| `session/config/update` | `sessions.write().await` |
| `session/cancel` | `sessions.read().await` |
| `session/close` | `sessions.write().await` |
| `session/set_mode` | `sessions.write().await` |
| `notifications` | `sessions.write().await` |

### Step 4: Pattern for `session/prompt`

The current code:

```rust
let workdir = sessions.workdir.clone();
let roko_config = sessions.roko_config.clone();
let session = match get_session_mut(sessions, &params.session_id) {
```

Becomes:

```rust
// Read-only data needed for the entire call
let (workdir, roko_config) = {
    let guard = sessions.read().await;
    (guard.workdir.clone(), guard.roko_config.clone())
};

// Hold the write lock only for the prompt body since handle_session_prompt
// needs &mut AcpSession and persistence happens after.
let mut guard = sessions.write().await;
let session = match get_session_mut(&mut *guard, &params.session_id) {
    Ok(session) => session,
    Err(error) => return send_error_response(transport, id, error).await,
};
```

Note: `handle_session_prompt` is `async`, so the write guard is held across an `.await`. This is intentional for now — the comment in `session.rs` says the lock is correctness-only, not performance. Document this in a `// TODO(daemon-mode):` comment.

### Step 5: `get_session_mut` signature

The existing helper at `handler.rs:304`:

```rust

    sessions: &'a mut SessionManager,
    session_id: &str,

```

stays unchanged in shape — callers already obtain `&mut SessionManager` from the lock guard.
