# AT_04: Run tournament arms in parallel via ParallelExecutor + collect verdicts

## Task
Drive the per-arm pipelines concurrently using `tokio::spawn` (or `roko_orchestrator::executor::ParallelExecutor` if present), one task per arm, each running `run_workflow_pipeline` against its own worktree. Collect per-arm results (gate verdicts, cost, files changed, commit SHA) into the parent `TournamentRun`. Halt early when the global cost ceiling is exceeded.

## Runner Context
Runner AT_B (ACP: Tournament Execution), batch 2 of 5. Depends on AT_03.

## Problem
After AT_03 we have N worktrees and N event-tagging senders. We need an executor that:
1. Runs N pipelines truly in parallel (no global mutex).
2. Streams events back to the editor with arm tags.
3. Aggregates outcomes into `TournamentRun`.
4. Honors `tournament.max_cost_usd` (cancel arms past the cap).
5. Returns when ALL arms terminate (or cap is hit).

### Step 1: Per-arm pipeline driver

In `crates/roko-acp/src/tournament_runner.rs` (add to AT_03's file):

```rust
use crate::pipeline::{PipelineConfig, WorkflowTemplate};
use crate::runner::{run_workflow_pipeline, SharedWorkflowRun};
use crate::session::CognitiveEvent;
use crate::workflow::WorkflowRun;
use roko_core::CancelToken;

/// Run a single arm. The returned `WorkflowRun` is the arm-local run with
/// its tournament_id / tournament_arm_id fields populated.
pub async fn run_arm(
    parent_run_id: &str,
    arm: &TournamentArm,
    user_prompt: &str,
    knowledge_context: String,
    cancel: CancelToken,
    parent_sender: tokio::sync::mpsc::Sender<CognitiveEvent>,
) -> anyhow::Result<WorkflowRun> {
    let arm_sender = arm_tagged_sender(arm, parent_sender);

    // Compose the strategy hint into the prompt so the strategist sees it.
    let prompt_with_strategy = format!(
        "{user_prompt}\n\n<tournament_strategy>\n{}\n</tournament_strategy>",
        arm.strategy
    );

    let shared_run: SharedWorkflowRun = std::sync::Arc::new(
        tokio::sync::Mutex::new(None)
    );

    let config = PipelineConfig {
        template: WorkflowTemplate::Standard,  // arms run Standard internally
        max_iterations: 3,
        affect: None,                           // affect handled at parent level
        // ...other PipelineConfig fields with sensible defaults
    };

    run_workflow_pipeline(
        &format!("{parent_run_id}-{}", arm.arm_id),
        &prompt_with_strategy,
        knowledge_context,
        None,
        &arm.worktree_path,
        config,
        cancel,
        arm_sender,
        shared_run.clone(),
    ).await?;

    let run = shared_run.lock().await.take()
        .ok_or_else(|| anyhow::anyhow!("arm {} produced no WorkflowRun", arm.arm_id))?;

    let mut run = run;
    run.tournament_id = Some(parent_run_id.to_string());
    run.tournament_arm_id = Some(arm.arm_id.clone());
    Ok(run)
}
```

### Step 2: Tournament driver

```rust
use tokio::task::JoinSet;
use tokio::sync::Mutex as TokioMutex;
use std::sync::Arc;

/// Drive all arms in parallel. Returns the populated TournamentRun.
pub async fn run_tournament(
    workdir: &std::path::Path,
    mut run: TournamentRun,
    knowledge_context: String,
    cancel: CancelToken,
    parent_sender: tokio::sync::mpsc::Sender<CognitiveEvent>,
) -> anyhow::Result<TournamentRun> {
    let manager = provision_worktrees(workdir, &mut run).await?;
    let run_arc: Arc<TokioMutex<TournamentRun>> = Arc::new(TokioMutex::new(run));

    let mut join_set = JoinSet::new();
    let arms_count = run_arc.lock().await.arms.len();
    for i in 0..arms_count {
        let run_for_task = Arc::clone(&run_arc);
        let cancel_for_task = cancel.clone();
        let prompt = run_arc.lock().await.user_prompt.clone();
        let parent_run_id = run_arc.lock().await.run_id.clone();
        let knowledge = knowledge_context.clone();
        let parent_sender = parent_sender.clone();

        join_set.spawn(async move {
            // Snapshot the arm we're driving (read-only).
            let arm_snapshot = {
                let g = run_for_task.lock().await;
                g.arms[i].clone()
            };

            // Mark Running.
            {
                let mut g = run_for_task.lock().await;
                g.arms[i].status = ArmStatus::Running;
            }

            let outcome = run_arm(
                &parent_run_id,
                &arm_snapshot,
                &prompt,
                knowledge,
                cancel_for_task,
                parent_sender,
            ).await;

            // Persist outcome back into the shared run.
            let mut g = run_for_task.lock().await;
            let arm = &mut g.arms[i];
            match outcome {
                Ok(workflow_run) => {
                    arm.cost_usd = workflow_run.total_cost_usd;
                    arm.tokens_used = workflow_run.total_tokens;
                    arm.commit_sha = workflow_run.commit_sha.clone();
                    arm.files_changed = workflow_run.files_changed.clone();
                    // Map gate verdicts from the embedded workflow run.
                    for (gate, passed) in workflow_run.gate_results.iter() {
                        arm.gate_results.insert(gate.clone(), *passed);
                    }
                    let all_required_passed = g.config.required_gates.iter()
                        .all(|gate| arm.gate_results.get(gate).copied().unwrap_or(false));
                    arm.status = if all_required_passed { ArmStatus::Passed } else { ArmStatus::Failed };
                    arm.workflow = Some(workflow_run);
                }
                Err(err) => {
                    tracing::warn!(arm = %arm.arm_id, error = %err, "tournament arm errored");
                    arm.status = ArmStatus::Errored;
                }
            }
        });
    }

    // Drive the join set with a budget watchdog.
    while let Some(result) = join_set.join_next().await {
        if let Err(err) = result {
            tracing::error!(error = %err, "tournament arm task panicked");
        }

        // Budget check after each completion.
        let cap_hit = {
            let g = run_arc.lock().await;
            g.budget_exhausted()
        };
        if cap_hit {
            tracing::warn!("tournament budget exhausted; cancelling remaining arms");
            cancel.cancel();
            // Mark in-flight arms as BudgetExhausted.
            let mut g = run_arc.lock().await;
            for arm in g.arms.iter_mut() {
                if matches!(arm.status, ArmStatus::Running | ArmStatus::Pending) {
                    arm.status = ArmStatus::BudgetExhausted;
                }
            }
            // Continue draining join_set (cancelled tasks return promptly).
        }
    }

    let mut final_run = Arc::try_unwrap(run_arc)
        .map(|m| m.into_inner())
        .unwrap_or_else(|arc| {
            // Fall back to clone if there are still strong refs (shouldn't happen).
            let g = arc.blocking_lock();
            g.clone()
        });
    final_run.completed_at = Some(chrono::Utc::now());

    cleanup_worktrees(&manager, &final_run).await;

    Ok(final_run)
}
```

### Step 3: Wire `WorkflowRun` extra fields

The Step 1 code references `workflow_run.commit_sha`, `workflow_run.files_changed`, and `workflow_run.gate_results`. If those fields don't exist on `WorkflowRun`:

1. Verify with `rg "commit_sha|files_changed|gate_results" crates/roko-acp/src/workflow.rs`.
2. If missing, ADD them with `#[serde(default)]`:
   ```rust
   #[serde(default)]
   pub commit_sha: Option<String>,
   #[serde(default)]
   pub files_changed: Vec<String>,
   #[serde(default)]
   pub gate_results: std::collections::HashMap<String, bool>,
   ```
3. Populate them inside the existing `run_workflow_pipeline` at the appropriate phase transitions (commit → set commit_sha; gate → set gate_results; commit → set files_changed from `git diff --name-only`).

### Step 4: Single-arm cancellation propagation

Each arm task accepts a CLONED CancelToken. Cancelling the parent token short-circuits all arms. This is enough for "cancel tournament" — single-arm cancel is a follow-up.

### Step 5: Persist tournament run

After `run_tournament` returns, persist to `<workdir>/.roko/tournaments/<run_id>.json`:

```rust
pub fn persist_tournament(workdir: &std::path::Path, run: &TournamentRun) -> std::io::Result<()> {
    let dir = workdir.join(".roko").join("tournaments");
    std::fs::create_dir_all(&dir)?;
    let path = dir.join(format!("{}.json", run.run_id));
    let json = serde_json::to_string_pretty(run)?;
    std::fs::write(path, json)?;
    Ok(())
}
```
