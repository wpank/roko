# CD_01: Remove blanket #![allow(dead_code)] from roko-cli

## Goal

Remove the `#![allow(dead_code, unused_imports, unused_variables)]` line
from `crates/roko-cli/src/lib.rs` and fix all resulting dead-code warnings
by either removing unused code or adding targeted `#[allow(dead_code)]`
annotations.

## Files to Edit

- `crates/roko-cli/src/lib.rs` (primary)
- Other files in `crates/roko-cli/src/` as needed for fixing warnings

## Steps

### 1. Remove the blanket allow

In `crates/roko-cli/src/lib.rs`, line 6, remove:

```rust
#![allow(dead_code, unused_imports, unused_variables)]
```

### 2. Build and collect warnings

```bash
cargo check -p roko-cli 2>&1 | grep "warning\[dead_code\]\|warning\[unused"
```

### 3. Triage each warning

For each dead-code warning, decide:

a) **Remove** the code if it is truly unused and not part of a planned feature.
   This is the preferred action.

b) **Add `pub`** if the item is used by tests or external crates but not
   within the lib itself.

c) **Add targeted `#[allow(dead_code)]`** with a comment explaining why,
   if the code is part of a known WIP feature:

   ```rust
   #[allow(dead_code)] // TODO(O1_05): used once parallel dispatch is wired
   fn clone_services(&self) -> EffectServices { ... }
   ```

### 4. Fix unused imports

For `unused_imports` warnings:
- Remove the import if the symbol is not used
- If the import is used conditionally (behind `#[cfg]`), gate the import too

### 5. Fix unused variables

For `unused_variables` warnings:
- Prefix with `_` if the variable is intentionally unused:
  ```rust
  let _result = some_call();
  ```
- Remove the variable if it is truly unnecessary

### 6. Iterate until clean

Repeat steps 2-5 until `cargo check -p roko-cli` produces zero
`dead_code`, `unused_imports`, or `unused_variables` warnings.

## Verification

```bash
cargo check -p roko-cli 2>&1 | grep -c "warning\[dead_code\]"
# Should output: 0

cargo check -p roko-cli 2>&1 | grep -c "warning\[unused"
# Should output: 0

cargo test -p roko-cli --lib 2>/dev/null || true
```

Zero dead-code and unused warnings. Tests still pass.

## Constraints

- Do NOT add new blanket allows — only targeted per-item annotations.
- Prefer removal over suppression.
- Do NOT remove code that is called from tests — make it `pub(crate)` or
  gate the test behind `#[cfg(test)]`.
- Keep `#![allow(clippy::...)]` lines — those are handled in CD_02.
- This may produce a large diff — that is expected and acceptable.
