# CD_02: Remove blanket clippy suppression from roko-cli

## Goal

Remove the blanket `#![cfg_attr(clippy, allow(clippy::all, ...))]` from
`crates/roko-cli/src/lib.rs` and fix the resulting clippy warnings with
targeted fixes or per-item suppressions.

## Files to Edit

- `crates/roko-cli/src/lib.rs` (primary)
- Other files in `crates/roko-cli/src/` as needed for fixing warnings

## Steps

### 1. Remove the blanket clippy suppression

In `crates/roko-cli/src/lib.rs`, remove lines 14-23:

```rust
// REMOVE:
#![cfg_attr(
    clippy,
    allow(
        clippy::all,
        clippy::pedantic,
        clippy::nursery,
        clippy::restriction,
        missing_docs
    )
)]
```

Also remove line 12 if present:

```rust
// REMOVE:
#![allow(clippy::module_name_repetitions)]
```

Keep `#![allow(missing_docs)]` if desired (missing docs on every item in a
large CLI crate is noisy).

### 2. Run clippy and collect warnings

```bash
cargo clippy -p roko-cli --no-deps 2>&1 | head -100
```

### 3. Fix warnings in priority order

**High priority (fix the code):**
- `clippy::needless_pass_by_value` — change `String` params to `&str`
- `clippy::redundant_clone` — remove unnecessary `.clone()`
- `clippy::manual_let_else` — use `let ... else { return }` syntax
- `clippy::map_unwrap_or` — use `.map_or(default, f)`
- `clippy::single_match_else` — use `if let` instead

**Medium priority (fix or suppress):**
- `clippy::too_many_arguments` — add targeted `#[allow]` or refactor
- `clippy::too_many_lines` — add targeted `#[allow]`
- `clippy::module_name_repetitions` — add targeted `#[allow]` per item

**Low priority (suppress with comment):**
- `clippy::missing_errors_doc` — add `#[allow(clippy::missing_errors_doc)]`
- `clippy::missing_panics_doc` — add targeted `#[allow]`

### 4. Add workspace-level lint configuration

The workspace already has `[lints]` in `crates/roko-cli/Cargo.toml`:

```toml
[lints]
workspace = true
```

Verify the workspace lint config in the root `Cargo.toml` applies appropriate
defaults.

### 5. Iterate until clean

```bash
cargo clippy -p roko-cli --no-deps -- -D warnings
```

Repeat until this produces zero warnings.

## Verification

```bash
cargo clippy -p roko-cli --no-deps -- -D warnings
cargo test -p roko-cli --lib 2>/dev/null || true
```

Zero clippy warnings with `-D warnings`. Tests still pass.

## Constraints

- Fix the code rather than suppressing where practical.
- Use per-item `#[allow(clippy::...)]` with a brief comment for suppressions.
- Do NOT add blanket `#![allow(...)]` back — that defeats the purpose.
- Keep `#![allow(missing_docs)]` if the crate has hundreds of public items.
- This task depends on CD_01 completing first (dead code must be removed
  before clippy can give meaningful warnings).
- This may produce a very large diff — that is expected.
