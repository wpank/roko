# CD_03: Delete dispatch_direct.rs (consolidate into dispatch_v2)

## Goal

Remove `dispatch_direct.rs` entirely once D1_11 has confirmed it is fully
gated behind the `legacy-orchestrate` feature flag and no default builds
use it.

## Files to Edit

- `crates/roko-cli/src/dispatch_direct.rs` (DELETE)
- `crates/roko-cli/src/lib.rs` (remove module declaration)

## Also Read

- `crates/roko-cli/Cargo.toml` — feature flags

## Steps

### 1. Verify no default build uses dispatch_direct

```bash
# Build without legacy feature — must succeed:
cargo check -p roko-cli

# Verify no non-gated imports exist:
grep -rn 'dispatch_direct' crates/roko-cli/src/ --include='*.rs' | \
  grep -v '#\[cfg' | grep -v '//\|/\*' | grep -v 'target/'
```

If any ungated imports are found, gate them with
`#[cfg(feature = "legacy-orchestrate")]` before proceeding.

### 2. Remove the module declaration

In `crates/roko-cli/src/lib.rs`, remove:

```rust
#[cfg(feature = "legacy-orchestrate")]
pub mod dispatch_direct;
```

### 3. Delete the file

```bash
rm crates/roko-cli/src/dispatch_direct.rs
```

### 4. Update callers to use dispatch_v2

Any remaining callers that import `DispatchResult` or `ToolOutput` from
`dispatch_direct` should import from `dispatch_v2` instead:

```rust
// Before:
use crate::dispatch_direct::{DispatchResult, ToolOutput};
// After:
use crate::dispatch_v2::{DispatchResult, ToolOutput};
```

These callers should already be gated behind `#[cfg(feature = "legacy-orchestrate")]`.
If they are, they can be updated or removed too.

### 5. Clean up the feature flag

If `legacy-orchestrate` is no longer needed by any code after removing
`dispatch_direct`, remove the feature flag from `Cargo.toml`:

```toml
# Remove if no longer used:
[features]
legacy-orchestrate = []
```

Search first:
```bash
grep -rn 'legacy-orchestrate' crates/roko-cli/ --include='*.rs' --include='*.toml'
```

Only remove the feature if zero Rust files reference it.

### 6. Verify the build

```bash
cargo check -p roko-cli
cargo check --workspace
```

## Verification

```bash
cargo check --workspace
test -f crates/roko-cli/src/dispatch_direct.rs && echo "FAIL: file still exists" || echo "OK: file deleted"
```

Workspace builds successfully. The file does not exist.

## Constraints

- Only proceed if D1_11 is complete (the feature gate is in place).
- If any ungated callers exist, gate them first — do NOT delete the file
  while it would break the default build.
- The `legacy-orchestrate` feature may still be used by other modules —
  check before removing the feature flag.
