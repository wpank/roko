# D1_01: Load CascadeRouter at CLI startup

## Goal

Add a function to `model_selection.rs` that loads or creates a `CascadeRouter`
from the `.roko/learn/cascade-router.json` file, so it can be passed into
`resolve_effective_model` and `ModelCallService` from all CLI entry points.

## Files to Edit

- `crates/roko-cli/src/model_selection.rs` (primary)

## Also Read

- `crates/roko-learn/src/cascade_router.rs` — `CascadeRouter::load_or_new`

## Steps

### 1. Add a `load_cascade_router` function

In `crates/roko-cli/src/model_selection.rs`, add:

```rust
use std::sync::Arc;

/// Load or create a `CascadeRouter` from the standard `.roko/learn/` path.
///
/// The router is loaded from `<workdir>/.roko/learn/cascade-router.json`.
/// If the file does not exist, a new router is created with model slugs
/// derived from the workspace config.
pub fn load_cascade_router(
    workdir: &Path,
    config: &RokoConfig,
) -> Arc<CascadeRouter> {
    let roko_dir = workdir.join(".roko");
    let router_path = roko_dir.join("learn").join("cascade-router.json");

    let default_model = config.agent.default_model.trim();
    let mut model_slugs = Vec::new();

    if !default_model.is_empty() {
        let resolved = resolve_model(config, default_model);
        if !resolved.slug.trim().is_empty() {
            model_slugs.push(resolved.slug);
        }
    }

    if let Some(fallback) = config.agent.fallback_model.as_deref() {
        let resolved = resolve_model(config, fallback);
        if !resolved.slug.trim().is_empty() && !model_slugs.contains(&resolved.slug) {
            model_slugs.push(resolved.slug);
        }
    }

    for tier_model in config.agent.tier_models.values() {
        let resolved = resolve_model(config, tier_model);
        if !resolved.slug.trim().is_empty() && !model_slugs.contains(&resolved.slug) {
            model_slugs.push(resolved.slug);
        }
    }

    Arc::new(CascadeRouter::load_or_new(&router_path, model_slugs))
}
```

### 2. Export the function

Ensure `load_cascade_router` is `pub` and importable from `crate::model_selection`.

### 3. Add `resolve_model` import if missing

The `resolve_model` function from `roko_core::agent` should already be imported.
Verify the import exists:

```rust
use roko_core::agent::resolve_model;
```

### 4. Verify the CascadeRouter import exists

```rust
use roko_learn::cascade_router::CascadeRouter;
```

This should already be present in the imports.

## Verification

```bash
cargo check -p roko-cli
```

Must succeed with no errors. This is a pure function addition with no callers
yet — callers are added in D1_04 and D1_05.

## Constraints

- Do NOT change `resolve_effective_model` or `resolve_effective_model_key` —
  those are updated in D1_02.
- Do NOT modify the `select_candidate` function — the cascade router parameter
  it already accepts will be threaded in D1_02.
- The function returns `Arc<CascadeRouter>` because the router is shared
  between model selection and ModelCallService.
