# D1_02: Thread CascadeRouter through resolve_effective_model_key

## Goal

Update `resolve_effective_model_key` to accept an optional `CascadeRouter`
and pass it through to `resolve_effective_model` -> `select_candidate`.

## Files to Edit

- `crates/roko-cli/src/model_selection.rs` (primary)

## Steps

### 1. Update `resolve_effective_model_key` signature

Change the function signature to accept an optional cascade router:

```rust
pub fn resolve_effective_model_key(
    workdir: &Path,
    cli_model: Option<String>,
    role: Option<&str>,
    context: &str,
    cascade_router: Option<&CascadeRouter>,
) -> anyhow::Result<String> {
    let config = load_roko_config(workdir)?;
    let selection = resolve_effective_model(
        cli_model,
        None,
        role.map(str::to_string),
        cascade_router,
        &config,
    )
    .map_err(|err| anyhow::anyhow!("resolve model selection for {context}: {err}"))?;
    selection.print_stderr();
    Ok(selection.effective_model_key)
}
```

### 2. Update `resolve_effective_model` to pass the router

The function `resolve_effective_model` already accepts
`cascade_router: Option<&CascadeRouter>` and passes it to `select_candidate`.
Verify this is the case. If the parameter is currently hardcoded to `None`,
thread it through.

### 3. Find all callers of `resolve_effective_model_key` and update them

Search for calls to `resolve_effective_model_key`:

```bash
grep -rn 'resolve_effective_model_key' crates/roko-cli/src/ --include='*.rs'
```

For each caller, add `None` as the cascade_router parameter (or pass a loaded
router if one is available). Common callers:

- `run.rs` — will be updated to pass a loaded router in D1_04
- `chat_inline.rs` — will be updated in D1_05
- Any other callers should pass `None` for now

### 4. Update test callers if any

If there are tests calling `resolve_effective_model_key`, add `None` for the
new parameter.

## Verification

```bash
cargo check -p roko-cli
cargo test -p roko-cli --lib 2>/dev/null || true
```

`cargo check` must succeed. Tests may need the `None` parameter added.

## Constraints

- This is a signature change only — no new behavior is added.
- All existing callers should compile with `None` for the new parameter.
- Do NOT load a CascadeRouter in this function — callers provide it.
