# D1_05: Wire CascadeRouter into roko chat

## Goal

Load a `CascadeRouter` at `roko chat` startup and pass it into model
selection so chat sessions benefit from learned model routing.

## Files to Edit

- `crates/roko-cli/src/chat_inline.rs` (primary)

## Also Read

- `crates/roko-cli/src/model_selection.rs` — `load_cascade_router`
- `crates/roko-agent/src/model_call_service.rs` — `with_cascade_router`

## Steps

### 1. Import `load_cascade_router`

Add to the imports at the top of `chat_inline.rs`:

```rust
use crate::model_selection::load_cascade_router;
```

### 2. Find the session initialization

Locate where `ChatAgentSession` or the dispatch backend is created. This is
typically in the main `run_chat_inline` or `chat_inline_main` function.

### 3. Load the CascadeRouter

Before the session is created, load the router:

```rust
let config = load_roko_config(workdir)?;
let cascade_router = load_cascade_router(workdir, &config);
```

### 4. Thread into model selection

If model selection happens via `resolve_effective_model_key`, pass the router:

```rust
let model_key = resolve_effective_model_key(
    workdir,
    cli_model,
    None,
    "roko chat",
    Some(&cascade_router),
)?;
```

### 5. Thread into ModelCallService if available

If the chat path constructs a `ModelCallService` directly, attach the router:

```rust
let model_call_service = ModelCallService::new(model.clone())
    .with_cascade_router(Arc::clone(&cascade_router))
    // ... other configuration
;
```

### 6. Persist on exit

At the end of the chat session (when the user exits), save the router state
so observations are preserved:

```rust
// After chat loop exits:
if let Err(err) = cascade_router.save(&router_path) {
    tracing::warn!("failed to save cascade router: {err}");
}
```

If `CascadeRouter` has an `autosave` or the `FeedbackService` handles
persistence, this step may already be covered.

## Verification

```bash
cargo check -p roko-cli
```

Must succeed. The chat session should start and function identically to
before — the router merely influences model selection when multiple models
are configured.

## Constraints

- Do NOT change the chat session protocol or streaming behavior.
- Router persistence is best-effort — warn on failure, do not abort.
- If `load_cascade_router` is not yet available (D1_01 not merged), add a
  TODO and pass `None` for the router parameter.
