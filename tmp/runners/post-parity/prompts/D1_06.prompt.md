# D1_06: Add Episode Emission to ModelCallService

## Goal

After each model call in `ModelCallService`, emit a structured episode event
via the event consumer system so all dispatch paths (workflow, chat, ACP)
produce consistent episode records.

## Files to Edit

- `crates/roko-agent/src/model_call_service.rs` (primary)

## Also Read

- `crates/roko-learn/src/episode_logger.rs` — `Episode` struct

## Steps

### 1. Define a `ModelCallEpisode` event struct

In `model_call_service.rs`, add a struct for the episode data emitted after
each call:

```rust
/// Episode data emitted after each model call for durable recording.
#[derive(Debug, Clone, serde::Serialize)]
pub struct ModelCallEpisode {
    /// Request id for correlation.
    pub request_id: Option<String>,
    /// Model slug used for the call.
    pub model: String,
    /// Role that initiated the call.
    pub role: String,
    /// Caller surface (cli, serve, etc.).
    pub caller: Option<String>,
    /// Workflow run id.
    pub run_id: Option<String>,
    /// Input tokens consumed.
    pub input_tokens: u64,
    /// Output tokens produced.
    pub output_tokens: u64,
    /// Total cost in USD.
    pub cost_usd: f64,
    /// Wall-clock latency in milliseconds.
    pub latency_ms: u64,
    /// Whether the call succeeded.
    pub success: bool,
    /// Prompt section ids used.
    pub prompt_section_ids: Vec<String>,
    /// Knowledge entry ids used.
    pub knowledge_ids: Vec<String>,
}
```

### 2. Emit the episode after each call

In the `ModelCaller::call` implementation, after the response is processed
(or on error), construct and emit the episode:

```rust
let episode = ModelCallEpisode {
    request_id: response.request_id.clone(),
    model: response.model.clone(),
    role: req.role.clone().unwrap_or_default(),
    caller: req.caller.clone(),
    run_id: req.run_id.clone(),
    input_tokens: response.usage.input_tokens,
    output_tokens: response.usage.output_tokens,
    cost_usd: response.usage.cost_usd,
    latency_ms,
    success: true,
    prompt_section_ids: req.prompt_section_ids.clone(),
    knowledge_ids: req.knowledge_ids.clone(),
};

// Emit via RuntimeEvent for event consumers
self.emit(RuntimeEvent::FeedbackRecorded {
    run_id: req.run_id.clone().unwrap_or_default(),
    kind: "model_call_episode".to_string(),
    summary: serde_json::to_string(&episode).unwrap_or_default(),
});
```

### 3. Handle failure episodes

On the error path, emit a failure episode:

```rust
let episode = ModelCallEpisode {
    request_id: None,
    model: effective_model.clone(),
    role: req.role.clone().unwrap_or_default(),
    caller: req.caller.clone(),
    run_id: req.run_id.clone(),
    input_tokens: 0,
    output_tokens: 0,
    cost_usd: 0.0,
    latency_ms,
    success: false,
    prompt_section_ids: req.prompt_section_ids.clone(),
    knowledge_ids: req.knowledge_ids.clone(),
};
```

### 4. Add an `emit` helper if not present

If `ModelCallService` does not have an `emit` method for `RuntimeEvent`,
add one:

```rust
fn emit(&self, event: RuntimeEvent) {
    for consumer in &self.event_consumers {
        consumer.consume(&event);
    }
}
```

## Verification

```bash
cargo check -p roko-agent
cargo test -p roko-agent --lib
```

Both must succeed. Existing tests pass because they either have no event
consumers or the consumers accept any event type.

## Constraints

- `ModelCallEpisode` is a local struct — do NOT add it to `roko-core`.
- Episode emission is fire-and-forget — do NOT await persistence.
- Serialization failure (serde_json) should produce an empty string, not panic.
