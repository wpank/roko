# D1_07: Wire Episode Logger to One-Shot CLI Paths

## Goal

Ensure `roko run` (one-shot) paths emit episode records through the
`ModelCallService` episode emission added in D1_06, by verifying the event
consumer chain is connected.

## Files to Edit

- `crates/roko-cli/src/run_inline.rs` (primary)

## Also Read

- `crates/roko-cli/src/run.rs` — `run_once`, `run_workflow_engine_report`
- `crates/roko-agent/src/model_call_service.rs` — episode emission

## Steps

### 1. Verify the event consumer chain in `run.rs`

In `crates/roko-cli/src/run.rs`, examine the workflow engine path
(`run_workflow_engine_with_services`). It already adds a `JsonlWorkflowConsumer`
via `engine.add_consumer(consumer)`. This consumer logs RuntimeEvents to the
JSONL file, which includes `FeedbackRecorded` events from the `ModelCallService`.

Verify this chain is intact:
1. `WorkflowEngine` receives `EventConsumer` instances
2. The engine forwards events to the `EffectDriver`
3. `EffectDriver::spawn_agent` calls `ModelCallService::call`
4. `ModelCallService::call` emits `RuntimeEvent::FeedbackRecorded` episodes
5. The `JsonlLogger` consumer writes them to `.roko/events.jsonl`

### 2. Verify the legacy `run_once` path

In `crates/roko-cli/src/run.rs`, the `run_once` function uses the old dispatch
path (Claude CLI agent, not `ModelCallService`). This path already records
episodes via `EpisodeLogger` directly. Verify no double-emission occurs.

If `run_once` constructs a `ModelCallService` internally, ensure it has an
event consumer attached:

```rust
let logger = JsonlLogger::from_roko_dir(&roko_dir);
let model_call_service = ModelCallService::new(model.clone())
    .with_event_consumer(Arc::new(logger))
    // ...
;
```

### 3. Wire the `run_inline.rs` path

In `crates/roko-cli/src/run_inline.rs`, the `run_once_inline` function wraps
`run_once`. Since `run_once` handles its own episode recording, no additional
wiring is needed in `run_inline.rs`.

However, verify that the inline path does NOT suppress events. Check that
the `external_hub` parameter is threaded through.

### 4. Add a trace log for episode emission

In the one-shot path, add a `tracing::debug!` to confirm episodes are emitted:

```rust
tracing::debug!(
    run_id = %run_id,
    "one-shot run complete; episode events emitted via event consumer chain"
);
```

## Verification

```bash
cargo check -p roko-cli
```

Must succeed. This is primarily a verification/wiring task — minimal code
changes expected.

## Constraints

- Do NOT duplicate episode recording — the `ModelCallService` handles it.
- Do NOT add new `EpisodeLogger` construction — use the event consumer chain.
- The `run_once` legacy path may still use `EpisodeLogger` directly; do NOT
  remove it until the legacy path is fully migrated.
