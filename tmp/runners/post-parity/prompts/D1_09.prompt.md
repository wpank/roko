# D1_09: Migrate chat_inline to ModelCallService

## Goal

Replace the direct provider dispatch in `chat_inline.rs` with calls through
`ModelCallService`, unifying the dispatch path with `roko run` and the
workflow engine.

## Files to Edit

- `crates/roko-cli/src/chat_inline.rs` (primary)

## Also Read

- `crates/roko-agent/src/model_call_service.rs` — `ModelCallService::call`
- `crates/roko-cli/src/dispatch_v2.rs` — current dispatch abstraction

## Steps

### 1. Identify the current dispatch call site

In `chat_inline.rs`, find where agent dispatch happens. This is typically in
the streaming loop or the `send_message` / `dispatch_turn` function. It may
call `dispatch_v2::dispatch_prompt` or use `ChatAgentSession`.

### 2. Replace with `ModelCallService::call`

If the chat path currently calls the provider directly, replace it with:

```rust
use roko_core::foundation::{ModelCallRequest, ChatMessage, MessageRole};

let request = ModelCallRequest {
    model: model_slug.clone(),
    system: Some(system_prompt.clone()),
    messages: vec![ChatMessage {
        role: MessageRole::User,
        content: user_input.clone(),
    }],
    max_tokens: Some(4096),
    temperature: Some(0.3),
    role: None,
    caller: Some("chat".to_string()),
    run_id: Some(session_id.clone()),
    ..Default::default()
};

let response = model_call_service.call(request).await?;
```

### 3. Construct `ModelCallService` at session start

If `ModelCallService` is not already available in the chat session, construct
it at initialization:

```rust
use roko_agent::ModelCallService;
use roko_orchestrator::{ServiceConfig, ServiceFactory};

let config = load_roko_config(workdir)?;
let service_config = ServiceConfig::production(workdir, config.clone());
let bundle = ServiceFactory::build(service_config)?;
let model_call_service = bundle.model_call_service;
```

### 4. Preserve streaming behavior

If the chat path uses streaming (SSE/WebSocket tokens), the `ModelCallService`
may not support streaming yet. In that case:

- Keep the streaming dispatch path as-is for now
- Add a TODO: "migrate to ModelCallService streaming once available"
- Use `ModelCallService` for the non-streaming fallback path only

### 5. Remove dead dispatch_direct imports

If `chat_inline.rs` imports from `dispatch_direct`, remove those imports
since `dispatch_direct` is deprecated:

```rust
// Remove:
#[cfg(feature = "legacy-orchestrate")]
use crate::dispatch_direct;
```

Replace with imports from `dispatch_v2` or `ModelCallService`.

## Verification

```bash
cargo check -p roko-cli
```

Must succeed. The chat experience should be identical — same model, same
streaming, same cost tracking.

## Constraints

- Do NOT break streaming — if `ModelCallService` does not support streaming,
  keep the current streaming path and only migrate the non-streaming fallback.
- Do NOT change the user-facing chat UX (prompts, formatting, shortcuts).
- The `ChatAgentSession` abstraction may already use `ModelCallService` — check
  before duplicating.
