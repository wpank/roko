# D1_10: Migrate ACP Runner to Provider System

## Goal

Replace the direct `ClaudeCliAgent` construction in the ACP runner with the
`ServiceFactory` / `ModelCallService` path so ACP sessions use the same
provider infrastructure as `roko run`.

## Files to Edit

- `crates/roko-acp/src/runner.rs` (primary)

## Also Read

- `crates/roko-orchestrator/src/service_factory.rs` — `ServiceFactory::build`
- `crates/roko-agent/src/model_call_service.rs` — `ModelCallService`

## Steps

### 1. Identify the current agent construction

In `crates/roko-acp/src/runner.rs`, find where `ClaudeCliAgent` is constructed
directly. This is typically in the `run_pipeline` or `execute_task` function.

### 2. Replace with ServiceFactory

Instead of constructing `ClaudeCliAgent` directly, use `ServiceFactory`:

```rust
use roko_orchestrator::{ServiceConfig, ServiceFactory};

let service_config = ServiceConfig::production(&workdir, workspace_config.clone());
let bundle = ServiceFactory::build(service_config)?;
```

### 3. Use the WorkflowEngine for pipeline execution

The ACP runner already has a `run_workflow_engine` path (around line 31 in the
imports). Verify it is used:

```rust
use roko_runtime::workflow_engine::{WorkflowEngine, WorkflowRunConfig, WorkflowRunReport};
```

If the runner has a fallback path that constructs `ClaudeCliAgent` directly,
remove it in favor of the `WorkflowEngine` path.

### 4. Thread the `ModelCallService` through

If ACP-specific operations (like direct message dispatch) need `ModelCallService`,
extract it from the bundle:

```rust
let model_call_service = bundle.model_call_service;
```

### 5. Remove direct ClaudeCliAgent imports if no longer needed

If all dispatch now goes through the service bundle, remove:

```rust
// Remove if unused:
use roko_agent::ClaudeCliAgent;
use roko_agent::claude_cli_agent::build_settings_json;
```

Keep them if they are used for plan execution (separate from ACP sessions).

### 6. Update the PipelineConfig handling

The ACP runner has a `PipelineConfig` struct. Ensure its `template` field
maps correctly to `WorkflowConfig`:

```rust
let workflow = match config.template {
    WorkflowTemplate::Express => WorkflowConfig::express(),
    WorkflowTemplate::Standard => WorkflowConfig::standard(),
    WorkflowTemplate::Full => WorkflowConfig::full(),
};
```

## Verification

```bash
cargo check -p roko-acp
cargo test -p roko-acp --lib 2>/dev/null || true
```

`cargo check` must succeed.

## Constraints

- Do NOT change the ACP protocol or session semantics — only the dispatch path.
- Keep the `AgentContract` / safety layer integration intact.
- If the runner uses `ClaudeCliAgent` for tool execution (not model calls),
  keep that path — only model calls should go through `ModelCallService`.
