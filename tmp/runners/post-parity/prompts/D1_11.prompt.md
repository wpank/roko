# D1_11: Gate dispatch_direct.rs behind feature flag

## Goal

Ensure `dispatch_direct.rs` is fully gated behind the `legacy-orchestrate`
feature flag so it can be cleanly removed in CD_03.

## Files to Edit

- `crates/roko-cli/src/dispatch_direct.rs` (primary)

## Also Read

- `crates/roko-cli/src/lib.rs` — module declarations

## Steps

### 1. Verify the module-level gate

In `crates/roko-cli/src/dispatch_direct.rs`, the file already starts with:

```rust
#![cfg(feature = "legacy-orchestrate")]
```

Verify this gate is present. If it uses `#[cfg(...)]` instead of `#![cfg(...)]`,
that is fine — both work for gating the entire module.

### 2. Gate the module declaration in lib.rs

In `crates/roko-cli/src/lib.rs`, find where `dispatch_direct` is declared as
a module. Ensure it is gated:

```rust
#[cfg(feature = "legacy-orchestrate")]
pub mod dispatch_direct;
```

If it is declared without `#[cfg]`, add the gate.

### 3. Gate all imports of dispatch_direct

Search the codebase for imports from `dispatch_direct`:

```bash
grep -rn 'dispatch_direct' crates/roko-cli/src/ --include='*.rs'
```

Every import must be gated:

```rust
#[cfg(feature = "legacy-orchestrate")]
use crate::dispatch_direct;
```

Common locations:
- `chat_inline.rs` — already has `#[cfg(feature = "legacy-orchestrate")]`
- `run.rs` — check for direct imports
- `commands/` — check subcommand files

### 4. Gate type re-exports

If `dispatch_direct` re-exports types (`DispatchResult`, `ToolOutput`), ensure
the re-exports are gated and callers that need these types import from
`dispatch_v2` instead:

```rust
// In dispatch_direct.rs (already present):
pub use crate::dispatch_v2::{DispatchResult, ToolOutput};

// Callers should import directly from dispatch_v2:
use crate::dispatch_v2::{DispatchResult, ToolOutput};
```

### 5. Verify the feature flag exists in Cargo.toml

In `crates/roko-cli/Cargo.toml`, verify:

```toml
[features]
legacy-orchestrate = []
```

If this feature is not defined, add it. Check that nothing in the workspace
enables it by default.

## Verification

```bash
# Without the feature flag — must compile:
cargo check -p roko-cli

# With the feature flag — must also compile:
cargo check -p roko-cli --features legacy-orchestrate
```

Both must succeed.

## Constraints

- Do NOT delete `dispatch_direct.rs` — that is CD_03.
- Do NOT change any runtime behavior — only add compile-time gates.
- All callers must compile both with and without the feature flag.
