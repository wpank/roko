# D1_12: Remove hardcoded model strings from auth_detect.rs

## Goal

Replace hardcoded model name strings in `auth_detect.rs` (like
`"claude-sonnet-4-6"`, `"gpt-5.4-mini"`) with references to a central
default model constant or config-driven resolution.

## Files to Edit

- `crates/roko-cli/src/auth_detect.rs` (primary)

## Also Read

- `crates/roko-core/src/config/schema.rs` — default model constants

## Steps

### 1. Identify hardcoded model strings

In `crates/roko-cli/src/auth_detect.rs`, find all hardcoded model name strings.
Based on the file header, these appear in the `AuthMethod` enum variants and
their `label()` method:

```rust
// Line 20-21: "claude-haiku-4-5" in comment
// Line 27: "gpt-5.4-mini", "glm-5.1" in comment
// Line 42: "claude-sonnet-4-6" in label()
// Line 49: "gpt-5.4-mini" in fallback comment
```

### 2. Define default model constants

At the top of `auth_detect.rs`, add constants:

```rust
/// Default Anthropic model used when no override is configured.
const DEFAULT_ANTHROPIC_MODEL: &str = "claude-sonnet-4-6";
/// Default OpenAI-compatible model used when no override is configured.
const DEFAULT_OPENAI_COMPAT_MODEL: &str = "gpt-4o-mini";
```

Or better, import from a central location if one exists:

```rust
use roko_core::config::defaults::{DEFAULT_ANTHROPIC_MODEL, DEFAULT_OPENAI_COMPAT_MODEL};
```

### 3. Replace hardcoded strings in `label()`

In the `AuthMethod::label()` method:

```rust
impl AuthMethod {
    pub fn label(&self) -> String {
        match self {
            Self::ClaudeCli => "claude CLI".to_string(),
            Self::AnthropicApi { model, .. } => {
                if let Some(m) = model {
                    format!("{m} (Anthropic API)")
                } else {
                    format!("{DEFAULT_ANTHROPIC_MODEL} (Anthropic API)")
                }
            }
            Self::OpenAiCompat { model, .. } => {
                if let Some(m) = model {
                    format!("{m} (OpenAI-compat)")
                } else {
                    format!("{DEFAULT_OPENAI_COMPAT_MODEL} (OpenAI-compat)")
                }
            }
            Self::NeedsSetup => "needs setup".to_string(),
        }
    }
}
```

### 4. Replace hardcoded strings in detection logic

If the detection functions (like `detect_auth`) use hardcoded model strings
as fallbacks, replace them:

```rust
// Before:
model: Some("gpt-5.4-mini".to_string()),
// After:
model: Some(DEFAULT_OPENAI_COMPAT_MODEL.to_string()),
```

### 5. Update doc comments

Replace any hardcoded model names in doc comments with generic descriptions:

```rust
/// Model override (e.g., "claude-haiku-4-5"). Defaults to the configured default.
```

## Verification

```bash
cargo check -p roko-cli
cargo test -p roko-cli --lib 2>/dev/null || true
```

`cargo check` must succeed.

## Constraints

- Do NOT change the authentication detection logic — only string literals.
- Constants are preferred over config-driven resolution here because
  `auth_detect` runs before config is loaded.
- Keep the `AuthMethod` enum variants unchanged — only update string values.
