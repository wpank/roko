# D1_14: Wire Health Checks into ModelCallService

## Goal

Add a `ProviderHealthTracker` to `ModelCallService` and consult it before
routing model calls. Record outcomes after each call.

## Files to Edit

- `crates/roko-agent/src/model_call_service.rs` (primary)

## Also Read

- `crates/roko-agent/src/provider/health.rs` — `ProviderHealthTracker`

## Steps

### 1. Add the tracker field

In the `ModelCallService` struct, add:

```rust
    /// Provider health tracker for routing decisions.
    health_tracker: Arc<ProviderHealthTracker>,
```

Import:

```rust
use crate::provider::health::ProviderHealthTracker;
```

### 2. Initialize in the constructor

In `ModelCallService::new()`, initialize:

```rust
health_tracker: Arc::new(ProviderHealthTracker::new()),
```

### 3. Add builder method

```rust
    /// Attach a shared health tracker.
    pub fn with_health_tracker(mut self, tracker: Arc<ProviderHealthTracker>) -> Self {
        self.health_tracker = tracker;
        self
    }
```

### 4. Check health before dispatch

In the `ModelCaller::call` implementation, before dispatching to the provider,
check health:

```rust
let provider_key = self.provider_key_for_model(&effective_model);
if !self.health_tracker.is_healthy(&provider_key) {
    tracing::warn!(
        provider = %provider_key,
        model = %effective_model,
        "provider unhealthy, attempting fallback"
    );
    // Try fallback models
    if let Some(fallback) = self.next_healthy_fallback(&provider_key) {
        effective_model = fallback;
    }
}
```

### 5. Record outcomes after dispatch

After the model call completes (success or failure), record the outcome:

```rust
let provider_key = self.provider_key_for_model(&response.model);
self.health_tracker.record(&provider_key, true, latency_ms);
```

On failure:

```rust
self.health_tracker.record(&provider_key, false, latency_ms);
```

### 6. Add helper methods

```rust
    /// Derive a provider key from a model slug.
    fn provider_key_for_model(&self, model: &str) -> String {
        // Simple heuristic: extract provider from model slug prefix
        if model.starts_with("claude") {
            "anthropic".to_string()
        } else if model.starts_with("gpt") || model.starts_with("o1") || model.starts_with("o3") {
            "openai".to_string()
        } else if model.starts_with("gemini") {
            "google".to_string()
        } else {
            "unknown".to_string()
        }
    }

    /// Find the next healthy fallback model.
    fn next_healthy_fallback(&self, unhealthy_provider: &str) -> Option<String> {
        for fallback in &self.fallback_models {
            let provider = self.provider_key_for_model(fallback);
            if provider != unhealthy_provider && self.health_tracker.is_healthy(&provider) {
                return Some(fallback.clone());
            }
        }
        None
    }
```

## Verification

```bash
cargo check -p roko-agent
cargo test -p roko-agent --lib
```

Both must succeed. Existing tests pass because the default health tracker
reports all providers as healthy.

## Constraints

- Health checking is best-effort — if no healthy fallback exists, use the
  original model anyway (the provider may have recovered).
- Do NOT change the `ModelCaller` trait signature.
- The health tracker is `Arc`-shared so multiple `ModelCallService` instances
  can share state.
