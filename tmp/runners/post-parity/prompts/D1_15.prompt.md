# D1_15: Add Retry Logic to ModelCallService

## Goal

Add automatic retry with exponential backoff for transient failures in
`ModelCallService`. Uses the health tracker to decide when to retry vs.
fail fast.

## Files to Edit

- `crates/roko-agent/src/model_call_service.rs` (primary)

## Also Read

- `crates/roko-agent/src/provider/health.rs` — `ProviderHealthTracker`

## Steps

### 1. Add retry configuration

At the top of `model_call_service.rs`, add constants:

```rust
/// Maximum retry attempts for transient failures.
const MAX_RETRIES: u32 = 2;
/// Initial backoff delay in milliseconds.
const INITIAL_BACKOFF_MS: u64 = 500;
/// Backoff multiplier per retry.
const BACKOFF_MULTIPLIER: u64 = 2;
```

### 2. Wrap the dispatch call in a retry loop

In the `ModelCaller::call` implementation, wrap the provider dispatch in a
retry loop:

```rust
let mut last_error = None;
let mut backoff_ms = INITIAL_BACKOFF_MS;

for attempt in 0..=MAX_RETRIES {
    if attempt > 0 {
        tracing::debug!(
            attempt,
            backoff_ms,
            model = %effective_model,
            "retrying model call after transient failure"
        );
        tokio::time::sleep(Duration::from_millis(backoff_ms)).await;
        backoff_ms *= BACKOFF_MULTIPLIER;
    }

    match self.dispatch_to_provider(&effective_model, &req).await {
        Ok(response) => {
            self.health_tracker.record(&provider_key, true, latency_ms);
            return Ok(response);
        }
        Err(err) if is_transient_error(&err) && attempt < MAX_RETRIES => {
            tracing::warn!(
                attempt,
                error = %err,
                model = %effective_model,
                "transient model call failure, will retry"
            );
            self.health_tracker.record(&provider_key, false, 0);
            last_error = Some(err);
            continue;
        }
        Err(err) => {
            self.health_tracker.record(&provider_key, false, latency_ms);
            return Err(err);
        }
    }
}

Err(last_error.unwrap_or_else(|| {
    RokoError::invalid("model call failed after retries")
}))
```

### 3. Add `is_transient_error` helper

```rust
/// Classify whether an error is transient (worth retrying).
fn is_transient_error(err: &RokoError) -> bool {
    let msg = err.to_string().to_lowercase();
    msg.contains("timeout")
        || msg.contains("rate limit")
        || msg.contains("429")
        || msg.contains("503")
        || msg.contains("502")
        || msg.contains("connection")
        || msg.contains("temporarily unavailable")
}
```

### 4. Ensure non-transient errors fail fast

Errors like "invalid API key", "model not found", and budget exceeded should
NOT be retried:

```rust
fn is_permanent_error(err: &RokoError) -> bool {
    let msg = err.to_string().to_lowercase();
    msg.contains("invalid api key")
        || msg.contains("authentication")
        || msg.contains("401")
        || msg.contains("403")
        || msg.contains("budget exceeded")
        || msg.contains("model not found")
}
```

### 5. Add retry count to episode data

If the model call episode struct (from D1_06) exists, add a `retries` field:

```rust
pub retries: u32,
```

## Verification

```bash
cargo check -p roko-agent
cargo test -p roko-agent --lib
```

Both must succeed. Existing tests pass because they do not trigger transient
errors (mock backends return immediately).

## Constraints

- Maximum 2 retries (3 total attempts) — do NOT make this unbounded.
- Backoff is exponential: 500ms, 1000ms.
- Do NOT retry budget-exceeded or authentication errors.
- `tokio::time::sleep` is used for backoff — do NOT busy-wait.
- Total retry time is bounded: 500ms + 1000ms = 1.5s maximum added latency.
