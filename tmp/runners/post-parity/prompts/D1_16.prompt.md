# D1_16: Add Budget Configuration to roko.toml

## Goal

Extend `BudgetConfig` with per-session and per-run budget fields so users can
control spend limits through `roko.toml`.

## Files to Edit

- `crates/roko-core/src/config/budget.rs` (primary)

## Also Read

- `crates/roko-core/src/config/schema.rs` — `RokoConfig.budget`

## Steps

### 1. Add new fields to `BudgetConfig`

In `crates/roko-core/src/config/budget.rs`, extend the `BudgetConfig` struct:

```rust
#[allow(clippy::derive_partial_eq_without_eq)]
#[derive(Clone, Debug, PartialEq, Serialize, Deserialize)]
pub struct BudgetConfig {
    /// Max dollars to spend per plan.
    #[serde(default = "default_max_plan_usd")]
    pub max_plan_usd: f32,
    /// Max dollars per single agent turn.
    #[serde(default = "default_max_turn_usd")]
    pub max_turn_usd: f32,
    /// Token budget for prompt composition.
    #[serde(default = "default_prompt_token_budget")]
    pub prompt_token_budget: usize,
    /// Max dollars per chat session (0.0 = unlimited).
    #[serde(default)]
    pub max_session_usd: f32,
    /// Max total tokens per single model call.
    #[serde(default)]
    pub max_tokens_per_call: Option<u32>,
    /// Whether to emit warnings when approaching budget limits.
    #[serde(default = "default_budget_warnings")]
    pub warn_on_threshold: bool,
    /// Warning threshold as a fraction of the budget (0.0 to 1.0).
    /// Warns when spend exceeds this fraction of the limit.
    #[serde(default = "default_warn_threshold")]
    pub warn_threshold: f32,
}
```

### 2. Add default functions

```rust
const fn default_budget_warnings() -> bool {
    true
}

const fn default_warn_threshold() -> f32 {
    0.8
}
```

### 3. Update the `Default` implementation

```rust
impl Default for BudgetConfig {
    fn default() -> Self {
        Self {
            max_plan_usd: default_max_plan_usd(),
            max_turn_usd: default_max_turn_usd(),
            prompt_token_budget: default_prompt_token_budget(),
            max_session_usd: 0.0,
            max_tokens_per_call: None,
            warn_on_threshold: default_budget_warnings(),
            warn_threshold: default_warn_threshold(),
        }
    }
}
```

### 4. Update the example generation

In `crates/roko-core/src/config/schema.rs`, find `write_example_budget` and
add the new fields to the generated example:

```rust
fn write_example_budget(out: &mut String, c: &Self) {
    let _ = writeln!(out, "# -- Spend / token budgets --");
    let _ = writeln!(out, "[budget]");
    let _ = writeln!(out, "max_plan_usd = {:.1}", c.budget.max_plan_usd);
    let _ = writeln!(out, "max_turn_usd = {:.1}", c.budget.max_turn_usd);
    let _ = writeln!(out, "# max_session_usd = 0.0  # 0 = unlimited");
    let _ = writeln!(out, "# max_tokens_per_call = 4096");
    let _ = writeln!(out, "# warn_on_threshold = true");
    let _ = writeln!(out, "# warn_threshold = 0.8");
    let _ = writeln!(out, "prompt_token_budget = {}\n", c.budget.prompt_token_budget);
}
```

### 5. Add a test

```rust
#[test]
fn budget_config_defaults() {
    let config = BudgetConfig::default();
    assert!((config.max_plan_usd - 25.0).abs() < f32::EPSILON);
    assert!((config.max_turn_usd - 3.0).abs() < f32::EPSILON);
    assert_eq!(config.prompt_token_budget, 10_000);
    assert!((config.max_session_usd - 0.0).abs() < f32::EPSILON);
    assert!(config.max_tokens_per_call.is_none());
    assert!(config.warn_on_threshold);
    assert!((config.warn_threshold - 0.8).abs() < f32::EPSILON);
}
```

## Verification

```bash
cargo check -p roko-core
cargo test -p roko-core --lib
```

Both must succeed. Existing deserialization tests pass because all new fields
have `#[serde(default)]`.

## Constraints

- All new fields must have `#[serde(default)]` to maintain backwards
  compatibility with existing `roko.toml` files.
- `max_session_usd = 0.0` means unlimited (not "zero budget").
- Do NOT change the meaning of existing fields.
