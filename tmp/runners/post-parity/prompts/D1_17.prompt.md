# D1_17: Set Default Budget in ModelCallService

## Goal

Read `BudgetConfig` from the workspace config and apply `max_turn_usd` and
`max_tokens_per_call` as default limits in `ModelCallService` so every model
call is budget-bounded.

## Files to Edit

- `crates/roko-agent/src/model_call_service.rs` (primary)

## Also Read

- `crates/roko-core/src/config/budget.rs` — `BudgetConfig` fields

## Steps

### 1. Add budget config fields to ModelCallService

The `ModelCallService` struct already has a `budget: BudgetCell` field (around
line 96). Examine its current implementation. It may already have per-call
budget tracking.

### 2. Thread `BudgetConfig` into construction

In the `with_config` builder method, extract budget settings:

```rust
pub fn with_config(mut self, config: RokoConfig) -> Self {
    // ... existing config processing ...

    // Apply budget defaults from config
    if config.budget.max_turn_usd > 0.0 {
        self.default_max_cost_usd = Some(config.budget.max_turn_usd as f64);
    }
    if let Some(max_tokens) = config.budget.max_tokens_per_call {
        self.default_max_tokens = Some(max_tokens);
    }

    self.config = config;
    self
}
```

### 3. Apply defaults to incoming requests

In the `ModelCaller::call` implementation, apply budget defaults to requests
that don't specify their own:

```rust
// Apply default budget if request doesn't specify one
let budget = req.budget.unwrap_or_else(|| {
    TokenBudget {
        max_input: None,
        max_output: req.max_tokens.map(u64::from),
        max_cost_usd: self.default_max_cost_usd,
    }
});
```

### 4. Apply default max_tokens

```rust
let max_tokens = req.max_tokens.or(self.default_max_tokens);
```

### 5. Add the new fields

If `ModelCallService` does not have these fields, add them:

```rust
    /// Default per-call cost limit from config.
    default_max_cost_usd: Option<f64>,
    /// Default per-call token limit from config.
    default_max_tokens: Option<u32>,
```

Initialize both to `None` in the constructor.

## Verification

```bash
cargo check -p roko-agent
cargo test -p roko-agent --lib
```

Both must succeed. Existing tests pass because the default budget config
has `max_turn_usd = 3.0` which is a reasonable default.

## Constraints

- Budget defaults are applied as fallbacks — explicit request budgets take
  precedence.
- A `max_turn_usd` of 0.0 means "no limit" (skip budget enforcement).
- Do NOT change the `TokenBudget` struct in `roko-core`.
