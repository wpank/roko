# D1_18: Implement Budget Exceeded Graceful Degradation

## Goal

When a model call would exceed the configured budget, gracefully degrade by
returning a `GatewayError::BudgetExceeded` error instead of silently over-
spending. Include the budget status in the error message.

## Files to Edit

- `crates/roko-agent/src/model_call_service.rs` (primary)

## Also Read

- `crates/roko-core/src/foundation.rs` — `GatewayError::BudgetExceeded`

## Steps

### 1. Locate the existing budget tracking

In `ModelCallService`, find the `budget: BudgetCell` field (around line 96).
Examine how it tracks cumulative spend across calls.

### 2. Add pre-call budget check

Before dispatching to the provider, check if the projected cost would exceed
the budget:

```rust
// Pre-call budget check
if let Some(max_cost) = self.default_max_cost_usd {
    let estimated = self.estimate_cost(&req);
    let spent = self.budget_spent();
    if spent + estimated.predicted_cost_usd > max_cost {
        return Err(RokoError::from(GatewayError::BudgetExceeded {
            detail: format!(
                "projected cost ${:.4} would exceed per-call limit ${:.2} \
                 (already spent ${:.4} this session)",
                estimated.predicted_cost_usd, max_cost, spent,
            ),
        }));
    }
}
```

### 3. Add session-level budget check

If the service has a session-level budget (from `max_session_usd`), check it:

```rust
if let Some(session_max) = self.session_budget_usd {
    let spent = self.session_total_cost();
    if spent >= session_max {
        return Err(RokoError::from(GatewayError::BudgetExceeded {
            detail: format!(
                "session budget ${:.2} exhausted (spent ${:.4})",
                session_max, spent,
            ),
        }));
    }
}
```

### 4. Add warning emission when approaching threshold

If `warn_on_threshold` is enabled and spend exceeds the warning threshold:

```rust
if self.warn_on_threshold {
    let spent = self.session_total_cost();
    if let Some(max) = self.session_budget_usd {
        let ratio = spent / max;
        if ratio >= self.warn_threshold as f64 {
            tracing::warn!(
                spent = spent,
                max = max,
                ratio = format!("{:.0}%", ratio * 100.0),
                "approaching session budget limit"
            );
        }
    }
}
```

### 5. Add `session_budget_usd` and `warn_on_threshold` fields

If not already present:

```rust
    /// Session-level cost budget (0.0 = unlimited).
    session_budget_usd: Option<f64>,
    /// Whether to warn when approaching budget limits.
    warn_on_threshold: bool,
    /// Warning threshold fraction (0.0 to 1.0).
    warn_threshold: f32,
```

Initialize from the config in `with_config`:

```rust
if config.budget.max_session_usd > 0.0 {
    self.session_budget_usd = Some(config.budget.max_session_usd as f64);
}
self.warn_on_threshold = config.budget.warn_on_threshold;
self.warn_threshold = config.budget.warn_threshold;
```

### 6. Add `session_total_cost` helper

```rust
fn session_total_cost(&self) -> f64 {
    self.budget.total_cost_usd()
}
```

If the existing `BudgetCell` does not expose this, add the accessor.

## Verification

```bash
cargo check -p roko-agent
cargo test -p roko-agent --lib
```

Both must succeed.

## Constraints

- Budget exceeded returns `GatewayError::BudgetExceeded`, not a panic.
- Pre-call cost estimation is approximate (uses the `CostEstimate` from
  `estimate_cost`) — actual cost may differ.
- Warnings are emitted via `tracing::warn`, not printed to stdout.
- Session budget of 0.0 means unlimited — do NOT enforce a zero budget.
