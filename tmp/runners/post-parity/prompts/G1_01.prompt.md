# G1_01: Add complexity and prior_failures to GateConfig

## Goal

Extend `GateConfig` with `complexity` and `prior_failures` fields so the gate
pipeline can adjust behavior based on task characteristics and history.

## Files to Edit

- `crates/roko-core/src/foundation.rs` (primary)

## Steps

### 1. Add fields to `GateConfig`

In `crates/roko-core/src/foundation.rs`, find the `GateConfig` struct (around
line 270) and add new fields:

```rust
#[derive(Debug, Clone, serde::Serialize, serde::Deserialize)]
pub struct GateConfig {
    /// Working directory to verify.
    pub workdir: PathBuf,
    /// Which gates to run (e.g., ["compile", "test", "clippy"]).
    pub enabled_gates: Vec<String>,
    /// Configured shell commands, consumed by shell/custom:shell gate entries.
    pub shell_gates: Vec<ShellGateCommand>,
    /// Maximum rung to run (0-6).
    pub max_rung: Option<u8>,
    /// Task complexity estimate (0.0 = trivial, 1.0 = complex).
    /// Used by adaptive thresholds to scale retry budgets.
    #[serde(default)]
    pub complexity: Option<f64>,
    /// Number of prior gate failures for this task across iterations.
    /// Used by adaptive thresholds to adjust skip behavior.
    #[serde(default)]
    pub prior_failures: u32,
}
```

### 2. Update all GateConfig construction sites

Search for `GateConfig {` in the codebase:

```bash
grep -rn 'GateConfig {' crates/ --include='*.rs' | grep -v target/
```

Add the new fields with defaults to each site:

```rust
    complexity: None,
    prior_failures: 0,
```

Key construction sites:
- `crates/roko-runtime/src/effect_driver.rs` — `run_gates` method (line ~285)
- `crates/roko-gate/src/gate_service.rs` — test fixtures
- Any other files that construct `GateConfig`

### 3. Add accessor methods

```rust
impl GateConfig {
    /// Returns the complexity estimate, defaulting to 0.5 (moderate).
    pub fn complexity_or_default(&self) -> f64 {
        self.complexity.unwrap_or(0.5)
    }

    /// Returns true if this task has had prior gate failures.
    pub fn has_prior_failures(&self) -> bool {
        self.prior_failures > 0
    }
}
```

## Verification

```bash
cargo check --workspace
cargo test -p roko-core --lib
cargo test -p roko-gate --lib
cargo test -p roko-runtime --lib
```

All must succeed. The `#[serde(default)]` on both fields ensures existing
serialized configs deserialize correctly.

## Constraints

- Both fields must be `#[serde(default)]` for backwards compatibility.
- `complexity` is `Option<f64>` (not `f64`) so callers can distinguish
  "not specified" from "zero complexity".
- Do NOT change the meaning of existing `GateConfig` fields.
