# G1_02: Add feedback and failure_classification to GateReport

## Goal

Extend `GateReport` with structured feedback and failure classification
fields so downstream consumers (prompt assembly, learning) can use richer
gate information.

## Files to Edit

- `crates/roko-core/src/foundation.rs` (primary)

## Steps

### 1. Add fields to `GateReport`

In `crates/roko-core/src/foundation.rs`, find the `GateReport` struct (around
line 299) and add:

```rust
#[derive(Debug, Clone)]
pub struct GateReport {
    pub verdicts: Vec<GateVerdict>,
    /// Structured feedback for prompt injection on retry.
    /// Each entry is a human-readable line describing what went wrong and
    /// what the next attempt should focus on.
    pub feedback: Vec<String>,
    /// Classification of the overall failure mode.
    pub failure_classification: Option<FailureClassification>,
}
```

### 2. Define `FailureClassification`

Add the enum near `GateReport`:

```rust
/// Classification of a gate pipeline failure.
#[derive(Debug, Clone, PartialEq, Eq, serde::Serialize, serde::Deserialize)]
pub enum FailureClassification {
    /// Compilation error — likely a syntax or type issue.
    CompileError,
    /// Lint/style failure — clippy or formatting.
    LintFailure,
    /// Test failure — logic or assertion error.
    TestFailure,
    /// Review rejection — human or LLM reviewer concerns.
    ReviewRejection,
    /// Custom gate failure — domain-specific.
    CustomGateFailure { gate_name: String },
    /// Multiple gates failed — compound issue.
    MultipleFailures,
    /// Unknown classification.
    Unknown,
}
```

### 3. Update `GateReport` constructors and methods

Update `all_passed()`, `first_failure()`, and `failure_summary()` to account
for the new fields. No logic changes needed — they operate on `verdicts`.

Update the method that constructs `GateReport` from verdicts to auto-populate
the classification:

```rust
impl GateReport {
    /// Create a report from a list of verdicts, auto-classifying failures.
    pub fn from_verdicts(verdicts: Vec<GateVerdict>) -> Self {
        let feedback = verdicts
            .iter()
            .filter(|v| !v.passed && !v.skipped)
            .map(|v| {
                format!(
                    "Gate '{}' failed: {}",
                    v.gate_name,
                    if v.output.is_empty() { "no output" } else { &v.output }
                )
            })
            .collect();

        let failures: Vec<_> = verdicts
            .iter()
            .filter(|v| !v.passed && !v.skipped)
            .collect();

        let classification = match failures.len() {
            0 => None,
            1 => Some(classify_single_failure(&failures[0].gate_name)),
            _ => Some(FailureClassification::MultipleFailures),
        };

        Self {
            verdicts,
            feedback,
            failure_classification: classification,
        }
    }
}

fn classify_single_failure(gate_name: &str) -> FailureClassification {
    match gate_name {
        "compile" | "compile:cargo" => FailureClassification::CompileError,
        "clippy" | "clippy:cargo" | "fmt" | "fmt:cargo" | "format" => {
            FailureClassification::LintFailure
        }
        "test" | "test:cargo" => FailureClassification::TestFailure,
        "judge" | "llm-judge" | "review" => FailureClassification::ReviewRejection,
        name => FailureClassification::CustomGateFailure {
            gate_name: name.to_string(),
        },
    }
}
```
