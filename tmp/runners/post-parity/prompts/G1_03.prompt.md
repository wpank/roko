# G1_03: Teach GateService rung selection

## Goal

Make `GateService` use `GateConfig.complexity` and `GateConfig.prior_failures`
to influence rung selection: skip higher rungs for simple tasks, require all
rungs for tasks with prior failures.

## Files to Edit

- `crates/roko-gate/src/gate_service.rs` (primary)

## Also Read

- `crates/roko-core/src/foundation.rs` — `GateConfig` fields

## Steps

### 1. Add complexity-aware rung filtering

In the `GateRunner::run_gates` implementation inside `GateService`, after
computing the ordered gate names, filter based on complexity:

```rust
// Adjust max_rung based on task complexity.
let effective_max_rung = match config.complexity {
    Some(c) if c < 0.3 => {
        // Simple tasks: skip rungs above 4 (no shell/judge)
        config.max_rung.map_or(Some(4), |r| Some(r.min(4)))
    }
    _ => config.max_rung,
};
```

### 2. Adjust skip behavior for prior failures

When checking adaptive thresholds for skip decisions, override the skip
if the task has prior failures:

```rust
// Never skip a rung if the task had prior failures on that gate
let should_skip = if config.prior_failures > 0 {
    false  // Force all rungs when there are prior failures
} else if let Some(ref adaptive) = self.adaptive {
    let thresholds = adaptive.lock().unwrap_or_else(|e| e.into_inner());
    rung.map_or(false, |r| thresholds.should_skip(r.into()))
} else {
    false
};
```

### 3. Add tracing for skip decisions

Log when a rung is skipped due to complexity:

```rust
if should_skip_for_complexity {
    tracing::debug!(
        gate = name,
        complexity = ?config.complexity,
        "skipping gate due to low task complexity"
    );
    verdicts.push(GateVerdict {
        gate_name: name.clone(),
        passed: true,
        skipped: true,
        skip_reason: Some("low complexity".to_string()),
        output: String::new(),
        duration_ms: 0,
    });
    continue;
}
```

### 4. Apply effective_max_rung to the gate loop

In the gate execution loop, skip gates above the effective max rung:

```rust
if let Some(max) = effective_max_rung {
    if let Some(rung) = Self::rung_for_name(&name) {
        if rung > max {
            verdicts.push(GateVerdict {
                gate_name: name.clone(),
                passed: true,
                skipped: true,
                skip_reason: Some(format!("rung {rung} above max {max}")),
                output: String::new(),
                duration_ms: 0,
            });
            continue;
        }
    }
}
```

### 5. Add a test

```rust
#[tokio::test]
async fn low_complexity_skips_high_rungs() {
    let service = GateService::new();
    let config = GateConfig {
        workdir: std::env::temp_dir(),
        enabled_gates: vec!["compile".to_string(), "judge".to_string()],
        shell_gates: vec![],
        max_rung: None,
        complexity: Some(0.1),
        prior_failures: 0,
    };
    let report = service.run_gates(config).await.unwrap();
    // judge (rung 6) should be skipped
    let judge = report.verdicts.iter().find(|v| v.gate_name == "judge");
    assert!(judge.map_or(true, |v| v.skipped));
}
```

## Verification

```bash
cargo check -p roko-gate
cargo test -p roko-gate --lib
```

Both must succeed.

## Constraints

- Rung 0 (compile) is NEVER skipped, regardless of complexity.
- Prior failures override complexity-based skipping — if a task failed before,
  run all configured gates.
- Default behavior (no complexity, no prior failures) is unchanged.
