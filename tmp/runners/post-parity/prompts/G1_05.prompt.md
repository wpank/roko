# G1_05: Wire GateService into run.rs

## Goal

Ensure the legacy `run_once` gate path in `run.rs` uses `GateService` (via
the `GateRunner` trait) instead of constructing gates individually.

## Files to Edit

- `crates/roko-cli/src/run.rs` (primary)

## Also Read

- `crates/roko-gate/src/gate_service.rs` — `GateService::run_gates`

## Steps

### 1. Identify the legacy gate path

In `crates/roko-cli/src/run.rs`, find where gates are run individually.
The legacy path constructs `CompileGate`, `ClippyGate`, `TestGate` etc.
directly and runs them in a loop:

```rust
// Look for patterns like:
let compile_gate = CompileGate::new(build_system);
let verdict = compile_gate.verify(&ctx, &signal).await;
```

### 2. Replace with GateService

Replace the manual gate construction with `GateService`:

```rust
use roko_gate::gate_service::GateService;
use roko_core::foundation::{GateConfig, GateRunner};

let gate_service = GateService::new();
let gate_config = GateConfig {
    workdir: workdir.to_path_buf(),
    enabled_gates: config.gates.iter().map(|g| g.name.clone()).collect(),
    shell_gates: config.gates.iter()
        .filter_map(|g| g.shell_command.as_ref().map(|cmd| {
            CoreShellGateCommand {
                program: cmd.program.clone(),
                args: cmd.args.clone(),
                timeout_ms: cmd.timeout_ms.unwrap_or(30_000),
            }
        }))
        .collect(),
    max_rung: None,
    complexity: None,
    prior_failures: 0,
};

let report = gate_service.run_gates(gate_config).await?;
```

### 3. Map GateReport back to RunReport verdicts

Convert the `GateReport` verdicts to the `RunReport.gate_verdicts` format:

```rust
let gate_verdicts: Vec<(String, bool)> = report
    .verdicts
    .iter()
    .filter(|v| !v.skipped)
    .map(|v| (v.gate_name.clone(), v.passed))
    .collect();
```

### 4. Thread feedback into the state machine

If the report has feedback, use it for the gate failure context:

```rust
let gate_feedback = if !report.feedback.is_empty() {
    Some(report.feedback.join("\n"))
} else {
    report.failure_summary().into()
};
```

### 5. Verify the workflow engine path

The workflow engine path (`run_workflow_engine_with_services`) already uses
`GateService` via the `ServiceBundle`. Verify this is the case — it should
construct `GateService::new()` in `ServiceFactory::build()` (line 195 of
`service_factory.rs`).

## Verification

```bash
cargo check -p roko-cli
```

Must succeed. Gate behavior should be identical since `GateService` wraps the
same gate implementations.

## Constraints

- Do NOT remove the individual gate imports yet — they may be used elsewhere.
- The legacy `run_once` path may be behind `#[cfg(feature = "legacy-orchestrate")]`
  — check before editing.
- Gate execution order must remain rung-ordered (compile -> clippy -> test -> ...).
