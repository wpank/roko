# G1_06: Wire GateService into ACP runner

## Goal

Replace the direct gate construction in the ACP runner with `GateService`
so ACP sessions use the same gate infrastructure as `roko run`.

## Files to Edit

- `crates/roko-acp/src/runner.rs` (primary)

## Also Read

- `crates/roko-gate/src/gate_service.rs` — `GateService`

## Steps

### 1. Identify the current gate path

In `crates/roko-acp/src/runner.rs`, find where gates are constructed directly.
The runner imports `CompileGate`, `ClippyGate`, `TestGate` (line 23):

```rust
use roko_gate::{
    AdaptiveThresholds, ClippyGate, CompileGate, GatePayload, TestGate,
    ...
};
```

### 2. Find the gate execution function

Look for a function that runs gates (possibly `run_gates`, `execute_gates`,
or inline gate logic). It likely constructs gates individually and runs them
in sequence.

### 3. Replace with GateService

If the ACP runner already uses `WorkflowEngine` (via the `ServiceFactory`
path), gates are already routed through `GateService`. Verify:

```rust
// In the ServiceFactory path:
let gate_runner: Arc<dyn GateRunner> = Arc::new(GateService::new());
```

### 4. If there is a separate gate path, replace it

If the runner has its own gate execution outside the workflow engine:

```rust
use roko_gate::gate_service::GateService;
use roko_core::foundation::{GateConfig, GateRunner};

let gate_service = GateService::new();
let config = GateConfig {
    workdir: workdir.clone(),
    enabled_gates: vec![
        "compile".to_string(),
        "clippy".to_string(),
        "test".to_string(),
    ],
    shell_gates: vec![],
    max_rung: None,
    complexity: None,
    prior_failures: iteration_failures,
};
let report = gate_service.run_gates(config).await?;
```

### 5. Thread `prior_failures` from session state

The ACP session tracks iteration count. Use it to set `prior_failures`:

```rust
let prior_failures = workflow_run.gate_failure_count();
```

If the `WorkflowRun` struct does not have this method, count failures from
the session state.

### 6. Remove unused direct gate imports

If `CompileGate`, `ClippyGate`, `TestGate` are no longer used directly,
remove them from the import block. Keep `AdaptiveThresholds` and `GatePayload`
if still used.

## Verification

```bash
cargo check -p roko-acp
```

Must succeed.

## Constraints

- Do NOT change the ACP protocol or session semantics.
- If the ACP runner uses `WorkflowEngine` exclusively, this task may be a
  verification-only pass — confirm and document.
- Keep `AdaptiveThresholds` usage if it exists — do NOT remove adaptive
  threshold integration.
