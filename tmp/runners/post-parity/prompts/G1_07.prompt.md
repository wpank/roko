# G1_07: Add Verdict::Skip constructor

## Goal

Add a `Verdict::skip()` constructor that creates a verdict with `passed: true`
and a clear indication that the gate was skipped rather than exercised.

## Files to Edit

- `crates/roko-core/src/verdict.rs` (primary)

## Steps

### 1. Add the `skip` constructor

In `crates/roko-core/src/verdict.rs`, add to the `impl Verdict` block:

```rust
    /// A skip verdict — the gate was not exercised.
    ///
    /// Skipped verdicts record `passed: true` (they do not block the pipeline)
    /// with `score: 0.0` (they contribute no confidence). The reason field
    /// explains why the gate was skipped.
    ///
    /// Downstream consumers can distinguish skips from genuine passes by
    /// checking `is_skip()`.
    #[must_use]
    pub fn skip(gate: impl Into<String>, reason: impl Into<String>) -> Self {
        Self {
            passed: true,
            reason: reason.into(),
            gate: gate.into(),
            score: 0.0, // No confidence contribution
            detail: Some("skipped".to_string()),
            test_count: None,
            error_digest: None,
            duration_ms: 0,
        }
    }
```

### 2. Add the `is_skip` predicate

```rust
    /// Returns true if this verdict represents a skipped gate.
    ///
    /// A skipped gate passed trivially (no check was performed) and
    /// should not count toward confidence calculations.
    #[must_use]
    pub fn is_skip(&self) -> bool {
        self.passed && self.score == 0.0 && self.detail.as_deref() == Some("skipped")
    }
```

### 3. Add tests

```rust
#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn skip_verdict_is_identified() {
        let v = Verdict::skip("symbol", "no SymbolManifest wired");
        assert!(v.passed);
        assert!(v.is_skip());
        assert_eq!(v.score, 0.0);
        assert_eq!(v.gate, "symbol");
    }

    #[test]
    fn pass_verdict_is_not_skip() {
        let v = Verdict::pass("compile");
        assert!(v.passed);
        assert!(!v.is_skip());
        assert_eq!(v.score, 1.0);
    }

    #[test]
    fn fail_verdict_is_not_skip() {
        let v = Verdict::fail("test", "assertion failed");
        assert!(!v.passed);
        assert!(!v.is_skip());
    }
}
```

## Verification

```bash
cargo check -p roko-core
cargo test -p roko-core --lib -- verdict
```

Both must succeed.

## Constraints

- `Verdict::skip` sets `passed: true` because skipped gates should NOT block
  the pipeline.
- `score: 0.0` distinguishes skips from genuine passes (`score: 1.0`).
- `detail: Some("skipped")` is the canonical marker — `is_skip()` checks it.
- Do NOT change `Verdict::pass` or `Verdict::fail` — those are stable.
