# G1_08: Replace stub pass with stub skip

## Goal

Replace the `stub_verdict` function in `rung_dispatch.rs` with
`Verdict::skip()` so that gates without wired inputs are recorded as
skips rather than false passes.

## Files to Edit

- `crates/roko-gate/src/rung_dispatch.rs` (primary)

## Also Read

- `crates/roko-core/src/verdict.rs` — `Verdict::skip()`

## Steps

### 1. Locate `stub_verdict`

In `crates/roko-gate/src/rung_dispatch.rs`, find the `stub_verdict` function
(around line 132):

```rust
fn stub_verdict(gate: &str, detail: impl Into<String>) -> Verdict {
    let message = format!("stub gate; {}", detail.into());
    let mut verdict = Verdict::pass(gate.to_string());
    verdict.reason.clone_from(&message);
    verdict.detail = Some(message);
    verdict
}
```

### 2. Replace with `Verdict::skip`

```rust
fn stub_verdict(gate: &str, detail: impl Into<String>) -> Verdict {
    Verdict::skip(gate, format!("stub gate; {}", detail.into()))
}
```

### 3. Verify callers

The `stub_verdict` function is called from several gate dispatch functions.
Ensure all callers still compile:

```bash
grep -n 'stub_verdict' crates/roko-gate/src/rung_dispatch.rs
```

Common call sites:
- `run_symbol_gate` — "no SymbolManifest wired into rung 3"
- `run_generated_test_gate` — "no artifact store wired"
- `run_fact_check_gate` — "no search oracle wired"
- `run_llm_judge_gate` — "no judge oracle wired"
- Others as applicable

### 4. Verify test expectations

If any tests check for `stub_verdict` output (checking `passed: true`,
`score: 1.0`), update them to expect `score: 0.0` and `is_skip() == true`:

```rust
// Before:
assert!(verdict.passed);
assert_eq!(verdict.score, 1.0);

// After:
assert!(verdict.passed);
assert!(verdict.is_skip());
assert_eq!(verdict.score, 0.0);
```

### 5. Check downstream consumers

Verify that consumers of rung dispatch results handle skipped verdicts
correctly. In particular:
- Adaptive threshold observers should NOT count skips as passes
- Episode recording should distinguish skips from passes
- TUI/dashboard displays should show "skipped" not "passed"

## Verification

```bash
cargo check -p roko-gate
cargo test -p roko-gate --lib
```

Both must succeed.

## Constraints

- `stub_verdict` still produces `passed: true` verdicts — they do NOT
  block the pipeline.
- The semantic change is from "fake pass" to "skip" — downstream consumers
  can now distinguish them.
- Do NOT remove `stub_verdict` — just change its implementation.
