# G1_09: Drain SPC alerts after pipeline run

## Goal

After `GateService` completes a gate pipeline run, drain any pending SPC
(Statistical Process Control) alerts from the adaptive thresholds and emit
them as runtime events.

## Files to Edit

- `crates/roko-gate/src/gate_service.rs` (primary)

## Also Read

- `crates/roko-gate/src/adaptive_threshold.rs` — `AdaptiveThresholds`
- `crates/roko-gate/src/spc.rs` — `SpcAlert`

## Steps

### 1. Locate the gate pipeline completion point

In `crates/roko-gate/src/gate_service.rs`, find where the `GateRunner::run_gates`
method returns the final `GateReport`. This is the point to drain SPC alerts.

### 2. Drain pending alerts

After all gates have run but before returning the report, drain SPC alerts:

```rust
// Drain SPC alerts from adaptive thresholds
if let Some(ref adaptive) = self.adaptive {
    let mut thresholds = adaptive.lock().unwrap_or_else(|e| e.into_inner());
    let alerts = thresholds.drain_pending_spc_alerts();
    for (rung, alert) in &alerts {
        tracing::info!(
            rung,
            alert_type = ?alert.kind(),
            message = %alert,
            "SPC alert from adaptive gate thresholds"
        );
    }
}
```

### 3. Check if `drain_pending_spc_alerts` exists

In `crates/roko-gate/src/adaptive_threshold.rs`, check if `AdaptiveThresholds`
has a method to drain pending alerts. Look for:

```rust
pub fn drain_pending_spc_alerts(&mut self) -> Vec<(u32, SpcAlert)>
```

If it does not exist, add it:

```rust
    /// Drain all pending SPC alerts, clearing the internal buffer.
    pub fn drain_pending_spc_alerts(&mut self) -> Vec<(u32, SpcAlert)> {
        std::mem::take(&mut self.pending_spc_alerts)
    }
```

### 4. Include alerts in the GateReport

Add alerts to the report feedback if any were detected:

```rust
if !alerts.is_empty() {
    for (rung, alert) in &alerts {
        report.feedback.push(format!(
            "Statistical process control alert on rung {rung}: {alert}"
        ));
    }
}
```

### 5. Add the alert to GateReport if desired

Optionally add an `spc_alerts` field to `GateReport`:

```rust
// In foundation.rs GateReport:
pub spc_alerts: Vec<String>,
```

Or use the `feedback` vector (simpler, no foundation change needed).

## Verification

```bash
cargo check -p roko-gate
cargo test -p roko-gate --lib
```

Both must succeed. No behavioral change for pipelines without adaptive
thresholds attached.

## Constraints

- SPC alert draining is fire-and-forget — alerts are logged, not acted upon.
- Use `tracing::info!` not `tracing::warn!` — SPC alerts are informational.
- Alerts are only drained when adaptive thresholds are attached to the service.
- Mutex poisoning is handled with `unwrap_or_else(|e| e.into_inner())`.
