# G1_10: Call Hotelling observe_pipeline

## Goal

After each complete gate pipeline run, call `AdaptiveThresholds::observe_pipeline`
(Hotelling T-squared detector) so the multivariate anomaly detector can track
joint gate outcomes.

## Files to Edit

- `crates/roko-gate/src/gate_service.rs` (primary)

## Also Read

- `crates/roko-gate/src/adaptive_threshold.rs` — `observe_pipeline`
- `crates/roko-gate/src/hotelling.rs` — `HotellingDetector`

## Steps

### 1. Locate `observe_pipeline` on `AdaptiveThresholds`

In `crates/roko-gate/src/adaptive_threshold.rs`, check if `observe_pipeline`
exists. It should accept a vector of per-rung pass/fail booleans and feed
them to the Hotelling detector:

```rust
pub fn observe_pipeline(&mut self, rung_outcomes: &[(u32, bool)])
```

If this method does not exist, add it:

```rust
    /// Feed a complete pipeline's rung outcomes into the Hotelling T-squared detector.
    ///
    /// Call this once per pipeline run, after all gates have executed.
    /// `rung_outcomes` maps rung index to pass/fail (true = pass).
    pub fn observe_pipeline(&mut self, rung_outcomes: &[(u32, bool)]) {
        if let Some(ref mut detector) = self.hotelling {
            // Convert to observation vector: 1.0 for pass, 0.0 for fail
            let max_rung = rung_outcomes.iter().map(|(r, _)| *r).max().unwrap_or(0);
            let mut obs = vec![0.5_f64; (max_rung + 1) as usize];
            for (rung, passed) in rung_outcomes {
                let idx = *rung as usize;
                if idx < obs.len() {
                    obs[idx] = if *passed { 1.0 } else { 0.0 };
                }
            }
            self.joint_anomaly_detected = detector.observe(&obs);
        }
    }
```

### 2. Call from GateService after pipeline completion

In `gate_service.rs`, after all gates have run and verdicts are collected,
call `observe_pipeline`:

```rust
// Feed outcomes to Hotelling detector
if let Some(ref adaptive) = self.adaptive {
    let mut thresholds = adaptive.lock().unwrap_or_else(|e| e.into_inner());

    let rung_outcomes: Vec<(u32, bool)> = verdicts
        .iter()
        .filter(|v| !v.skipped)
        .filter_map(|v| {
            Self::rung_for_name(&v.gate_name)
                .map(|rung| (u32::from(rung), v.passed))
        })
        .collect();

    if !rung_outcomes.is_empty() {
        thresholds.observe_pipeline(&rung_outcomes);
    }
}
```

### 3. Log joint anomaly detection

```rust
if thresholds.joint_anomaly_detected() {
    tracing::info!(
        "Hotelling T-squared joint anomaly detected across gate pipeline"
    );
}
```

### 4. Add `joint_anomaly_detected` accessor if missing

In `adaptive_threshold.rs`:

```rust
    /// Whether the last pipeline observation triggered a joint anomaly.
    pub fn joint_anomaly_detected(&self) -> bool {
        self.joint_anomaly_detected
    }
```

## Verification

```bash
cargo check -p roko-gate
cargo test -p roko-gate --lib
```

Both must succeed.

## Constraints

- The Hotelling detector is optional — only call `observe_pipeline` if the
  detector is initialized (`self.hotelling.is_some()`).
- Skip verdicts are excluded from the observation vector.
- The observation vector dimension is determined by the max rung observed.
- Mutex poisoning is handled with `unwrap_or_else(|e| e.into_inner())`.
