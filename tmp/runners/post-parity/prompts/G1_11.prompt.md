# G1_11: Instantiate domain profiles at plan start

## Goal

When a plan run starts, select the appropriate `ThresholdProfile` (coding,
research, security) based on the task domain and apply it to the
`AdaptiveThresholds` before gates execute.

## Files to Edit

- `crates/roko-gate/src/adaptive_threshold.rs` (primary)

## Also Read

- `crates/roko-gate/src/gate_service.rs` — `with_adaptive_thresholds`

## Steps

### 1. Add `apply_profile` method to `AdaptiveThresholds`

In `crates/roko-gate/src/adaptive_threshold.rs`, add:

```rust
    /// Apply a domain-specific threshold profile.
    ///
    /// Sets prior pass rates for each rung from the profile's priors.
    /// Only affects rungs that have not yet accumulated enough observations
    /// to override the prior (fewer than 5 observations).
    pub fn apply_profile(&mut self, profile: &ThresholdProfile) {
        for (rung, prior) in &profile.rung_priors {
            let stats = self.rungs.entry(*rung).or_default();
            // Only apply prior if we don't have enough real data
            if stats.total_observations < 5 {
                stats.ema_pass_rate = *prior;
            }
        }

        // Apply CUSUM sensitivity override if the profile specifies one
        if let Some(sensitivity) = profile.cusum_sensitivity_override {
            self.cusum_sensitivity = sensitivity;
        }
    }
```

### 2. Add `profile_for_domain` factory

```rust
    /// Select the appropriate profile for a domain label.
    ///
    /// Common labels: "coding", "research", "security", "general".
    /// Unknown labels get the coding profile as a reasonable default.
    pub fn profile_for_domain(domain: &str) -> ThresholdProfile {
        ThresholdProfile::by_name(domain)
    }
```

### 3. Add `with_profile` builder to `GateService`

In `crates/roko-gate/src/gate_service.rs`, add:

```rust
    /// Apply a domain profile to the attached adaptive thresholds.
    ///
    /// Has no effect if no adaptive thresholds are attached.
    pub fn apply_domain_profile(&self, domain: &str) {
        if let Some(ref adaptive) = self.adaptive {
            let profile = AdaptiveThresholds::profile_for_domain(domain);
            let mut thresholds = adaptive.lock().unwrap_or_else(|e| e.into_inner());
            thresholds.apply_profile(&profile);
            tracing::debug!(
                domain,
                profile_name = %profile.name,
                "applied domain threshold profile"
            );
        }
    }
```

### 4. Add domain detection from task role

Add a helper that maps agent roles to domains:

```rust
/// Map an agent role to a threshold domain.
fn domain_for_role(role: &str) -> &str {
    match role {
        "implementer" | "autofix" | "coder" => "coding",
        "researcher" | "analyst" => "research",
        "security" | "auditor" => "security",
        _ => "coding", // Default to coding domain
    }
}
```

### 5. Add a test

```rust
#[test]
fn apply_coding_profile_sets_rung_priors() {
    let mut thresholds = AdaptiveThresholds::new();
    let profile = ThresholdProfile::coding();
    thresholds.apply_profile(&profile);

    let stats_0 = thresholds.stats_for_rung(0);
    assert!((stats_0.ema_pass_rate - 0.90).abs() < 0.01);

    let stats_2 = thresholds.stats_for_rung(2);
    assert!((stats_2.ema_pass_rate - 0.65).abs() < 0.01);
}

#[test]
fn profile_does_not_override_with_enough_data() {
    let mut thresholds = AdaptiveThresholds::new();
    // Record enough observations to override the prior
    for _ in 0..10 {
        thresholds.observe(0, true);
    }
    let profile = ThresholdProfile::coding();
    thresholds.apply_profile(&profile);

    // Rung 0 should keep its observed rate, not the prior
    let stats = thresholds.stats_for_rung(0);
    assert!(stats.total_observations >= 5);
    // EMA should be close to 1.0 (all passes), not 0.90 (prior)
}
```

### 6. Add `stats_for_rung` accessor if missing

```rust
    /// Get the stats for a rung, creating default if missing.
    pub fn stats_for_rung(&self, rung: u32) -> &RungStats {
        self.rungs.get(&rung).unwrap_or(&DEFAULT_RUNG_STATS)
    }
```

With a static default:

```rust
static DEFAULT_RUNG_STATS: RungStats = RungStats {
    ema_pass_rate: 0.5,
    total_observations: 0,
    consecutive_passes: 0,
    cusum_high: 0.0,
    cusum_low: 0.0,
    cusum_shift_detected: false,
};
```

Or return a clone of `RungStats::default()`.

Both must succeed.

## Constraints

- Profile priors are only applied when observations are insufficient (< 5).
- Real observations always take precedence over priors.
- Unknown domains default to "coding" — never panic on unknown input.
- Mutex poisoning is handled with `unwrap_or_else(|e| e.into_inner())`.
