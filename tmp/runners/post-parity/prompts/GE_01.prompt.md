# GE_01: Scaffold roko-eval crate with core types

## Task
Create a new `roko-eval` crate with the core types for the next-generation gate evolution framework: `EvidenceKind`, `Finding`, `CriterionResult`, and `EvalProfile`.

## Runner Context
Runner GE, batch 1 of 8. No dependencies.

## Problem
The existing `roko-gate` crate provides a 7-rung gate pipeline, but it is monolithic and hard to extend. The `roko-eval` crate will define a composable evaluation framework where individual criteria produce typed findings, evidence is collected by trait, and an `EvalService` orchestrates the flow. This batch scaffolds the crate and defines the core type vocabulary.

### Step 3: Create `crates/roko-eval/src/types.rs`

```rust
//! Core types for the evaluation framework.

use serde::{Deserialize, Serialize};
use std::collections::HashMap;
use std::time::Duration;

/// Classification of evidence produced during evaluation.
#[derive(Clone, Debug, PartialEq, Eq, Hash, Serialize, Deserialize)]
pub enum EvidenceKind {
    /// Output from a compiler or build tool.
    CompilerOutput,
    /// Output from a linter or static analysis tool.
    LintOutput,
    /// Output from running tests.
    TestOutput,
    /// A diff between expected and actual.
    Diff,
    /// Free-form text from a shell command.
    ShellOutput,
    /// LLM judge verdict.
    LlmJudge,
    /// Metrics or measurements.
    Metric,
    /// Custom evidence type.
    Custom(String),
}

/// Severity level for a finding.
#[derive(Clone, Copy, Debug, PartialEq, Eq, PartialOrd, Ord, Hash, Serialize, Deserialize)]
pub enum Severity {
    /// Informational, does not affect pass/fail.
    Info,
    /// Warning, may indicate a problem but does not block.
    Warning,
    /// Error, blocks the evaluation.
    Error,
    /// Critical, blocks and should escalate immediately.
    Critical,
}

/// A single finding produced by a criterion evaluation.
#[derive(Clone, Debug, Serialize, Deserialize)]
pub struct Finding {
    /// Unique identifier for this finding within the criterion.
    pub id: String,
    /// Human-readable description.
    pub message: String,
    /// Severity level.
    pub severity: Severity,
    /// Evidence type this finding is based on.
    pub evidence_kind: EvidenceKind,
    /// Optional file path where the issue was found.
    pub file: Option<String>,
    /// Optional line number.
    pub line: Option<u32>,
    /// Raw evidence text (truncated for storage).
    pub evidence_text: Option<String>,
}

/// Result of evaluating a single criterion.
#[derive(Clone, Debug, Serialize, Deserialize)]
pub struct CriterionResult {
    /// Name of the criterion (e.g., "compile", "lint", "test").
    pub criterion_name: String,
    /// Whether this criterion passed.
    pub passed: bool,
    /// Time taken to evaluate.
    pub duration_ms: u64,
    /// Findings produced by the evaluation.
    pub findings: Vec<Finding>,
    /// Optional numeric score (0.0 to 1.0).
    pub score: Option<f64>,
    /// Arbitrary metadata.
    pub metadata: HashMap<String, String>,
}

impl CriterionResult {
    /// Create a passing result with no findings.
    pub fn pass(name: impl Into<String>, duration: Duration) -> Self {
        Self {
            criterion_name: name.into(),
            passed: true,
            duration_ms: duration.as_millis() as u64,
            findings: Vec::new(),
            score: Some(1.0),
            metadata: HashMap::new(),
        }
    }

    /// Create a failing result with findings.
    pub fn fail(name: impl Into<String>, duration: Duration, findings: Vec<Finding>) -> Self {
        Self {
            criterion_name: name.into(),
            passed: false,
            duration_ms: duration.as_millis() as u64,
            findings,
            score: Some(0.0),
            metadata: HashMap::new(),
        }
    }

    /// Count findings at or above the given severity.
    pub fn count_at_severity(&self, min: Severity) -> usize {
        self.findings.iter().filter(|f| f.severity >= min).count()
    }
}

/// Evaluation profile — which criteria to run and their configuration.
#[derive(Clone, Debug, Default, Serialize, Deserialize)]
pub struct EvalProfile {
    /// Profile name (e.g., "standard", "express", "full").
    pub name: String,
    /// Criteria to include, in order.
    pub criteria: Vec<String>,
    /// Whether to stop at the first failing criterion.
    pub fail_fast: bool,
    /// Maximum total evaluation time.
    pub timeout_ms: Option<u64>,
    /// Per-criterion configuration overrides.
    pub config: HashMap<String, HashMap<String, String>>,
}

/// Aggregated report from a full evaluation run.
#[derive(Clone, Debug, Serialize, Deserialize)]
pub struct EvalReport {
    /// Profile used for this evaluation.
    pub profile: String,
    /// Results from each criterion, in evaluation order.
    pub results: Vec<CriterionResult>,
    /// Overall pass/fail.
    pub passed: bool,
    /// Total evaluation time.
    pub total_duration_ms: u64,
    /// Total findings across all criteria.
    pub total_findings: usize,
}

impl EvalReport {
    /// Count total errors across all criteria.
    pub fn error_count(&self) -> usize {
        self.results.iter().map(|r| r.count_at_severity(Severity::Error)).sum()
    }
}
```
