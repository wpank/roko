# GT_03: Wire AdapterRegistry into CLI startup

## Task
Create an `AdapterRegistry` during CLI startup and pass it to the command dispatch context so adapters are available to subcommands and the serve loop.

## Runner Context
Runner GT_A (GTM: Adapter Foundation), batch 3 of 3. Depends on GT_01.

## Problem
Even though the `AdapterRegistry` type exists, there is no code path that creates one, populates it from config, and makes it available at runtime.

## Current Code

**CLI main** -- `crates/roko-cli/src/main.rs:80+`:
Parses args, loads config, dispatches to subcommands. No adapter registry setup.

**RokoConfig** -- `crates/roko-core/src/config/schema.rs`:
Config struct. Does not have adapter config fields yet.

## Exact Changes

### Step 1: Add adapter config to RokoConfig

In the config schema (or in a new section):

```rust
/// Adapter configuration section in roko.toml.
#[derive(Debug, Clone, Default, Serialize, Deserialize)]
pub struct AdapterConfig {
    /// GitHub adapter settings.
    #[serde(default)]
    pub github: Option<GitHubAdapterConfig>,
    /// Linear adapter settings.
    #[serde(default)]
    pub linear: Option<LinearAdapterConfig>,
}

#[derive(Debug, Clone, Default, Serialize, Deserialize)]
pub struct GitHubAdapterConfig {
    /// GitHub owner/org.
    pub owner: String,
    /// Repository name.
    pub repo: String,
    /// Labels to filter issues by.
    #[serde(default)]
    pub labels: Vec<String>,
}

#[derive(Debug, Clone, Default, Serialize, Deserialize)]
pub struct LinearAdapterConfig {
    /// Linear team key.
    pub team: String,
    /// Project ID.
    #[serde(default)]
    pub project: Option<String>,
}
```

### Step 3: Wire into CLI startup

In `main.rs`, after loading config and before dispatching subcommands:

```rust
let adapter_registry = build_adapter_registry(&config);
let adapter_results = adapter_registry.init_all().await;
for (name, result) in &adapter_results {
    match result {
        Ok(()) => tracing::info!(adapter = %name, "adapter initialized"),
        Err(err) => tracing::warn!(adapter = %name, %err, "adapter init failed"),
    }
}
```
