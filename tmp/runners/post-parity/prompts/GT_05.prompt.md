# GT_05: GitHub webhook receiver

## Task
Extend the existing GitHub webhook handler to dispatch received events to the adapter event system, converting GitHub webhook payloads into roko-native work items and events.

## Runner Context
Runner GT_B (GTM: GitHub), batch 2 of 3. No dependencies.

## Problem
The existing webhook handler (`webhooks.rs`) verifies the HMAC signature and persists the raw payload as an `Engram`, but does not convert the payload into typed events that the adapter system can process (e.g., "issue opened" -> create a PRD idea).

## Current Code

**github_webhook** -- `crates/roko-serve/src/routes/webhooks.rs:38-60`:
Verifies HMAC, parses JSON, creates `Engram`, publishes to event bus. Does not interpret the event type.

## Exact Changes

### Step 1: Add event type parsing

After HMAC verification and before creating the Engram, parse the GitHub event type:

```rust
let event_type = headers
    .get("X-GitHub-Event")
    .and_then(|v| v.to_str().ok())
    .unwrap_or("unknown");

let payload: Value = serde_json::from_slice(&body)
    .map_err(|e| ApiError::bad_request(format!("invalid JSON: {e}")))?;

let action = payload.get("action")
    .and_then(|v| v.as_str())
    .unwrap_or("unknown");
```

### Step 2: Add typed event conversion

```rust
/// Convert a GitHub webhook event into adapter events.
fn github_event_to_adapter_events(
    event_type: &str,
    action: &str,
    payload: &Value,
) -> Vec<AdapterEvent> {
    match (event_type, action) {
        ("issues", "opened") => {
            let title = payload["issue"]["title"].as_str().unwrap_or("");
            let body = payload["issue"]["body"].as_str().unwrap_or("");
            let number = payload["issue"]["number"].as_u64().unwrap_or(0);
            vec![AdapterEvent {
                kind: "work_item.created".to_string(),
                payload: json!({
                    "source": "github",
                    "external_id": number.to_string(),
                    "title": title,
                    "body": body,
                }),
                timestamp: chrono::Utc::now().to_rfc3339(),
            }]
        }
        ("issues", "closed") => {
            let number = payload["issue"]["number"].as_u64().unwrap_or(0);
            vec![AdapterEvent {
                kind: "work_item.closed".to_string(),
                payload: json!({
                    "source": "github",
                    "external_id": number.to_string(),
                }),
                timestamp: chrono::Utc::now().to_rfc3339(),
            }]
        }
        ("pull_request", "opened" | "synchronize") => {
            let number = payload["pull_request"]["number"].as_u64().unwrap_or(0);
            let title = payload["pull_request"]["title"].as_str().unwrap_or("");
            vec![AdapterEvent {
                kind: "pull_request.updated".to_string(),
                payload: json!({
                    "source": "github",
                    "external_id": number.to_string(),
                    "title": title,
                }),
                timestamp: chrono::Utc::now().to_rfc3339(),
            }]
        }
        _ => vec![],
    }
}
```

### Step 3: Dispatch adapter events

After creating the Engram, dispatch adapter events:

```rust
let adapter_events = github_event_to_adapter_events(event_type, action, &payload);
for event in adapter_events {
    tracing::info!(kind = %event.kind, "GitHub webhook -> adapter event");
    // Emit to event bus for observer adapters
}
```
