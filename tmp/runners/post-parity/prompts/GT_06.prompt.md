# GT_06: Issue-to-plan pipeline

## Task
Create a workflow that converts a GitHub issue (via the adapter) into a roko PRD idea, optionally drafts a PRD, and optionally generates a plan.

## Runner Context
Runner GT_B (GTM: GitHub), batch 3 of 3. Depends on GT_04.

## Problem
When a GitHub issue is labeled "roko" or "automated", there is no pipeline to automatically convert it into a plan that roko can execute. This is the key GTM integration: external work items become executable plans.

```rust
//! Issue-to-plan pipeline (GT_06).
//!
//! Converts an external work item (GitHub issue, Linear ticket) into
//! a roko PRD and optionally generates an implementation plan.

use std::path::Path;

use roko_core::adapter::WorkItem;

/// Pipeline configuration.
#[derive(Debug, Clone)]
pub struct IssueToPlanConfig {
    /// Whether to auto-draft a PRD from the issue.
    pub auto_draft: bool,
    /// Whether to auto-generate a plan from the PRD.
    pub auto_plan: bool,
    /// Labels that trigger the pipeline.
    pub trigger_labels: Vec<String>,
}

impl Default for IssueToPlanConfig {
    fn default() -> Self {
        Self {
            auto_draft: true,
            auto_plan: false,
            trigger_labels: vec!["roko".to_string(), "automated".to_string()],
        }
    }
}

/// Result of the issue-to-plan pipeline.
#[derive(Debug)]
pub struct IssueToPlanResult {
    /// PRD idea ID created.
    pub idea_id: Option<String>,
    /// PRD draft slug created.
    pub draft_slug: Option<String>,
    /// Plan directory created.
    pub plan_dir: Option<String>,
    /// Steps completed.
    pub steps: Vec<String>,
}

/// Run the issue-to-plan pipeline for a work item.
pub async fn run_issue_to_plan(
    work_item: &WorkItem,
    roko_dir: &Path,
    config: &IssueToPlanConfig,
) -> Result<IssueToPlanResult, String> {
    let mut result = IssueToPlanResult {
        idea_id: None,
        draft_slug: None,
        plan_dir: None,
        steps: Vec::new(),
    };

    // Check if the work item matches trigger labels
    let has_trigger = config.trigger_labels.is_empty()
        || work_item.labels.iter().any(|l| config.trigger_labels.contains(l));
    if !has_trigger {
        return Ok(result);
    }

    // Step 1: Create PRD idea
    let idea_text = format!(
        "{}\n\nSource: {} #{}\n\n{}",
        work_item.title,
        work_item.source,
        work_item.external_id,
        work_item.body
    );

    // Write idea to .roko/prd/ideas/
    let ideas_dir = roko_dir.join("prd").join("ideas");
    std::fs::create_dir_all(&ideas_dir)
        .map_err(|e| format!("create ideas dir: {e}"))?;

    let slug = slugify(&work_item.title);
    let idea_path = ideas_dir.join(format!("{slug}.md"));
    std::fs::write(&idea_path, &idea_text)
        .map_err(|e| format!("write idea: {e}"))?;

    result.idea_id = Some(slug.clone());
    result.steps.push(format!("Created PRD idea: {slug}"));

    tracing::info!(
        slug = %slug,
        source = %work_item.source,
        external_id = %work_item.external_id,
        "issue-to-plan: created PRD idea"
    );

    // Step 2: Auto-draft PRD (if enabled)
    if config.auto_draft {
        let drafts_dir = roko_dir.join("prd").join("drafts");
        std::fs::create_dir_all(&drafts_dir)
            .map_err(|e| format!("create drafts dir: {e}"))?;

        let draft_path = drafts_dir.join(format!("{slug}.md"));
        let draft_content = format!(
            "# {}\n\n## Background\n\n{}\n\n## Requirements\n\nTBD (auto-generated from {} #{})\n",
            work_item.title,
            work_item.body,
            work_item.source,
            work_item.external_id,
        );
        std::fs::write(&draft_path, &draft_content)
            .map_err(|e| format!("write draft: {e}"))?;

        result.draft_slug = Some(slug.clone());
        result.steps.push(format!("Created PRD draft: {slug}"));
    }

    Ok(result)
}

fn slugify(title: &str) -> String {
    title
        .to_lowercase()
        .chars()
        .map(|c| if c.is_alphanumeric() { c } else { '-' })
        .collect::<String>()
        .split('-')
        .filter(|s| !s.is_empty())
        .collect::<Vec<_>>()
        .join("-")
}
```

- Add agent dispatch (the pipeline creates files, not runs agents)
