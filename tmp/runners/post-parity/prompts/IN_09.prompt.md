# IN_09: Wire speculative execution (parallel model racing)

## Task
Add a `speculative_call` method to `ModelCallService` that dispatches the same request to multiple models in parallel and returns the first successful result.

## Runner Context
Runner IN, batch 9 of 9. No dependencies.

## Problem
For latency-critical tasks, dispatching to a single model and waiting for the full response is slow. Speculative execution sends the same request to 2-3 models in parallel, uses the first successful response, and cancels the rest. This trades cost for latency.

    pub response: ModelCallResponse,
    /// Which model won the race.

impl ModelCallService {
    /// Race a request across multiple models and return the first success.
    ///
    /// Dispatches the same request to the primary model and all race
    /// candidates. Returns as soon as the first model responds
    /// successfully. Remaining tasks are cancelled.
    ///
    /// Falls back to primary-only if no race models are configured.
    pub async fn speculative_call(
        &self,
        request: &ModelCallRequest,
        config: &SpeculativeConfig,

        use tokio::time::timeout;

        let all_models: Vec<String> = std::iter::once(self.default_model.clone())
            .chain(config.race_models.iter().cloned())
            .collect();

        if all_models.len() <= 1 {
            // No racing, dispatch normally

        }

        let candidates_raced = all_models.len();

        let cancel = tokio_util::sync::CancellationToken::new();
        let (tx, mut rx) = tokio::sync::mpsc::channel::<(String, ModelCallResponse, u64)>(candidates_raced);

        for model in &all_models {
            let model = model.clone();
            let tx = tx.clone();
            let cancel = cancel.clone();
