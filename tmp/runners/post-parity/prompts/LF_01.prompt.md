# LF_01: Wire FeedbackService to chat_session.rs

## Task
Make `ChatAgentSession` instantiate and use a `FeedbackService` to emit feedback events on each turn completion.

## Runner Context
Runner LF, batch 1 of 6. No dependencies.

## Problem
`FeedbackService` exists with full support for recording model call metrics, gate results, and workflow outcomes. `ChatAgentSession` completes turns (via `dispatch_turn` or similar) but never records anything to the feedback system. Chat turns are invisible to the learning subsystem.

## Exact Changes

### Step 1: Add feedback_service field to ChatAgentSession

In `crates/roko-cli/src/chat_session.rs`, add to the struct:

```rust
    /// Feedback service for recording turn metrics to the learning subsystem.
    feedback_service: Option<Arc<dyn FeedbackSink>>,
```

### Step 2: Initialize in constructor

In `ChatAgentSession::new()`, after other fields are set:

```rust
    let feedback_service: Option<Arc<dyn FeedbackSink>> = {
        let roko_dir = workdir.join(".roko");
        if roko_dir.exists() {
            Some(Arc::new(roko_learn::FeedbackService::from_roko_dir_with_episodes(&roko_dir)))
        } else {
            None
        }
    };
```

Add `feedback_service` to the struct initializer.

### Step 3: Add emit_turn_feedback method

```rust
    /// Record a completed turn to the feedback system.
    fn emit_turn_feedback(&self, turn: &TurnResult) {
        if let Some(ref svc) = self.feedback_service {
            let event = roko_core::foundation::FeedbackEvent::ModelCall {
                request_id: uuid::Uuid::new_v4().to_string(),
                run_id: None,
                model: turn.model.clone(),
                role: "chat".to_string(),
                tokens_in: turn.tokens_in,
                tokens_out: turn.tokens_out,
                cost_usd: turn.cost_usd,
                duration_ms: turn.duration.as_millis() as u64,
                success: true,
                prompt_section_ids: vec![],
                knowledge_ids: vec![],
            };
            if let Err(e) = svc.record(event) {
                tracing::debug!(error = %e, "failed to record turn feedback");
            }
        }
    }
```

### Step 4: Call after each turn

At every point where a `TurnResult` is returned to the caller, call `self.emit_turn_feedback(&result)` before returning.

### Step 5: Flush on drop or session end

Add a method and call it when the session ends:

```rust
    /// Flush any buffered feedback events.
    pub fn flush_feedback(&self) {
        if let Some(ref svc) = self.feedback_service {
            let _ = svc.flush();
        }
    }
```

### Step 6: Add imports

```rust
use roko_core::foundation::FeedbackSink;
use roko_learn::FeedbackService;
use std::sync::Arc;
```

- `ChatAgentSession` creates a `FeedbackService` when `.roko/` exists
- Each completed turn emits a `FeedbackEvent::ModelCall`
- Missing `.roko/` dir = no feedback (graceful degradation)
- `flush_feedback()` is available for session cleanup

- Change `FeedbackService` or `FeedbackSink` trait
- Change `TurnResult` struct
- Add gate feedback (chat doesn't run gates)
