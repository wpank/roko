# LF_02: Wire FeedbackService to chat_inline.rs

## Task
Wire `FeedbackService` into the inline chat REPL so that each completed dispatch records feedback events.

## Runner Context
Runner LF, batch 2 of 6. No dependencies.

## Problem
The inline chat REPL (`chat_inline.rs`) dispatches prompts and receives `DispatchResult` responses but never records any metrics to the feedback system. The REPL tracks costs via `CostMeter` for display purposes, but this data never reaches the learning subsystem.

## Current Code

**ChatSession struct** -- `crates/roko-cli/src/chat_inline.rs` (private struct):
```rust
struct ChatSession {
    phase: Phase,
    input: InputState,
    streaming: StreamingState,
    cost: CostMeter,
    cost_table: CostTable,
    agent_id: String,
    tick: u64,
    started_at: Instant,
    dispatch: DispatchMode,
    response_rx: Option<tokio::sync::mpsc::Receiver<Result<DispatchResult, String>>>,
}
```

**CostMeter** tracks cumulative cost for display.

**DispatchResult handling** -- after receiving a response in the main loop, the REPL updates `cost` and `streaming` but records nothing to feedback.

## Exact Changes

### Step 1: Add feedback_sink field to ChatSession

In `crates/roko-cli/src/chat_inline.rs`, add to the `ChatSession` struct:

```rust
    feedback_sink: Option<Arc<dyn FeedbackSink>>,
```

### Step 2: Initialize during session setup

Where `ChatSession` is constructed, add:

```rust
    let feedback_sink: Option<Arc<dyn FeedbackSink>> = {
        let roko_dir = workdir.join(".roko");
        if roko_dir.exists() {
            Some(Arc::new(roko_learn::FeedbackService::from_roko_dir_with_episodes(&roko_dir)))
        } else {
            None
        }
    };
```

Add `feedback_sink` to the struct init.

### Step 3: Record feedback on dispatch result

Where the `DispatchResult` is received and processed, add:

```rust
    if let Some(ref sink) = session.feedback_sink {
        let event = FeedbackEvent::ModelCall {
            request_id: uuid::Uuid::new_v4().to_string(),
            run_id: None,
            model: result.model.clone().unwrap_or_default(),
            role: "chat".to_string(),
            tokens_in: result.tokens_in.unwrap_or(0),
            tokens_out: result.tokens_out.unwrap_or(0),
            cost_usd: result.cost_usd.unwrap_or(0.0),
            duration_ms: result.duration_ms.unwrap_or(0),
            success: true,
            prompt_section_ids: vec![],
            knowledge_ids: vec![],
        };
        let _ = sink.record(event);
    }
```

### Step 4: Flush on session exit

In the session cleanup path (where the REPL exits), add:

```rust
    if let Some(ref sink) = session.feedback_sink {
        let _ = sink.flush();
    }
```

### Step 5: Add imports

```rust
use roko_core::foundation::{FeedbackEvent, FeedbackSink};
use roko_learn::FeedbackService;
```

## Acceptance Criteria
- Each dispatch result emits a `FeedbackEvent::ModelCall`
- Feedback is flushed when the REPL session exits
- Missing `.roko/` = no feedback (graceful)
- No panic if feedback recording fails

## Do NOT
- Change `DispatchResult` or `CostMeter` types
- Add gate events (chat REPL doesn't run gates)
- Touch files outside write scope
