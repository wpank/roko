# LF_04: Wire CascadeRouter observations from runner event loop

## Task
Ensure the runner v2 event loop records routing observations to the `CascadeRouter` after each task completion.

## Runner Context
Runner LF, batch 4 of 6. No dependencies.

## Problem
The `CascadeRouter` needs `observe_multi_objective()` calls with quality, cost, and latency to update its UCB1/Thompson models. The runner event loop knows which model was used and what the outcome was, but may not be feeding observations back to the router consistently.

### Step 2: Record observation

```rust
    if let Some(router) = &cascade_router {
        // Record binary outcome
        router.record_outcome(&model_slug, gate_passed);

        // Record multi-objective observation for richer learning
        if let Some(model_idx) = router.model_index_for_slug(&model_slug) {
            let quality = if gate_passed { 1.0 } else { 0.0 };
            let weights = roko_learn::model_router::RewardWeights::default();
            router.observe_multi_objective(
                routing_ctx.to_features(),
                model_idx,
                quality,
                completion.cost_usd,
                completion.duration.as_millis() as f64,
                &weights,
            );
        }
    }
```

### Step 3: Persist router state periodically

After recording, check if it is time to persist:

```rust
    // Persist router every 10 observations
    if router.total_observations() % 10 == 0 {
        if let Some(ref path) = router_persist_path {
            if let Err(e) = router.save(path) {
                debug!(error = %e, "failed to persist cascade router");
            }
        }
    }
```

- Each task completion records both `record_outcome` (binary) and `observe_multi_objective` (quality/cost/latency)
- Router state is persisted every 10 observations
- Missing router = no observation (graceful)
- `RoutingContext` fields are populated from task metadata
