# LF_05: Wire gate threshold updates from runner event loop

## Task
Ensure the runner v2 event loop updates `AdaptiveThresholds` after each gate result so that gate skip decisions improve over time.

## Runner Context
Runner LF, batch 5 of 6. No dependencies.

## Problem
`AdaptiveThresholds` tracks per-rung consecutive pass streaks and can skip gates that reliably pass. `GateService` accepts thresholds via `with_adaptive_thresholds()`. But the runner event loop may not be feeding gate results back to the thresholds after each gate run.

## Exact Changes

### Step 1: Ensure AdaptiveThresholds is loaded at run start

If not already present, load thresholds:

```rust
    let thresholds_path = roko_dir.join("learn/gate-thresholds.json");
    let adaptive_thresholds = Arc::new(Mutex::new(
        AdaptiveThresholds::load(&thresholds_path).unwrap_or_else(|_| AdaptiveThresholds::new())
    ));
```

### Step 2: Record gate results

After each gate completion in the event loop:

```rust
    if let Some(rung) = GateService::rung_for_name(&gate_name) {
        let mut thresholds = adaptive_thresholds.lock().unwrap_or_else(|e| e.into_inner());
        thresholds.record(rung, gate_passed);
    }
```

### Step 3: Persist thresholds periodically

After recording:

```rust
    // Persist thresholds after each gate result
    let thresholds = adaptive_thresholds.lock().unwrap_or_else(|e| e.into_inner());
    if let Err(e) = thresholds.save(&thresholds_path) {
        debug!(error = %e, "failed to persist adaptive thresholds");
    }
```

### Step 4: Log threshold decisions

```rust
    if let Some(rung) = GateService::rung_for_name(&gate_name) {
        let thresholds = adaptive_thresholds.lock().unwrap_or_else(|e| e.into_inner());
        if thresholds.should_skip(rung) {
            debug!(gate = %gate_name, rung, "gate would be skipped by adaptive thresholds");
        }
    }
```
