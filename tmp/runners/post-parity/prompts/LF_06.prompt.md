# LF_06: Wire efficiency events from runner event loop

## Task
Ensure the runner v2 event loop emits `AgentEfficiencyEvent` records per turn to `.roko/learn/efficiency.jsonl`.

## Runner Context
Runner LF, batch 6 of 6. No dependencies.

## Problem
`AgentEfficiencyEvent` captures per-turn efficiency metrics (tokens, cost, duration, model, role, gate outcome). The runner event loop has all this data after each task completion but may not be constructing and writing efficiency events.

## Current Code

**AgentEfficiencyEvent** -- `crates/roko-learn/src/efficiency.rs`:
```rust
pub struct AgentEfficiencyEvent {
    pub timestamp: String,
    pub run_id: String,
    pub task_id: String,
    pub role: String,
    pub model: String,
    pub tokens_in: u64,
    pub tokens_out: u64,
    pub cost_usd: f64,
    pub duration_ms: u64,
    pub gate_passed: bool,
    pub attempt: u32,
}
```

Typically appended to `.roko/learn/efficiency.jsonl`.

**Runner event loop** -- `crates/roko-cli/src/runner/event_loop.rs`:
Has access to all fields needed after task completion. Already imports `AgentEfficiencyEvent`.

## Exact Changes

### Step 1: Build efficiency event on task completion

After each task completion in the event loop (where agent result and gate result are both available):

```rust
    let efficiency_event = AgentEfficiencyEvent {
        timestamp: chrono::Utc::now().to_rfc3339(),
        run_id: run_id.clone(),
        task_id: task_id.clone(),
        role: agent_role.to_string(),
        model: model_slug.clone(),
        tokens_in: completion.tokens_in,
        tokens_out: completion.tokens_out,
        cost_usd: completion.cost_usd,
        duration_ms: completion.duration.as_millis() as u64,
        gate_passed,
        attempt: state.attempt_count(&task_id),
    };
```

### Step 2: Append to efficiency JSONL

```rust
    let efficiency_path = roko_dir.join("learn/efficiency.jsonl");
    if let Err(e) = append_efficiency_event(&efficiency_path, &efficiency_event) {
        debug!(error = %e, "failed to write efficiency event");
    }
```

### Step 3: Add append helper if not present

```rust
fn append_efficiency_event(
    path: &std::path::Path,
    event: &AgentEfficiencyEvent,
) -> std::io::Result<()> {
    if let Some(parent) = path.parent() {
        std::fs::create_dir_all(parent)?;
    }
    let mut file = std::fs::OpenOptions::new()
        .create(true)
        .append(true)
        .open(path)?;
    let json = serde_json::to_string(event)?;
    use std::io::Write;
    writeln!(file, "{json}")?;
    Ok(())
}
```

## Acceptance Criteria
- Every task completion emits an `AgentEfficiencyEvent`
- Events are appended to `.roko/learn/efficiency.jsonl`
- `attempt` field reflects the retry count for the task
- Failed writes emit debug log but do not abort the run

## Do NOT
- Change `AgentEfficiencyEvent` struct
- Buffer events (append-only JSONL is fine per-task)
- Touch files outside write scope
