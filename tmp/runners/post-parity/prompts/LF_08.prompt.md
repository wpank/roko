# LF_08: Wire BudgetGuardrail into live paths

## Task
Instantiate `BudgetGuardrail` at session start and enforce budget limits at the task, session, and day levels.

## Runner Context
Runner LF, batch 8 of 9. No dependencies.

## Problem
`BudgetGuardrail` exists with `record_cost()` returning `BudgetAction` (Ok/Warn/RouteToCheaper/BlockNewSessions/Block). It enforces per-task, per-session, and per-day limits. But no live code path instantiates it or acts on its decisions. Budget overruns go unchecked.

## Exact Changes

### Step 1: Instantiate BudgetGuardrail at run start

In `crates/roko-cli/src/run.rs`, at run initialization:

```rust
use roko_learn::budget::{BudgetGuardrail, BudgetAction};

let mut budget_guard = BudgetGuardrail::new(
    config.budget.per_task_limit_usd.unwrap_or(5.0),
    config.budget.per_session_limit_usd.unwrap_or(50.0),
    config.budget.per_day_limit_usd.unwrap_or(200.0),
    0.75, // warn at 75%
);
```

### Step 2: Record task costs

After each task dispatch result with cost data:

```rust
    let task_action = budget_guard.record_cost(result.cost_usd, "task");
    let session_action = budget_guard.record_cost(result.cost_usd, "session");

    match task_action {
        BudgetAction::Warn { percent_used, .. } => {
            warn!(percent = format!("{:.0}%", percent_used * 100.0), "task budget warning");
        }
        BudgetAction::RouteToCheaper => {
            info!("task budget high — routing to cheaper model for remaining work");
            // Set flag for model selection to prefer cheaper models
        }
        BudgetAction::Block => {
            error!("task budget exhausted — skipping remaining task retries");
            // Skip further retries for this task
        }
        _ => {}
    }

    match session_action {
        BudgetAction::Warn { percent_used, .. } => {
            warn!(percent = format!("{:.0}%", percent_used * 100.0), "session budget warning");
        }
        BudgetAction::Block => {
            error!("session budget exhausted — aborting run");
            break;
        }
        _ => {}
    }
```

### Step 3: Reset task budget between tasks

```rust
    // At the start of each new task
    budget_guard.reset_task();
```

### Step 4: Add default budget config fallbacks

If `config.budget` doesn't have these fields, use sensible defaults that allow normal operation without config changes.
