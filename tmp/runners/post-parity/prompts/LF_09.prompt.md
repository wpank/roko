# LF_09: Wire ConductorBandit into plan runner retry logic

## Task
Replace hardcoded retry logic in the plan runner with `ConductorBandit` decisions.

## Runner Context
Runner LF, batch 9 of 9. No dependencies.

## Problem
The plan runner has hardcoded retry behavior: on task failure, it retries with the same model a fixed number of times. `ConductorBandit` exists with a learned policy that decides whether to continue, inject a hint, switch models, restart, or abort based on the failure context. But it is never instantiated or consulted.

### Step 4: Add error classifier helper

```rust
fn classify_error(output: &str) -> ErrorPattern {
    let lower = output.to_lowercase();
    if lower.contains("error[e") || lower.contains("cannot find") {
        ErrorPattern::Compile
    } else if lower.contains("test result: failed") || lower.contains("assertion failed") {
        ErrorPattern::Test
    } else if lower.contains("tool_use") || lower.contains("tool error") {
        ErrorPattern::ToolCall
    } else if lower.contains("timeout") || lower.contains("timed out") {
        ErrorPattern::Timeout
    } else if lower.contains("rate limit") || lower.contains("429") {
        ErrorPattern::RateLimit
    } else if lower.contains("context length") || lower.contains("too many tokens") {
        ErrorPattern::ContextOverflow
    } else {
        ErrorPattern::Unknown
    }
}
```
