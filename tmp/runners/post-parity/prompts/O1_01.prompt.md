# O1_01: Add WorktreeManager to EffectServices

## Goal

Add an optional `WorktreeManager` field to the `EffectServices` struct so the
effect driver can create per-task git worktrees for parallel execution.

## Files to Edit

- `crates/roko-runtime/src/effect_driver.rs` (primary)
- `crates/roko-runtime/Cargo.toml` (no new deps needed -- git commands via tokio::process)

## Also Read

- `crates/roko-runtime/Cargo.toml` — confirm available dependencies

## Steps

### 1. Define the `WorktreeManager` struct

In `crates/roko-runtime/src/effect_driver.rs`, after the `EffectServices` struct
(around line 50), add:

```rust
/// Manages per-task git worktrees for parallel execution.
///
/// Each worktree is a lightweight checkout sharing the parent repo's objects.
/// The manager tracks active worktrees and cleans them up on drop/explicit call.
#[derive(Debug, Clone)]
pub struct WorktreeManager {
    /// Root repo directory (the directory containing `.git`).
    repo_root: PathBuf,
    /// Base directory for worktrees (defaults to `<repo_root>/.roko/worktrees`).
    worktree_base: PathBuf,
}
```

### 2. Add basic constructor and accessor methods

```rust
impl WorktreeManager {
    /// Create a new `WorktreeManager` rooted at the given repo.
    pub fn new(repo_root: PathBuf) -> Self {
        let worktree_base = repo_root.join(".roko").join("worktrees");
        Self {
            repo_root,
            worktree_base,
        }
    }

    /// Create a `WorktreeManager` with a custom worktree base directory.
    pub fn with_base(repo_root: PathBuf, worktree_base: PathBuf) -> Self {
        Self {
            repo_root,
            worktree_base,
        }
    }

    /// Returns the repo root path.
    pub fn repo_root(&self) -> &Path {
        &self.repo_root
    }

    /// Returns the worktree base directory.
    pub fn worktree_base(&self) -> &Path {
        &self.worktree_base
    }
}
```

### 3. Add `worktree_manager` field to `EffectServices`

In the `EffectServices` struct, add after the `affect_policy` field:

```rust
    /// Optional worktree manager for parallel task isolation.
    pub worktree_manager: Option<WorktreeManager>,
```

### 4. Update all construction sites

In the existing code that creates `EffectServices`, add `worktree_manager: None`.

There are two construction sites to update:

1. In the tests at the bottom of `effect_driver.rs` (around line 822), where
   `EffectServices` is constructed in `spawn_agent_applies_affect_modulation_to_model_request`:
   add `worktree_manager: None,` to the struct literal.

2. In `crates/roko-orchestrator/src/service_factory.rs` (line ~86), in the
   `effect_services()` method of `ServiceBundle`. Add `worktree_manager: None,`
   to the `EffectServices` struct literal. (This site will be updated to
   `Some(...)` in O1_02.)

### 5. Export from the module

Ensure `WorktreeManager` is visible to downstream crates by confirming it has
`pub` visibility. It is already defined in the `effect_driver` module which is
`pub` in `crates/roko-runtime/src/lib.rs`.
