# O1_02: Wire WorktreeManager construction in ServiceFactory

## Goal

Construct a `WorktreeManager` in `ServiceFactory::build()` and pass it into
the `ServiceBundle` so that `effect_services()` includes a live manager.

## Files to Edit

- `crates/roko-orchestrator/src/service_factory.rs` (primary)

## Also Read

- `crates/roko-runtime/src/effect_driver.rs` — `WorktreeManager` struct

## Steps

### 1. Add `WorktreeManager` to `ServiceBundle`

In `crates/roko-orchestrator/src/service_factory.rs`, add a new field to the
`ServiceBundle` struct (after `affect_policy`):

```rust
    /// Optional worktree manager for parallel task isolation.
    pub worktree_manager: Option<roko_runtime::effect_driver::WorktreeManager>,
```

### 2. Construct `WorktreeManager` in `ServiceFactory::build()`

In the `build()` method, just before the `Ok(ServiceBundle { ... })` return
(around line 201), add:

```rust
        let worktree_manager = Some(
            roko_runtime::effect_driver::WorktreeManager::new(config.workdir.clone())
        );
```

### 3. Pass into the `ServiceBundle` return

Add `worktree_manager,` to the `ServiceBundle { ... }` struct literal.

### 4. Update `effect_services()` to thread the manager through

In the `effect_services()` method of `ServiceBundle` (around line 84), change:

```rust
        EffectServices {
            default_model: self.model.clone(),
            model_caller,
            prompt_assembler: Arc::clone(&self.prompt_assembler),
            feedback_sink: Arc::clone(&self.feedback_sink),
            gate_runner: Arc::clone(&self.gate_runner),
            affect_policy: self.affect_policy.clone(),
            worktree_manager: self.worktree_manager.clone(),
        }
```

### 5. Import the type

Add to the import block at the top of `service_factory.rs`:

```rust
use roko_runtime::effect_driver::WorktreeManager;
```

Then simplify the field types to use `WorktreeManager` directly.

## Verification

```bash
cargo check -p roko-orchestrator
cargo test -p roko-runtime --lib
```

Both commands must succeed. No runtime behavior changes — this only threads
the construction.

## Constraints

- Do NOT change the `ServiceConfig` struct — the worktree manager is derived
  from `config.workdir`, not from a separate config field.
- Do NOT add any new crate dependencies to `roko-orchestrator/Cargo.toml`.
- The `WorktreeManager` is always created (not gated by a config flag) since
  it is cheap — just two `PathBuf` values.
