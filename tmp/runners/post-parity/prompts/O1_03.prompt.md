# O1_03: Worktree lifecycle management (create/cleanup)

## Goal

Implement `create_worktree` and `remove_worktree` on `WorktreeManager` using
`git worktree add/remove` via `tokio::process::Command`.

## Files to Edit

- `crates/roko-runtime/src/effect_driver.rs` (primary)

## Also Read

## Steps

### 1. Add `create_worktree` method

In the `impl WorktreeManager` block in `crates/roko-runtime/src/effect_driver.rs`,
add:

```rust
    /// Create a new worktree for the given task id.
    ///
    /// Runs `git worktree add <worktree_base>/<task_id> -b roko/<task_id>` from
    /// `repo_root`. Returns the absolute path to the new worktree directory.
    ///
    /// If the branch already exists, uses `--no-track` to avoid conflicts.
    pub async fn create_worktree(&self, task_id: &str) -> Result<PathBuf> {
        let worktree_dir = self.worktree_base.join(task_id);
        let branch_name = format!("roko/{task_id}");

        tokio::fs::create_dir_all(&self.worktree_base).await?;

        let output = tokio::process::Command::new("git")
            .args([
                "worktree", "add",
                &worktree_dir.display().to_string(),
                "-b", &branch_name,
            ])
            .current_dir(&self.repo_root)
            .output()
            .await?;

        if !output.status.success() {
            let stderr = String::from_utf8_lossy(&output.stderr);
            // Branch may already exist — retry without -b
            if stderr.contains("already exists") {
                let retry = tokio::process::Command::new("git")
                    .args([
                        "worktree", "add",
                        &worktree_dir.display().to_string(),
                        &branch_name,
                    ])
                    .current_dir(&self.repo_root)
                    .output()
                    .await?;
                if !retry.status.success() {
                    let err = String::from_utf8_lossy(&retry.stderr);
                    return Err(format!("git worktree add failed: {err}").into());
                }
            } else {
                return Err(format!("git worktree add failed: {stderr}").into());
            }
        }

        Ok(worktree_dir)
    }
```

### 2. Add `remove_worktree` method

```rust
    /// Remove a worktree by task id.
    ///
    /// Runs `git worktree remove --force <path>` from `repo_root`.
    /// Errors are logged but do not propagate — cleanup is best-effort.
    pub async fn remove_worktree(&self, task_id: &str) -> Result<()> {
        let worktree_dir = self.worktree_base.join(task_id);

        if !worktree_dir.exists() {
            return Ok(());
        }

        let output = tokio::process::Command::new("git")
            .args([
                "worktree", "remove", "--force",
                &worktree_dir.display().to_string(),
            ])
            .current_dir(&self.repo_root)
            .output()
            .await?;

        if !output.status.success() {
            let stderr = String::from_utf8_lossy(&output.stderr);
            tracing::warn!(
                task_id,
                path = %worktree_dir.display(),
                "worktree removal failed: {stderr}"
            );
        }

        Ok(())
    }
```

### 3. Add `list_worktrees` method

```rust
    /// List active worktrees managed by this instance.
    ///
    /// Returns a list of task ids with active worktree directories.
    pub async fn list_worktrees(&self) -> Vec<String> {
        let Ok(mut entries) = tokio::fs::read_dir(&self.worktree_base).await else {
            return Vec::new();
        };

        let mut ids = Vec::new();
        while let Ok(Some(entry)) = entries.next_entry().await {
            if entry.path().is_dir() {
                if let Some(name) = entry.file_name().to_str() {
                    ids.push(name.to_string());
                }
            }
        }
        ids
    }
```

### 4. Add `cleanup_all` method

```rust
    /// Remove all worktrees created by this manager.
    ///
    /// Called at the end of a plan run to ensure no stale worktrees remain.
    pub async fn cleanup_all(&self) {
        let ids = self.list_worktrees().await;
        for id in ids {
            let _ = self.remove_worktree(&id).await;
        }
    }
```
