# O1_04: Add parallel config to WorkflowConfig

## Goal

Extend `WorkflowConfig` with fields that control parallel task execution:
`max_parallel_tasks` and `use_worktrees`. These are configuration-only
changes with no runtime effect yet.

## Files to Edit

- `crates/roko-runtime/src/pipeline_state.rs` (primary)

## Steps

### 1. Add fields to `WorkflowConfig`

In the `WorkflowConfig` struct (around line 37 of `pipeline_state.rs`), add
two new fields after `max_autofix_attempts`:

```rust
    /// Maximum number of independent tasks to run in parallel.
    /// 1 = sequential (default). Must be >= 1.
    pub max_parallel_tasks: u32,
    /// Whether to create per-task git worktrees for isolation.
    /// Only meaningful when `max_parallel_tasks > 1`.
    pub use_worktrees: bool,
```

### 2. Update `Default` impl

In the `Default for WorkflowConfig` impl (calls `Self::standard()`), the
`standard()` method already covers the default. Update `standard()`:

```rust
    pub fn standard() -> Self {
        Self {
            has_strategy: false,
            has_review: true,
            max_iterations: 2,
            max_autofix_attempts: 2,
            max_parallel_tasks: 1,
            use_worktrees: false,
        }
    }
```

### 3. Update `express()` and `full()` constructors

Add the new fields to both:

```rust
    // In express():
    max_parallel_tasks: 1,
    use_worktrees: false,

    // In full():
    max_parallel_tasks: 1,
    use_worktrees: false,
```

### 4. Add TOML parsing for new fields

In the `WorkflowConfigToml` struct (around line 54), add:

```rust
    max_parallel_tasks: Option<u32>,
    use_worktrees: Option<bool>,
```

In `parse_workflow_config_toml`, initialize both to `None` in the constructor.

In the key match block (around line 216), add:

```rust
    "max_parallel_tasks" => {
        workflow.max_parallel_tasks = Some(parse_toml_u32(value, line_number)?);
    }
    "use_worktrees" => {
        workflow.use_worktrees = Some(parse_toml_bool(value, line_number)?);
    }
```

In `WorkflowConfig::from_toml_str`, after the existing override applications,
add:

```rust
        if let Some(max_parallel) = raw.max_parallel_tasks {
            config.max_parallel_tasks = max_parallel.max(1);
        }
        if let Some(use_worktrees) = raw.use_worktrees {
            config.use_worktrees = use_worktrees;
        }
```

### 5. Add tests

Add two tests:

```rust
    #[test]
    fn toml_parallel_config() {
        let cfg = WorkflowConfig::from_toml_str(
            "max_parallel_tasks = 4\nuse_worktrees = true"
        ).unwrap();
        assert_eq!(cfg.max_parallel_tasks, 4);
        assert!(cfg.use_worktrees);
    }

    #[test]
    fn parallel_defaults_to_sequential() {
        let cfg = WorkflowConfig::standard();
        assert_eq!(cfg.max_parallel_tasks, 1);
        assert!(!cfg.use_worktrees);
    }
```

## Verification

```bash
cargo test -p roko-runtime --lib
cargo check -p roko-runtime
```

All existing tests must still pass, plus the two new tests.

## Constraints

- Do NOT change `PipelineStateV2` — the parallel execution logic will be
  added to `WorkflowEngine` in O1_05.
- `max_parallel_tasks` must be clamped to >= 1 in the TOML parser.
- Checkpoint serialization works automatically since both fields are primitive
  types with `Serialize`/`Deserialize`.
