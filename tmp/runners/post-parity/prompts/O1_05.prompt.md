# O1_05: Concurrent task dispatch in WorkflowEngine

## Goal

When `WorkflowConfig::max_parallel_tasks > 1`, spawn multiple implementer
agents concurrently using `tokio::JoinSet`. Each task runs in its own worktree
(if `use_worktrees` is true) or in the shared workdir (if false).

## Files to Edit

- `crates/roko-runtime/src/workflow_engine.rs` (primary)

## Also Read

- `crates/roko-runtime/src/pipeline_state.rs` — `WorkflowConfig` struct
- `crates/roko-runtime/src/effect_driver.rs` — `WorktreeManager`, `EffectDriver`

## Steps

### 1. Add a parallel dispatch helper method to `WorkflowEngine`

Add a new method `spawn_parallel_agents` that fans out agent calls:

```rust
    /// Spawn up to `n` parallel implementer agents.
    ///
    /// When worktrees are available, each agent gets its own worktree via
    /// `WorktreeManager::create_worktree`. Otherwise all agents share the
    /// main workdir. Results are collected and the first successful output
    /// is used; if all fail, the last error is propagated.
    async fn spawn_parallel_agents(
        &self,
        n: u32,
        prompt: &str,
        context: Option<&str>,
        workdir: &std::path::Path,
    ) -> PipelineInput {
        use tokio::task::JoinSet;

        if n <= 1 {
            // Fall back to sequential single-agent dispatch
            let driver = self.create_effect_driver(workdir);
            return driver.spawn_agent("implementer", prompt, context).await;
        }

        let mut set = JoinSet::new();
        let prompt_owned = prompt.to_string();
        let context_owned = context.map(String::from);

        for i in 0..n {
            let task_prompt = prompt_owned.clone();
            let task_context = context_owned.clone();
            let services = self.clone_services();
            let task_workdir = workdir.to_path_buf();
            let run_id = format!("parallel_{i}");

            set.spawn(async move {
                let driver = EffectDriver::new(services, run_id, task_workdir);
                driver
                    .spawn_agent("implementer", &task_prompt, task_context.as_deref())
                    .await
            });
        }

        // Collect results: use first success, fall back to last error
        let mut last_error = None;
        while let Some(result) = set.join_next().await {
            match result {
                Ok(input @ PipelineInput::AgentCompleted { .. }) => return input,
                Ok(input @ PipelineInput::AgentFailed { .. }) => {
                    last_error = Some(input);
                }
                Ok(other) => {
                    last_error = Some(other);
                }
                Err(join_err) => {
                    last_error = Some(PipelineInput::AgentFailed {
                        error: format!("parallel task panicked: {join_err}"),
                    });
                }
            }
        }

        last_error.unwrap_or(PipelineInput::AgentFailed {
            error: "no parallel tasks completed".to_string(),
        })
    }
```

### 2. Add `clone_services` helper

```rust
    fn clone_services(&self) -> EffectServices {
        EffectServices {
            default_model: self.services.default_model.clone(),
            model_caller: Arc::clone(&self.services.model_caller),
            prompt_assembler: Arc::clone(&self.services.prompt_assembler),
            feedback_sink: Arc::clone(&self.services.feedback_sink),
            gate_runner: Arc::clone(&self.services.gate_runner),
            affect_policy: self.services.affect_policy.clone(),
            worktree_manager: self.services.worktree_manager.clone(),
        }
    }
```

### 3. Wire into the run loop

In `run_with_cancel`, when matching `PipelineOutput::SpawnImplementer`, check
if parallel dispatch is requested:

```rust
PipelineOutput::SpawnImplementer { prompt, context } => {
    if config.workflow.max_parallel_tasks > 1 {
        self.spawn_parallel_agents(
            config.workflow.max_parallel_tasks,
            prompt,
            context.as_deref(),
            &config.workdir,
        ).await
    } else {
        driver
            .spawn_agent("implementer", prompt, context.as_deref())
            .await
    }
}
```

### 4. Add `create_effect_driver` helper

Factor out the EffectDriver creation used in the single-agent path:

```rust
    fn create_effect_driver(&self, workdir: &std::path::Path) -> EffectDriver {
        EffectDriver::new(
            self.clone_services(),
            generate_run_id(),
            workdir.to_path_buf(),
        )
    }
```

## Verification

```bash
cargo check -p roko-runtime
cargo test -p roko-runtime --lib
```

Both must succeed. Existing tests continue to pass because they use
`max_parallel_tasks = 1` (the default).

## Constraints

- Do NOT change the `PipelineStateV2` state machine — parallel dispatch is
  transparent to it; it still sees a single `AgentCompleted` or `AgentFailed`.
- The first successful result wins; others are dropped.
- Worktree creation/cleanup integration is deferred to the caller (plan runner).
