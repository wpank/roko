# O1_06: Wire CLI plan-run to parallel config

## Goal

Parse `--parallel N` and `--worktrees` CLI flags in `roko run` and thread them
into the `WorkflowConfig` used by the workflow engine.

## Files to Edit

- `crates/roko-cli/src/run.rs` (primary)

## Also Read

- `crates/roko-runtime/src/pipeline_state.rs` â€” `WorkflowConfig` fields
- `crates/roko-runtime/src/workflow_engine.rs` â€” `WorkflowRunConfig`

## Steps

### 1. Find the `workflow_config_for_template` function

In `crates/roko-cli/src/run.rs`, locate the function that builds a
`WorkflowConfig` from a template string. It should look like:

```rust
fn workflow_config_for_template(template: &str) -> WorkflowConfig { ... }
```

### 2. Add a parameterized variant

Add a new function or extend the existing one:

```rust
fn workflow_config_for_template_with_parallel(
    template: &str,
    max_parallel: Option<u32>,
    use_worktrees: bool,
) -> WorkflowConfig {
    let mut config = workflow_config_for_template(template);
    if let Some(n) = max_parallel {
        config.max_parallel_tasks = n.max(1);
    }
    config.use_worktrees = use_worktrees;
    config
}
```

### 3. Thread through the call chain

Find where `run_workflow_engine_report` or `run_workflow_engine_report_with_hub`
is called and ensure the parallel config reaches the `WorkflowRunConfig`.

The key call site is `run_workflow_engine_report_with_hub` (around line 604).
Add optional parameters:

```rust
pub async fn run_workflow_engine_report_with_hub(
    prompt: &str,
    workdir: &std::path::Path,
    workflow_template: &str,
    enabled_gates: Vec<String>,
    shell_gates: Vec<CoreShellGateCommand>,
    external_hub: Option<&StateHub>,
    max_parallel: Option<u32>,
    use_worktrees: bool,
) -> anyhow::Result<WorkflowRunReport> {
```

Update the body to use `workflow_config_for_template_with_parallel`.

### 4. Update callers

Find all call sites of `run_workflow_engine_report_with_hub` and add the
default values `None, false` for the new parameters. Typical callers:
- `run_workflow_engine_report` (if it exists as a wrapper)
- Bench strategy runner
- ACP session runner

### 5. Wire CLI flags

In the `roko run` command handler (look for `clap` argument parsing or the
function that matches `RunCommand` or `RunArgs`), add:

```rust
// If using clap derive:
/// Maximum parallel agents (1 = sequential)
#[arg(long, default_value = "1")]
parallel: u32,

/// Use git worktrees for parallel isolation
#[arg(long)]
worktrees: bool,
```

Thread these values through to `run_workflow_engine_report_with_hub`.

Both must succeed. No runtime behavior change for the default case
(`parallel = 1, worktrees = false`).
