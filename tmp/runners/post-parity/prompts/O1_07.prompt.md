# O1_07: CumulativeContext struct and builder

## Goal

Define a `CumulativeContext` struct that accumulates cross-task state
(diff summaries, prior outputs, gate feedback) during a workflow run. This
struct will be injected into subsequent prompts to give later tasks awareness
of what earlier tasks produced.

## Files to Edit

- `crates/roko-runtime/src/workflow_engine.rs` (primary)

## Also Read

- `crates/roko-runtime/src/pipeline_state.rs` — `PipelineStateV2` fields

## Steps

### 1. Define `CumulativeContext`

Add near the top of `workflow_engine.rs` (after the existing structs):

```rust
/// Accumulated context from prior workflow steps.
///
/// Fed into the prompt assembly for subsequent implementation iterations so
/// later agents see what earlier passes produced.
#[derive(Debug, Clone, Default, Serialize, Deserialize)]
pub struct CumulativeContext {
    /// One entry per completed implementation pass.
    pub entries: Vec<ContextEntry>,
}

/// A single entry of cumulative context.
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct ContextEntry {
    /// Which iteration produced this entry.
    pub iteration: u32,
    /// Role of the agent that produced the output.
    pub role: String,
    /// Short summary of the diff produced (file list + line counts).
    pub diff_summary: Option<String>,
    /// Gate feedback from this iteration, if any.
    pub gate_feedback: Option<String>,
    /// Truncated agent output (first 500 chars).
    pub output_preview: Option<String>,
}
```

### 2. Add builder methods to `CumulativeContext`

```rust
impl CumulativeContext {
    /// Create a new empty context.
    pub fn new() -> Self {
        Self::default()
    }

    /// Record an implementation pass result.
    pub fn record_pass(
        &mut self,
        iteration: u32,
        role: &str,
        output: &str,
        diff_summary: Option<String>,
    ) {
        let preview = if output.len() > 500 {
            Some(format!("{}...", &output[..500]))
        } else if !output.is_empty() {
            Some(output.to_string())
        } else {
            None
        };

        self.entries.push(ContextEntry {
            iteration,
            role: role.to_string(),
            diff_summary,
            gate_feedback: None,
            output_preview: preview,
        });
    }

    /// Record gate feedback for the most recent entry.
    pub fn record_gate_feedback(&mut self, feedback: &str) {
        if let Some(last) = self.entries.last_mut() {
            last.gate_feedback = Some(feedback.to_string());
        }
    }

    /// Render the cumulative context as a markdown string suitable for
    /// injection into a system or user prompt.
    pub fn render(&self) -> Option<String> {
        if self.entries.is_empty() {
            return None;
        }

        let mut lines = vec!["## Prior Implementation Context".to_string()];

        for entry in &self.entries {
            lines.push(format!(
                "\n### Iteration {} ({})",
                entry.iteration, entry.role
            ));
            if let Some(ref diff) = entry.diff_summary {
                lines.push(format!("**Files changed:** {diff}"));
            }
            if let Some(ref preview) = entry.output_preview {
                lines.push(format!("**Output:** {preview}"));
            }
            if let Some(ref feedback) = entry.gate_feedback {
                lines.push(format!("**Gate feedback:** {feedback}"));
            }
        }

        Some(lines.join("\n"))
    }
}
```

### 3. Add `cumulative_context` field to the run loop state

In `run_with_cancel`, just after creating the `pipeline` variable, add:

```rust
let mut cumulative_context = CumulativeContext::new();
```

This variable will be populated in O1_08 and injected in O1_09.

## Verification

```bash
cargo check -p roko-runtime
cargo test -p roko-runtime --lib
```

Both must succeed. This is a data structure definition — no runtime behavior
changes.

## Constraints

- Do NOT modify `PipelineStateV2` — cumulative context is engine-level, not
  state-machine-level.
- The `render()` method returns `None` for empty contexts so callers can
  easily skip injection.
- `output_preview` is capped at 500 chars to prevent prompt bloat.
