# O1_08: Compute git diff summary after task

## Goal

After each `AgentCompleted` event in the workflow engine, compute a short git
diff summary (file names and line counts) and record it in the
`CumulativeContext`.

## Files to Edit

- `crates/roko-runtime/src/effect_driver.rs` (primary)

## Also Read

- `crates/roko-runtime/src/workflow_engine.rs` — `CumulativeContext`

## Steps

### 1. Add `diff_summary` method to `EffectDriver`

In `crates/roko-runtime/src/effect_driver.rs`, add a public method:

```rust
    /// Compute a short git diff summary for the working directory.
    ///
    /// Runs `git diff --stat HEAD` and returns the summary lines.
    /// Returns `None` if git is unavailable or the diff is empty.
    pub async fn diff_summary(&self) -> Option<String> {
        let result = tokio::process::Command::new("git")
            .args(["diff", "--stat", "HEAD"])
            .current_dir(&self.workdir)
            .output()
            .await;

        match result {
            Ok(output) if output.status.success() => {
                let stdout = String::from_utf8_lossy(&output.stdout).trim().to_string();
                if stdout.is_empty() {
                    None
                } else {
                    Some(stdout)
                }
            }
            _ => None,
        }
    }

    /// Compute a short list of changed file names.
    ///
    /// Runs `git diff --name-only HEAD` and returns the file list.
    /// Returns `None` if git is unavailable or no files changed.
    pub async fn changed_files_list(&self) -> Option<String> {
        let result = tokio::process::Command::new("git")
            .args(["diff", "--name-only", "HEAD"])
            .current_dir(&self.workdir)
            .output()
            .await;

        match result {
            Ok(output) if output.status.success() => {
                let stdout = String::from_utf8_lossy(&output.stdout).trim().to_string();
                if stdout.is_empty() {
                    None
                } else {
                    // Cap to first 20 files to prevent prompt bloat
                    let lines: Vec<&str> = stdout.lines().take(20).collect();
                    let truncated = if stdout.lines().count() > 20 {
                        format!("{}\n... and {} more", lines.join("\n"), stdout.lines().count() - 20)
                    } else {
                        lines.join("\n")
                    };
                    Some(truncated)
                }
            }
            _ => None,
        }
    }
```

### 2. Wire into the workflow engine

In `crates/roko-runtime/src/workflow_engine.rs`, in the `run_with_cancel`
method, find the `PipelineOutput::SpawnImplementer` match arm. After the
agent completes and produces an `AgentCompleted` input, add context recording.

Locate where `PipelineInput::AgentCompleted` is produced from the implementer
and add:

```rust
// After the agent call returns an AgentCompleted:
if let PipelineInput::AgentCompleted { ref output, files_changed } = input {
    let diff = driver.diff_summary().await;
    cumulative_context.record_pass(
        pipeline.iteration,
        "implementer",
        output,
        diff,
    );
}
```

This goes inside the match arm for `PipelineOutput::SpawnImplementer`, right
after the `driver.spawn_agent()` call returns.

### 3. Record gate feedback on failure

In the match arm for `PipelineOutput::RunGates`, when the result is
`PipelineInput::GateFailed`, record the feedback:

```rust
if let PipelineInput::GateFailed { ref output, .. } = input {
    cumulative_context.record_gate_feedback(output);
}
```

## Verification

```bash
cargo check -p roko-runtime
cargo test -p roko-runtime --lib
```

Both must succeed. Existing tests pass because they do not exercise git diffs
(the `diff_summary` methods return `None` in test environments without git).

## Constraints

- `diff_summary()` and `changed_files_list()` must be infallible — return
  `None` on any error.
- Cap file lists to 20 entries to prevent prompt bloat.
- Do NOT change `PipelineStateV2` — context accumulation is in the engine.
