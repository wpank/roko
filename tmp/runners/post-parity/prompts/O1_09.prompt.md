# O1_09: Inject cumulative context into prompts

## Goal

When `CumulativeContext` has entries, append its rendered markdown to the
`PromptSpec::gate_feedback` vector so subsequent implementation passes see
what prior passes produced.

## Files to Edit

- `crates/roko-compose/src/prompt_assembly_service.rs` (primary)

## Also Read

- `crates/roko-runtime/src/workflow_engine.rs` — `CumulativeContext::render()`
- `crates/roko-core/src/foundation.rs` — `PromptSpec` struct

## Steps

### 1. Add `cumulative_context` to `PromptSpec`

In `crates/roko-core/src/foundation.rs`, add a new field to `PromptSpec`:

```rust
    /// Rendered cumulative context from prior iterations.
    pub cumulative_context: Option<String>,
```

### 2. Update `PromptAssemblyService::assemble` to inject context

In `crates/roko-compose/src/prompt_assembly_service.rs`, inside the `assemble`
method, after building the system prompt sections and before calling
`builder.build()`, check for cumulative context:

```rust
        // Inject cumulative context from prior workflow iterations.
        if let Some(ref context_text) = spec.cumulative_context {
            if !context_text.is_empty() {
                builder = builder.with_context_section(
                    "cumulative_context",
                    context_text.clone(),
                );
                section_ids.push("cumulative_context".to_string());
            }
        }
```

If `SystemPromptBuilder` does not have a `with_context_section` method, use
the existing layer mechanism. Look for how gate feedback is injected (it uses
a similar pattern) and follow the same approach.

If the builder uses a `with_gate_feedback` or similar pattern, add the
cumulative context to the gate feedback vector:

```rust
        if let Some(ref context_text) = spec.cumulative_context {
            if !context_text.is_empty() {
                gate_feedback_lines.push(context_text.clone());
            }
        }
```

### 3. Wire from WorkflowEngine

In `crates/roko-runtime/src/workflow_engine.rs`, in the `run_with_cancel`
method, when building the `SpawnImplementer` action on iteration > 1, inject
the cumulative context.

Find where the implementer is spawned and the context/user prompt is prepared.
The `EffectDriver::spawn_agent` takes an optional `context: Option<&str>`.

Before calling `spawn_agent`, append the cumulative context to the `context`
parameter:

```rust
PipelineOutput::SpawnImplementer { prompt, context } => {
    let enriched_context = match (context.as_deref(), cumulative_context.render()) {
        (Some(ctx), Some(cumul)) => Some(format!("{ctx}\n\n{cumul}")),
        (Some(ctx), None) => Some(ctx.to_string()),
        (None, Some(cumul)) => Some(cumul),
        (None, None) => None,
    };
    driver
        .spawn_agent("implementer", prompt, enriched_context.as_deref())
        .await
}
```

## Verification

```bash
cargo check -p roko-compose
cargo check -p roko-runtime
cargo test -p roko-runtime --lib
cargo test -p roko-compose --lib
```

All four must succeed.

## Constraints

- The cumulative context is injected as a `context` string (the existing
  agent `context` parameter), not as a new system prompt layer.
- Empty cumulative contexts produce `None` and are not injected.
- Do NOT change the `SystemPromptBuilder` API in this task — use existing
  injection points.
