# OB_02: Add OpenTelemetry dependencies to workspace

## Task
Add `opentelemetry`, `opentelemetry-otlp`, and `tracing-opentelemetry` dependencies to the workspace `Cargo.toml` so crates can opt in to OTLP export.

## Runner Context
Runner OB_A (Observability: Tracing), batch 2 of 4. No dependencies.

## Problem
The codebase uses `tracing` extensively but has no way to export spans to an OpenTelemetry collector. Adding workspace-level deps lets individual crates opt in behind a feature flag.

## Current Code

The workspace `Cargo.toml` at the repo root defines shared dependencies. Check the `[workspace.dependencies]` section for existing tracing deps.

## Exact Changes

### Step 1: Add to `[workspace.dependencies]` in root Cargo.toml

```toml
# OpenTelemetry (opt-in via crate features)
opentelemetry = { version = "0.28", default-features = false, features = ["trace"] }
opentelemetry_sdk = { version = "0.28", default-features = false, features = ["trace", "rt-tokio"] }
opentelemetry-otlp = { version = "0.28", default-features = false, features = ["trace", "tonic"] }
tracing-opentelemetry = { version = "0.28" }
```

Check the latest compatible versions. The key constraint is compatibility with the existing `tracing` version in the workspace.

### Step 2: Add optional deps to roko-runtime

In `crates/roko-runtime/Cargo.toml`:

```toml
[dependencies]
opentelemetry = { workspace = true, optional = true }
opentelemetry_sdk = { workspace = true, optional = true }
opentelemetry-otlp = { workspace = true, optional = true }
tracing-opentelemetry = { workspace = true, optional = true }

[features]
default = []
otel = ["opentelemetry", "opentelemetry_sdk", "opentelemetry-otlp", "tracing-opentelemetry"]
```

### Step 3: Add optional deps to roko-agent

In `crates/roko-agent/Cargo.toml`:

```toml
[dependencies]
opentelemetry = { workspace = true, optional = true }
tracing-opentelemetry = { workspace = true, optional = true }

[features]
default = []
otel = ["opentelemetry", "tracing-opentelemetry"]
```

## Write Scope
- `Cargo.toml` (workspace root)
- `crates/roko-runtime/Cargo.toml`
- `crates/roko-agent/Cargo.toml`

## Read-Only Context
- `crates/roko-runtime/Cargo.toml` (existing deps)
- `crates/roko-agent/Cargo.toml` (existing deps)

## Verify
```bash
cargo check --workspace 2>&1 | head -30
cargo check -p roko-runtime --features otel 2>&1 | head -30
```

## Acceptance Criteria
- OpenTelemetry deps in workspace-level dependencies
- `otel` feature flag on roko-runtime and roko-agent
- Default build does NOT pull in OpenTelemetry (feature-gated)
- `cargo check --workspace` passes without the feature
- `cargo check -p roko-runtime --features otel` passes

## Do NOT
- Enable `otel` by default
- Add OpenTelemetry to crates that do not need it
- Pin specific versions without checking compatibility with existing tracing
