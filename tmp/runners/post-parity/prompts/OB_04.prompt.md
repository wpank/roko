# OB_04: Lazy event serialization in JSONL logger

## Task
Defer JSON serialization in `JsonlLogger` until the write is actually performed, avoiding wasted serialization when the logger is disabled or the event is filtered.

## Runner Context
Runner OB_A (Observability: Tracing), batch 4 of 4. No dependencies.

## Problem
`JsonlLogger::write_event()` serializes every `RuntimeEvent` to JSON even when the writer is not initialized or the event would be filtered. In high-throughput scenarios (many tool calls per second), this wastes CPU on serialization that is never written.

## Current Code

**JsonlLogger::write_event** -- `crates/roko-runtime/src/jsonl_logger.rs:62-82`:
```rust
fn write_event(&self, event: &RuntimeEvent) -> std::io::Result<()> {
    self.ensure_writer()?;

    let envelope = RuntimeEventEnvelope::new(
        event.run_id(),
        self.seq.fetch_add(1, Ordering::Relaxed),
        "jsonl_logger",
        event.clone(),
    );

    let json = serde_json::to_string(&envelope)
        .map_err(|err| std::io::Error::new(std::io::ErrorKind::InvalidData, err))?;

    let mut writer = self.writer.lock()
        .unwrap_or_else(std::sync::PoisonError::into_inner);
    if let Some(ref mut w) = *writer {
        writeln!(w, "{json}")?;
        // ...
    }
    Ok(())
}
```

The event is cloned and serialized before checking if the writer is available.

## Exact Changes

### Step 1: Reorder to avoid unnecessary serialization

Move the writer availability check before serialization:

```rust
fn write_event(&self, event: &RuntimeEvent) -> std::io::Result<()> {
    // Check writer availability BEFORE serialization
    self.ensure_writer()?;

    let mut writer = self.writer.lock()
        .unwrap_or_else(std::sync::PoisonError::into_inner);

    let w = match writer.as_mut() {
        Some(w) => w,
        None => return Ok(()), // no writer, skip serialization
    };

    let seq = self.seq.fetch_add(1, Ordering::Relaxed);
    let envelope = RuntimeEventEnvelope::new(
        event.run_id(),
        seq,
        "jsonl_logger",
        event.clone(),
    );

    let json = serde_json::to_string(&envelope)
        .map_err(|err| std::io::Error::new(std::io::ErrorKind::InvalidData, err))?;

    writeln!(w, "{json}")?;
    // flush periodically, not every write
    if seq % 16 == 0 {
        w.flush()?;
    }

    Ok(())
}
```

### Step 2: Add event filtering

```rust
/// Optional filter for events. When set, only events matching the filter
/// are serialized and written.
pub struct JsonlLogger {
    path: PathBuf,
    seq: AtomicU64,
    writer: Mutex<Option<std::io::BufWriter<std::fs::File>>>,
    /// Minimum event severity to log. Events below this are silently dropped.
    min_severity: Option<EventSeverity>,
}

/// Event severity for filtering.
#[derive(Debug, Clone, Copy, PartialEq, Eq, PartialOrd, Ord)]
pub enum EventSeverity {
    Debug,
    Info,
    Warn,
    Error,
}
```

### Step 3: Add builder method

```rust
impl JsonlLogger {
    /// Set a minimum severity filter. Events below this level are not serialized.
    pub fn with_min_severity(mut self, severity: EventSeverity) -> Self {
        self.min_severity = Some(severity);
        self
    }
}
```

## Acceptance Criteria
- Serialization deferred until writer is confirmed available
- Periodic flush (every 16 writes) instead of per-write flush
- Optional `EventSeverity` filter for dropping low-severity events
- Existing API unchanged (backward compatible)
- No new dependencies

## Do NOT
- Change the `RuntimeEventEnvelope` format
- Remove the `ensure_writer` method
- Break the `EventConsumer` trait implementation
