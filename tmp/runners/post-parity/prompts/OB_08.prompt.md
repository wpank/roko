# OB_08: WebSocket feed endpoint for live events

## Task
Add a WebSocket endpoint to `roko-serve` that streams `RuntimeEvent`s to connected clients in real time, enabling live dashboards.

## Runner Context
Runner OB_B (Observability: Dashboard), batch 4 of 4. No dependencies.

## Problem
The HTTP API has SSE endpoints but no WebSocket feed for live events. WebSocket connections are bidirectional and more efficient for high-frequency event streams. The TUI already has a WS client (`tui/ws_client.rs`) but the server-side feed endpoint is incomplete.

## Current Code

**WebSocket route module** -- `crates/roko-serve/src/routes/ws.rs`:
Check what exists. The routes module declares `mod ws` but the implementation may be partial.

**EventBus** -- `crates/roko-runtime/src/event_bus.rs`:
Has `subscribe()` returning a `broadcast::Receiver`.

**RuntimeEvent** -- `crates/roko-core/src/runtime_event.rs`:
The event type that flows through the bus. Implements `Serialize`.

## Exact Changes

### Step 1: Implement WebSocket feed handler

In `crates/roko-serve/src/routes/ws.rs`, add or complete the feed handler:

```rust
use axum::extract::ws::{Message, WebSocket, WebSocketUpgrade};
use axum::extract::State;
use axum::response::IntoResponse;
use std::sync::Arc;
use tokio::sync::broadcast;
use tracing::{debug, warn};

use crate::state::AppState;

/// `GET /ws/events` -- WebSocket feed of RuntimeEvents.
pub async fn events_feed(
    ws: WebSocketUpgrade,
    State(state): State<Arc<AppState>>,
) -> impl IntoResponse {
    ws.on_upgrade(|socket| handle_events_socket(socket, state))
}

async fn handle_events_socket(mut socket: WebSocket, state: Arc<AppState>) {
    debug!("WebSocket client connected to /ws/events");

    let mut rx = state.event_bus.subscribe();

    loop {
        tokio::select! {
            result = rx.recv() => {
                match result {
                    Ok(envelope) => {
                        let json = match serde_json::to_string(&envelope) {
                            Ok(j) => j,
                            Err(err) => {
                                warn!(%err, "failed to serialize event for WS");
                                continue;
                            }
                        };
                        if socket.send(Message::Text(json.into())).await.is_err() {
                            debug!("WebSocket client disconnected");
                            break;
                        }
                    }
                    Err(broadcast::error::RecvError::Lagged(n)) => {
                        warn!(n, "WebSocket client lagged, dropped events");
                    }
                    Err(broadcast::error::RecvError::Closed) => {
                        debug!("Event bus closed, closing WebSocket");
                        break;
                    }
                }
            }
            msg = socket.recv() => {
                match msg {
                    Some(Ok(Message::Close(_))) | None => {
                        debug!("WebSocket client closed connection");
                        break;
                    }
                    Some(Ok(Message::Ping(data))) => {
                        let _ = socket.send(Message::Pong(data)).await;
                    }
                    _ => {} // ignore other messages
                }
            }
        }
    }
}
```

In the router builder (likely in `routes/mod.rs` or `ws.rs`):

## Acceptance Criteria
- `GET /ws/events` upgrades to WebSocket
- Each `RuntimeEvent` from the event bus is serialized to JSON and sent
- Handles client disconnect gracefully
- Handles event bus lagging with warning (not crash)
- Responds to Ping with Pong

## Do NOT
- Remove existing SSE endpoints
- Change the EventBus API
- Add authentication (that is a separate concern)
