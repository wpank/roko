# OB_11: Latency surge detector

## Task
Add a latency surge detector that monitors model call latency and alerts when response times exceed a configurable multiple of the running average.

## Runner Context
Runner OB_C (Observability: Alerting), batch 3 of 3. No dependencies.

## Problem
Latency surges (e.g., a model suddenly taking 30s instead of 3s) can indicate provider degradation, rate limiting, or oversized prompts. There is no automated detection -- operators discover latency issues after the fact.

## Current Code

**LatencyRegistry** -- `crates/roko-learn/src/latency.rs`:
Tracks per-provider latency stats but does not emit alerts.

## Exact Changes

### Step 1: Add latency surge detector

In `crates/roko-runtime/src/event_bus.rs` (alongside other detectors):

```rust
/// Detects latency surges using EMA comparison.
pub struct LatencySurgeDetector {
    /// EMA of latency in milliseconds.
    ema_ms: f64,
    /// EMA decay factor.
    alpha: f64,
    /// Alert when latency exceeds this multiple of EMA.
    surge_multiplier: f64,
    /// Minimum samples before alerting.
    min_samples: usize,
    /// Current sample count.
    sample_count: usize,
    /// Absolute minimum latency (ms) below which no alert fires.
    /// Avoids false positives when EMA is very low.
    min_alert_latency_ms: u64,
}

impl LatencySurgeDetector {
    pub fn new(surge_multiplier: f64, min_alert_latency_ms: u64) -> Self {
        Self {
            ema_ms: 0.0,
            alpha: 0.1,
            surge_multiplier,
            min_samples: 10,
            sample_count: 0,
            min_alert_latency_ms,
        }
    }

    /// Default detector: alert on 3x EMA, minimum 5000ms absolute.
    pub fn with_defaults() -> Self {
        Self::new(3.0, 5000)
    }

    /// Feed a latency observation. Returns alert if surge detected.
    pub fn observe(&mut self, latency_ms: u64, model: &str) -> Option<String> {
        self.sample_count += 1;
        let latency = latency_ms as f64;

        if self.sample_count <= 1 {
            self.ema_ms = latency;
            return None;
        }

        // Update EMA
        self.ema_ms = self.alpha * latency + (1.0 - self.alpha) * self.ema_ms;

        // Check for surge
        if self.sample_count >= self.min_samples
            && latency_ms >= self.min_alert_latency_ms
            && latency > self.ema_ms * self.surge_multiplier
        {
            Some(format!(
                "Latency surge: {latency_ms}ms is {:.1}x the average {:.0}ms (model: {model})",
                latency / self.ema_ms,
                self.ema_ms,
            ))
        } else {
            None
        }
    }

    /// Current EMA latency in milliseconds.
    pub fn current_ema_ms(&self) -> f64 {
        self.ema_ms
    }
}
```
