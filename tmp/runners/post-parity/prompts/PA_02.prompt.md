# PA_02: Wire shared HTTP client into ClaudeApiAgent and provider backends

## Task
Change all LLM provider backends that create their own `ReqwestPoster` to accept `Arc<dyn HttpPoster>` from the caller.

## Runner Context
Runner PA, batch 2 of 5. Depends on PA_01.

## Problem
Each provider backend creates a fresh `ReqwestPoster::new()` per construction, bypassing connection pooling. After PA_01 provides the factory, we wire it into the agent layer.

## Exact Changes

### Step 1: Find all `ReqwestPoster::new()` in `crates/roko-agent/src/`

Search pattern: `ReqwestPoster::new()` and `reqwest::Client::new()` in `crates/roko-agent/src/`

For each site, replace the inline construction with an accepted parameter.

### Step 2: Update provider structs to accept poster

For each provider that constructs its own poster, change the constructor to accept `Arc<dyn HttpPoster>`:

```rust
// BEFORE (example pattern)
impl SomeProvider {
    pub fn new(model: &str) -> Self {
        Self {
            poster: Arc::new(ReqwestPoster::new()),
            // ...
        }
    }
}

// AFTER
impl SomeProvider {
    pub fn new(model: &str, poster: Arc<dyn HttpPoster>) -> Self {
        Self {
            poster,
            // ...
        }
    }
}
```

### Step 3: Update call sites in `crates/roko-agent/src/`

Any internal call that constructs a provider and passes `ReqwestPoster::new()` should instead accept the poster from its caller. Propagate the parameter upward.

### Step 4: Update call sites in `crates/roko-cli/src/`

Search `crates/roko-cli/src/` for construction of any provider type modified above. At each site, pass the shared client (from `roko_agent::shared_http_client()` or threaded from startup).
