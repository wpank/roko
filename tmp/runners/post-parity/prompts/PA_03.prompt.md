# PA_03: Wire shared HTTP client into roko-serve AppState

## Task
Replace the bare `reqwest::Client::new()` in roko-serve startup with `roko_agent::shared_http_client_from()` and ensure all route handlers use `state.http_client`.

## Runner Context
Runner PA, batch 3 of 5. Depends on PA_01.

## Problem
`roko-serve` already has `http_client: reqwest::Client` in `AppState` (state.rs:397), created via `reqwest::Client::new()` (state.rs:500). But many route handlers create additional clients instead of using the shared one.

## Current Code

**AppState struct** — `crates/roko-serve/src/state.rs:397`:
```rust
pub struct AppState {
    // ... other fields
    pub http_client: reqwest::Client,
```

**AppState construction** — `crates/roko-serve/src/state.rs:500`:
```rust
let http_client = reqwest::Client::new();
```

**Routes already using state.http_client correctly:**
- `routes/agents.rs:1040` — `state.http_client.get(url)`
- `routes/agents.rs:1133` — `state.http_client.post(url)`
- `routes/aggregator.rs:967` — `state.http_client.get(&card_uri)`

**Routes creating their own clients (NEEDS FIX):**
- `routes/secrets.rs:182,206,228,247` — 4 separate `reqwest::Client::new()` calls
- `routes/deployments.rs:399` — per-request `reqwest::Client::new()`
- `feedback.rs:67` — `Client::builder().user_agent(...).build()?`
- `relay.rs:272` — `reqwest::Client::new()`
- `deploy/railway_api.rs:110` — `reqwest::Client::new()` in constructor

## Exact Changes

### Step 1: Change AppState to use `Arc<ReqwestPoster>` (optional upgrade)

In `crates/roko-serve/src/state.rs:500`, replace:
```rust
// BEFORE
let http_client = reqwest::Client::new();

// AFTER
let http_client = reqwest::Client::builder()
    .pool_max_idle_per_host(20)
    .build()
    .unwrap_or_else(|_| reqwest::Client::new());
```

This is a minor improvement — the real wins come from Step 2.

### Step 2: Fix route handlers that create their own clients

**`routes/secrets.rs` — lines 182, 206, 228, 247:**
```rust
// BEFORE (at each of the 4 sites)
let client = reqwest::Client::new();

// AFTER (at each of the 4 sites)
let client = &state.http_client;
```

**`routes/deployments.rs:399`:**
```rust
// BEFORE
let client = reqwest::Client::new();

// AFTER
let client = &state.http_client;
```

**`feedback.rs:67`:**
```rust
// BEFORE
let client = Client::builder().user_agent("roko-serve-feedback/0.1").build()?;

// AFTER — use state.http_client with per-request header
let client = &state.http_client;
// Add .header("User-Agent", "roko-serve-feedback/0.1") to the request builder
```

**`relay.rs:272`:**
```rust
// BEFORE
let client = reqwest::Client::new();

// AFTER
let client = &state.http_client;
```

**`deploy/railway_api.rs:110`:**
```rust
// BEFORE
pub fn new(token: String) -> Self {
    Self { client: reqwest::Client::new(), token }
}

// AFTER — accept client parameter
pub fn new(token: String, client: reqwest::Client) -> Self {
    Self { client, token }
}
```

### Step 3: Thread state.http_client to RailwayApiBackend construction

Find where `RailwayApiBackend::new()` is called and pass `state.http_client.clone()`.
