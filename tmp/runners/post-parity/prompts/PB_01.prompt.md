# PB_01: Document current chat dispatch data flow

## Task
Read and document the current chat dispatch path from user input to LLM call, identifying every gap where session state is lost.

## Runner Context
Runner PB, batch 1 of 8. Depends on PA_01.

## Problem
Chat dispatch is broken because session state (system prompt, tools, history, model, effort) is stored in `ChatAgentSession` but never makes it to the actual LLM call. This batch is documentation-only — map the exact data flow.
