# PB_03: Wire system_message into dispatch request

## Task
Ensure that when `/system` sets a system message, it reaches `ChatAgentSession.system_prompt` and flows through `build_dispatch_request()`.

## Runner Context
Runner PB, batch 3 of 8. Depends on PB_02.

## Problem

`/system` handler in `chat_inline.rs:2125-2141` stores the message:
```rust
session.system_message = Some(msg.to_string());
```

But `ChatAgentSession.system_prompt` (chat_session.rs:51) is set once at construction and never updated. The `/system` command writes to a **different field** (`session.system_message` on the REPL `ChatSession` struct at chat_inline.rs:742) that never reaches `ChatAgentSession`.

## Current Code

**`/system` handler** — `chat_inline.rs:2125-2141`:
```rust
_ if cmd.starts_with("/system") => {
    let msg = cmd.strip_prefix("/system").unwrap().trim();
    if msg.is_empty() {
        // display current
    } else {
        session.system_message = Some(msg.to_string());  // Writes to ChatSession, NOT ChatAgentSession
    }
}
```

**ChatSession** (REPL state) — `chat_inline.rs:742-753` — may or may not have a `system_message` field. Check if it exists or if it's on a sub-struct.

**ChatAgentSession.system_prompt** — `chat_session.rs:51`:
```rust
pub system_prompt: String,
```
Set once at line 80: `let system_prompt = build_chat_system_prompt(&workdir, config);`

## Exact Changes

### Step 1: Find where REPL stores `/system` value

In `chat_inline.rs`, search for `system_message`. Determine whether it's a field on the REPL `ChatSession` struct (line 742) or if the REPL holds a `ChatAgentSession` reference.

### Step 2: Bridge the REPL `/system` value to ChatAgentSession

Option A — If the REPL holds a `ChatAgentSession` reference:
```rust
// In /system handler, update the agent session directly:
agent_session.system_prompt = msg.to_string();
```

Option B — If the REPL doesn't hold a `ChatAgentSession`:
Add a `system_prompt_override: Option<String>` field to the REPL `ChatSession` struct. In `build_dispatch_request()`, prefer the override:
```rust
system_prompt: self.system_prompt_override.clone()
    .unwrap_or_else(|| self.agent_session.system_prompt.clone()),
```

### Step 3: Handle `/system clear`

In the `/system` handler, add:
```rust
if msg == "clear" || msg.is_empty() {
    // Reset to auto-built prompt
    session.system_prompt_override = None;
} else {
    session.system_prompt_override = Some(msg.to_string());
}
```
