# PB_05: Wire message history into dispatch request

## Task
Ensure conversation history from `ChatAgentSession.api_history` flows through dispatch and that responses are appended back.

## Runner Context
Runner PB, batch 5 of 8. Depends on PB_02.

## Problem
`ChatAgentSession.api_history: Vec<ChatMessage>` (chat_session.rs:60) starts empty and is never populated. Multi-turn chat sends only the latest message, losing all context.

## Current Code

**History field** — `chat_session.rs:60`:

**Initialized empty** — `chat_session.rs:93`:
```rust

```

**session_id field** — `chat_session.rs:59`:
```rust
pub session_id: Option<String>,
```
Always `None` — never captured from Claude CLI response.

**Claude CLI session_id capture** — `crates/roko-agent/src/provider/claude_cli/stream.rs:23-29`:
```rust
pub enum ClaudeStreamEvent {
    System(ClaudeSystemEvent),  // Contains session_id

```
The `SystemInit` event at stream.rs:134-221 contains `session_id` and `model`.

**REPL response handling** — `chat_inline.rs:1350-1415`:
Receives `DispatchResult` with text, model, tokens — but does NOT:
1. Append user message to `api_history`
2. Append assistant response to `api_history`
3. Capture `session_id` from the response

## Exact Changes

### Step 1: Append user message before dispatch

In `chat_inline.rs`, find where `dispatch_prompt()` is called (around line 1459). Before the call, append the user message:

```rust
// Before dispatch_prompt() call:
agent_session.api_history.push(ChatMessage {
    role: "user".to_string(),
    content: prompt.to_string(),
});
```

### Step 2: Append assistant response after dispatch

In `chat_inline.rs`, find where `DispatchResult` is received (around line 1350-1415). After receiving the response:

```rust
// After receiving response:
agent_session.update_from_response(result.session_id, &result.text);
// update_from_response() was added in PB_02
```

### Step 3: Ensure `DispatchResult` includes `session_id`

Check `DispatchResult` struct definition. If it doesn't have a `session_id: Option<String>` field, add one. The CLI stream parser already extracts it from `SystemInit` events.

### Step 4: For CLI dispatch — use `--resume` with session_id

In the adapter method (PB_07), when building `ClaudeCliAgent`:
```rust
if let Some(sid) = &request.session_id {
    agent = agent.with_resume(sid.clone());
}
```

The `ClaudeCliAgent` already supports `--resume` at claude_cli_agent.rs:349-351.
