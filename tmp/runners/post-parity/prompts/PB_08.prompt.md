# PB_08: Wire one-shot (roko "prompt") through ChatAgentSession

## Task
Make `roko "prompt"` one-shot mode go through `ChatAgentSession` in single-turn mode, using the same dispatch path as interactive chat.

## Runner Context
Runner PB, batch 8 of 8. Depends on PB_07.

## Problem

**unified.rs:84-130** — `cmd_oneshot_inline()` has a separate dispatch chain:
```rust
#[cfg(feature = "legacy-orchestrate")]
let result = dispatch_direct::dispatch_prompt(&auth, &prompt).await;
#[cfg(not(feature = "legacy-orchestrate"))]
let result = crate::dispatch_v2::dispatch_via_model_call_service(&prompt).await;
```

This bypasses `ChatAgentSession` entirely — no system prompt, no tools, no MCP, no workspace context.

**unified.rs:23-79** — `cmd_unified_chat()` creates the interactive REPL which correctly uses `ChatAgentSession`. One-shot should use the same session in single-turn mode.

## Exact Changes

### Step 1: Rewrite `cmd_oneshot_inline()` in `unified.rs`

Find `cmd_oneshot_inline` (around line 84). Replace the body:

```rust
// BEFORE
pub async fn cmd_oneshot_inline(prompt: &str, config_path: Option<&Path>) -> Result<i32> {
    let auth = detect_auth();
    // ... direct dispatch without session
}

// AFTER
pub async fn cmd_oneshot_inline(prompt: &str, config_path: Option<&Path>) -> Result<i32> {
    let auth = detect_auth();
    if matches!(auth, AuthMethod::NeedsSetup) {
        print_setup_instructions();
        return Ok(1);
    }

    let workdir = std::env::current_dir().unwrap_or_else(|_| PathBuf::from("."));
    ensure_workspace(&workdir)?;
    let config = load_config_or_defaults(config_path, &workdir)?;

    // Use ChatAgentSession for full context
    let model_selection = resolve_model_selection(&config, &auth);
    let mut session = ChatAgentSession::new(&config, workdir, model_selection)?;

    // Add the prompt as a user message
    session.api_history.push(ChatMessage {
        role: "user".to_string(),
        content: prompt.to_string(),
    });

    // Dispatch through the same path as interactive chat
    let request = session.build_dispatch_request();
    let result = dispatch_chat_request(request).await;

    match result {
        Ok(dispatch_result) => {
            println!("{}", dispatch_result.text);
            Ok(0)
        }
        Err(e) => {
            eprintln!("error: {e:#}");
            Ok(1)
        }
    }
}
```

### Step 2: Ensure resolve_model_selection() exists
