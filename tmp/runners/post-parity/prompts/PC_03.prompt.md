# PC_03: Wire Claude CLI stream events to StreamingCallback

## Task
When using Claude CLI backend, pipe `MessageDelta` events through `StreamingCallback` as they arrive.

## Runner Context
Runner PC, batch 3 of 6. Depends on PC_02.

## Problem
`ClaudeCliAgent` spawns `claude --print --output-format stream-json`. The stream parser at `provider/claude_cli/stream.rs:134-221` extracts `MessageDelta { text }` events (line 164-165) but only concatenates them internally. No real-time forwarding.

## Current Code

**Stream parser** — `crates/roko-agent/src/provider/claude_cli/stream.rs:134-221`:
```rust
// Line 164-165: Text content block
ClaudeContentBlock::Text { text } => {
    events.push(AgentRuntimeEvent::MessageDelta { text: text.clone() });
}
```

**AgentRuntimeEvent::MessageDelta** — carries individual text deltas.

**Where the agent processes events** — search `claude_cli_agent.rs` for where `AgentRuntimeEvent::MessageDelta` is handled. The agent collects these into a String buffer and returns the final result.

**dispatch_prompt()** in `chat_inline.rs:1459-1496` — spawns a tokio task that calls the agent and sends the final result via mpsc channel. No streaming callback is passed.

## Exact Changes

### Step 1: Add callback parameter to dispatch

In the dispatch function created by PB_07 (`dispatch_chat_request` or equivalent), add a callback parameter:

```rust
async fn dispatch_chat_request(
    request: ChatDispatchRequest,
    callback: Option<Box<dyn StreamingCallback>>,
) -> Result<DispatchResult> { ... }
```

### Step 2: Wire callback into CLI agent execution

In `dispatch_via_cli()` (from PB_07), find where the agent runs. The `ClaudeCliAgent` likely has a `run()` or `execute()` method that processes stdout lines.

Find the loop that processes `AgentRuntimeEvent` variants. When `MessageDelta { text }` is encountered:

```rust
AgentRuntimeEvent::MessageDelta { text } => {
    // Existing: append to buffer
    buffer.push_str(&text);

    // NEW: forward to callback
    if let Some(cb) = &callback {
        cb.on_token(&text);
    }
}
```

### Step 3: If the agent doesn't expose runtime events

If `ClaudeCliAgent` processes events internally without exposing them, add a callback parameter to the run method:

```rust
// In claude_cli_agent.rs, modify the run method:
pub async fn run_with_callback(
    &self,
    prompt: &str,
    callback: Option<&dyn StreamingCallback>,
) -> Result<AgentOutput> {
    // In the stdout reading loop:
    for event in parse_stream_line(&line)? {
        match event {
            AgentRuntimeEvent::MessageDelta { text } => {
                buffer.push_str(&text);
                if let Some(cb) = callback {
                    cb.on_token(&text);
                }
            }
            // ... handle other events
        }
    }
}
```

### Step 4: Call `on_start()` and `on_complete()`

At the beginning of agent execution:
```rust
if let Some(cb) = &callback { cb.on_start(&self.model); }
```

After agent completes:
```rust
if let Some(cb) = &callback { cb.on_complete(token_count); }
```
