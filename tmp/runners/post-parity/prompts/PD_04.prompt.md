# PD_04: Wire /gate to write gate config to roko.toml

## Task
Make `/gate enable compile` actually write the gate configuration to `roko.toml`.

## Runner Context
Runner PD, batch 4 of 6. Depends on PD_01.

## Problem
`/gate` handler prints "update roko.toml" but doesn't write. This is the SC-2 anti-pattern: "Config writes go through the config system."

## Current Code

Search `chat_inline.rs` for the `/gate` handler. It should be in the command match block around lines 2279-2315.

The config system:
- `crates/roko-core/src/config/mod.rs` — `Config` struct with serialization
- Search for `save`, `write_config`, or `to_toml` methods on the Config struct
- Config is loaded via `Config::load()` or `Config::from_path()`

Gate configuration in roko.toml:
```toml

compile = true
test = true

```

## Exact Changes

### Step 1: Find the config write method

In `crates/roko-core/src/config/mod.rs`, search for methods that write config back to `roko.toml`. Expected patterns:
- `config.save()` or `config.write_to(path)`
- Or use `toml::to_string_pretty(&config)` + `fs::write()`

### Step 2: Update the /gate handler

Replace the print-only handler with actual config writes:

```rust
_ if cmd.starts_with("/gate") => {
    let arg = cmd.strip_prefix("/gate").unwrap().trim();
    let parts: Vec<&str> = arg.splitn(2, ' ').collect();

    match parts.as_slice() {
        ["list"] | [] => {
            // Show current gate config
            // Read from config.gates or equivalent
            term.push_lines(&[styled::continuation(theme, "gates", "current config:", None)])?;
            // List each gate and its status
        }
        ["enable", name] => {
            // Update config and write to roko.toml
            // Use config system's write method, NOT hand-written TOML
            term.push_lines(&[styled::continuation(theme, "gate",
                &format!("{name} enabled"), None)])?;
        }
        ["disable", name] => {
            // Same pattern as enable
            term.push_lines(&[styled::continuation(theme, "gate",
                &format!("{name} disabled"), None)])?;
        }
        _ => {
            term.push_lines(&[styled::continuation(theme, "gate",
                "usage: /gate list | /gate enable <name> | /gate disable <name>",
                None)])?;
        }
    }
}
```
