# PE_03: Add --dangerously-skip-permissions CLI flag for explicit opt-in

## Task
Add a `--dangerously-skip-permissions` CLI flag to `roko run` and `roko plan run` for explicit opt-in.

## Runner Context
Runner PE, batch 3 of 8. Depends on PE_02.

## Problem
After PE_02 flips all defaults to `false`, automated/CI usage needs explicit opt-in. The flag name is deliberately ugly.

## Current Code

**CLI arg parsing** — `crates/roko-cli/src/main.rs` or `crates/roko-cli/src/commands/`:
Uses `clap` for argument parsing. Find the `Run` and `PlanRun` command structs.

**`plan run` command** — `crates/roko-cli/src/commands/plan.rs`:
Around line 258-291, the `PlanRunArgs` struct. Search for the clap derive struct.

**`roko run` command** — `crates/roko-cli/src/run.rs` or `commands/run.rs`:
Find the `RunArgs` struct.

## Exact Changes

### Step 1: Add flag to `PlanRunArgs` in `commands/plan.rs`

Find the clap-derived struct for `plan run`. Add:
```rust
/// Skip Claude's permission system (UNSAFE — only for automated/CI usage).
/// Sets dangerously_skip_permissions=true for all spawned agents.
#[arg(long, default_value_t = false)]
dangerously_skip_permissions: bool,
```

### Step 2: Wire into RunConfig in `plan run` handler

In the `plan run` handler (around line 291):
```rust
let mut run_config = RunConfig::from_roko_config(workdir, plan_dir, config);
if args.dangerously_skip_permissions {
    run_config.dangerously_skip_permissions = true;
}
```

### Step 3: Add flag to `roko run` command

Find the run command args struct. Add the same flag:
```rust
#[arg(long, default_value_t = false)]
dangerously_skip_permissions: bool,
```

Wire into the dispatch options.

### Step 4: Add to `roko chat` if applicable

If `roko chat` also needs the flag (for interactive sessions with agent tools), add it there too.

### Step 5: Add config file support

In `crates/roko-core/src/config/mod.rs`, find the runner config section and add:
```rust
pub struct RunnerConfig {
    // ... existing fields
    pub dangerously_skip_permissions: Option<bool>,
}
```

This allows `[runner] dangerously_skip_permissions = true` in `roko.toml`.
