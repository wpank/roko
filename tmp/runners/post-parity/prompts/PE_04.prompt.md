# PE_04: Replace production AgentContract::permissive() with restricted fallback

## Task
Find and replace all production uses of `AgentContract::permissive()` with `load_for_role_with_mode()` using `RestrictedFallback`. Keep `permissive()` available for tests.

## Runner Context
Runner PE, batch 4 of 8. Depends on PE_01.

## Problem
SA-2 anti-pattern: "Permissive contracts in production." `AgentContract::permissive()` (contract.rs:71-83) creates a contract that allows everything. This is correct for tests but dangerous in production — a typo in a role name silently gives the agent unrestricted access.

## Current Code

**`permissive()` method** — `crates/roko-agent/src/safety/contract.rs:71-83`:
```rust
pub fn permissive(role: impl Into<String>) -> Self {
    Self {
        role: role.into(),
        ..Self::default()
    }
}
```

**`restricted()` method** — returns deny-all contract.

**`load_for_role_with_mode()` method** — `contract.rs:148-163`:
On error with `RestrictedFallback` mode, returns `Self::restricted(role_ref)` (line 158-163).

## Known Call Sites (VERIFIED — do not rely on a simple search)

1. **`contract.rs:71`** — Definition. KEEP.
2. **`safety/mod.rs:254`** — `SafetyLayer::with_defaults()` uses `AgentContract::permissive("default")` as the default contract. **PRODUCTION — MUST FIX.** This is the default constructor for SafetyLayer; gating `permissive()` behind `#[cfg(test)]` without fixing this would break compilation.
3. **`safety/mod.rs:873`** — `contract_for_role()` falls back to `AgentContract::permissive(role)` when a role override is configured but no bundled contract exists (`MissingAsset` error). **PRODUCTION — MUST FIX.**
4. **`safety/mod.rs:~1070`** — `permissive_layer()` test helper. Already in `#[cfg(test)]` module. OK.
5. **Other types** — `AllowlistGuard::permissive()`, `HallucinationDetector::permissive()` are different types, not targeted by this prompt. Ignore them.

**WARNING**: If you `#[cfg(test)]`-gate `permissive()` BEFORE fixing sites 2 and 3, the build will fail.

## Exact Changes

### Step 1: Fix SafetyLayer::with_defaults() (site 2)

```rust
// BEFORE (safety/mod.rs:254):
contract: AgentContract::permissive("default"),

// AFTER:
contract: AgentContract::load_for_role_with_mode(
    "default",
    ContractLoadMode::RestrictedFallback,
).unwrap_or_else(|_| AgentContract::restricted("default")),
```

### Step 2: Fix contract_for_role() fallback (site 3)

```rust
// BEFORE (safety/mod.rs:873):
Err(MissingAsset { .. }) => AgentContract::permissive(role),

// AFTER:
Err(MissingAsset { .. }) => {
    warn!(role = %role, "no contract found for role — using restricted fallback");
    AgentContract::restricted(role)
},
```

### Step 3: Gate permissive() behind test/test-helpers feature

ONLY after fixing sites 2 and 3:

```rust
/// Create a permissive contract for testing only.
/// **NEVER use in production** — use `load_for_role_with_mode()` instead.
#[cfg(any(test, feature = "test-helpers"))]
pub fn permissive(role: impl Into<String>) -> Self {
    Self { role: role.into(), ..Self::default() }
}
```

### Step 4: Add test-helpers feature to Cargo.toml if needed

Check if integration tests in other crates (e.g., `roko-learn/tests/`) call `permissive()`. If so, add to `crates/roko-agent/Cargo.toml`:

```toml

```

And in dependent crates' `Cargo.toml`:

### Step 5: Verify compilation
