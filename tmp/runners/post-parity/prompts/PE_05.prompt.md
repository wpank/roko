# PE_05: Verify contract_for_role uses RestrictedFallback, not permissive

## Task
Ensure `load_for_role_with_mode()` uses `ContractLoadMode::RestrictedFallback` at all call sites, never `Strict` without error handling.

## Runner Context
Runner PE, batch 5 of 8. Depends on PE_04.

## Problem
When a role's YAML contract file is missing, the system must fail-closed (deny-all), not fail-open (allow-all). AP-2: "Missing contract YAML → restricted fallback, never permissive."

## Current Code

**`load_for_role_with_mode()`** — `contract.rs:148-163`:
```rust
pub fn load_for_role_with_mode(role: impl AsRef<str>, mode: ContractLoadMode) -> Result<Self, ContractLoadError> {
    match Self::load_for_role(role_ref) {
        Ok(contract) => Ok(contract),
        Err(err) => match mode {
            ContractLoadMode::Strict => Err(err),
            ContractLoadMode::RestrictedFallback => {
                tracing::warn!(role = %role_ref, %err, "no contract for role; using restricted fallback");
                Ok(Self::restricted(role_ref))
            }
        }
    }
}
```
