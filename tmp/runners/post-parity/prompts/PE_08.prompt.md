# PE_08: Fix --share flag to error when unsupported by current engine

## Task
Make `--share` flag produce a clear error when used with the v2 engine instead of silently doing nothing.

## Runner Context
Runner PE, batch 8 of 8. No dependencies.

## Problem
`--share` flag is parsed but only works with legacy engine. The v2 engine silently ignores it. Users think they're sharing runs but nothing happens.

## Current Code

**Flag parsing** — `crates/roko-cli/src/run.rs`:
Search for `share` in the clap args struct. Find where it's checked.

**Engine selection** — search `run.rs` for `engine`, `Engine::Legacy`, `engine_from_config`.

## Exact Changes

### Step 1: Find the --share flag

In `run.rs`, search for the clap field:
```rust
#[arg(long)]
share: bool,
```

### Step 2: Add early validation

In the command handler, after parsing args but before execution:
```rust
if args.share && !is_legacy_engine {
    anyhow::bail!(
        "--share is not yet supported with the v2 engine. \
         Use --engine legacy for share functionality, or omit --share."
    );
}
```

### Step 3: Find `is_legacy_engine` check

Determine how the engine is selected. Look for:
- `cfg!(feature = "legacy-orchestrate")` compile-time check
- Runtime engine selection from config or CLI flag

Use the appropriate check:
```rust
let is_legacy = cfg!(feature = "legacy-orchestrate") && !args.use_v2_engine;
```
