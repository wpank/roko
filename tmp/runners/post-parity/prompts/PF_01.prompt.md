# PF_01: Change legacy-orchestrate default to off

## Task
Remove `legacy-orchestrate` from the default features in `roko-cli/Cargo.toml`.

## Runner Context
Runner PF, batch 1 of 3. No dependencies.

## Problem
`crates/roko-cli/Cargo.toml:16-17`:
```toml
[features]
default = ["legacy-orchestrate"]
legacy-orchestrate = []
```

orchestrate.rs is 21,577 lines of dead monolith code behind this feature flag. It's already gated with `#[cfg(feature = "legacy-orchestrate")]` throughout the codebase. Removing it from defaults means it stops compiling by default, saving build time and reducing binary size.

## Changes Required

1. Change `Cargo.toml`:
```toml
[features]
default = []
legacy-orchestrate = []
```

2. Verify that all `#[cfg(feature = "legacy-orchestrate")]` guards are in place. Check:
   - `crates/roko-cli/src/lib.rs` (lines 62, 83, 134)
   - `crates/roko-cli/src/run.rs` (16 guards)
   - `crates/roko-cli/src/unified.rs` (line 99)
   - `crates/roko-cli/src/dispatch_direct.rs` (line 10, entire file)

3. Ensure that without the feature, `cargo build -p roko-cli` should still compile. All the `#[cfg]` guards should already handle this. If any code outside the guards references orchestrate.rs symbols, add the missing guard.

## Write Scope

## Read-Only Context
