# PF_03: Ensure all cfg(legacy-orchestrate) guards compile clean without feature

## Task
Verify that all `#[cfg(feature = "legacy-orchestrate")]` guards are complete — no dangling references when the feature is off.

## Runner Context
Runner PF, batch 3 of 3. Depends on PF_01.

## Problem
After PF_01 removes the feature from defaults, the crate must compile without it. Any code that references orchestrate.rs symbols without a `#[cfg]` guard will cause a compile error.

## Changes Required

1. Read through the cfg guards in:
   - `crates/roko-cli/src/lib.rs` (lines 62, 83, 134)
   - `crates/roko-cli/src/run.rs` (16 guards)
   - `crates/roko-cli/src/unified.rs` (line 99)

2. For each guard, verify:
   - The `use` import is guarded
   - The function call is guarded
   - Any match arms referencing orchestrate types are guarded

3. Common pattern to look for:
```rust
#[cfg(feature = "legacy-orchestrate")]
use crate::orchestrate;

// ... later in the code ...
#[cfg(feature = "legacy-orchestrate")]
orchestrate::run(...);
```

4. If there are unguarded references, add the `#[cfg]` attribute.

5. Check for conditional compilation in `mod` declarations:
```rust
#[cfg(feature = "legacy-orchestrate")]
pub mod orchestrate;

#[cfg(feature = "legacy-orchestrate")]
pub mod dispatch_direct;
```
