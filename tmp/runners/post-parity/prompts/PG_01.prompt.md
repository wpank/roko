# PG_01: Drain efficiency_events Vec periodically to disk

## Task
Add periodic flushing of `efficiency_events: Vec<AgentEfficiencyEvent>` in orchestrate.rs to prevent unbounded memory growth.

## Runner Context
Runner PG, batch 1 of 4. No dependencies.

## Problem
`crates/roko-cli/src/orchestrate.rs:2530-2531`:
```rust
/// In-memory efficiency events collected during this run.
efficiency_events: Vec<AgentEfficiencyEvent>,
```

This Vec grows unbounded during long plan runs. With many tasks, it can reach 9.5GB RSS. Events should be flushed to `.roko/learn/efficiency.jsonl` periodically.

## Current State
- Vec is initialized at lines 4183, 4397, 4608
- Accessor at line 5546-5547: `pub fn efficiency_events(&self) -> &[AgentEfficiencyEvent]`
- Used in cfactor computation at line 6673
- Used for total cost at line 7175
- Loaded from disk or fallback at lines 7330-7341

## Changes Required

1. Add a `flush_efficiency_events()` method:
```rust
fn flush_efficiency_events(&mut self) -> Result<()> {
    if self.efficiency_events.is_empty() {
        return Ok(());
    }
    let path = self.data_dir.join("learn/efficiency.jsonl");
    let file = std::fs::OpenOptions::new()
        .create(true)
        .append(true)
        .open(&path)?;
    let mut writer = std::io::BufWriter::new(file);
    for event in self.efficiency_events.drain(..) {
        serde_json::to_writer(&mut writer, &event)?;
        writeln!(writer)?;
    }
    Ok(())
}
```

2. Call `flush_efficiency_events()` after each task completes (find the task completion point in the executor loop).

3. Update `efficiency_events()` accessor to also read flushed events from disk when computing aggregates (for cfactor, total cost).

4. Add a `flush_count` field to track how many events have been flushed (for logging).
