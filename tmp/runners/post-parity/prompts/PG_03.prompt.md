# PG_03: Fix learn all to read from correct efficiency.jsonl path

## Task
Fix `roko learn all` to read efficiency events from the correct file path.

## Runner Context
Runner PG, batch 3 of 4. Depends on PG_01.

## Problem
`roko learn all` says "empty" even though `.roko/learn/efficiency.jsonl` has 22+ entries. The reader is looking at a different path than the writer.

EXISTING ANTI-PATTERN from mega-parity:
> Execution writes to `.roko/learn/efficiency.jsonl`. `roko learn all` reads from a different expected path and says "empty."

## Changes Required

1. Find where `roko learn all` reads efficiency events:
   - `crates/roko-cli/src/commands/learn.rs`

2. Find where execution writes efficiency events:
   - After PG_01: `flush_efficiency_events()` writes to `.roko/learn/efficiency.jsonl`

3. Ensure the read path matches the write path exactly. Common issues:
   - Reader uses `data_dir/efficiency.jsonl` but writer uses `data_dir/learn/efficiency.jsonl`
   - Reader uses `learn/efficiency_events.jsonl` but writer uses `learn/efficiency.jsonl`

4. Fix the reader to use the correct path.
