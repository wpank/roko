# PG_04: Add efficiency event count to roko status output

## Task
Include efficiency event count and total cost in `roko status` output.

## Runner Context
Runner PG, batch 4 of 4. Depends on PG_03.

## Problem
`roko status` shows signal counts and episode counts but not efficiency/cost information. After PG_01 and PG_03 fix the write and read paths, `status` should report:
- Total efficiency events recorded
- Total cost across all events (if non-zero)

## Changes Required

1. In `crates/roko-cli/src/commands/status.rs`, add a section:
```rust
// Efficiency events
let efficiency_path = data_dir.join("learn/efficiency.jsonl");
if efficiency_path.exists() {
    let count = count_jsonl_lines(&efficiency_path)?;
    let total_cost = sum_efficiency_cost(&efficiency_path)?;
    println!("Efficiency events: {}", count);
    if total_cost > 0.0 {
        println!("Total cost: ${:.4}", total_cost);
    }
}
```

2. Add helper functions for counting lines and summing cost (or use existing ones from roko-learn).

3. Keep the output format consistent with existing `status` sections.

## Write Scope
- `crates/roko-cli/src/commands/status.rs`

## Read-Only Context
- `crates/roko-learn/src/efficiency.rs`
- `crates/roko-cli/src/commands/learn.rs` (reference for reading events)


## Verify
```bash
cargo build --workspace 2>&1 | head -30
cargo test --workspace 2>&1 | tail -20
```
## Acceptance Criteria
- `roko status` shows efficiency event count
- Shows total cost when non-zero
- Zero/missing events show "0" or skip the section
- Output format matches existing status sections

## Do NOT
- Add new status sections beyond efficiency
- Change existing status output
- Add dependencies
