# PH_01: Make max_concurrent_tasks configurable (currently hardcoded to 1)

## Task
Change `max_concurrent_tasks` from hardcoded 1 to a configurable value from `roko.toml` and CLI flag.

## Runner Context
Runner PH, batch 1 of 5. No dependencies.

## Problem

`crates/roko-cli/src/runner/event_loop.rs:113-119`:
```rust
let exec_config = ExecutorConfig {
    max_concurrent_plans: 4,
    max_concurrent_tasks: 1,  // ← BOTTLENECK
    max_auto_fix_iterations: config.max_retries,
    task_timeout_secs: config.timeout_secs,
    ..Default::default()
};
```

`ExecutorConfig` in `roko-orchestrator/src/executor/mod.rs:148-180` declares `max_concurrent_tasks` with a default of **8** (line 209-211). But `event_loop.rs` overrides it to **1**, negating the DAG.

The architecture for parallel dispatch already exists:
- `task_dag.rs:194-219` has `ready_tasks()` returning ALL concurrently-ready tasks
- `task_dag.rs:224-231` has `mark_running()` preventing double-dispatch
- `PlanDag.running: HashSet<TaskId>` tracks in-flight tasks per plan

But `task_dag.rs` is **declared in `mod.rs` but never imported or used by `event_loop.rs`**. The event loop instead holds a single `Option<AgentHandle>` (line 249) and guards with `ctx.agent_handle.is_some()` (line 1740), enforcing max 1 agent regardless of config.

## Exact Changes

### Step 1: Add field to RunConfig

In `crates/roko-cli/src/runner/types.rs`, find the `RunConfig` struct (around line 1241). Add:

```rust
pub struct RunConfig {
    // ... existing fields ...
    /// Maximum number of tasks executing concurrently within a plan.
    /// Overridden by --max-tasks CLI flag.
    pub max_concurrent_tasks: usize,
}
```

### Step 2: Read from roko.toml in `from_roko_config()`

In `RunConfig::from_roko_config()` (around line 1337):

```rust
max_concurrent_tasks: config.runner.as_ref()
    .and_then(|r| r.max_concurrent_tasks)
    .unwrap_or(4),
```

### Step 3: Set in Default impl

In `RunConfig::default()` (around line 1382):

```rust
max_concurrent_tasks: 4,
```

### Step 4: Add CLI flag

In `crates/roko-cli/src/commands/plan.rs`, find the `plan run` subcommand struct. Add:

```rust
/// Maximum concurrent tasks per plan (default: from config, fallback: 4)
#[arg(long, default_value_t = 0)]
max_tasks: usize,
```

In the handler, override RunConfig if flag is set:

```rust
if args.max_tasks > 0 {
    run_config.max_concurrent_tasks = args.max_tasks;
}
```

### Step 5: Thread into ExecutorConfig

In `event_loop.rs:113-119`, replace the hardcoded value:

```rust
let exec_config = ExecutorConfig {
    max_concurrent_plans: 4,
    max_concurrent_tasks: config.max_concurrent_tasks,  // ← from RunConfig
    max_auto_fix_iterations: config.max_retries,
    task_timeout_secs: config.timeout_secs,
    ..Default::default()
};
```

### Step 6: Add config schema

In `roko-core/src/config/mod.rs`, add to the runner config section:

```rust
#[derive(Debug, Clone, Deserialize, Serialize, Default)]
pub struct RunnerConfig {
    pub max_concurrent_tasks: Option<usize>,
    pub max_concurrent_plans: Option<usize>,
}
```
