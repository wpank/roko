# PH_02: Wire ready_tasks() parallel dispatch when max_concurrent_tasks > 1

## Task
Replace the single `Option<AgentHandle>` with a concurrent agent pool and wire `TaskDag::ready_tasks()` for parallel task dispatch.

## Runner Context
Runner PH, batch 2 of 5. Depends on PH_01.

## Problem
Three mechanisms enforce serial execution:
1. **`Option<AgentHandle>` (line 249)** — single agent slot, at most 1 agent alive
2. **`state.agent_active` guard (line 1740)** — suppresses spawn if any agent is running
3. **`PlanStateMachine::next_action` in orchestrator** — returns 1 `SpawnAgent` per plan per tick

The infrastructure for parallelism exists but is unwired:
- `TaskDag` (task_dag.rs:132-136) with `PlanDag.running: HashSet<TaskId>` tracking in-flight tasks
- `ready_tasks()` (task_dag.rs:194-219) returns ALL concurrently-ready tasks
- `mark_running()` (task_dag.rs:224-231) prevents double-dispatch, returns false if already running

## Exact Changes

### Step 1: Replace `Option<AgentHandle>` with `HashMap<String, AgentHandle>`

In `event_loop.rs`, find line 249:
```rust
// BEFORE
let mut agent_handle: Option<AgentHandle> = None;

// AFTER
let mut agent_handles: HashMap<String, AgentHandle> = HashMap::new();  // task_id → handle
```

### Step 2: Update RunContext struct

In `RunContext` (lines 90-102):
```rust
// BEFORE
agent_handle: &'a mut Option<AgentHandle>,

// AFTER
agent_handles: &'a mut HashMap<String, AgentHandle>,
```

### Step 3: Wire TaskDag into event_loop

Add import at the top of event_loop.rs:
```rust
use super::task_dag::TaskDag;
```

Initialize before the main loop:
```rust
let mut task_dag = TaskDag::default();
// Initialize with plan tasks from the loaded plans
for (plan_id, tasks) in &task_index {
    for task in tasks.values() {
        task_dag.register_task(plan_id, &task.id, &task.deps);
    }
}
```

### Step 4: Replace the spawn guard

At line 1740-1748, replace:
```rust
// BEFORE
if ctx.state.agent_active || ctx.agent_handle.is_some() {
    debug!("agent already active — suppressing duplicate spawn");
    return;
}

// AFTER
let active_count = ctx.agent_handles.len();
if active_count >= ctx.config.max_concurrent_tasks {
    debug!(active = active_count, max = ctx.config.max_concurrent_tasks,
        "at concurrent task limit — deferring spawn");
    return;
}
// Also check if this specific task is already running
if ctx.agent_handles.contains_key(&task_id) {
    debug!(task = %task_id, "task already has active agent — suppressing");
    return;
}
```

### Step 5: Track spawned agents by task_id

At lines 1952-1977 (on successful spawn):
```rust
// BEFORE
ctx.state.agent_active = true;
*ctx.agent_handle = Some(handle);

// AFTER
ctx.agent_handles.insert(task_id.clone(), handle);
```

### Step 6: Handle agent completion — remove from map

In the `agent_rx.recv()` branch (around line 338), when an agent completes:
```rust
// Remove the completed agent's handle
ctx.agent_handles.remove(&completed_task_id);
// Mark in TaskDag
task_dag.mark_completed(plan_id, &completed_task_id);
```

### Step 7: Modify the tick branch to dispatch multiple tasks

At lines 838-855:
```rust
_ = tick_interval.tick() => {
    // Get all ready tasks across all plans
    for (plan_id, plan_tasks) in &task_index {
        let slots_available = config.max_concurrent_tasks
            .saturating_sub(agent_handles.len());
        if slots_available == 0 { break; }

        let completed: Vec<String> = task_dag.completed_tasks(plan_id);
        let ready = task_dag.ready_tasks(plan_id, plan_tasks, &completed, &completed_plans);
        for task_def in ready.into_iter().take(slots_available) {
            if !agent_handles.contains_key(&task_def.id) {
                task_dag.mark_running(plan_id, &task_def.id);
                dispatch_task(task_def, &mut ctx).await;
            }
        }
    }
}
```

### Step 8: Update state.agent_active checks

Search for all `state.agent_active` reads and replace with `agent_handles.is_empty()` or `agent_handles.len()` checks as appropriate.

- Task completion correctly unblocks dependent tasks for next tick
- No double-dispatch of the same task (guarded by `mark_running` + HashMap key check)
- Agent completion removes the handle from the map
