# PH_03: Add cargo build semaphore for concurrent gate runs

## Task
Add a `tokio::sync::Semaphore` that limits concurrent `cargo` invocations across all gate runs.

## Runner Context
Runner PH, batch 3 of 5. Depends on PH_02.

## Problem
PX-2 anti-pattern: "Cargo is a shared resource." When multiple tasks run in parallel (after PH_02), their gate pipelines each invoke `cargo test`, `cargo clippy`, etc. Multiple concurrent cargo builds thrash the build cache.

## Current Code

**Gate dispatch** — `crates/roko-cli/src/runner/gate_dispatch.rs`:
Search for `spawn_gate` or the gate evaluation entry point (around line 28-118).

**Event loop** — `crates/roko-cli/src/runner/event_loop.rs`:
- Line 115: `max_concurrent_tasks: 1` (will be configurable after PH_01)
- No semaphore for cargo specifically

**Per-task timeout** — `event_loop.rs:117`:
```rust
task_timeout_secs: config.timeout_secs,
```
Timeout exists per-task, but no cargo-specific limiting.

## Exact Changes

### Step 1: Create semaphore at runner startup

In `event_loop.rs`, after config setup (around line 115):
```rust
let cargo_semaphore = Arc::new(tokio::sync::Semaphore::new(
    config.max_concurrent_cargo.unwrap_or(2)
));
```

### Step 2: Add `max_concurrent_cargo` to RunConfig

In `crates/roko-cli/src/runner/types.rs`, add to `RunConfig` struct:
```rust
pub max_concurrent_cargo: Option<usize>,
```

Default: `Some(2)`. Wire from `roko.toml`:
```toml
[runner]
max_concurrent_cargo = 2
```

### Step 3: Pass semaphore to gate dispatch

Find the gate dispatch function. Add parameter:
```rust
pub async fn run_gate(
    // ... existing params ...
    cargo_semaphore: Arc<tokio::sync::Semaphore>,
) -> GateResult {
    // Acquire permit ONLY for cargo-based gates
    let _permit = if is_cargo_gate(&gate) {
        Some(cargo_semaphore.acquire().await.expect("semaphore closed"))
    } else {
        None
    };

    // Run gate command...
}

fn is_cargo_gate(gate: &Gate) -> bool {
    // Compile, test, clippy gates invoke cargo
    matches!(gate.kind(), GateKind::Compile | GateKind::Test | GateKind::Clippy)
}
```

### Step 4: Thread semaphore through dispatch

From `event_loop.rs`, pass `cargo_semaphore.clone()` to wherever gates are spawned.

## Acceptance Criteria
- At most N cargo processes run simultaneously (default 2)
- Non-cargo gates (diff, lint) are unaffected
- Configurable via `[runner] max_concurrent_cargo` in roko.toml

## Do NOT
- Add file-based locking
- Change the gate pipeline structure
- Apply semaphore to non-cargo gates
