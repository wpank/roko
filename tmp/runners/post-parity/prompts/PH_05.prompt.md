# PH_05: Wire exclusive_files conflict detection for parallel tasks

## Task
Prevent parallel tasks from running simultaneously if they declare overlapping `exclusive_files`.

## Runner Context
Runner PH, batch 5 of 5. Depends on PH_02.

## Problem
PX-4 anti-pattern: "Exclusive files prevent conflicts." Two tasks modifying the same file must not run simultaneously. After PH_02 enables parallel dispatch, file conflicts need detection.

## Current Code

**MergeQueue locked_conflicts** — `crates/roko-orchestrator/src/merge_queue.rs:179-214`:

The merge queue already tracks file locks. This mechanism can be extended to task dispatch.

**TaskDef in tasks.toml** — search `crates/roko-orchestrator/src/` for `TaskDef` struct. It may have a `files` or `exclusive_files` field.

## Exact Changes

### Step 1: Add `exclusive_files` field to TaskDef

If not already present, add:
```rust
/// Files this task exclusively modifies. Tasks with overlapping exclusive_files
/// must not run simultaneously.
#[serde(default)]
pub exclusive_files: Vec<String>,
```

### Step 2: Track locked files in TaskDag or event_loop state

```rust
// In event_loop state:
let mut locked_files: HashMap<String, String> = HashMap::new();  // file_path → task_id
```

### Step 3: Filter ready_tasks by file conflicts

In the tick branch (event_loop.rs, modified by PH_02):
```rust
let ready = task_dag.ready_tasks(plan_id, plan_tasks, &completed, &completed_plans);
for task_def in ready.into_iter().take(slots_available) {
    // Check file conflicts
    let has_conflict = task_def.exclusive_files.iter().any(|f| locked_files.contains_key(f));
    if has_conflict {
        debug!(task = %task_def.id, "file conflict — deferring");
        continue;
    }

    // Lock files
    for file in &task_def.exclusive_files {
        locked_files.insert(file.clone(), task_def.id.clone());
    }

    task_dag.mark_running(plan_id, &task_def.id);
    dispatch_task(task_def, &mut ctx).await;
}
```

### Step 4: Release file locks on task completion

In the agent completion handler:
```rust
// Remove file locks for completed task
locked_files.retain(|_file, owner| owner != &completed_task_id);
```

## Acceptance Criteria
- Tasks with overlapping `exclusive_files` never run simultaneously
- File locks released on task completion
- Tasks without `exclusive_files` are unaffected
- Deferred tasks dispatch on next tick when conflict resolves
