## Task
Ensure episodes record the actual model (from response), actual provider, and files changed by the agent.

## Runner Context

## Problem

The episode model field is populated from `state.agent_model` (set by `AgentEvent::Started` at agent_events.rs:20 and `AgentEvent::SystemInit` at agent_events.rs:27). For CLI dispatch, `SystemInit` has the actual model from Claude. For API dispatch, verify the actual model is returned.

## Exact Changes

### Step 1: Verify model comes from response (not request)

The `SystemInit` event carries the actual model the CLI reports. For API dispatch, check if the response includes the model used. If the API response has `model` in the body, extract it.

If `state.agent_model` is empty after dispatch (for providers that don't report model), fall back to the requested model:
```rust
if ep.model.is_empty() {
    ep.model = config.model.clone();  // requested, as fallback only
}
```

### Step 2: Populate provider field

Verify the episode has a `provider` field. If empty:
```rust
ep.provider = if state.agent_provider.is_empty() {
    "unknown".to_string()
} else {
    state.agent_provider.clone()
};
```

### Step 3: Populate files_changed from git diff

After task completion, before episode construction:
```rust
let files_changed: Vec<String> = match git_diff_names(&config.workdir, &task_branch) {
    Ok(files) => files,
    Err(err) => {
        debug!(error = %err, "could not get files changed");
        Vec::new()
    }
};
ep.files_changed = files_changed;
```

Add helper:
```rust
fn git_diff_names(workdir: &Path, branch: &str) -> Result<Vec<String>> {
    let output = std::process::Command::new("git")
        .args(["diff", "--name-only", "main", branch])
        .current_dir(workdir)
        .output()?;
    Ok(String::from_utf8_lossy(&output.stdout)
        .lines()
        .map(|l| l.to_string())
        .collect())
}
```
