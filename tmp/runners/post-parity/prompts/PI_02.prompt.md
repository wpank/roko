# PI_02: Load gate thresholds from disk at runner startup

## Task
Load persisted gate thresholds from `.roko/learn/gate-thresholds.json` at runner startup.

## Runner Context
Runner PI, batch 2 of 5. No dependencies.

## Problem
LL-2 anti-pattern: "Thresholds must load." Gate thresholds are saved to disk after each run (`update_gate_thresholds()` at event_loop.rs:2354-2390) but may not be loaded on the next run.

## Current Code

**Threshold save** — `event_loop.rs:2354-2390`:
```rust
fn update_gate_thresholds(workdir: &Path, rung: u32, passed: bool) {
    let path = workdir.join(".roko").join("learn").join("gate-thresholds.json");
    let mut thresholds: serde_json::Value = std::fs::read_to_string(&path)
        .ok()
        .and_then(|s| serde_json::from_str(&s).ok())
        .unwrap_or_else(|| serde_json::json!({"rungs": {}}));
    // EMA update per rung...
}
```

**Threshold load on startup** — search `event_loop.rs` for gate threshold loading at startup.

**PersistPaths** — `persist.rs:31-44`:
```rust

```

## Exact Changes

### Step 1: Verify thresholds are loaded at startup

In `event_loop.rs`, at the start of the run function, search for `gate_thresholds`. If not loaded, add:

```rust
let gate_thresholds = persist::load_gate_thresholds(&paths)
    .unwrap_or_default();
```

### Step 2: Add load function if missing

In `persist.rs`:
```rust
pub fn load_gate_thresholds(paths: &PersistPaths) -> Result<GateThresholds> {
    let content = std::fs::read_to_string(&paths.gate_thresholds_json)?;
    let thresholds: GateThresholds = serde_json::from_str(&content)?;
    Ok(thresholds)
}
```

### Step 3: Pass loaded thresholds to gate evaluation

Thread the loaded thresholds into the gate dispatch path so they're used (not fresh defaults).

### Step 4: Verify the EMA update persists correctly

The write at `event_loop.rs:2389-2392` may use `unwrap_or_default()` which overwrites with empty on serialization failure (PV_05 bug). Verify the write path is correct.
