# PI_03: Record force_backend overrides as dampened routing observations

## Task
When `--force-backend` or `model_hint` overrides model selection, record the outcome as a dampened routing observation.

## Runner Context
Runner PI, batch 3 of 5. No dependencies.

## Problem
LL-3 anti-pattern: "Manual overrides are data." The `force_backend` flag (event_loop.rs:1819) passes the override to the dispatcher, but `observe_cascade_router()` at event_loop.rs:2347-2454 records the observation at full strength. Manual overrides should be dampened (e.g., 0.3x weight).

## Current Code

**force_backend passed to dispatcher** — `event_loop.rs:1819`:
```rust
force_backend: ctx.config.cli_model_override.clone(),
```

**observe_cascade_router()** — `event_loop.rs:2347-2454`:
Records quality (0.0 or 1.0), normalized cost, normalized latency — all at full strength.

Does NOT check whether the model was manually overridden.

**observe_multi_objective()** — `cascade_router.rs:1107-1135`:
```rust
pub fn observe_multi_objective(
    &self,
    context_vec: Vec<f64>,
    model_idx: usize,
    quality: f64,
    normalized_cost: f64,
    normalized_latency: f64,
    weights: &RewardWeights,
)
```

## Exact Changes

### Step 1: Detect manual override in observe function

In `observe_cascade_router()` (event_loop.rs:2347), add a check:

```rust
fn observe_cascade_router(
    config: &RunConfig,
    state: &RunState,
    completion: &GateCompletion,
    tui: &TuiBridge,
) {
    // ... existing code ...

    let is_manual_override = config.cli_model_override.is_some()
        || state.task_model_hint.is_some();

    // Dampen quality signal for manual overrides
    let quality = if completion.passed { 1.0 } else { 0.0 };
    let dampened_quality = if is_manual_override {
        quality * 0.3  // 30% weight for manual overrides
    } else {
        quality
    };

    router.observe_multi_objective(
        ctx.to_features(),
        model_idx,
        dampened_quality,
        normalized_cost,
        normalized_latency,
        &weights,
    );
}
```

### Step 2: Track whether current task used model_hint

In `RunState` or the dispatch context, add a flag:
```rust
pub task_model_hint: Option<String>,
```

Set it when the task has a `model_hint` field in the task definition. Clear it after observation.

### Step 3: Log the dampening

```rust
if is_manual_override {
    debug!(
        model = %model,
        dampening = 0.3,
        "manual override detected — recording dampened observation"
    );
}
```
