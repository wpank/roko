# PI_04: Verify cascade router state persists and reloads across runs

## Task
Audit and verify that the CascadeRouter's learned state (arm weights, observation counts, role table) persists to disk after observations and reloads correctly on the next run.

## Runner Context
Runner PI, batch 4 of 5. Depends on PI_02.

## Problem
LL-4 anti-pattern: "Learning state must survive restarts." The router records observations during a run, but if the save/load cycle has gaps, every run starts cold and the bandit never converges.

pub fn save(&self, path: &Path) -> Result<(), std::io::Error> {
    let json = self.snapshot_json();  // line 1511-1548
    let parent = path.parent().unwrap();
    std::fs::create_dir_all(parent)?;
    let tmp = path.with_extension("json.tmp");

    std::fs::rename(&tmp, path)?;  // atomic write-then-rename

}
