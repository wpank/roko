# PI_05: Wire provider health pre-filtering into routing context

## Task
Use the existing `ProviderHealthTracker::filter_arms_or_best()` to pre-filter unhealthy providers before CascadeRouter model selection, replacing hardcoded binary availability checks.

## Runner Context
Runner PI, batch 5 of 5. Depends on PI_04.

## Problem
LL-5 anti-pattern: "Route around known failures." Provider health is tracked via `ProviderHealthTracker` (circuit breaker with Closed/Open/HalfOpen states) but the CascadeRouter's UCB1 bandit selection doesn't incorporate pass-rate data. A provider with 60% success rate should be weighted lower than one with 99%.

**Key insight**: The codebase already has `ProviderHealthTracker::filter_arms_or_best()` which is designed exactly for this purpose. It's already called in `model_router.rs:1338` and `runtime_feedback.rs:1439`. The v2 runner's event loop just needs to wire this call into its routing path.

### Step 2: Pre-filter model slugs before CascadeRouter selection

In the routing path (around `event_loop.rs:2396-2454`), add health pre-filtering BEFORE calling `select_model()`:

```rust
// Collect candidate model slugs
let all_slugs: Vec<String> = cascade_router.model_slugs().to_vec();

// Pre-filter using existing health tracker API
let provider_of = |slug: &str| -> Option<String> {
    // Extract provider from model slug (e.g., "anthropic/claude-sonnet" -> "anthropic")
    slug.split('/').next().map(String::from)
};
let healthy_slugs = health_tracker.filter_arms_or_best(&all_slugs, &provider_of);

// Select from healthy candidates only
let ctx = RoutingContext { /* ... existing fields ... */ };
let selected = cascade_router.select_model_from(
    ctx.to_features(),
    &healthy_slugs,
    &weights,
);
```

If `select_model_from` doesn't exist (it may only have `select_model` which uses all arms), then filter the slugs before passing to the router:

```rust
// Alternative: temporarily mask unhealthy arms
let selected = cascade_router.select_model(ctx.to_features(), &weights);

// If selected model is unhealthy, check if filter_arms_or_best gives a better option
if !healthy_slugs.contains(&selected.model) && !healthy_slugs.is_empty() {
    // Re-select from healthy subset only
    // Use the first healthy slug as override
    debug!(
        original = %selected.model,
        override_to = %healthy_slugs[0],
        "overriding unhealthy provider selection"
    );
    selected.model = healthy_slugs[0].clone();
}
```

### Step 3: Record health outcomes after task completion

After each task, update provider health based on success/failure:

```rust
let provider = provider_of(&selected_model);
if let Some(provider_id) = provider {
    if gate_passed {
        health_tracker.record_success(&provider_id);
    } else {
        health_tracker.record_failure(&provider_id);
    }
}
```

### Step 4: Log health influence

```rust
if healthy_slugs.len() < all_slugs.len() {
    debug!(
        total = all_slugs.len(),

}
```
