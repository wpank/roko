# PJ_01: Add cascade router state to run snapshot

## Task
Include the CascadeRouter's learned state in the run snapshot so it survives crash+resume.

## Runner Context
Runner PJ, batch 1 of 5. No dependencies.

## Problem
PV-1 anti-pattern: "Snapshot everything or lose it." The run snapshot (`save_snapshot()` at event_loop.rs:1405-1453) saves executor state, completed tasks, cost, and fingerprints — but NOT the CascadeRouter's learned weights. On crash+resume, routing knowledge is lost.

## Current Code

**RunStateSnapshot struct** — `persist.rs:81-122`:
```rust
pub struct RunStateSnapshot {
    pub schema_version: u32,
    pub run_id: String,
    pub started_at_ms: u64,
    pub timestamp_ms: u64,
    pub tasks_total: usize,
    pub tasks_completed: usize,
    pub tasks_failed: usize,
    pub total_tokens_in: u64,
    pub total_tokens_out: u64,
    pub total_cost_usd: f64,

    pub completed_tasks: HashMap<String, Vec<String>>,
    pub snapshot_fail_streak: u32,
    pub fingerprints: Vec<TaskDefFingerprint>,
    // NOTE: No cascade_router_state field
}
```

**save_snapshot()** — `event_loop.rs:1405-1453`:
Called 11+ times during execution (lines 421, 489, 834, 860, 872, 882, 1013, 1123, 2218, 2271, 2279). Builds RunStateSnapshot from RunState fields and writes via `persist::save_run_state()`.

**prepare_resume()** — `resume.rs:130-214`:
Loads run-state snapshot, validates fingerprints, runs JSONL recovery. Does NOT restore router state.

**Router persistence path**: `.roko/learn/cascade-router.json` — already saved/loaded independently by PI_04. However, the snapshot provides a consistent checkpoint. The router file may be ahead of the snapshot (more observations recorded after last snapshot).

## Exact Changes

### Step 1: Add `cascade_router_json` field to RunStateSnapshot

In `persist.rs:81-122`, add an optional field:

```rust
pub struct RunStateSnapshot {
    // ... existing 15 fields (schema_version through fingerprints) ...

    /// CascadeRouter snapshot JSON, captured at save time.
    /// None for old snapshots or when no router is configured.
    #[serde(default, skip_serializing_if = "Option::is_none")]
    pub cascade_router_json: Option<String>,
}
```

Use `Option<String>` (not `Option<serde_json::Value>`) because `CascadeRouter::snapshot_json()` at cascade_router.rs:1511-1548 already returns a `String`. This avoids double-serialization.

### Step 2: Populate in save_snapshot()

In `event_loop.rs:1405-1453`, in the RunStateSnapshot construction block (around line 1430-1445):

```rust
// After existing field population:
cascade_router_json: config.cascade_router.as_ref().map(|r| r.snapshot_json()),
```

### Step 3: Restore in prepare_resume()

In `resume.rs`, after loading the snapshot and before returning the ResumeReport, add router restoration:

```rust
// In prepare_resume(), after snapshot validation:
if let Some(router_json) = &snapshot.cascade_router_json {
    debug!("restoring cascade router from snapshot");
    // The caller should use this to initialize the router instead of loading from file
    report.cascade_router_json = Some(router_json.clone());
}
```

The caller (event loop startup) should prefer the snapshot's router state over the file-based one when resuming:

```rust
let cascade_router = if let Some(json) = resume_report.cascade_router_json {
    CascadeRouter::from_snapshot_json(&json, model_slugs)
        .unwrap_or_else(|_| CascadeRouter::load_or_new(&router_path, model_slugs))
} else {
    CascadeRouter::load_or_new(&router_path, model_slugs)
};
```

### Step 4: Add from_snapshot_json if missing

If `CascadeRouter` doesn't have a `from_snapshot_json()` method, add it in `cascade_router.rs` near the existing `load_or_new()` at line 1569:

```rust
pub fn from_snapshot_json(json: &str, model_slugs: Vec<String>) -> Result<Self, serde_json::Error> {
    let snapshot: CascadeSnapshot = serde_json::from_str(json)?;
    Ok(Self::from_snapshot(snapshot, model_slugs))
}
```
