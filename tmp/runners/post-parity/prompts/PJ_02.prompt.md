# PJ_02: Add daimon state to run snapshot

## Task
Include the DaimonState (affect engine) in the run snapshot for crash+resume.

## Runner Context
Runner PJ, batch 2 of 5. No dependencies.

## Problem
PV-2 anti-pattern: "Affect must survive restarts." DaimonState tracks the system's affective state (arousal, valence, pleasure, dominance via PAD vector, somatic landscape, crate confidence map). On crash+resume, this resets to neutral defaults. The system loses accumulated affect signals (e.g., high arousal from repeated failures) which influences dispatch modulation.

## Current Code

**DaimonState** — `crates/roko-daimon/src/lib.rs:1973-1990`:
```rust
pub struct DaimonState {
    pub state: AffectState,          // PAD vector (arousal, valence, pleasure, dominance)
    pub half_life_hours: f64,
    pub somatic_landscape: SomaticLandscape,
    pub crate_confidence_map: HashMap<String, f64>,
    // ... additional fields ...
}
```

**Current usage in event loop** — `event_loop.rs:2430`:
```rust
daimon_policy: roko_core::DaimonPolicy::default(),
```
DaimonState is NOT loaded from disk in the v2 runner — it always uses `DaimonPolicy::default()`. This is a separate bug (daimon not loaded at all), but the snapshot should still include whatever state exists.

**RunStateSnapshot** — `persist.rs:81-122`:
No daimon field.

## Exact Changes

### Step 1: Add `daimon_state_json` field to RunStateSnapshot

In `persist.rs:81-122`:

```rust
pub struct RunStateSnapshot {
    // ... existing fields ...

    /// DaimonState snapshot, captured at save time.
    #[serde(default, skip_serializing_if = "Option::is_none")]
    pub daimon_state_json: Option<String>,
}
```

### Step 2: Populate in save_snapshot()

In `event_loop.rs:1405-1453`, in the snapshot construction block:

```rust
daimon_state_json: state.daimon_state.as_ref()
    .and_then(|d| serde_json::to_string(d).ok()),
```

### Step 3: Add daimon_state to RunState if missing

If `RunState` (state.rs:37-79) doesn't have a `daimon_state` field, add it:

```rust
pub struct RunState {
    // ... existing fields ...
    pub daimon_state: Option<DaimonState>,
}
```

Load it at startup:

```rust
let daimon_path = config.workdir.join(".roko/daimon.json");
state.daimon_state = std::fs::read_to_string(&daimon_path)
    .ok()
    .and_then(|s| serde_json::from_str(&s).ok());
```

### Step 4: Restore on resume

In the resume path, restore from snapshot:

```rust
if let Some(daimon_json) = &snapshot.daimon_state_json {
    state.daimon_state = serde_json::from_str(daimon_json).ok();
}
```

### Step 5: Use loaded DaimonState in routing context

Replace the hardcoded default at event_loop.rs:2430:

```rust
// BEFORE:
daimon_policy: roko_core::DaimonPolicy::default(),

// AFTER:
daimon_policy: state.daimon_state.as_ref()
    .map(|d| d.to_policy())
    .unwrap_or_default(),
```
