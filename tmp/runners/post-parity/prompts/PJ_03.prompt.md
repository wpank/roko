# PJ_03: Verify cost/efficiency aggregates survive crash+resume

## Task
Verify that cumulative cost and token counts in the run snapshot are correctly restored on resume.

## Runner Context
Runner PJ, batch 3 of 5. No dependencies.

## Problem
PV-3 anti-pattern: "Cost tracking must be continuous." This is actually an AUDIT batch — the cost fields already exist in `RunStateSnapshot`. The concern is whether `prepare_resume()` correctly restores them into `RunState` so post-resume cost reporting is cumulative (not reset to $0.00).

**Cost rollup** — `state.rs:401-409`:
```rust
fn roll_into_totals(&mut self) {
    self.total_tokens_in += self.tokens_in;
    self.total_tokens_out += self.tokens_out;
    self.total_cost_usd += self.cost_usd;
    if !self.plan_id.is_empty() {
        *self.plan_costs.entry(self.plan_id.clone()).or_default() += self.cost_usd;
    }
}
```
