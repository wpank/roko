# PK_01: Wire neuro store query into CascadeRouter routing context

## Task
Make the CascadeRouter consult the neuro knowledge store for model routing advice before selecting a model.

## Runner Context
Runner PK, batch 1 of 6. No dependencies.

## Problem
KR-1 anti-pattern: "Knowledge exists but isn't consulted." The neuro store holds durable knowledge entries about which models/strategies worked for which task types. The CascadeRouter selects models via UCB1 bandit but doesn't incorporate this knowledge. This is the most powerful cross-run learning feature and it's completely disconnected from routing.

## Exact Changes

### Step 1: Query neuro store before model selection

In `event_loop.rs`, at the dispatch path (before the agent is spawned for a task), query the neuro store:

```rust
// Before model selection for dispatch
let knowledge_hints = if let Some(neuro) = &config.neuro_store {
    match neuro.query(&task.title, 3) {
        Ok(entries) => entries.iter()
            .filter_map(|e| e.metadata.get("recommended_model").cloned())
            .collect::<Vec<String>>(),
        Err(_) => Vec::new(),
    }
} else {
    Vec::new()
};
```

### Step 2: Pass hints as prior bias to router selection

Rather than modifying the 18-dim feature vector (which would break the bandit's observation space), apply knowledge hints as a post-selection bias:

```rust
let selected = router.select_model(ctx.to_features(), &weights);

// Apply knowledge bias: if knowledge recommends a different model, boost it
let knowledge_bias_weight: f64 = 0.2;  // configurable
if !knowledge_hints.is_empty() {
    let hint_matches = knowledge_hints.iter()
        .any(|h| h == &selected.model);
    if !hint_matches {
        // Check if any hinted model is available and score-competitive
        for hint_model in &knowledge_hints {
            if let Some(hinted) = router.score_model(hint_model, ctx.to_features(), &weights) {
                if hinted.score + knowledge_bias_weight > selected.score {
                    debug!(
                        from = %selected.model,
                        to = %hint_model,
                        "knowledge store recommends different model — switching"
                    );
                    selected = hinted;
                    break;
                }
            }
        }
    }
}
```

### Step 3: Log knowledge influence

```rust
if !knowledge_hints.is_empty() {
    debug!(
        hints = ?knowledge_hints,
        selected = %selected.model,
        "knowledge store consulted for routing"
    );
}
```

## Acceptance Criteria
- Router queries neuro store before model selection
- Knowledge hints influence (don't override) model selection via 0.2 weight bias
- Missing knowledge → no change from current behavior (empty hints = no bias)
- Weight of knowledge bias is configurable (default 0.2)
- Debug log shows when knowledge store is consulted

## Do NOT
- Change the neuro store API or KnowledgeEntry struct
- Change CONTEXT_DIM or the feature vector encoding
- Add hard overrides from knowledge (bias only, not force)
- Change the UCB1 algorithm internals
