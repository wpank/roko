# PK_02: Wire DaimonState affect into CascadeRouter routing bias

## Task
Use `roko_daimon::DaimonPolicy::modulate_dispatch()` to populate the `RoutingContext::daimon_policy` field with real affect-based values instead of `DaimonPolicy::default()`.

## Runner Context
Runner PK, batch 2 of 6. Depends on PK_01.

## Problem
KR-2 anti-pattern: "Affect exists but doesn't modulate routing." The `roko-daimon` crate already implements `DaimonPolicy::modulate_dispatch()` which computes `DispatchModulation { tier_bias, exploration_rate, turn_limit_factor }` from the PAD vector. But the v2 runner hardcodes `roko_core::DaimonPolicy::default()` in the routing context, ignoring all of this.

**Key insight**: There are TWO `DaimonPolicy` types. The one in `roko-daimon/src/policy.rs:18` wraps `DaimonState` and implements the `AffectPolicy` trait with `modulate_dispatch()`. The one in `roko-core/src/affect.rs:67-88` is a lightweight snapshot:
```rust
// roko-core/src/affect.rs:67-88
pub struct DaimonPolicy {
    pub affect_confidence: f64,
    pub behavioral_state: BehavioralState,
}
```

The `RoutingContext::daimon_policy` at `model_router.rs:160` uses the `roko_core` version. We need to populate its fields from the `roko-daimon` version's `modulate_dispatch()` output.

## Current Code

**Hardcoded default in routing** — `event_loop.rs:2430`:
```rust
daimon_policy: roko_core::DaimonPolicy::default(),
```

**DaimonState** — `crates/roko-daimon/src/lib.rs:1973-2009`:

**AffectState** — contains `pad: PadVector` (with `pleasure`, `arousal`, `dominance` fields) and `confidence: f64`. Note: there is NO `valence` field — it's `pleasure` in the PAD model.

**DaimonPolicy::modulate_dispatch()** — `crates/roko-daimon/src/policy.rs`:
Returns `DispatchModulation { tier_bias, exploration_rate, turn_limit_factor }` computed from the PAD vector. This is the logic we want to use — do NOT reimplement it.

**roko-core DaimonPolicy** — `crates/roko-core/src/affect.rs:67-88`:
```rust
pub struct DaimonPolicy {
    pub affect_confidence: f64,
    pub behavioral_state: BehavioralState,
}
```

**DaimonState persistence** — loaded via `DaimonState::load_or_new()` which reads from `.roko/daimon.json`.

## Exact Changes

### Step 1: Load DaimonState at runner startup

In `event_loop.rs`, at the top of the run function, load the daimon state:
```rust
use roko_daimon::{DaimonState, policy::DaimonPolicy as DaimonPolicyFull};

// Load persisted daimon state (or fresh default)
let daimon_path = config.workdir.join(".roko/daimon.json");
let mut daimon_state = DaimonState::load_or_new(&daimon_path);
let daimon_policy_full = DaimonPolicyFull::new(daimon_state.clone(), daimon_path.clone());
```

### Step 2: Build roko_core::DaimonPolicy from modulation output

Replace the hardcoded default at event_loop.rs:2430:

```rust
// BEFORE:
daimon_policy: roko_core::DaimonPolicy::default(),

// AFTER:
daimon_policy: {
    let modulation = daimon_policy_full.modulate_dispatch();
    roko_core::DaimonPolicy {
        affect_confidence: daimon_state.state.confidence,
        behavioral_state: if modulation.tier_bias < -0.1 {
            BehavioralState::Conservative  // stress → cheaper models
        } else if modulation.tier_bias > 0.05 {
            BehavioralState::Exploratory   // confidence → allow expensive
        } else {
            BehavioralState::Balanced
        },
    }
},
```

### Step 3: Update DaimonState after task outcomes

After each task completion/failure, use the `AffectPolicy` trait methods:

```rust
// After task completes:
use roko_daimon::policy::AffectPolicy;

if gate_passed {
    daimon_policy_full.on_task_outcome(/* success params */);
} else {
    daimon_policy_full.on_gate_result(/* failure params */);
}

// Persist updated state
if let Ok(json) = serde_json::to_string_pretty(daimon_policy_full.state()) {
    let _ = persist::atomic_write(&daimon_path, json.as_bytes());
}
```

### Step 4: Log affect influence at debug level

```rust
let modulation = daimon_policy_full.modulate_dispatch();
if modulation.tier_bias.abs() > 0.05 {
    debug!(
        arousal = daimon_state.state.pad.arousal,
        pleasure = daimon_state.state.pad.pleasure,
        tier_bias = modulation.tier_bias,
        exploration_rate = modulation.exploration_rate,
        "daimon affect influencing model selection"
    );
}
```
