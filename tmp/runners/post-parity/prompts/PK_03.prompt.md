# PK_03: Wire knowledge injection into SystemPromptBuilder domain layer

## Task
Inject relevant neuro knowledge entries into the agent's system prompt via the SystemPromptBuilder's domain context layer.

## Runner Context
Runner PK, batch 3 of 6. Depends on PK_01.

## Problem
KR-3 anti-pattern: "Knowledge exists but agents can't see it." SystemPromptBuilder has 9 layers. The domain layer is set via `with_domain()` (system_prompt_builder.rs:169) and pheromone context via `with_pheromones()` (line 183). The `PromptAssembler` (dispatch/prompt_builder.rs:247-284) uses `WorkdirKnowledgeSource` for some context injection. But the neuro store's durable knowledge (cross-run lessons, patterns, strategies) is not queried for the system prompt.

## Exact Changes

### Step 1: Query neuro store for task-relevant knowledge

In `event_loop.rs`, in the dispatch path (before building the system prompt for a task):

```rust
let domain_knowledge = if let Some(neuro) = &config.neuro_store {
    match neuro.query(&task.title, 5) {
        Ok(entries) if !entries.is_empty() => {
            let formatted = entries.iter()
                .take(5)
                .map(|e| format!("- {}: {}", e.topic, e.summary))
                .collect::<Vec<_>>()
                .join("\n");
            Some(format!(
                "## Relevant Knowledge (from previous runs)\n{}",
                formatted
            ))
        }
        _ => None,
    }
} else {
    None
};
```

### Step 2: Inject into SystemPromptBuilder domain layer

When constructing the builder, append knowledge to the existing domain context:

```rust
let builder = SystemPromptBuilder::new()
    .with_role(&role_spec)
    .with_conventions(&conventions)
    .with_domain(&{
        let mut domain = task.domain.clone().unwrap_or_default();
        if let Some(knowledge) = &domain_knowledge {
            if !domain.is_empty() {
                domain.push_str("\n\n");
            }
            domain.push_str(knowledge);
        }
        domain
    })
    .with_task(&task_context)
    // ... remaining layers ...
    ;
```

### Step 3: Respect context budget

Cap knowledge injection at ~500 tokens (~2000 chars) to avoid bloating the system prompt:

```rust
const MAX_KNOWLEDGE_CHARS: usize = 2000;

let knowledge_text = if formatted.len() > MAX_KNOWLEDGE_CHARS {
    format!("{}...", &formatted[..MAX_KNOWLEDGE_CHARS])
} else {
    formatted
};
```

### Step 4: Log injection

```rust
if domain_knowledge.is_some() {
    debug!(
        task = %task.id,
        entries = entries.len(),
        "injected neuro knowledge into system prompt Layer 3"
    );
}
```

- Bounded to ~2000 chars (~500 tokens)
- Missing knowledge → Layer 3 gets existing domain content only (no error)
- Knowledge entries include topic and summary for agent context

## Do NOT
- Change the SystemPromptBuilder's 9-layer structure
- Add more than 9 layers
- Change the with_domain() or with_pheromones() API
- Inject knowledge into non-system-prompt locations
