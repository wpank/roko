# PK_05: Wire dream routing advice into CascadeRouter context

## Task
Feed dream-generated routing advice into the CascadeRouter so cross-run insights influence model selection.

## Runner Context
Runner PK, batch 5 of 6. Depends on PK_04.

## Problem
KR-5 anti-pattern: "Dream output exists but nothing reads it." After PK_04, dream consolidation produces routing advice (e.g., "claude-sonnet works best for Rust gate failures"). This may be written to `.roko/learn/dream-routing-advice.json` or embedded in the dream report. The CascadeRouter never reads it.

## Exact Changes

### Step 1: Define DreamRoutingAdvice struct

If not already defined, add to a shared location (e.g., `roko-learn/src/dream_advice.rs`):

```rust
#[derive(Debug, Serialize, Deserialize, Default)]
pub struct DreamRoutingAdvice {
    pub recommendations: Vec<DreamRecommendation>,
    pub generated_at: u64,
}

#[derive(Debug, Serialize, Deserialize)]
pub struct DreamRecommendation {
    pub task_category: String,   // e.g., "implementation", "refactoring"
    pub recommended_model: String,
    pub confidence: f64,         // 0.0 - 1.0
    pub reason: String,
}
```

### Step 2: Load dream advice at runner startup

In `event_loop.rs`, at startup (near router initialization):

```rust
let dream_advice_path = config.workdir.join(".roko/learn/dream-routing-advice.json");
let dream_advice: DreamRoutingAdvice = std::fs::read_to_string(&dream_advice_path)
    .ok()
    .and_then(|s| serde_json::from_str(&s).ok())
    .unwrap_or_default();

if !dream_advice.recommendations.is_empty() {
    debug!(
        recommendations = dream_advice.recommendations.len(),
        "loaded dream routing advice"
    );
}
```

### Step 3: Apply advice as weak prior during model selection

In the dispatch path, after CascadeRouter selects a model (similar pattern to PK_01):

```rust
let dream_advice_weight: f64 = 0.15;  // weaker than knowledge bias (0.2)

if let Some(rec) = dream_advice.recommendations.iter()
    .find(|r| r.task_category == task_category_str && r.confidence > 0.5)
{
    if rec.recommended_model != selected.model {
        if let Some(dream_model) = router.score_model(&rec.recommended_model, features, &weights) {
            if dream_model.score + dream_advice_weight > selected.score {
                debug!(
                    from = %selected.model,
                    to = %rec.recommended_model,
                    reason = %rec.reason,
                    "dream advice recommends different model — switching"
                );
                selected = dream_model;
            }
        }
    }
}
```

### Step 4: Dream cycle should write advice file

In the dream consolidation output (handled by `DreamCycle`), ensure routing advice is written to `.roko/learn/dream-routing-advice.json`. If the dream cycle doesn't produce routing advice yet, this batch creates the consumer side; the producer side is a follow-up.

## Acceptance Criteria
- Dream advice loaded from `.roko/learn/dream-routing-advice.json` at startup
- Advice influences routing with 0.15 weight (weaker than knowledge store's 0.2)
- Missing advice file → empty advice (no error, no influence)
- Only recommendations with confidence > 0.5 are applied
- Advice refreshed on next startup after dream cycle runs

- Change the dream cycle's internal output format
