# PK_06: Fix episode_completion.rs to use ModelCaller instead of direct API key

## Task
Change episode distillation to use the `ModelCaller` trait (which `ModelCallService` implements) instead of reading `ANTHROPIC_API_KEY` directly from the environment.

## Runner Context
Runner PK, batch 6 of 6. No dependencies.

## Problem
KR-6 anti-pattern: "Raw provider HTTP in library code." `spawn_episode_distillation()` at episode_completion.rs:23-40 reads `ANTHROPIC_API_KEY` from env and makes direct HTTP calls, bypassing:
- Provider config resolution (which key? which base URL?)
- Cost tracking (distillation costs are invisible)
- Episode recording for the distillation itself
- Model routing (always uses whatever model is hardcoded)

## Current Code

**spawn_episode_distillation** — `episode_completion.rs:23-40`:
```rust
pub fn spawn_episode_distillation(
    workdir: PathBuf,
    episode: Episode,
    model_caller: Option<Arc<dyn ModelCaller>>,
    // ...
) -> JoinHandle
```
Already accepts `Option<Arc<dyn ModelCaller>>` but falls back to direct API key read when `None`.

**ANTHROPIC_API_KEY env read** — `episode_completion.rs:46`:
```rust
let Some(api_key) = std::env::var("ANTHROPIC_API_KEY").ok() else {
    // bail — no distillation without key
};
```

**ModelCallService struct** — `model_call_service.rs:49-80`:
```rust
pub struct ModelCallService {
    default_model,
    config,
    cost_table,
    event_consumers,
    feedback_sink,
    gateway_event_writer,
    knowledge_store,
    model_router,
}
```

## Exact Changes

### Step 1: Always pass ModelCaller from caller sites

In `event_loop.rs`, where `spawn_episode_distillation` is called, ensure the `ModelCaller` is always provided:

```rust
// Find the call site (search for spawn_episode_distillation in event_loop.rs)
// BEFORE:
roko_neuro::spawn_episode_distillation(workdir.clone(), episode, None);

// AFTER:
roko_neuro::spawn_episode_distillation(
    workdir.clone(),
    episode,
    Some(Arc::clone(&model_caller)),  // pass the runner's ModelCallService
);
```

Similarly in `run.rs` (search for the same call):
```rust
runtime.set_episode_completion_hook(|episode| {
    roko_neuro::spawn_episode_distillation(
        distillation_workdir.clone(),
        episode,
        Some(Arc::clone(&model_service)),  // ← pass service
    );
});
```

### Step 2: Remove direct API key fallback

In `episode_completion.rs:46`, remove or deprecate the env var fallback:

```rust
// BEFORE:
let Some(api_key) = std::env::var("ANTHROPIC_API_KEY").ok() else { return; };
// ... direct HTTP call with api_key ...

// AFTER:
let Some(caller) = model_caller else {
    debug!("no ModelCaller provided — skipping episode distillation");
    return;
};
// Use caller for the LLM call
let response = caller.call(ModelCallRequest {
    prompt: build_distillation_prompt(&episode),
    model: Some("claude-haiku-4-5".into()),  // cheap model for distillation
    caller_id: "neuro_distillation".into(),
    // ...
}).await?;
```

### Step 3: Specify cheap model for distillation

Distillation should always use the cheapest available model (haiku), not the default:

```rust
model: Some("claude-haiku-4-5".into()),
```

### Step 4: Cost tracking is automatic

Once using `ModelCallService`, the cost is automatically tracked via the service's `cost_table` and `event_consumers`. No additional wiring needed.
