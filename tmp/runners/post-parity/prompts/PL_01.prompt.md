# PL_01: Wire MCP tool dispatch in ACP pipeline runner

## Task
Route tool calls from ACP pipeline agents to their declared MCP servers by passing `--mcp-config` to the Claude CLI subprocess.

## Runner Context
Runner PL, batch 1 of 4. No dependencies.

## Problem

## Exact Changes

### Step 1: Add mcp_config_path helper to AcpSession (or runner context)

```rust
fn write_mcp_config(workdir: &Path, mcp_servers: &[McpServerConfig]) -> Option<PathBuf> {
    if mcp_servers.is_empty() {
        return None;
    }
    let path = workdir.join(".roko/mcp-config-acp.json");
    let config = serde_json::json!({
        "mcpServers": mcp_servers.iter().map(|s| {
            (s.name.clone(), serde_json::json!({
                "command": s.command,
                "args": s.args,
                "env": s.env,
            }))
        }).collect::<serde_json::Map<String, serde_json::Value>>()
    });
    match std::fs::write(&path, serde_json::to_string_pretty(&config).unwrap_or_default()) {
        Ok(()) => Some(path),
        Err(err) => {
            debug!(%err, "failed to write MCP config for ACP");
            None
        }
    }
}
```

### Step 2: Pass --mcp-config to run_claude_cli

In `runner.rs:1752-1785`, modify the Command construction:

```rust
async fn run_claude_cli(
    prompt: &str,
    workdir: &Path,
    cancel_token: &CancelToken,
    mcp_config_path: Option<&Path>,  // ← ADD parameter
) -> Result<String> {
    let mut cmd = Command::new("claude");
    cmd.arg("--print")
       .arg("--dangerously-skip-permissions")
       .arg(prompt)
       .current_dir(workdir);

    // Wire MCP config if available
    if let Some(mcp_path) = mcp_config_path {
        cmd.arg("--mcp-config").arg(mcp_path);
    }

    let output = cmd.output().await?;
    // ... existing stdout handling ...
}
```
