# PL_03: Wire cost and token tracking into WorkflowRun

## Task
Update `WorkflowRun.total_cost_usd` and `total_tokens` from actual LLM usage during pipeline execution.

## Runner Context
Runner PL, batch 3 of 4. Depends on PL_02.

## Problem
ACP-3 anti-pattern: "Cost fields exist but read zero." `WorkflowRun` at workflow.rs:15-30 has `total_cost_usd: f64` and `total_tokens: u64` initialized to 0 and never updated. Users can't see what ACP workflows cost.

## Current Code

**WorkflowRun** — `workflow.rs:15-30`:
```rust
pub struct WorkflowRun {
    pub run_id: String,
    pub pipeline: String,
    pub started_at: u64,
    pub completed_at: Option<u64>,
    pub total_cost_usd: f64,     // always 0.0
    pub total_tokens: u64,        // always 0
    pub agents_spawned: u32,
    // ...
}
```

**run_claude_cli output** — returns `String` (stdout only). Claude CLI with `--print` doesn't include usage stats in stdout. Usage is in stderr JSON when using `--output-format stream-json`.

## Exact Changes

### Step 1: For CLI path — parse usage from stream-json

Change `run_claude_cli` to use `--output-format stream-json` instead of `--print`, then parse the final `result` event for usage:

```rust
// In run_claude_cli, change output format:
cmd.arg("--output-format").arg("stream-json");
// Remove --print (incompatible with stream-json)

// Parse stdout for the result event containing usage:
let events: Vec<serde_json::Value> = stdout.lines()
    .filter_map(|line| serde_json::from_str(line).ok())
    .collect();

let usage = events.iter()
    .find(|e| e["type"] == "result")
    .and_then(|e| {
        Some(UsageStats {
            input_tokens: e["usage"]["input_tokens"].as_u64()?,
            output_tokens: e["usage"]["output_tokens"].as_u64()?,
            cost_usd: estimate_cost(e),
        })
    });

let content = events.iter()
    .filter(|e| e["type"] == "assistant" || e["type"] == "result")
    .filter_map(|e| e["content"].as_str())
    .collect::<Vec<_>>()
    .join("");
```

### Step 2: For API path — extract usage from ModelCallService response

The `ModelCallService::call()` response already includes usage metadata:
```rust
let response = model_service.call(request).await?;
let usage = response.usage;  // Already populated by ModelCallService
```

### Step 3: Accumulate into WorkflowRun after each phase

```rust
// After each phase completes:
if let Some(usage) = &phase_result.usage {
    workflow_run.total_tokens += usage.input_tokens + usage.output_tokens;
    workflow_run.total_cost_usd += usage.cost_usd;
}
```

### Step 4: Return PhaseResult with usage

Update the phase result type to include usage:
```rust
struct PhaseResult {
    pub output: String,
    pub usage: Option<UsageStats>,
    pub success: bool,
}
```

### Step 5: Log cost at workflow completion

```rust
info!(
    tokens = workflow_run.total_tokens,
    cost_usd = format!("{:.4}", workflow_run.total_cost_usd),
    "ACP workflow completed"
);
```
