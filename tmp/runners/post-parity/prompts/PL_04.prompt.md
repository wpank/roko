# PL_04: Remove hardcoded dangerously_skip_permissions from ACP phases

## Task
Make ACP pipeline phases respect safety defaults instead of hardcoding `--dangerously-skip-permissions`.

## Runner Context
Runner PL, batch 4 of 4. No dependencies.

## Problem
ACP-4 anti-pattern: "Safety bypass hardcoded." At runner.rs:1765, every Claude CLI invocation in the ACP pipeline includes `--dangerously-skip-permissions`. This bypasses ALL safety checks for ALL ACP workflows regardless of user preference. The Reviewer phase (read-only analysis) especially should never need this flag.

## Current Code

**Hardcoded flag** — `runner.rs:1765`:
```rust
.arg("--dangerously-skip-permissions")  // ← always passed
```

## Exact Changes

### Step 1: Add safety config to ACP session/runner

```rust
// In AcpSession or runner config:
pub struct AcpSafetyConfig {
    /// Whether to skip Claude CLI permission prompts.
    /// Default: false (safe). Set true only for automated pipelines.
    pub dangerously_skip_permissions: bool,
}

impl Default for AcpSafetyConfig {
    fn default() -> Self {
        Self { dangerously_skip_permissions: false }
    }
}
```

### Step 2: Conditionally pass the flag

In `run_claude_cli()` at runner.rs:1752-1785:

```rust
async fn run_claude_cli(
    prompt: &str,
    workdir: &Path,
    cancel_token: &CancelToken,
    mcp_config_path: Option<&Path>,
    skip_permissions: bool,  // ← ADD parameter
) -> Result<String> {
    let mut cmd = Command::new("claude");
    cmd.arg("--print")
       .arg(prompt)
       .current_dir(workdir);

    if skip_permissions {
        cmd.arg("--dangerously-skip-permissions");
    }

    if let Some(mcp_path) = mcp_config_path {
        cmd.arg("--mcp-config").arg(mcp_path);
    }

    let output = cmd.output().await?;
    // ...
}
```

### Step 3: Never skip for Reviewer phase

The Reviewer phase performs read-only analysis — it should never need shell or file write access:

```rust
let skip_permissions = match phase.kind {
    PhaseKind::Reviewer => false,  // Always safe for read-only analysis
    _ => session.safety.dangerously_skip_permissions,
};
```

### Step 4: Wire from CLI flag and config

Users opt in via:
- CLI: `roko acp run --dangerously-skip-permissions`
- Config: `[acp] dangerously_skip_permissions = true`

In the CLI argument parser:
```rust
#[arg(long)]
pub dangerously_skip_permissions: bool,
```

Merge CLI flag into session config:
```rust
session.safety.dangerously_skip_permissions = args.dangerously_skip_permissions
    || config.acp.dangerously_skip_permissions.unwrap_or(false);
```
