# PM_01: Add merge outcome persistence and audit trail

## Task
Persist merge outcomes (success/conflict/error) to disk so the merge history survives restarts and can inform conflict prediction.

## Runner Context
Runner PM (Merge Quality), batch 1 of 3. No dependencies.

## Problem
`GitMergeBackend` at merge.rs:151-246 executes real `git merge --no-ff` commands, and the post-merge regression gate (`CargoCheckRegressionGate`) validates results. However, merge outcomes are ephemeral — once the event loop processes them, the result is lost. There's no way to know: which branches merged successfully, which conflicted, which failed regression, or how long merges took. This data is needed for conflict prediction (PM_02) and future merge queue optimization.

### Step 1: Add MergeOutcome record type

In `crates/roko-cli/src/runner/merge.rs`, after the existing types:

```rust
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct MergeOutcome {
    pub task_id: String,
    pub branch_name: String,
    pub target_branch: String,
    pub result: MergeOutcomeResult,
    pub started_at_ms: u64,
    pub completed_at_ms: u64,
    pub regression_gate_passed: Option<bool>,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub enum MergeOutcomeResult {
    Success,
    Conflict { stderr: String },
    GitError { stderr: String },
    RegressionFailed { gate_output: String },
}
```

### Step 2: Add persistence to PlanMerger

```rust
impl PlanMerger {
    fn outcomes_path(&self) -> PathBuf {
        self.config.workdir.join(".roko/state/merge-outcomes.jsonl")
    }

    pub fn record_outcome(&self, outcome: MergeOutcome) -> Result<()> {
        let path = self.outcomes_path();
        if let Some(parent) = path.parent() {
            std::fs::create_dir_all(parent)?;
        }
        let mut file = std::fs::OpenOptions::new()
            .create(true)
            .append(true)
            .open(&path)?;
        let line = serde_json::to_string(&outcome)?;
        use std::io::Write;
        writeln!(file, "{}", line)?;
        Ok(())
    }

    pub fn load_outcomes(&self) -> Vec<MergeOutcome> {
        let path = self.outcomes_path();
        let Ok(content) = std::fs::read_to_string(&path) else { return vec![] };
        content.lines()
            .filter_map(|line| serde_json::from_str(line).ok())
            .collect()
    }
}
```

- Outcomes include: task_id, branches, result (success/conflict/error), timing, regression gate result
- Missing file → empty history (no crash)
- JSONL format for easy append + streaming reads
- `cargo build --workspace` passes

## Do NOT
- Change the merge execution logic (git merge --no-ff stays)
- Change the MergeQueue ordering logic
- Auto-retry conflicted merges (that's PM_02)
- Block the event loop on persistence I/O
