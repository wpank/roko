# PM_03: Wire prompt pre-assembly during gate execution window

## Task
Pre-assemble the next task's system prompt and context during the gate execution window, saving 1-3s of prompt assembly time per task.

## Runner Context
Runner PM, batch 3 of 3. No dependencies.

## Problem
MQ-3 optimization: "Use idle time productively." During gate execution (compile, test, clippy), the system is idle for 10-60s. We cannot pre-spawn agents (Claude CLI processes require the prompt at spawn time — there is no way to "pre-spawn then send prompt later"). However, we CAN pre-assemble the next task's system prompt, context files, and routing decision during this window.

**Why NOT a warm agent pool**: `AgentHandle` wraps an OS process spawned via `Command::new("claude")`. The prompt is passed as a CLI argument at spawn time. There is no "pre-spawn and send prompt later" mechanism. The agent sidecar supports this pattern, but the v2 runner uses CLI processes. Pre-assembly of prompts is the feasible optimization.

## Exact Changes

### Step 1: Add PreAssembledContext struct

In `event_loop.rs` or a new `pre_assembly.rs`:

```rust
struct PreAssembledContext {
    plan_id: String,
    task_id: String,
    system_prompt: String,
    code_context: Option<String>,
    routing_decision: Option<String>,  // pre-selected model slug
    assembled_at: Instant,
}

struct PreAssemblyCache {
    cached: Option<PreAssembledContext>,
}

impl PreAssemblyCache {
    fn new() -> Self { Self { cached: None } }

    fn take(&mut self, plan_id: &str, task_id: &str) -> Option<PreAssembledContext> {
        if let Some(ctx) = &self.cached {
            if ctx.plan_id == plan_id && ctx.task_id == task_id
                && ctx.assembled_at.elapsed() < Duration::from_secs(300) // 5min TTL
            {
                return self.cached.take();
            }
        }
        self.cached = None;  // stale — discard
        None
    }
}
```

### Step 2: Pre-assemble during gate execution

After dispatching a task's agent and starting gate execution:

```rust
// After task completes, while gates are running:
let ready = task_dag.ready_tasks(plan_id, tasks, &completed, &plans);
if let Some(next_task) = ready.first() {
    if pre_assembly.cached.is_none() {
        debug!(task = %next_task.id, "pre-assembling context during gate window");

        // Build the system prompt (the expensive part)
        let system_prompt = build_system_prompt_for_task(
            next_task,
            &config,
            &knowledge_store,
            &code_index,
        ).await;

        // Optionally pre-select model via CascadeRouter
        let routing_decision = cascade_router.as_ref().map(|r| {
            let ctx = build_routing_context(next_task, &state);
            r.select_model(ctx.to_features(), &weights).model
        });

        pre_assembly.cached = Some(PreAssembledContext {
            plan_id: plan_id.clone(),
            task_id: next_task.id.clone(),
            system_prompt,
            code_context: None,  // included in system_prompt via builder
            routing_decision,
            assembled_at: Instant::now(),
        });
    }
}
```

### Step 3: Use pre-assembled context at dispatch time

When dispatching a new task, check the cache first:

```rust
let (system_prompt, model_hint) = if let Some(pre) = pre_assembly.take(&plan_id, &task.id) {
    debug!(task = %task.id, "using pre-assembled context");
    (pre.system_prompt, pre.routing_decision)
} else {
    // Cold assembly on the hot path (existing behavior)
    let prompt = build_system_prompt_for_task(&task, &config, &knowledge_store, &code_index).await;
    let model = cascade_router.as_ref().map(|r| {
        r.select_model(build_routing_context(&task, &state).to_features(), &weights).model
    });
    (prompt, model)
};
```

### Step 4: Invalidate cache on DAG changes

After any task failure that could change the ready set:

```rust
if let Some(cached) = &pre_assembly.cached {
    let still_next = task_dag.ready_tasks(plan_id, tasks, &completed, &plans)
        .first()
        .map(|t| t.id == cached.task_id)
        .unwrap_or(false);
    if !still_next {
        debug!(task = %cached.task_id, "pre-assembled context no longer valid — discarding");
        pre_assembly.cached = None;
    }
}
```

## Acceptance Criteria
- Next task's system prompt pre-assembled during gate execution window
- Pre-assembled context used when available (skips hot-path assembly)
- Stale cache (>5min or wrong task) discarded
- Cache invalidated when DAG changes (task failure, replan)
- Cold assembly used as fallback (no behavioral change when cache empty)

## Do NOT
- Pre-spawn Claude CLI processes (they require the prompt at spawn time)
- Reference `AgentHandle::cancel()` (it doesn't exist — only `kill(grace)`)
- Change the agent spawn interface
- Add persistent agent connections or process pooling
