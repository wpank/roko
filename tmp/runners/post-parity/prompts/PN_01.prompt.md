# PN_01: Implement /api/dream/journal GET endpoint

## Task
Add a GET endpoint at `/api/dream/journal` that returns dream journal entries for the DreamPhaseViz component.

## Runner Context
Runner PN, batch 1 of 7. No dependencies.

## Problem
DS-1 anti-pattern: "Frontend calls backend that doesn't exist." `demo/demo-app/src/components/DreamPhaseViz.tsx:237` calls `GET /api/dream/journal`. The dream routes file (`routes/dream.rs:17-19`) only registers `POST /api/dream/run`. The journal GET endpoint is missing.

## Current Code

**Dream route registration** — `routes/dream.rs:17-19`:
```rust
Router::new()
    .route("/dream/run", post(dream_run))
    // No /dream/journal GET route
```

**Dream journal path**: `.roko/learn/dream-journal.json` (if it exists after dream consolidation).

## Exact Changes

### Step 1: Add GET handler in routes/dream.rs

```rust
pub async fn get_dream_journal(
    State(state): State<Arc<AppState>>,
) -> impl IntoResponse {
    let journal_path = state.workdir.join(".roko/learn/dream-journal.json");
    match std::fs::read_to_string(&journal_path) {
        Ok(data) => {
            match serde_json::from_str::<serde_json::Value>(&data) {
                Ok(json) => Json(json),
                Err(_) => Json(serde_json::json!([])),
            }
        }
        Err(_) => Json(serde_json::json!([])),  // No journal yet → empty array
    }
}
```

### Step 2: Register the route

In `routes/dream.rs`, update the router:

```rust
Router::new()
    .route("/dream/run", post(dream_run))
    .route("/dream/journal", get(get_dream_journal))  // ← ADD
```

### Step 3: Verify response format matches DreamPhaseViz

Check `DreamPhaseViz.tsx:237` for the expected response shape. It likely expects an array of entries with `date`, `phase`, `summary`, `entries` fields. Ensure the journal JSON matches.
