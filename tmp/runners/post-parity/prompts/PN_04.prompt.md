# PN_04: Publish cost and token events via SSE to /api/statehub/events

## Task
Emit cost and token usage events through StateHub so SSE clients (demo Explorer) receive live updates.

## Runner Context
Runner PN, batch 4 of 7. No dependencies.

## Problem
DS-4 anti-pattern: "Events defined but never published." The StateHub has a `DashboardEvent` enum (dashboard_snapshot.rs:25-122 with 18+ variants including `AgentSpawned`, `TaskCompleted`, etc.) but no `Cost` or `TokenUsage` variants. The Explorer page polls SSE but gets no cost data.

## Current Code

**DashboardEvent enum** — `dashboard_snapshot.rs:25-122`:
```rust
pub enum DashboardEvent {
    PlanStarted { plan_id: String },
    PlanCompleted { success: bool },
    TaskStarted { title: String, phase: String },
    TaskCompleted { /* ... */ },
    AgentSpawned { role: String, model: String },
    GateResult { /* ... */ },
    // ... 12+ more variants ...
    // NOTE: No Cost or TokenUsage variants
}
```

**StateHub::publish()** — `state_hub.rs:134-142`:
```rust
pub fn publish(&self, event: DashboardEvent) {
    self.event_bus.emit(event);
    self.snapshot_tx.send_modify(|_| {});
}
```

## Exact Changes

### Step 1: Add CostEvent and TokenUsageEvent variants

In `dashboard_snapshot.rs`, add to `DashboardEvent`:

```rust
pub enum DashboardEvent {
    // ... existing variants ...

    /// Emitted after each agent turn with cost data.
    CostRecorded {
        cost_usd: f64,
        model: String,
        provider: String,
        task_id: Option<String>,
    },

    /// Emitted after each agent turn with token breakdown.
    TokenUsage {
        input_tokens: u64,
        output_tokens: u64,
        total_tokens: u64,
        model: String,
    },
}
```

### Step 2: Publish from runner event loop

In `event_loop.rs`, after each agent turn completes and usage data is available:

```rust
// After extracting usage from agent result:
if let Some(usage) = &agent_result.usage {
    state_hub.publish(DashboardEvent::CostRecorded {
        cost_usd: usage.cost_usd,
        model: state.agent_model.clone(),
        provider: state.agent_provider.clone(),
        task_id: Some(state.current_task_id.clone()),
    });
    state_hub.publish(DashboardEvent::TokenUsage {
        input_tokens: usage.input_tokens,
        output_tokens: usage.output_tokens,
        total_tokens: usage.input_tokens + usage.output_tokens,
        model: state.agent_model.clone(),
    });
}
```

### Step 3: Verify SSE serialization

The SSE endpoint at `/api/statehub/events` serializes `DashboardEvent` via serde. Ensure the new variants serialize correctly with `#[serde(tag = "type")]` or whatever the existing convention is.
