# PN_06: Add /api/runs/{id}/events SSE endpoint for live run streaming

## Task
Add an SSE endpoint that streams events for a specific run, enabling per-run live progress tracking.

## Runner Context
Runner PN, batch 6 of 7. Depends on PN_04.

## Problem
DS-6 anti-pattern: "Global events but no per-run filter." After PN_04 publishes cost/token events, the global SSE stream at `/api/statehub/events` broadcasts everything. The demo has no way to subscribe to a specific run's events for focused progress tracking.

## Current Code

**StateHub::publish()** — `state_hub.rs:134-142`:
Broadcasts to all subscribers. No per-run filtering.

**DashboardEvent variants** — include `task_id` and `plan_id` fields that can be used for filtering.

## Exact Changes

### Step 1: Add run-specific SSE handler

```rust
pub async fn run_events_sse(
    State(state): State<Arc<AppState>>,
    Path(run_id): Path<String>,
) -> impl IntoResponse {
    // Subscribe to the global event bus
    let mut rx = state.state_hub.subscribe();

    let stream = async_stream::stream! {
        loop {
            match rx.recv().await {
                Ok(event) => {
                    // Filter events by run_id
                    let belongs_to_run = match &event {
                        DashboardEvent::TaskStarted { plan_id, .. } => plan_id == &run_id,
                        DashboardEvent::TaskCompleted { plan_id, .. } => plan_id == &run_id,
                        DashboardEvent::CostRecorded { task_id, .. } => {
                            task_id.as_deref() == Some(&run_id) || true // include all cost events for now
                        }
                        DashboardEvent::PlanCompleted { .. } => true,
                        _ => false,
                    };

                    if belongs_to_run {
                        let json = serde_json::to_string(&event).unwrap_or_default();
                        yield Ok::<_, std::convert::Infallible>(
                            axum::response::sse::Event::default()
                                .event(event_type_name(&event))
                                .data(json)
                        );
                    }

                    // Close stream when plan completes
                    if matches!(&event, DashboardEvent::PlanCompleted { .. }) {
                        break;
                    }
                }
                Err(_) => break,
            }
        }
    };

    axum::response::Sse::new(stream)
        .keep_alive(axum::response::sse::KeepAlive::default())
}

fn event_type_name(event: &DashboardEvent) -> &'static str {
    match event {
        DashboardEvent::TaskStarted { .. } => "task_status",
        DashboardEvent::TaskCompleted { .. } => "task_status",
        DashboardEvent::CostRecorded { .. } => "cost",
        DashboardEvent::TokenUsage { .. } => "token_usage",
        DashboardEvent::GateResult { .. } => "gate_result",
        _ => "event",
    }
}
```

### Step 2: Register the route

```rust
.route("/runs/:id/events", get(run_events_sse))
```

If `StateHub` doesn't have a `subscribe()` method that returns a broadcast receiver:
