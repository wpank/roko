# PO_01: Publish CostEvent to StateHub after each agent turn

## Task
Emit a `DashboardEvent::CostRecorded` to StateHub after every agent turn so cost data is observable in real-time.

## Runner Context
Runner PO, batch 1 of 6. No dependencies.

## Problem
OB-1 anti-pattern: "Cost computed but never broadcast." Cost is tracked internally in `RunState.cost_usd` (state.rs:47) but never published as a DashboardEvent. TUI, HTTP endpoints, and SSE clients can't show live cost. Dashboards display $0.00 during execution.

## Current Code

**DashboardEvent enum** — `dashboard_snapshot.rs:25-122`:
Has 18+ variants but no `CostRecorded` variant (added by PN_04 if it runs first; otherwise add here).

**StateHub::publish()** — `state_hub.rs:134-142`:
```rust
pub fn publish(&self, event: DashboardEvent) { ... }
```

**Cost extraction in event loop** — search for where `state.cost_usd` is updated from agent results. Typically after parsing agent output or AgentEvent::CostUpdate.

## Exact Changes

This batch overlaps with PN_04. If PN_04 has already added the `CostRecorded` variant, skip Step 1.

### Step 1: Add CostRecorded variant (if not already added by PN_04)

In `dashboard_snapshot.rs`:
```rust
CostRecorded {
    cost_usd: f64,
    model: String,
    provider: String,
    task_id: Option<String>,
},
```

### Step 2: Publish after each agent turn

In `event_loop.rs`, in the agent event handler where cost/usage is extracted (search for where `state.cost_usd += ...` or `state.tokens_in += ...`):

```rust
// After extracting cost from agent event:
if cost_usd > 0.0 {
    tui.publish(DashboardEvent::CostRecorded {
        cost_usd,
        model: state.agent_model.clone(),
        provider: state.agent_provider.clone(),
        task_id: Some(state.current_task_id.clone()),
    });
}
```

### Step 3: Include cumulative totals in event (optional enrichment)

For dashboard display, it's useful to include running totals:
```rust
// Consider adding cumulative fields:
CostRecorded {
    cost_usd: f64,          // this turn's cost
    cumulative_cost_usd: f64, // running total
    model: String,
    provider: String,
    task_id: Option<String>,
},
```
