# PO_02: Publish TokenUsageEvent with input/output/total per turn

## Task
Emit a `DashboardEvent::TokenUsage` with token breakdown after every agent turn.

## Runner Context
Runner PO, batch 2 of 6. Depends on PO_01.

## Problem
OB-2 anti-pattern: "Tokens counted but never surfaced." Token counts are extracted from provider responses and stored in `state.tokens_in` / `state.tokens_out` (state.rs) but not published as events. Dashboards can't show token usage trends or per-model breakdowns.

## Current Code

**Token extraction** — in event_loop.rs, agent events carry usage data:
- Claude CLI: `AgentEvent::SystemInit` has model, parsed from stream-json
- API: response includes `usage.input_tokens`, `usage.output_tokens`
- Stored in `state.tokens_in` and `state.tokens_out`

## Exact Changes

### Step 1: Add TokenUsage variant (if not already added by PN_04)

In `dashboard_snapshot.rs`:
```rust
TokenUsage {
    input_tokens: u64,
    output_tokens: u64,
    total_tokens: u64,
    model: String,
},
```

### Step 2: Publish after each agent turn

In the same location as PO_01's cost event (after extracting usage):

```rust
if tokens_in > 0 || tokens_out > 0 {
    tui.publish(DashboardEvent::TokenUsage {
        input_tokens: tokens_in,
        output_tokens: tokens_out,
        total_tokens: tokens_in + tokens_out,
        model: state.agent_model.clone(),
    });
}
```

### Step 3: Handle unknown token counts

When provider doesn't return token counts (e.g., some CLI paths), don't emit:

```rust
// Only emit when we have actual data, not zeros from default initialization
if tokens_in > 0 || tokens_out > 0 {
    // publish event
}
// Don't publish with zeros — that's misleading
```

## Write Scope
- `crates/roko-core/src/dashboard_snapshot.rs` (add variant if not present)
- `crates/roko-cli/src/runner/event_loop.rs` (publish token event)

## Acceptance Criteria
- Token usage event emitted per turn with accurate counts
- Unknown token counts → no event (not zeros)
- Both CLI and API paths emit when data available
- SSE/TUI subscribers can build token trend charts

## Do NOT
- Add token estimation for CLI paths that don't report tokens
- Change the token extraction logic
