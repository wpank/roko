# PO_04: Persist provider health state to provider-health.json

## Task
Save provider health metrics to disk so they survive server restarts.

## Runner Context
Runner PO, batch 4 of 6. No dependencies.

## Problem
OB-4 anti-pattern: "Health state is volatile." `ProviderHealthTracker` (provider_health.rs:501-550) uses in-memory state with `RwLock` and `Instant`-based timers. Server restart loses all health observations. A provider that was rate-limited 30 seconds ago appears healthy after restart.

## Current Code

**ProviderHealthTracker** — `provider_health.rs:501-550`:
In-memory with `Instant` for cooldown timers (not serializable).

**ProviderHealth struct** — `provider_health.rs:82-99`:
```rust
pub struct ProviderHealth {
    pub provider_id: String,
    pub state: CircuitState,
    pub consecutive_failures: u32,
    pub total_requests: u64,
    pub total_failures: u64,
    pub last_failure_at: Option<i64>,   // ← timestamp, serializable
    pub cooldown_until: Option<i64>,     // ← timestamp, serializable
    pub failure_window: VecDeque<FailureRecord>,
}
```
The struct itself IS serializable (uses i64 timestamps). Only the tracker's `Instant` fields are not.

## Exact Changes

### Step 1: Add save/load to ProviderHealthTracker

```rust
impl ProviderHealthTracker {
    pub fn save(&self, path: &Path) -> Result<()> {
        let providers = self.all_providers_sync();  // collect snapshots
        let json = serde_json::to_string_pretty(&providers)?;
        persist::atomic_write(path, json.as_bytes())
    }

    pub fn load(path: &Path) -> Result<Self> {
        let content = std::fs::read_to_string(path)?;
        let providers: Vec<ProviderHealth> = serde_json::from_str(&content)?;
        let tracker = Self::new();
        for health in providers {
            tracker.restore_provider(health);
        }
        Ok(tracker)
    }
}
```

### Step 2: Save after each health update

In `roko-serve` state, after each `record_success()` or `record_failure()`:

```rust
// In the serve routes, after updating provider health:
state.provider_health.record_success(&provider);
let _ = state.provider_health.save(&state.health_path);  // best-effort persist
```

Or debounce: save every 10 updates or every 30 seconds, whichever comes first.

### Step 3: Load at serve startup

In `roko serve` startup:

```rust
let health_path = workdir.join(".roko/state/provider-health.json");
let provider_health = ProviderHealthTracker::load(&health_path)
    .unwrap_or_else(|_| ProviderHealthTracker::new());
```

### Step 4: Handle Instant conversion

The `cooldown_until: Option<i64>` field is already a Unix timestamp. When restoring, convert to `Instant` for the tracker's internal use:

```rust
fn restore_provider(&self, health: ProviderHealth) {
    // Convert i64 timestamp to Instant if cooldown is still in the future
    let cooldown_instant = health.cooldown_until
        .filter(|&ts| ts > chrono::Utc::now().timestamp())
        .map(|ts| {
            let remaining = ts - chrono::Utc::now().timestamp();
            Instant::now() + Duration::from_secs(remaining as u64)
        });

}
```
