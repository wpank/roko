# PP_01: Wire full learning loop into roko run

## Task
Fix `roko run` to use the actual provider name (not heuristic) in `CompletedRunInput`, and ensure the API tool loop path produces proper episodes.

## Runner Context
Runner PP, batch 1 of 6. No dependencies.

## Problem
CL-1 anti-pattern: "Inferred where actual data exists." `run.rs` calls `LearningRuntime::record_completed_run()` (line 2530) with `CompletedRunInput`, but the `provider` field uses `infer_provider()` (line 2746-2765) — a string heuristic — instead of the actual provider from the dispatch result. The `run_anthropic_api_tool_loop` path builds its own mini-episode but skips routing observations.

## Current Code

**Learning wiring** — `run.rs:2513-2532`:
```rust
let mut runtime = LearningRuntime::open_under(learn_root).await?;
runtime.set_episode_completion_hook(|episode| {
    roko_neuro::spawn_episode_distillation(distillation_workdir.clone(), episode, None);
});
let mut completed = CompletedRunInput::from_episode(episode);
completed.provider = Some(infer_provider(config));      // ← heuristic
completed.task_metric = Some(build_task_metric(...));
runtime.record_completed_run(completed).await?;
```

**infer_provider()** — `run.rs:2746-2765`:
Matches command strings to guess the provider. Unreliable for API dispatch.

**LearningRuntime** — `runtime_feedback.rs:1210-1233`:
```rust
pub struct LearningRuntime {
    paths: LearningPaths,
    episode_logger: EpisodeLogger,
    affect_engine: DaimonState,
    // ...
}
```

**record_completed_run** — `runtime_feedback.rs:1977-2040`:
Updates episode logger, applies affect signature, records provider outcomes.

## Exact Changes

### Step 1: Use actual provider from dispatch result

The dispatch result carries the actual provider/model used. Thread it through:

```rust
// BEFORE (heuristic):
completed.provider = Some(infer_provider(config));

// AFTER (actual from dispatch):
completed.provider = Some(agent_result.provider.clone()
    .unwrap_or_else(|| infer_provider(config)));  // fallback only if result missing
```

### Step 2: Pass actual model from dispatch result

Similarly for the model field:
```rust
completed.model = Some(agent_result.model.clone()
    .unwrap_or_else(|| config.model.clone()));
```

### Step 3: Wire episode recording for API tool loop

In the `run_anthropic_api_tool_loop` path (around line 2131), after collecting the final response, produce a full episode:

```rust
// After API tool loop completes:
let episode = Episode {
    task_id: run_id.clone(),
    model: agent_result.model.clone().unwrap_or_default(),
    provider: "anthropic_api".into(),
    prompt: prompt.clone(),
    output: agent_result.content.clone(),
    success: agent_result.success,
    tokens_in: agent_result.usage.input_tokens,
    tokens_out: agent_result.usage.output_tokens,
    cost_usd: agent_result.usage.cost_usd,
    // ...
};
persist::append_jsonl(&episodes_path, &episode)?;
```

### Step 4: Emit AgentEvent::ModelSelected

```rust
// After dispatch, before recording:
event_bus.emit(AgentEvent::ModelSelected {
    model: agent_result.model.clone().unwrap_or_default(),
    provider: agent_result.provider.clone().unwrap_or_default(),
    reason: "run_dispatch".into(),
});
```
