# PP_02: Wire session_id capture and --resume for roko chat multi-turn

## Task
Capture the `session_id` returned by the Claude backend and reuse it on subsequent turns so `roko chat` maintains a true multi-turn conversation.

## Runner Context
Runner PP, batch 2 of 6. No dependencies.

## Problem
CL-2 anti-pattern: "Session field exists but never populated." `ChatAgentSession.session_id` (chat_session.rs:58) is always `None`. Each turn starts a fresh conversation instead of continuing the previous one via `--resume`.

## Current Code

**ChatSession.session_id** — `chat_session.rs:51-102`:
```rust
pub struct ChatSession {
    pub session_id: Option<String>,  // always None
    pub api_history: Vec<ChatMessage>,
    // ...
}
```

**HTTP response conversion** — `chat_inline.rs:3554`:
```rust
session_id: None,  // hardcoded
```

**Resume in run.rs** — `run.rs:1839-1867`:
The `--resume` path works via `split_resume_arg` and env vars. `roko chat` has no equivalent.

## Exact Changes

### Step 1: Extract session_id from dispatch result

In `chat_inline.rs`, after receiving a dispatch result, check for session_id:

```rust
// After dispatch returns:
if let Some(sid) = &dispatch_result.session_id {
    session.session_id = Some(sid.clone());
    debug!(session_id = %sid, "captured session_id from backend");
}
```

### Step 2: Pass session_id as --resume on next turn

In the dispatch path, when `session.session_id` is `Some`:

```rust
// When building the ClaudeCliAgent for the next turn:
if let Some(sid) = &session.session_id {
    agent_builder = agent_builder.with_resume(sid);
}
```

Or for HTTP dispatch:
```rust
// In the HTTP request payload:
if let Some(sid) = &session.session_id {
    payload.insert("session_id", sid.clone());
}
```

### Step 3: Persist session_id to last.json

In the session save logic (search for `save_session` in chat_inline.rs):

```rust
// When saving session state:
let snapshot = SessionSnapshot {
    session_id: session.session_id.clone(),
    api_history: session.api_history.clone(),
    // ...
};
std::fs::write(
    &session_path,
    serde_json::to_string_pretty(&snapshot)?,
)?;
```

### Step 4: Load session_id from last.json on restart

```rust
// When loading session:
if let Ok(snapshot) = load_session_snapshot(&session_path) {
    session.session_id = snapshot.session_id;
    session.api_history = snapshot.api_history;
}
```

## Acceptance Criteria
- Second turn in `roko chat` reuses session_id from first turn's response
- Session_id persisted to last.json and loadable on restart
- HTTP dispatch path also carries session_id when available
- Missing session_id (first turn or unsupported backend) → fresh conversation

## Do NOT
- Change the Claude CLI agent's --resume interface
- Add session management beyond simple ID passthrough
