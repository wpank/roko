# PP_03: Deduplicate chat event loops into shared run_inline_loop

## Task
Extract the ~150-line duplicated event loop from `run_chat_inline` and `run_unified_inline` into a shared `run_inline_loop` function.

## Runner Context
Runner PP (Pipeline Simplification), batch 3 of 6. No dependencies.

## Problem
`chat_inline.rs` has two nearly identical async event loops:
- `run_chat_inline` (lines 947-1206, 260 lines) — HTTP backend mode
- `run_unified_inline` (lines 1212-1452, 241 lines) — direct dispatch mode

Line 1301 has an explicit comment acknowledging the duplication: `// Main event loop (identical structure to run_chat_inline)`.

Any bug fix (e.g., PP_02 session_id wiring) must be applied to both loops.

### Step 2: Extract shared event loop

Create `run_inline_loop` containing the ~150 lines that are identical between both functions:

```rust
async fn run_inline_loop(
    chat_session: &mut ChatSession,
    dispatch_mode: &mut DispatchMode<'_>,
    terminal: &mut Terminal<impl Backend>,
) -> Result<()> {
    loop {
        // 33ms poll
        if crossterm::event::poll(Duration::from_millis(33))? {
            match crossterm::event::read()? {
                // ... key event handling (shared) ...
                // ... slash command parsing (shared) ...
                _ => {}
            }
        }

        // Dispatch: the ONLY divergent part
        if chat_session.has_pending_input() {
            let response = match dispatch_mode {
                DispatchMode::Http { agent_id, serve_url, client } => {
                    // HTTP dispatch to sidecar/serve
                    dispatch_http(client, serve_url, agent_id, &input).await?
                }
                DispatchMode::Direct { session } => {
                    // Direct dispatch via ChatAgentSession
                    session.send_message(&input).await?
                }
            };
            chat_session.push_agent_response(&response);
        }

        // Render viewport (shared)
        terminal.draw(|f| chat_session.render(f))?;

        // Save session every N turns (shared)
        if chat_session.turn_count() % 5 == 0 {
            chat_session.save_session()?;
        }

        // Exit check (shared)
        if chat_session.should_exit() {
            break;
        }
    }
    Ok(())
}
```

### Step 3: Reduce run_chat_inline to thin wrapper

```rust
pub async fn run_chat_inline(
    agent_id: &str,
    serve_url: &str,
    /* ... other params ... */
) -> Result<()> {
    // HTTP-specific init (~20 lines)
    let client = reqwest::Client::new();
    let mut chat_session = ChatSession::new(/* ... */)?;
    let mut terminal = setup_terminal()?;

    let mut mode = DispatchMode::Http {
        agent_id,
        serve_url,
        client: &client,
    };

    let result = run_inline_loop(&mut chat_session, &mut mode, &mut terminal).await;

    // HTTP-specific cleanup
    restore_terminal(&mut terminal)?;
    chat_session.save_session()?;
    result
}
```
