# PP_05: Wire CostMeter to all dispatch modes

## Task
Ensure `CostMeter` tracks costs in `roko run` and `roko plan run`, not just `roko chat`.

## Runner Context
Runner PP, batch 5 of 6. No dependencies.

## Problem
CL-5 anti-pattern: "Cost tracking in one path but not others." `CostMeter` (inline/primitives/cost_meter.rs:10-27) is only wired into `chat_inline.rs`. `roko run` tracks tokens via `RunUsage` fields (run.rs:1287-1290) without `CostMeter`. `roko plan run` has its own cost accounting via `RunState`. None of these use `CostMeter`, so `SessionSummaryData` (with savings ratio, cache hit rate, per-model breakdown) is only available in chat mode.

## Current Code

**CostMeter** — `inline/primitives/cost_meter.rs:10-27`:
```rust
pub struct CostMeter {
    pub total_cost: f64,
    pub input_tokens: u64,
    pub output_tokens: u64,
    pub cache_hits: u64,
    pub cache_misses: u64,
    pub model_tokens: HashMap<String, u64>,
    pub run_count: u32,
    pub naive_baseline: f64,
}
```

**Chat wiring** — `chat_inline.rs` calls `session.cost.record_run(cost, input_tokens, output_tokens, &model, naive)`.

**Run path** — `run.rs:1287-1290` tracks tokens directly into `RunUsage`.

## Exact Changes

### Step 1: Instantiate CostMeter in roko run

In `run.rs`, at the start of `run_once`:

```rust
let mut cost_meter = CostMeter::default();
```

### Step 2: Record after each dispatch

After `dispatch_agent` returns:
```rust
if let Some(usage) = &agent_result.usage {
    cost_meter.record_run(
        usage.cost_usd,
        usage.input_tokens,
        usage.output_tokens,
        &agent_result.model.clone().unwrap_or_default(),
        compute_naive_baseline(usage),  // what it would cost without caching
    );
}
```

### Step 3: Render SessionSummaryData at end of run

After `run_once` completes:
```rust
let summary = SessionSummaryData::from_meter(&cost_meter, gate_totals, elapsed);
render_session_summary(&summary);  // or print to terminal
```

### Step 4: Thread CostMeter through plan runner

For `roko plan run`, add a shared `CostMeter` to `RunState`:

```rust
// In RunState (state.rs):
pub cost_meter: CostMeter,
```

Record after each task's agent completes. Show cumulative summary at plan completion.
