# PP_06: Warn on missing MCP config instead of silent fallback

## Task
Emit a visible warning when MCP config is expected but missing, instead of silently skipping MCP.

## Runner Context
Runner PP, batch 6 of 6. No dependencies.

## Problem
CL-6 anti-pattern: "Silent fallback to None." Multiple dispatch paths silently drop MCP:
1. `chat_session.rs` — `discover_mcp_config_stub` stub returns `None`
2. `run.rs:1921,1957` — known-protocol and generic subprocess paths hardcode `mcp_config: None`
3. Claude CLI adapter — if `mcp_config` is `None`, no `--mcp-config` flag passed with no log

Users who configure `agent.mcp_config` in `roko.toml` get no feedback when MCP isn't being used.

## Exact Changes

### Step 1: Replace discover_mcp_config_stub with real discovery

```rust
fn discover_mcp_config(workdir: &Path, config: &Config) -> Option<PathBuf> {
    // 1. Check explicit config
    if let Some(mcp_path) = &config.agent.mcp_config {
        let path = PathBuf::from(mcp_path);
        if path.exists() {
            return Some(path);
        }
        warn!(path = %path.display(), "configured MCP config not found");
    }

    // 2. Check workdir conventions
    for candidate in [".roko/mcp.json", ".mcp.json", "mcp-config.json"] {
        let path = workdir.join(candidate);
        if path.exists() {
            debug!(path = %path.display(), "discovered MCP config");
            return Some(path);
        }
    }

    None
}
```

### Step 2: Warn when MCP config is configured but dispatch doesn't support it

In `run.rs`, for paths that can't use MCP:
