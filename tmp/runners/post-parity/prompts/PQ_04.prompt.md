# PQ_04: Enable SQLite persistent index backend

## Task
Wire the feature-gated `SqliteIndex` as an alternative to in-memory `WorkspaceIndex` so the code index persists across invocations.

## Runner Context
Runner PQ, batch 4 of 4. No dependencies.

## Problem
CI-4 anti-pattern: "Rebuild from scratch every time." Every `WorkspaceIndex::load(workdir)` does a full filesystem scan, parse, graph construction, and PageRank computation. For large workspaces this takes seconds. `SqliteIndex` exists at `crates/roko-index/src/sqlite.rs` behind `#[cfg(feature = "sqlite")]` with `open()`, `create_tables()`, `insert_symbol()`, `fts_search()`, `incremental_update()` — but it's not wired.

## Exact Changes

### Step 1: Enable sqlite feature in roko-cli dependency

In `crates/roko-cli/Cargo.toml`, add sqlite feature to roko-index dependency:
```toml
roko-index = { path = "../roko-index", features = ["sqlite"] }
```

Or add to roko-index's default features:
```toml
[features]
default = ["sqlite"]
sqlite = ["rusqlite"]
```

### Step 2: Prefer SQLite in code_context_for_task

In `prompt_helpers.rs:206-239`:

```rust
pub(crate) fn code_context_for_task(
    workdir: &Path,
    task_description: &str,
    cached_index: Option<&WorkspaceIndex>,
) -> Vec<String> {
    let db_path = workdir.join(".roko/index.db");

    // Prefer SQLite if available
    #[cfg(feature = "sqlite")]
    if db_path.exists() {
        if let Ok(sqlite_index) = SqliteIndex::open(&db_path) {
            // Use sqlite_index for search
            let results = sqlite_index.fts_search(task_description, 20);
            if !results.is_empty() {
                return format_results(results);
            }
        }
    }

    // Fallback to in-memory WorkspaceIndex
    let index = cached_index
        .cloned()
        .or_else(|| WorkspaceIndex::load(workdir).ok());
    // ... existing search logic ...
}
```

### Step 3: Wire `roko index build` to create SQLite

In the CLI `index build` command:

```rust
// After building the in-memory WorkspaceIndex:
#[cfg(feature = "sqlite")]
{
    let db_path = workdir.join(".roko/index.db");
    match SqliteIndex::open(&db_path) {
        Ok(mut db) => {
            db.create_tables()?;
            for symbol in &index.symbols {
                db.insert_symbol(symbol)?;
            }
            info!(path = %db_path.display(), "SQLite index created");
        }
        Err(err) => warn!(%err, "failed to create SQLite index"),
    }
}
```

### Step 4: Wire `roko index build` with incremental update

```rust
if db_path.exists() {
    // Incremental: only update changed files

} else {

}
```

## Acceptance Criteria
- `roko index build` creates `.roko/index.db`
- Subsequent builds use incremental update (< 500ms on unchanged workspace)
- `code_context_for_task` loads from SQLite when .roko/index.db exists
- Missing `.roko/index.db` gracefully falls back to in-memory WorkspaceIndex::load()
- sqlite feature is opt-in (default features or explicit)

## Do NOT
- Remove in-memory WorkspaceIndex (keep as fallback)
- Change the SqliteIndex API or schema
- Add index versioning (incremental_update handles staleness)
