# PR_01: Migrate PRD auto-plan trigger from legacy PlanRunner to runner v2

## Task
Verify the PRD auto-plan trigger uses the v2 runner exclusively, add DashboardEvent emissions, and clean up any legacy references.

## Runner Context
Runner PR, batch 1 of 2. No dependencies.

## Problem
LM-1 audit: "Verify v2 migration is complete." Research confirms `run_generated_plans` (prd.rs:728-749) already calls `crate::runner::run()` (v2). However: (1) `prd_publish_subscriber` (roko-serve/src/routes/prds.rs:223-254) may still reference legacy paths, (2) error handling doesn't emit DashboardEvents, (3) the Strategist agent prompt may generate plans incompatible with v2 runner format.

```rust

}
```

## Exact Changes

### Step 1: Audit prd_publish_subscriber

In `routes/prds.rs:223-254`, trace `handle_prd_published_event` to verify it calls the v2 runner path. If it references `orchestrate.rs` functions, update to use `crate::runner::run`.

### Step 2: Add DashboardEvent emissions

```rust
// Before auto-plan starts:
state.state_hub.publish(DashboardEvent::AutoPlanTriggered {
    slug: slug.clone(),
    timestamp_ms: now_ms(),
});

// On success:
state.state_hub.publish(DashboardEvent::AutoPlanCompleted {
    slug: slug.clone(),
    timestamp_ms: now_ms(),
});

// On failure:
state.state_hub.publish(DashboardEvent::AutoPlanFailed {
    slug: slug.clone(),
    reason: format!("{}", err),
    timestamp_ms: now_ms(),
});
```

Add these variants to `DashboardEvent` if not present.

### Step 3: Verify plan format compatibility

Ensure `generate_plan_from_prd_with_failure_context` (prd.rs:790) generates plans that `crate::runner::load_plans()` can parse. The v2 runner expects `tasks.toml` with specific fields.

### Step 4: Remove any legacy-gated code

Search for `#[cfg(feature = "legacy-orchestrate")]` in prd.rs. If any auto-plan code is behind this gate, make it unconditional (since it now uses v2).
