# PR_02: Wire learning and episode recording into cloud worker

## Task
Wire `LearningRuntime` into the cloud worker's plan execution path so cloud runs produce episodes, efficiency events, and routing observations.

## Runner Context
Runner PR, batch 2 of 2. No dependencies.

## Problem
LM-2 anti-pattern: "Cloud runs are invisible to learning." `worker/cloud.rs:435-459` calls `crate::runner::run()` but doesn't open `LearningRuntime` afterward. Cloud task outcomes don't record episodes, efficiency events, or routing observations. Provider health uses hardcoded `"default"` key.

## Current Code

**Cloud execution** — `cloud.rs:435-459`:
```rust
pub async fn run_code_implementer_cloud(
    params: &HashMap<String, String>,
) -> Result<TaskResult> {
    // clone, checkout, load config
    let plans = crate::runner::load_plans(&plan_dir)?;
    let run_config = crate::runner::RunConfig::from_roko_config(...);
    crate::runner::run(plans, &run_config, &state_hub, CancellationToken::new()).await?;
    // commit, push, create PR
    // NOTE: No LearningRuntime, no episode recording
}
```

**Git identity** — `cloud.rs:306`: hardcoded to `roko@nunchi.dev`.

## Exact Changes

### Step 1: Wire LearningRuntime after plan execution

```rust
// After crate::runner::run() completes:
let learn_root = workdir.join(".roko/learn");
if let Ok(mut runtime) = LearningRuntime::open_under(&learn_root).await {
    runtime.set_episode_completion_hook(|episode| {
        roko_neuro::spawn_episode_distillation(workdir.clone(), episode, None);
    });

    let completed = CompletedRunInput {
        provider: Some(run_config.provider_name().to_string()),
        model: Some(run_config.model_name().to_string()),
        plan_id: Some(plan_id.clone()),
        success: plan_result.is_ok(),
        // ... populate from run result ...
    };
    if let Err(err) = runtime.record_completed_run(completed).await {
        warn!(%err, "failed to record cloud run to learning store");
    }
}
```

### Step 2: Use actual provider name for health tracking

```rust
// BEFORE:
state.provider_health.record_success("default");

// AFTER:
let provider_name = run_config.provider_name();
state.provider_health.record_success(provider_name);
```

### Step 3: Read MCP config from cloned repo

```rust
// After loading roko.toml from cloned repo:
let mcp_config = config.agent.mcp_config.as_ref().map(|p| {
    workdir.join(p)  // resolve relative to cloned repo
});
run_config.mcp_config = mcp_config;
```

### Step 4: Make git identity configurable

```rust
// BEFORE:
let git_email = "roko@nunchi.dev";
let git_name = "roko";

// AFTER:
let git_email = config.worker.as_ref()
    .and_then(|w| w.git_email.as_deref())
    .unwrap_or("roko@nunchi.dev");
let git_name = config.worker.as_ref()
    .and_then(|w| w.git_name.as_deref())
    .unwrap_or("roko");
```
