# PS_01: Unify serve route dispatch into a DispatchService

## Task
Replace the scattered `runtime.run_once()` calls in serve routes with a single `DispatchService` that provides consistent provider tracking and event emission.

## Runner Context
Runner PS, batch 1 of 4. No dependencies.

## Problem
PD-1 anti-pattern: "Same dispatch logic in 4 places." Four route modules each call `runtime.run_once()` with independent error handling. Provider health uses hardcoded `"default"` key. `ModelCallService` exists with proper fallback chains and cost tracking but serve routes bypass it.

## Current Code

**ModelCallService** — `model_call_service.rs:49-80`:

Each has: `runtime.run_once(workdir, &prompt)` → `provider_health.record_success("default")`.

## Exact Changes

### Step 1: Create DispatchService wrapper

In `crates/roko-serve/src/dispatch.rs`:

```rust
pub struct DispatchService {
    model_service: Option<Arc<ModelCallService>>,
    runtime: Arc<CliRuntime>,
    provider_health: Arc<ProviderHealthTracker>,
    state_hub: Arc<StateHub>,
}

impl DispatchService {
    pub async fn dispatch(
        &self,
        workdir: &Path,
        prompt: &str,
        caller: &str,  // e.g., "serve_run", "serve_research"
    ) -> Result<DispatchResult> {
        let start = Instant::now();

        // Resolve provider name from config
        let provider = self.resolve_provider();

        // Emit start event
        self.state_hub.publish(DashboardEvent::AgentDispatched {
            caller: caller.into(),
            provider: provider.clone(),
        });

        // Dispatch via ModelCallService if available, else fallback to CLI runtime
        let result = if let Some(service) = &self.model_service {
            service.call(ModelCallRequest {
                user_content: prompt.to_string(),
                caller_id: caller.into(),
                ..Default::default()
            }).await
        } else {
            self.runtime.run_once(workdir, prompt).await
        };

        let elapsed = start.elapsed();

        // Record provider health with actual name
        match &result {
            Ok(_) => self.provider_health.record_success(&provider),
            Err(_) => self.provider_health.record_failure(&provider, "dispatch_error"),
        }

        // Emit completion event
        self.state_hub.publish(DashboardEvent::AgentCompleted {
            caller: caller.into(),
            success: result.is_ok(),
            elapsed_ms: elapsed.as_millis() as u64,
        });

        result
    }
}
```

### Step 2: Update route handlers

Replace direct `runtime.run_once()` calls:

```rust
// BEFORE (in each route):
match runtime.run_once(workdir, &prompt).await {
    Ok(result) => { state.provider_health.record_success("default"); ... }
    Err(err) => { state.provider_health.record_failure("default", &err.to_string()); ... }
}

// AFTER:
match state.dispatch.dispatch(workdir, &prompt, "serve_run").await {
    Ok(result) => { ... }
    Err(err) => { ... }
}
```

### Step 3: Add DispatchService to AppState

```rust
pub struct AppState {

}
```
