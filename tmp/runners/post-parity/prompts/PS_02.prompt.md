# PS_02: Wire provider fallback chain in runner dispatch

## Task
Wire `ModelCallService`'s provider fallback chain into the v2 runner's dispatch path so failed providers automatically retry on alternatives.

## Runner Context
Runner PS, batch 2 of 4. Depends on PS_01.

## Problem
PD-2 anti-pattern: "Single provider, no fallback." The v2 runner dispatches via a single provider. If it fails, the task enters retry backoff. `ModelCallService` already has fallback chains that iterate through alternatives — but the runner doesn't use it.

## Current Code

**ModelCallService fallback** — `model_call_service.rs:1235-1298` (ProviderCallCell):
```rust
async fn execute(&self, model, system_prompt, user_content, options, fallback_models) {
    for (i, attempt_model) in std::iter::once(model.to_owned())
        .chain(fallback_models.iter().cloned())
        .enumerate()
    {
        // create_agent_for_model, agent.run, check retryable
    }
}
```

**Runner dispatch** — dispatches through its own path without fallback models.

## Exact Changes

### Step 1: Read fallback models from config

In the runner dispatch path (event_loop.rs):

### Step 2: Implement fallback on dispatch failure

```rust
async fn dispatch_with_fallback(
    primary_model: &str,
    fallback_models: &[String],
    task: &TaskDef,
    config: &RunConfig,
    tui: &TuiBridge,
) -> Result<AgentResult> {
    // Try primary
    match dispatch_agent(primary_model, task, config).await {
        Ok(result) => return Ok(result),
        Err(err) => {
            let retryable = is_retryable_error(&err);
            if !retryable || fallback_models.is_empty() {
                return Err(err);
            }
            warn!(
                model = primary_model,
                error = %err,
                "primary model failed — trying fallbacks"
            );
        }
    }

    // Try fallbacks
    for fallback in fallback_models {
        tui.publish(DashboardEvent::ModelFallback {
            requested_model: primary_model.to_string(),
            fallback_model: fallback.clone(),
            reason: "primary_failed".into(),
            task_id: Some(task.id.clone()),
            timestamp_ms: now_ms(),
        });

        match dispatch_agent(fallback, task, config).await {
            Ok(result) => {
                info!(model = %fallback, "fallback model succeeded");
                return Ok(result);
            }
            Err(err) => {
                warn!(model = %fallback, %err, "fallback model also failed");
                continue;
            }
        }
    }

    Err(anyhow!("all models failed (primary + {} fallbacks)", fallback_models.len()))
}
```

### Step 3: Wire into the dispatch path

Replace the current single-model dispatch with `dispatch_with_fallback`.

## Acceptance Criteria
- Runner retries on fallback models before failing a task
- Retryable vs permanent errors distinguished
- `DashboardEvent::ModelFallback` emitted with original + fallback names
- Non-retryable errors skip fallback and fail immediately
- No fallback models configured → single-attempt behavior (no change)

## Do NOT
- Change the ModelCallService fallback implementation
- Add infinite retry loops (bounded by fallback list length)
