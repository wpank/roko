# PS_03: Add provider error classification

## Task
Add a structured error classification enum for provider failures so retry logic and health tracking can make informed decisions.

## Runner Context
Runner PS, batch 3 of 4. No dependencies.

## Problem
PD-3 anti-pattern: "String matching for error classification." Provider errors are currently matched by string heuristics. Rate limits and auth failures get the same retry behavior. Without structured classification, retry logic is dumb.

## Current Code

**AgentEvent::ProviderError** — already has `ErrorClass` in `roko-learn/src/events.rs`:
```rust
ProviderError {
    provider_id: String,
    error_class: ErrorClass,
    status: u16,
},
```
But `ErrorClass` exists as an event type, not used for dispatch decisions.

## Exact Changes

### Step 1: Define ProviderErrorKind enum

In `crates/roko-agent/src/provider/mod.rs`:

```rust
#[derive(Debug, Clone, PartialEq)]
pub enum ProviderErrorKind {
    RateLimit { retry_after: Option<Duration> },
    AuthFailure,
    NetworkTimeout,
    ServiceUnavailable,      // 503
    InvalidRequest,          // 400, malformed prompt
    ModelNotFound,           // 404
    QuotaExceeded,
    ContextLengthExceeded,   // prompt too long
    Unknown { status: Option<u16>, message: String },
}
```

### Step 2: Add classification function

```rust
pub fn classify_provider_error(err: &anyhow::Error) -> ProviderErrorKind {
    let msg = err.to_string().to_lowercase();

    // Check HTTP status codes first
    if let Some(status) = extract_status_code(err) {
        return match status {
            429 => ProviderErrorKind::RateLimit {
                retry_after: extract_retry_after(err),
            },
            401 | 403 => ProviderErrorKind::AuthFailure,
            400 => {
                if msg.contains("context") || msg.contains("too long") || msg.contains("max tokens") {
                    ProviderErrorKind::ContextLengthExceeded
                } else {
                    ProviderErrorKind::InvalidRequest
                }
            }
            404 => ProviderErrorKind::ModelNotFound,
            503 => ProviderErrorKind::ServiceUnavailable,
            _ => ProviderErrorKind::Unknown { status: Some(status), message: msg },
        };
    }

    // Fallback to string matching
    if msg.contains("rate limit") || msg.contains("too many requests") {
        ProviderErrorKind::RateLimit { retry_after: None }
    } else if msg.contains("timeout") || msg.contains("timed out") {
        ProviderErrorKind::NetworkTimeout
    } else if msg.contains("auth") || msg.contains("api key") || msg.contains("unauthorized") {
        ProviderErrorKind::AuthFailure
    } else if msg.contains("quota") || msg.contains("billing") {
        ProviderErrorKind::QuotaExceeded
    } else {
        ProviderErrorKind::Unknown { status: None, message: msg }
    }
}
```

### Step 3: Add is_retryable and backoff_hint

```rust
impl ProviderErrorKind {
    pub fn is_retryable(&self) -> bool {
        matches!(self,
            Self::RateLimit { .. } |
            Self::ServiceUnavailable |
            Self::NetworkTimeout
        )
    }

    pub fn backoff_hint(&self) -> Option<Duration> {
        match self {
            Self::RateLimit { retry_after: Some(d) } => Some(*d),
            Self::RateLimit { retry_after: None } => Some(Duration::from_secs(1)),
            Self::NetworkTimeout => Some(Duration::from_secs(5)),
            Self::ServiceUnavailable => Some(Duration::from_secs(2)),
            _ => None, // non-retryable, no backoff
        }
    }
}
```

## Acceptance Criteria
- All provider errors classified into a structured enum
- `is_retryable()` correctly distinguishes transient from permanent
- `RateLimit` extracts `retry-after` header when available
- Covers at least: rate limit, auth, network, 503, 400, 404, quota, context length
- String-based classification as fallback when no HTTP status available

## Do NOT
- Change the existing AgentEvent::ProviderError (that's for event logging)
- Add provider-specific classification (generic across all providers)
