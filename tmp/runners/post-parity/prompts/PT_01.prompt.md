# PT_01: Wire episode recording into ACP pipeline phases

## Task
Record episodes from ACP pipeline task execution so the learning layer has visibility into ACP-originated work.

## Runner Context
Runner PT, batch 1 of 3. Depends on PK_06 (ModelCaller fix).

## Problem
AL-1 anti-pattern: "Execution without learning." The ACP pipeline runner dispatches via `run_claude_cli` (runner.rs:1752-1785) — a bare `Command::new("claude")` subprocess with no learning integration. No episodes are recorded. `analyze_gate_failure` (line 276) reads past episodes for context but no episodes are ever written from ACP runs.

## Current Code

**run_claude_cli** — `runner.rs:1752-1785`:
```rust
async fn run_claude_cli(prompt: &str, workdir: &Path, cancel_token: &CancelToken) -> Result<String> {
```

**IMPORTANT**: PT_01, PT_02, and PT_03 all modify `crates/roko-acp/src/runner.rs`. They should share a single `AcpPipelineContext` struct initialized once at pipeline startup, NOT independently load LearningRuntime / EventBus / CascadeRouter per call. See Step 0 below.

## Exact Changes

### Step 0: Create shared AcpPipelineContext (shared across PT_01/PT_02/PT_03)

At the top of the pipeline run function, initialize all learning state once:

```rust
/// Shared learning context for the entire ACP pipeline run.
/// Initialized once at startup; passed by reference into each phase.
struct AcpPipelineContext {
    learning_runtime: Option<LearningRuntime>,
    model_caller: Option<Arc<dyn ModelCaller>>,  // from PK_06 — for episode distillation
}

impl AcpPipelineContext {
    async fn new(workdir: &Path, model_caller: Option<Arc<dyn ModelCaller>>) -> Self {
        let learn_root = workdir.join(".roko/learn");
        let learning_runtime = match LearningRuntime::open_under(&learn_root).await {
            Ok(rt) => Some(rt),
            Err(err) => {
                debug!(%err, "LearningRuntime unavailable — ACP episodes will not be recorded");
                None
            }
        };
        Self { learning_runtime, model_caller }
    }
}
```


### Step 2: Record episode via shared LearningRuntime

```rust
if let Some(runtime) = &mut pipeline_ctx.learning_runtime {
    // Set episode completion hook with real ModelCaller (from PK_06)
    // IMPORTANT: Pass the ModelCaller, NOT None. PK_06 removes the env-var fallback.
    if let Some(caller) = &pipeline_ctx.model_caller {
        runtime.set_episode_completion_hook({
            let workdir = workdir.to_path_buf();
            let caller = Arc::clone(caller);
            move |episode| {
                roko_neuro::spawn_episode_distillation(workdir.clone(), episode, Some(caller.clone()));
            }
        });
    }

    let completed = CompletedRunInput::from_episode(episode);
    if let Err(err) = runtime.record_completed_run(completed).await {
        debug!(%err, "failed to record ACP episode — continuing");
    }
}
```

### Step 3: Don't block pipeline on recording failure

Episode recording is best-effort. Missing `.roko/` dir or serialization errors must not crash the pipeline:

```rust
// Wrap the entire record step:
if let Err(err) = record_acp_episode(&pipeline_ctx, &episode).await {
    debug!(%err, "episode recording failed — pipeline continues");
}
```

## Acceptance Criteria
- Each `run_claude_cli` call produces an episode in `.roko/episodes.jsonl`
- Episode includes task_id, prompt, output, success, model
- Episode completion hook passes a real `ModelCaller` (not `None`) — requires PK_06
- LearningRuntime opened ONCE at pipeline startup via `AcpPipelineContext`
- Missing `.roko/` dir doesn't crash the pipeline
- Unknown token counts use 0 (not None — Episode fields are u64)

## Do NOT
- Pass `None` as the ModelCaller to `spawn_episode_distillation` (that's the anti-pattern PK_06 fixes)
- Open a new `LearningRuntime` per `run_claude_cli` call (use the shared pipeline context)
- Change the Episode struct
- Change the LearningRuntime API
- Block the pipeline on recording failures
