# PT_02: Wire efficiency events from ACP pipeline into learning store

## Task
Emit `AgentEvent` efficiency events from ACP pipeline phases so the learning layer can track ACP performance.

## Runner Context
Runner PT, batch 2 of 3. Depends on PT_01.

## Problem
AL-2 anti-pattern: "CognitiveEvents but no AgentEvents." The ACP pipeline emits `CognitiveEvent` types for its UI but never emits the `AgentEvent` types that the learning layer consumes. The learning `EventBus` is blind to ACP work.

## Current Code

**AgentEvent types** — `roko-learn/src/events.rs:15-80`:
```rust
AgentEvent::TurnStarted { task_id, model, provider, timestamp_ms }
AgentEvent::TurnCompleted { turn, usage, tool_call_count, gate_passed, finish_reason }
AgentEvent::CostRecorded { model, provider, cost_usd, tokens }
AgentEvent::GateResult { gate_name, passed, score, duration_ms }
```

## Exact Changes

### Step 1: Emit TurnStarted before run_claude_cli

```rust
// Before dispatching to Claude CLI:
event_bus.emit(AgentEvent::TurnStarted {
    task_id: task_id.clone(),
    model: "claude-default".to_string(),
    provider: "claude_cli".to_string(),
    timestamp_ms: now_ms(),
});
```

### Step 2: Emit TurnCompleted after run_claude_cli

```rust
// After run_claude_cli returns:
event_bus.emit(AgentEvent::TurnCompleted {
    turn: phase_index as u32,
    usage: None,  // CLI --print doesn't report tokens; use None not zeros
    tool_call_count: 0,  // bare --print mode
    gate_passed: None,   // gates run separately
    finish_reason: if success { "stop" } else { "error" }.to_string(),
});
```

### Step 3: Emit GateResult for each gate

In the gate execution path (around line 1577 in runner.rs):

```rust
// After each gate completes:
event_bus.emit(AgentEvent::GateResult {
    gate_name: gate.name.clone(),
    passed: gate_result.passed,
    score: gate_result.score,
    duration_ms: gate_result.elapsed.as_millis() as u64,
});
```

### Step 4: Emit CostRecorded (estimated)

If cost estimation is available:
```rust
if let Some(cost) = estimate_cost_from_output_length(&output) {
    event_bus.emit(AgentEvent::CostRecorded {
        model: "claude-default".to_string(),
        provider: "claude_cli".to_string(),
        cost_usd: cost,
        tokens: None,
    });
}
```

### Step 5: Wire EventBus into ACP runner

If the ACP runner doesn't have access to `EventBus<AgentEvent>`, add it to the runner context:

```rust
pub struct AcpRunner {
    // ... existing fields ...
    event_bus: Option<EventBus<AgentEvent>>,
}
```

## Acceptance Criteria
- ACP pipeline phases emit TurnStarted and TurnCompleted
- Gate results emit GateResult events
- Events available on the EventBus for subscribers
- Unknown token counts use `None` (not hardcoded zeros)
- Missing EventBus → no events (no crash)

## Do NOT
- Change the AgentEvent enum
- Remove CognitiveEvent emissions (keep both)
- Add token estimation heuristics (accept None when unknown)
