# PT_03: Wire CascadeRouter observations from ACP task outcomes

## Task
Feed ACP task success/failure outcomes into the `CascadeRouter` so model routing bandits learn from ACP pipeline results.

## Runner Context
Runner PT, batch 3 of 3. Depends on PT_01 (shared AcpPipelineContext).

## Problem
AL-3 anti-pattern: "Two execution paths, one learns." The v2 runner records CascadeRouter observations after each task (event_loop.rs:2396-2455). The ACP pipeline runs tasks but never reports outcomes to the CascadeRouter. The routing bandits are blind to ACP results.

## Current Code

**ACP adaptive gates** — `runner.rs:1602-1641`:
```rust
thresholds.observe(rung, result);  // gate thresholds updated ✓
// CascadeRouter NOT updated ✗
```

**v2 runner observation pattern** — `event_loop.rs:2396-2455`:
Constructs `RoutingContext`, calls `router.observe_multi_objective()`. This is the pattern to replicate.

**IMPORTANT**: PT_01 introduces `AcpPipelineContext` with shared `LearningRuntime`. This prompt extends that context to also hold the `CascadeRouter`. Do NOT load the router independently.

## Exact Changes

### Step 1: Extend AcpPipelineContext with CascadeRouter (from PT_01)

In `runner.rs`, add CascadeRouter to the shared context created by PT_01:

```rust
struct AcpPipelineContext {
    learning_runtime: Option<LearningRuntime>,
    model_caller: Option<Arc<dyn ModelCaller>>,
    cascade_router: Option<CascadeRouter>,  // ← ADD THIS
    router_path: PathBuf,                    // ← ADD THIS
}

impl AcpPipelineContext {
    async fn new(workdir: &Path, model_caller: Option<Arc<dyn ModelCaller>>, model_slugs: Vec<String>) -> Self {
        // ... existing LearningRuntime init from PT_01 ...

        // Load CascadeRouter (shared for entire pipeline run)
        let router_path = workdir.join(".roko/learn/cascade-router.json");
        let cascade_router = if model_slugs.is_empty() {
            None
        } else {
            Some(CascadeRouter::load_or_new(&router_path, model_slugs))
        };

        Self { learning_runtime, model_caller, cascade_router, router_path }
    }
}
```

### Step 2: Record observation after each phase

After `run_claude_cli` completes and gates run:

```rust
if let Some(router) = &mut pipeline_ctx.cascade_router {
    // Build RoutingContext (simplified for ACP — no full task metadata available)
    let ctx = RoutingContext {
        task_category: TaskCategory::Implementation,
        complexity: TaskComplexityBand::Medium,
        iteration: attempt as u32,
        role: AgentRole::Implementer,
        crate_familiarity: 0.5,  // default for ACP
        has_prior_failure: attempt > 0,
        ..Default::default()
    };

    // Determine model from output (or use placeholder)
    let model_name = parse_model_from_output(&output)
        .unwrap_or_else(|| "claude-default".to_string());

    if let Some(model_idx) = router.model_index(&model_name) {
        let quality = if gate_passed { 1.0 } else { 0.0 };
        router.observe_multi_objective(
            ctx.to_features(),
            model_idx,
            quality,
            normalized_cost,
            normalized_latency,
            &RewardWeights::default(),
        );
    }
}
```

### Step 3: Persist router periodically (not per-observation)

Save after each TASK (not each observation) and at pipeline completion:

```rust
fn persist_router(ctx: &AcpPipelineContext) {
    if let Some(router) = &ctx.cascade_router {
        if let Err(err) = router.save(&ctx.router_path) {
            debug!(%err, "failed to persist cascade router from ACP");
        }
    }
}

// Call after each task completes (not after each phase/observation):
persist_router(&pipeline_ctx);

// Also call at pipeline shutdown (in a Drop guard or finally block):
persist_router(&pipeline_ctx);
```

### Step 4: Parse actual model from CLI output if possible

If the CLI output contains model metadata (e.g., from stream-json), extract it:

```rust
fn parse_model_from_output(output: &str) -> Option<String> {
    // Look for model in JSON metadata if stream-json output available
    // Fall back to None if bare --print output (no metadata)
    output.lines()
        .filter_map(|line| serde_json::from_str::<serde_json::Value>(line).ok())
        .find_map(|v| v.get("model").and_then(|m| m.as_str()).map(String::from))
}
```

## Acceptance Criteria
- ACP task outcomes feed into CascadeRouter observations
- CascadeRouter loaded ONCE at pipeline startup (shared via `AcpPipelineContext`)
- Router persisted per-task (not per-observation — avoids excessive serialization)
- Router also persisted at pipeline shutdown
- Model name reflects actual model when parseable from output
- Missing model slugs → no router loaded (no crash)

## Do NOT
- Load CascadeRouter independently (use shared `AcpPipelineContext` from PT_01)
- Save router after every `observe_multi_objective` call (too expensive — save per-task)
- Change the CascadeRouter API
- Change the RewardWeights defaults
- Add ACP-specific observation weights
