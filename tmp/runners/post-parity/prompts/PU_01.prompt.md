# PU_01: Wire file watcher to trigger apply_hot_reload on roko.toml changes

## Task
Replace the 2-second polling watcher with `notify::RecommendedWatcher` for sub-second config change detection.

## Runner Context

## Problem

## Exact Changes

### Step 1: Replace polling with notify watcher

```rust
use notify::{RecommendedWatcher, RecursiveMode, Watcher, Event, EventKind};
use notify_debouncer_full::{new_debouncer, Debouncer};

pub fn start_config_watcher(state: Arc<AppState>) -> JoinHandle<()> {
    tokio::spawn(async move {
        let (tx, mut rx) = tokio::sync::mpsc::channel(32);

        // Create debounced watcher (500ms debounce)
        let mut watcher = match new_debouncer(
            Duration::from_millis(500),
            None,
            move |result: Result<Vec<_>, _>| {
                if let Ok(events) = result {
                    for event in events {
                        let _ = tx.blocking_send(event);
                    }
                }
            },
        ) {
            Ok(w) => w,
            Err(err) => {
                warn!(%err, "failed to create config watcher — falling back to polling");
                return start_polling_watcher(state).await;
            }
        };

        // Watch roko.toml and STRATEGY.md
        let config_path = state.workdir.join("roko.toml");
        let strategy_path = state.workdir.join("STRATEGY.md");
        for path in [&config_path, &strategy_path] {
            if path.exists() {
                if let Err(err) = watcher.watcher().watch(path, RecursiveMode::NonRecursive) {
                    warn!(path = %path.display(), %err, "failed to watch config file");
                }
            }
        }

        // Process events
        while let Some(_event) = rx.recv().await {
            debug!("config file changed — reloading");
            reload_config_from_disk(&state).await;
        }
    })
}
```

### Step 2: Keep polling as fallback

If `notify` fails to initialize (e.g., no OS support), fall back to the existing 2-second polling:

```rust
async fn start_polling_watcher(state: Arc<AppState>) {
    let mut interval = tokio::time::interval(Duration::from_secs(2));
    let mut last_mtime = get_mtime(&state.config_path);
    loop {
        interval.tick().await;
        let current_mtime = get_mtime(&state.config_path);
        if current_mtime != last_mtime {
            last_mtime = current_mtime;
            reload_config_from_disk(&state).await;
        }
    }
}
```

### Step 3: Verify SSE forwarding

Check that when `reload_config_from_disk` succeeds, it publishes `DashboardEvent::ConfigReloaded` (or `ServerEvent::ConfigReloaded`) to the SSE bridge. If not:

```rust
async fn reload_config_from_disk(state: &AppState) {
    match load_and_apply_config(state).await {
        Ok(result) => {
            state.state_hub.publish(DashboardEvent::ConfigReloaded {
                hot_reloaded_sections: result.applied.clone(),
                needs_restart_sections: result.needs_restart.clone(),
            });
            info!(
                applied = ?result.applied,
                needs_restart = ?result.needs_restart,
                "config reloaded"
            );
        }
        Err(err) => warn!(%err, "config reload failed"),
    }
}
```
