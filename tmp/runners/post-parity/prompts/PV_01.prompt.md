# PV_01: Fill AgentOutcome in TaskCompleted feedback with real model/tokens/cost

## Task
Populate the `AgentOutcome` struct in the `FeedbackEvent::TaskCompleted` event with actual dispatch data instead of empty/zero defaults.

## Runner Context
Runner PV, batch 1 of 5. No dependencies.

## Problem
`event_loop.rs:1299-1321` constructs an `AgentOutcome` with all fields zeroed:

```rust
// event_loop.rs:1299-1321 — CURRENT (broken)
let agent_outcome = AgentOutcome {
    task_id: attempt.task_id.clone(),
    plan_id: attempt.plan_id.clone(),
    model: String::new(),       // always empty
    provider: String::new(),    // always empty
    output: String::new(),      // always empty
    tokens_in: 0,               // always zero
    tokens_out: 0,              // always zero
    cost_usd: 0.0,              // always zero
    duration_ms: 0,             // always zero
    ...
};
```

This is the primary record feeding `LearningRuntime` routing observations. Empty model/provider means the CascadeRouter gets no signal. Zero tokens means efficiency tracking is blind.

## Exact Changes

### Step 1: Find where `TaskAttemptCompleted` is constructed

The `TaskAttemptCompleted` event is produced in `event_loop.rs` after a task dispatch returns. Find the call site that populates the `completion` struct — it has access to `RunState` which holds `model`, `provider`, `tokens_in`, `tokens_out`, `cost_usd`, and `duration_ms`.

### Step 2: Thread state into `runner_event_to_feedback`

`runner_event_to_feedback` (around line 1270) converts `RunnerEvent` → `FeedbackEvent`. It currently doesn't receive the `RunState`. Change the function signature:

```rust
// BEFORE
fn runner_event_to_feedback(event: &RunnerEvent) -> Option<FeedbackEvent> {
// AFTER
fn runner_event_to_feedback(event: &RunnerEvent, run_states: &HashMap<String, RunState>) -> Option<FeedbackEvent> {
```

### Step 3: Fill fields from RunState

In the `RunnerEvent::TaskAttemptCompleted` arm, look up the task's `RunState`:

```rust
RunnerEvent::TaskAttemptCompleted(attempt) => {
    let state = run_states.get(&attempt.task_id);
    let agent_outcome = AgentOutcome {
        task_id: attempt.task_id.clone(),
        plan_id: attempt.plan_id.clone(),
        model: state.map(|s| s.model.clone()).unwrap_or_default(),
        provider: state.map(|s| s.provider.clone()).unwrap_or_default(),
        output: state.map(|s| s.output_tail.clone()).unwrap_or_default(),
        tokens_in: state.map(|s| s.tokens_in).unwrap_or(0),
        tokens_out: state.map(|s| s.tokens_out).unwrap_or(0),
        cost_usd: state.map(|s| s.cost_usd).unwrap_or(0.0),
        duration_ms: state.map(|s| s.duration_ms).unwrap_or(0),
        ...
    };
```

### Step 4: Update all call sites of `runner_event_to_feedback`

Pass the `run_states` map from the event loop's local state into every call to `runner_event_to_feedback`.
