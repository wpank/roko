# PV_03: Derive RoutingContext fields from task metadata instead of hardcoded defaults

## Task
Replace the hardcoded `RoutingContext` in the cascade router observation with values derived from the actual task and executor state.

## Runner Context
Runner PV, batch 3 of 5. No dependencies.

## Problem
`event_loop.rs:2419-2436` hardcodes every `RoutingContext` field:

```rust
// event_loop.rs:2419-2436 — CURRENT (degraded learning signal)
let ctx = RoutingContext {
    task_category: TaskCategory::Implementation,  // always Implementation
    complexity: TaskComplexityBand::Standard,      // always Standard
    crate_familiarity: 0.5,                        // always 0.5
    conductor_load: 0.0,                           // always 0.0
    active_agents: 1,                              // always 1
    ready_queue_depth: 0,                          // always 0
    max_queue_wait_hours: 0.0,                     // always 0.0
};
```

Every task looks identical to the router. The UCB1 bandit can't learn task-specific preferences.

## Exact Changes

### Step 1: Map task domain/role to TaskCategory

The `TaskDef` struct has `domain: Option<String>` and `role: Option<String>`. Map these:

```rust
let task_category = match task_def.domain.as_deref().or(task_def.role.as_deref()) {
    Some("research" | "researcher") => TaskCategory::Research,
    Some("test" | "testing" | "qa") => TaskCategory::Testing,
    Some("review" | "reviewer") => TaskCategory::Review,
    Some("plan" | "strategist" | "planning") => TaskCategory::Planning,
    _ => TaskCategory::Implementation,
};
```

### Step 2: Map task tier/size to complexity

```rust
let complexity = match task_def.tier.unwrap_or(0) {
    0..=1 => TaskComplexityBand::Trivial,
    2..=3 => TaskComplexityBand::Standard,
    4..=5 => TaskComplexityBand::Complex,
    _ => TaskComplexityBand::Epic,
};
```

### Step 3: Read active_agents and queue depth from RunState

The executor's `RunState` or the parallel dispatch state tracks how many tasks are currently running:

```rust
active_agents: executor_state.active_task_count() as u32,
ready_queue_depth: executor_state.ready_queue_len() as u32,
```

If these methods don't exist, approximate from the event loop's local tracking of dispatched tasks.

### Step 4: Update the cost/latency normalization

Also in this area (lines 2415-2417), make the normalization denominators more reasonable:

```rust
// BEFORE: clamps nearly everything to 1.0
let normalized_cost = (state.cost_usd / 1.0).clamp(0.0, 1.0);
let normalized_latency = (wall_secs / 300.0).clamp(0.0, 1.0);

// AFTER: use task-tier-aware references
let cost_ref = match complexity {
    TaskComplexityBand::Trivial => 0.50,
    TaskComplexityBand::Standard => 2.00,
    TaskComplexityBand::Complex => 10.00,
    TaskComplexityBand::Epic => 50.00,
};
let latency_ref = match complexity {
    TaskComplexityBand::Trivial => 60.0,
    TaskComplexityBand::Standard => 300.0,
    TaskComplexityBand::Complex => 900.0,
    TaskComplexityBand::Epic => 3600.0,
};
let normalized_cost = (state.cost_usd / cost_ref).clamp(0.0, 1.0);
let normalized_latency = (wall_secs / latency_ref).clamp(0.0, 1.0);
```
