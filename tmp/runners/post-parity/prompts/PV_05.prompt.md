# PV_05: Fix gate threshold write to not overwrite with empty on serialization failure

## Task
Prevent the gate threshold file from being destroyed when `serde_json::to_string_pretty` fails.

## Runner Context
Runner PV, batch 5 of 5. No dependencies.

## Problem
`event_loop.rs:2389-2392`:

```rust
// CURRENT (destructive on failure)
let _ = std::fs::write(
    &path,
    serde_json::to_string_pretty(&thresholds).unwrap_or_default(),
);
```

If serialization fails, `unwrap_or_default()` produces `""`. The file is overwritten with empty content, destroying the entire gate-threshold history. The `let _ = ...` also silently drops the write error.

## Exact Changes

### Step 1: Replace with match

```rust
// AFTER (safe)
match serde_json::to_string_pretty(&thresholds) {
    Ok(json) => {
        if let Err(e) = std::fs::write(&path, &json) {
            warn!(path = %path.display(), error = %e, "failed to write gate thresholds");
        }
    }
    Err(e) => {
        warn!(error = %e, "failed to serialize gate thresholds, skipping write");
    }
}
```

### Step 2: Apply same fix to any other `unwrap_or_default` + `fs::write` patterns

Search the file for other instances of `serde_json::to_string*.unwrap_or_default()` followed by `fs::write`. Apply the same pattern.
