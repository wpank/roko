# PW_01: Wire GET /api/bench/cost-summary endpoint for CostRace component

## Task
Implement the `/api/bench/cost-summary` endpoint that the demo app's `CostRace` component expects.

## Runner Context
Runner PW, batch 1 of 6. No dependencies.

## Problem
`demo/demo-app/src/components/CostRace.tsx:174` calls:
```ts
const payload = await get<CostRaceResponse>('/api/bench/cost-summary');
```

Expected response shape:
```ts
{ models: { model: string; cost_usd: number; tokens: number; runs: number; }[] }
```

The endpoint does not exist in `roko-serve`. `useApiWithFallback` falls back to `DEMO_COST_RACE` demo data.

## Exact Changes

### Step 1: Add handler in bench.rs

In `crates/roko-serve/src/routes/bench.rs`, add:

```rust
async fn cost_summary(
    State(state): State<Arc<AppState>>,
) -> Result<impl IntoResponse, ApiError> {
    // Read completed bench runs from state.bench_runs (or from disk at .roko/bench/)
    // Aggregate per-model: cost_usd sum, token sum, run count
    let runs = state.bench_store.list_runs().await?;
    let mut by_model: HashMap<String, (f64, u64, u32)> = HashMap::new();
    for run in &runs {
        for result in &run.results {
            let entry = by_model.entry(result.model.clone()).or_default();
            entry.0 += result.cost_usd;
            entry.1 += result.tokens;
            entry.2 += 1;
        }
    }
    let models: Vec<_> = by_model.into_iter().map(|(model, (cost, tokens, count))| {
        json!({ "model": model, "cost_usd": cost, "tokens": tokens, "runs": count })
    }).collect();
    Ok(Json(json!({ "models": models })))
}
```

### Step 2: Register route

Add `.route("/bench/cost-summary", get(cost_summary))` to the bench routes function.
