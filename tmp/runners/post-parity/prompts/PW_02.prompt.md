# PW_02: Wire GET /api/bench/runs/{id} and GET /api/bench/runs list endpoints

## Task
Implement the bench runs list and individual run detail endpoints the demo app expects.

## Runner Context
Runner PW, batch 2 of 6. No dependencies.

## Problem
`demo/demo-app/src/hooks/useBench.ts:116` calls `GET /api/bench/runs` (list).
`demo/demo-app/src/hooks/useBench.ts:233` and `BenchRunDetail.tsx:257` call `GET /api/bench/runs/{id}`.
Neither endpoint exists. The demo falls back to `DEMO_BENCH_RUNS`.

Expected shapes:
```ts
// List: BenchRun[]
// Detail: BenchRun
interface BenchRun {
    id: string; status: string; config: BenchConfig;
    results: BenchTaskResult[]; summary?: BenchRunSummary;
    started_at: string; finished_at?: string;
}
```

## Exact Changes

### Step 1: Add list handler

```rust
async fn list_bench_runs(
    State(state): State<Arc<AppState>>,
) -> Result<impl IntoResponse, ApiError> {
    let runs = state.bench_store.list_runs().await?;
    Ok(Json(runs))
}
```

### Step 2: Add detail handler

```rust
async fn get_bench_run(
    State(state): State<Arc<AppState>>,
    Path(id): Path<String>,
) -> Result<impl IntoResponse, ApiError> {
    let run = state.bench_store.get_run(&id).await?
        .ok_or_else(|| ApiError::not_found(format!("bench run {id} not found")))?;
    Ok(Json(run))
}
```

### Step 3: Register routes

```rust
.route("/bench/runs", get(list_bench_runs))
.route("/bench/runs/{id}", get(get_bench_run))
```

### Step 4: Also add export endpoint

`useBench.ts:276` calls `GET /api/bench/export/{runId}`. Add:
```rust
.route("/bench/export/{id}", get(get_bench_run))  // same handler, same data
```
