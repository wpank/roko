# PX_02: Wire DispatchModulation into ModelCallRequest (CRIT-06)

## Task
Apply the computed `DispatchModulation` (tier_bias, turn_limit_factor, exploration_rate) to the `ModelCallRequest` instead of discarding it.

## Runner Context
Runner PX, batch 2 of 4. Depends on PX_01.

## Problem
`effect_driver.rs:133-138` computes `DispatchModulation` from `policy.modulate_dispatch()` but never applies the result:

```rust
// CURRENT: computed but discarded
let modulation = policy.modulate_dispatch(&affect_state, &task_context).await?;
// modulation.tier_bias, modulation.turn_limit_factor, modulation.exploration_rate
// ... never used ...
```

## Exact Changes

### Step 1: Find where ModelCallRequest is constructed after modulation

The `DispatchModulation` should influence the `ModelCallRequest` that goes to the model call service. Look for where the request is built after `modulate_dispatch` returns.

### Step 2: Apply tier_bias to model selection

```rust
// Apply tier_bias: positive = prefer higher tier (more capable), negative = prefer cheaper
if modulation.tier_bias > 0.3 {
    request.model_hint = Some(ModelTierHint::PreferCapable);
} else if modulation.tier_bias < -0.3 {
    request.model_hint = Some(ModelTierHint::PreferCheap);
}
```

### Step 3: Apply turn_limit_factor

```rust
// Scale the turn limit by the factor (e.g., 0.5 = halve allowed turns under stress)
if let Some(max_turns) = request.max_turns {
    request.max_turns = Some((max_turns as f64 * modulation.turn_limit_factor) as u32);
}
```

### Step 4: Apply exploration_rate

```rust
// Pass exploration rate to CascadeRouter so it adjusts UCB1 exploration term
request.exploration_rate = Some(modulation.exploration_rate);
```
