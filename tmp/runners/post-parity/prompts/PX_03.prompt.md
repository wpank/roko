# PX_03: Fix feedback_service flush_async race condition (CRIT-07)

## Task
Fix the race condition in `flush_async` where `episode_logger` is checked and buffer is read in separate lock scopes.

## Runner Context
Runner PX (Critical Fixes), batch 3 of 4. No dependencies.

## Problem
`feedback_service.rs:609-627` has a subtle race: `flush_async()` checks `self.episode_logger.is_some()` at line 610, then calls `pending_workflow_episodes()` which acquires `self.buffer` lock at line 631. Between the check and the lock, another thread could modify the buffer or the logger, causing stale reads or missed flushes.

## Current Code

**flush_async** — `crates/roko-learn/src/feedback_service.rs:609-627`:
```rust
pub async fn flush_async(&self) -> Result<()> {
    let episodes = if self.episode_logger.is_some() {      // L610: check (no lock)
        self.pending_workflow_episodes()?                    // L611: lock acquired inside
    } else {
        Vec::new()
    };

    self.flush()?;                                           // L616: separate lock scope

            if let Err(err) = logger.append(&episode).await {
                tracing::warn!("failed to append episode: {err}");
            }
        }
    }

    Ok(())
}
```

## Race Scenario
1. Thread A calls `flush_async()`, checks `episode_logger.is_some()` → true (line 610)
2. Thread B calls `flush()`, acquires buffer lock, drains all events (line 616 equivalent)
3. Thread A calls `pending_workflow_episodes()`, acquires lock → buffer is now EMPTY
4. Thread A's `episodes` vec is empty — events were flushed by Thread B without episode logging

## Exact Changes

### Step 1: Combine episode extraction and buffer flush under one lock

Replace `flush_async` at line 609 with:

```rust
pub async fn flush_async(&self) -> Result<()> {
    // Acquire lock once: extract episodes AND drain buffer atomically
    let (events, episodes) = {
        let mut buf = self
            .buffer
            .lock()
            .map_err(|error| RokoError::Invalid(format!("lock poisoned: {error}")))?;

        let episodes = if self.episode_logger.is_some() {
            self.extract_episodes_from(&buf)
        } else {
            Vec::new()
        };

        let events: Vec<FeedbackEvent> = buf.drain(..).collect();
        (events, episodes)
    };
    // Lock released here — I/O happens without holding lock

    // Write feedback events to disk
    if !events.is_empty() {
        self.write_events(&events)?;
    }

    // Append episodes to logger
    if let Some(ref logger) = self.episode_logger {
        for episode in episodes {
            if let Err(err) = logger.append(&episode).await {
                tracing::warn!("failed to append episode: {err}");
            }
        }
    }

    Ok(())
}
```

### Step 2: Extract episode filtering into helper

If `pending_workflow_episodes` filters buffer contents into Episodes, extract that logic into a non-locking helper:

```rust
/// Extract episodes from an already-locked buffer slice.
/// Does NOT acquire the lock — caller must hold it.
fn extract_episodes_from(&self, buf: &[FeedbackEvent]) -> Vec<Episode> {
    buf.iter()
        .filter_map(|event| event.as_episode())
        .collect()
}
```

### Step 3: Update `flush()` (sync version) similarly

If `flush()` (the sync counterpart) has the same pattern, fix it too — acquire lock once, drain + extract in one scope.
