# PX_04: Fix WorkflowEngine resume missing StateCheckpointed event (CRIT-09/10)

## Task
Add `StateCheckpointed` event emission to `WorkflowEngine::resume()` so observers know the checkpoint file that was loaded.

## Runner Context
Runner PX (Critical Fixes), batch 4 of 4. No dependencies.

## Problem
CRIT-09 was previously marked as "PhaseTransition missing on resume" — but `resume()` already emits `PhaseTransition` at line 381 via `emit_phase_transition()`. The real remaining issue is CRIT-10: `resume()` does NOT emit `StateCheckpointed` at all, so TUI/SSE observers don't know which checkpoint file was loaded or that a resume occurred.

During normal execution, `effect_driver.rs:458` emits `StateCheckpointed` when writing checkpoints. But the resume path (loading a checkpoint) has no corresponding event, creating an asymmetry.

## Current Code

**resume()** — `crates/roko-runtime/src/workflow_engine.rs:309-478`:
```rust
pub async fn resume(&mut self, checkpoint_path: &Path) -> Result<WorkflowRunReport> {
    // ... loads checkpoint from disk ...
    // Line 381: emits PhaseTransition (CRIT-09 is already fixed)
    self.emit_phase_transition(&run_id, "checkpoint", pipeline.phase.label());
    // ... resumes pipeline execution ...
    // NO StateCheckpointed event emitted
}
```

**emit_phase_transition helper** — `workflow_engine.rs:488-496`:
```rust
fn emit_phase_transition(&self, run_id: &str, from: &str, to: &str) {
    let event = RuntimeEvent::PhaseTransition {
        run_id: run_id.to_string(),
        from: from.to_string(),
        to: to.to_string(),
    };
    self.emit(event);
}
```

**Normal checkpoint emission** — `effect_driver.rs:458`:
Emits `StateCheckpointed` when writing checkpoints during execution. Resume path has no analogous emission.

## Exact Changes

### Step 1: Add StateCheckpointed emission after checkpoint load

In `workflow_engine.rs`, inside `resume()` after the checkpoint is successfully loaded and before pipeline execution resumes (around line 381, after the PhaseTransition emission):

```rust
// After PhaseTransition emission at line 381:
self.emit(RuntimeEvent::StateCheckpointed {
    run_id: run_id.clone(),
    path: checkpoint_path.display().to_string(),
});
```

### Step 2: Add a ResumeStarted event variant (if needed)

If observers need to distinguish "resumed from checkpoint" from "wrote new checkpoint", add a distinct event:

```rust
// In runtime_event.rs, add variant:
ResumedFromCheckpoint {
    run_id: String,
    checkpoint_path: String,
    phase: String,
},
```

Then in `resume()`:
```rust
self.emit(RuntimeEvent::ResumedFromCheckpoint {
    run_id: run_id.clone(),
    checkpoint_path: checkpoint_path.display().to_string(),
    phase: pipeline.phase.label().to_string(),
});
```

**Prefer Step 1** (reuse existing StateCheckpointed) unless the distinction matters for TUI display. Step 2 is optional.
