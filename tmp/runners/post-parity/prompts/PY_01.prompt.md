# PY_01: Audit direct agent_exec call sites and document dispatch bypass paths

## Task
Audit all direct `agent_exec` call sites and document which CLI entrypoints bypass the WorkflowEngine.

## Runner Context
Runner PY, batch 1 of 4. No dependencies.

## Problem
`crates/roko-cli/src/agent_exec.rs` provides direct agent execution that bypasses: provider-neutral events, prompt diagnostics, feedback fan-out, crash-safe resume, and episode attribution. Multiple CLI commands use it:
- `commands/prd.rs` — PRD draft, plan generation
- `commands/research.rs` — research topic, enhance
- `commands/plan.rs` — plan generate/regenerate
- `unified.rs` — unified dispatch
- `chat_inline.rs` — dispatch_direct path

## Exact Changes

### Step 1: Grep for all import sites

Search for `use crate::agent_exec` and `use super::agent_exec` and `agent_exec::` across `crates/roko-cli/src/`. Document each call site with:
- File and line number
- Which function is called
- What the caller expects back
- What's missing (events, episodes, resume, etc.)

### Step 2: Categorize by migration difficulty

For each call site, classify:
- **Easy**: Can replace with `run_with_workflow_engine()` directly (same input/output shape)
- **Medium**: Needs adapter (different return type, but same semantics)
- **Hard**: Deeply integrated with caller's state (e.g., streaming callbacks)

### Step 3: Document in a migration table

Add a comment block at the top of `agent_exec.rs`:
```rust
// Migration status: agent_exec → WorkflowEngine
// | Caller | Function | Difficulty | Status |
// | commands/prd.rs:N | run_agent | Easy | PY_02 |
// | commands/research.rs:N | run_agent | Easy | PY_03 |
// | ... | ... | ... | ... |
```
