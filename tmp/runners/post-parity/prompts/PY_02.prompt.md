# PY_02: Route PRD draft/plan generation through WorkflowEngine dispatch

## Task
Replace direct `agent_exec` calls in PRD commands with `run_with_workflow_engine` so PRD work gets episode recording and event publishing.

## Runner Context
Runner PY, batch 2 of 4. Depends on PY_01.

## Problem
`commands/prd.rs` calls `agent_exec::run_agent(...)` for draft generation and plan generation. This bypasses the WorkflowEngine, so:
- No episodes recorded for PRD agent work
- No efficiency events emitted
- No cost tracking
- No routing observations

## Exact Changes

### Step 1: Find the agent_exec calls in prd.rs

From PY_01's audit, locate the exact lines where `agent_exec` is called.

### Step 2: Replace with WorkflowEngine dispatch

```rust
// BEFORE
let result = agent_exec::run_agent(&workdir, &config, &prompt, &system_prompt).await?;

// AFTER
let result = crate::run::run_with_workflow_engine(
    &workdir,
    &config,
    &prompt,
    Some(&system_prompt),
    None, // external_hub
).await?;
```

### Step 3: Adapt return type if needed

`agent_exec` may return a raw string output. `run_with_workflow_engine` returns a `RunReport`. Extract the output text:
```rust
let report = crate::run::run_with_workflow_engine(...).await?;
let output = report.output_text.unwrap_or_default();
```
