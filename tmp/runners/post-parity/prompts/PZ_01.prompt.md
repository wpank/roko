# PZ_01: Fix ProcessSupervisor orphan PID race

## Task
Persist the agent PID to `.roko/runtime/agent-pids.json` before the spawn function returns, preventing orphan PIDs on crash.

## Runner Context
Runner PZ, batch 1 of 4. No dependencies.

## Problem
`crates/roko-agent/src/process/registry.rs` uses a global `OnceLock<Mutex<HashSet<u32>>>` to track PIDs. If Roko exits between spawning an agent process and calling `persist_pids()`, the PID is lost and `cleanup_orphaned_agents()` on next start can't find it.

## Exact Changes

### Step 1: Find the spawn → register → persist sequence

Search `registry.rs` for the function that spawns a child process and registers its PID. The pattern is likely:
```rust
let child = Command::new(cmd).spawn()?;
let pid = child.id();
register_pid(pid);      // adds to in-memory set
persist_pids();          // writes to disk — may not reach here on crash
```

### Step 2: Persist BEFORE returning the child handle

```rust
// AFTER: persist before returning
let child = Command::new(cmd).spawn()?;
let pid = child.id();
register_pid(pid);
persist_pids()?;  // disk write before caller gets the handle
Ok(child)
```

### Step 3: Make persist_pids non-fallible for the caller

If `persist_pids()` fails (disk full, permissions), log a warning but don't fail the spawn:
```rust
if let Err(e) = persist_pids() {
    warn!(pid, error = %e, "failed to persist agent PID, orphan possible on crash");
}
```

## Write Scope
- `crates/roko-agent/src/process/registry.rs`

## Acceptance Criteria
- PID written to disk before spawn returns
- Crash between spawn and persist is impossible (no window)
- `cleanup_orphaned_agents()` finds all previously spawned PIDs
- Disk write failure logs warning, doesn't block spawn
