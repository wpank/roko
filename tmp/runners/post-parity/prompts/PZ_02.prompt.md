# PZ_02: Fix contract_for_role fail-open to use RestrictedFallback

## Task
Change `contract_for_role` to use `ContractLoadMode::RestrictedFallback` instead of `AgentContract::permissive()` when contract loading fails.

## Runner Context
Runner PZ, batch 2 of 4. No dependencies.

## Problem
`crates/roko-agent/src/safety/mod.rs:864-871`:
```rust
// CURRENT: fail-open — typo in role name silently removes ALL enforcement
contract_for_role(role)
    .unwrap_or_else(|_| AgentContract::permissive(role))
```

A typo in the role name (e.g., "implemnter" instead of "implementer") means the contract YAML isn't found, and the agent gets full permissions with no restrictions.

## Exact Changes

### Step 1: Find `contract_for_role` usage

Search `safety/mod.rs` for `contract_for_role` and its `unwrap_or_else` fallback.

### Step 2: Replace permissive fallback with restricted

```rust
// AFTER: fail-closed — missing contract = restrictive defaults
contract_for_role(role).unwrap_or_else(|e| {
    warn!(role, error = %e, "contract not found for role, using restricted fallback");
    AgentContract::restricted_fallback(role)
})
```

### Step 3: Verify RestrictedFallback exists

Check if `AgentContract::restricted_fallback` exists. If not, create it:
```rust
impl AgentContract {
    pub fn restricted_fallback(role: &str) -> Self {
        Self {
            role: role.to_string(),
            allowed_tools: vec![],  // no tools allowed
            allowed_paths: vec![],  // no file access
            max_turns: 5,           // strict turn limit
            permissions: Permissions::read_only(),
            ..Default::default()
        }
    }
}
```
