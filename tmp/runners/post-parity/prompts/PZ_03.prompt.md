# PZ_03: Wire ToolLoop checkpoint resume into runner prepare_resume flow

## Task
Connect the ToolLoop's checkpoint mechanism to the runner's resume flow so tool-loop state survives crashes.

## Runner Context
Runner PZ, batch 3 of 4. No dependencies.

## Problem
`crates/roko-agent/src/tool_loop/checkpoint.rs` builds resumable state snapshots, but the runner's `prepare_resume` in `resume.rs` doesn't know about them. A crash mid-tool-loop re-executes side-effecting tools (write_file, edit_file, bash) from the beginning.

## Exact Changes

### Step 1: Find the checkpoint save in tool_loop

Search `checkpoint.rs` for the snapshot save function. It likely writes to a file or returns a serializable state.

### Step 2: Find prepare_resume in runner/resume.rs

Search `resume.rs` for `prepare_resume` or the resume entry point.

### Step 3: Wire checkpoint into RunStateSnapshot

In `persist.rs`, the `RunStateSnapshot` struct stores run state. Add a field for tool-loop checkpoints:
```rust
pub struct RunStateSnapshot {
    // ... existing fields ...
    pub tool_loop_checkpoints: HashMap<String, ToolLoopCheckpoint>,  // task_id → checkpoint
}
```

### Step 4: Save checkpoints during execution

After each tool-loop iteration, save the checkpoint:
```rust
// In the dispatch path, after each tool iteration
if let Some(checkpoint) = tool_loop.take_checkpoint() {
    snapshot.tool_loop_checkpoints.insert(task_id.clone(), checkpoint);
    save_snapshot(&snapshot)?;
}
```

### Step 5: Restore checkpoints on resume

In `prepare_resume`, load tool-loop checkpoints and pass them to the task dispatch:
```rust
if let Some(checkpoint) = snapshot.tool_loop_checkpoints.get(&task_id) {
    tool_loop.restore_from_checkpoint(checkpoint)?;
}
```
