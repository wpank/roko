# PZ_04: Fix dangerously_skip_permissions default to false in ClaudeCliAgent

## Task
Change the default for `dangerously_skip_permissions` from `true` to `false` in `ClaudeCliAgent`.

## Runner Context
Runner PZ, batch 4 of 4. No dependencies.

## Problem
`crates/roko-agent/src/claude_cli_agent.rs:128` defaults `dangerously_skip_permissions` to `true`. This means every Claude CLI agent spawned without explicit safety configuration runs with no permission checks — the agent can execute any tool, write any file, run any bash command without user confirmation.

## Exact Changes

### Step 1: Find the default

Search `claude_cli_agent.rs` for `dangerously_skip_permissions`. The default is set in a builder or constructor.

### Step 2: Change default to false

```rust
// BEFORE

// AFTER
dangerously_skip_permissions: false,
```

Search the codebase for places that explicitly need `dangerously_skip_permissions: true`:

- Test helpers

For plan runner: set it explicitly in the dispatch config:
