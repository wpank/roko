# QA_01: Wire ContextTier into PromptAssemblyService

## Task
Make `PromptAssemblyService` accept a `ContextTier` and use it to set per-tier token budgets when assembling prompts.

## Runner Context
Runner QA, batch 1 of 5. No dependencies.

## Problem
`ContextTier` (Surgical/Focused/Full) exists in `context_provider.rs` with per-tier budgets (4K/12K/24K) and model-slug detection. `PromptAssemblyService` has an optional `token_budget` field but never consults `ContextTier`. The result: every prompt gets the same budget regardless of model capability.

## Current Code

**ContextTier enum** -- `crates/roko-compose/src/context_provider.rs:39-46`:
```rust
pub enum ContextTier {
    Surgical,  // 4K budget
    Focused,   // 12K budget
    Full,      // 24K budget
}
```

**ContextTier::from_task_and_model** -- `context_provider.rs:54-64`:
```rust
pub fn from_task_and_model(task_tier: &str, model_slug: &str) -> Self {
    if is_local_model(model_slug) { return Self::Surgical; }
    match task_tier {
        "mechanical" => Self::Surgical,
        "architectural" => Self::Full,
        _ => Self::Focused,
    }
}
```

**ContextTier::default_token_budget** -- `context_provider.rs:68-74`:
```rust
pub const fn default_token_budget(self) -> usize {
    match self {
        Self::Surgical => 4_000,
        Self::Focused => 12_000,
        Self::Full => 24_000,
    }
}
```

**PromptAssemblyService struct** -- `prompt_assembly_service.rs:47-70`:
```rust
pub struct PromptAssemblyService {
    // ...
    token_budget: Option<usize>,
    section_effectiveness: Option<HashMap<String, f64>>,
}
```

## Exact Changes

### Step 1: Add context_tier field to PromptAssemblyService

In `crates/roko-compose/src/prompt_assembly_service.rs`, add to the struct after `section_effectiveness`:

```rust
    /// Context tier for model-aware token budgeting.
    context_tier: Option<ContextTier>,
```

Add to the `new()` method's Self initializer:

```rust
    context_tier: None,
```

### Step 2: Add builder method

After the existing `with_section_effectiveness` method, add:

```rust
    /// Set the context tier for model-aware token budgeting.
    ///
    /// When set, overrides `token_budget` with the tier's default budget
    /// unless an explicit `token_budget` was already provided.
    #[must_use]
    pub fn with_context_tier(mut self, tier: ContextTier) -> Self {
        self.context_tier = Some(tier);
        self
    }

    /// Derive and set the context tier from task tier and model slug.
    #[must_use]
    pub fn with_model_aware_budget(mut self, task_tier: &str, model_slug: &str) -> Self {
        self.context_tier = Some(ContextTier::from_task_and_model(task_tier, model_slug));
        self
    }
```

### Step 3: Use context_tier in assemble()

In the `assemble()` method (the `PromptAssembler` trait impl), resolve the effective budget at the top:

```rust
    let effective_budget = self.token_budget.unwrap_or_else(|| {
        self.context_tier
            .map(|t| t.default_token_budget())
            .unwrap_or(12_000) // Focused default
    });
```

Replace any existing `self.token_budget.unwrap_or(...)` calls with `effective_budget`.

### Step 4: Add import

At the top of `prompt_assembly_service.rs`, add:

```rust
use crate::context_provider::ContextTier;
```
