# QA_02: Wire BudgetPredictor into live prompt assembly path

## Task
Make `PromptAssemblyService` consult the `BudgetPredictor` to set data-driven token budgets instead of fixed defaults.

## Runner Context
Runner QA, batch 2 of 5. No dependencies.

## Problem
`BudgetPredictor` exists with `predict()` and `record()` methods, plus persistence helpers. It learns optimal token budgets from historical task outcomes via EMA. But it is never called from `PromptAssemblyService` -- budgets are always the fixed tier defaults or an explicit override.

## Exact Changes

### Step 1: Add predictor field to PromptAssemblyService

In `crates/roko-compose/src/prompt_assembly_service.rs`, add to the struct:

```rust
    /// Optional budget predictor for data-driven token allocation.
    budget_predictor: Option<Arc<Mutex<BudgetPredictor>>>,
```

Initialize in `new()`:
```rust
    budget_predictor: None,
```

### Step 2: Add builder method

```rust
    /// Attach a budget predictor for data-driven token allocation.
    ///
    /// When attached, `assemble()` will call `predictor.predict()` using
    /// the task's role, complexity, and domain to determine the budget.
    /// Falls back to tier defaults if the predictor has no history.
    #[must_use]
    pub fn with_budget_predictor(mut self, predictor: Arc<Mutex<BudgetPredictor>>) -> Self {
        self.budget_predictor = Some(predictor);
        self
    }
```

### Step 3: Add predict method

```rust
    /// Predict a token budget from task features, falling back to the
    /// explicit budget or tier default.
    fn predicted_budget(&self, role: &str, complexity: &str, domain: &str) -> Option<usize> {
        let predictor = self.budget_predictor.as_ref()?;
        let features = TaskFeatures::new(role, complexity, domain);
        let guard = predictor.lock().unwrap_or_else(|e| e.into_inner());
        if guard.has_history(&features) {
            Some(guard.predict(&features) as usize)
        } else {
            None
        }
    }
```

### Step 4: Integrate into budget resolution in assemble()

Where the effective budget is resolved (either existing or after QA_01):

```rust
    let effective_budget = self.token_budget
        .or_else(|| self.predicted_budget(
            spec.role_name().unwrap_or("implementer"),
            spec.complexity().unwrap_or("standard"),
            spec.domain().unwrap_or("code"),
        ))
        .unwrap_or(12_000);
```
