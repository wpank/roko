# QA_03: Wire SectionInfluence feedback into prompt assembly

## Task
Make `PromptAssemblyService` use `SectionInfluence` weights to prioritize high-lift prompt sections and deprioritize low-lift ones during assembly.

## Runner Context
Runner QA, batch 3 of 5. Depends on QA_01.

## Problem
`SectionInfluence` tracks per-section success rates via leave-one-out approximation. It has `weights()` returning per-section multipliers and `lift_for()` returning marginal lift. `PromptAssemblyService` has a `section_effectiveness: Option<HashMap<String, f64>>` field and `with_section_effectiveness()` builder, but the influence data never flows from `SectionInfluence` into this field.

### Step 2: Add builder method

```rust
    /// Attach a live section influence tracker.
    ///
    /// When attached, `assemble()` merges influence weights into
    /// `section_effectiveness` scores, giving data-driven priority to
    /// sections that historically correlate with task success.
    #[must_use]
    pub fn with_section_influence(mut self, influence: Arc<Mutex<SectionInfluence>>) -> Self {
        self.section_influence = Some(influence);
        self
    }
```

### Step 3: Merge influence weights in assemble()

At the start of `assemble()`, before sections are scored, merge:

```rust
    // Merge live influence weights into section effectiveness
    let merged_effectiveness = {
        let mut base = self.section_effectiveness.clone().unwrap_or_default();
        if let Some(influence) = &self.section_influence {
            let guard = influence.lock().unwrap_or_else(|e| e.into_inner());
            for (section, weight) in guard.weights() {
                let existing = base.entry(section).or_insert(1.0);
                // Blend: 70% existing static score, 30% live influence
                *existing = *existing * 0.7 + weight * 0.3;
            }
        }
        if base.is_empty() { None } else { Some(base) }
    };
```

Use `merged_effectiveness` wherever `self.section_effectiveness` was previously used.

### Step 4: Add method to record outcome for feedback loop

```rust
    /// Record which sections were included and whether the task succeeded.
    ///
    /// Call this after gate results are known to update influence tracking.
    pub fn record_section_outcome(
        &self,
        included_sections: &[String],
        all_sections: &[String],
        success: bool,
    ) {
        if let Some(influence) = &self.section_influence {
            let mut guard = influence.lock().unwrap_or_else(|e| e.into_inner());
            guard.record(included_sections, all_sections, success);
        }
    }
```
