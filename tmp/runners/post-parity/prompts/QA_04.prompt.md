# QA_04: Wire conversation compaction for long chat sessions

## Task
Add a conversation compaction method to `PromptAssemblyService` that summarizes chat history when it exceeds the token budget.

## Runner Context
Runner QA, batch 4 of 5. No dependencies.

## Problem
Long chat sessions accumulate message history that eventually exceeds the model's context window. There is no mechanism to compact older messages into a summary while preserving recent turns. The `TokenCounter` exists but is never used to detect when compaction is needed.

## Current Code

**TokenCounter** -- `crates/roko-compose/src/token_counter.rs`:
Provides `count_tokens(text: &str) -> usize` for approximate token counting.

**PromptAssemblyService** -- `prompt_assembly_service.rs:47-70`:
Has `token_budget: Option<usize>` but no history management.

**ChatAgentSession api_history** -- `chat_session.rs:307+`:
Stores `Vec<ChatMessage>` that grows unbounded.

## Exact Changes

### Step 1: Add compaction method to PromptAssemblyService

In `crates/roko-compose/src/prompt_assembly_service.rs`, add:

```rust
    /// Compact a conversation history to fit within the token budget.
    ///
    /// Strategy:
    /// 1. Count tokens in the full history
    /// 2. If within budget, return as-is
    /// 3. If over budget, keep the most recent `preserve_recent` messages
    ///    verbatim and summarize older messages into a single system message
    ///
    /// The summary prefix is "[Conversation summary] " followed by a
    /// concatenation of older message roles and truncated content.
    pub fn compact_history(
        &self,
        history: &[ChatMessage],
        budget_tokens: usize,
        preserve_recent: usize,
    ) -> Vec<ChatMessage> {
        use crate::token_counter::count_tokens;

        let total_tokens: usize = history.iter().map(|m| count_tokens(&m.content)).sum();

        if total_tokens <= budget_tokens || history.len() <= preserve_recent {
            return history.to_vec();
        }

        let split_point = history.len().saturating_sub(preserve_recent);
        let older = &history[..split_point];
        let recent = &history[split_point..];

        // Build summary of older messages
        let mut summary_parts = Vec::with_capacity(older.len());
        for msg in older {
            let role_label = match msg.role {
                MessageRole::User => "User",
                MessageRole::Assistant => "Assistant",
                MessageRole::System => "System",
                _ => "Other",
            };
            // Take first 200 chars of each older message
            let truncated: String = msg.content.chars().take(200).collect();
            summary_parts.push(format!("[{role_label}]: {truncated}"));
        }

        let summary_text = format!(
            "[Conversation summary — {} earlier messages compacted]\n{}",
            older.len(),
            summary_parts.join("\n")
        );

        // Trim summary if still too large
        let summary_budget = budget_tokens.saturating_sub(
            recent.iter().map(|m| count_tokens(&m.content)).sum::<usize>()
        );
        let summary_trimmed = if count_tokens(&summary_text) > summary_budget {
            summary_text.chars().take(summary_budget * 3).collect() // ~3 chars per token
        } else {
            summary_text
        };

        let mut compacted = Vec::with_capacity(1 + recent.len());
        compacted.push(ChatMessage {
            role: MessageRole::System,
            content: summary_trimmed,
        });
        compacted.extend_from_slice(recent);
        compacted
    }

    /// Check whether a conversation history exceeds the budget and needs
    /// compaction. Returns the estimated token count.
    pub fn history_token_count(&self, history: &[ChatMessage]) -> usize {
        use crate::token_counter::count_tokens;
        history.iter().map(|m| count_tokens(&m.content)).sum()
    }
```

## Acceptance Criteria
- `compact_history()` returns unmodified history when within budget
- Over-budget history is split: older messages summarized, recent preserved
- `preserve_recent` parameter controls how many recent messages to keep verbatim
- Summary is itself trimmed if it would exceed remaining budget
- `history_token_count()` gives callers a way to check before compacting

## Do NOT
- Change `TokenCounter` or `ChatMessage` types
- Add LLM-based summarization (that is a separate, heavier feature)
- Modify `ChatAgentSession` (caller is responsible for using compact_history)
- Touch files outside write scope
