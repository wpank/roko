# QA_05: Activate VCG auction for tight budgets

## Task
Wire the existing `vcg_allocate` function into `PromptAssemblyService` so it activates when the token budget is tight relative to the total section demand.

## Runner Context
Runner QA, batch 5 of 5. Depends on QA_01.

## Problem
`vcg_allocate()` exists in `auction.rs` and is fully implemented with greedy value-density allocation, VCG payments, and diagnostics. But `PromptAssemblyService` never calls it -- the greedy path always dominates. When the budget is tight (total section tokens > budget), VCG should allocate optimally.

## Exact Changes

### Step 1: Add VCG threshold constant

In `crates/roko-compose/src/prompt_assembly_service.rs`, add near the top:

```rust
/// Activate VCG auction when total section demand exceeds this fraction
/// of the available budget. Below this threshold, greedy inclusion is fine.
const VCG_ACTIVATION_RATIO: f64 = 0.85;
```

### Step 2: Add auction method

```rust
    /// Run VCG auction to allocate token budget across sections.
    ///
    /// Called when total section tokens exceed `VCG_ACTIVATION_RATIO` of
    /// the budget. Returns the names of sections that won the auction.
    fn auction_sections(
        &self,
        sections: &[(String, usize, f64)], // (name, tokens, effectiveness_score)
        budget: usize,
    ) -> Vec<String> {
        use crate::auction::{AffectModulation, VcgBid, vcg_allocate};

        let bids: Vec<VcgBid> = sections
            .iter()
            .map(|(name, tokens, score)| VcgBid {
                bidder: crate::prompt::AttentionBidder::TaskContext, // default bidder
                section_name: name.clone(),
                tokens: *tokens,
                raw_bid: *score,
                adjusted_bid: *score,
                valence: 0.0,
            })
            .collect();

        let modulation = AffectModulation {
            pleasure: 0.0,
            arousal: 0.0,
            modulation_strength: 0.0,
        };

        let allocation = vcg_allocate(bids, budget, &modulation);

        tracing::debug!(
            winners = allocation.winners.len(),
            excluded = allocation.excluded.len(),
            utilization = allocation.budget_utilization,
            "VCG auction completed"
        );

        allocation
            .winners
            .into_iter()
            .map(|w| w.section_name)
            .collect()
    }

    /// Check whether VCG auction should be activated.
    fn should_auction(&self, total_section_tokens: usize, budget: usize) -> bool {
        budget > 0 && (total_section_tokens as f64 / budget as f64) > VCG_ACTIVATION_RATIO
    }
```

### Step 3: Integrate into assemble()

In the `assemble()` method, after sections are collected and scored but before they are concatenated into the final prompt, add a gate:

```rust
    // If sections exceed budget, run VCG auction to select optimally
    let included_section_names = if self.should_auction(total_section_tokens, effective_budget) {
        let section_data: Vec<(String, usize, f64)> = sections.iter()
            .map(|s| (s.name.clone(), s.estimated_tokens, s.effectiveness_score))
            .collect();
        self.auction_sections(&section_data, effective_budget)
    } else {
        sections.iter().map(|s| s.name.clone()).collect()
    };
```

Then filter sections to only those in `included_section_names`.
