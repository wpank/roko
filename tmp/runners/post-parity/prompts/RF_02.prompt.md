# RF_02: Shared config cache -- load once, Arc through

## Task
Load config once at run start and pass it as `Arc<Config>` through the entire run path instead of reloading from disk at each usage point.

## Runner Context
Runner RF, batch 2 of 11. No dependencies.

## Problem
Config is loaded from `roko.toml` multiple times during a single run: once at startup, potentially again in dispatch, again in gate configuration, etc. Each load involves disk I/O, TOML parsing, and validation. Loading once and sharing via `Arc` eliminates redundant work.

## Current Code

**Config loading** -- various call sites load config via `Config::load()` or `load_layered()`.

**run.rs** -- `crates/roko-cli/src/run.rs`:
Entry point that sets up the run. Multiple functions may call config loading.

## Exact Changes

### Step 1: Load config once and wrap in Arc

In `crates/roko-cli/src/run.rs`, at the start of the run function:

```rust
use std::sync::Arc;

// Load config once for the entire run
let config = Arc::new(load_layered(&workdir)?);
```

### Step 2: Pass Arc<Config> to subsystems

Change function signatures that currently take `&Config` or reload config to accept `Arc<Config>`:

```rust
// Before
fn setup_dispatch(workdir: &Path, config: &Config) -> Result<Dispatcher> { ... }

// After
fn setup_dispatch(workdir: &Path, config: &Arc<Config>) -> Result<Dispatcher> { ... }
```

### Step 3: Replace config reloads with Arc clones

Search for any `Config::load()`, `load_layered()`, or `RokoConfig::load()` calls that happen after the initial load and replace them with `Arc::clone(&config)`:

```rust
// Before
let config = Config::load(&workdir)?;

// After
let config = Arc::clone(&shared_config);
```

### Step 4: Ensure no mutable access to config

If any code mutates config after loading, move those mutations to happen before the Arc wrap:

```rust
let mut config = load_layered(&workdir)?;
// Apply any runtime overrides
if let Some(model) = args.model.as_ref() {
    config.agent.model = model.clone();
}
let config = Arc::new(config);
```

## Acceptance Criteria
- Config loaded exactly once per run
- Shared via `Arc<Config>` to all subsystems that need it
- No redundant `Config::load()` calls after the initial load
- All existing config access patterns still work (via Arc deref)
- Config mutations happen before the Arc wrap

## Do NOT
- Change the `Config` or `RokoConfig` types
- Add Mutex (config is read-only after initial load)
- Change config loading in other crates
- Touch files outside write scope
