# RF_03: LearningRuntime single-open -- avoid redundant file opens

## Task
Open `LearningRuntime` once at session start and reuse it across the entire run, avoiding redundant file opens for episodes, efficiency, and routing data.

## Runner Context
Runner RF, batch 3 of 11. No dependencies.

## Problem
`LearningRuntime` manages persistence for episodes, efficiency events, cascade router, and experiments. If it is opened multiple times during a run, each open re-reads files from disk. Opening once and reusing avoids redundant I/O.

## Current Code

**LearningRuntime** -- `crates/roko-learn/src/runtime_feedback.rs`:
```rust
pub struct LearningRuntime {
    pub paths: LearningPaths,
    // ...
}
impl LearningRuntime {
    pub fn from_roko_dir(roko_dir: &Path) -> Self { ... }
    pub fn record_run(&mut self, input: CompletedRunInput) { ... }
}
```

**run.rs** -- `crates/roko-cli/src/run.rs`:
May create `LearningRuntime` instances at multiple points.

## Exact Changes

### Step 1: Create LearningRuntime once at run start

In `crates/roko-cli/src/run.rs`, at initialization:

```rust
use roko_learn::runtime_feedback::LearningRuntime;

let learning_runtime = if roko_dir.exists() {
    Some(Arc::new(Mutex::new(LearningRuntime::from_roko_dir(&roko_dir))))
} else {
    None
};
```

### Step 2: Pass to subsystems that need it

Thread `Option<Arc<Mutex<LearningRuntime>>>` through to functions that record learning data.

### Step 3: Remove any redundant constructions

Search for other `LearningRuntime::from_roko_dir` or `LearningRuntime::new` calls in `run.rs` and replace with the shared instance:

```rust
// Before
let mut lr = LearningRuntime::from_roko_dir(&roko_dir);
lr.record_run(input);

// After
if let Some(ref lr) = learning_runtime {
    let mut guard = lr.lock().unwrap_or_else(|e| e.into_inner());
    guard.record_run(input);
}
```

### Step 4: Flush at run end

```rust
if let Some(ref lr) = learning_runtime {
    let guard = lr.lock().unwrap_or_else(|e| e.into_inner());
    if let Err(e) = guard.flush() {
        tracing::debug!(error = %e, "failed to flush learning runtime");
    }
}
```

## Acceptance Criteria
- `LearningRuntime` opened at most once per run
- Shared via `Arc<Mutex<>>` for thread-safe access
- Flushed at run end
- Missing `.roko/` = no learning runtime (graceful)

## Do NOT
- Change `LearningRuntime` struct or API
- Add async mutex (sync Mutex is fine for this)
- Touch files outside write scope
