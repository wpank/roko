# RF_04: Safety contract caching -- skip redundant YAML loads

## Task
Ensure `AgentContract` uses its existing `CONTRACT_CACHE` effectively and doesn't reload YAML files for the same role within a session.

## Runner Context
Runner RF, batch 4 of 11. No dependencies.

## Problem
`AgentContract` has a `CONTRACT_CACHE: LazyLock<RwLock<HashMap<String, AgentContract>>>` but it may not be used consistently. Every dispatch call that checks safety contracts should hit the cache instead of re-parsing YAML.

### Step 1: Verify cache-first pattern in load_for_role

In `crates/roko-agent/src/safety/contract.rs`, ensure `load_for_role()` checks the cache:

```rust
pub fn load_for_role(role: &str) -> Result<Self> {
    // Check cache first
    {
        let cache = CONTRACT_CACHE.read().unwrap_or_else(|e| e.into_inner());
        if let Some(contract) = cache.get(role) {
            return Ok(contract.clone());
        }
    }

    // Load from YAML
    let contract = Self::load_from_yaml(role)?;

    // Cache for reuse
    {
        let mut cache = CONTRACT_CACHE.write().unwrap_or_else(|e| e.into_inner());
        cache.insert(role.to_string(), contract.clone());
    }

    Ok(contract)
}
```
