# RF_07: Wire Express Gate Mode into GateService

## Task
Make `GateService` respect `GateMode` when deciding which gates to run.

## Runner Context
Runner RF, batch 7 of 11. Depends on RF_06.

## Problem
`GateService` runs all configured gates. With `GateMode` defined (RF_06), the service needs to filter its gate list based on the mode before executing.

## Exact Changes

### Step 1: Add gate_mode field

In `crates/roko-gate/src/gate_service.rs`:

```rust
pub struct GateService {
    adaptive: Option<Arc<Mutex<AdaptiveThresholds>>>,
    /// Gate execution mode -- filters which gates are actually run.
    gate_mode: Option<roko_runtime::pipeline_state::GateMode>,
}
```

### Step 2: Add builder method

```rust
    /// Set the gate execution mode.
    ///
    /// When set, only gates matching the mode's filter are executed.
    /// Express = compile only, Quick = compile + test, Full = all.
    #[must_use]
    pub fn with_gate_mode(mut self, mode: roko_runtime::pipeline_state::GateMode) -> Self {
        self.gate_mode = Some(mode);
        self
    }
```

### Step 3: Filter gates in the GateRunner impl

In the `GateRunner::run()` implementation, at the top, filter the config list:

```rust
    let effective_configs: Vec<&GateConfig> = if let Some(mode) = &self.gate_mode {
        let allowed_names = mode.filter_gates(
            &config.iter().map(|g| g.name.clone()).collect::<Vec<_>>()
        );
        config
            .iter()
            .filter(|g| allowed_names.contains(&g.name.as_str()))
            .collect()
    } else {
        config.iter().collect()
    };
```

Then iterate `effective_configs` instead of `config`.

### Step 4: Log mode

    if let Some(mode) = &self.gate_mode {
