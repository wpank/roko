# RF_08: Add --gates CLI flag to roko run

## Task
Add a `--gates` flag to `roko run` that lets users select the gate mode (express/quick/full).

## Runner Context
Runner RF, batch 8 of 11. Depends on RF_06.

## Problem
Users can't control gate mode from the CLI. The `--gates` flag should accept "express", "quick", or "full" and thread the mode through to `GateService`.

## Current Code

**main.rs** -- `crates/roko-cli/src/main.rs`:
Defines CLI args via clap.

**GateMode** (RF_06): Express/Quick/Full enum with `FromStr` impl.

## Exact Changes

### Step 1: Add --gates flag to CLI

In `crates/roko-cli/src/main.rs`, in the args struct:

```rust
    /// Gate execution mode: express (compile only), quick (compile+test), full (all gates)
    #[arg(long, default_value = "full")]
    gates: String,
```

### Step 2: Parse and thread to run function

```rust
    let gate_mode = roko_runtime::pipeline_state::GateMode::from_str_loose(&args.gates);
    tracing::info!(mode = %gate_mode, "gate mode selected");
```

### Step 3: Pass to run function

Thread `gate_mode` through to wherever `GateService` is constructed:

```rust
    let gate_service = GateService::new()
        .with_gate_mode(gate_mode)
        .with_adaptive_thresholds(thresholds);
```

### Step 4: Also accept --gates for plan run

If the `plan run` subcommand has a separate args path, add `--gates` there too.

## Write Scope
- `crates/roko-cli/src/main.rs`

## Verify
```bash
cargo build -p roko-cli 2>&1 | head -30
cargo test -p roko-cli 2>&1 | tail -20
```

## Acceptance Criteria
- `roko run "prompt" --gates express` runs only compile gates
- `roko run "prompt" --gates quick` runs compile + test
- `roko run "prompt" --gates full` runs all gates (default)
- `roko plan run plans/ --gates express` also works
- Invalid values default to "full"
