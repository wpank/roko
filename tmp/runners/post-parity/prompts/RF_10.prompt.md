# RF_10: Wire WarmDispatchPool into EffectDriver

## Task
Wire `WarmDispatchPool` into `EffectDriver` so model calls check the pool before cold-starting.

## Runner Context
Runner RF, batch 10 of 11. Depends on RF_09.

## Problem
`EffectDriver` dispatches model calls via `EffectServices.model_caller`. It always cold-starts. With `WarmDispatchPool`, it should check for a pre-warmed worker first.

## Exact Changes

### Step 1: Add warm pool to EffectServices

In `crates/roko-runtime/src/effect_driver.rs`:

```rust
pub struct EffectServices {
    // ... existing fields ...
    /// Optional warm dispatch pool for reducing cold-start latency.
    pub warm_pool: Option<Arc<crate::warm_dispatch_pool::WarmDispatchPool>>,
}
```

### Step 2: Check pool before model calls

In the model call dispatch path inside `EffectDriver`, before calling `model_caller`:

```rust
    // Check warm pool for a pre-warmed worker
    let warm_worker = self.services.warm_pool.as_ref()
        .and_then(|pool| pool.take(&model));

    if warm_worker.is_some() {
        tracing::debug!(model = %model, "using warm dispatch worker");
    }
```

### Step 3: Return worker after call

```rust
    // Return worker to pool if it was from the warm pool
    if let (Some(worker), Some(pool)) = (warm_worker, self.services.warm_pool.as_ref()) {
        pool.return_worker(worker);
    }
```

### Step 4: Log pool stats periodically

```rust
    if let Some(pool) = &self.services.warm_pool {
        let stats = pool.stats();
        tracing::debug!(
            hits = stats.hits,
            misses = stats.misses,
            idle = stats.idle_workers,
            "warm pool stats"
        );
    }
```
