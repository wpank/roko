# RF_11: Wire WarmDispatchPool into roko run startup

## Task
Create and pre-warm a `WarmDispatchPool` at `roko run` startup and pass it through to `EffectDriver`.

## Runner Context
Runner RF, batch 11 of 11. Depends on RF_09.

## Problem
`WarmDispatchPool` exists (RF_09) and `EffectDriver` supports it (RF_10), but nobody creates one at startup. The run command should create the pool, pre-warm it with the configured default model, and thread it through.

## Current Code

**run.rs** -- `crates/roko-cli/src/run.rs`:
Sets up config, dispatch, gates, and runs.

**WarmDispatchPool** (RF_09): `new()`, `warm_up()`.

## Exact Changes

### Step 1: Create pool at run start

In `crates/roko-cli/src/run.rs`, at initialization:

```rust
use roko_runtime::warm_dispatch_pool::{WarmDispatchPool, WarmPoolConfig};

let warm_pool = {
    let config = WarmPoolConfig {
        max_per_model: 2,
        idle_ttl: std::time::Duration::from_secs(300),
        pre_warm_models: vec![model_slug.clone()],
    };
    let pool = WarmDispatchPool::new(config);

    // Pre-warm the default model
    pool.warm_up(&model_slug);

    Arc::new(pool)
};
```

### Step 2: Pass to EffectServices

When constructing `EffectServices`:

```rust
    let services = EffectServices {
        default_model: model_slug.clone(),
        model_caller: model_caller.clone(),
        prompt_assembler: prompt_assembler.clone(),
        feedback_sink: feedback_sink.clone(),
        gate_runner: gate_runner.clone(),
        affect_policy: None,
        warm_pool: Some(warm_pool.clone()),
    };
```

### Step 3: Evict expired workers periodically

If the run has a periodic tick (e.g., every 60 seconds), add:

```rust
    warm_pool.evict_expired();
```

### Step 4: Log pool stats at run end

```rust
    let stats = warm_pool.stats();
    tracing::info!(
        hits = stats.hits,
        misses = stats.misses,
        evicted = stats.evicted,
        "warm dispatch pool summary"
    );
```
