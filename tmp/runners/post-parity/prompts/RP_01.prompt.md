# RP_01: Wire WorktreeManager per-task worktrees

## Task
Wire `WorktreeManager` into the `WorkflowEngine` so each agent dispatch gets its own isolated git worktree, preventing file conflicts between concurrent agents.

## Runner Context
Runner RP_A (Runner Patterns), batch 1 of 5. No dependencies.

## Problem
`WorktreeManager` is fully implemented in `crates/roko-orchestrator/src/worktree.rs` with create/remove/list/prune/health-check methods, but the `WorkflowEngine` dispatches agents into the main working directory. Multiple concurrent agents will step on each other's files.

## Current Code

**WorktreeManager** -- `crates/roko-orchestrator/src/worktree.rs:36-53`:

## Exact Changes

### Step 1: Add worktree option to WorkflowRunConfig

```rust
pub struct WorkflowRunConfig {
    pub prompt: String,
    pub workdir: PathBuf,
    pub workflow: WorkflowConfig,
    pub enabled_gates: Vec<String>,
    pub shell_gates: Vec<ShellGateCommand>,
    pub commit_prefix: Option<String>,
    /// Optional worktree manager for per-agent isolation.
    pub worktree_manager: Option<Arc<roko_orchestrator::WorktreeManager>>,
}
```

### Step 2: Create per-agent worktree in EffectDriver

In `spawn_agent()`, if a worktree manager is available, create a worktree:

```rust
let agent_workdir = if let Some(ref wt_mgr) = self.worktree_manager {
    let handle = wt_mgr.create(&agent_id, None).await.map_err(|e| {
        PipelineInput::AgentFailed {
            error: format!("worktree creation failed: {e}"),
        }
    })?;
    tracing::info!(
        agent_id = %agent_id,
        worktree = %handle.path.display(),
        branch = %handle.branch,
        "created per-agent worktree"
    );
    handle.path.clone()
} else {
    self.workdir.clone()
};
```

### Step 3: Clean up worktree after agent completes

After the agent finishes:
```rust
if let Some(ref wt_mgr) = self.worktree_manager {
    if let Err(e) = wt_mgr.remove(&agent_id).await {
        tracing::warn!(%e, agent_id = %agent_id, "failed to remove worktree");
    }
}
```

- `WorkflowRunConfig` has optional `worktree_manager` field
- Per-agent worktree created before dispatch when manager is present
- Worktree cleaned up after agent completes
