# RP_02: Wire serialized merge via MergeQueue

## Task
Wire the `MergeQueue` into the `WorkflowEngine` so completed agent worktrees are merged serialized through the queue, preventing file-level conflicts.

## Runner Context
Runner RP_A (Runner Patterns), batch 2 of 5. No dependencies.

## Problem
`MergeQueue` is fully implemented in `crates/roko-orchestrator/src/merge_queue.rs` with file-conflict-aware enqueueing, priority-based dequeuing, and retry logic. But the `WorkflowEngine` does not use it -- commits go directly to the working directory.

## Current Code

**MergeQueue** -- `crates/roko-orchestrator/src/merge_queue.rs:30-43`:
```rust
pub struct MergeRequest {
    pub plan_id: String,
    pub branch_name: String,
    pub files_changed: Vec<String>,
    pub priority: u32,
    pub retry_count: u32,
}
```
Has `enqueue()`, `try_dequeue()`, `complete()`, `fail()`.

## Exact Changes

### Step 1: Add MergeQueue option to WorkflowRunConfig


### Step 2: Add merge step in commit flow

In the workflow engine's commit phase, instead of committing directly:

```rust
if let Some(ref queue) = self.merge_queue {
    let request = MergeRequest::new(
        &self.run_id,
        &branch_name,
        files_changed,
        priority,
    );
    queue.enqueue(request);
    tracing::info!(run_id = %self.run_id, "enqueued merge request");

    // Wait for merge slot
    loop {
        if let Some(slot) = queue.try_dequeue() {
            if slot.plan_id == self.run_id {
                // Our turn -- perform the merge
                match perform_merge(&slot).await {
                    Ok(()) => {
                        queue.complete(&slot.plan_id);
                        break;
                    }
                    Err(e) => {
                        queue.fail(&slot.plan_id);
                        tracing::warn!(%e, "merge failed, will retry");
                    }
                }
            }
        }
        tokio::time::sleep(Duration::from_millis(500)).await;
    }
} else {
    // Direct commit (no queue)
    perform_commit(&self.workdir, &commit_message).await?;
}
```

### Step 3: Add merge helper

```rust
async fn perform_merge(slot: &MergeRequest) -> Result<(), String> {
    let output = tokio::process::Command::new("git")
        .args(["merge", "--no-ff", &slot.branch_name])
        .output()
        .await
        .map_err(|e| format!("git merge: {e}"))?;

    if output.status.success() {
        Ok(())
    } else {
        Err(String::from_utf8_lossy(&output.stderr).to_string())
    }
}
```
