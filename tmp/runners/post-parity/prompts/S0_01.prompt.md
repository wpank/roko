# S0_01: Fix `roko config mcp` unreachable panic

## Task
Replace the `unreachable!()` in the `ConfigCmd::Mcp` match arm of `dispatch_config()` with a handler that delegates to the existing `dispatch_mcp_cmd()` function.

## Runner Context
Post-parity Phase 0 (Stability), batch 1 of 25. No dependencies. Group S0A (Panics & Security).

## Problem
In `crates/roko-cli/src/commands/config_cmd.rs:208-210`, the `ConfigCmd::Mcp` arm contains:
```rust
ConfigCmd::Mcp { .. } => {
    unreachable!("mcp dispatched in dispatch_subcommand")
}
```

While this is normally intercepted at `main.rs:2141` before reaching `dispatch_config()`, there
are code paths (e.g. tests, library callers, future refactors) that could call `dispatch_config()`
directly with a `ConfigCmd::Mcp` variant. The same pattern exists for `Experiments`, `Plugins`,
and `Secrets`, but MCP is the most likely to be hit because `dispatch_mcp_cmd()` already exists
at `main.rs:2836` and is a natural candidate for extraction.

A real `dispatch_mcp_cmd()` function already exists in `main.rs:2836-2906` that handles
`ConfigMcpCmd::List`, `ConfigMcpCmd::Test`, and `ConfigMcpCmd::Add`. The fix is to call through
to it instead of panicking.

## Exact Changes

### Step 1: Make `dispatch_mcp_cmd` callable from config_cmd.rs

In `crates/roko-cli/src/main.rs`, find the function at line 2836:
```rust
fn dispatch_mcp_cmd(cmd: &ConfigMcpCmd, workdir: &Path) -> Result<()> {
```

Change its visibility from `fn` to `pub(crate) fn` so it can be called from `config_cmd.rs`:
```rust
pub(crate) fn dispatch_mcp_cmd(cmd: &ConfigMcpCmd, workdir: &Path) -> Result<()> {
```

### Step 2: Replace unreachable with delegation in config_cmd.rs

In `crates/roko-cli/src/commands/config_cmd.rs`, replace lines 207-210:
```rust
// ── MCP (intercepted in dispatch_subcommand) ─────────────────
ConfigCmd::Mcp { .. } => {
    unreachable!("mcp dispatched in dispatch_subcommand")
}
```

With:
```rust
// ── MCP ──────────────────────────────────────────────────────
ConfigCmd::Mcp { cmd } => {
    let wd = resolve_workdir(cli);
    crate::dispatch_mcp_cmd(&cmd, &wd)
}
```

Ensure `resolve_workdir` is already imported (it is used extensively in the same function).

### Step 3: Add `ConfigMcpCmd` import if needed

At the top of `config_cmd.rs`, verify that `ConfigMcpCmd` is in scope. If the `ConfigCmd` enum
is defined in `main.rs`, the `cmd` field type should already be accessible through the enum.
If not, add:
```rust
use crate::ConfigMcpCmd;
```
