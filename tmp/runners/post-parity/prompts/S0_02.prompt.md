# S0_02: Verify share route is behind auth middleware

## Task
Audit the `POST /api/runs/{id}/share` route placement and confirm it is behind the auth middleware, or move it if not.

## Runner Context
Post-parity Phase 0 (Stability), batch 2 of 25. No dependencies. Group S0A (Panics & Security).

## Problem
The share endpoint creates publicly accessible links to run transcripts. If exposed without
authentication, any network neighbor could create share links for any run.

The current code already has the correct structure:
- `shared_runs::auth_routes()` at `shared_runs.rs:854` returns `POST /runs/{id}/share`
- `shared_runs::public_routes()` at `shared_runs.rs:864` returns only GET routes
- `routes/mod.rs:117` merges `auth_routes()` into the `api` router
- `routes/mod.rs:124-132` applies auth middleware to the `api` router
- `routes/mod.rs:170` merges `public_routes()` outside the auth-gated nest

However, there is no test asserting this invariant. A future refactor could accidentally move
the route. This batch adds a test to lock the behavior.

## Exact Changes

### Step 1: Add integration test asserting share requires auth

In `crates/roko-serve/src/routes/shared_runs.rs`, find the `#[cfg(test)] mod tests` block
(around line 872). Add a test that verifies unauthenticated POST to `/api/runs/{id}/share`
returns 401:

```rust
#[tokio::test]
async fn share_route_requires_auth() {
    // Build a minimal app with auth enabled
    let state = test_state_with_auth();
    let app = crate::routes::build_router(state);

    let response = app
        .oneshot(
            axum::http::Request::builder()
                .method("POST")
                .uri("/api/runs/test-id/share")
                .header("content-type", "application/json")
                .body(axum::body::Body::from("{}"))
                .unwrap(),
        )
        .await
        .unwrap();

    assert_eq!(
        response.status(),
        axum::http::StatusCode::UNAUTHORIZED,
        "POST /api/runs/{{id}}/share must require authentication"
    );
}
```

If `test_state_with_auth()` does not exist, create a helper that builds an `AppState` with
`auth.enabled = true` and a known API key. Look at existing test helpers in the same file
or in `crates/roko-serve/src/routes/mod.rs` for patterns.

### Step 2: Add doc comment to `auth_routes()` function

At `shared_runs.rs:850-856`, add a doc comment reinforcing the security invariant:

```rust
/// Routes that **require** authentication. Wire these inside the auth layer.
///
/// **Security invariant**: The `POST /runs/{id}/share` endpoint MUST be behind

///
/// Endpoints:
/// * `POST /runs/{id}/share` -- create a shareable token for a completed run.
pub fn auth_routes() -> axum::Router<Arc<AppState>> {
```
