# S0_03: Verify secret scrubbing in CLI Gist share path

## Task
Audit the CLI share path in `crates/roko-cli/src/share.rs` and confirm secret scrubbing is applied before Gist upload, adding any missing coverage.

## Runner Context
Post-parity Phase 0 (Stability), batch 3 of 25. No dependencies. Group S0A (Panics & Security).

## Problem
The CLI `share_run()` function uploads run transcripts to GitHub Gist. If secrets (API keys,
tokens) leak into the uploaded markdown, they become publicly visible.

The current code at `share.rs:96-105` already scrubs:
```rust
let scrubbed_prompt = scrub_share_text(prompt);
let scrubbed_report = RunReport {
    output_text: report.output_text.as_deref().map(scrub_share_text),
    ..report.clone()
};
```

However, `RunReport` has additional text fields that may contain secrets:
- `tool_outputs` (tool call results may contain API responses)
- `error_text` (error messages may contain URLs with tokens)
- `system_prompt` (may contain injected API keys)

These fields flow into `render_markdown_transcript()` and may appear in the Gist.

## Exact Changes

### Step 1: Audit RunReport fields for unscrubbed text

In `crates/roko-cli/src/share.rs`, find the `share_run()` function. Read the `RunReport` struct
definition (likely in `crates/roko-cli/src/run.rs` or similar) to identify all String/text fields.

For each text field in `RunReport` that could contain secrets, ensure it is scrubbed before
being passed to `render_markdown_transcript()`. Add scrubbing for any missing fields:

```rust
let scrubbed_report = RunReport {
    output_text: report.output_text.as_deref().map(scrub_share_text),
    error_text: report.error_text.as_deref().map(scrub_share_text),
    tool_outputs: report.tool_outputs.iter().map(|t| ToolOutput {
        output: scrub_share_text(&t.output),
        ..t.clone()
    }).collect(),
    system_prompt: report.system_prompt.as_deref().map(scrub_share_text),
    ..report.clone()
};
```

Adjust field names to match the actual `RunReport` struct. If a field does not exist, skip it.

### Step 2: Add test for scrubbing coverage

Add a test in the `#[cfg(test)]` module of `share.rs` that verifies secret patterns are
removed from all text fields:

```rust
#[test]
fn scrub_share_text_removes_api_keys() {
    let input = "key is sk-ant-api03-AAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAA";
    let scrubbed = scrub_share_text(input);
    assert!(!scrubbed.contains("sk-ant-api03"), "API key should be scrubbed: {scrubbed}");
}

#[test]
fn scrub_share_text_removes_bearer_tokens() {
    let input = "Authorization: Bearer ghp_1234567890abcdef1234567890abcdef12345678";
    let scrubbed = scrub_share_text(input);
    assert!(!scrubbed.contains("ghp_"), "Bearer token should be scrubbed: {scrubbed}");
}
```

### Step 3: Add doc comment documenting the scrubbing invariant

At the top of `share_run()`, add:
```rust
/// Share a run report. Tries GitHub Gist first, falls back to local file.
///
/// **Security invariant**: All text embedded in the transcript (prompt, output,
/// tool results, errors) MUST be scrubbed for secrets before being written to
/// disk or uploaded. See `scrub_share_text()`.
```
