# S0_04: Fix stub gate verdicts giving false PASS

## Task
Change `stub_verdict()` in `crates/roko-gate/src/rung_dispatch.rs` to return `Verdict::skip()` instead of `Verdict::pass()`, and add the `Verdict::skip()` constructor to `crates/roko-core/src/verdict.rs`.

## Runner Context
Post-parity Phase 0 (Stability), batch 4 of 25. No dependencies. Group S0A (Panics & Security).

## Problem
When a rung 3-6 gate lacks required inputs (e.g. no `SymbolManifest` for symbol gate, no
fact-check oracle configured), `stub_verdict()` at `rung_dispatch.rs:132-137` returns a
**passing** verdict:

```rust
fn stub_verdict(gate: &str, detail: impl Into<String>) -> Verdict {
    let message = format!("stub gate; {}", detail.into());
    let mut verdict = Verdict::pass(gate.to_string());
    verdict.reason.clone_from(&message);
    verdict.detail = Some(message);
    verdict
}
```

This means unconfigured gates silently report PASS, inflating gate pass rates and hiding
the fact that verification was skipped. Downstream policies (adaptive thresholds, replan
triggers, C-factor metrics) all treat these as real passes.

The fix is to introduce `Verdict::skip()` -- a verdict that is neither pass nor fail but
explicitly marks the gate as not-run. The `passed` field stays `false` so it does not
inflate pass counts, but the `score` is set to a sentinel value (e.g., `f32::NAN` or a
dedicated `skipped` field) so policies can distinguish skip from fail.

## Exact Changes

### Step 1: Add `Verdict::skip()` constructor in verdict.rs

In `crates/roko-core/src/verdict.rs`, after the `fail()` constructor (line ~101), add:

```rust
/// A skip verdict — the gate was not run (missing inputs, not configured).
///
/// Skipped verdicts have `passed = false` and `score = -1.0` so they are
/// distinguishable from real failures (score 0.0) and real passes (score 1.0).
/// Policies should filter these out before computing pass rates.
#[must_use]
pub fn skip(gate: impl Into<String>, reason: impl Into<String>) -> Self {
    Self {
        passed: false,
        reason: reason.into(),
        gate: gate.into(),
        score: -1.0,
        detail: None,
        test_count: None,
        error_digest: None,
        duration_ms: 0,
    }
}

/// Returns `true` if this verdict represents a skipped gate (not actually run).
#[must_use]
pub fn is_skipped(&self) -> bool {
    self.score < 0.0
}
```

### Step 2: Update `stub_verdict()` in rung_dispatch.rs

In `crates/roko-gate/src/rung_dispatch.rs`, replace lines 132-137:

```rust
fn stub_verdict(gate: &str, detail: impl Into<String>) -> Verdict {
    let message = format!("stub gate; {}", detail.into());
    let mut verdict = Verdict::pass(gate.to_string());
    verdict.reason.clone_from(&message);
    verdict.detail = Some(message);
    verdict
}
```

With:

```rust
fn stub_verdict(gate: &str, detail: impl Into<String>) -> Verdict {
    let message = format!("stub gate; {}", detail.into());
    Verdict::skip(gate.to_string(), &message).with_detail(message)
}
```

### Step 3: Add tests for Verdict::skip

In the `#[cfg(test)] mod tests` block of `crates/roko-core/src/verdict.rs`, add:

```rust
#[test]
fn skip_verdict_is_not_passed() {
    let v = Verdict::skip("symbol", "no manifest wired");
    assert!(!v.passed);
    assert!(v.is_skipped());
    assert!(v.score < 0.0);
}

#[test]
fn skip_verdict_is_distinguishable_from_fail() {
    let skip = Verdict::skip("symbol", "not configured");
    let fail = Verdict::fail("symbol", "check failed");
    assert!(skip.is_skipped());
    assert!(!fail.is_skipped());
    assert_ne!(skip.score, fail.score);
}

#[test]
fn pass_verdict_is_not_skipped() {
    let v = Verdict::pass("compile");
    assert!(!v.is_skipped());
}
```
