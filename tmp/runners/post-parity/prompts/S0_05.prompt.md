# S0_05: Normalize gate config format

## Task
Ensure `GatesConfig` in `crates/roko-core/src/config/gates.rs` parses correctly from the `[gates]` table format used in `roko.toml`, and document the canonical format.

## Runner Context
Post-parity Phase 0 (Stability), batch 5 of 25. No dependencies. Group S0B (Config Consistency).

## Problem
There is confusion between two gate config formats in the codebase:

1. **`[gates]` table** (canonical): Used in `roko.toml` and parsed as `GatesConfig` struct
   ```toml
   [gates]
   clippy_enabled = true
   skip_tests = false
   max_iterations = 3
   ```

2. **`[[gate]]` array** (init output): Written by `append_shell_gate()` in `init.rs:129-148`
   ```toml
   [[gate]]
   kind = "shell"
   program = "cargo"
   args = ["check"]
   timeout_ms = 600000
   ```

The `[[gate]]` array entries are NOT parsed by `GatesConfig` -- they are a different concept
(per-task verification commands vs global gate settings). But the naming overlap causes
confusion. `roko init` emits `[[gate]]` entries but the schema expects `[gates]`.

The `GatesConfig` struct at `gates.rs:13-27` is correct and parses `[gates]` properly.
The issue is that `roko init` at `init.rs:102-148` emits `[[gate]]` entries (shell commands)
alongside the `[gates]` table, and there is no struct to parse `[[gate]]` entries.

## Exact Changes

### Step 1: Add ShellGateConfig struct for `[[gate]]` entries

In `crates/roko-core/src/config/gates.rs`, after the `GatesConfig` struct and its `Default`
impl (line ~42), add a struct for `[[gate]]` array entries:

```rust
/// A single shell-based verification gate command.
///
/// Written as `[[gate]]` entries in `roko.toml` by `roko init`.
/// These are per-task verification commands, distinct from the global
/// `[gates]` settings in `GatesConfig`.
#[derive(Clone, Debug, PartialEq, Eq, Serialize, Deserialize)]
pub struct ShellGateEntry {
    /// Gate kind (currently only "shell" is supported).
    #[serde(default = "default_gate_kind")]
    pub kind: String,
    /// Program to execute.
    pub program: String,
    /// Arguments to pass to the program.
    #[serde(default)]
    pub args: Vec<String>,
    /// Timeout in milliseconds.
    #[serde(default = "default_gate_timeout")]
    pub timeout_ms: u64,
}

fn default_gate_kind() -> String { "shell".to_string() }
const fn default_gate_timeout() -> u64 { 600_000 }
```

### Step 2: Add `gate` field to RokoConfig

In `crates/roko-core/src/config/schema.rs`, add the `gate` array field to `RokoConfig`:

```rust
/// Per-task shell verification commands (`[[gate]]` entries).
#[serde(default, skip_serializing_if = "Vec::is_empty")]
pub gate: Vec<ShellGateEntry>,
```

Add the import:
```rust
use super::gates::ShellGateEntry;
```

### Step 3: Re-export ShellGateEntry

In `crates/roko-core/src/config/mod.rs`, add `ShellGateEntry` to the public exports if
not already there.

### Step 4: Add parsing test

In the test module of `gates.rs` or `schema.rs`, add:

```rust
#[test]
fn gate_array_and_gates_table_coexist() {
    let toml = r#"
[gates]
clippy_enabled = false
skip_tests = true

[[gate]]
kind = "shell"
program = "cargo"
args = ["check"]
timeout_ms = 600000

[[gate]]
kind = "shell"
program = "cargo"
args = ["test"]
timeout_ms = 600000
"#;
    let cfg = RokoConfig::from_toml(toml).expect("parse");
    assert!(!cfg.gates.clippy_enabled);
    assert!(cfg.gates.skip_tests);
    assert_eq!(cfg.gate.len(), 2);
    assert_eq!(cfg.gate[0].program, "cargo");
    assert_eq!(cfg.gate[0].args, vec!["check"]);
}
```
