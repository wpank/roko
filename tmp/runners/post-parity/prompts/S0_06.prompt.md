# S0_06: Fix roko init emitting wrong gate format

## Task
Update `append_verification_gates()` in `crates/roko-cli/src/commands/init.rs` to emit both `[gates]` table settings and `[[gate]]` shell entries in the generated `roko.toml`.

## Runner Context
Post-parity Phase 0 (Stability), batch 6 of 25. Depends on S0_05. Group S0B (Config Consistency).

## Problem
`roko init` currently emits only `[[gate]]` array entries (shell commands) via
`append_shell_gate()` at `init.rs:129-148`, but does not emit a `[gates]` table with
global settings like `clippy_enabled`, `skip_tests`, `max_iterations`.

After S0_05, the config schema properly parses both. But `roko init` should emit a
complete config that includes the `[gates]` table so users see all available settings.

Current output from `roko init --profile rust`:
```toml
# -- Verification gates --
# Rust projects use cargo for compile, test, and lint checks.

[[gate]]
kind = "shell"
program = "cargo"
args = ["check"]
timeout_ms = 600000
```

Expected output:
```toml
# -- Verification gates --
[gates]
clippy_enabled = true
skip_tests = false
max_iterations = 3

# Rust projects use cargo for compile, test, and lint checks.

[[gate]]
kind = "shell"
program = "cargo"
args = ["check"]
timeout_ms = 600000
```

## Exact Changes

### Step 1: Add `[gates]` table before shell entries

In `crates/roko-cli/src/commands/init.rs`, modify `append_verification_gates()` at line 102.
Before the profile-specific `[[gate]]` entries, emit the `[gates]` table:

```rust
fn append_verification_gates(out: &mut String, profile: Option<&str>) {
    out.push_str("\n# -- Verification gates --\n");

    // Global gate settings
    out.push_str("[gates]\n");
    match profile {
        Some("rust") => {
            out.push_str("clippy_enabled = true\n");
            out.push_str("skip_tests = false\n");
        }
        Some("typescript") => {
            out.push_str("clippy_enabled = false\n");
            out.push_str("skip_tests = false\n");
        }
        _ => {
            out.push_str("clippy_enabled = false\n");
            out.push_str("skip_tests = false\n");
        }
    }
    out.push_str("max_iterations = 3\n");

    // Per-task shell gate commands
    match profile {
        Some("rust") => {
            out.push_str("\n# Rust projects use cargo for compile, test, and lint checks.\n");
            append_shell_gate(out, "cargo", &["check"], 600_000);
            append_shell_gate(out, "cargo", &["test"], 600_000);
            append_shell_gate(out, "cargo", &["clippy"], 600_000);
        }
        Some("typescript") => {
            out.push_str("\n# TypeScript projects use npx tsc and npm test.\n");
            append_shell_gate(out, "npx", &["tsc", "--noEmit"], 600_000);
            append_shell_gate(out, "npm", &["test"], 600_000);
        }
        _ => {
            out.push_str(
                "\n# No default gates were written because no supported project profile was supplied.\n",
            );
            out.push_str("# Supported profiles: rust, typescript.\n");
            out.push_str("# Add [[gate]] entries manually to run your own validation commands.\n");
            out.push_str(
                "# Or rerun `roko init --profile rust` / `roko init --profile typescript`.\n",
            );
        }
    }
}
```
