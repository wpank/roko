# S0_07: Wire default_model from config into all dispatch

## Task
Ensure `config.agent.default_model` is respected as the primary fallback in `resolve_effective_model()` in `crates/roko-cli/src/model_selection.rs`.

## Runner Context
Post-parity Phase 0 (Stability), batch 7 of 25. No dependencies. Group S0B (Config Consistency).

## Problem
The model selection cascade in `resolve_effective_model()` at `model_selection.rs:141` has
the correct priority order:
1. CLI override (`--model`)
2. Task hint (from plan task definition)
3. Role config (`agent.roles.{role}.model`)
4. CascadeRouter recommendation
5. Project default (`config.agent.default_model`)
6. Built-in default (`RokoConfig::default().agent.default_model`)

At line 252-258, the project default IS consulted:
```rust
let default_model = config.agent.default_model.trim();
if !default_model.is_empty() {
    return Ok(ModelCandidate {
        source: SelectionSource::ProjectDefault,
        model: default_model.to_string(),
    });
}
```

However, the CascadeRouter step (priority 4) is only consulted if `cascade_router: Option<&CascadeRouter>`
is passed as `Some`. In practice, most callers pass `None` because loading the router from
disk is not wired into the call sites.

This batch ensures all callers of `resolve_effective_model()` pass the loaded CascadeRouter
when available, and adds a log warning when the router is not available.

## Exact Changes

### Step 1: Add tracing warning when CascadeRouter is None

In `crates/roko-cli/src/model_selection.rs`, in the `resolve_effective_model()` function,
find the cascade router check. Add a `tracing::debug!` when the router is None:

```rust
// Step 4: CascadeRouter recommendation
if let Some(router) = cascade_router {
    if let Some(recommendation) = router.recommend(/* ... */) {
        return Ok(ModelCandidate {
            source: SelectionSource::CascadeRouter,
            model: recommendation,
        });
    }
} else {
    tracing::debug!("cascade router not loaded; skipping adaptive model selection");
}
```

### Step 2: Add helper to load CascadeRouter from disk

In `crates/roko-cli/src/model_selection.rs`, add a helper function:

```rust
/// Load the CascadeRouter from the standard disk location.
///
/// Returns `None` if the file does not exist or cannot be parsed (not an error).
pub fn load_cascade_router(workdir: &std::path::Path) -> Option<CascadeRouter> {
    let path = workdir.join(".roko/learn/cascade-router.json");
    match std::fs::read_to_string(&path) {
        Ok(json) => match serde_json::from_str(&json) {
            Ok(router) => Some(router),
            Err(e) => {
                tracing::warn!("failed to parse cascade router at {}: {e}", path.display());
                None
            }
        },
        Err(_) => None,
    }
}
```

### Step 3: Add test for config default priority

In the test module, add:

```rust
#[test]
fn config_default_model_takes_priority_over_builtin() {
    let mut config = RokoConfig::default();
    config.agent.default_model = "my-custom-model".to_string();

    let selection =
        resolve_effective_model(None, None, None, None, &config).expect("selection");

    assert_eq!(selection.effective_model_key, "my-custom-model");
    assert_eq!(selection.source, SelectionSource::ProjectDefault);
}
```
