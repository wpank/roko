# S0_08: Remove unsafe set_var for --provider override

## Task
Replace the `unsafe { std::env::set_var("ROKO_PROVIDER", p) }` call in `crates/roko-cli/src/commands/util.rs` with safe config threading.

## Runner Context
Post-parity Phase 0 (Stability), batch 8 of 25. No dependencies. Group S0B (Config Consistency).

## Problem
In `crates/roko-cli/src/commands/util.rs:233-237`, the `--provider` CLI flag is propagated
by mutating the process environment:

```rust
#[allow(unsafe_code)]
if let Some(ref p) = provider {
    // SAFETY: single-threaded at this point (before tokio runtime starts).
    unsafe { std::env::set_var("ROKO_PROVIDER", p) };
}
```

This is unsafe and fragile. If the call site moves after the tokio runtime starts (which
it might -- there are already async call paths), this becomes undefined behavior in Rust
2024 edition. The provider override should be threaded through the config struct instead.

The config resolution at `resolve_config_for_workdir()` already merges env vars via
`apply_env()`. We can replace the `set_var` with a direct config mutation after loading.

## Exact Changes

### Step 1: Remove the unsafe set_var block

In `crates/roko-cli/src/commands/util.rs`, find the function containing the `set_var` call
(around line 226). Remove lines 231-237:

```rust
// DELETE THESE LINES:
// Expose --provider as the ROKO_PROVIDER env var so that apply_env()
// picks it up during config resolution (same mechanism as the env var).
#[allow(unsafe_code)]
if let Some(ref p) = provider {
    // SAFETY: single-threaded at this point (before tokio runtime starts).
    unsafe { std::env::set_var("ROKO_PROVIDER", p) };
}
```

### Step 2: Apply provider override after config load

After the `resolve_config_for_workdir()` call (which is a few lines later), add the
provider override directly to the config:

```rust
let workdir = workdir.unwrap_or_else(|| resolve_workdir(cli));
prepare_runtime_hooks(&workdir, cli.quiet);
let mut config = resolve_config_for_workdir(cli, &workdir)?;

// Apply --provider override directly to config (replaces unsafe set_var).
if let Some(ref p) = provider {
    config.agent.default_backend = p.clone();
}

apply_resume_session_override(&mut config, cli.resume.clone());
```

### Step 3: Verify no other set_var calls exist for ROKO_PROVIDER

Search for other `set_var.*ROKO_PROVIDER` calls in the codebase. If found, apply the same
pattern: remove the set_var and thread through config.

### Step 4: Remove the #[allow(unsafe_code)] if no other unsafe blocks exist in the file

If the file has no remaining `unsafe` blocks after the removal, remove any file-level
`#[allow(unsafe_code)]` attribute.
