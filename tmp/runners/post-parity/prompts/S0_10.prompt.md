# S0_10: Wire feedback recording to roko chat

## Task
Add feedback event emission at the end of `roko chat` sessions so learning signals are recorded.

## Runner Context
Post-parity Phase 0 (Stability), batch 10 of 25. No dependencies. Group S0C (Learning Wiring).

## Problem
The `roko chat` command (`chat_session.rs` and `chat_inline.rs`) does not emit any
`FeedbackEvent` or `CompletedRunInput` when a session ends. This means the learning
runtime never sees chat interactions and cannot improve model routing or prompt quality
based on chat usage.

The `roko run` path (in `run.rs`) already emits `CompletedRunInput` via `LearningRuntime`.
The chat path should do the same on session exit.

## Exact Changes

### Step 1: Collect session metrics during chat

In `crates/roko-cli/src/chat_inline.rs`, find the `ChatSession` struct (around line 742).
Add fields to track session metrics:

```rust
/// Total tokens used across all turns (accumulated from DispatchResult).
total_input_tokens: u64,
total_output_tokens: u64,
/// Total cost in USD (accumulated from DispatchResult).
total_cost_usd: f64,
```

Initialize these to 0 in the constructor(s) (around lines 1100-1140 and 1510-1530).

### Step 2: Accumulate metrics from dispatch results

In the response handling code (around line 1280 where `Ok(Ok(result))` is matched),
accumulate the metrics:

```rust
Ok(Ok(result)) => {
    // ... existing handling ...
    session.total_input_tokens += result.input_tokens.unwrap_or(0);
    session.total_output_tokens += result.output_tokens.unwrap_or(0);
    session.total_cost_usd += result.cost_usd.unwrap_or(0.0);
    // ...
}
```

Adjust field names to match what `DispatchResult` actually contains.

### Step 3: Emit feedback event on session exit

Find the session exit point (where `Phase::Done` is handled or after the main loop ends).
Before returning, emit a learning event:

```rust
// Emit feedback for learning runtime
if session.turn_count > 0 {
    let feedback_path = workdir.join(".roko/learn/chat-feedback.jsonl");
    let event = serde_json::json!({
        "kind": "chat_session_complete",
        "model": session.agent_id,
        "turns": session.turn_count,
        "input_tokens": session.total_input_tokens,
        "output_tokens": session.total_output_tokens,
        "cost_usd": session.total_cost_usd,
        "elapsed_s": session.started_at.elapsed().as_secs(),
        "timestamp": chrono::Utc::now().to_rfc3339(),
    });
    if let Ok(mut f) = std::fs::OpenOptions::new()
        .create(true).append(true).open(&feedback_path)
    {
        use std::io::Write;
        let _ = writeln!(f, "{}", event);
    }
}
```

### Step 4: Add the same to the direct REPL path

If there is a separate chat REPL function (the non-inline path in `chat_session.rs`),
add the same feedback emission there.

## Acceptance Criteria
- Chat sessions emit a JSONL feedback event to `.roko/learn/chat-feedback.jsonl`
- Event includes: model, turn count, token usage, cost, elapsed time
- Event only emitted if at least one turn was completed
- Feedback file is append-only (no truncation)
- No crash if the file cannot be written (graceful failure)

## Do NOT
- Change the chat UX or interaction flow
- Add feedback to the streaming event channel
- Modify the learning runtime itself
- Block session exit on feedback write failure
- Touch any files outside write scope
