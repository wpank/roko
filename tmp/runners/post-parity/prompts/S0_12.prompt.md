# S0_12: Wire build_repo_context() into plan generate

## Task
Call `build_repo_context()` before agent dispatch in `crates/roko-cli/src/commands/plan.rs` for plan generation, injecting the result into the system prompt.

## Runner Context
Post-parity Phase 0 (Stability), batch 12 of 25. No dependencies. Group S0C (Learning Wiring).

## Problem
The `roko prd plan <slug>` command (in `commands/prd.rs:877`) already calls
`build_repo_context()` and injects it into the system prompt. But `roko plan generate`
(in `commands/plan.rs`) does not -- it generates plans without knowing the repo structure.

This means `plan generate` produces plans that may reference wrong file paths, miss
existing implementations, or duplicate code. The `build_repo_context()` function at
`repo_context.rs:282` returns a `RepoContextPack` with:
- Crate/package layout
- File tree structure
- Feature keywords from the repo
- "Do not create" file list
- Context root verification

## Exact Changes

### Step 1: Find the plan generate entry point

In `crates/roko-cli/src/commands/plan.rs`, find the handler for `plan generate` (or
`plan regenerate`). Look for where the agent is dispatched to generate a plan from
a prompt or PRD.

### Step 2: Call build_repo_context before dispatch

Before the agent dispatch call, add repo context loading:

```rust
// Load repo context for plan generation
let feature_keywords: Vec<String> = extract_keywords_from_prompt(&prompt_text);
let keyword_refs: Vec<&str> = feature_keywords.iter().map(String::as_str).collect();
let repo_context = match crate::repo_context::build_repo_context(&workdir, &keyword_refs).await {
    Ok(pack) => {
        if !pack.context_root_verified {
            eprintln!("warning: repo context root not verified; context may be incomplete");
        }
        Some(pack)
    }
    Err(e) => {
        tracing::warn!("failed to build repo context: {e}");
        None
    }
};
```

If `extract_keywords_from_prompt` does not exist, create a simple version:
```rust
fn extract_keywords_from_prompt(prompt: &str) -> Vec<String> {
    prompt
        .split_whitespace()
        .filter(|w| w.len() > 3)
        .take(10)
        .map(|w| w.trim_matches(|c: char| !c.is_alphanumeric()).to_lowercase())
        .collect()
}
```

### Step 3: Inject repo context into system prompt

Append the repo context to the system prompt or agent instructions:

```rust
let mut system_prompt = base_system_prompt.clone();
if let Some(ref pack) = repo_context {
    system_prompt.push_str("\n\n## Repository Context\n\n");
    system_prompt.push_str(&pack.render_for_prompt());
}
```

If `render_for_prompt()` does not exist on `RepoContextPack`, use the `Display` impl
or call `pack.to_string()`. Check `repo_context.rs` for available methods.
