# S0_14: Replace PERPLEXITY_API_KEY env read in web_search

## Task
Replace the direct `std::env::var("PERPLEXITY_API_KEY")` read in `crates/roko-std/src/tool/builtin/web_search.rs` with a config-sourced key from `ToolContext`.

## Runner Context
Post-parity Phase 0 (Stability), batch 14 of 25. No dependencies. Group S0D (Env Var Elimination).

## Problem
In `crates/roko-std/src/tool/builtin/web_search.rs:332-339`, the web search tool reads
the Perplexity API key directly from the environment:

```rust
let api_key = match std::env::var("PERPLEXITY_API_KEY") {
    Ok(k) if !k.is_empty() => k,
    _ => {
        return ToolResult::Err(ToolError::Other(
            "web_search: PERPLEXITY_API_KEY environment variable is not set."
                .into(),
        ));
    }
};
```

This bypasses the config system. The key should come from the `ToolContext` or from
a config field, falling back to the env var for backward compatibility.

## Exact Changes

### Step 1: Check if ToolContext carries config

Read the `ToolContext` struct definition (likely in `crates/roko-core/src/tool/mod.rs`
or similar). Check if it has a `config` field or a way to pass provider-specific keys.

### Step 2: Add api_key lookup via ToolContext

If `ToolContext` has a `config` or `secrets` field:
```rust
let api_key = ctx.config()
    .and_then(|c| c.providers.get("perplexity"))
    .and_then(|p| p.resolve_api_key())
    .or_else(|| std::env::var("PERPLEXITY_API_KEY").ok().filter(|k| !k.is_empty()));

let Some(api_key) = api_key else {
    return ToolResult::Err(ToolError::Other(
        "web_search: no Perplexity API key found. \
         Set providers.perplexity.api_key in roko.toml or \
         PERPLEXITY_API_KEY environment variable.".into(),
    ));
};
```

If `ToolContext` does NOT carry config, add a `search_api_key` field:

In the `ToolContext` struct, add:
```rust
/// API key for search providers (Perplexity). Populated from config at construction.
pub search_api_key: Option<String>,
```

Then in web_search.rs:
```rust
let api_key = ctx.search_api_key.clone()
    .or_else(|| std::env::var("PERPLEXITY_API_KEY").ok().filter(|k| !k.is_empty()));

let Some(api_key) = api_key else {
    return ToolResult::Err(ToolError::Other(
        "web_search: no Perplexity API key found. \
         Set it via config or PERPLEXITY_API_KEY env var.".into(),
    ));
};
```

### Step 3: Keep env var fallback for backward compatibility

The env var should remain as a fallback, but the primary source should be config.
Update the error message to mention both sources.

### Step 4: Update the test

Update the existing test at line 400-420 to reflect the new error message:

```rust
ToolResult::Err(ToolError::Other(msg)) => {
    assert!(
        msg.contains("API key") || msg.contains("PERPLEXITY_API_KEY"),
        "error should mention API key, got: {msg}"
    );
}
```
