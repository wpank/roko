# S0_15: Replace PERPLEXITY_API_KEY reads in research paths

## Task
Add a `resolve_provider_api_key()` helper and use it in `crates/roko-cli/src/commands/research.rs` to replace direct `std::env::var("PERPLEXITY_API_KEY")` calls.

## Runner Context
Post-parity Phase 0 (Stability), batch 15 of 25. No dependencies. Group S0D (Env Var Elimination).

## Problem
In `crates/roko-cli/src/commands/research.rs:723-724`, the research search command reads
the API key directly:

```rust
let api_key =
    std::env::var("PERPLEXITY_API_KEY").context("PERPLEXITY_API_KEY not set")?;
```

And at line 867, a fallback provider config hardcodes the env var name:
```rust
api_key_env: Some("PERPLEXITY_API_KEY".to_string()),
```

The research code should resolve the API key through the config system first, falling
back to the env var.

## Exact Changes

### Step 1: Add resolve_provider_api_key helper

In `crates/roko-cli/src/commands/research.rs` (or a shared helper module), add:

```rust
/// Resolve an API key for a provider, checking config first, then env var.
///
/// Priority:
/// 1. `config.providers[provider_key].api_key` (if set directly in config)
/// 2. `std::env::var(config.providers[provider_key].api_key_env)` (config-named env var)
/// 3. `std::env::var(default_env_var)` (hardcoded fallback env var name)
fn resolve_provider_api_key(
    config: &RokoConfig,
    provider_key: &str,
    default_env_var: &str,
) -> Option<String> {
    // Check config provider entry
    if let Some(provider) = config.providers.get(provider_key) {
        // Direct API key in config
        if let Some(ref key) = provider.api_key {
            if !key.is_empty() {
                return Some(key.clone());
            }
        }
        // Config-specified env var name
        if let Some(ref env_name) = provider.api_key_env {
            if let Ok(key) = std::env::var(env_name) {
                if !key.is_empty() {
                    return Some(key);
                }
            }
        }
    }

    // Fallback: default env var
    std::env::var(default_env_var)
        .ok()
        .filter(|k| !k.is_empty())
}
```

### Step 2: Replace direct env read at line 723

Replace:
```rust
let api_key =
    std::env::var("PERPLEXITY_API_KEY").context("PERPLEXITY_API_KEY not set")?;
```

With:
```rust
let api_key = resolve_provider_api_key(&config, "perplexity", "PERPLEXITY_API_KEY")
    .context(
        "Perplexity API key not found. Set providers.perplexity.api_key in roko.toml \
         or PERPLEXITY_API_KEY environment variable."
    )?;
```

### Step 3: Use resolve_provider_api_key at other research call sites

Search for other `PERPLEXITY_API_KEY` references in research.rs and replace them with
the helper. The fallback provider config at line 867 can keep its `api_key_env` field
since that is a config struct definition, not a direct read.

### Step 4: Add a test

```rust
#[test]
fn resolve_provider_api_key_prefers_config_over_env() {
    let mut config = RokoConfig::default();
    config.providers.insert("perplexity".to_string(), ProviderConfig {
        api_key: Some("config-key".to_string()),
        ..Default::default()
    });

    let key = resolve_provider_api_key(&config, "perplexity", "PERPLEXITY_API_KEY");
    assert_eq!(key, Some("config-key".to_string()));
}
```
