# S0_17: Route all CLI entry points through ServiceFactory

## Task
Replace direct `create_agent_for_model()` calls with `ServiceFactory::build()` in `run.rs`, `chat_session.rs`, `chat_inline.rs`, and `prd.rs`.

## Runner Context
Post-parity Phase 0 (Stability), batch 17 of 25. Depends on S0_07. Group S0E (ServiceFactory Completeness).

## Problem
`ServiceFactory::build()` in `roko-orchestrator` constructs a fully-configured dispatch
stack with model routing, safety layers, feedback hooks, and tool registries. But several
CLI entry points bypass it by calling `create_agent_for_model()` directly, which skips:

- CascadeRouter consultation
- Safety contract loading
- Feedback event wiring
- Tool registry enrichment
- MCP config passthrough

The `roko run` path at `run.rs:466` already uses `ServiceFactory::build()`:
```rust
let services = ServiceFactory::build(ServiceConfig {
    workdir: workdir.to_path_buf(),
    roko_dir: workdir.join(".roko"),
    workspace_config: model_config,
    // ...
});
```

Other paths should follow this pattern.

## Exact Changes

### Step 1: Find all direct create_agent_for_model calls

```bash
grep -rn 'create_agent_for_model\|ClaudeCliAgent::new\|agent_exec::' crates/roko-cli/src/ --include='*.rs'
```

For each hit in non-test code, determine if it should go through `ServiceFactory`.

### Step 2: Replace in chat_session.rs

Find the agent construction in `chat_session.rs`. Replace the direct agent creation with:

```rust
use roko_orchestrator::{ServiceConfig, ServiceFactory};

let service_config = ServiceConfig {
    workdir: workdir.clone(),
    roko_dir: workdir.join(".roko"),
    workspace_config: config.clone(),
    // Fill other fields from the ServiceConfig struct
    ..Default::default()
};
let services = ServiceFactory::build(service_config)?;
```

Use the `services` struct to get the agent/dispatcher instead of creating one directly.

### Step 3: Replace in chat_inline.rs

Apply the same pattern to the chat inline dispatch path. The `DispatchMode::Direct`
variant should use `ServiceFactory` to build its dispatch stack.

### Step 4: Replace in prd.rs

Find the agent dispatch in `commands/prd.rs` (for `prd draft` and `prd plan`). These
likely call `agent_exec::run_agent_task()` or similar. Route through `ServiceFactory`.

### Step 5: Document the canonical pattern

Add a doc comment to `ServiceFactory::build()` or to a helper in `run.rs`:

```rust

/// Direct agent construction bypasses safety, routing, and feedback.
```
