# S0_18: Wire CascadeRouter into all live paths

## Task
Verify that `CascadeRouter` observations flow from dispatch outcomes back to the router, enabling it to learn from every model call.

## Runner Context
Post-parity Phase 0 (Stability), batch 18 of 25. Depends on S0_17. Group S0E (ServiceFactory Completeness).

## Problem
The `CascadeRouter` in `roko-learn` makes model selection recommendations based on past
observations. But observations (success/failure, latency, cost) are only recorded in the
`orchestrate.rs` path. Calls through `ServiceFactory` in chat, PRD, and research paths
do not feed outcomes back to the router.

After S0_17 routes all paths through `ServiceFactory`, the router must also receive
observations from all those paths.

## Exact Changes

### Step 1: Find where observations are recorded

Search for `cascade_router` observation recording:
```bash
grep -rn 'observe\|record_outcome\|add_observation' crates/roko-learn/src/cascade_router.rs
```

Identify the method that records an outcome (likely `observe()` or `record()`).

### Step 2: Find where outcomes are available

After a model call completes, the caller has:
- Model used
- Success/failure
- Latency (ms)
- Token counts
- Cost estimate

These are available in `DispatchResult`, `TurnResult`, or equivalent return types.

### Step 3: Wire observation recording in ServiceFactory dispatch

If `ServiceFactory::build()` returns a service that wraps dispatch, add observation
recording in the dispatch wrapper. Otherwise, add it at each call site after dispatch
completes:

```rust
// After dispatch completes:
if let Some(router) = cascade_router.as_mut() {
    let observation = roko_learn::cascade_router::Observation {
        model: selection.effective_model_key.clone(),
        success: result.is_ok(),
        latency_ms: elapsed.as_millis() as u64,
        input_tokens: result.as_ref().ok().and_then(|r| r.input_tokens),
        output_tokens: result.as_ref().ok().and_then(|r| r.output_tokens),
    };
    router.observe(observation);
    // Persist to disk
    let router_path = workdir.join(".roko/learn/cascade-router.json");
    if let Ok(json) = serde_json::to_string_pretty(router) {
        let _ = std::fs::write(&router_path, json);
    }
}
```

### Step 4: Add persistence after batch of observations

To avoid writing to disk after every single call, consider buffering:

```rust
/// Persist cascade router state if it has new observations.
fn persist_cascade_router(router: &CascadeRouter, workdir: &Path) {
    let path = workdir.join(".roko/learn/cascade-router.json");
    match serde_json::to_string_pretty(router) {
        Ok(json) => {
            if let Err(e) = std::fs::write(&path, &json) {
                tracing::warn!("failed to persist cascade router: {e}");
            }
        }
        Err(e) => tracing::warn!("failed to serialize cascade router: {e}"),
    }
}
```

- CascadeRouter observations recorded after every model call in chat path
- CascadeRouter observations recorded after every model call in PRD path
- Router state persisted to disk after observations

- Persistence failures logged but do not cause errors
