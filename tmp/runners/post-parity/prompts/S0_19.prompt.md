# S0_19: Add FeedbackService to roko chat path

## Task
Wire `LearningRuntime` feedback recording into the `roko chat` dispatch path through `ServiceFactory`.

## Runner Context
Post-parity Phase 0 (Stability), batch 19 of 25. Depends on S0_17. Group S0E (ServiceFactory Completeness).

## Problem
The `roko run` path emits `CompletedRunInput` via `LearningRuntime` after each run,
enabling the learning system to track model performance, adjust routing, and update
playbooks. The `roko chat` path does not emit these events.

After S0_17, chat uses `ServiceFactory`. This batch wires the `LearningRuntime`
feedback recording into the chat session lifecycle.

## Exact Changes

### Step 1: Find LearningRuntime wiring in run.rs

In `crates/roko-cli/src/run.rs`, find where `LearningRuntime` is created and how
`CompletedRunInput` is emitted after a run completes. This is the reference pattern.

### Step 2: Create LearningRuntime in chat_session.rs

In the chat session constructor or initialization, create a `LearningRuntime` instance:

```rust
let learning_paths = LearningPaths::from_workdir(&workdir);
let learning_runtime = LearningRuntime::new(learning_paths);
```

Store it in the `ChatAgentSession` struct:
```rust
pub struct ChatAgentSession {
    // ... existing fields ...
    pub learning_runtime: Option<LearningRuntime>,
}
```

### Step 3: Emit feedback on turn completion

After each turn completes successfully, emit a feedback event:

```rust
if let Some(ref learning) = self.learning_runtime {
    let input = CompletedRunInput {
        model: self.model_selection.effective_model_key.clone(),
        role: "chat".to_string(),
        success: true,
        input_tokens: result.input_tokens.unwrap_or(0) as u32,
        output_tokens: result.output_tokens.unwrap_or(0) as u32,
        latency_ms: elapsed.as_millis() as u64,
        cost_usd: result.cost_usd.unwrap_or(0.0),
        // ... other fields
    };
    if let Err(e) = learning.record_completion(input) {
        tracing::debug!("failed to record chat feedback: {e}");
    }
}
```

### Step 4: Emit session summary on exit

On session exit (when the user types `/exit` or Ctrl-D), emit a session-level summary:

```rust
if let Some(ref learning) = self.learning_runtime {
    let summary = RunnerFeedbackEvent::SessionComplete {
        model: self.model_selection.effective_model_key.clone(),
        turns: self.turn_count,
        total_cost_usd: self.total_cost_usd,
        elapsed_s: self.started_at.elapsed().as_secs(),
    };
    if let Err(e) = learning.record_event(summary) {
        tracing::debug!("failed to record session summary: {e}");
    }
}
```

Adjust the event types and field names to match the actual `LearningRuntime` API.
