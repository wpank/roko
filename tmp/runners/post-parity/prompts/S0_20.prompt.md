# S0_20: Add config validation on load

## Task
Implement `RokoConfig::validate()` in `crates/roko-core/src/config/schema.rs` that checks for common misconfigurations.

## Runner Context
Post-parity Phase 0 (Stability), batch 20 of 25. No dependencies. Group S0F (Config Validation & Migration).

## Problem
`RokoConfig::from_toml()` deserializes the TOML without semantic validation. Users can
write configs that parse correctly but produce runtime failures:

- `default_model` set to empty string
- `max_iterations` set to 0 (infinite loop)
- `max_plan_usd` set to 0.0 (no budget)
- Provider with no `api_key` and no `api_key_env`
- `serve.auth.enabled = true` with no `api_key`
- `schema_version` higher than `CURRENT_CONFIG_VERSION`

These should be caught at load time with clear error messages.

## Exact Changes

### Step 1: Add validate() method

In `crates/roko-core/src/config/schema.rs`, add a `validate()` method to `RokoConfig`:

```rust
impl RokoConfig {
    /// Validate config for common misconfigurations.
    ///
    /// Returns a list of warnings (non-fatal) and errors (fatal).
    /// Call after `from_toml()` to catch semantic issues early.
    pub fn validate(&self) -> ConfigValidation {
        let mut warnings = Vec::new();
        let mut errors = Vec::new();

        // Schema version check
        if self.schema_version > CURRENT_CONFIG_VERSION {
            warnings.push(format!(
                "schema_version {} is newer than supported {}; some fields may be ignored",
                self.schema_version, CURRENT_CONFIG_VERSION
            ));
        }

        // Agent config
        if self.agent.default_model.trim().is_empty() {
            warnings.push("agent.default_model is empty; built-in default will be used".into());
        }

        // Gate config
        if self.gates.max_iterations == 0 {
            errors.push("gates.max_iterations = 0 would cause infinite retries; set to >= 1".into());
        }

        // Budget config
        if self.budget.max_plan_usd <= 0.0 {
            warnings.push("budget.max_plan_usd <= 0; plans will fail budget checks".into());
        }
        if self.budget.max_turn_usd <= 0.0 {
            warnings.push("budget.max_turn_usd <= 0; turns will fail budget checks".into());
        }

        // Auth config
        if self.serve.auth.enabled && self.serve.auth.api_key.is_none() {
            errors.push("serve.auth.enabled = true but no api_key set".into());
        }

        // Provider config
        for (name, provider) in &self.providers {
            let has_key = provider.api_key.as_ref().is_some_and(|k| !k.is_empty());
            let has_env = provider.api_key_env.as_ref().is_some_and(|e| !e.is_empty());
            let has_cmd = provider.command.as_ref().is_some_and(|c| !c.is_empty());
            if !has_key && !has_env && !has_cmd {
                warnings.push(format!(
                    "providers.{name}: no api_key, api_key_env, or command configured"
                ));
            }
        }

        ConfigValidation { warnings, errors }
    }
}
```

### Step 2: Add ConfigValidation struct

```rust
/// Result of config validation.
#[derive(Debug, Default)]
pub struct ConfigValidation {
    /// Non-fatal issues (user should fix but config is usable).
    pub warnings: Vec<String>,
    /// Fatal issues (config should not be used).
    pub errors: Vec<String>,
}

impl ConfigValidation {
    /// Returns true if there are no fatal errors.
    pub fn is_ok(&self) -> bool {
        self.errors.is_empty()
    }

    /// Format all issues for display.
    pub fn display(&self) -> String {
        let mut out = String::new();
        for w in &self.warnings {
            out.push_str(&format!("  warning: {w}\n"));
        }
        for e in &self.errors {
            out.push_str(&format!("  error: {e}\n"));
        }
        out
    }
}
```

### Step 3: Add tests

```rust
#[test]
fn validate_catches_zero_max_iterations() {
    let mut cfg = RokoConfig::default();
    cfg.gates.max_iterations = 0;
    let result = cfg.validate();
    assert!(!result.is_ok());
    assert!(result.errors.iter().any(|e| e.contains("max_iterations")));
}

#[test]
fn validate_warns_on_empty_default_model() {
    let mut cfg = RokoConfig::default();
    cfg.agent.default_model.clear();
    let result = cfg.validate();
    assert!(result.is_ok()); // warning, not error
    assert!(result.warnings.iter().any(|w| w.contains("default_model")));
}

#[test]
fn validate_default_config_passes() {
    let cfg = RokoConfig::default();
    let result = cfg.validate();
    assert!(result.is_ok(), "default config should pass validation: {}", result.display());
}
```

## Acceptance Criteria
- `RokoConfig::validate()` returns `ConfigValidation` with warnings and errors
- Default config passes validation with no errors
- Zero `max_iterations` is a fatal error
- Empty `default_model` is a warning (not error)
- Auth enabled without key is a fatal error
- At least 3 tests covering different validation cases

## Do NOT
- Call `validate()` automatically in `from_toml()` (caller should decide)
- Add the `ConfigValidation` to serde (it is runtime-only)
- Change how `from_toml()` works
- Touch any files outside write scope
