# S0_21: Normalize model aliases at load time

## Task
Add `normalize_model_slug()` in `crates/roko-core/src/agent.rs` that maps common model aliases to canonical names, and call it during config loading.

## Runner Context
Post-parity Phase 0 (Stability), batch 21 of 25. No dependencies. Group S0F (Config Validation & Migration).

## Problem
Users write model names inconsistently in `roko.toml`:
- `"sonnet"` vs `"claude-sonnet-4-6"` vs `"claude-3-5-sonnet"` vs `"claude-sonnet-4-20250514"`
- `"opus"` vs `"claude-opus-4-6"` vs `"claude-3-opus"`
- `"haiku"` vs `"claude-haiku-4-5"` vs `"claude-3-5-haiku"`
- `"gpt-4o"` vs `"gpt-4o-2024-08-06"`

These aliases should be normalized to canonical model identifiers at config load time
so downstream code can rely on consistent names.

## Exact Changes

### Step 1: Add normalize_model_slug function

In `crates/roko-core/src/agent.rs`, add:

```rust
/// Normalize a model name/alias to its canonical identifier.
///
/// Maps common shorthand names to their full model identifiers.
/// Unknown names are returned as-is (they may be valid model IDs
/// from new providers).
pub fn normalize_model_slug(slug: &str) -> String {
    let lower = slug.trim().to_lowercase();
    match lower.as_str() {
        // Claude Opus
        "opus" | "claude-opus" | "claude-3-opus" => "claude-opus-4-6".to_string(),
        // Claude Sonnet
        "sonnet" | "claude-sonnet" | "claude-3-5-sonnet" | "claude-3-sonnet"
            => "claude-sonnet-4-6".to_string(),
        // Claude Haiku
        "haiku" | "claude-haiku" | "claude-3-5-haiku" | "claude-3-haiku"
            => "claude-haiku-4-5".to_string(),
        // Pass through anything else unchanged
        _ => slug.trim().to_string(),
    }
}
```

### Step 2: Apply normalization in config loading

In `crates/roko-core/src/config/schema.rs`, in the `from_toml()` method (or after it),
normalize the `default_model` field:

```rust
impl RokoConfig {
    pub fn from_toml(toml_str: &str) -> Result<Self> {
        let mut config: Self = toml::from_str(toml_str)?;
        config.normalize();
        Ok(config)
    }

    /// Normalize model names and other config values.
    fn normalize(&mut self) {
        self.agent.default_model = crate::agent::normalize_model_slug(&self.agent.default_model);

        // Normalize role-specific models
        for role_config in self.agent.roles.values_mut() {
            role_config.model = crate::agent::normalize_model_slug(&role_config.model);
        }

        // Normalize routing models
        self.routing.fast_task_model =
            crate::agent::normalize_model_slug(&self.routing.fast_task_model);
        self.routing.standard_task_model =
            crate::agent::normalize_model_slug(&self.routing.standard_task_model);
        self.routing.complex_task_model =
            crate::agent::normalize_model_slug(&self.routing.complex_task_model);
    }
}
```

If `from_toml()` already exists and cannot be modified, add the `normalize()` call
after deserialization.

In `crates/roko-core/src/lib.rs` or `crates/roko-core/src/agent.rs`, ensure

### Step 4: Add tests

```rust
#[test]
fn normalize_opus_aliases() {
    assert_eq!(normalize_model_slug("opus"), "claude-opus-4-6");
    assert_eq!(normalize_model_slug("claude-opus"), "claude-opus-4-6");
    assert_eq!(normalize_model_slug("claude-3-opus"), "claude-opus-4-6");
}

#[test]
fn normalize_sonnet_aliases() {
    assert_eq!(normalize_model_slug("sonnet"), "claude-sonnet-4-6");
    assert_eq!(normalize_model_slug("claude-3-5-sonnet"), "claude-sonnet-4-6");
}

#[test]
fn normalize_unknown_passes_through() {
    assert_eq!(normalize_model_slug("gpt-4o"), "gpt-4o");
    assert_eq!(normalize_model_slug("my-custom-model"), "my-custom-model");
}

#[test]
fn normalize_trims_whitespace() {
    assert_eq!(normalize_model_slug("  opus  "), "claude-opus-4-6");
}
```
