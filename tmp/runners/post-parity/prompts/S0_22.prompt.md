# S0_22: Audit legacy-orchestrate feature flag

## Task
Document all `#[cfg(feature = "legacy-orchestrate")]` gated blocks across the workspace and determine which can be removed.

## Runner Context
Post-parity Phase 0 (Stability), batch 22 of 25. No dependencies. Group S0F (Config Validation & Migration).

## Problem
The `legacy-orchestrate` feature flag gates code paths throughout the CLI crate. Some of
these paths are the only working implementation (the "v2" path is incomplete), while others
are truly legacy and should be removed.

Without a clear audit, developers do not know:
- Which feature flag is active in production builds
- Which gated blocks are dead code
- Which gated blocks are the ONLY working path

## Exact Changes

### Step 1: Find all feature-gated blocks

Search the workspace for all `legacy-orchestrate` references:

```bash
grep -rn 'legacy.orchestrate\|legacy_orchestrate' crates/ --include='*.rs'
grep -rn 'legacy-orchestrate' crates/ --include='*.toml'
```

### Step 2: Create audit document

For each gated block found, record:
- File and line number
- What the block does
- Whether the non-legacy alternative exists and works
- Recommendation: keep, remove, or merge

Create the audit as a doc comment in a central location. Add it to the top of
`crates/roko-cli/src/lib.rs`:

```rust
//! ## Feature Flag Audit: `legacy-orchestrate`
//!
//! The `legacy-orchestrate` feature flag controls which dispatch path is used
//! for agent execution. This audit documents all gated blocks.
//!
//! | File | Lines | Purpose | Non-legacy exists? | Recommendation |
//! |------|-------|---------|--------------------|----------------|
//! | dispatch_direct.rs | 1 | Entire file | Yes (dispatch_v2) | Remove after v2 verified |
//! | chat_inline.rs | NNN | Direct dispatch mode | Yes (Session mode) | Remove after Session mode verified |
//! | ... | ... | ... | ... | ... |
//!
//! **Current default**: `legacy-orchestrate` is ENABLED in `Cargo.toml`.
//! The v2 path is incomplete for: [list gaps].
//!
//! **Migration plan**: Verify v2 paths work for all commands, then remove
//! the feature flag and delete the legacy code.
```

### Step 3: Check Cargo.toml for default features

```bash
grep -A5 '\[features\]' crates/roko-cli/Cargo.toml
```

Record whether `legacy-orchestrate` is in the `default` feature set.

### Step 4: Add TODO comments to each gated block

At each `#[cfg(feature = "legacy-orchestrate")]` block, add a comment referencing
the audit:

```rust
// AUDIT(S0_22): Legacy orchestrate path. See lib.rs for removal plan.
#[cfg(feature = "legacy-orchestrate")]
```

## Write Scope
- `crates/roko-cli/src/lib.rs` (audit doc comment)
- All files containing `legacy-orchestrate` gates (TODO comments only)

## Read-Only Context
- `crates/roko-cli/Cargo.toml` (feature definitions)
- `crates/roko-cli/src/dispatch_direct.rs` (legacy dispatch)
- `crates/roko-cli/src/dispatch_v2.rs` (v2 dispatch)

- TODO comment at each gated block referencing the audit

- No code changes beyond doc comments and TODO markers

- Change the default features
