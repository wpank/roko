# S0_23: Remove/migrate ROKO_ACP_LEGACY env var

## Task
Replace `std::env::var_os("ROKO_ACP_LEGACY")` checks in `crates/roko-acp/src/bridge_events.rs` with a config field.

## Runner Context
Post-parity Phase 0 (Stability), batch 23 of 25. No dependencies. Group S0G (Feature Flag & Safety).

## Problem
In `crates/roko-acp/src/bridge_events.rs`, there are three `ROKO_ACP_LEGACY` env var
checks (lines 1133, 2330, 2373) that switch between legacy and v2 workflow pipelines:

```rust
if std::env::var_os("ROKO_ACP_LEGACY").is_some() {
    let legacy_run = shared_run.clone();
    let result = crate::runner::run_workflow_pipeline(
        &session_id,
        &prompt_text,
        // ...
    );
}
```

This env var is undocumented, hard to discover, and bypasses the config system.
It should be a config field so it can be set in `roko.toml` and appears in
`roko config show`.

## Exact Changes

### Step 1: Add config field

In `crates/roko-core/src/config/schema.rs`, find the ACP or agent config section.
Add a field for the legacy pipeline toggle:

If there is an `AcpConfig` or similar struct:
```rust
/// Use the legacy (v1) workflow pipeline instead of the v2 pipeline.
/// Set to `true` for backward compatibility during migration.
#[serde(default)]
pub use_legacy_pipeline: bool,
```

If no ACP config section exists, add it to `AgentConfig` or create a new section:
```rust
#[derive(Clone, Debug, Default, PartialEq, Serialize, Deserialize)]
pub struct AcpConfig {
    /// Use the legacy (v1) workflow pipeline.
    #[serde(default)]
    pub use_legacy_pipeline: bool,
}
```

And add to `RokoConfig`:
```rust
#[serde(default)]
pub acp: AcpConfig,
```

### Step 2: Replace env var checks with config field

In `crates/roko-acp/src/bridge_events.rs`, replace each occurrence of:
```rust
if std::env::var_os("ROKO_ACP_LEGACY").is_some() {
```

With:
```rust
if config.acp.use_legacy_pipeline {
```

The `config` variable should be accessible from the function's parameters. If the
function does not have access to config, thread it through from the caller.

If threading config is too invasive, use a simpler approach -- read the config field
once at session creation and store it as a bool:

```rust
let use_legacy = config.acp.use_legacy_pipeline
    || std::env::var_os("ROKO_ACP_LEGACY").is_some(); // env var fallback during migration
```

### Step 3: Keep env var as temporary fallback

During migration, keep the env var as a fallback so existing deployments are not broken:

```rust
let use_legacy = config.acp.use_legacy_pipeline
    || std::env::var_os("ROKO_ACP_LEGACY").is_some();

if use_legacy {
    if std::env::var_os("ROKO_ACP_LEGACY").is_some() {
        tracing::warn!(
            "ROKO_ACP_LEGACY env var is deprecated; set acp.use_legacy_pipeline = true in roko.toml"
        );
    }
    // ... legacy path ...
}
```

### Step 4: Document the migration

Add a doc comment explaining the migration path:
```rust
/// # Migration from ROKO_ACP_LEGACY env var
///
/// The `ROKO_ACP_LEGACY` environment variable is deprecated.
/// Use `acp.use_legacy_pipeline = true` in `roko.toml` instead.
/// The env var will be removed in a future release.
```
