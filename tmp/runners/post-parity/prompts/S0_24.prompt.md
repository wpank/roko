# S0_24: Remove dangerously_skip_permissions default

## Task
Change `dangerously_skip_permissions` default to `false` in `crates/roko-cli/src/commands/plan.rs` and add a `--skip-permissions` CLI flag.

## Runner Context
Post-parity Phase 0 (Stability), batch 24 of 25. No dependencies. Group S0G (Feature Flag & Safety).

## Problem
In `crates/roko-cli/src/commands/plan.rs:394`, `dangerously_skip_permissions` is hardcoded
to `true`:

```rust
dangerously_skip_permissions: true,
```

This means `roko plan run` always skips the Claude CLI permission system, which is designed
to prevent agents from executing destructive operations without user approval. This is a
significant safety gap -- agents can modify files, run commands, and access the network
without any permission checks.

The fix is to default to `false` (permissions enforced) and add a `--skip-permissions`
CLI flag for users who explicitly want to bypass the safety check.

## Exact Changes

### Step 1: Add CLI flag to plan run subcommand

In `crates/roko-cli/src/main.rs`, find the `plan run` subcommand definition. Add:

```rust
/// Skip Claude CLI permission prompts (DANGEROUS: agents can modify files
/// and run commands without approval).
#[arg(long, alias = "dangerously-skip-permissions")]
skip_permissions: bool,
```

Find the exact enum variant for `plan run` (likely `PlanCmd::Run { ... }`) and add
the field there.

### Step 2: Thread the flag to the runner config

In `crates/roko-cli/src/commands/plan.rs`, find where `dangerously_skip_permissions` is
set (around line 394). Replace the hardcoded `true` with the CLI flag value:

```rust
dangerously_skip_permissions: skip_permissions,
```

Where `skip_permissions` comes from the parsed CLI args.

### Step 3: Default to false

Ensure the field defaults to `false`:
```rust
dangerously_skip_permissions: skip_permissions, // default: false (permissions enforced)
```

### Step 4: Add warning when permissions are skipped

When `skip_permissions` is true, log a warning:

```rust
if skip_permissions {
    tracing::warn!(
        "permission checks disabled (--skip-permissions). \
         Agents can modify files and run commands without approval."
    );
    eprintln!(
        "WARNING: Permission checks disabled. Agents can modify files \
         and run commands without approval."
    );
}
```

### Step 5: Check other callers

Search for other places where `dangerously_skip_permissions` is set:

```bash

```

For each hit, evaluate whether it should also default to `false`.
