# S0_25: Wire AgentContract into runner v2

## Task
Load `AgentContract` for the task role in `crates/roko-cli/src/runner/event_loop.rs` and pass `allowed_tools` to the agent dispatch.

## Runner Context
Post-parity Phase 0 (Stability), batch 25 of 25. No dependencies. Group S0G (Feature Flag & Safety).

## Problem
The `AgentContract` system in `roko-agent/src/safety/contract.rs` defines per-role
tool restrictions, token budgets, and invariants. It is wired in the ACP path
(`roko-acp/src/runner.rs:15,824`) but NOT in the runner v2 event loop.

The runner v2 at `crates/roko-cli/src/runner/event_loop.rs` dispatches agents for
plan tasks without loading or enforcing contracts. This means:
- All tools are available regardless of agent role
- No token budget enforcement
- No invariant checking

The `SafetyLayer` in `roko-agent/src/safety/mod.rs` already supports contracts via
`with_contract()`. The runner just needs to load the contract and pass it.

## Exact Changes

### Step 1: Find the agent dispatch point in event_loop.rs

In `crates/roko-cli/src/runner/event_loop.rs`, find where agents are spawned for tasks.
Look for `AgentSpawnConfig`, `spawn_agent`, or `dispatch_agent` calls.

### Step 2: Load contract for the task role

Before agent spawn, load the contract for the task's role:

```rust
use roko_agent::safety::contract::AgentContract;

// Load contract for the task role
let contract = match &task.role {
    Some(role) => {
        let contract_path = workdir.join(format!(".roko/contracts/{role}.yaml"));
        match std::fs::read_to_string(&contract_path) {
            Ok(yaml) => match serde_yaml::from_str::<AgentContract>(&yaml) {
                Ok(contract) => {
                    tracing::info!(role, "loaded agent contract");
                    Some(contract)
                }
                Err(e) => {
                    tracing::warn!(role, "failed to parse contract: {e}; using restricted fallback");
                    Some(AgentContract::restricted(role))
                }
            },
            Err(_) => {
                tracing::debug!(role, "no contract file; using restricted fallback");
                Some(AgentContract::restricted(role))
            }
        }
    }
    None => {
        tracing::debug!("no role specified; using restricted fallback");
        Some(AgentContract::restricted("default"))
    }
};
```

### Step 3: Extract allowed_tools from contract

From the loaded contract, extract the allowed tool list:

```rust
let allowed_tools: Option<Vec<String>> = contract.as_ref().map(|c| {
    c.allowed_tools.clone().unwrap_or_default()
});
```

Check the `AgentContract` struct for the exact field name for allowed tools.

### Step 4: Pass to agent dispatch

Thread `allowed_tools` into the `AgentSpawnConfig` or `AgentDispatchRequest`:

```rust
let spawn_config = AgentSpawnConfig {
    // ... existing fields ...
    allowed_tools,
    contract,
    // ...
};
```

If the spawn config does not have these fields, add them:

```rust
pub struct AgentSpawnConfig {
    // ... existing ...
    /// Allowed tools from the agent contract (None = all tools).
    pub allowed_tools: Option<Vec<String>>,
    /// Loaded agent contract for safety enforcement.
    pub contract: Option<AgentContract>,
}
```

### Step 5: Pass to SafetyLayer

If the dispatch path creates a `SafetyLayer`, attach the contract:

```rust
let safety = SafetyLayer::with_defaults();
let safety = if let Some(contract) = contract {
    safety.with_contract(contract)
} else {
    safety
};
```
