# SF_03: Implement trust domain split (privileged/quarantined)

## Task
Define a `TrustDomain` enum and `TrustDomainPolicy` struct in a new `trust_domain.rs` module that classifies agent roles into privilege tiers and restricts capabilities accordingly.

## Runner Context
Runner SF_A (Safety: Contract Enforcement), batch 3 of 5. No dependencies.

## Problem
All agents currently run with the same effective privilege level. There is no formal distinction between a trusted "implementer" agent (which needs file write access) and an untrusted "researcher" agent (which should only read). The safety layer applies policies uniformly, but there is no domain classification to vary policies by trust level.

### Step 1: Create `crates/roko-agent/src/safety/trust_domain.rs`

```rust
//! Trust domain classification for agent roles (SF_03).
//!
//! Agents are classified into trust domains that determine their
//! baseline capability set. Contracts can further restrict within
//! a domain, but cannot escalate beyond it.

use serde::{Deserialize, Serialize};

/// Trust domain classification.
#[derive(Debug, Clone, Copy, PartialEq, Eq, Hash, Serialize, Deserialize)]
#[serde(rename_all = "snake_case")]
pub enum TrustDomain {
    /// Full access: file read/write, bash, network, git.
    /// Roles: implementer, auto-fixer.
    Privileged,
    /// Read + limited write: file read, restricted bash (no rm -rf, no sudo).
    /// Roles: reviewer, strategist.
    Standard,
    /// Read-only: file read, grep, glob. No bash, no write, no network.
    /// Roles: researcher, observer.
    Quarantined,
}

impl TrustDomain {
    /// Classify a role name into its trust domain.
    pub fn for_role(role: &str) -> Self {
        match role.to_lowercase().as_str() {
            "implementer" | "auto-fixer" | "auto_fixer" | "fixer" => Self::Privileged,
            "reviewer" | "strategist" | "planner" => Self::Standard,
            "researcher" | "observer" | "auditor" => Self::Quarantined,
            _ => Self::Standard, // unknown roles get standard
        }
    }

    /// Maximum tool set allowed for this domain.
    pub fn max_allowed_tools(&self) -> &'static [&'static str] {
        match self {
            Self::Privileged => &[
                "read_file", "write_file", "edit_file", "multi_edit",
                "apply_patch", "notebook_edit", "glob", "grep", "ls",
                "bash", "run_tests", "web_fetch", "web_search",
            ],
            Self::Standard => &[
                "read_file", "edit_file", "glob", "grep", "ls",
                "bash",
            ],
            Self::Quarantined => &[
                "read_file", "glob", "grep", "ls",
            ],
        }
    }

    /// Whether this domain allows the given tool.
    pub fn allows_tool(&self, tool_name: &str) -> bool {
        self.max_allowed_tools().contains(&tool_name)
    }
}

/// Policy that combines trust domain with contract.
#[derive(Debug, Clone)]
pub struct TrustDomainPolicy {
    pub domain: TrustDomain,
    /// Effective allowed tools = intersection(domain.max_allowed_tools, contract.allowed_tools).
    pub effective_tools: Vec<String>,
}

impl TrustDomainPolicy {
    /// Compute the effective policy for a role with an optional contract tool list.
    pub fn for_role(role: &str, contract_tools: Option<&[String]>) -> Self {
        let domain = TrustDomain::for_role(role);
        let domain_tools = domain.max_allowed_tools();
        let effective_tools = match contract_tools {
            Some(ct) => ct
                .iter()
                .filter(|t| domain_tools.contains(&t.as_str()))
                .cloned()
                .collect(),
            None => domain_tools.iter().map(|s| (*s).to_string()).collect(),
        };
        Self { domain, effective_tools }
    }
}
```
