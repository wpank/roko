# SF_04: Immutable gate configs -- load at pipeline start, freeze

## Task
Make `GateService` load its configuration once at pipeline start and freeze it so gate configs cannot be mutated mid-run by agent output or side effects.

## Runner Context
Runner SF_A (Safety: Contract Enforcement), batch 4 of 5. No dependencies.

## Problem
`GateService` currently constructs gate implementations on-the-fly in `gate_for_name()` and reads adaptive thresholds from a `Mutex`-guarded store. In theory, an agent could modify files that influence gate behavior between iterations. Gate configs should be snapshotted at pipeline start and made immutable.

## Exact Changes

### Step 1: Add a frozen config snapshot

```rust
/// Frozen gate configuration captured at pipeline start.
///
/// Once created, this snapshot is immutable. It captures the gate
/// names, ordering, and adaptive thresholds at the moment the pipeline
/// begins, preventing mid-run mutations.
#[derive(Debug, Clone)]
pub struct FrozenGateConfig {
    /// Ordered list of gate names to run.
    pub gates: Vec<String>,
    /// Snapshot of adaptive thresholds at freeze time.
    pub adaptive_snapshot: Option<AdaptiveThresholds>,
    /// Whether the config has been frozen (used for debug assertions).
    frozen: bool,
}

impl FrozenGateConfig {
    /// Create a frozen config from the current gate service state.
    pub fn freeze(gates: Vec<String>, adaptive: Option<&Mutex<AdaptiveThresholds>>) -> Self {
        let adaptive_snapshot = adaptive.map(|a| a.lock().unwrap().clone());
        Self {
            gates,
            adaptive_snapshot,
            frozen: true,
        }
    }

    /// Returns true if this config is frozen.
    pub fn is_frozen(&self) -> bool {
        self.frozen
    }
}
```

### Step 2: Add freeze method to GateService

```rust
impl GateService {
    /// Freeze the current configuration into an immutable snapshot.
    ///
    /// Call this at pipeline start. The returned `FrozenGateConfig` captures
    /// the current state and cannot be mutated.
    pub fn freeze_config(&self, gates: Vec<String>) -> FrozenGateConfig {
        FrozenGateConfig::freeze(
            gates,
            self.adaptive.as_ref().map(|a| a.as_ref()),
        )
    }
}
```

### Step 3: Add run_with_frozen method

```rust
impl GateService {
    /// Run gates using a frozen config snapshot rather than live state.
    ///
    /// This ensures gate ordering and thresholds cannot change mid-pipeline.
    pub async fn run_gates_frozen(
        &self,
        frozen: &FrozenGateConfig,
        workdir: &std::path::Path,
        shell_gates: &[roko_core::foundation::ShellGateCommand],
    ) -> roko_core::foundation::GateReport {
        debug_assert!(frozen.is_frozen(), "gate config must be frozen before use");
        // Delegate to existing run_gates with the frozen gate list
        self.run_gates_inner(&frozen.gates, workdir, shell_gates).await
    }
}
```
