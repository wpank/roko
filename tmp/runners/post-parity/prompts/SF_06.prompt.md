# SF_06: Tiered permission model -- define PermissionTier enum

## Task
Define a `PermissionTier` enum that encodes graduated access levels (None, ReadOnly, Suggest, Write, Admin) and wire it into the `AuthzDecision` system.

## Runner Context
Runner SF_B (Safety: Permission System), batch 1 of 3. Depends on SF_01.

## Problem
The current authorization system (`authz.rs`) has `AuthzDecision` (Permit/Deny/Escalate) but no graduated permission model. Every tool is either fully allowed or fully denied. There is no concept of "read access is free but write requires approval" or "suggest changes but don't apply them."

## Exact Changes

### Step 1: Define PermissionTier in authz.rs

```rust
/// Graduated permission tier for tool access.
///
/// Tiers are ordered from least to most privileged. A tool operation
/// requires a minimum tier; if the agent's effective tier is below
/// that threshold, the operation is either denied or escalated.
#[derive(Debug, Clone, Copy, PartialEq, Eq, PartialOrd, Ord, Hash, Serialize, Deserialize)]
#[serde(rename_all = "snake_case")]
pub enum PermissionTier {
    /// No access at all.
    None,
    /// Read-only access: file reads, grep, glob, ls.
    ReadOnly,
    /// Can suggest changes (diff preview) but not apply them.
    Suggest,
    /// Full write access: edit, write, bash, network.
    Write,
    /// Administrative: can modify safety configs, contracts, policies.
    Admin,
}

impl PermissionTier {
    /// The minimum tier required for a given tool.
    pub fn required_for_tool(tool_name: &str) -> Self {
        match tool_name {
            "read_file" | "glob" | "grep" | "ls" => Self::ReadOnly,
            "edit_file" | "write_file" | "multi_edit" | "apply_patch"
            | "notebook_edit" => Self::Write,
            "bash" | "run_tests" => Self::Write,
            "web_fetch" | "web_search" => Self::ReadOnly,
            _ => Self::Write, // default to write for unknown tools
        }
    }

    /// Check if this tier satisfies the requirement.
    pub fn satisfies(self, required: Self) -> bool {
        self >= required
    }
}
```

### Step 2: Add tier to AuthzDecision::Permit

Extend the `Permit` variant (or add a helper) to carry the effective tier:

```rust
impl AuthzDecision {
    /// Create a permit decision with an explicit permission tier.
    pub fn permit_with_tier(evidence: AuthorizationEvidence, tier: PermissionTier) -> Self {
        Self::Permit { evidence }
        // Note: tier is used by callers for downstream filtering,
        // not stored in the decision itself. Keep the existing variant shape.
    }

    /// Check a tool call against the given tier.
    pub fn check_tier(tier: PermissionTier, tool_name: &str) -> Self {
        let required = PermissionTier::required_for_tool(tool_name);
        if tier.satisfies(required) {
            Self::Permit {
                evidence: AuthorizationEvidence::role_grant(
                    format!("tier {tier:?} satisfies {required:?} for {tool_name}")
                ),
            }
        } else {
            Self::Deny {
                reason: format!(
                    "tool '{tool_name}' requires {required:?} but agent has {tier:?}"
                ),
            }
        }
    }
}
```
