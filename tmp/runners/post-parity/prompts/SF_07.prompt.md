# SF_07: Worktree isolation enforcement -- path policy for agents

## Task
Extend `PathPolicy` to enforce per-agent worktree isolation: each agent can only access files within its assigned worktree directory, not the main repo or other agents' worktrees.

## Runner Context
Runner SF_B (Safety: Permission System), batch 2 of 3. Depends on SF_01.

## Problem
`PathPolicy` currently validates paths against a single worktree root. When multiple agents run in parallel with separate worktrees (via `WorktreeManager`), each agent should be confined to its own worktree. Currently there is no enforcement binding a specific agent to a specific worktree path.

## Exact Changes

### Step 1: Add agent-scoped worktree field to PathPolicy

```rust
pub struct PathPolicy {
    pub deny_symlinks: bool,
    pub prevent_escapes: bool,
    /// Optional agent-specific worktree root. When set, paths are validated
    /// against this root instead of the main worktree, providing per-agent
    /// isolation in multi-agent parallel execution.
    pub agent_worktree: Option<PathBuf>,
}
```

### Step 2: Update canonicalize_with_policy

In `canonicalize_with_policy`, use `agent_worktree` when present:

```rust
let effective_root = policy.agent_worktree.as_deref().unwrap_or(worktree);
// Use effective_root for all subsequent path validation
```

### Step 3: Add factory method for agent-scoped policy

```rust
impl PathPolicy {
    /// Create a policy that confines an agent to a specific worktree directory.
    pub fn for_agent_worktree(worktree_path: PathBuf) -> Self {
        Self {
            deny_symlinks: false,
            prevent_escapes: true,
            agent_worktree: Some(worktree_path),
        }
    }
}
```

### Step 4: Update Default impl

```rust
impl Default for PathPolicy {
    fn default() -> Self {
        Self {
            deny_symlinks: false,
            prevent_escapes: true,
            agent_worktree: None,
        }
    }
}
```
