# SF_08: Bash command categorization and risk-level blocking

## Task
Add risk-level categorization to `BashPolicy` so commands can be classified as low/medium/high/critical risk, and blocking decisions can be made based on the agent's trust level rather than a flat denylist.

## Runner Context
Runner SF_B (Safety: Permission System), batch 3 of 3. No dependencies.

## Problem
`BashPolicy` currently has a flat denylist: if a command matches any `DenyPattern`, it is rejected. There is no distinction between "this command could destroy data" (critical) and "this command is slightly risky" (medium). Different agent trust levels should tolerate different risk thresholds.

## Exact Changes

### Step 1: Add RiskLevel enum

```rust
/// Risk classification for bash commands.
#[derive(Debug, Clone, Copy, PartialEq, Eq, PartialOrd, Ord, Hash)]
pub enum RiskLevel {
    /// Safe commands: ls, cat, echo, grep, cargo build.
    Low,
    /// Moderate: git commit, cargo test, file writes.
    Medium,
    /// Dangerous: rm (with flags), chmod, network tools.
    High,
    /// Catastrophic: rm -rf /, sudo, mkfs, fork bombs.
    Critical,
}
```

### Step 2: Add CategorizedDenyPattern

```rust
/// A deny pattern with an associated risk level.
#[derive(Debug, Clone)]
pub struct CategorizedDenyPattern {
    pub pattern: DenyPattern,
    pub risk: RiskLevel,
    pub description: String,
}
```

### Step 3: Add categorize_command function

```rust
/// Categorize a command's risk level based on pattern matching.
///
/// Returns the highest risk level matched, or `Low` if no patterns match.
pub fn categorize_command(command: &str, patterns: &[CategorizedDenyPattern]) -> RiskLevel {
    patterns
        .iter()
        .filter(|p| p.pattern.matches(command))
        .map(|p| p.risk)
        .max()
        .unwrap_or(RiskLevel::Low)
}
```

### Step 4: Add default categorized patterns to BashPolicy

```rust
impl BashPolicy {
    /// Build the default categorized deny patterns.
    pub fn default_categorized_patterns() -> Vec<CategorizedDenyPattern> {
        vec![
            CategorizedDenyPattern {
                pattern: DenyPattern::Substring("rm -rf /".into()),
                risk: RiskLevel::Critical,
                description: "recursive root deletion".into(),
            },
            CategorizedDenyPattern {
                pattern: DenyPattern::Substring("sudo ".into()),
                risk: RiskLevel::Critical,
                description: "privilege escalation".into(),
            },
            CategorizedDenyPattern {
                pattern: DenyPattern::Substring("mkfs".into()),
                risk: RiskLevel::Critical,
                description: "filesystem format".into(),
            },
            CategorizedDenyPattern {
                pattern: DenyPattern::Substring("chmod 777 /".into()),
                risk: RiskLevel::Critical,
                description: "world-writable root".into(),
            },
            CategorizedDenyPattern {
                pattern: DenyPattern::Substring("rm -rf".into()),
                risk: RiskLevel::High,
                description: "recursive forced deletion".into(),
            },
            CategorizedDenyPattern {
                pattern: DenyPattern::Substring("chmod".into()),
                risk: RiskLevel::Medium,
                description: "permission change".into(),
            },
            CategorizedDenyPattern {
                pattern: DenyPattern::Substring("curl".into()),
                risk: RiskLevel::Medium,
                description: "network download".into(),
            },
        ]
    }

    /// Check a command against a risk threshold.
    ///
    /// Commands at or above the threshold are rejected.
    pub fn check_with_risk_threshold(
        &self,
        command: &str,
        max_allowed_risk: RiskLevel,
    ) -> Result<(), ToolError> {
        let risk = categorize_command(command, &Self::default_categorized_patterns());
        if risk > max_allowed_risk {
            return Err(ToolError::Other(format!(
                "command risk level {risk:?} exceeds threshold {max_allowed_risk:?}"
            )));
        }
        // Also run the existing flat denylist
        check_command_with_policy(command, self)
    }
}
```
