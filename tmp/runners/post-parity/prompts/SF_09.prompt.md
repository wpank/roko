# SF_09: MCP server version pinning -- require explicit version

## Task
Add a `version` field to the MCP server configuration schema so that MCP servers must declare an explicit version, preventing silent upgrades that could introduce security vulnerabilities.

## Runner Context
Runner SF_C (Safety: MCP & Audit), batch 1 of 4. No dependencies.

## Problem
MCP servers are currently configured by command path without any version constraint. A malicious or buggy update to an MCP binary on PATH could silently change the tool surface available to agents. Version pinning adds an integrity check.

## Current Code

The MCP configuration lives in `roko.toml` under `agent.mcp_config` as a path to a JSON file. The JSON file follows the Claude MCP config format with `mcpServers` entries that specify `command` and `args` but no version.

Example `mcp.json`:
```json
{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"]
    }
  }
}
```

## Exact Changes

### Step 1: Add MCP version config types

In the roko-core config module (find the appropriate file, likely `config/mod.rs` or create a section):

```rust
/// Configuration for an MCP server with version pinning.
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct McpServerConfig {
    /// Command to invoke the MCP server.
    pub command: String,
    /// Arguments to pass.
    #[serde(default)]
    pub args: Vec<String>,
    /// Required version string (semver). When set, the server's
    /// `--version` output is checked at startup.
    #[serde(default, skip_serializing_if = "Option::is_none")]
    pub version: Option<String>,
    /// SHA-256 hash of the server binary. When set, the binary is
    /// verified before execution.
    #[serde(default, skip_serializing_if = "Option::is_none")]
    pub sha256: Option<String>,
}
```

### Step 2: Add version verification function

```rust
/// Verify an MCP server matches its pinned version.
///
/// Runs `<command> --version` and checks the output contains the
/// expected version string. Returns Ok(()) if no version is pinned.
pub async fn verify_mcp_server_version(config: &McpServerConfig) -> Result<(), String> {
    let Some(ref expected) = config.version else {
        return Ok(());
    };

    let output = tokio::process::Command::new(&config.command)
        .arg("--version")
        .output()
        .await
        .map_err(|e| format!("failed to run {} --version: {e}", config.command))?;

    let stdout = String::from_utf8_lossy(&output.stdout);
    let stderr = String::from_utf8_lossy(&output.stderr);
    let combined = format!("{stdout}{stderr}");

    if combined.contains(expected.as_str()) {
        Ok(())
    } else {
        Err(format!(
            "MCP server '{}' version mismatch: expected '{}', got '{}'",
            config.command,
            expected,
            combined.trim()
        ))
    }
}
```
