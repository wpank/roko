# SF_12: Secret detection in agent context -- wire ScrubPolicy

## Task
Wire `ScrubPolicy` into the agent prompt assembly path so that secrets in agent context (system prompts, user messages, tool outputs fed back into context) are scrubbed before reaching the LLM.

## Runner Context
Runner SF_C (Safety: MCP & Audit), batch 4 of 4. No dependencies.

## Problem
`ScrubPolicy` and `scrub_secrets()` exist in `crates/roko-agent/src/safety/scrub.rs` with comprehensive regex patterns for API keys, JWTs, AWS credentials, GitHub tokens, etc. `ResultFilter` uses them for tool outputs. But the agent context assembly path (system prompt + user prompt + prior tool outputs) does not run through scrubbing, so secrets in file contents or environment can leak into the LLM context.

## Exact Changes

### Step 1: Add context scrubbing function

In `scrub.rs`, add a high-level function for scrubbing agent context:

```rust
/// Scrub secrets from agent context text.
///
/// This is a convenience wrapper around `scrub_secrets` intended for
/// scrubbing system prompts, user messages, and assembled context
/// before they reach the LLM.
pub fn scrub_agent_context(content: &str) -> String {
    scrub_secrets(content, &ScrubPolicy::default())
}

/// Scrub a batch of chat messages in-place.
///
/// Applies secret scrubbing to the content of each message.
pub fn scrub_messages(messages: &mut [ChatMessage]) {
    let policy = ScrubPolicy::default();
    for msg in messages {
        msg.content = scrub_secrets(&msg.content, &policy);
    }
}
```

Where `ChatMessage` is the message struct used in the agent dispatch path. Check if it is `roko_core::foundation::ChatMessage` or similar.

### Step 2: Add scrub_before_dispatch flag to SafetyLayer

```rust
// In SafetyLayer:
/// Whether to scrub secrets from context before dispatch.
pub scrub_context: bool,
```

Default to `true` in `with_defaults()`.

### Step 3: Add context scrubbing helper to SafetyLayer

```rust
impl SafetyLayer {
    /// Scrub secrets from a prompt string if context scrubbing is enabled.
    pub fn scrub_if_enabled(&self, content: &str) -> String {
        if self.scrub_context {
            scrub_secrets(content, &self.scrub_policy)
        } else {
            content.to_string()
        }
    }
}
```
