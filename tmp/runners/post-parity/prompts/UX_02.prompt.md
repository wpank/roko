# UX_02: Add run summary output after each roko run

## Task
Print a structured summary after each `roko run` completes, showing verdicts, timing, costs, and gate results.

## Runner Context
Runner UX, batch 2 of 5. No dependencies.

## Problem
After `roko run` completes, the user gets no summary. They have to dig through logs or JSONL files to understand what happened. A concise summary table showing pass/fail verdicts, timing, token costs, and gate results is essential UX.

## Current Code

**RunReport** -- `crates/roko-cli/src/run.rs:65-87`:
```rust
pub struct RunReport {
    pub plan_id: Option<String>,
    pub task_count: usize,
    pub tasks_completed: usize,
    pub tasks_failed: usize,
    pub total_cost_usd: f64,
    pub total_tokens: u64,
    pub duration: Duration,
    pub gate_results: Vec<(String, bool)>,
}
```

**run.rs** -- returns `RunReport` but it may not be printed to the user.

## Exact Changes

### Step 1: Add print_run_summary function

In `crates/roko-cli/src/run.rs`, add:

```rust
/// Print a human-readable summary after a run completes.
pub fn print_run_summary(report: &RunReport) {
    println!();
    println!("=== Run Summary ===");
    println!();

    // Verdict
    let verdict = if report.tasks_failed == 0 { "PASS" } else { "FAIL" };
    let color = if report.tasks_failed == 0 { "\x1b[32m" } else { "\x1b[31m" };
    println!("  Verdict:  {color}{verdict}\x1b[0m");

    if let Some(ref plan_id) = report.plan_id {
        println!("  Plan:     {plan_id}");
    }

    // Tasks
    println!("  Tasks:    {}/{} completed ({} failed)",
        report.tasks_completed, report.task_count, report.tasks_failed);

    // Timing
    let secs = report.duration.as_secs();
    let mins = secs / 60;
    let remaining_secs = secs % 60;
    if mins > 0 {
        println!("  Duration: {mins}m {remaining_secs}s");
    } else {
        println!("  Duration: {remaining_secs}s");
    }

    // Cost
    if report.total_cost_usd > 0.0 {
        println!("  Cost:     ${:.4}", report.total_cost_usd);
    }
    if report.total_tokens > 0 {
        println!("  Tokens:   {}", format_tokens(report.total_tokens));
    }

    // Gate results
    if !report.gate_results.is_empty() {
        println!();
        println!("  Gates:");
        for (name, passed) in &report.gate_results {
            let icon = if *passed { "\x1b[32m+\x1b[0m" } else { "\x1b[31mx\x1b[0m" };
            println!("    {icon} {name}");
        }
    }

    println!();
}

fn format_tokens(tokens: u64) -> String {
    if tokens >= 1_000_000 {
        format!("{:.1}M", tokens as f64 / 1_000_000.0)
    } else if tokens >= 1_000 {
        format!("{:.1}K", tokens as f64 / 1_000.0)
    } else {
        tokens.to_string()
    }
}
```

### Step 2: Call print_run_summary after run completion

At every exit point where a `RunReport` is available, call:

```rust
    print_run_summary(&report);
```

This includes both successful and failed runs.

### Step 3: Also emit JSON summary when --json flag is set

```rust
    if json_output {
        let json = serde_json::to_string_pretty(&report)?;
        println!("{json}");
    } else {
        print_run_summary(&report);
    }
```

- Every `roko run` prints a summary showing verdict, tasks, duration, cost, gates
- Verdict is colored: green for PASS, red for FAIL
- Gate results listed with pass/fail indicators
- `--json` flag outputs JSON instead of human-readable
- Token counts formatted with K/M suffixes
