# UX_04: Implement roko pack create command

## Task
Create the `roko pack create` command that aggregates sources (directories, files, URLs) into a single context pack file.

## Runner Context
Runner UX, batch 4 of 5. No dependencies.

## Problem
Users need to feed context from multiple sources into agents. Currently, they manually copy-paste or use ad-hoc scripts. A `roko pack create` command should aggregate file contents, directory listings, and URL text into a single `.pack` file that can be fed to agents.

**commands/mod.rs** -- `crates/roko-cli/src/commands/mod.rs`:
Lists all command modules. Does not include `pack`.

### Step 1: Create `crates/roko-cli/src/commands/pack.rs`

```rust
//! `roko pack` — context pack creation and synthesis.

use anyhow::{Result, bail, Context as _};
use std::path::{Path, PathBuf};
use std::io::Write;

/// Create a context pack from multiple sources.
pub fn create_pack(
    name: &str,
    sources: &[String],
    output_dir: &Path,
    max_tokens: usize,
) -> Result<PathBuf> {
    let mut sections = Vec::new();

    for source in sources {
        let path = Path::new(source);
        if path.exists() {
            if path.is_dir() {
                let section = pack_directory(path)?;
                sections.push(section);
            } else if path.is_file() {
                let section = pack_file(path)?;
                sections.push(section);
            }
        } else {
            // Treat as a glob pattern
            let matched = glob_source(source)?;
            for file in matched {
                let section = pack_file(&file)?;
                sections.push(section);
            }
        }
    }

    if sections.is_empty() {
        bail!("No sources matched. Provide file paths, directories, or glob patterns.");
    }

    // Assemble pack
    let mut content = String::new();
    content.push_str(&format!("# Context Pack: {name}\n\n"));
    content.push_str(&format!("Sources: {}\n", sources.join(", ")));
    content.push_str(&format!("Sections: {}\n\n", sections.len()));

    let mut total_chars = 0usize;
    let char_budget = max_tokens * 4; // ~4 chars per token

    for section in &sections {
        if total_chars + section.content.len() > char_budget {
            content.push_str(&format!("## {} (TRUNCATED)\n\n", section.source));
            let remaining = char_budget.saturating_sub(total_chars);
            let truncated: String = section.content.chars().take(remaining).collect();
            content.push_str(&truncated);
            content.push_str("\n\n[...truncated to fit token budget...]\n\n");
            break;
        }

        content.push_str(&format!("## {}\n\n", section.source));
        content.push_str(&section.content);
        content.push_str("\n\n");
        total_chars += section.content.len();
    }

    // Write pack file
    std::fs::create_dir_all(output_dir)?;
    let output_path = output_dir.join(format!("{name}.pack"));
    let mut file = std::fs::File::create(&output_path)?;
    file.write_all(content.as_bytes())?;

    let approx_tokens = total_chars / 4;
    println!("Pack created: {}", output_path.display());
    println!("  Sections:   {}", sections.len());
    println!("  Characters: {total_chars}");
    println!("  Est tokens: ~{approx_tokens}");

    Ok(output_path)
}

struct PackSection {
    source: String,
    content: String,
}

fn pack_file(path: &Path) -> Result<PackSection> {
    let content = std::fs::read_to_string(path)
        .with_context(|| format!("reading {}", path.display()))?;
    Ok(PackSection {
        source: path.display().to_string(),
        content,
    })
}

fn pack_directory(dir: &Path) -> Result<PackSection> {
    let mut content = String::new();
    content.push_str(&format!("Directory: {}\n\n", dir.display()));

    let mut file_count = 0usize;
    for entry in walkdir(dir)? {
        let entry_path = entry;
        if entry_path.is_file() {
            let relative = entry_path.strip_prefix(dir).unwrap_or(&entry_path);
            // Skip binary files
            if is_likely_binary(&entry_path) {
                continue;
            }
            match std::fs::read_to_string(&entry_path) {
                Ok(file_content) => {
                    content.push_str(&format!("### {}\n```\n", relative.display()));
                    // Limit per-file to 500 lines
                    for line in file_content.lines().take(500) {
                        content.push_str(line);
                        content.push('\n');
                    }
                    content.push_str("```\n\n");
                    file_count += 1;
                }
                Err(_) => continue,
            }
        }
    }

    content.push_str(&format!("\n({file_count} files)\n"));

    Ok(PackSection {
        source: format!("dir:{}", dir.display()),
        content,
    })
}

fn walkdir(dir: &Path) -> Result<Vec<PathBuf>> {
    let mut files = Vec::new();
    fn walk_inner(dir: &Path, files: &mut Vec<PathBuf>, depth: usize) -> Result<()> {
        if depth > 5 { return Ok(()); } // limit recursion
        for entry in std::fs::read_dir(dir)? {
            let entry = entry?;
            let path = entry.path();
            if path.is_dir() {
                let name = path.file_name().unwrap_or_default().to_string_lossy();
                // Skip hidden dirs and common noise
                if name.starts_with('.') || name == "target" || name == "node_modules" {
                    continue;
                }
                walk_inner(&path, files, depth + 1)?;
            } else {
                files.push(path);
            }
        }
        Ok(())
    }
    walk_inner(dir, &mut files, 0)?;
    files.sort();
    Ok(files)
}

fn glob_source(pattern: &str) -> Result<Vec<PathBuf>> {
    let paths: Vec<PathBuf> = glob::glob(pattern)
        .with_context(|| format!("invalid glob pattern: {pattern}"))?
        .filter_map(|r| r.ok())
        .collect();
    Ok(paths)
}

fn is_likely_binary(path: &Path) -> bool {
    let ext = path.extension().and_then(|e| e.to_str()).unwrap_or("");
    matches!(ext, "png" | "jpg" | "jpeg" | "gif" | "bmp" | "ico" |
        "woff" | "woff2" | "ttf" | "otf" | "eot" |
        "zip" | "tar" | "gz" | "br" | "zst" |
        "exe" | "dll" | "so" | "dylib" | "o" | "a" |
        "wasm" | "pdf" | "mp3" | "mp4" | "webm")
}

## Acceptance Criteria
- `roko pack create --name <name> --source <path>...` creates a `.pack` file
- Supports: files, directories, glob patterns
- Directories are walked recursively (depth 5 max), skipping binary and hidden
- Token budget respected via truncation
- Output shows section count and estimated token count

- Add LLM-based synthesis (that is UX_05)
