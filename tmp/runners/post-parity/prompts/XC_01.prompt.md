# XC_01: Migrate public APIs from anyhow to structured errors

## Task
Define a structured error enum in `roko-core` that replaces `anyhow::Error` in public API boundaries, providing typed error handling for callers.

## Runner Context
Runner XC_A (Cross-Cutting: Error Handling), batch 1 of 3. No dependencies.

## Problem
Many public APIs return `anyhow::Result` which loses type information at crate boundaries. Callers cannot pattern-match on error kinds, making retry logic and error-specific handling impossible without string parsing.

## Current Code

**roko_core::Result** -- `crates/roko-core/src/error.rs` or `lib.rs`:
Check what the current `Result` type alias is.

## Exact Changes

### Step 1: Define structured error enum

In `crates/roko-core/src/error.rs`:

```rust
//! Structured error types for roko public APIs.

use thiserror::Error;

/// Top-level error type for roko-core public APIs.
#[derive(Debug, Error)]
pub enum RokoError {
    /// Configuration error (missing field, invalid value, parse failure).
    #[error("config: {0}")]
    Config(String),

    /// I/O error (file read/write, directory creation).
    #[error("io: {source}")]
    Io {
        #[from]
        source: std::io::Error,
    },

    /// Serialization/deserialization error.
    #[error("serde: {0}")]
    Serde(String),

    /// Agent dispatch error.
    #[error("dispatch: {0}")]
    Dispatch(String),

    /// Gate verification error.
    #[error("gate: {gate}: {message}")]
    Gate { gate: String, message: String },

    /// Safety policy violation.
    #[error("safety: {0}")]
    Safety(String),

    /// Resource exhausted (budget, rate limit, timeout).
    #[error("resource: {0}")]
    Resource(String),

    /// External service error (API call, webhook).
    #[error("external: {service}: {message}")]
    External { service: String, message: String },

    /// Not found (plan, task, agent, knowledge entry).
    #[error("not found: {kind}: {id}")]
    NotFound { kind: String, id: String },

    /// Invalid state (wrong phase, missing prerequisite).
    #[error("invalid state: {0}")]
    InvalidState(String),

    /// Catch-all for errors from dependencies using anyhow.
    #[error("{0}")]
    Other(#[from] anyhow::Error),
}

impl RokoError {
    /// Create a config error.
    pub fn config(msg: impl Into<String>) -> Self {
        Self::Config(msg.into())
    }

    /// Create a dispatch error.
    pub fn dispatch(msg: impl Into<String>) -> Self {
        Self::Dispatch(msg.into())
    }

    /// Create a gate error.
    pub fn gate(gate: impl Into<String>, message: impl Into<String>) -> Self {
        Self::Gate {
            gate: gate.into(),
            message: message.into(),
        }
    }

    /// Create a safety error.
    pub fn safety(msg: impl Into<String>) -> Self {
        Self::Safety(msg.into())
    }

    /// Create a not-found error.
    pub fn not_found(kind: impl Into<String>, id: impl Into<String>) -> Self {
        Self::NotFound {
            kind: kind.into(),
            id: id.into(),
        }
    }

    /// Whether this error is retryable.
    pub fn is_retryable(&self) -> bool {
        matches!(
            self,
            Self::Io { .. }
            | Self::External { .. }
            | Self::Resource(..)
        )
    }
}

impl From<serde_json::Error> for RokoError {
    fn from(e: serde_json::Error) -> Self {
        Self::Serde(e.to_string())
    }
}

impl From<toml::de::Error> for RokoError {
    fn from(e: toml::de::Error) -> Self {
        Self::Serde(format!("toml: {e}"))
    }
}
```

### Step 2: Ensure backward compatibility

The existing `type Result<T> = std::result::Result<T, RokoError>` should remain. If `RokoError` already exists, extend it. If the current `Result` uses `anyhow::Error`, add the `Other` variant to accept `anyhow::Error` via `From`.

## Acceptance Criteria
- `RokoError` enum with typed variants for each error category
- `From` impls for `std::io::Error`, `serde_json::Error`, `anyhow::Error`
- `is_retryable()` method for retry logic
- Constructor helpers (`.config()`, `.dispatch()`, etc.)
- Backward compatible with existing `anyhow::Error` usage

## Do NOT
- Migrate existing callers (that is a gradual process)
- Remove the `anyhow` dependency
- Change function signatures in other crates
