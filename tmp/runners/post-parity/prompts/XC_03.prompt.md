# XC_03: Unify logging init (tracing subscriber)

## Task
Create a shared `init_tracing()` function that both the CLI binary and the serve binary use, ensuring consistent log format, filtering, and output across all entry points.

## Runner Context
Runner XC_A (Cross-Cutting: Error Handling), batch 3 of 3. No dependencies.

## Problem
The CLI (`main.rs`) and the server (`roko-serve`) each initialize their own tracing subscriber with slightly different configurations. Log format, filtering, and output targets diverge, making it harder to diagnose issues across components.

## Current Code

**CLI tracing init** -- `crates/roko-cli/src/main.rs`:
Has a custom tracing subscriber setup with format options.

**Serve tracing** -- `crates/roko-serve/src/lib.rs`:
Uses `tower_http::TraceLayer` but may have its own subscriber init.

## Exact Changes

### Step 1: Add shared init function to roko-cli lib

In a suitable location (e.g., `crates/roko-cli/src/lib.rs` or a new `logging.rs`):

```rust
//! Shared tracing initialization.

use tracing_subscriber::fmt::format::FmtSpan;
use tracing_subscriber::layer::SubscriberExt;
use tracing_subscriber::util::SubscriberInitExt;
use tracing_subscriber::EnvFilter;

/// Log format options.
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum LogFormat {
    /// Human-readable text output.
    Text,
    /// Structured JSON output (for log aggregation).
    Json,
    /// Compact single-line format.
    Compact,
}

/// Configuration for tracing initialization.
#[derive(Debug, Clone)]
pub struct TracingConfig {
    /// Log format.
    pub format: LogFormat,
    /// Whether to include ANSI colors.
    pub ansi: bool,
    /// Whether to include timestamps.
    pub timestamps: bool,
    /// Whether to include span events (enter/exit).
    pub span_events: bool,
    /// Custom env filter string (overrides RUST_LOG).
    pub filter: Option<String>,
}

impl Default for TracingConfig {
    fn default() -> Self {
        Self {
            format: LogFormat::Text,
            ansi: std::io::IsTerminal::is_terminal(&std::io::stderr()),
            timestamps: true,
            span_events: false,
            filter: None,
        }
    }
}

/// Initialize the global tracing subscriber.
///
/// Call this once at program start. Both the CLI and serve binaries
/// should use this function for consistent log formatting.
pub fn init_tracing(config: &TracingConfig) {
    let env_filter = config.filter.as_deref()
        .map(|f| EnvFilter::new(f))
        .unwrap_or_else(|| {
            EnvFilter::try_from_default_env()
                .unwrap_or_else(|_| EnvFilter::new("info,roko=debug"))
        });

    let span_events = if config.span_events {
        FmtSpan::NEW | FmtSpan::CLOSE
    } else {
        FmtSpan::NONE
    };

    match config.format {
        LogFormat::Text => {
            tracing_subscriber::registry()
                .with(env_filter)
                .with(
                    tracing_subscriber::fmt::layer()
                        .with_ansi(config.ansi)
                        .with_target(false)
                        .with_span_events(span_events)
                        .with_timer(
                            if config.timestamps {
                                tracing_subscriber::fmt::time::SystemTime::default()
                            } else {
                                // Use a no-op timer when timestamps are disabled
                                tracing_subscriber::fmt::time::SystemTime::default()
                            }
                        )
                )
                .init();
        }
        LogFormat::Json => {
            tracing_subscriber::registry()
                .with(env_filter)
                .with(
                    tracing_subscriber::fmt::layer()
                        .json()
                        .with_span_events(span_events)
                )
                .init();
        }
        LogFormat::Compact => {
            tracing_subscriber::registry()
                .with(env_filter)
                .with(
                    tracing_subscriber::fmt::layer()
                        .compact()
                        .with_ansi(config.ansi)
                        .with_target(false)
                )
                .init();
        }
    }
}
```

### Step 2: Wire into CLI main

Replace the existing tracing init in `main.rs` with:
```rust
roko_cli::init_tracing(&TracingConfig {
    format: if args.json { LogFormat::Json } else { LogFormat::Text },
    ansi: !args.no_color,
    ..Default::default()
});
```
