# XC_04: Propagate CancelToken for graceful shutdown

## Task
Ensure `CancelToken` is propagated through all dispatch paths so that `Ctrl+C` or API-triggered cancellation gracefully stops agents, gates, and I/O operations.

## Runner Context
Runner XC_B (Cross-Cutting: Lifecycle), batch 1 of 3. No dependencies.

## Problem
`CancelToken` exists in `roko-runtime` but may not be checked at every async await point. Long-running operations (agent dispatch, gate execution, merge queue polling) should check the token periodically and abort cleanly.

## Current Code

**CancelToken** -- `crates/roko-runtime/src/cancel.rs`:
Check the existing implementation. It likely wraps a `tokio::sync::watch` or `CancellationToken`.

**WorkflowEngine** -- `crates/roko-runtime/src/workflow_engine.rs`:
Uses `CancelToken` at the top-level loop but may not pass it to sub-operations.

## Exact Changes

### Step 1: Audit cancel token usage

Check all long-running operations and ensure they accept a `CancelToken`:

```rust
// In EffectDriver::spawn_agent:
// Existing: spawns agent without cancel awareness
// Changed: select! on cancel token + agent completion

tokio::select! {
    result = agent_future => {
        // normal completion
        result
    }
    _ = cancel_token.cancelled() => {
        // kill the agent process
        tracing::info!(agent_id = %agent_id, "cancelling agent due to shutdown");
        PipelineInput::UserCancel
    }
}
```

### Step 2: Add cancel check to gate execution

```rust
// In GateService run_gates:
for gate in &gates {
    if cancel_token.is_cancelled() {
        return GateReport::cancelled();
    }
    // run gate...
}
```

### Step 3: Add cancel token to WorkflowRunConfig

```rust
pub struct WorkflowRunConfig {
    // ... existing fields ...
    /// Cancellation token for graceful shutdown.
    pub cancel_token: CancelToken,
}
```

### Step 4: Propagate to EffectDriver

```rust
pub struct EffectDriver {
    services: EffectServices,
    run_id: String,
    workdir: PathBuf,
    feedback_totals: tokio::sync::Mutex<WorkflowFeedbackTotals>,
    cancel_token: CancelToken,
}
```
