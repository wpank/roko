# XC_05: Serve shutdown hooks

## Task
Add graceful shutdown hooks to `roko serve` that drain active connections, stop background tasks, and flush pending writes before exit.

## Runner Context
Runner XC_B (Cross-Cutting: Lifecycle), batch 2 of 3. No dependencies.

## Problem
`roko serve` does not handle shutdown gracefully. On SIGTERM/SIGINT, active WebSocket connections are dropped, pending event bus writes are lost, and background subscribers may leave inconsistent state.

## Exact Changes

### Step 1: Add shutdown handler to serve startup

In `crates/roko-serve/src/lib.rs` (or wherever the server is started):

```rust
/// Graceful shutdown handler.
async fn shutdown_signal() {
    let ctrl_c = async {
        tokio::signal::ctrl_c()
            .await
            .expect("failed to install Ctrl+C handler");
    };

    #[cfg(unix)]
    let terminate = async {
        tokio::signal::unix::signal(tokio::signal::unix::SignalKind::terminate())
            .expect("failed to install SIGTERM handler")
            .recv()
            .await;
    };

    #[cfg(not(unix))]
    let terminate = std::future::pending::<()>();

    tokio::select! {
        _ = ctrl_c => {
            tracing::info!("received Ctrl+C, initiating graceful shutdown");
        }
        _ = terminate => {
            tracing::info!("received SIGTERM, initiating graceful shutdown");
        }
    }
}
```

### Step 2: Add shutdown hooks registry

```rust
/// Registry of shutdown hooks called during graceful shutdown.
pub struct ShutdownHooks {
    hooks: Vec<Box<dyn FnOnce() + Send>>,
}

impl ShutdownHooks {
    pub fn new() -> Self {
        Self { hooks: Vec::new() }
    }

    pub fn register(&mut self, hook: impl FnOnce() + Send + 'static) {
        self.hooks.push(Box::new(hook));
    }

    pub fn run(self) {
        for hook in self.hooks {
            hook();
        }
    }
}
```

### Step 3: Wire into server

```rust
// In the serve function:
let server = axum::serve(listener, app)
    .with_graceful_shutdown(shutdown_signal());

// After server completes:
tracing::info!("server stopped, running shutdown hooks");
state.flush_pending_writes();
state.close_event_bus();
shutdown_hooks.run();
tracing::info!("shutdown complete");
```

## Acceptance Criteria
- Graceful shutdown on SIGTERM and SIGINT
- Pending writes flushed before exit
- Event bus closed (subscribers notified)
- Shutdown hooks registry for extensibility
- Log messages for shutdown progress

## Do NOT
- Add forced shutdown timeout (that is an OS responsibility)
- Kill active agent processes (they manage their own lifecycle)
- Add new dependencies beyond tokio::signal
