# D01: Add `max_cost_per_{plan,task,session}_usd` config + enforcement in orchestrate

## Goal

Three new config fields under `[agent]` (with sane defaults) and enforcement in `orchestrate.rs` that, before each dispatch, looks up cumulative cost via `costs_db.query_by_plan` and raises a `BudgetExceeded` error (the existing `RokoError` variant) if the next call would exceed any of the three limits.

Per-call enforcement at the gateway is already partly done in `model_call_service.rs`. This batch adds *plan*-level and *session*-level rollups in addition to the existing per-call check.

## Step 1: config fields

Open `crates/roko-core/src/config/agent.rs`. Locate `AgentConfig`. Add:

```rust
/// Hard limit on total USD spent by a single plan run. 0.0 = unlimited.
/// On exceed, in-flight dispatches still complete; subsequent ones return
/// `RokoError::BudgetExceeded`.
#[serde(default = "default_max_cost_per_plan_usd")]
pub max_cost_per_plan_usd: f64,

/// Hard limit on total USD spent by a single task. 0.0 = unlimited.
#[serde(default = "default_max_cost_per_task_usd")]
pub max_cost_per_task_usd: f64,

/// Hard limit on total USD spent in this server-process session. 0.0 = unlimited.
#[serde(default = "default_max_cost_per_session_usd")]
pub max_cost_per_session_usd: f64,
```

fn default_max_cost_per_session_usd() -> f64 { 50.0 }

## Step 2: helpers on the cost layer

In `costs_db.rs`, add (or confirm existence of):

```rust
impl CostsDb {
    /// Cumulative USD spent on the given plan_id this session.
    pub fn plan_cost_usd(&self, plan_id: &str) -> f64 {
        self.query_by_plan(plan_id)
            .iter()
            .map(|r| r.cost_usd)
            .sum()
    }

    /// Cumulative USD spent on the given (plan_id, task_id) this session.
    pub fn task_cost_usd(&self, plan_id: &str, task_id: &str) -> f64 {
        self.query_by_plan(plan_id)
            .iter()
            .filter(|r| r.task_id.as_deref() == Some(task_id))
            .map(|r| r.cost_usd)
            .sum()
    }
}
```

Match the actual `CostRecord` field names (`cost_usd`, `task_id`, etc.). If `query_by_plan` doesn't exist (it should — search), use a private iter on `self.records()`.

If `total_cost` already exists, the *session* total is `costs_db.total_cost()`.

## Step 3: pre-dispatch enforcement in orchestrate

Find the pre-dispatch validation block at `orchestrate.rs:15169–15188`. Insert the budget check immediately AFTER the provider-availability check and AFTER P09's context-overflow check (so the order is: model-availability → context-window → budget).

```rust
// D01: budget enforcement (plan / task / session).
let plan_id = self.current_plan_id();   // existing accessor
let task_id = task.id.as_str();          // assumes `task: &TaskDef` is in scope

let plan_limit    = self.config.agent.max_cost_per_plan_usd;
let task_limit    = self.config.agent.max_cost_per_task_usd;
let session_limit = self.config.agent.max_cost_per_session_usd;

let costs = self.learning.costs_db();
let plan_spent    = costs.plan_cost_usd(plan_id);
let task_spent    = costs.task_cost_usd(plan_id, task_id);
let session_spent = costs.total_cost();

// Estimate cost for THIS call so we trip before, not after.
let est = self.config.cost_table()
    .estimate_cost(prompt_tokens_est, expected_completion_tokens, &selected_model)
    .unwrap_or(0.0);

if plan_limit > 0.0 && plan_spent + est > plan_limit {
    tracing::error!(
        target: "roko::budget",
        plan_id = %plan_id,
        plan_spent,
        plan_limit,
        est,
        "plan budget would be exceeded; skipping task"
    );
    return Err(roko_core::RokoError::BudgetExceeded {
        detail: format!(
            "plan '{plan_id}' has spent ${plan_spent:.4}; this call (~${est:.4}) would exceed limit ${plan_limit:.2}"
        ),
    }.into());
}
if task_limit > 0.0 && task_spent + est > task_limit {
    return Err(roko_core::RokoError::BudgetExceeded {
        detail: format!(
            "task '{plan_id}::{task_id}' has spent ${task_spent:.4}; this call (~${est:.4}) would exceed limit ${task_limit:.2}"
        ),
    }.into());
}
if session_limit > 0.0 && session_spent + est > session_limit {
    return Err(roko_core::RokoError::BudgetExceeded {
        detail: format!(
            "session has spent ${session_spent:.4}; this call (~${est:.4}) would exceed limit ${session_limit:.2}"
        ),
    }.into());
}
```

The variables `plan_id`, `task_id`, `prompt_tokens_est`, `selected_model`, etc., must match what's actually in scope at the insertion point. Do not make up accessors that don't exist; `grep -n 'self.current_plan_id\|fn plan_id\|self.plan_id' crates/roko-cli/src/orchestrate.rs` to find the real getter.

If `cost_table()` isn't a method on `RokoConfig`, look at how `model_call_service.rs:992` accesses pricing — copy that pattern.
