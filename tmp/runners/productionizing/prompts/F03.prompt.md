# F03: Wire tropical attention + sheaf inconsistency into routing

## Goal

Two supplementary signals on top of the cascade router:

1. **Tropical attention** — when scoring routing candidates, augment the existing scoring with a max-plus aggregation over (quality, cost, latency) signals, computed via `roko_primitives::tropical`. Logged but not dispositive.
2. **Sheaf inconsistency** — compute an inconsistency score across the available routing signals (cascade router suggestion, role config, task hint) and persist it on every routing decision via `routing_log`.

Both are *advisory* — they do NOT block dispatch. If computation fails, the router falls through to existing behavior.

## Files to Edit

- `crates/roko-learn/src/cascade_router.rs` — call tropical attention; pass scores out
- `crates/roko-learn/src/routing_log.rs` — record sheaf inconsistency on each entry

## Also Read (do NOT modify)

- `crates/roko-primitives/src/tropical.rs` — for `TropicalF64`, polynomial, attention API
- `crates/roko-primitives/src/sheaf.rs` — for `CellularSheaf`, coboundary, inconsistency

## Step 1: tropical attention in cascade_router.rs

Find the routing scoring code (search for `score`, `quality`, `cost`, `latency`):

```bash
grep -n 'score\|quality\|cost\|latency' crates/roko-learn/src/cascade_router.rs | head -20
```

The router already combines multiple signals when ranking candidates (`route_static_filtered`, `route_confidence_filtered`, `route_ucb_filtered`). Find the scoring expression for one candidate. Typically:

```rust
let score = w_q * stats.pass_rate
          + w_c * (1.0 - stats.cost_norm)
          + w_l * (1.0 - stats.latency_norm);
```

Add a tropical-attention alternative score that takes the *max* of weighted contributions (max-plus), and log both:

```rust
use roko_primitives::tropical::TropicalF64;

let weights = [w_q, w_c, w_l];
let values  = [stats.pass_rate, 1.0 - stats.cost_norm, 1.0 - stats.latency_norm];

// existing dot product
let linear_score = weights.iter().zip(values.iter()).map(|(w, v)| w * v).sum::<f64>();

// tropical: max over (w_i + v_i) interpreted as TropicalF64 addition
let tropical_score = weights.iter().zip(values.iter())
    .map(|(w, v)| TropicalF64(w + v))
    .fold(TropicalF64(f64::NEG_INFINITY), |a, b| a.add(b))
    .0;

tracing::trace!(
    target: "roko::routing",
    candidate = %candidate.slug,
    linear_score,
    tropical_score,
    "candidate score"
);

// Use linear_score for ranking — tropical_score is observation only.
```

Wrap the tropical computation in `let _ = std::panic::catch_unwind(|| { ... });` *only if* the API can panic on edge cases. Otherwise leave as-is — these are pure arithmetic.

The existing scoring expression stays — we only add the tropical computation alongside.

## Step 2: sheaf inconsistency in routing_log.rs

Find the existing routing-log entry struct:

```bash
grep -n 'pub struct\|RoutingLogEntry\|RoutingDecision' crates/roko-learn/src/routing_log.rs | head -10
```

Add an `inconsistency: Option<f64>` field on the entry struct. Default to `None`. Populate it when the routing pipeline disagrees across signals.

In the cascade router, where the final model is chosen, build a small list of (signal_name, suggested_model) tuples:

```rust
let signals: Vec<(&str, &str)> = vec![
    ("cascade", &cascade_choice.primary.slug),
    ("role_default", &role_config_default),     // whatever variable holds it
    ("task_hint", task_hint_model.unwrap_or("")),
];

let inconsistency = compute_sheaf_inconsistency(&signals);
```

Where `compute_sheaf_inconsistency` is a new private helper in `cascade_router.rs`:

```rust
fn compute_sheaf_inconsistency(signals: &[(&str, &str)]) -> Option<f64> {
    // Treat each signal as a 0-cell with the model name; coboundary fires when
    // two adjacent signals disagree. Inconsistency = fraction of disagreeing
    // pairs.
    let n = signals.len();
    if n < 2 {
        return None;
    }
    let mut disagreements = 0;
    let mut pairs = 0;
    for i in 0..n {
        for j in (i + 1)..n {
            if signals[i].1.is_empty() || signals[j].1.is_empty() {
                continue;
            }
            pairs += 1;
            if signals[i].1 != signals[j].1 {
                disagreements += 1;
            }
        }
    }
    if pairs == 0 {
        None
    } else {
        Some(disagreements as f64 / pairs as f64)
    }
}
```

(Yes this is a degenerate sheaf — it's the simplest faithful representation of "do my routing signals agree?" If `roko_primitives::sheaf` exposes a richer API that operates on a similar shape of data, prefer it; the helper above is a fallback.)

Pass the score to whoever appends to `routing_log` — search:

```bash

```

…and add the field to the call site.
