# F05: Implement `NoveltyArchive` (HDC fingerprints) + wire into curriculum

## Goal

Discover novel task types via HDC fingerprint similarity. The archive stores fingerprints of all previously *successfully completed* tasks; new tasks are scored as `1.0 - max_similarity` to existing archive entries. The curriculum scheduler (in Adaptive or Adas mode) biases task selection toward novelty when configured.

Depends on F01 (the curriculum-scheduler hooks for advisory signals).

## Step 1: write `crates/roko-learn/src/novelty_search.rs`

```rust
//! Novelty search via HDC fingerprint archive.
//!
//! Persists the fingerprint of every successfully completed task. New tasks
//! are scored against the archive: novelty = 1.0 - max_similarity.
//!
//! - Lookup is O(N) over archive size; bounded by `max_entries` (default 10k).
//! - Persistence is binary (`bincode`) for size + speed; format is versioned.
//! - Opt-in via `learning.novelty_search = true` in roko.toml.

use serde::{Deserialize, Serialize};
use std::collections::VecDeque;
use std::path::Path;

/// On-disk format version. Bump on incompatible changes.
const FORMAT_VERSION: u32 = 1;

/// Storage shape on disk. We deliberately do NOT store raw task text — only
/// fingerprints and a small label for diagnostics.
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct ArchiveEntry {
    pub label: String,
    /// Hypervector serialized as raw bytes.
    pub fingerprint: Vec<u64>,
    pub added_at_ms: i64,
}

/// Bounded archive of HDC fingerprints.
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct NoveltyArchive {
    version: u32,
    max_entries: usize,
    entries: VecDeque<ArchiveEntry>,
}

impl NoveltyArchive {
    pub fn new(max_entries: usize) -> Self {
        Self {
            version: FORMAT_VERSION,
            max_entries,
            entries: VecDeque::new(),
        }
    }

    pub fn len(&self) -> usize {
        self.entries.len()
    }

    pub fn is_empty(&self) -> bool {
        self.entries.is_empty()
    }

    /// Add a fingerprint. Evicts oldest when capacity is reached.
    pub fn add(&mut self, label: impl Into<String>, fingerprint: Vec<u64>) {
        let entry = ArchiveEntry {
            label: label.into(),
            fingerprint,
            added_at_ms: chrono::Utc::now().timestamp_millis(),
        };
        if self.entries.len() >= self.max_entries {
            self.entries.pop_front();
        }
        self.entries.push_back(entry);
    }

    /// Compute novelty score in `[0.0, 1.0]`.
    ///
    /// Returns 1.0 when the archive is empty (everything is novel until we
    /// see something).
    pub fn novelty(&self, fingerprint: &[u64]) -> f64 {
        if self.entries.is_empty() {
            return 1.0;
        }
        let mut max_sim = 0.0_f64;
        for e in &self.entries {
            let sim = hamming_similarity(&e.fingerprint, fingerprint);
            if sim > max_sim {
                max_sim = sim;
            }
        }
        (1.0 - max_sim).clamp(0.0, 1.0)
    }

    // ── Persistence (bincode for compact binary). ──────────────────────────

    pub fn save(&self, path: &Path) -> std::io::Result<()> {
        let bytes = bincode::serialize(self).map_err(std::io::Error::other)?;
        roko_fs::atomic::write_atomic(path, &bytes)
    }

    pub fn load_or_default(path: &Path, max_entries: usize) -> Self {
        match std::fs::read(path) {
            Ok(bytes) => match bincode::deserialize::<Self>(&bytes) {
                Ok(a) if a.version == FORMAT_VERSION => a,
                _ => Self::new(max_entries),
            },
            Err(_) => Self::new(max_entries),
        }
    }
}

/// Hamming similarity in `[0.0, 1.0]` over packed u64 vectors of equal length.
///
/// If the vectors differ in length, the prefix overlap is used.
fn hamming_similarity(a: &[u64], b: &[u64]) -> f64 {
    let n = a.len().min(b.len());
    if n == 0 {
        return 0.0;
    }
    let total_bits = (n as u64) * 64;
    let differing: u32 = a
        .iter()
        .zip(b.iter())
        .take(n)
        .map(|(x, y)| (x ^ y).count_ones())
        .sum();
    1.0 - (differing as f64 / total_bits as f64)
}

#[cfg(test)]
mod tests {
    use super::*;

    fn fp(seed: u64) -> Vec<u64> {
        (0..8).map(|i| seed.wrapping_mul(i + 1)).collect()
    }

    #[test]
    fn empty_archive_returns_full_novelty() {
        let a = NoveltyArchive::new(100);
        assert_eq!(a.novelty(&fp(1)), 1.0);
    }

    #[test]
    fn identical_fingerprint_has_zero_novelty() {
        let mut a = NoveltyArchive::new(100);
        a.add("seed", fp(42));
        assert_eq!(a.novelty(&fp(42)), 0.0);
    }

    #[test]
    fn different_fingerprint_is_novel() {
        let mut a = NoveltyArchive::new(100);
        a.add("seed", fp(42));
        let n = a.novelty(&fp(7));
        assert!(n > 0.0 && n <= 1.0, "expected (0, 1], got {n}");
    }

    #[test]
    fn evicts_oldest_on_overflow() {
        let mut a = NoveltyArchive::new(2);
        a.add("a", fp(1));
        a.add("b", fp(2));
        a.add("c", fp(3));
        assert_eq!(a.len(), 2);
        // "a" should be evicted; archive holds "b" and "c"
        let labels: Vec<_> = a.entries.iter().map(|e| e.label.clone()).collect();
        assert_eq!(labels, vec!["b", "c"]);
    }
}
```

```rust
impl CurriculumScheduler {
    pub fn with_novelty(mut self, archive: crate::novelty_search::NoveltyArchive, weight: f64) -> Self {
        self.novelty = Some(archive);
        self.novelty_weight = weight.clamp(0.0, 1.0);
        self
    }

    /// After a task succeeds, register its fingerprint in the archive.
    pub fn record_success_fingerprint(&mut self, label: &str, fp: Vec<u64>) {
        if let Some(ref mut a) = self.novelty {
            a.add(label, fp);
        }
    }
}
```

The exact arithmetic depends on existing scoring; if scores are in `[0, 1]` already, additive is fine. If they're in a different range, scale.

UCB1 already balances exploration vs exploitation in `bandits.rs`. Reuse — do NOT add a second exploration mechanism. The simplest sane choice: `score += novelty_weight * (1.0 - exploitation_pressure) * novelty`, where `exploitation_pressure` already exists in the curriculum state.
