# F06: Add `parallel_subtasks` field + wire `MultiAgentPool` fan-out in DAG node

## Goal

Allow a single DAG task to fan out into N parallel agents within the wave, merging the N results via existing `MergeStrategy`. The orchestrator currently dispatches one agent per task; this batch turns a `parallel_subtasks > 1` task into N parallel dispatches against the existing `MultiAgentPool`.

## Files to Edit

- `crates/roko-core/src/agent.rs` (or whichever file declares `TaskDef`) — add the field
- `crates/roko-cli/src/orchestrate.rs` — fan out + merge

## Also Read (do NOT modify)

- `crates/roko-agent/src/multi_pool.rs` — `MultiAgentPool` API
- `crates/roko-agent/src/composition.rs` — `MergeStrategy { Concatenate, Aggregate, Vote, BestOfN }`

## Step 2: orchestrate fan-out

Find the per-task dispatch site. Search:

```bash
grep -n 'fn dispatch_agent_with\|fn execute_task\|self.agent_pool' crates/roko-cli/src/orchestrate.rs | head -20
```

The single-agent dispatch path is something like:

```rust
async fn dispatch_agent_with(&mut self, task: &TaskDef, ...) -> Result<TaskOutcome> {
    // selected_model, dispatch_params, agent_pool.acquire_*, run, return outcome
}
```

Wrap it. Pseudocode:

```rust
async fn dispatch_task(&mut self, task: &TaskDef, ...) -> Result<TaskOutcome> {
    let n = u8::min(task.parallel_subtasks.max(1), self.config.agent.max_parallel as u8) as usize;
    if n <= 1 {
        return self.dispatch_agent_with(task, ...).await;
    }

    let strategy = task.merge_strategy
        .map(translate_merge_strategy)
        .unwrap_or_else(|| default_strategy_for_role(task.role));

    // Spawn N parallel dispatches against the MultiAgentPool.
    let futs = (0..n).map(|i| {
        let mut sub_task = task.clone();
        // Tag the prompt for the i-th sub so caches and logs are distinguishable
        sub_task.id = format!("{}#sub{}", task.id, i);
        self.dispatch_agent_with(&sub_task, ...) // still a future, do not await
    });

    let outcomes: Vec<TaskOutcome> = futures::future::join_all(futs).await
        .into_iter()
        .collect::<Result<Vec<_>>>()?;

    Ok(merge_outcomes(outcomes, strategy))
}
```

You'll need:

```rust
use roko_agent::composition::MergeStrategy;

fn translate_merge_strategy(s: roko_core::agent::TaskMergeStrategy) -> MergeStrategy {
    use roko_core::agent::TaskMergeStrategy as T;
    match s {
        T::BestOfN => MergeStrategy::BestOfN,
        T::Concatenate => MergeStrategy::Concatenate,
        T::Aggregate => MergeStrategy::Aggregate,
        T::Vote => MergeStrategy::Vote,
    }
}

fn default_strategy_for_role(role: Role) -> MergeStrategy {
    match role {
        Role::Implementer | Role::Patch => MergeStrategy::BestOfN,
        Role::Researcher => MergeStrategy::Concatenate,
        Role::Reviewer => MergeStrategy::Vote,
        _ => MergeStrategy::BestOfN,
    }
}
```

(Match the actual `Role` variant names used by `orchestrate.rs`.)

`merge_outcomes` should already exist if `composition.rs` provides one — search:

```bash
grep -n 'pub fn merge\|fn merge_outcomes\|MergeStrategy' crates/roko-agent/src/composition.rs
```

If a public merge function exists, use it. If not, inline the merge here:
- `BestOfN` → pick the outcome with the highest gate score
- `Concatenate` → concat the texts, sum tokens
- `Vote` → majority verdict; on tie, highest gate score
- `Aggregate` → role-specific aggregation, fallback to `Concatenate`

## Step 3: gates run per-subtask, merge runs after

Each subtask result goes through gates *independently* (existing pipeline). Only the post-gate outcomes are merged. Do not change the gate pipeline.

## Step 4: cap on `agent.max_parallel`

The config already has an `agent.max_parallel` (or similar) field — search:

```bash
grep -n 'max_parallel\|max_concurrent' crates/roko-core/src/config/agent.rs crates/roko-core/src/config/schema.rs
```

The dispatch must cap N at this value. If the user sets `parallel_subtasks = 100` with `max_parallel = 4`, dispatch 4.

## Step 5: warm pool reuse

`MultiAgentPool` already supports warm reuse. Fan-out reads N agents from the pool. If fewer than N are warm, the pool spawns new ones. Don't bypass the pool.
