# F07: Wire `CollusionDetector` into marketplace assignments (advisory)

## Goal

After every agent assignment in `crates/roko-chain/src/marketplace.rs`, feed the assignment edge into `CollusionDetector::record_assignment`. Periodically (or on-demand) run `detect()`. When a ring is detected, log a warning and apply the existing `CollisionFeedbackDilution` reputation penalty from `reputation_registry.rs`.

This is **observation only**. It does not block assignments.

## Files to Edit

- `crates/roko-chain/src/marketplace.rs` (only)

## Also Read (do NOT modify)

- `crates/roko-chain/src/collusion.rs` — `AssignmentEdge`, `CollusionConfig`, `CollusionDetector::{new, record_assignment, assignment_count, detect}`
- `crates/roko-chain/src/reputation_registry.rs` — `CollisionFeedbackDilution` penalty values

## Step 2: hook the assignment site

Find the function where an agent gets assigned to a task — the `assign`, `hire`, or similar method. Pseudo-code for the hook:

```rust
pub fn assign(&mut self, client: AgentId, agent: AgentId, score: f64) -> Result<...> {
    // ... existing assignment logic ...
    let assignment = /* the existing variable holding the result */;

    // Record for collusion detection.
    self.collusion.record_assignment(AssignmentEdge {
        client,
        agent,
        weight: score,
        // match actual fields
    });

    // Check periodically (not every assignment — it's O(V^2) at worst).
    if self.collusion.assignment_count() % self.collusion_check_every == 0 {
        self.run_collusion_check_async();
    }

    Ok(assignment)
}
```

(If `assign` is not async, do not change it. The check itself should be cheap; we'll synchronously call detect with `tracing::warn!` on hit.)

```rust
fn run_collusion_check_async(&self) {
    let report = self.collusion.detect();
    if report.rings.is_empty() {
        return;
    }
    for ring in &report.rings {
        tracing::warn!(
            target: "roko::marketplace::collusion",
            ring_size = ring.size(), // or however it exposes participants
            members = ?ring.members(), // or .participants()
            "collusion ring detected; applying reputation dilution"
        );
        // Apply existing penalty for each member.
        if let Some(ref mut rep) = self.reputation {
            for member in ring.members() {
                rep.apply_dilution(member, CollisionFeedbackDilution::default());
            }
        }
    }
}
```

The exact penalty API depends on what `reputation_registry.rs` exposes. Read it before writing the call. If `apply_dilution` doesn't exist but `record_dilution_event` or `record_collision` does, use that. Do NOT redefine the penalty values.

## Step 3: optional — make the check async

If marketplace operations sit on a hot path, run `detect()` on a `tokio::task::spawn_blocking` and log from the spawned task. For this batch, synchronous detection is fine — the detector is in-memory and small (assignment counts cap in the thousands).

## Step 4: do NOT add chain integration

This batch must NOT make any blockchain calls. The collusion check is in-memory. Phase 2 will anchor results to the chain — out of scope here.

## Step 5: do NOT block assignments

Even if a ring is detected on the *current* assignment, the assignment proceeds. Penalties dampen *future* feedback weight; they do not retroactively undo the assignment.

## Acceptance Criteria

- [ ] `Marketplace` carries a `CollusionDetector` initialized with `CollusionConfig::default()` (or equivalent)
- [ ] `record_assignment` called on every successful assignment with the right edge fields
- [ ] `detect()` runs at most every N assignments (default N=50)
- [ ] Detected rings emit a `tracing::warn!` with member ids
- [ ] Each ring member is penalized via the existing `CollisionFeedbackDilution` API
- [ ] No `unwrap`/`expect` introduced
- [ ] No new chain calls
- [ ] No blocking on assignment

## Do NOT

- Do not run collusion detection on every assignment — bound it.
- Do not add a runtime dependency on `roko-chain` from `roko-cli`.
- Do not redefine penalty values — read them from the existing registry.
- Do not block assignments on collusion checks — advisory only.
- Do not run `cargo *`.
