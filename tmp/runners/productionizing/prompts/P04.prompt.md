## How to access `default_model`

The struct has a `self.config: RokoConfig` field. Access via:

```rust
let default_model = self.config.agent.default_model.clone();
```

Look at line 15169 (the existing pre-dispatch validation block) for the canonical access pattern.

For free functions (not methods), accept a `&str` parameter or accept `&AgentConfig` if the call site has it.

## Step-by-step

### 1. Confirm grep

```bash

```

If the line numbers have shifted, use the *new* line numbers — the strings are unique enough to match by context. Only replace occurrences that are NOT inside `#[cfg(test)]` or `mod tests`.

### 2. Replace each non-test occurrence

For 6 of the 7 production sites, the pattern is the same: a string literal. The replacement is `self.config.agent.default_model.clone()` (in methods) or `config.agent.default_model.clone()` (when `config` is a parameter).

Example BEFORE (line 10354 area):
```rust
let fallback = override_model
    .clone()
    .unwrap_or_else(|| "claude-sonnet-4-6".into());
```
AFTER:
```rust
let fallback = override_model
    .clone()
    .unwrap_or_else(|| self.config.agent.default_model.clone());
```

### 3. The escalation array (line 14232)

```rust
let escalation_models = ["claude-haiku-4-5", "claude-sonnet-4-6", "claude-opus-4-6"];
```

This is a haiku → sonnet → opus tier chain. Without a config-driven equivalent, leave it but mark for follow-up:

```rust
// TODO(P15): move escalation tiers to `[agent.escalation]` config table.
// All three models must already be in `config.models`; the cascade router
// + provider availability filter will skip any whose provider lacks credentials.

```

DO NOT replace with `default_model` — the whole point of the array is escalation.

### 4. The `resolve_enrichment_backend` call (line 20377)



If `self` is not available, find the nearest config in scope (search for `config:` or `roko_config:` in the function signature) and use it.
