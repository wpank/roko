## Step 1: create `crates/roko-fs/src/flock.rs`

```rust
//! Cross-process advisory file locking.
//!
//! Uses `flock(2)` on Unix; falls back to a no-op + warning on non-Unix.
//! The lock is keyed by `{path}.lock` to avoid touching the data file's metadata.

use std::fs::{File, OpenOptions};
use std::io;
use std::path::{Path, PathBuf};

/// Guard returned by [`exclusive`]. Releases on drop.
pub struct FileLockGuard {
    _file: File,
    lock_path: PathBuf,
}

impl Drop for FileLockGuard {
    fn drop(&mut self) {
        // The OS releases the lock when the file descriptor closes; the .lock
        // sidecar can stay (small, deterministic name) or be removed. We keep
        // it to avoid races with new lockers between our remove and their open.
        // If you want to clean it up, do so on a separate sweep.
        let _ = &self.lock_path; // silence unused
    }
}

/// Acquire an exclusive cross-process lock for `path`.
///
/// Blocks until the lock is available. The lock file is `{path}.lock`.
///
/// On non-Unix targets, returns a no-op guard with a single `tracing::warn!`
/// (process-level state writers should still serialize via tokio mutexes).
pub fn exclusive(path: &Path) -> io::Result<FileLockGuard> {
    let lock_path = lock_path_for(path);
    if let Some(parent) = lock_path.parent() {
        std::fs::create_dir_all(parent)?;
    }
    let file = OpenOptions::new()
        .create(true)
        .write(true)
        .read(true)
        .open(&lock_path)?;

    #[cfg(unix)]
    {
        use std::os::unix::io::AsRawFd;
        // SAFETY: file owns the fd for its lifetime; we hold `file` in the
        // guard so the fd outlives every `flock` call.
        let ret = unsafe { libc::flock(file.as_raw_fd(), libc::LOCK_EX) };
        if ret != 0 {
            return Err(io::Error::last_os_error());
        }
    }

    #[cfg(not(unix))]
    {
        tracing::warn!(
            path = %path.display(),
            "exclusive file lock requested on non-Unix target; \
             cross-process safety NOT enforced"
        );
    }

    Ok(FileLockGuard {
        _file: file,
        lock_path,
    })
}

/// Try to acquire the lock without blocking.
///
/// Returns `Ok(None)` when another process holds it; `Ok(Some(guard))` on
/// success. Errors are real I/O errors.
pub fn try_exclusive(path: &Path) -> io::Result<Option<FileLockGuard>> {
    let lock_path = lock_path_for(path);
    if let Some(parent) = lock_path.parent() {
        std::fs::create_dir_all(parent)?;
    }
    let file = OpenOptions::new()
        .create(true)
        .write(true)
        .read(true)
        .open(&lock_path)?;

    #[cfg(unix)]
    {
        use std::os::unix::io::AsRawFd;
        let ret = unsafe { libc::flock(file.as_raw_fd(), libc::LOCK_EX | libc::LOCK_NB) };
        if ret != 0 {
            let err = io::Error::last_os_error();
            if err.raw_os_error() == Some(libc::EWOULDBLOCK) {
                return Ok(None);
            }
            return Err(err);
        }
    }

    Ok(Some(FileLockGuard {
        _file: file,
        lock_path,
    }))
}

fn lock_path_for(path: &Path) -> PathBuf {
    let mut s = path.as_os_str().to_owned();
    s.push(".lock");
    PathBuf::from(s)
}

#[cfg(test)]
mod tests {
    use super::*;
    use tempfile::tempdir;

    #[test]
    fn exclusive_lock_serializes_threads() {
        let dir = tempdir().unwrap();

        let _g1 = exclusive(&path).unwrap();
        // try_exclusive in the same process succeeds because flock is
        // per-process — but we still get a sidecar file:
        assert!(path.with_extension("jsonl.lock").exists());
    }
}
```

## Step 4: `crates/roko-learn/src/episode_logger.rs`

Find `pub async fn append(&self, episode: &Episode) -> Result<(), LoggerError>` (line ~964).

Wrap the `OpenOptions::new()...append(true).open(...)` block in a `spawn_blocking` that holds the flock for the duration of the write.

BEFORE:
```rust
pub async fn append(&self, episode: &Episode) -> Result<(), LoggerError> {
    // ... existing logic that opens self.path with append=true and writes ...
    let mut file = OpenOptions::new()
        .create(true)
        .append(true)
        .open(&self.path)
        .await?;
    file.write_all(&line).await?;
    file.flush().await?;
    Ok(())
}
```

AFTER:
```rust
pub async fn append(&self, episode: &Episode) -> Result<(), LoggerError> {
    // ... existing logic that prepares `line: Vec<u8>` ...

    let path = self.path.clone();
    let line = line; // already a Vec<u8> from serialization above

    tokio::task::spawn_blocking(move || -> std::io::Result<()> {
        let _lock = roko_fs::flock::exclusive(&path)?;
        let mut file = std::fs::OpenOptions::new()
            .create(true)
            .append(true)
            .open(&path)?;
        std::io::Write::write_all(&mut file, &line)?;
        file.sync_data()?;
        Ok(())
    })
    .await
    .map_err(|join_err| LoggerError::from(std::io::Error::other(join_err)))??;

    Ok(())
}
```

The `LoggerError::from(std::io::Error::...)` part assumes `LoggerError: From<std::io::Error>`. If it doesn't, use whatever variant exists — check the existing `?` usage at the top of the function.

If `roko-fs` is not already in `roko-learn/Cargo.toml`'s `[dependencies]`, add it (workspace dep).
