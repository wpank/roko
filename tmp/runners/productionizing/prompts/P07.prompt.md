# P07: HTTP timeout layer + 503 on degraded `/api/health`

## Goal

Two changes to `roko-serve`:

1. Add `tower_http::timeout::TimeoutLayer(Duration::from_secs(30))` to the top-level router so hung handlers don't accumulate.
2. Make `/api/health` (the rich one) return `503 Service Unavailable` when the embedded status is `down`, while keeping the response body shape unchanged.

`/health` (top-level liveness probe at `routes/mod.rs:193`) MUST stay 200-always. Don't touch it.

## Files to Edit

- `crates/roko-serve/src/routes/mod.rs` (router construction)
- `crates/roko-serve/src/routes/status/health.rs` (rich health handler)
- `crates/roko-serve/Cargo.toml` (add `timeout` feature if missing)

## Also Read (do NOT modify)

- `crates/roko-serve/src/routes/status/mod.rs` — exports for the status submodule
- `crates/roko-serve/src/routes/mod.rs:170–195` — current router shape

## Step 1: Cargo.toml

Open `crates/roko-serve/Cargo.toml`. Find the `tower-http` dependency. Confirm `timeout` is in the feature list. The current spec is:

```toml
tower-http = { workspace = true, features = ["...some-set..."] }
```

If `timeout` is missing, add it. Don't remove existing features. Example after edit:

```toml
tower-http = { workspace = true, features = ["limit", "cors", "trace", "timeout"] }
```

(Match whatever features were already enabled — just append `"timeout"` if not present.)

## Step 2: routes/mod.rs

Open `crates/roko-serve/src/routes/mod.rs`. Around line 60, the imports include:

```rust
use tower_http::limit::RequestBodyLimitLayer;
```

Add (alphabetical, near `limit::`):

```rust
use tower_http::timeout::TimeoutLayer;
```

Around line 183 the router build chain looks like:

```rust
router
    .layer(RequestBodyLimitLayer::new(32 * 1024 * 1024))
    .layer(TraceLayer::new_for_http())
    .layer(cors)
    .with_state(state)
```

Insert a `TimeoutLayer` *before* `with_state(state)`:

```rust
router
    .layer(RequestBodyLimitLayer::new(32 * 1024 * 1024))
    .layer(TimeoutLayer::new(std::time::Duration::from_secs(30)))
    .layer(TraceLayer::new_for_http())
    .layer(cors)
    .with_state(state)
```

This applies a 30-second deadline to every route. Streaming routes (SSE, WebSocket) are wrapped before this layer is hit, so they're unaffected; the same holds for the upgrade-handled WS routes that use `axum::extract::ws::WebSocketUpgrade` (the upgrade completes well within 30s, then the protocol takes over).

If you find any HTTP route that legitimately runs longer than 30s (e.g. a synchronous `cargo test` proxy), exempt it via `.route_layer(<a longer TimeoutLayer>)` on that specific route. Do not raise the global limit. Search:

```bash
grep -rn 'route("/api/' crates/roko-serve/src/routes/ | grep -v '\.rs:.*#' | head -20
```

The known long ones are `routes/plans/run.rs` (plan execution) and `routes/bench/*.rs` (bench runs). If those are top-level routes (not nested behind streaming), wrap them with a 600s `route_layer`. If they're SSE-based, no change needed.

## Step 3: routes/status/health.rs

Open `crates/roko-serve/src/routes/status/health.rs`. Find the handler (typically `pub async fn health(...) -> Json<HealthResponse>` or similar).

Locate the place where the status string is computed. The existing logic produces a struct with at least a `status: String` (or enum) field with values like `"ok"` / `"degraded"` / `"down"`.

Change the return type to `(StatusCode, Json<HealthResponse>)`:

BEFORE:
```rust
pub async fn health(State(state): State<Arc<AppState>>) -> Json<HealthResponse> {
    let body = compute_health(state).await;
    Json(body)
}
```

AFTER:
```rust
pub async fn health(State(state): State<Arc<AppState>>) -> (StatusCode, Json<HealthResponse>) {
    let body = compute_health(state).await;
    let code = match body.status.as_str() {
        "ok" | "degraded" => StatusCode::OK,
        _ => StatusCode::SERVICE_UNAVAILABLE,
    };
    (code, Json(body))
}
```

If `status` is an enum, match on the enum variants instead of the string:

```rust
let code = match body.status {
    HealthStatus::Ok | HealthStatus::Degraded => StatusCode::OK,
    HealthStatus::Down => StatusCode::SERVICE_UNAVAILABLE,
};
```

Add the import if missing:

```rust
use axum::http::StatusCode;
```

The body shape MUST stay the same; only the HTTP status code changes when degraded → down.

## Step 4: do NOT touch top-level /health

Open `crates/roko-serve/src/routes/mod.rs` around line 193:

```rust
async fn top_level_health() -> Json<Value> {
    Json(json!({"status": "ok"}))

```

Leave it. This is the unconditional liveness probe Railway hits (`railway.toml: healthcheckPath = "/health"`). Changing it would break deploys.
