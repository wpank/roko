## Goal

Replace `let _ = ...` and `.ok()` with logged error-handling at the **security-critical and bootstrap-critical** sites identified in `tmp/productionizing/06-AUDIT-FINDINGS.md` H3. Do not try to fix every silent swallow in the workspace â€” fix only the named lines.

## Files to Edit

## Site 1 â€” `crates/roko-serve/src/jwks.rs:145`

This is **security-critical**: a silent JWKS refresh failure means JWT validation continues with stale keys.

The current code:

```rust
if needs_refresh {
    let _ = self.refresh_jwks().await;
}
```

Replace with:

```rust
if needs_refresh {
    if let Err(err) = self.refresh_jwks().await {
        tracing::error!(
            error = %err,
            "JWKS refresh failed â€” JWT validation may use stale keys"
        );
    }
}
```

Use `tracing::error!` (not `warn!`) because this is a security-impacting failure.

## Site 2 â€” `crates/roko-serve/src/lib.rs` bootstrap

Run this grep to locate:

```bash
grep -n 'let _ = state\.state_hub\.bootstrap_from_workdir\|state_hub\.bootstrap' crates/roko-serve/src/lib.rs
```

The audit pinned this at line ~830. The current call is:

```rust
let _ = state.state_hub.bootstrap_from_workdir(&state.workdir);
```

Replace with:

```rust
if let Err(err) = state.state_hub.bootstrap_from_workdir(&state.workdir) {
    tracing::warn!(
        error = %err,
        workdir = %state.workdir.display(),
        "StateHub bootstrap failed; starting with empty state"
    );
}
```

Use `warn!` because the system can recover from an empty StateHub (it just loses prior history).

## Site 4 â€” `crates/roko-serve/src/terminal.rs` (5+ ops)

```bash
grep -n 'let _ = ' crates/roko-serve/src/terminal.rs
```

For each `let _ = ...` that wraps a `Result`-returning call (resize, write, kill, close, etc.), wrap with logging:

BEFORE:
```rust
let _ = state.terminal_sessions.resize(&id, cols, rows);
```

AFTER:
```rust
if let Err(err) = state.terminal_sessions.resize(&id, cols, rows) {
    tracing::warn!(
        error = %err,
        session_id = %id,
        cols, rows,
        "terminal resize failed"
    );
}
```

Pattern variations:

| Original op | Suggested log message |
|---|---|
| `resize` | `terminal resize failed` |
| `write_input` / `write` | `terminal write failed` |
| `kill` / `close` | `terminal close failed` |
| `set_size` | `terminal set_size failed` |
| `notify`/`broadcast` | `terminal event broadcast failed` |

Match the existing parameter names. If `id`, `session_id`, `pid`, etc. are in scope, include them as structured fields.

**Skip** any `let _ = tx.send(...)` where `tx` is a `tokio::sync::broadcast::Sender` and a no-receivers error is intentional. Add a comment instead:

```rust
// best-effort: receivers may have hung up
let _ = tx.send(event);
```

## What NOT to change

- Skip `let _ = ...` inside `Drop` impls (cleanup paths).
- Skip `let _ = std::fs::remove_file(...)` cleanup of `*.tmp` / `*.lock` files.
- Skip `let _ = sender.send(...)` where the no-receivers case is OK (broadcast channels often have ephemeral subscribers). But add a one-line comment.
- Skip occurrences inside `#[cfg(test)]` blocks.
- Do not touch `*.ok()` chained on `Option` (only `Result`).
