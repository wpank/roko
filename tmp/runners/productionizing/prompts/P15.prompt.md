# P15: Create `roko.production.toml` (anthropic, openai, perplexity, gemini, ollama only)

## Goal

Create a `roko.production.toml` next to the existing `roko.toml`. This file is what Railway and other production deploys load via `ROKO_CONFIG=roko.production.toml`. It must:

1. Have **only** providers that have API keys in our env (anthropic, openai, perplexity, gemini, ollama).
2. Strip the providers we never had keys for (cerebras, zhipu, openrouter, moonshot, zai).
3. Strip every `[models.*]` entry that points at a stripped provider.
4. Set `serve.auth.enabled = true`.
5. Reference API keys via `api_key_env`, never inline.

## Files to Edit

- `roko.production.toml` (NEW file at repo root)

## Also Read (do NOT modify)

- `roko.toml` — the source of truth for the schema
- `tmp/productionizing/05-ROKO-TOML-PRODUCTION.md` — the spec
- `crates/roko-core/src/config/schema.rs` — the canonical struct

## Step 1: copy roko.toml as a starting point

```bash
cp roko.toml roko.production.toml
```

(You'll edit it in place after.)

## Step 2: strip stripped providers

Remove every section whose header is one of:

- `[providers.cerebras]`
- `[providers.zhipu]`
- `[providers.openrouter]`
- `[providers.moonshot]`
- `[providers.zai]`

The header line plus all key/value lines until the next `[...]` header. For example, this block goes:

```toml
[providers.cerebras]
kind = "openai_compat"
base_url = "https://api.cerebras.ai/v1"
api_key_env = "CEREBRAS_API_KEY"
# ... (a few more lines) ...
```

Repeat for all five.

## Step 3: strip models that reference stripped providers

Search for `[models.*]` blocks whose `provider = "<stripped>"`. Common ones:

- Anything mentioning `qwen`, `glm`, `deepseek` via cerebras/zhipu
- `kimi`, `moonshot-*` via moonshot
- Anything `openrouter/*` proxied via openrouter

Use the spec doc as the canonical list, but verify against `roko.toml` because the source can drift:

```bash
grep -n 'provider = "cerebras"\|provider = "zhipu"\|provider = "openrouter"\|provider = "moonshot"\|provider = "zai"' roko.toml
```

For each match, find the enclosing `[models.<key>]` header (search backwards for the nearest `[models.`) and remove the entire model block.

## Step 4: keep these providers

These five must remain:

- `[providers.anthropic]` (`api_key_env = "ANTHROPIC_API_KEY"` — confirm)
- `[providers.openai]`    (`api_key_env = "OPENAI_API_KEY"`)
- `[providers.perplexity]`(`api_key_env = "PERPLEXITY_API_KEY"`)
- `[providers.gemini]`    (`api_key_env = "GEMINI_API_KEY"`)
- `[providers.ollama]`    (`api_key_env = ""` — local, no key needed)

If `[providers.anthropic]` currently has `kind = "claude_cli"` + `command = "claude"`, set it to:

```toml
[providers.anthropic]
kind = "anthropic_api"
api_key_env = "ANTHROPIC_API_KEY"
```

(Drop the `command = "claude"` line — production runs the API backend, not the CLI.)

If a separate `[providers.claude_cli]` block exists, **delete it** for the production config. Production has no `claude` binary in the container.

## Step 5: keep models that reference kept providers

After stripping, verify every `[models.*]` block's `provider = ` field references one of the five kept providers. If not, remove the model.

The spec doc lists the canonical kept set:

| Provider | Models to keep |
|---|---|
| anthropic | claude-opus-4-6, claude-sonnet-4-6, claude-haiku-4-5 (and any `claude-sonnet-4-5` variants that exist) |
| openai    | gpt-5.4, gpt-5-mini, o3, o4-mini |
| perplexity| sonar-deep-research, sonar-reasoning-pro |
| gemini    | gemini-2.5-pro, gemini-2.5-flash |
| ollama    | (whatever is configured locally — keep the entries that reference `provider = "ollama"`) |

Do NOT add models that aren't in the source `roko.toml` — only filter.

## Step 6: serve.auth and bind

Find `[serve]` (and `[serve.auth]`). Set:

```toml
[serve]
acknowledge_public_risk = false  # require auth on public binds

[serve.auth]
enabled = true
# Privy app id is auto-set by lib.rs; keep this section minimal.
```

`bind` (under `[server]` or `[serve]`) should be `"0.0.0.0"` for production (the Docker container binds it; the `validate_bind_safety` check only allows it because `auth.enabled = true`).

## Step 7: agent.default_model

`[agent].default_model` must reference a model that is still present after the strip. If `roko.toml` had `default_model = "kimi-k2"` (or anything via stripped providers), change it to:

```toml
[agent]
default_model = "claude-sonnet-4-6"
```

## Step 8: routing config

`[routing]` (if present) must only reference kept models. Same rule as `default_model`.

## Step 9: gc, learning, etc.

These sections are configuration without API surface. Keep them as-is.

## Verification

```bash
ls roko.production.toml                                                       # exists

# Stripped providers gone:
! grep -E '^\[providers\.(cerebras|zhipu|openrouter|moonshot|zai)\]' roko.production.toml

# Stripped models gone (no model points at a stripped provider):
! grep -E 'provider = "(cerebras|zhipu|openrouter|moonshot|zai)"' roko.production.toml

# Auth enabled:
grep -nE '^\[serve\.auth\]|^enabled' roko.production.toml | head -3

# default_model still resolves:
grep -nE '^default_model' roko.production.toml

# claude_cli provider gone:
! grep -E '^\[providers\.claude_cli\]' roko.production.toml

# Anthropic uses API backend:
grep -A 2 '^\[providers\.anthropic\]$' roko.production.toml | grep 'kind = "anthropic_api"'
```

All `!`-prefixed lines must produce no output (returning non-zero from grep).
