# P16: Add `Dockerfile.runtime` (binary-only image for fast deploys)

## Goal

Create the Tier-1 Dockerfile (`Dockerfile.runtime`). It does NOT contain a Rust toolchain — it copies a pre-built binary from `target/x86_64-unknown-linux-gnu/release/roko` (built locally with `cargo zigbuild`) and a pre-built frontend from `demo/demo-app/dist/`.

Result: ~5-second Docker build.

## Files to Edit

- `Dockerfile.runtime` (NEW file at repo root)
- `.dockerignore` (NEW or amended at repo root — add an entry that allows the cross-compiled binary path)

## Also Read (do NOT modify)

- `Dockerfile` (root) — for current runtime layers
- `tmp/productionizing/08-FAST-BUILD-DEPLOY.md`

## Step 1: write Dockerfile.runtime

```dockerfile
# syntax=docker/dockerfile:1.7
#
# Dockerfile.runtime — Tier-1 build (no Rust toolchain in image).
#
# Prereqs (one-time on the build host):
#   brew install zig            # macOS; or `apt install zig` on Linux
#   cargo install cargo-zigbuild
#   rustup target add x86_64-unknown-linux-gnu
#
# Build pipeline (typical use via deploy.sh from P17):
#   (cd demo/demo-app && npm run build)
#   cargo zigbuild --release --target x86_64-unknown-linux-gnu --bin roko
#   docker build -f Dockerfile.runtime -t ghcr.io/nunchi-trade/roko:latest .
#
# Image is ~80-120 MB and builds in ~5 seconds.

FROM debian:bookworm-slim

RUN apt-get update \
    && apt-get install -y --no-install-recommends \
        ca-certificates libssl3 git curl coreutils bash \
    && rm -rf /var/lib/apt/lists/*

# Cross-compiled binary (pre-built outside Docker).
COPY target/x86_64-unknown-linux-gnu/release/roko /usr/local/bin/roko

# Pre-built SPA assets.
COPY demo/demo-app/dist/ /usr/share/roko/spa/

# Production config (from P15). Falls back to roko.toml if production isn't there yet.
COPY roko.production.toml /workspace/roko.toml

# Minimal terminal prompt.
RUN echo 'export PS1="> "' >> /root/.bashrc

VOLUME ["/workspace/.roko"]
WORKDIR /workspace

ENV RUST_LOG=info
ENV SHELL=/bin/bash
ENV ROKO_SPA_DIR=/usr/share/roko/spa
EXPOSE 6677

ENTRYPOINT ["/usr/local/bin/roko"]
CMD ["serve", "--bind", "0.0.0.0", "--port", "6677"]
```

If `roko.production.toml` doesn't exist yet (P15 hasn't merged), the `COPY roko.production.toml ...` line will fail the build. To make this batch independent of P15, change that line to:

```dockerfile
# Use roko.production.toml if present, otherwise fall back to roko.toml.
COPY roko*.toml /workspace/_configs/
RUN if [ -f /workspace/_configs/roko.production.toml ]; then \
        mv /workspace/_configs/roko.production.toml /workspace/roko.toml; \
    else \
        mv /workspace/_configs/roko.toml /workspace/roko.toml; \
    fi && rm -rf /workspace/_configs
```

That way deploys before P15 lands still work.

## Step 2: amend .dockerignore

The existing `.dockerignore` may include `target/` to keep `Dockerfile` small. `Dockerfile.runtime` *needs* `target/x86_64-unknown-linux-gnu/release/roko` to be copyable.

Strategy: keep `target` excluded by default and explicitly un-exclude the runtime-build artifact.

Read `.dockerignore`. If it has:

```
target
```

…replace that with:

```
target
!target/x86_64-unknown-linux-gnu/release/roko
```

If `.dockerignore` doesn't exist, create it:

```dockerignore
# Default exclusions
.git
.github
.idea
.vscode
.roko
node_modules
**/node_modules
*.log
.DS_Store

# Cargo target — exclude everything except the cross-compiled release binary,
# which Dockerfile.runtime needs.
target
!target/x86_64-unknown-linux-gnu/release/roko

# tmp: working scratch
tmp

# demo-app build artifacts
demo/demo-app/.parcel-cache
demo/demo-app/dist
!demo/demo-app/dist/
```

Note: we explicitly **un-exclude** `demo/demo-app/dist/` because `Dockerfile.runtime` copies it. Without this, the build context will lack the SPA assets.

## Step 3: do not break the existing Dockerfile

The existing `Dockerfile` does its own multi-stage build inside the container. `.dockerignore` changes here must not exclude anything that `Dockerfile` (the unoptimized one) needs:

- `crates/`, `Cargo.toml`, `Cargo.lock`, `contracts/`, `roko.toml`, `demo/demo-app/` (sources, not just dist)

If your `.dockerignore` excludes any of these, the existing build will break. Read the file twice.

## Verification

```bash
ls Dockerfile.runtime                                # exists
ls .dockerignore                                     # exists

# Toolchain absent:
! grep -E 'rust:|cargo|rustup' Dockerfile.runtime

# Binary copy in:
grep -n 'target/x86_64-unknown-linux-gnu/release/roko' Dockerfile.runtime

# SPA copy in:
grep -n 'demo/demo-app/dist' Dockerfile.runtime

# Runtime contract matches:
diff <(grep -E 'EXPOSE|VOLUME|CMD|ENTRYPOINT' Dockerfile) \
     <(grep -E 'EXPOSE|VOLUME|CMD|ENTRYPOINT' Dockerfile.runtime)
# Expect: identical
```
