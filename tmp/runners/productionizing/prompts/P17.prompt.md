# P17: Add `deploy.sh` convenience script

## Goal

Create `deploy.sh` at the repo root that automates: build SPA → cross-compile binary → build runtime image → push → deploy. Targets railway/fly/local. Depends on `Dockerfile.runtime` from P16.

## Files to Edit

- `deploy.sh` (NEW file at repo root)

## Also Read (do NOT modify)

- `Dockerfile.runtime` (from P16) — for the build context expectations
- `tmp/productionizing/08-FAST-BUILD-DEPLOY.md`

## Step 1: write deploy.sh

```bash
#!/usr/bin/env bash
#
# deploy.sh — fast-deploy pipeline.
#
# Usage:
#   ./deploy.sh                # default: railway, latest tag
#   ./deploy.sh railway        # explicit
#   ./deploy.sh fly            # fly.io
#   ./deploy.sh local          # docker run locally for smoke testing
#   ./deploy.sh railway v1.2.3 # tagged release
#
# Prereqs:
#   - cargo install cargo-zigbuild
#   - rustup target add x86_64-unknown-linux-gnu
#   - brew install zig            # or apt install zig
#   - docker, railway / flyctl as appropriate
#
# Env vars:
#   - REGISTRY  (default: ghcr.io/nunchi-trade/roko)
#   - PLATFORM  (default: linux/amd64)
#   - SKIP_PUSH (set to 1 to build but not push)

set -euo pipefail

TARGET="${1:-railway}"
TAG="${2:-latest}"
REGISTRY="${REGISTRY:-ghcr.io/nunchi-trade/roko}"
PLATFORM="${PLATFORM:-linux/amd64}"
SKIP_PUSH="${SKIP_PUSH:-0}"

cd "$(dirname "$0")"

log() { printf '\033[1;36m==>\033[0m %s\n' "$*"; }

# ─── 1. Frontend ─────────────────────────────────────────────────────────────
log "Building frontend (demo-app)…"
(cd demo/demo-app && npm ci --prefer-offline && npm run build)

# ─── 2. Cross-compile Rust binary ────────────────────────────────────────────
log "Cross-compiling roko for x86_64-unknown-linux-gnu…"
if ! command -v cargo-zigbuild >/dev/null 2>&1; then
    echo "error: cargo-zigbuild not installed. Run:"
    echo "  cargo install cargo-zigbuild"
    echo "  rustup target add x86_64-unknown-linux-gnu"
    echo "  brew install zig   # or: apt install zig"
    exit 1
fi
cargo zigbuild --release --target x86_64-unknown-linux-gnu --bin roko

# Sanity check.
test -x target/x86_64-unknown-linux-gnu/release/roko \
    || { echo "error: cross-compiled binary not found"; exit 1; }

# ─── 3. Docker build (Tier 1 — Dockerfile.runtime) ───────────────────────────
log "Building docker image $REGISTRY:$TAG (platform=$PLATFORM)…"

DOCKER_BUILD_ARGS=(
    "buildx" "build"
    "-f" "Dockerfile.runtime"
    "-t" "$REGISTRY:$TAG"
    "--platform" "$PLATFORM"
)

if [ "$SKIP_PUSH" = "1" ] || [ "$TARGET" = "local" ]; then
    DOCKER_BUILD_ARGS+=("--load")
else
    DOCKER_BUILD_ARGS+=("--push")
fi

docker "${DOCKER_BUILD_ARGS[@]}" .

# ─── 4. Deploy ───────────────────────────────────────────────────────────────
case "$TARGET" in
  railway)
    log "Deploying to Railway from $REGISTRY:$TAG"
    railway up --image "$REGISTRY:$TAG"
    ;;
  fly)
    log "Deploying to Fly from $REGISTRY:$TAG"
    fly deploy --image "$REGISTRY:$TAG"
    ;;
  local)
    log "Running locally on http://localhost:6677"
    docker run --rm \
      -p 6677:6677 \
      -e ANTHROPIC_API_KEY="${ANTHROPIC_API_KEY:-}" \
      -e OPENAI_API_KEY="${OPENAI_API_KEY:-}" \
      -e PERPLEXITY_API_KEY="${PERPLEXITY_API_KEY:-}" \
      -e GEMINI_API_KEY="${GEMINI_API_KEY:-}" \
      -v roko-state:/workspace/.roko \
      "$REGISTRY:$TAG"
    ;;
  *)
    echo "error: unknown target '$TARGET'"
    echo "Usage: $0 [railway|fly|local] [tag]"
    exit 1
    ;;
esac

log "Done."
```

## Step 2: chmod +x

The runner cannot run `chmod`, so add a comment at the top reminding the user, and also commit the file with the executable bit set if your tooling allows. If the runner can't set the bit, the user will run `chmod +x deploy.sh` once.

(In the worktree, `chmod +x deploy.sh` should be the last command before adding to git. The runner's commit pipeline preserves the bit.)

## Step 3: handle missing tools gracefully

The script must:

1. Fail fast on missing `cargo-zigbuild` (already does — see step 2 of the script).
2. Fail fast on missing `docker` (the `docker buildx` call will fail with a clear error; that's OK).
3. Fail fast on missing `railway` / `flyctl` only at the deploy step, after the image is built (so the user can re-run with `SKIP_PUSH=1` and try a different deploy target).

Don't add additional checks — the existing `set -euo pipefail` plus the `command -v` check is enough.
