# AG02: InsightBoardReader in roko-chain (read-only via alloy::sol!)

## Goal

Add a read-only client for the `InsightBoard` contract to
`crates/roko-chain`. AG05 will plug this into the aggregator's
knowledge endpoints.

## Context

The contract spec is at `contracts/src/InsightBoard.sol`. Today
`roko-chain` has no reader for it. The pattern to mirror is
`crates/roko-chain/src/marketplace.rs`.

## Changes Required

1. Create `crates/roko-chain/src/insight_board.rs`:

            string  uri;

/// Stable JSON-friendly view of an on-chain insight.
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct InsightView {
    pub id: String,
    pub poster: String,
    pub content_hash: String,
    pub uri: String,
    pub posted_at: u64,
    pub pheromone: u64,
}

impl From<(U256, IInsightBoard::Insight)> for InsightView {
    fn from((id, raw): (U256, IInsightBoard::Insight)) -> Self {
        Self {
            id: format!("{id:#x}"),
            poster: format!("{:#x}", raw.poster),
            content_hash: format!("{:#x}", raw.contentHash),
            uri: raw.uri,
            posted_at: raw.postedAt,
            pheromone: raw.pheromone,
        }
    }
}

        let mut out = Vec::with_capacity(n as usize);

## Acceptance Criteria

- [ ] `crates/roko-chain/src/insight_board.rs` exists and follows the
  shape above.
- [ ] `cargo build -p roko-chain` succeeds.
- [ ] `cargo test -p roko-chain --lib insight_board` passes.
- [ ] Public types `InsightBoardReader`, `InsightView`,
  `InsightEvent` are re-exported from `roko-chain`.
- [ ] No `unimplemented!()` remains by the time the batch lands; the
  scaffold must be filled in using the marketplace.rs primitives.

## Do NOT

- Do not use a different ABI library than `alloy_sol_types`. The
  workspace standardised on it.
- Do not introduce a new error type. Use `anyhow::Result` (matching
  marketplace.rs style) — typed errors are a separate refactor.
- Do not fetch contract state via raw `reqwest` to JSON-RPC. Go
  through the existing `ChainClientBase` (or whatever marketplace.rs
  uses).
- Do not export internal types (e.g. the alloy-generated `Insight`
  struct) at the crate root. Only export the JSON-friendly `View`
  shape.
