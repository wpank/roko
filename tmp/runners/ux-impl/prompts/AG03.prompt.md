# AG03: AgentCardFetcher (HTTP / IPFS / data-URI) in roko-chain

## Goal

Add `AgentCardFetcher` that resolves any `agentCardUri` returned by
`IdentityRegistry` and produces a `roko_agent_server::AgentCard`.

## Context

ERC-8004 stores a `cardUri` per passport. The URI can be:

- `http://...` / `https://...` — fetch via `reqwest`.
- `ipfs://<cid>` — translate to a configured gateway URL.
- `data:application/json;base64,<b64>` — decode locally.

Used by AG04 (`KnowledgeSource`) and CH05 (chain-discovery merge).

## Changes Required

1. Create `crates/roko-chain/src/agent_card_fetcher.rs`:

```rust
//! Resolve any ERC-8004 agentCardUri to an AgentCard JSON payload.

use std::time::Duration;

use anyhow::{Context, Result, anyhow, bail};
use base64::Engine;
use reqwest::Client;
use serde::{Deserialize, Serialize};

const MAX_CARD_BYTES: usize = 64 * 1024;
const FETCH_TIMEOUT: Duration = Duration::from_secs(5);

/// JSON shape compatible with `roko_agent_server::registration::AgentCard`.
/// Re-declared locally to avoid a `roko-chain → roko-agent-server` dependency
/// edge.
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct AgentCardPayload {
    pub name: String,
    #[serde(default)]
    pub capabilities: Vec<String>,
    pub endpoints: serde_json::Value,
    #[serde(default)]
    pub domain_tags: Vec<String>,
    #[serde(default = "default_card_version")]
    pub version: String,
}

fn default_card_version() -> String { "1.0".into() }

pub struct AgentCardFetcher {
    http: Client,
    ipfs_gateway: String,
}

impl Default for AgentCardFetcher {
    fn default() -> Self { Self::new() }
}

impl AgentCardFetcher {
    pub fn new() -> Self {
        let http = Client::builder()
            .timeout(FETCH_TIMEOUT)
            .build()
            .expect("reqwest client builds");
        Self {
            http,
            ipfs_gateway: "https://ipfs.io/ipfs/".into(),
        }
    }

    pub fn with_ipfs_gateway(mut self, gateway: impl Into<String>) -> Self {
        self.ipfs_gateway = gateway.into();
        self
    }

    /// Fetch and decode an AgentCard for the given URI.
    ///
    /// # Errors
    ///
    /// Returns an error if the URI scheme is unsupported, the fetch
    /// times out, the response exceeds `MAX_CARD_BYTES`, or the body
    /// is not valid JSON in the expected shape.
    pub async fn fetch(&self, uri: &str) -> Result<AgentCardPayload> {
        let body = match scheme(uri) {
            Some("http" | "https") => self.fetch_http(uri).await?,
            Some("ipfs") => self.fetch_ipfs(uri).await?,
            Some("data") => decode_data_uri(uri)?,
            other => bail!("unsupported AgentCard URI scheme: {:?}", other),
        };
        let card: AgentCardPayload =
            serde_json::from_slice(&body).context("decode AgentCard JSON")?;
        validate(&card)?;
        Ok(card)
    }

    async fn fetch_http(&self, uri: &str) -> Result<Vec<u8>> {
        let res = self.http.get(uri).send().await?;
        ensure_ok(&res)?;
        body_bytes_capped(res).await
    }

    async fn fetch_ipfs(&self, uri: &str) -> Result<Vec<u8>> {
        let cid = uri
            .strip_prefix("ipfs://")
            .ok_or_else(|| anyhow!("malformed ipfs URI: {uri}"))?;
        let resolved = format!("{}{cid}", self.ipfs_gateway.trim_end_matches('/').to_string() + "/");
        let res = self.http.get(&resolved).send().await?;
        ensure_ok(&res)?;
        body_bytes_capped(res).await
    }
}

fn scheme(uri: &str) -> Option<&str> {
    uri.split_once(':').map(|(s, _)| s)
}

fn ensure_ok(res: &reqwest::Response) -> Result<()> {
    if !res.status().is_success() {
        bail!("AgentCard fetch returned status {}", res.status());
    }
    Ok(())
}

async fn body_bytes_capped(res: reqwest::Response) -> Result<Vec<u8>> {
    let bytes = res.bytes().await?;
    if bytes.len() > MAX_CARD_BYTES {
        bail!(
            "AgentCard exceeds {} byte cap (got {})",
            MAX_CARD_BYTES,
            bytes.len()
        );
    }
    Ok(bytes.to_vec())
}

fn decode_data_uri(uri: &str) -> Result<Vec<u8>> {
    let payload = uri
        .strip_prefix("data:application/json;base64,")
        .ok_or_else(|| anyhow!("data URI must be application/json;base64: {uri}"))?;
    let decoded = base64::engine::general_purpose::STANDARD_NO_PAD
        .decode(payload)
        .or_else(|_| base64::engine::general_purpose::STANDARD.decode(payload))
        .context("decode data URI base64")?;
    if decoded.len() > MAX_CARD_BYTES {
        bail!("AgentCard data URI exceeds {} bytes", MAX_CARD_BYTES);
    }
    Ok(decoded)
}

fn validate(card: &AgentCardPayload) -> Result<()> {
    if card.name.is_empty() {
        bail!("AgentCard.name must be non-empty");
    }
    if !card.endpoints.is_object() {
        bail!("AgentCard.endpoints must be a JSON object");
    }
    Ok(())
}

#[cfg(test)]
mod tests {
    use super::*;

    fn fixture_card_b64() -> String {
        let card = serde_json::json!({
            "name": "test-agent",
            "capabilities": ["messaging"],
            "endpoints": {"rest": "http://localhost:9100"},
            "domain_tags": ["roko"],
            "version": "1.0"
        });
        base64::engine::general_purpose::STANDARD_NO_PAD
            .encode(serde_json::to_vec(&card).unwrap())
    }

    #[tokio::test]
    async fn data_uri_decodes() {
        let uri = format!("data:application/json;base64,{}", fixture_card_b64());
        let card = AgentCardFetcher::new().fetch(&uri).await.unwrap();
        assert_eq!(card.name, "test-agent");
    }

    #[tokio::test]
    async fn unsupported_scheme_errors() {
        let err = AgentCardFetcher::new()
            .fetch("ftp://example.com/card.json")
            .await
            .unwrap_err();
        assert!(format!("{err}").contains("unsupported"));
    }
}
```

## Acceptance Criteria

- [ ] `crates/roko-chain/src/agent_card_fetcher.rs` matches the shape above.
- [ ] `cargo build -p roko-chain` succeeds.
- [ ] `cargo test -p roko-chain --lib agent_card_fetcher` passes (data URI + unsupported scheme).
- [ ] `AgentCardFetcher` and `AgentCardPayload` are re-exported from `roko-chain`.

## Do NOT

- Do not depend on `roko-agent-server`. The fetcher must be usable
  from layers below the agent server. Re-declare the shape locally if
  needed.
- Do not raise `MAX_CARD_BYTES`. 64 KB is generous; larger means
  malicious inputs can exhaust memory.
- Do not bake the IPFS gateway URL into a constant. Operators
  override it via `AgentCardFetcher::with_ipfs_gateway`.
