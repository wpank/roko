# AG04: KnowledgeSource enum on AppState

## Goal

Add a `KnowledgeSource` enum to `crates/roko-serve/src/state.rs` with
three variants: `Local`, `Chain`, `Hybrid`. Wire its construction into
`AppState::new` (or whatever the existing constructor path is).

## Changes Required

1. Add to `crates/roko-serve/src/state.rs`:

```rust
use std::sync::Arc;

use roko_chain::{InsightBoardReader, AgentCardFetcher};
use roko_neuro::knowledge_store::KnowledgeStore;

/// Where the aggregator pulls knowledge entries from.
#[derive(Clone)]
pub enum KnowledgeSource {
    /// File-backed local cache only (test/dev fallback).
    Local(Arc<KnowledgeStore>),
    /// On-chain InsightBoard only (rare; typically used for read-only nodes).
    Chain(Arc<InsightBoardReader>, Arc<AgentCardFetcher>),
    /// Production: chain is source-of-truth, local cache absorbs reads.
    Hybrid {
        chain: Arc<InsightBoardReader>,
        cache: Arc<KnowledgeStore>,
        fetcher: Arc<AgentCardFetcher>,
    },
}

impl KnowledgeSource {
    pub fn local(store: Arc<KnowledgeStore>) -> Self { Self::Local(store) }
    pub fn chain(reader: Arc<InsightBoardReader>, fetcher: Arc<AgentCardFetcher>) -> Self {
        Self::Chain(reader, fetcher)
    }
    pub fn hybrid(
        chain: Arc<InsightBoardReader>,
        cache: Arc<KnowledgeStore>,
        fetcher: Arc<AgentCardFetcher>,
    ) -> Self {
        Self::Hybrid { chain, cache, fetcher }
    }
}
```

2. Add a field on `AppState`:

   ```rust
   pub knowledge_source: KnowledgeSource,
   ```

   (Place near the other domain-state fields, alongside whatever
   already exists for layout / process_supervisor.)

3. In whichever constructor builds `AppState`, decide the variant:

   ```rust
   let knowledge_source = match (
       config.chain.insight_board_address.as_ref(),
       config.chain.rpc_url.as_ref(),
   ) {
       (Some(addr), Some(rpc)) => {
           let provider = build_chain_provider(rpc).await?;  // existing helper or new
           let reader = Arc::new(InsightBoardReader::new(addr.parse()?, provider));
           let fetcher = Arc::new(AgentCardFetcher::new());
           let local_cache = Arc::new(KnowledgeStore::for_layout(&layout));
           KnowledgeSource::hybrid(reader, local_cache, fetcher)
       }
       _ => {
           let store = Arc::new(KnowledgeStore::for_layout(&layout));
           KnowledgeSource::local(store)
       }
   };
   ```

   If the config schema has no `chain.insight_board_address` field
   yet, leave a `// TODO(CH09): config-driven chain wiring` comment
   and default to `Local` for now. AG04 must compile without
   contract config; CH09 will refine.

4. In `crates/roko-serve/Cargo.toml`, ensure `roko-chain` and
   `roko-neuro` are dependencies. They almost certainly already are;
   add if missing.

## Acceptance Criteria

- [ ] `KnowledgeSource` enum present in `state.rs` with the three variants.
- [ ] `AppState` has a `knowledge_source` field.
- [ ] `AppState::new` (or equivalent constructor) populates it.
- [ ] `cargo build -p roko-serve` succeeds.
- [ ] `cargo test -p roko-serve` passes (AG05/AG09 add the new tests; existing ones must not break).
- [ ] When chain config is absent, the variant is `Local`. When
  present, the variant is `Hybrid` (or `Chain` if cache is disabled).

## Do NOT

- Do not import internal `alloy_*` types into `state.rs`. The
  `InsightBoardReader` and `AgentCardFetcher` types are the contract.
- Do not add a fourth variant for "neither chain nor local". The enum
  must always have a usable backend; the caller picks which.
- Do not put state mutation behind a `Mutex`. The variants are
  `Arc`-shared and read-only at the aggregator level.
