# AG07: Add `/api/pheromones/*` routes to aggregator

## Goal

Mount six pheromone routes on `crates/roko-serve/src/routes/aggregator.rs`
that read from `state.pheromone_field` and produce responses matching
the captured fixtures.

## Routes (exact shape)

| Route | Method | Body | Auth |
|-------|--------|------|------|
| `/api/pheromones` | GET | none | public |
| `/api/pheromones/{kind}` | GET | none | public |
| `/api/pheromones/topology` | GET | none | public |
| `/api/pheromones/active` | GET | none | public |
| `/api/pheromones/decay` | POST | `{ "kind": "<optional>" }` | bearer |
| `/api/pheromones/decay/{kind}` | GET | none | public |

Response shapes match
`tmp/runners/ux-impl/fixtures/pheromones*.json`. Diff at runtime
(AG09).

## Changes Required

1. Add the routes block in `aggregator.rs::routes()`:

   ```rust
   pub fn routes() -> Router<Arc<AppState>> {
       Router::new()
           // ... existing routes ...
           .route("/pheromones", get(list_pheromones))
           .route("/pheromones/active", get(list_active_pheromones))
           .route("/pheromones/topology", get(pheromone_topology))
           .route("/pheromones/{kind}", get(list_pheromones_by_kind))
           .route("/pheromones/decay", post(trigger_decay))
           .route("/pheromones/decay/{kind}", get(decay_schedule))
           // ... existing routes ...
   }
   ```

   Place `/{kind}` *after* `/active` and `/topology` so axum's
   matcher prefers the more specific routes.

2. Implement each handler:

   ```rust
   async fn list_pheromones(
       State(state): State<Arc<AppState>>,
   ) -> Result<Json<Value>, ApiError> {
       let samples = state.pheromone_field.list_all();
       Ok(Json(serialize_pheromone_list(samples)))
   }

   async fn list_active_pheromones(
       State(state): State<Arc<AppState>>,
   ) -> Result<Json<Value>, ApiError> {
       let samples = state.pheromone_field.list_active();
       Ok(Json(serialize_pheromone_list(samples)))
   }

   async fn pheromone_topology(
       State(state): State<Arc<AppState>>,
   ) -> Result<Json<Value>, ApiError> {
       Ok(Json(state.pheromone_field.topology()))
   }

   async fn list_pheromones_by_kind(
       State(state): State<Arc<AppState>>,
       Path(kind_str): Path<String>,
   ) -> Result<Json<Value>, ApiError> {
       let kind = roko_serve::pheromone::PheromoneKind::parse(&kind_str)
           .ok_or_else(|| ApiError::bad_request(format!("unknown pheromone kind: {kind_str}")))?;
       let samples = state.pheromone_field.list_by_kind(kind);
       Ok(Json(serialize_pheromone_list(samples)))
   }

   #[derive(Debug, serde::Deserialize)]
   struct DecayRequest { kind: Option<String> }

   async fn trigger_decay(
       State(state): State<Arc<AppState>>,
       Json(req): Json<DecayRequest>,
   ) -> Result<Json<Value>, ApiError> {
       // Body is intentionally unused for now; bearer-auth makes this
       // a debug-only knob. We could later honour `req.kind` to decay
       // only that kind.
       let _ = req;
       state.pheromone_field.decay_tick();
       Ok(Json(serde_json::json!({ "status": "ok" })))
   }

   async fn decay_schedule(
       Path(kind_str): Path<String>,
   ) -> Result<Json<Value>, ApiError> {
       let kind = roko_serve::pheromone::PheromoneKind::parse(&kind_str)
           .ok_or_else(|| ApiError::bad_request(format!("unknown pheromone kind: {kind_str}")))?;
       Ok(Json(serde_json::json!({
           "kind": kind.as_str(),
           "default_half_life_seconds": kind.half_life().as_secs(),
       })))
   }

   fn serialize_pheromone_list(samples: Vec<roko_serve::pheromone::PheromoneSample>) -> Value {
       serde_json::json!({
           "items": samples.iter().map(|s| serde_json::json!({
               "agent_id": s.agent_id,
               "kind": s.kind.as_str(),
               "intensity": s.intensity,
               "last_update": s.last_update.to_rfc3339(),
           })).collect::<Vec<_>>(),
           "total": samples.len(),
       })
   }
   ```

3. Bearer-auth enforcement on `POST /api/pheromones/decay`: rely on
   the existing middleware in `crates/roko-serve/src/routes/middleware.rs`
   that already gates write paths. If the routes module mounts public
   GETs and bearer-protected POSTs separately, place this route in
   the protected sub-router.

4. Validate that the response shape matches
   `tmp/runners/ux-impl/fixtures/pheromones.json` field-by-field.
   Likely keys: `items`, `total`, plus per-row `agent_id`, `kind`,
   `intensity`, `last_update`. If the captured fixture uses
   `samples` instead of `items`, match the fixture (it's the
   contract).

## Acceptance Criteria

- [ ] All six routes in `routes()`.
- [ ] `cargo build -p roko-serve` succeeds.
- [ ] `cargo test -p roko-serve` passes.
- [ ] After `state.pheromone_field.deposit(...)`, `GET /api/pheromones`
  returns the deposited sample.
- [ ] `GET /api/pheromones/threat` filters by kind.
- [ ] `GET /api/pheromones/badkind` returns 400 with `unknown pheromone kind`.
- [ ] `POST /api/pheromones/decay` without a bearer header returns 401
  (or whatever the existing middleware sends).
- [ ] Response shape matches `tmp/runners/ux-impl/fixtures/pheromones.json`.

## Do NOT

- Do not return `200 + empty array` for an unknown kind. That hides bugs.
- Do not skip bearer auth on `/decay` because "it's only debug".
- Do not duplicate `serialize_pheromone_list` if a similar helper
  already exists.
- Do not add the routes outside the aggregator module. They live next
  to `/api/agents`, `/api/predictions`, etc. for discoverability.
