# AG08: Subscribe to InsightBoard events and invalidate aggregator cache

## Goal

When an `InsightPosted` or `InsightConfirmed` event lands, invalidate
every cached `aggregator:knowledge:*` entry within ~1 s.

## Changes Required

1. In `crates/roko-serve/src/state.rs`, expose a cache-invalidation
   helper if not already present:

   ```rust
   impl AppState {
       /// Invalidate every cache entry whose key starts with `prefix`.
       pub async fn invalidate_cache_prefix(&self, prefix: &str) {
           // Drop matching keys from the moka / dashmap / whatever the
           // cache is. If the cache lib doesn't support prefix scans,
           // record the prefix in a tombstone set checked at read.
       }
   }
   ```

2. In `crates/roko-serve/src/lib.rs::serve(...)` startup, after
   `AppState` and `decay_loop` setup:

   ```rust
   if let KnowledgeSource::Chain(reader, _)
        | KnowledgeSource::Hybrid { chain: reader, .. }
        = &state.knowledge_source
   {
       let reader = Arc::clone(reader);
       let st = Arc::clone(&state);
       tokio::spawn(async move {
           let mut events = reader.subscribe_events();
           while let Some(event) = events.recv().await {
               tracing::debug!(?event, "InsightBoard event; invalidating knowledge cache");
               st.invalidate_cache_prefix("aggregator:knowledge:").await;
           }
           tracing::warn!("InsightBoard event subscription ended; falling back to TTL only");
       });
   }
   ```

3. If the cache layer in `state.rs` does not currently support prefix
   invalidation, add a minimal version:

   ```rust
   pub async fn invalidate_cache_prefix(&self, prefix: &str) {
       let to_remove: Vec<String> = self
           .json_cache
           .iter()
           .map(|kv| kv.key().clone())
           .filter(|k| k.starts_with(prefix))
           .collect();
       for k in to_remove {
           self.json_cache.invalidate(&k).await;
       }
   }
   ```

   Adapt to whichever cache library is in use. If it's `moka`, the
   above is correct. If it's a raw `DashMap`, replace `.invalidate`
   with `.remove`.

4. Add a smoke test in `crates/roko-serve/tests/aggregator_cache.rs`:

   ```rust
   #[tokio::test]
   async fn invalidate_prefix_drops_matching_keys() {
       let state = test_app_state();
       state.cache_json("aggregator:knowledge:entries", &json!({"k":1}), Duration::from_secs(60)).await;
       state.cache_json("aggregator:agents:list",        &json!({"k":2}), Duration::from_secs(60)).await;

       state.invalidate_cache_prefix("aggregator:knowledge:").await;

       assert!(state.cached_json("aggregator:knowledge:entries").await.is_none());
       assert!(state.cached_json("aggregator:agents:list").await.is_some());
   }
   ```

## Acceptance Criteria

- [ ] `AppState::invalidate_cache_prefix(&str)` exists and is callable from any task.
- [ ] `serve(...)` spawns one subscription task when knowledge source is `Chain` / `Hybrid`.
- [ ] `aggregator_cache.rs` smoke test passes.
- [ ] `cargo build --workspace` succeeds.
- [ ] On a sample posted insight (via cast call against anvil), the
  next aggregator GET returns the new entry within 35 s (well under
  the 30 s TTL fallback).

## Do NOT

- Do not poll `eth_getLogs` if the provider supports `eth_subscribe`.
  Polling is the fallback inside `subscribe_events`, not the default
  here.
- Do not invalidate `aggregator:agents:` on `InsightBoard` events.
  They're orthogonal.
- Do not block startup on the subscription. If it fails, log once and
  continue — the TTL fallback keeps the dashboard usable.
