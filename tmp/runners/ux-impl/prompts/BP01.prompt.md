# BP01: parity_kit.rs test helper module

## Goal

Single helper that drives the canonical 5 scenarios across every
LLM backend: happy path, streaming, tool call, error path,
session continuation. Every BP02-BP07 batch reuses this kit.

## Changes Required

1. Create `crates/roko-agent/tests/_helpers/mod.rs`:

   ```rust
   pub mod parity_kit;
   ```

2. Create `crates/roko-agent/tests/_helpers/parity_kit.rs`:

```rust
//! Cross-backend parity kit. Each `assert_*` function takes a backend
//! constructor closure that yields a fully-configured backend pointed
//! at a mockito server. The kit drives the backend through one
//! deterministic interaction and asserts the post-condition.
//!
//! Usage:
//!
//! ```ignore
//! use crate::_helpers::parity_kit::*;
//!
//! #[tokio::test]
//! async fn happy_path() {
//!     let mock = mockito::Server::new_async().await;
//!     // configure mock for happy_path scenario
//!     let backend = MyBackend::new(mock.url());
//!     assert_happy_path(&backend, "expected response").await;
//! }
//! ```

use std::time::Duration;

use roko_agent::tool_loop::LlmBackend;

pub struct ScenarioResult { pub output: String, pub took: Duration }

/// Drive a single non-streaming `send_turn` and assert the output
/// equals `expected`.
pub async fn assert_happy_path<B: LlmBackend + ?Sized>(
    backend: &B,
    prompt: &str,
    expected: &str,
) {
    let req = roko_core::ModelCallRequest::new(prompt).expect("request");
    let res = backend.send_turn(req).await.expect("send_turn");
    let body = res.text().expect("response text");
    assert_eq!(body.trim(), expected, "happy_path output mismatch");
}

/// Drive `send_turn_streaming` and assert the concatenated chunks
/// equal `expected`. Backends that don't support streaming should
/// `#[ignore]` this scenario with a documented reason.
pub async fn assert_streaming<B: LlmBackend + ?Sized>(
    backend: &B,
    prompt: &str,
    expected: &str,
) {
    let req = roko_core::ModelCallRequest::new(prompt).expect("request");
    let mut rx = backend.send_turn_streaming(req).await.expect("send_turn_streaming");
    let mut chunks = String::new();
    while let Some(chunk) = rx.recv().await {
        chunks.push_str(&chunk);
    }
    assert_eq!(chunks.trim(), expected, "streaming output mismatch");
}

/// Issue a request that should trigger one tool-call frame, then a
/// response. Assert the dispatcher routed the tool call.
pub async fn assert_tool_call<B: LlmBackend + ?Sized>(
    backend: &B,
    tool_name: &str,
) {
    let req = roko_core::ModelCallRequest::new(format!("call tool {tool_name}"))
        .expect("request");
    let res = backend.send_turn(req).await.expect("send_turn");
    let body = res.text().expect("response text");
    assert!(body.contains(tool_name), "tool call not surfaced: {body}");
}

/// Send a request the mock will respond to with HTTP `status`. Assert
/// the backend produces a `DispatchError` (not a panic).
pub async fn assert_error_path<B: LlmBackend + ?Sized>(
    backend: &B,
    status: u16,
) {
    let req = roko_core::ModelCallRequest::new("bad").expect("request");
    let res = backend.send_turn(req).await;
    match res {
        Err(_) => {}
        Ok(r) => panic!("expected error for status {status}, got: {r:?}"),
    }
}

/// Two consecutive `send_turn` calls; the second carries prior context
/// (verified by the mock's session-continuation script).
pub async fn assert_session_continuation<B: LlmBackend + ?Sized>(
    backend: &B,
    first: &str,
    second: &str,
) {
    let r1 = backend.send_turn(
        roko_core::ModelCallRequest::new(first).expect("req1"),
    ).await.expect("turn1");
    let _ = r1;
    let r2 = backend.send_turn(
        roko_core::ModelCallRequest::new(second).expect("req2"),
    ).await.expect("turn2");
    let _ = r2;
}
```

   The exact return types from `send_turn` / `send_turn_streaming`
   come from `crates/roko-agent/src/tool_loop/mod.rs`. Adjust the
   `expect`s in the kit to match the real signatures.

3. The `_helpers` directory must be excluded from cargo's test
   discovery (cargo will not compile it as a test if it doesn't have
   `mod tests` or `#[test]` at the top level — placing it inside a
   `mod _helpers` referenced by other test files is the trick).

## Acceptance Criteria

- [ ] `crates/roko-agent/tests/_helpers/parity_kit.rs` exists.
- [ ] Re-exported via `crates/roko-agent/tests/_helpers/mod.rs`.
- [ ] All five `assert_*` functions compile against the actual
  `LlmBackend` trait.
- [ ] No test in the kit itself fails (the kit has no tests; it's a
  helper).

## Do NOT

- Do not bake mockito setup into the kit. Each backend test owns
  its own mock; the kit only drives the backend.
- Do not include real provider credentials in the kit.
- Do not import anything from a specific backend (e.g.
  `claude_agent::ClaudeAgent`) — the kit must be backend-agnostic.
