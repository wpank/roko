# BP08: ExecAgent / ClaudeCliAgent consolidation (Option A)

## Goal

Consolidate `ExecAgent` and `ClaudeCliAgent` into a single `CliAgent`
with Claude-specific defaults provided as a thin wrapper. Net LOC drop
~400.

## Context

`crates/roko-agent/src/exec.rs` (~600 LOC) and
`crates/roko-agent/src/claude_cli_agent.rs` (~878 LOC) wrap a
JSON-line CLI. Their core logic overlaps. The plan recommends Option A
(consolidate).

## Changes Required

1. Create `crates/roko-agent/src/cli_agent.rs` containing the union
   of mechanics:
   - Spawn process with given binary + args.
   - Send JSON-line requests on stdin; read JSON-line responses on
     stdout.
   - Stream chunks via the `LlmBackend::send_turn_streaming` path.
   - Cancellation via tokio `CancellationToken`.
   - Tool call routing via the dispatcher.
   - Configurable timeout, env vars, cwd.

   Public type:

   ```rust
   pub struct CliAgent {
       config: CliAgentConfig,
       // ...
   }

   #[derive(Clone, Debug)]
   pub struct CliAgentConfig {
       pub binary: PathBuf,
       pub args: Vec<String>,
       pub env: Vec<(String, String)>,
       pub cwd: Option<PathBuf>,
       pub timeout: Duration,
       pub send_format: CliFormat,        // raw stdin JSON / claude-cli specific / etc
       pub recv_format: CliFormat,
   }

   pub enum CliFormat {
       JsonLine,
       ClaudeCliV1,                       // claude --print --json mode
       ExecGeneric,
   }

   impl LlmBackend for CliAgent { ... }
   ```

2. Re-implement `ClaudeCliAgent` as a thin builder around `CliAgent`:

   ```rust
   // crates/roko-agent/src/claude_cli_agent.rs
   use crate::cli_agent::{CliAgent, CliAgentConfig, CliFormat};

   pub struct ClaudeCliAgent { inner: CliAgent }

   impl ClaudeCliAgent {
       pub fn new(binary: impl Into<PathBuf>) -> Self {
           let config = CliAgentConfig {
               binary: binary.into(),
               args: vec!["--print".into(), "--output-format".into(), "stream-json".into(), "--include-partial-messages".into()],
               env: Vec::new(),
               cwd: None,
               timeout: Duration::from_secs(600),
               send_format: CliFormat::ClaudeCliV1,
               recv_format: CliFormat::ClaudeCliV1,
           };
           Self { inner: CliAgent::new(config) }
       }
   }

   impl LlmBackend for ClaudeCliAgent {
       async fn send_turn(&self, req: ModelCallRequest) -> Result<ModelCallResult, DispatchError> {
           self.inner.send_turn(req).await
       }
       async fn send_turn_streaming(&self, req: ModelCallRequest) -> Result<mpsc::Receiver<String>, DispatchError> {
           self.inner.send_turn_streaming(req).await
       }
   }
   ```

3. Re-implement `ExecAgent` as a similarly thin builder:

   ```rust
   // crates/roko-agent/src/exec.rs
   pub struct ExecAgent { inner: CliAgent }

   impl ExecAgent {
       pub fn new(binary: impl Into<PathBuf>, args: Vec<String>) -> Self {
           let config = CliAgentConfig {
               binary: binary.into(),
               args,
               env: Vec::new(),
               cwd: None,
               timeout: Duration::from_secs(600),
               send_format: CliFormat::ExecGeneric,
               recv_format: CliFormat::ExecGeneric,
           };
           Self { inner: CliAgent::new(config) }
       }
   }
   // ... LlmBackend forwards to inner ...
   ```

4. Move tests:
   - The thick logic tests (claude streaming, tool routing) move to
     `crates/roko-agent/tests/cli_agent_*.rs` against `CliAgent`
     directly with both formats.
   - Backend-specific defaults tests stay in `claude_cli_agent.rs` /
     `exec.rs` `mod tests`.

5. In `crates/roko-agent/src/lib.rs`, add `pub mod cli_agent;`.

## Acceptance Criteria

- [ ] `crates/roko-agent/src/cli_agent.rs` is the single source of
  CLI-spawn logic.
- [ ] `claude_cli_agent.rs` and `exec.rs` are thin wrappers (each
  < 100 LOC).
- [ ] `cargo build -p roko-agent` succeeds.
- [ ] `cargo test -p roko-agent` passes including the parity-kit tests
  from BP02.
- [ ] `tokei crates/roko-agent/src` shows the LOC drop is real (~400).

## Do NOT

- Do not introduce the `CliAgent` and leave the old `ExecAgent`/
  `ClaudeCliAgent` thick. The whole point is consolidation; partial
  consolidation is worse than none.
- Do not break the public API. All `ClaudeCliAgent::*` and
  `ExecAgent::*` methods that callers use must continue to work.
- Do not place CLI-format-specific parsing in the wrapper. It belongs
  in `CliAgent` keyed off `CliFormat`.
