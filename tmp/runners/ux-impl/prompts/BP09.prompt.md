# BP09: File-layout cleanup (Gemini, Ollama, Perplexity)

## Goal

For each multi-protocol backend, pick **single file** OR **directory
with mod.rs**, never both.

## Decisions

| Backend | Today | Recommend |
|---------|-------|-----------|
| Gemini | `src/gemini/native.rs` 883 LOC | single file `src/gemini.rs` |
| Ollama | `src/ollama_agent.rs` + `src/ollama_backend.rs` | single file `src/ollama.rs` |
| Perplexity | `src/perplexity/{chat,deep_research,tool_loop,search}.rs` 2 562 LOC total | keep directory; clean `mod.rs` to be the public surface |

## Changes Required

### Gemini

1. Move `crates/roko-agent/src/gemini/native.rs` → `crates/roko-agent/src/gemini.rs`.
2. Update `crates/roko-agent/src/lib.rs`:
   - Remove `pub mod gemini;` (the directory mod) if it only existed
     to re-export `native::*`.
   - Add `pub mod gemini;` (the new file).
3. Search for `roko_agent::gemini::native::` and replace with
   `roko_agent::gemini::`.

### Ollama

1. Read both `ollama_agent.rs` and `ollama_backend.rs`. They likely
   carry overlapping types.
2. Merge into `crates/roko-agent/src/ollama.rs`:
   - The `LlmBackend` impl from `ollama_agent.rs`.
   - The HTTP client + types from `ollama_backend.rs`.
3. Delete the two original files (`git rm`).
4. Update `lib.rs` exports.
5. Update parity test's import in `BP07` test (or whatever path it
   uses).

### Perplexity

1. Keep the `crates/roko-agent/src/perplexity/` directory.
2. Make `crates/roko-agent/src/perplexity/mod.rs` the *only* place
   that re-exports public types. Internal modules
   (`chat`, `deep_research`, `tool_loop`, `search`) become `pub(super)`
   if other crates only need the `mod.rs` surface.
3. Clean up the public surface: confirm what other crates actually
   import (grep `roko_agent::perplexity::`) and re-export only those.

## Acceptance Criteria

- [ ] `crates/roko-agent/src/gemini.rs` exists; `src/gemini/native.rs` does not.
- [ ] `crates/roko-agent/src/ollama.rs` exists; the two old files do not.
- [ ] `crates/roko-agent/src/perplexity/mod.rs` is the public-export
  point; internal modules are not directly re-exported from
  `lib.rs`.
- [ ] `cargo build -p roko-agent --all-features` succeeds.
- [ ] `cargo test -p roko-agent` passes (parity tests from BP05/06/07
  still green).
- [ ] `git grep 'roko_agent::gemini::native::'` returns 0 hits.
- [ ] `git grep 'roko_agent::ollama_(agent|backend)::'` returns 0 hits.

## Do NOT

- Do not change *behaviour* in this batch. Pure refactor.
- Do not reorganise other backends (Claude, Codex, Cursor, OpenAI).
- Do not lose `pub` visibility on a type that has external callers.
  The `git grep` checks catch the easy cases; manual verification
  catches the rest.
