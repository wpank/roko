# BP10: Cascade router integration test against mock dispatcher

## Goal

`crates/roko-learn/tests/cascade_router_integration.rs` boots
`Dispatcher` with two mock backends, runs cascade until one wins,
asserts state at `.roko/learn/cascade-router.json`. Closes followup
items 40a + 60c.

## Changes Required

1. Read the existing mock infrastructure at `crates/roko-std/src/mock.rs`
   (the test references it). Build your test backends using that
   pattern.

2. Create `crates/roko-learn/tests/cascade_router_integration.rs`:

```rust
use std::sync::Arc;

use serde_json::Value;
use tempfile::TempDir;

use roko_learn::cascade_router::CascadeRouter;
use roko_agent::dispatcher::Dispatcher;
use roko_std::mock::MockBackend;

#[tokio::test]
async fn cascade_persists_winner_through_real_trait() {
    let tmp = TempDir::new().unwrap();
    let cascade_path = tmp.path().join("cascade-router.json");

    // Two mock backends with different latency / cost profiles.
    let cheap = MockBackend::new("cheap-model")
        .with_response("answer-cheap")
        .with_latency_ms(10);
    let expensive = MockBackend::new("expensive-model")
        .with_response("answer-expensive")
        .with_latency_ms(200);

    let dispatcher = Dispatcher::new(vec![
        Arc::new(cheap),
        Arc::new(expensive),
    ]);
    let router = CascadeRouter::new(&cascade_path);

    // 10 routing decisions; cheap should win on cost-efficiency.
    for i in 0..10 {
        let prompt = format!("hello {i}");
        let result = router.route(&dispatcher, prompt).await
            .expect("cascade should succeed");
        assert_eq!(result.backend_name(), "cheap-model");
    }

    // Assert state landed on disk.
    let raw = std::fs::read_to_string(&cascade_path).expect("file exists");
    let parsed: Value = serde_json::from_str(&raw).expect("valid JSON");
    assert!(parsed["winner"].as_str() == Some("cheap-model"),
        "winner not persisted: {parsed}");
    assert!(parsed["arms"].as_array().is_some(),
        "arm matrix missing");
}

#[tokio::test]
async fn cascade_falls_back_to_expensive_on_cheap_failure() {
    let tmp = TempDir::new().unwrap();
    let cascade_path = tmp.path().join("cascade-router.json");

    let failing = MockBackend::new("cheap-model")
        .with_error("upstream busy");
    let working = MockBackend::new("expensive-model")
        .with_response("answer-expensive");

    let dispatcher = Dispatcher::new(vec![
        Arc::new(failing),
        Arc::new(working),
    ]);
    let router = CascadeRouter::new(&cascade_path);

    let result = router.route(&dispatcher, "hello").await
        .expect("cascade should fall back");
    assert_eq!(result.backend_name(), "expensive-model");
}
```

3. If `MockBackend` doesn't expose `with_latency_ms` / `with_error` /
   `with_response`, extend it in `crates/roko-std/src/mock.rs`. These
   are general-purpose test helpers.

4. The `Dispatcher::new` and `CascadeRouter::new` signatures may differ
   slightly — adjust the test to match the actual public API.

## Acceptance Criteria

- [ ] `cargo test -p roko-learn --test cascade_router_integration` passes.
- [ ] Test runs in < 5 s.
- [ ] No real network calls.
- [ ] `cascade-router.json` is created during the test under a
  `TempDir` and cleaned up automatically.

## Do NOT

- Do not run the test against real backends. Mocks only.
- Do not assert specific arm-matrix values; assert the *shape* (winner
  string, arms array). Algorithm tweaks shouldn't break the integration test.
- Do not write the cascade state to a fixed path like `.roko/learn/`.
  Tests must be hermetic.
