# CH01: Add `updateAgentCard(uint256,string,uint64)` to IdentityRegistry.sol

## Goal

Add storage + a function + an event that lets agents update their
`agentCardUri` and `capabilityMask` atomically. Preserve the existing
`updateAgentCardUri(uint256,string)` for back-compat.

## Changes Required

1. In `contracts/src/IdentityRegistry.sol`:
   - Add storage: `mapping(uint256 => uint64) public capabilityMask;`
   - Add function:
     ```solidity
     function updateAgentCard(
         uint256 passportId,
         string calldata cardUri,
         uint64  mask
     ) external onlyPassportOwner(passportId) {
         agentCardUri[passportId] = cardUri;
         capabilityMask[passportId] = mask;
         emit AgentCardUpdated(passportId, cardUri, mask);
     }
     ```
   - Add event:
     ```solidity
     event AgentCardUpdated(uint256 indexed passportId, string cardUri, uint64 capabilityMask);
     ```
   - Keep the existing `updateAgentCardUri(uint256,string)` and its
     event untouched.

2. In `contracts/test/IdentityRegistry.t.sol`:
   - Add a test that registers a passport, calls `updateAgentCard`,
     reads back `agentCardUri[id]` and `capabilityMask[id]`, asserts
     equality.
   - Add a test that asserts the `AgentCardUpdated` event fires with
     the correct topics.
   - Add a test that asserts non-owner calls revert.
   - Keep all existing tests; the legacy `updateAgentCardUri` path
     still passes.

3. Run `forge fmt` and `forge build`. The build must succeed; the
   workspace expects clean compile in CI.

## Acceptance Criteria

- [ ] `forge build` succeeds.
- [ ] `forge test --match-contract IdentityRegistryTest` passes (existing + new tests).
- [ ] `cast call <addr> 'capabilityMask(uint256)(uint64)' 0 --rpc-url <anvil>` returns a value (zero pre-update, non-zero post-update).
- [ ] `cast call <addr> 'agentCardUri(uint256)(string)' 0 --rpc-url <anvil>` returns the URI set by `updateAgentCard`.

## Do NOT

- Do not break the existing `updateAgentCardUri(uint256,string)` ABI.
  Old agents must continue to register.
- Do not change the visibility of `agentCardUri` or other existing
  storage. Storage layout stability matters for upgrades.
- Do not pack `capabilityMask` into the same slot as
  `agentCardUri` mapping value. uint64 mappings get their own slots
  in solidity; that's fine.
