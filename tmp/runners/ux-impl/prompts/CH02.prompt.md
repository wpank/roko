# CH02: IdentityRegistryReader in roko-chain (enumerate, get, watch)

## Goal

Read-only client for `IdentityRegistry` analogous to
`crates/roko-chain/src/insight_board.rs` (AG02) and
`crates/roko-chain/src/marketplace.rs` (existing template).

## Changes Required

1. Create `crates/roko-chain/src/identity_registry.rs`:

```rust
//! Read-only client for the ERC-8004 IdentityRegistry contract.

use std::sync::Arc;

use alloy_primitives::{Address, U256};
use alloy_sol_types::sol;
use anyhow::{Context, Result};
use serde::{Deserialize, Serialize};
use tokio::sync::mpsc;

use crate::client::ChainClientBase;

sol! {
    interface IIdentityRegistry {
        function registeredCount() external view returns (uint256);
        function registeredAt(uint256 i) external view returns (uint256 passportId);
        function passport(uint256 passportId) external view returns (
            address owner, string memory agentCardUri, uint256 registeredAt
        );
        function capabilityMask(uint256 passportId) external view returns (uint64);
        event AgentCardUpdated(uint256 indexed passportId, string cardUri, uint64 capabilityMask);
    }
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct PassportRecord {
    pub passport_id: String,    // hex
    pub owner: String,          // hex
    pub agent_card_uri: String,
    pub capability_mask: u64,
    pub registered_at: u64,
}

#[derive(Debug, Default, Clone)]
pub struct PassportFilter {
    pub require_capability_bits: Option<u64>,
    pub forbid_capability_bits: Option<u64>,
    pub since_passport_id: Option<U256>,
    pub max_count: Option<usize>,
}

pub struct IdentityRegistryReader {
    address: Address,
    provider: Arc<dyn ChainClientBase>,
}

impl IdentityRegistryReader {
    pub fn new(address: Address, provider: Arc<dyn ChainClientBase>) -> Self {
        Self { address, provider }
    }

    pub async fn count(&self) -> Result<u64> {
        // call registeredCount() via provider
        unimplemented!("port from marketplace.rs pattern")
    }

    pub async fn passport(&self, id: U256) -> Result<PassportRecord> {
        // call passport(id) + capabilityMask(id) and merge
        unimplemented!("port from marketplace.rs pattern")
    }

    pub async fn enumerate_passports(&self, filter: PassportFilter) -> Result<Vec<PassportRecord>> {
        let count = self.count().await?;
        let start = filter.since_passport_id.map(|u| u.to::<u64>()).unwrap_or(0);
        let max = filter.max_count.unwrap_or(usize::MAX);

        let mut out = Vec::new();
        for id in start..count {
            if out.len() >= max { break; }
            let rec = self.passport(U256::from(id)).await?;
            if let Some(req) = filter.require_capability_bits {
                if rec.capability_mask & req != req { continue; }
            }
            if let Some(forbid) = filter.forbid_capability_bits {
                if rec.capability_mask & forbid != 0 { continue; }
            }
            out.push(rec);
        }
        Ok(out)
    }

    pub fn watch_registrations(self: Arc<Self>) -> mpsc::Receiver<PassportRecord> {
        let (tx, rx) = mpsc::channel(64);
        // subscribe to AgentCardUpdated; on event, call self.passport(id)
        // and forward through tx. Fall back to polling at 30s if WS
        // unavailable. Pattern: marketplace.rs.
        let _spawned = tokio::spawn(async move {
            let _ = (self, tx);
        });
        rx
    }
}
```

## Acceptance Criteria

- [ ] `crates/roko-chain/src/identity_registry.rs` matches the shape above with no `unimplemented!()`.
- [ ] `cargo build -p roko-chain` succeeds.
- [ ] `cargo test -p roko-chain --lib identity_registry` passes (3 unit tests).
- [ ] `IdentityRegistryReader`, `PassportRecord`, `PassportFilter` re-exported from `roko-chain`.

## Do NOT

- Do not introduce a new ABI binding library. Use `alloy_sol_types`.
- Do not page `enumerate_passports` in batches of 1; pre-allocate
  the vec with the count and enumerate sequentially. (Future
  optimisation: call `multicall3`. Out of scope for this batch.)
- Do not return raw `B256` / `Address` types from public methods. The
  hex-string `String` is the JSON-friendly contract.
