# CH03: roko-core::capability_bits constants

## Goal

Add a single source-of-truth module `crates/roko-core/src/capability_bits.rs`
with bit assignments and a string→bit helper.

## Changes Required

1. Create `crates/roko-core/src/capability_bits.rs`:

```rust
//! Canonical capability bitmask for ERC-8004 IdentityRegistry.
//!
//! `IdentityRegistry::capabilityMask(passportId)` is a 64-bit value.
//! Bits 0..14 are skill flags. Bit 15 is the "Roko-compatible" flag.
//! Bits 16..63 are reserved; do not repurpose without a contract upgrade.

pub const BIT_MESSAGING:       u64 = 1 << 0;
pub const BIT_PREDICTIONS:     u64 = 1 << 1;
pub const BIT_RESEARCH:        u64 = 1 << 2;
pub const BIT_TASKS:           u64 = 1 << 3;
pub const BIT_DEFI_ANALYSIS:   u64 = 1 << 4;
pub const BIT_CODE_GEN:        u64 = 1 << 5;
pub const BIT_KNOWLEDGE:       u64 = 1 << 6;
pub const BIT_PHEROMONES:      u64 = 1 << 7;
// bits 8..14 reserved for future skill flags
pub const BIT_ROKO_COMPATIBLE: u64 = 1 << 15;
pub const BITS_RESERVED:       u64 = 0xFFFF_FFFF_FFFF_0000;

/// Translate a capability tag (as declared in `AgentCardPayload.capabilities`)
/// into its corresponding bit. Returns `None` for unknown tags so callers
/// can decide whether to ignore or warn.
#[must_use]
pub fn capability_string_to_bit(name: &str) -> Option<u64> {
    Some(match name {
        "messaging"     => BIT_MESSAGING,
        "predictions"   => BIT_PREDICTIONS,
        "research"      => BIT_RESEARCH,
        "tasks"         => BIT_TASKS,
        "defi-analysis" => BIT_DEFI_ANALYSIS,
        "code-gen"      => BIT_CODE_GEN,
        "knowledge"     => BIT_KNOWLEDGE,
        "pheromones"    => BIT_PHEROMONES,
        _ => return None,
    })
}

/// Compose a mask from a slice of capability tags. Always sets
/// `BIT_ROKO_COMPATIBLE` so discovery filters by bit 15 catch the
/// agent.
#[must_use]
pub fn mask_for_capabilities(caps: &[String]) -> u64 {
    let mut m = BIT_ROKO_COMPATIBLE;
    for c in caps {
        if let Some(b) = capability_string_to_bit(c) {
            m |= b;
        }
    }
    m
}

#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn known_capability_maps() {
        assert_eq!(capability_string_to_bit("predictions"), Some(BIT_PREDICTIONS));
    }

    #[test]
    fn unknown_returns_none() {
        assert_eq!(capability_string_to_bit("something-new"), None);
    }

    #[test]
    fn mask_always_includes_bit_15() {
        let m = mask_for_capabilities(&[]);
        assert_eq!(m & BIT_ROKO_COMPATIBLE, BIT_ROKO_COMPATIBLE);
    }

    #[test]
    fn mask_unions_known_bits_only() {
        let m = mask_for_capabilities(&["messaging".into(), "unknown".into()]);
        assert_eq!(m & BIT_MESSAGING, BIT_MESSAGING);
        assert_eq!(m & BIT_ROKO_COMPATIBLE, BIT_ROKO_COMPATIBLE);
    }
}
```

2. Re-export from `crates/roko-core/src/lib.rs`:

   ```rust
   pub mod capability_bits;
   ```

   No further re-exports — consumers explicitly use
   `roko_core::capability_bits::BIT_*` for clarity.

## Do NOT

- Do not place the constants in `roko-agent-server`. The bits are a
  cross-crate contract; they live in `roko-core`.
- Do not change the bit positions later. Bit positions are fixed by
  the on-chain contract.
- Do not add bits beyond what a current consumer needs. Reservations
  are fine; speculative additions are not.
