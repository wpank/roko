# CH04: Agent-server registration uses v2 selector + bit 15

## Goal

`crates/roko-agent-server/src/registration.rs::AgentRegistration::update_identity_registry`
calls the new `updateAgentCard(uint256,string,uint64)` selector and
includes the capability mask. Old 2-arg `updateAgentCardUri` path
preserved behind a `use_v2: bool` field for back-compat.

## Changes Required

1. Add to `AgentRegistration`:

   ```rust
   /// Use v2 selector `updateAgentCard(uint256,string,uint64)` (default: true).
   /// Set false to fall back to the legacy `updateAgentCardUri(uint256,string)`.
   pub use_v2: bool,
   ```

   Update `Default` impl to set `use_v2: true`.

2. In `update_identity_registry`, branch on `self.use_v2`:

   ```rust
   async fn update_identity_registry(
       &self,
       state: &AgentState,
       card_uri: &str,
   ) -> Result<Option<String>> {
       let Some((wallet, registry, passport_id)) = self.chain_update_config() else {
           return Ok(None);
       };
       let request = if self.use_v2 {
           let mask = roko_core::capability_bits::mask_for_capabilities(state.capabilities());
           TxRequest {
               to: Some(registry.to_string()),
               data: build_update_agent_card_calldata(passport_id, card_uri, mask)?,
               ..TxRequest::default()
           }
       } else {
           TxRequest {
               to: Some(registry.to_string()),
               data: build_update_agent_card_uri_calldata(passport_id, card_uri)?,
               ..TxRequest::default()
           }
       };
       let hash = wallet
           .sign_and_submit(request)
           .await
           .context("submit identity-registry transaction")?;
       Ok(Some(tx_hash_string(&hash)))
   }
   ```

   The existing `build_update_agent_card_uri_calldata` stays. Add a
   new `build_update_agent_card_calldata`:

   ```rust
   fn build_update_agent_card_calldata(
       passport_id: &str,
       card_uri: &str,
       mask: u64,
   ) -> Result<Vec<u8>> {
       let selector = &keccak256("updateAgentCard(uint256,string,uint64)".as_bytes())[..4];
       let encoded_passport = encode_passport_id_word(passport_id)?;
       let encoded_uri = abi_encode_string(card_uri);
       let encoded_mask = encode_word(mask);
       // ABI: uint256, string (offset), uint64
       //   slot 0: passportId
       //   slot 1: offset to string (= 0x60 = 96, since uint256 + uint64 + offset = 3 slots)
       //   slot 2: mask
       //   slots 3+: string-length + bytes
       let mut data = Vec::with_capacity(4 + 96 + encoded_uri.len());
       data.extend_from_slice(selector);
       data.extend_from_slice(&encoded_passport);
       data.extend_from_slice(&encode_word(0x60));
       data.extend_from_slice(&encoded_mask);
       data.extend_from_slice(&encoded_uri);
       Ok(data)
   }
   ```

3. Pass `state.capabilities()` (already exists on `AgentState`) so
   that the mask reflects the agent's enabled features. The
   `mask_for_capabilities` helper unconditionally sets `BIT_ROKO_COMPATIBLE`,
   so we don't need to do it twice.

4. Add to `AgentRegistration::default()`:

   ```rust
   use_v2: true,
   ```

5. In `crates/roko-agent-server/Cargo.toml`, ensure `roko-core` is a
   dependency (it almost certainly is; add if missing).

6. Add a test in
   `crates/roko-agent-server/tests/registration.rs` (create if
   missing) that asserts the published card always has `domain_tags`
   containing `"roko"`. Use the `StubPublisher` from the existing
   inline tests at the bottom of `registration.rs`.

## Acceptance Criteria

- [ ] `AgentRegistration { use_v2, .. }` field exists.
- [ ] When `use_v2 = true`, the submitted tx data starts with
  `keccak256("updateAgentCard(uint256,string,uint64)")[..4]`.
- [ ] When `use_v2 = false`, the legacy selector still works.
- [ ] `Default::default()` sets `use_v2 = true`.
- [ ] `cargo build -p roko-agent-server` succeeds.
- [ ] `cargo test -p roko-agent-server` passes; new test asserts
  domain tags include `"roko"`.

## Do NOT

- Do not delete `build_update_agent_card_uri_calldata` or its caller
  branch. Old contracts must keep working.
- Do not move `roko-core::capability_bits` into roko-agent-server.
- Do not auto-detect "is the new contract deployed". The operator's
  `use_v2` flag is the contract.
