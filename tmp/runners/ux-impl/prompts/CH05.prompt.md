# CH05: Aggregator merges chain ∪ local registry

## Goal

`crates/roko-serve/src/routes/aggregator.rs::known_agents` returns the
union of (chain-derived agents from `IdentityRegistryReader`) and
(local roko-serve-managed agents). Chain entries dominate on ID
conflicts.

## Changes Required

1. Extend `AppState`:

   ```rust
   pub struct AppState {
       // ... existing fields ...
       pub identity_reader: Option<Arc<IdentityRegistryReader>>,
       pub agent_card_fetcher: Arc<AgentCardFetcher>,
       // (process_supervisor + DiscoveredAgent etc already exist)
   }
   ```

   Build them in the constructor, mirroring the `KnowledgeSource`
   wiring from AG04. If no chain config is supplied,
   `identity_reader` is `None`.

2. Add a `discovery_source` enum field to `DiscoveredAgent`:

   ```rust
   pub enum DiscoverySource {
       Chain { passport_id: String },
       LocalRegistry,
       Both { passport_id: String },
   }

   pub struct DiscoveredAgent {
       // ... existing fields ...
       pub discovery_source: DiscoverySource,
   }
   ```

   Update every `DiscoveredAgent::from_*` constructor to set this.

3. Replace `known_agents()` (in `aggregator.rs`):

   ```rust
   async fn known_agents(state: &Arc<AppState>) -> Result<Vec<DiscoveredAgent>, ApiError> {
       let cache_key = "aggregator:agents:merged:v1";
       if let Some(cached) = state.cached_value::<Vec<DiscoveredAgent>>(cache_key).await {
           return Ok(cached);
       }

       let local_agents = state.process_supervisor.discovered_agents();

       let chain_agents: Vec<DiscoveredAgent> =
           if let Some(reader) = &state.identity_reader {
               let filter = roko_chain::PassportFilter {
                   require_capability_bits: Some(roko_core::capability_bits::BIT_ROKO_COMPATIBLE),
                   ..Default::default()
               };
               let records = reader.enumerate_passports(filter).await
                   .map_err(|e| ApiError::internal(format!("enumerate passports: {e}")))?;
               let fetcher = Arc::clone(&state.agent_card_fetcher);
               let cards = futures::future::join_all(records.iter().map(|r| {
                   let f = Arc::clone(&fetcher);
                   let uri = r.agent_card_uri.clone();
                   async move { f.fetch(&uri).await.ok() }
               })).await;
               records.into_iter().zip(cards)
                   .filter_map(|(rec, card)| {
                       let card = card?;
                       if !card.domain_tags.iter().any(|t| t == "roko") {
                           return None;
                       }
                       Some(DiscoveredAgent::from_chain(rec, card))
                   })
                   .collect()
           } else {
               Vec::new()
           };

       let mut merged: HashMap<String, DiscoveredAgent> = local_agents
           .into_iter()
           .map(|a| (a.agent_id.clone(), a))
           .collect();
       for agent in chain_agents {
           let id = agent.agent_id.clone();
           merged.entry(id)
               .and_modify(|local| {
                   if let DiscoverySource::Chain { passport_id } = &agent.discovery_source {
                       local.discovery_source = DiscoverySource::Both { passport_id: passport_id.clone() };
                   }
                   // Chain endpoints take precedence over local-registry endpoints.
                   local.endpoints = agent.endpoints.clone();
               })
               .or_insert(agent);
       }
       let result: Vec<_> = merged.into_values().collect();
       state.cache_value(cache_key, &result, AGENT_LIST_TTL).await;
       Ok(result)
   }
   ```

4. Add `DiscoveredAgent::from_chain(record, card) -> Self` constructor
   in `state.rs`. It maps `PassportRecord.passport_id` → `agent_id`
   (for new chain-only agents that don't have a local id, use the
   passport hex), reads `endpoints` from the card, and sets
   `discovery_source: DiscoverySource::Chain { passport_id }`.

5. Add an `AgentCardUpdated` event subscription in `serve(...)`,
   parallel to AG08:

   ```rust
   if let Some(reader) = &state.identity_reader {
       let reader = Arc::clone(reader);
       let st = Arc::clone(&state);
       tokio::spawn(async move {
           let mut events = Arc::clone(&reader).watch_registrations();
           while let Some(_event) = events.recv().await {
               st.invalidate_cache_prefix("aggregator:agents:").await;
           }
       });
   }
   ```

## Acceptance Criteria

- [ ] `cargo build -p roko-serve` succeeds.
- [ ] When `identity_reader` is `None`, `/api/agents` matches the previous shape (only local agents).
- [ ] When `identity_reader` is `Some(...)` and a chain agent has bit 15 set + `"roko"` domain tag, it appears in `/api/agents`.
- [ ] When the same `agent_id` exists locally and on chain, the merged entry has `discovery_source: Both` and chain endpoints.
- [ ] Test `crates/roko-serve/tests/aggregator_chain_discovery.rs` exists with at least the four cases (chain-only, local-only, both, neither).

## Do NOT

- Do not crash if a chain agent has a malformed Agent Card. Skip it
  with a `tracing::warn!`.
- Do not call `enumerate_passports` on every request. Cache for
  `AGENT_LIST_TTL` (existing constant).
- Do not change the response shape of `/api/agents` even when chain
  entries are added. The fields surface naturally through the
  existing serialiser.
