# CH06: Bearer-protected GET /api/agents/discover-chain debug route

## Goal

Add a debug endpoint that returns the *raw* chain enumeration (no
merging). Useful for diagnosing "why isn't this agent showing up?"

## Changes Required

1. In `crates/roko-serve/src/routes/aggregator.rs`:

   ```rust
   .route("/agents/discover-chain", get(discover_chain))
   ```

   Place it inside the bearer-protected sub-router (mirror the existing
   `pheromones/decay` placement).

2. Implement:

   ```rust
   async fn discover_chain(
       State(state): State<Arc<AppState>>,
   ) -> Result<Json<Value>, ApiError> {
       let Some(reader) = &state.identity_reader else {
           return Ok(Json(serde_json::json!({
               "configured": false,
               "passports": [],
               "note": "no identity_reader configured; chain discovery disabled",
           })));
       };

       let filter = roko_chain::PassportFilter::default();
       let records = reader.enumerate_passports(filter).await
           .map_err(|e| ApiError::internal(format!("enumerate passports: {e}")))?;

       Ok(Json(serde_json::json!({
           "configured": true,
           "passports": records,
       })))
   }
   ```

3. Document under OpenAPI tag `agents`:

   ```rust
   #[utoipa::path(
       get,
       path = "/api/agents/discover-chain",
       tag = "agents",
       responses(
           (status = 200, description = "Raw chain enumeration; bearer-protected debug route"),
           (status = 401, description = "Missing or invalid bearer token"),
       )
   )]
   ```

   (Skip the macro if existing handlers in this file don't carry it.)

## Acceptance Criteria

- [ ] `curl -H "Authorization: Bearer <tok>" http://127.0.0.1:6677/api/agents/discover-chain` returns 200 with a `passports` array.
- [ ] Without the bearer header, the route returns 401.
- [ ] When `identity_reader` is `None`, the route returns
  `{ "configured": false, "passports": [] }`.
- [ ] `cargo test -p roko-serve` passes.

## Do NOT

- Do not include the merged local-registry agents in this response.
  The whole point is "what does chain say".
- Do not cache the response. Debug routes should reflect current
  state.
- Do not skip auth. Anonymous chain enumeration is fine in principle
  (it's public chain data) but we don't want this leaking the
  identity_reader's RPC URL or causing unbounded RPC calls.
