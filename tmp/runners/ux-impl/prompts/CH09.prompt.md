```markdown
# ERC-8004 chain-based agent discovery

The roko-serve aggregator (`/api/agents`) merges two sources:

1. **Chain enumeration** — walks `IdentityRegistry::registeredCount()`
   and `passport(i)`, filters by capability bitmask bit 15 (Roko-compatible),
   fetches each Agent Card, and confirms the card's `domain_tags`
   contains `"roko"`.
2. **Local registry** — agents spawned via `ProcessSupervisor` in this
   same roko-serve process.

Conflicts on `agent_id` resolve to the chain entry (chain endpoints
take precedence).

## Capability bitmask

`IdentityRegistry::capabilityMask(passportId)` is a 64-bit value.
Allocation lives at `crates/roko-core/src/capability_bits.rs`:

| Bit | Constant | Meaning |
|-----|----------|---------|
| 0 | BIT_MESSAGING | Implements `/message` + `/stream` |
| 1 | BIT_PREDICTIONS | Implements `/predictions/*` |
| 2 | BIT_RESEARCH | Implements `/research` |
| 3 | BIT_TASKS | Implements `/tasks/*` |
| 4 | BIT_DEFI_ANALYSIS | Skill flag |
| 5 | BIT_CODE_GEN | Skill flag |
| 6 | BIT_KNOWLEDGE | Skill flag |
| 7 | BIT_PHEROMONES | Skill flag |
| 8-14 | _reserved_ | Future skill flags |
| 15 | BIT_ROKO_COMPATIBLE | Required for discovery to surface this agent |
| 16-63 | _reserved_ | Do not repurpose without contract upgrade |

## Domain-tag filter

The aggregator additionally requires the off-chain Agent Card's
`domain_tags` array to include `"roko"`. This is a defence-in-depth
filter: anyone can flip bit 15 on chain, but they cannot easily
spoof a published Agent Card without controlling the underlying
URI.

## Network-only mode (dashboard)

Setting `VITE_NETWORK_ONLY=true` in nunchi-dashboard skips its
roko-serve calls and enumerates the chain directly via wagmi/viem.
Pheromone and knowledge tabs degrade in this mode (they need
roko-serve until the matching contracts ship).

## Demo bootstrap

`roko demo init` populates 5 Roko passports against a fresh mirage
fork. See `crates/roko-demo/src/scenarios/bootstrap_passports.rs` and
the post-run results at `tmp/demo-bootstrap.json`.
```
