# FG01: Add `default-members` to root Cargo.toml

## Goal

`cargo build --workspace` builds only the shipped slice. `cargo build
--workspace --all` still builds everything including Phase 2 crates.

## Changes Required

1. Open `Cargo.toml` (workspace root).

2. Locate the `[workspace]` block. Identify the existing `members = [...]`
   list. Add a `default-members` list that excludes the Phase 2
   crates:

   ```toml
   [workspace]
   members = [
     # ... keep existing list, including phase-2 crates ...
   ]

   default-members = [
     "apps/agent-relay",
     "apps/mirage-rs",
     "apps/roko-chain-watcher",
     "crates/roko-acp",
     "crates/roko-agent",
     "crates/roko-agent-server",
     "crates/roko-chain",
     "crates/roko-cli",
     "crates/roko-compose",
     "crates/roko-conductor",
     "crates/roko-core",
     "crates/roko-demo",
     "crates/roko-fs",
     "crates/roko-gate",
     "crates/roko-index",
     "crates/roko-lang-go",
     "crates/roko-lang-rust",
     "crates/roko-lang-typescript",
     "crates/roko-learn",
     "crates/roko-mcp-code",
     "crates/roko-neuro",
     "crates/roko-orchestrator",
     "crates/roko-primitives",
     "crates/roko-runtime",
     "crates/roko-serve",
     "crates/roko-std",
   ]
   ```

   Phase 2 crates **excluded** from `default-members`:
   - `crates/roko-dreams`
   - `crates/roko-daimon`
   - `crates/roko-plugin` (decision deferred to FG03)

   `roko-mcp-{github,slack,scripts,stdio}`: include in
   `default-members` only if MC02 marked them `shipped-default`.
   Default: include all (audit hasn't dictated otherwise).

3. Run `cargo build --workspace` from the repo root. Verify only the
   default-members get compiled (use `--timings` and inspect output if
   needed).

4. Run `cargo build --workspace --all`. Verify all members including
   Phase 2 build.

5. Run `cargo build --workspace --no-default-features` for a sanity
   pass.

## Acceptance Criteria

- [ ] `Cargo.toml` `[workspace]` has a `default-members` list.
- [ ] `cargo build --workspace` succeeds (only default-members).
- [ ] `cargo build --workspace --all` succeeds (everything).
- [ ] `cargo test --workspace --all` succeeds.
- [ ] Phase 2 crates (`roko-dreams`, `roko-daimon`, `roko-plugin`) are
  *not* in `default-members`.
- [ ] All currently-shipped crates ARE in `default-members`.

## Do NOT

- Do not remove any crate from `members`. The shape is still the
  whole workspace; only the *default* set narrows.
- Do not gate `roko-chain` as Phase 2. It's a hard dependency for
  AG02 / CH02.
- Do not introduce per-crate `[features]` to control inclusion. The
  `default-members` mechanism is the contract.
- Do not edit any crate's `Cargo.toml` in this batch. FG02-FG04 do
  per-crate work.
