# FG03: roko-plugin audit + decision

## Goal

Decide the fate of `crates/roko-plugin` (item 54). Three options:
gate as Phase 2, delete, or promote to default.

## Changes Required

1. Read `crates/roko-plugin/src/lib.rs` and the rest of its source.

2. Grep for external consumers:

   ```bash
   rg --type rust -l 'roko_plugin::' crates/ apps/ | grep -v 'crates/roko-plugin/'
   ```

3. Decide using these rules:

   | Evidence | Decision |
   |----------|----------|
   | Has external callers, well-tested | **promote**: add to `default-members` |
   | No external callers, near-term plan documented | **gate**: stay out of `default-members`, file tracking issue |
   | No external callers, no plan, no tests | **delete**: remove from workspace |

4. Apply the decision:

   - **promote**: edit `Cargo.toml` to add
     `crates/roko-plugin` to `default-members`; update CLAUDE.md row
     to "Wired".

   - **gate**: leave as-is (FG01 already excluded it from
     `default-members`); add a `crates/roko-plugin/README.md` (or
     update existing) noting the Phase 2 status and the entry
     conditions.

   - **delete**: remove `crates/roko-plugin` from `Cargo.toml`
     `members`; `git rm -r crates/roko-plugin/`; update CLAUDE.md to
     drop the row.

5. Record the decision in
   `tmp/ux/ux-followup/08-phase-2-vision.md` item 54 with a
   `**Decided YYYY-MM-DD: <bucket>**` line and the audit rationale.

## Acceptance Criteria

- [ ] Decision recorded in `tmp/ux/ux-followup/08-phase-2-vision.md` item 54.
- [ ] `cargo build --workspace` and `cargo build --workspace --all` succeed.
- [ ] CLAUDE.md row reflects the chosen option.
- [ ] If deleted: no `roko_plugin::` references remain in any other
  crate (grep output empty).

## Do NOT

- Do not pick "gate" by default. It's the easy choice but parks
  decisions indefinitely. Only choose `gate` if there's a documented
  near-term plan in `tmp/ux/ux-followup/08-phase-2-vision.md`.
- Do not delete the crate without zero external callers. Verify with
  grep first.
- Do not skip CLAUDE.md update. Stale doc entries are an anti-pattern.
