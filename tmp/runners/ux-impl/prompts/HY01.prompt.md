# HY01: SystemPromptBuilder snapshot tests (one per role)

## Goal

`crates/roko-compose/tests/system_prompt_snapshots.rs` produces an
`insta` snapshot per agent role. Reviewers run `cargo insta review` to
bless changes.

## Context

Followup item 19. The crate compiles but has no per-role golden
output; silent regressions in prompt composition are undetectable.

## Changes Required

1. Add `insta` to `crates/roko-compose/Cargo.toml` `[dev-dependencies]`:

   ```toml
   insta = { version = "1.39", features = ["yaml"] }
   ```

   (Use `cargo add insta --dev --features yaml -p roko-compose` to
   pull the latest.)

2. Identify the canonical role list. Read
   `crates/roko-core/src/config/schema.rs` for `RoleOverride` keys and
   any `[agent.<role>]` defaults; the typical list is
   `implementer`, `reviewer`, `planner`, `researcher`, `tester`,
   `documenter`. Confirm against actual config before locking in.

3. Create `crates/roko-compose/tests/system_prompt_snapshots.rs`:

```rust
use roko_compose::system_prompt_builder::SystemPromptBuilder;
use roko_core::config::schema::RokoConfig;

fn fixture_config() -> RokoConfig {
    // Build a deterministic, minimal config. Use the existing
    // RokoConfig::default() if it is hermetic; otherwise hand-build a
    // small one. Constants for sample skills, tools, memory items.
    let mut cfg = RokoConfig::default();
    // Inject any required test fixtures here:
    // - skills: [...]
    // - tools: [...]
    // - memory_excerpts: [...]
    cfg
}

fn render_for_role(role: &str) -> String {
    let cfg = fixture_config();
    SystemPromptBuilder::new(&cfg, role)
        .render_default()                  // adapt to actual API
        .expect("render must succeed")
}

macro_rules! role_snapshot {
    ($name:ident, $role:expr) => {
        #[test]
        fn $name() {
            let prompt = render_for_role($role);
            insta::assert_snapshot!(stringify!($name), prompt);
        }
    };
}

role_snapshot!(role_implementer, "implementer");
role_snapshot!(role_reviewer, "reviewer");
role_snapshot!(role_planner, "planner");
role_snapshot!(role_researcher, "researcher");
role_snapshot!(role_tester, "tester");
role_snapshot!(role_documenter, "documenter");
```

4. Run `cargo insta test --accept` once locally to seed the
   `*.snap` files. Commit them under
   `crates/roko-compose/tests/snapshots/`.

5. Add to `.github/workflows/<test-job>.yml` (or wherever workspace
   tests run) a step that fails on uncommitted snapshot updates:
   `cargo insta test --check`. If the workflow doesn't exist yet,
   leave a TODO in the test file's header.

## Acceptance Criteria

- [ ] `crates/roko-compose/tests/system_prompt_snapshots.rs` exists.
- [ ] One `*.snap` file per role under `tests/snapshots/`.
- [ ] `cargo test -p roko-compose --test system_prompt_snapshots` passes.
- [ ] `cargo insta test --check` is green (no pending changes).
- [ ] `insta` is in `[dev-dependencies]` of `roko-compose/Cargo.toml`.

## Do NOT

- Do not commit `*.snap.new` files. Only `*.snap` (blessed).
- Do not add prompt-rendering changes in this batch. The whole point
  is *capturing* the current output. HY02 expands the assertions.
- Do not skip roles. If a role's render fails, that's a bug to surface,
  not to silence.
