# HY02: 6-layer-presence assertion across all roles

## Goal

A single test asserts every role's rendered prompt contains all 6
expected layer headers. Catches a layer disappearing without a
snapshot bless.

## Context

The 6 layers (per the SystemPromptBuilder design):
`role`, `skills`, `tools`, `context`, `memory`, `policy`. The actual
header text used in rendering may vary — adjust the strings below to
match `system_prompt_builder.rs` literals.

## Changes Required

1. In `crates/roko-compose/tests/system_prompt_snapshots.rs` (already
   created in HY01), append:

```rust
const ALL_ROLES: &[&str] = &[
    "implementer", "reviewer", "planner", "researcher", "tester", "documenter",
];

const REQUIRED_LAYER_MARKERS: &[&str] = &[
    "# Role",
    "# Skills",
    "# Tools",
    "# Context",
    "# Memory",
    "# Policy",
];

#[test]
fn six_layers_present_in_every_role() {
    for role in ALL_ROLES {
        let prompt = render_for_role(role);
        for marker in REQUIRED_LAYER_MARKERS {
            assert!(
                prompt.contains(marker),
                "role={role}: expected marker {marker} missing from rendered prompt"
            );
        }
    }
}
```

2. **Adapt the markers** to match the actual section headers emitted
   by `system_prompt_builder.rs`. Likely candidates: `# Role`,
   `## Role`, `[Role]`. Read the source to determine.

3. If a role intentionally omits a layer, document the exception:

   ```rust
   const REQUIRED_PER_ROLE: &[(&str, &[&str])] = &[
       ("documenter", &["# Role", "# Skills", "# Tools", "# Context", "# Memory", "# Policy"]),
       // future exceptions go here
   ];
   ```

   Update the test to use a per-role lookup.

## Acceptance Criteria

- [ ] `cargo test -p roko-compose --test system_prompt_snapshots six_layers_present_in_every_role` passes.
- [ ] Markers list matches the actual headers in
  `system_prompt_builder.rs`.
- [ ] No marker is omitted.

## Do NOT

- Do not assert the layer *content*. Snapshots cover that. This test
  asserts only structural presence.
- Do not weaken the test to "at least 4 of 6". Either every role has
  every layer, or the exception is explicit per-role.
