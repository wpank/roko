# HY04: Same as HY03 but for roko-gate, roko-compose

## Goal

Apply the same `# Errors` / `# Panics` cleanup in `roko-gate` and
`roko-compose`.

## Changes Required

Identical procedure to HY03, scoped to:

```
crates/roko-gate/
crates/roko-compose/
```

1. `rg -t rust '#\\[allow\\(clippy::missing_(errors|panics)_doc\\)\\]' crates/roko-gate crates/roko-compose`
2. Remove + fill in.
3. `cargo clippy -p roko-gate --no-deps -- -D warnings`
4. `cargo clippy -p roko-compose --no-deps -- -D warnings`
5. `cargo build -p roko-gate -p roko-compose`

## Acceptance Criteria

- [ ] No `#[allow(clippy::missing_*_doc)]` in either crate.
- [ ] Both crates' clippy with `-D warnings` is green.
- [ ] Existing tests still pass.

## Do NOT

- Do not introduce per-function `#[allow(...)]`.
- Do not add `# Panics` sections to functions that don't panic.
- Do not touch other crates.
