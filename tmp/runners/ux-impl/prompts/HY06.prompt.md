# HY06: Per-gate timeline rendered in TUI Gate tab

## Goal

Render a 24-bucket sparkline per gate (compile, test, clippy, symbol,
generated_test, fact_check, property_test, verify_chain, llm_judge,
integration) on the TUI Gate tab. Closes followup item 87.

## Context

`crates/roko-cli/src/tui/verdicts.rs` already maintains rolling
24×1h pass/fail buckets via `VerdictsAggregator`. Today the dashboard
only renders a single rolling EMA per rung. This batch surfaces the
per-gate breakdown.

2. In `crates/roko-cli/src/tui/views/dashboard_view.rs`, extend the
   existing `render_gate_trend_grid` to render one row per gate:

   ```rust
   fn render_gate_trend_rows(stats: &[GateStats]) -> Vec<Line<'_>> {
       let mut rows = Vec::with_capacity(stats.len());
       for s in stats {
           let mut spans = Vec::with_capacity(2 + 24);
           spans.push(Span::styled(format!("{:<14}", s.name), Style::default()));
           for b in &s.buckets {
               let pass_rate = b.pass_rate();        // 0.0..=1.0
               let block = sparkline_block(pass_rate);
               let color = bucket_color(pass_rate);
               spans.push(Span::styled(block.to_string(), Style::default().fg(color)));
           }
           let last = s.buckets.last().map(|b| b.pass_rate()).unwrap_or(0.0);
           spans.push(Span::raw(format!("  {:>5.1}%", last * 100.0)));
           rows.push(Line::from(spans));
       }
       rows
   }

   fn sparkline_block(rate: f64) -> char {
       const LEVELS: &[char] = &['▁','▂','▃','▄','▅','▆','▇','█'];
       let idx = ((rate * (LEVELS.len() as f64 - 1.0)).round() as usize).min(LEVELS.len() - 1);
       LEVELS[idx]
   }

   fn bucket_color(rate: f64) -> Color {
       if rate >= 0.9 { Color::Green }
       else if rate >= 0.7 { Color::Yellow }
       else { Color::Red }
   }
   ```

## Acceptance Criteria

- [ ] `cargo test -p roko-cli --lib tui` passes the new render test.
- [ ] Running the TUI against a fixture state with multiple gates
  shows one row per gate.
- [ ] Pass-rate >= 90% renders green, 70-89% yellow, < 70% red.
- [ ] When > 6 gates exist, only top-6 by recent activity render
  with a "+ N more" footer.

## Do NOT

- Do not introduce a different sparkline character set. Use the 8-level
  Unicode blocks.
- Do not auto-resize the sparkline width per gate. Fixed 24 buckets.
- Do not show milliseconds-level timing in the row; the layout has
  enough information already.
