# HY08: Smoke test — TUI event responsiveness

## Goal

`crates/roko-cli/tests/smoke_tui.rs` drives the TUI's app loop with a
synthetic event source and asserts a `RokoEvent::TaskCompleted`
updates the dashboard pane within 1 s.

## Changes Required

```rust
// crates/roko-cli/tests/smoke_tui.rs

use std::time::Duration;

use ratatui::backend::TestBackend;
use ratatui::Terminal;
use roko_runtime::event_bus::RokoEvent;
use tokio::time::Instant;

#[tokio::test]
async fn task_completed_event_updates_pane_within_1s() {
    let backend = TestBackend::new(160, 48);
    let terminal = Terminal::new(backend).unwrap();

    // Build the App with an in-process StateHub (the unconditional
    // subscribe path landed previously).
    let mut app = roko_cli::tui::App::for_smoke();    // add this if absent
    let bus = app.event_bus_handle();

    // Drive one render to settle initial state.
    app.tick_render(&mut terminal_handle(terminal)).await;

    // Emit a synthetic TaskCompleted event.
    let started = Instant::now();
    bus.emit(RokoEvent::TaskCompleted {
        task_id: "smoke-task".into(),
        outcome: "ok".into(),
    }).await;

    // Tick until the dashboard reports the new task or 1 s elapses.
    let mut got = false;
    while started.elapsed() < Duration::from_secs(1) {
        app.tick_render(&mut terminal_handle(terminal)).await;
        if app.state().task_pane_contains("smoke-task") {
            got = true;
            break;
        }
        tokio::time::sleep(Duration::from_millis(20)).await;
    }
    assert!(got, "task pane did not update within 1 s");
}

fn terminal_handle<B: ratatui::backend::Backend>(t: Terminal<B>) -> Terminal<B> {
    t   // stub; ratatui terminals are reused per render in the smoke
}
```

   Implement `App::for_smoke()`, `App::tick_render`, `App::event_bus_handle`,
   `App::state`, `state.task_pane_contains` if any are missing. Mark
   them `#[cfg(any(test, feature = "test-helpers"))]` if exposed only
   for testing.

## Acceptance Criteria

- [ ] `cargo test -p roko-cli --test smoke_tui` passes.
- [ ] Test completes in < 5 s (single 1 s window + setup).
- [ ] No real terminal access — uses `TestBackend`.

## Do NOT

- Do not enter a busy loop polling the state. The 20 ms sleep between
  `tick_render` calls is the contract.
- Do not block the test on actual file I/O. The synthetic event bus
  is the source of truth.
- Do not exercise multiple panes; this smoke is targeted.
