# M02: Drop the chain → dashboard-api implication

## Goal

Make the `chain` Cargo feature in `apps/mirage-rs/Cargo.toml` no longer
imply `dashboard-api`. After this batch, `chain` enables only
HDC/InsightEntry/pheromone *implementation*, not the REST surface.
This is a one-week shim before M04 deletes both features outright.

## Context

Today (`apps/mirage-rs/Cargo.toml`):

```toml
[features]
default = ["binary", "chain"]
binary = []
library = []
sim-gas = []
chain = ["dep:roko-primitives", "dashboard-api"]   # ← implies dashboard-api
dashboard-api = []
roko = ["chain", "dep:roko-core", "dep:async-trait"]
```

After M02:

```toml
[features]
default = ["binary", "chain"]
binary = []
library = []
sim-gas = []
chain = ["dep:roko-primitives"]                    # ← no longer implies
dashboard-api = []
roko = ["chain", "dep:roko-core", "dep:async-trait"]
```

`default` keeps `chain` for back-compat in this brief shim window.
M04 deletes the whole `chain`/`dashboard-api`/`roko` feature surface.

## Changes Required

1. Edit `apps/mirage-rs/Cargo.toml`:
   - Change `chain = ["dep:roko-primitives", "dashboard-api"]` to
     `chain = ["dep:roko-primitives"]`.
   - Add a comment line above the `chain` feature explaining this is
     a deprecation shim that M04 will remove.

2. Verify `cargo build -p mirage-rs --features binary,chain` still
   succeeds (it will, because `chain`-only HDC code doesn't import
   `http_api`).

3. Verify `cargo build -p mirage-rs --no-default-features --features binary,chain,dashboard-api`
   still succeeds (operators who want REST temporarily must opt in).

## Acceptance Criteria

- [ ] `cargo build -p mirage-rs` succeeds with the new feature shape.
- [ ] `cargo build -p mirage-rs --no-default-features --features binary` builds (pure-EVM).
- [ ] `cargo build -p mirage-rs --no-default-features --features binary,chain` builds (HDC, no REST).
- [ ] `cargo build -p mirage-rs --no-default-features --features binary,chain,dashboard-api` builds (HDC + REST).
- [ ] The comment above `chain` references "M04 deletes both" so the next reader knows the plan.

## Do NOT

- Do not modify the `default` line (M04 will narrow it).
- Do not delete `dashboard-api` or `chain`; they must stay
  declarable for one shim release.
- Do not modify anything outside `Cargo.toml`. The cfg gates in source
  remain `#[cfg(feature = "dashboard-api")]` until M04.
