# M03: Add slim `http_health.rs` (`/health` + `/stats` only)

## Goal

Create `apps/mirage-rs/src/http_health.rs` exposing exactly two routes
backed by EVM-only state. M04 will mount this in place of the deleted
`http_api/` module.

## Context

Plan 01 keeps `/health` and `/stats` from the original mirage REST
surface. Everything else (agents, knowledge, pheromones, tasks,
predictions, topology, ws, isfr, skills) goes away in M04. The new
module must:

- Hold zero application state (no `ChainContext`, no
  `KnowledgeStore`, no `PheromoneField`).
- Be unconditionally compilable — no `#[cfg(feature = "...")]` guards.
- Take only `(started_at: Instant, current_block_fn, upstream:
  Option<String>)` as state.
- Be wired in `apps/mirage-rs/src/lib.rs` (export) and
  `apps/mirage-rs/src/main.rs` (mount), but **only** mounted when
  `--features binary` is on.

## Changes Required

1. Create `apps/mirage-rs/src/http_health.rs` exactly as below:

```rust
//! Slim HTTP surface kept after the Wave M extraction.
//!
//! `/health` and `/stats` are EVM-only and require no application
//! state from the deleted `http_api/` module. All other routes live
//! on roko-serve's aggregator (`crates/roko-serve/src/routes/aggregator.rs`).

use std::sync::Arc;
use std::time::Instant;

use axum::extract::State;
use axum::routing::get;
use axum::{Json, Router};
use serde_json::{Value, json};

/// Read-only state for the slim HTTP module.
#[derive(Clone)]
pub struct HealthState {
    pub started_at: Instant,
    pub current_block: Arc<dyn Fn() -> u64 + Send + Sync>,
    pub upstream: Option<String>,
}

impl HealthState {
    pub fn new(
        started_at: Instant,
        current_block: Arc<dyn Fn() -> u64 + Send + Sync>,
        upstream: Option<String>,
    ) -> Self {
        Self { started_at, current_block, upstream }
    }
}

/// Build the slim `/health` + `/stats` router.
pub fn router(state: HealthState) -> Router {
    Router::new()
        .route("/health", get(health))
        .route("/stats", get(stats))
        .with_state(state)
}

async fn health(State(s): State<HealthState>) -> Json<Value> {
    Json(json!({
        "status": "ok",
        "uptime_secs": s.started_at.elapsed().as_secs(),
        "block_number": (s.current_block)(),
    }))
}

async fn stats(State(s): State<HealthState>) -> Json<Value> {
    Json(json!({
        "uptime_secs": s.started_at.elapsed().as_secs(),
        "block_number": (s.current_block)(),
        "upstream": s.upstream,
    }))
}

#[cfg(test)]
mod tests {
    use super::*;
    use axum::body::{Body, to_bytes};
    use axum::http::{Request, StatusCode};
    use tower::ServiceExt;

    fn fixture_state() -> HealthState {
        HealthState::new(Instant::now(), Arc::new(|| 1234), Some("https://upstream".into()))
    }

    #[tokio::test]
    async fn health_returns_ok() {
        let r = router(fixture_state());
        let res = r
            .oneshot(Request::builder().uri("/health").body(Body::empty()).unwrap())
            .await
            .unwrap();
        assert_eq!(res.status(), StatusCode::OK);
        let body = to_bytes(res.into_body(), 1024).await.unwrap();
        let v: serde_json::Value = serde_json::from_slice(&body).unwrap();
        assert_eq!(v["status"], "ok");
        assert_eq!(v["block_number"], 1234);
    }

    #[tokio::test]
    async fn stats_includes_upstream() {
        let r = router(fixture_state());
        let res = r
            .oneshot(Request::builder().uri("/stats").body(Body::empty()).unwrap())
            .await
            .unwrap();
        assert_eq!(res.status(), StatusCode::OK);
        let body = to_bytes(res.into_body(), 1024).await.unwrap();
        let v: serde_json::Value = serde_json::from_slice(&body).unwrap();
        assert_eq!(v["upstream"], "https://upstream");
    }
}
```
