# M04: Delete chain/, http_api/, roko_bridge/ + Cargo cleanup

## Goal

Remove every file in `apps/mirage-rs/src/{chain,http_api,roko_bridge}/`
and collapse the Cargo features to `["binary", "library", "sim-gas"]`.
After this batch, mirage-rs is a pure EVM substrate with `/health`,
`/stats`, JSON-RPC, scenarios, and replay â€” nothing else.

## Context

This is the destructive step of Wave M. M01 verified no live
consumers; M02 dropped the `chain â†’ dashboard-api` implication;
M03 added the slim `http_health.rs`. After M04, the only HTTP routes
mirage exposes are the two from M03.

Net LOC drop: approximately 4 500 (the original Phase 3 estimate).

This batch must run with `verify = "full"` because it touches the
entire crate's compilation surface and Cargo features.

## Changes Required

1. In `apps/mirage-rs/Cargo.toml`:

   Replace the `[features]` block exactly with:

   ```toml
   [features]
   # Phase 3 final. Pure EVM substrate.
   default = ["binary"]
   binary = []
   library = []
   sim-gas = []
   ```

   In the `[dependencies]` block, change `roko-primitives` and
   `roko-core` from optional to absent:
   - Delete the line `roko-primitives = { path = "...", optional = true }`.
   - Delete the line `roko-core = { path = "...", optional = true }`.
   - Delete the `async-trait = { workspace = true, optional = true }` line if
     present.

   Update the package `description` to:

   ```toml
   description = "In-process Ethereum fork simulator with lazy upstream reads, copy-on-write scenario branching, and JSON-RPC server."
   ```

2. In `apps/mirage-rs/src/lib.rs`:

   - Delete `pub mod chain;` (and any `#[cfg(feature = "chain")]` guard).
   - Delete `#[cfg(feature = "dashboard-api")] pub mod http_api;`.
   - Delete `#[cfg(feature = "roko")] pub mod roko_bridge;`.
   - Keep `pub mod http_health;` (added in M03).
   - Keep all other `pub mod` lines (`fork`, `rpc`, `replay`, `scenario`,
     `persist`, `provider`, `events`, `cow`, `chain_rpc`, `precompiles`,
     `integration`, `rate_limit`, `resources`).

3. In `apps/mirage-rs/src/main.rs`:

   - Remove the `#[cfg(feature = "dashboard-api")] { app = app.merge(...); }`
     block added in M03.
   - Remove the `--enable-hdc`, `--enable-knowledge`, `--enable-stigmergy`
     CLI flags. Update the `clap` derive struct to remove the matching
     fields.
   - Remove any startup banner lines that print "HDC enabled / Knowledge
     enabled / Stigmergy enabled" â€” they no longer have backing state.
   - Keep the slim `http_health::router(...)` mount.

4. Delete the directories (use `git rm -r`):

   ```
   apps/mirage-rs/src/http_api/
   apps/mirage-rs/src/chain/
   apps/mirage-rs/src/roko_bridge/
   ```

5. Delete `apps/mirage-rs/src/integration.rs` only if it references
   `chain::` or `http_api::` types that no longer exist. If it touches
   only EVM machinery, keep it.

6. Run `cargo build -p mirage-rs --all-features` and fix any compile
   errors that surface. Acceptable fixes:
   - Removing `use mirage_rs::chain::...` lines in tests.
   - Removing `use mirage_rs::http_api::...` lines anywhere.
   Unacceptable fixes:
   - Re-introducing the deleted modules under another name.
   - Stubbing functions that used to live in `chain/`.
