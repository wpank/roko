# M05: Drop chain_* JSON-RPC methods from rpc.rs

## Goal

Remove every JSON-RPC method whose name starts with `chain_` from
`apps/mirage-rs/src/rpc.rs`. Keep all `mirage_*` extension methods and
all standard `eth_*` / `net_*` / `web3_*` methods.

## Context

After M04 deleted `apps/mirage-rs/src/chain/`, any RPC method that
called into that module is dead code. The `chain_*` methods were
bolted-on InsightEntry/pheromone helpers; their backing state lives on
roko-serve now.

`mirage_*` methods (e.g. `mirage_fork_to`, `mirage_take_snapshot`,
`mirage_replay_block_range`, `mirage_set_balance`) are simulator
helpers and stay.

## Changes Required

1. In `apps/mirage-rs/src/rpc.rs`, for each method whose RPC name
   begins with `chain_`:
   - Remove its handler arm in the dispatch `match` (typical pattern:
     `"chain_postInsight" => self.chain_post_insight(params).await,`).
   - Remove the corresponding `async fn chain_*` implementation
     function.
   - Remove any `use crate::chain::...` import that the deletion
     orphans.

2. Search for any helper helpers introduced solely for `chain_*` (e.g.
   `parse_insight_params`, `serialize_pheromone_response`). If a
   helper is unused after the deletion, remove it. If it has other
   callers, keep it.

3. Compile-check: `cargo check -p mirage-rs`. The fix-up loop is to
   remove any orphaned imports the compiler flags as unused.

4. Run the existing rpc tests: `cargo test -p mirage-rs --lib rpc`.
   Any test that asserts a `chain_*` method *exists* must be deleted
   (the method is gone). Tests that assert error responses ("method
   not found") should be updated to use the new error.

## Acceptance Criteria

- [ ] `git grep '"chain_' apps/mirage-rs/src/rpc.rs` returns 0 hits.
- [ ] `git grep 'fn chain_' apps/mirage-rs/src/rpc.rs` returns 0 hits.
- [ ] `git grep '"mirage_' apps/mirage-rs/src/rpc.rs` is unchanged from before this batch (preserved).
- [ ] `cargo build -p mirage-rs` succeeds.
- [ ] `cargo test -p mirage-rs --lib` passes.
- [ ] A POST of `{"jsonrpc":"2.0","method":"chain_postInsight",...}` to a running mirage returns an `unknown method` error (not a 500).
- [ ] A POST of `{"jsonrpc":"2.0","method":"eth_blockNumber",...}` still works.

## Do NOT

- Do not touch `mirage_*` methods. They are not application state.
- Do not touch `eth_*`, `net_*`, `web3_*`. Those are standard JSON-RPC.
- Do not stub a placeholder `chain_*` method that returns an error. The
  whole point is method-not-found.
- Do not modify `chain_rpc.rs`. The file name is misleading — it's the
  *chain RPC* (i.e. JSON-RPC) plumbing, not chain-application-state
  helpers.
