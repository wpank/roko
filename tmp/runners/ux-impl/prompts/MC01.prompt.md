# MC01: MCP audit (symbol + call-site + handshake matrix)

## Goal

Produce `docs/v2/MCP-AUDIT.md` recording the actual state of every
`roko-mcp-*` crate. No bucket decisions yet (that's MC02); just the
inputs.

## Context

Five MCP crates exist (`roko-mcp-code`, `-github`, `-slack`, `-scripts`,
`-stdio`). Only `roko-mcp-code` is documented as wired in CLAUDE.md.
The others have unknown ship-gate status.

## Changes Required

1. Create `docs/v2/MCP-AUDIT.md` with this scaffold:

```markdown
# MCP coverage audit (YYYY-MM-DD)

## Matrix

| Crate | LOC | External call sites | Handshake works | Tool listing works | One tool exercise | Bucket | Notes |
|-------|-----|---------------------|-----------------|--------------------|-------------------|--------|-------|
| roko-mcp-code   |  | | | | | | |
| roko-mcp-github |  | | | | | | |
| roko-mcp-slack  |  | | | | | | |
| roko-mcp-scripts|  | | | | | | |
| roko-mcp-stdio  |  | | | | | | |

## Per-crate notes

### roko-mcp-code

**Public symbols** (from `rg -t rust '^pub (struct|fn|trait|enum)' crates/roko-mcp-code/src/`):

(...verbatim grep output...)

**External call sites** (consumers outside the crate):

(...verbatim grep output...)

**Behaviour smoke**: handshake / tools/list / tools/call transcripts
under `tmp/runners/ux-impl/fixtures/mcp-roko-mcp-code-*.txt`.

(repeat for each crate)
```

2. For each crate, run and paste verbatim into the doc:

   ```bash
   crate=roko-mcp-github   # iterate
   echo "## $crate"
   echo "## Public API"
   rg -t rust '^pub (struct|fn|trait|enum)' "crates/$crate/src/"

   echo "## External call sites"
   sym=$(echo "$crate" | tr - _)
   rg --type rust -l "${sym}::" crates/ apps/ | grep -v "crates/$crate/"

   echo "## LOC"
   tokei "crates/$crate/src/"
   ```

3. For each crate, attempt a behaviour smoke. Since Codex can't spawn
   processes:
   - Read `crates/<crate>/src/main.rs` (or `lib.rs`) to identify the
     transport (stdio / TCP / sse).
   - For stdio: classify by reading whether the `tools/list` JSON-RPC
     handler is implemented.
   - Mark "Handshake works" / "Tool listing works" / "One tool exercise"
     as `?` (unknown — needs human run) or `yes`/`no` (verifiable from
     code).

   Acceptable: "yes (from code)" or "?" with a note. Do not fabricate.

4. Document the audit-only scope in the file's header so MC02 can
   safely add the bucket column without conflating decisions with
   evidence.

## Acceptance Criteria

- [ ] `docs/v2/MCP-AUDIT.md` exists.
- [ ] Matrix has all 5 crates with verbatim grep outputs.
- [ ] LOC numbers present per crate.
- [ ] Bucket column exists but is empty / `?` (MC02 fills it).
- [ ] Per-crate sections include the public API and external-call-site greps.

## Do NOT

- Do not claim "yes" for handshake / tool listing without explicit
  verification from source. Use `?` if unsure.
- Do not skip the LOC measurement. It's used for triage in MC02.
- Do not classify `roko-mcp-stdio` as a server without reading
  `lib.rs`. It might be a transport library; mark accordingly.
