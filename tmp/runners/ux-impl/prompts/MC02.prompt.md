# MC02: MCP bucket decisions

## Goal

Fill the `Bucket` column in `docs/v2/MCP-AUDIT.md` for every crate.
One of: `shipped-default`, `shipped-opt-in`, `deprecate`, `wip`.

## Decision rules

- **shipped-default**: external call sites > 0 AND handshake works AND
  tools/list works AND at least one tool exercise verifiable. Wired
  into a default `roko.toml.mcp_config` row.
- **shipped-opt-in**: handshake + tools/list work, but the crate is
  niche or expensive. Operator opts in via config.
- **deprecate**: no external call sites, handshake doesn't work, or
  superseded by another crate.
- **wip**: explicit "not yet" — file a tracking issue and mark
  CLAUDE.md row as `Phase 2+ (in progress)`.

## Changes Required

1. Update `docs/v2/MCP-AUDIT.md`'s matrix with the bucket per crate.

2. For each crate, append a "**Decision** : `<bucket>` — <reason>" line
   under its per-crate section.

3. If a crate is bucketed `deprecate`, add a "**Replaced by** : ..."
   line. If no replacement, write "no replacement; remove from
   workspace under MC03 follow-up".

4. If a crate is bucketed `wip`, add a "**Tracking** : <issue link or
   placeholder>" line. Don't open issues from this batch — record
   intent only.

5. Suggested defaults if the audit is ambiguous:
   - `roko-mcp-code` — shipped-default (it's the reference)
   - `roko-mcp-stdio` — likely a transport library; if so mark
     `transport-only` (a fifth implicit bucket; document and move on)
   - others — shipped-opt-in unless evidence shows otherwise

## Acceptance Criteria

- [ ] Every crate row in the matrix has a non-empty Bucket.
- [ ] Every crate has a `**Decision**` line in its per-crate section.
- [ ] No `deprecate` crate is left without a follow-up plan.
- [ ] CLAUDE.md "Key crates" rows match the buckets *after* MC05; this
  batch only updates the audit doc.

## Do NOT

- Do not move crates out of the workspace in this batch (MC03/MC04 do
  the wiring; deprecation moves are separate).
- Do not bucket every unclear crate as `shipped-default` to "make it
  green". A false-green here causes user-visible breakage when an
  unverified crate fails at runtime.
- Do not add new buckets beyond the four (plus the `transport-only`
  exception for `roko-mcp-stdio` if applicable). Bucket proliferation
  defeats the purpose.
