# MC03: Integration tests for shipped-default MCP crates

## Goal

For every crate bucketed `shipped-default` in `docs/v2/MCP-AUDIT.md`,
add a `tests/integration.rs` that exercises handshake → tools/list →
single tool call.

## Changes Required

1. Read `docs/v2/MCP-AUDIT.md`. For each crate marked
   `shipped-default`, create
   `crates/<crate>/tests/integration.rs` if not present:

```rust
//! Integration test: handshake → tools/list → one tool call.

use std::io::{Read, Write};
use std::process::{Command, Stdio};
use std::time::Duration;

#[test]
fn handshake_lists_tools_and_calls_one() {
    let mut child = Command::new(env!("CARGO_BIN_EXE_<binary-name>"))
        .stdin(Stdio::piped())
        .stdout(Stdio::piped())
        .stderr(Stdio::piped())
        .spawn()
        .expect("spawn MCP server");

    let mut stdin = child.stdin.take().unwrap();
    let mut stdout = child.stdout.take().unwrap();

    // 1) initialize
    writeln!(stdin, r#"{{"jsonrpc":"2.0","id":1,"method":"initialize","params":{{"protocolVersion":"2024-11-05","capabilities":{{}}}}}}"#).unwrap();
    let init = read_one(&mut stdout);
    assert!(init.contains("\"result\""), "initialize response: {init}");

    // 2) tools/list
    writeln!(stdin, r#"{{"jsonrpc":"2.0","id":2,"method":"tools/list","params":{{}}}}"#).unwrap();
    let list = read_one(&mut stdout);
    assert!(list.contains("\"tools\""), "tools/list response: {list}");

    // 3) tools/call (use first tool from the listing — parse list dynamically)
    let v: serde_json::Value = serde_json::from_str(&list).unwrap();
    let first_tool = v["result"]["tools"][0]["name"].as_str().unwrap_or("noop");
    writeln!(stdin, r#"{{"jsonrpc":"2.0","id":3,"method":"tools/call","params":{{"name":"{first_tool}","arguments":{{}}}}}}"#).unwrap();
    let call = read_one(&mut stdout);
    assert!(call.contains("\"id\":3"), "tools/call response: {call}");

    drop(stdin);
    let _ = child.wait_timeout_ms(2_000);   // tolerant cleanup
}

fn read_one(out: &mut std::process::ChildStdout) -> String {
    let mut buf = [0u8; 8192];
    let n = out.read(&mut buf).unwrap();
    String::from_utf8_lossy(&buf[..n]).to_string()
}
```

   Replace `<binary-name>` with the actual binary the crate emits
   (look at its `Cargo.toml` `[[bin]]` entry).

2. If a `shipped-default` crate has *no* binary (it's a library
   embedded inside another binary, e.g. via `roko-mcp-stdio`), write
   the test against the Rust API directly: spin up the dispatch
   handler, write JSON-RPC requests, assert responses.

3. Each test must complete in < 5 s. If a tool call hangs (server
   bug), the test fails fast.

4. Add `#[ignore]` if the test requires an external service (e.g.
   `roko-mcp-github` needs a `GITHUB_TOKEN`). Operators run with
   `cargo test --ignored`.

## Acceptance Criteria

- [ ] One `crates/<crate>/tests/integration.rs` per `shipped-default` crate.
- [ ] `cargo test -p <crate>` passes for each (or `#[ignore]`-d with a documented reason).
- [ ] Each test exercises all three steps (handshake / list / call).
- [ ] No test relies on global state (no `/tmp/<hardcoded>` paths).

## Do NOT

- Do not hit external services in non-ignored tests.
- Do not assert specific tool names in the matrix (each crate's tools
  are different); use the first listed tool for the call.
- Do not silence failing assertions to make CI green.
