# MC04: Update roko.toml.example + docs/v2/MCP-INTEGRATION.md

## Goal

Mirror the bucket decisions from `docs/v2/MCP-AUDIT.md` into the
example config and integration guide.

## Changes Required

1. `roko.toml.example`:
   - Under `[mcp]` (or wherever MCP server config lives), include an
     entry per `shipped-default` crate with sensible defaults.
   - For `shipped-opt-in` crates, include a commented-out template
     with a "Uncomment to enable" line above.
   - For `deprecate` and `wip` crates, do *not* include an entry.

   Example:

   ```toml
   [mcp]
   # ── Shipped, default-on ────────────────────────────────────────
   [[mcp.servers]]
   name = "code"
   command = "roko-mcp-code"
   transport = "stdio"

   # ── Shipped, opt-in ────────────────────────────────────────────
   # Uncomment to enable GitHub MCP. Requires GITHUB_TOKEN in env.
   # [[mcp.servers]]
   # name = "github"
   # command = "roko-mcp-github"
   # transport = "stdio"
   # env = { GITHUB_TOKEN = "$GITHUB_TOKEN" }
   ```

2. `docs/v2/MCP-INTEGRATION.md` (create if absent):

```markdown
# MCP Integration

Roko ships with these MCP servers:

| Server | Crate | Bucket | How to enable |
|--------|-------|--------|---------------|
| code | roko-mcp-code | shipped-default | on by default |
| github | roko-mcp-github | shipped-opt-in | uncomment in roko.toml; set GITHUB_TOKEN |
| ... | ... | ... | ... |

## Configuration

Each server lives under `[[mcp.servers]]` in `roko.toml`. Required
fields:

| Field | Purpose |
|-------|---------|
| `name` | Logical name; referenced by tool calls |
| `command` | Binary to spawn (in `$PATH` or workspace target) |
| `transport` | `stdio` (default) or `tcp` |
| `env` | Optional env-var overrides |

## Audit trail

Bucket decisions live in `docs/v2/MCP-AUDIT.md`. To re-audit, see
`tmp/runners/ux-impl/prompts/MC01.prompt.md`.

## Adding a new MCP server

1. Implement the crate (mirror `roko-mcp-code`).
2. Run the MC01 / MC02 audit flow.
3. Add to `roko.toml.example` under the appropriate bucket.
4. Add an integration test under `crates/<crate>/tests/integration.rs`.
```

   Fill in the per-server rows from the actual buckets in
   `MCP-AUDIT.md`.

## Acceptance Criteria

- [ ] `roko.toml.example` lists every `shipped-default` crate.
- [ ] Every `shipped-opt-in` crate is present as a commented block.
- [ ] `docs/v2/MCP-INTEGRATION.md` exists with the bucket table.
- [ ] No `deprecate` or `wip` crate appears in either file.

## Do NOT

- Do not invent config keys not consumed by the dispatcher. Look at
  `crates/roko-agent/src/dispatcher/mod.rs` (or wherever MCP config is
  parsed) for the actual schema.
- Do not enable secrets-bearing servers (e.g. github, slack) by
  default. Always opt-in.
