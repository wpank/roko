# MC05: CLAUDE.md Key crates rows reflect MCP buckets

## Goal

CLAUDE.md's "Key crates" table tells the truth about each MCP crate
after MC02's bucket decisions.

## Changes Required

1. Open `CLAUDE.md`. Find the "Key crates" section (it's typically a
   markdown table).

2. For each `roko-mcp-*` row, set the status column based on bucket:

   | Bucket | CLAUDE.md status |
   |--------|------------------|
   | shipped-default | Wired |
   | shipped-opt-in | Wired (opt-in) |
   | deprecate | Deprecated; pending removal |
   | wip | Phase 2+ (in progress) |
   | transport-only | Library used by other MCP crates |

3. Add a "Default config" sub-column or a footnote that points readers
   at `roko.toml.example`.

4. If CLAUDE.md aggregates the MCP crates into one row ("roko-mcp-* —
   ..."), expand that row into one per crate so the buckets are
   visible.

## Acceptance Criteria

- [ ] Each `roko-mcp-*` crate has its own row.
- [ ] Status column matches the bucket per the mapping table above.
- [ ] At least one row links to `docs/v2/MCP-INTEGRATION.md`.

## Do NOT

- Do not change unrelated rows in the Key crates table.
- Do not delete `roko-mcp-*` rows for `deprecate` crates yet — keep
  them with the explicit "Deprecated" status.
