# TU01: Learning trio incremental (experiments / gate-thresholds / cascade)

## Goal

Replace the file-stamp polling for `experiments.jsonl`,
`gate-thresholds.json`, and `cascade-router.json` in
`crates/roko-cli/src/tui/dashboard.rs` with `IncrementalTailer<T>` (for
the JSONL) or single-snapshot reload via `fs_watch` (for JSON).

## Context

`dashboard.rs` already uses `IncrementalTailer` for `efficiency.jsonl`
(line ~402) and `c-factor.jsonl` (line ~404). The pattern works.
Lines ~565-586 still poll `learn/experiments`, `learn/gate-thresholds`,
`learn/cascade-router` via `file_stamp(...)`.

Closes followup item 76 in `tmp/ux/ux-followup/12-tui-event-parity.md`.

## Changes Required

1. In `crates/roko-cli/src/tui/dashboard.rs`, identify the three
   stamp+reload sites for the learning trio (search for `file_stamp(&experiments_path)`,
   `file_stamp(&gate_thresholds_path)`, `file_stamp(&cascade_router_path)`).

2. For `experiments.jsonl` (JSONL append-only): convert to
   `IncrementalTailer<ExperimentEvent>`. Pattern:

   ```rust
   pub struct DashboardData {
       // existing fields ...
       efficiency_tailer: IncrementalTailer<AgentEfficiencyEvent>,
       cfactor_tailer: IncrementalTailer<CFactor>,
       experiments_tailer: IncrementalTailer<ExperimentEvent>,    // NEW
   }

   // In tick():
   let added = self.experiments_tailer.tick().unwrap_or(0);
   if added > 0 { generation_changed = true; }
   ```

   Update `experiment_winners` computation to read from
   `self.experiments_tailer.items()` instead of re-reading the file.

3. For `gate-thresholds.json` (single JSON snapshot, not append-only):
   add a one-shot reload triggered by `fs_watch` notifications. Keep
   the parsed value in a `RwLock` or similar; reload on change:

   ```rust
   gate_thresholds_loader: SnapshotLoader<AdaptiveThresholds>,
   ```

   `SnapshotLoader` is a small new helper in `tui/jsonl_tailer.rs` (or
   sibling). Behaviour: lazy, parses on first call after a change; the
   change signal comes from `fs_watch` over `.roko/learn/`.

4. For `cascade-router.json` (single JSON snapshot): same
   `SnapshotLoader` pattern.

5. Where the previous `if stamp_changed { reload_*; generation_changed = true; }`
   blocks lived, remove them. Replace with one call to the tailer's
   `tick()` (JSONL) or the snapshot loader's `tick()` (JSON).

6. Leave `efficiency_tailer` and `cfactor_tailer` alone — they
   already use the right pattern.

## Acceptance Criteria

- [ ] `rg 'file_stamp\\(&experiments_path' crates/roko-cli/src/tui/dashboard.rs` returns 0 hits.
- [ ] `rg 'file_stamp\\(&gate_thresholds_path' crates/roko-cli/src/tui/dashboard.rs` returns 0 hits.
- [ ] `rg 'file_stamp\\(&cascade_router_path' crates/roko-cli/src/tui/dashboard.rs` returns 0 hits.
- [ ] `cargo build -p roko-cli` succeeds.
- [ ] `cargo test -p roko-cli --lib tui` passes.
- [ ] When a fixture writes to `.roko/learn/experiments.jsonl`, the
  `experiment_winners` reflect the new entry within one tick.
- [ ] No `IncrementalTailer<T>` is constructed where `T` doesn't actually parse from the JSONL line shape — sanity-check at first read.

## Do NOT

- Do not introduce new polling. The whole point of the wave is
  removing it.
- Do not unify `gate-thresholds.json` and `cascade-router.json` parsing
  into one struct. They have different schemas.
- Do not write the `SnapshotLoader` helper inside `dashboard.rs`. Place
  it in `crates/roko-cli/src/tui/jsonl_tailer.rs` (or a new sibling
  `snapshot_loader.rs`) so other panels can reuse it.
