# TU02: Episode log incremental tail with bounded ring buffer

## Goal

Replace the `file_stamp` poll for `.roko/episodes.jsonl` (around
`dashboard.rs:561`) with `IncrementalTailer<Episode>`.

## Context

Followup item 73 in `tmp/ux/ux-followup/12-tui-event-parity.md`.

## Changes Required

1. Add a tailer field to `DashboardData`:

   ```rust
   episodes_tailer: IncrementalTailer<Episode>,
   ```

   `Episode` lives at `crates/roko-learn/src/episode_logger.rs`.

2. Initialise:

   ```rust
   let mut episodes_tailer = IncrementalTailer::<Episode>::new(&episodes_path);
   let _ = episodes_tailer.tick();   // catch up on existing file
   ```

3. Replace the polling block (search for `refresh_episodes(&episodes_path, stamp)`):

   ```rust
   let added = self.episodes_tailer.tick().unwrap_or(0);
   if added > 0 {
       generation_changed = true;
       self.cap_episodes_to(10_000);
   }
   ```

4. Add a `cap_episodes_to(max: usize)` helper:

   ```rust
   fn cap_episodes_to(&mut self, max: usize) {
       let len = self.episodes_tailer.len();
       if len > max {
           // IncrementalTailer doesn't expose drain; the cleanest path
           // is to add a `pub fn truncate_oldest(&mut self, n: usize)`
           // method on IncrementalTailer in jsonl_tailer.rs and call it here.
           self.episodes_tailer.truncate_oldest(len - max);
       }
   }
   ```

   Add `pub fn truncate_oldest(&mut self, n: usize)` in
   `crates/roko-cli/src/tui/jsonl_tailer.rs`:

   ```rust
   /// Drop the first `n` items. Cursor offset is unchanged so future
   /// ticks pick up from where they left off.
   pub fn truncate_oldest(&mut self, n: usize) {
       let drop = n.min(self.items.len());
       self.items.drain(0..drop);
   }
   ```

5. Consumers (e.g. `views/episodes_view.rs`) that previously read from
   `data.episodes_state.entries` now read from
   `data.episodes_tailer.items()`.

6. Remove the `refresh_episodes` method and the `episodes_stamp` field
   from `DashboardData` if no other caller remains.

## Acceptance Criteria

- [ ] `rg 'refresh_episodes\\b' crates/roko-cli/src/tui/dashboard.rs` returns 0 hits.
- [ ] `rg 'file_stamp\\(&episodes_path' crates/roko-cli/src/tui/dashboard.rs` returns 0 hits.
- [ ] `cargo build -p roko-cli` succeeds.
- [ ] `cargo test -p roko-cli --lib tui` passes.
- [ ] After 10 001 episodes are appended, `data.episodes_tailer.len() == 10_000`.
- [ ] Adding a single episode after a steady-state run reflects in the
  Episodes view within one TUI tick.

## Do NOT

- Do not unbound the tailer's internal Vec. Memory matters for long
  sessions.
- Do not remove episodes by re-reading the file from offset 0. The
  cap is on in-memory state only; the on-disk JSONL stays append-only.
- Do not mutate `Episode` struct fields to fit the dashboard. The
  dashboard renders what `roko-learn` writes.
