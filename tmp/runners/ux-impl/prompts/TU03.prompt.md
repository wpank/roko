# TU03: Gate signals incremental tail

## Goal

Replace the per-tick re-parse of `.roko/signals.jsonl` (around
`dashboard.rs:414`) with `IncrementalTailer<SignalEntry>`.

## Context

Followup item 71. Today the helpers
`load_signal_state(&signals_path)`, `gate_signal_summaries`, and
`signal_gate_results` re-parse the whole file on every refresh.

## Changes Required

1. Add a tailer:

   ```rust
   signals_tailer: IncrementalTailer<SignalEntry>,
   ```

   `SignalEntry` lives wherever the existing `load_signal_state`
   loader expects (likely `crates/roko-learn/...` or local to
   `dashboard.rs`).

2. Refactor `load_signal_state(&signals_path) -> (Vec<RecentSignal>, GateSummaries, GateResults, SignalState)`
   into a *reducer* that takes the previous accumulated state plus the
   slice of newly-tailed entries:

   ```rust
   fn fold_signal_state(
       prev: SignalState,
       new_entries: &[SignalEntry],
   ) -> (Vec<RecentSignal>, GateSummaries, GateResults, SignalState) {
       // ... existing fold logic, but starting from `prev` instead of empty ...
   }
   ```

   The `prev` argument captures running totals (per-gate counters,
   recent ring) so the per-tick cost is O(new) not O(N).

3. In `tick()`, replace the existing block with:

   ```rust
   let added = self.signals_tailer.tick().unwrap_or(0);
   if added > 0 {
       let new_slice = &self.signals_tailer.items()[self.signals_tailer.len() - added..];
       let (recent, summaries, results, state) =
           fold_signal_state(self.signal_state.clone(), new_slice);
       self.recent_signals = recent;
       self.gate_signal_summaries = summaries;
       self.signal_gate_results = results;
       self.signal_state = state;
       generation_changed = true;
   }
   ```

4. Cap `signals_tailer` items at 50 000 with the same
   `truncate_oldest` helper from TU02.

## Acceptance Criteria

- [ ] `rg 'load_signal_state\\(' crates/roko-cli/src/tui/dashboard.rs` returns at most one hit (its definition; no callers).
- [ ] `rg 'file_stamp\\(&signals_path' crates/roko-cli/src/tui/dashboard.rs` returns 0 hits.
- [ ] `cargo build -p roko-cli` and `cargo test -p roko-cli` pass.
- [ ] Adding a single signal entry to the JSONL surfaces in the Gate
  tab within one TUI tick.
- [ ] `data.signals_tailer.len() <= 50_000`.

## Do NOT

- Do not delete the existing `load_signal_state` immediately. Keep it
  as a one-shot warm-start helper if needed:
  `let warm = load_signal_state(&signals_path); /* seed self.signal_state */`.
  Then delete after the warm-start is removed in a follow-up.
- Do not mix the warm start and the tailer's first `tick()`. They
  cover different ranges; pick one or the other.
- Do not add a new file format. Keep `signals.jsonl` as is.
