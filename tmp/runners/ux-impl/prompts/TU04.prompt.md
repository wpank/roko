# TU04: Task outputs watched per-file via fs_watch (cap 20 concurrent)

## Goal

Stop walking `.roko/task-outputs/` on every refresh. Subscribe to
`fs_watch` events and tail only changed files.

## Context

Followup item 72. Today `dashboard.rs:448-451` calls `load_task_outputs`
which walks the directory + tails each file every tick.

## Changes Required

1. Move the per-file logic into
   `crates/roko-cli/src/tui/task_outputs.rs`:

   ```rust
   pub struct TaskOutputs {
       cursors: HashMap<PathBuf, JsonlCursor>,   // per file offset
       tails: HashMap<PathBuf, RingBuffer<String>>,   // per file last N lines
       cap_files: usize,
   }

   impl TaskOutputs {
       pub fn new(cap_files: usize) -> Self {
           Self { cursors: HashMap::new(), tails: HashMap::new(), cap_files }
       }

       /// Apply a batch of fs_watch events. Returns the set of file
       /// paths that produced new lines (so the caller can decide
       /// whether render state changed).
       pub fn apply_events(
           &mut self,
           events: &[FsWatchEvent],
       ) -> Vec<PathBuf> {
           let mut changed = Vec::new();
           for ev in events {
               if !ev.path.is_file() { continue; }
               let cursor = self.cursors.entry(ev.path.clone())
                   .or_insert_with(|| JsonlCursor::new(&ev.path));
               match cursor.read_new_lines() {
                   Ok(lines) if !lines.is_empty() => {
                       let tail = self.tails.entry(ev.path.clone())
                           .or_insert_with(|| RingBuffer::with_capacity(200));
                       for line in lines { tail.push(line); }
                       changed.push(ev.path.clone());
                   }
                   Ok(_) => {}
                   Err(e) => tracing::debug!(?e, ?ev.path, "task output read failed"),
               }
           }
           // Cap concurrent files. Drop the least-recently-changed.
           while self.cursors.len() > self.cap_files {
               // Drop oldest by... need a recency stamp. Easiest: keep
               // an LRU vec of paths in self.lru.
           }
           changed
       }

       pub fn tail_for(&self, path: &Path) -> Option<&RingBuffer<String>> {
           self.tails.get(path)
       }
   }
   ```

   `RingBuffer` either reuses an existing helper (search the workspace
   for `RingBuffer<` first) or implement inline as a tiny `VecDeque`
   wrapper with a cap.

   Implement an LRU vec of paths to make the "drop oldest" cap real.

2. In `dashboard.rs`:
   - Add field `task_outputs: TaskOutputs`.
   - Initialise: `task_outputs: TaskOutputs::new(20)`.
   - In `tick()`, replace the directory-walk block with consumption of
     `fs_watch_rx` events that targeted `.roko/task-outputs/`.

3. The `fs_watch.rs` module already produces a stream of refresh
   events. If it doesn't include path filtering, add a helper that
   filters events to a sub-tree:

   ```rust
   impl FsWatchHandle {
       pub fn try_recv_under<'a>(
           &'a mut self,
           prefix: &'a Path,
       ) -> impl Iterator<Item = FsWatchEvent> + 'a { ... }
   }
   ```

4. Consumers (the Tasks panel) read from
   `data.task_outputs.tail_for(&path)`.

## Acceptance Criteria

- [ ] `rg 'load_task_outputs\\b' crates/roko-cli/src/tui/dashboard.rs` returns 0 hits at the call site (definition can stay in `task_outputs.rs` if useful).
- [ ] `cargo build -p roko-cli` and `cargo test -p roko-cli` pass.
- [ ] Appending to one file under `.roko/task-outputs/` re-tails *only* that file (assert via tracing log line count).
- [ ] After 21 distinct files are touched, the oldest 1 has its cursor + tail dropped.

## Do NOT

- Do not poll. The whole point is event-driven.
- Do not store the entire file body in memory; tail length 200 lines
  per file is enough.
- Do not abuse `tokio::fs::read_dir`. The watcher provides events;
  trust them.
