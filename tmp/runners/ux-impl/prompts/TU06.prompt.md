# TU06: Persist generation counter to .roko/state/dashboard-gen.json

## Goal

The `OnceLock<HashMap<PathBuf, DashboardGenerationState>>` in
`dashboard.rs:104-110` resets to 0 on every process restart. Persist
the map to disk so cross-process / multi-tab consumers see monotonic
generation numbers.

## Context

Followup item 78. Atomic write on update; load once at startup.

## Changes Required

1. Create `crates/roko-cli/src/tui/dashboard_gen_persist.rs`:

```rust
//! Persist + load the dashboard generation counter map.
//!
//! Stored at `.roko/state/dashboard-gen.json` as a single JSON object:
//! `{ "<canonical-path>": { "fingerprint": <u64>, "generation": <u64> } }`.

use std::collections::HashMap;
use std::path::{Path, PathBuf};

use serde::{Deserialize, Serialize};

const STATE_FILE: &str = "dashboard-gen.json";
const MAX_ENTRIES: usize = 1_000;

#[derive(Debug, Clone, Copy, Default, Serialize, Deserialize)]
pub struct DashboardGenerationState {
    pub fingerprint: u64,
    pub generation: u64,
}

#[derive(Debug, Default, Serialize, Deserialize)]
pub struct GenerationStore {
    pub entries: HashMap<String, DashboardGenerationState>,
}

impl GenerationStore {
    pub fn path_under(state_dir: &Path) -> PathBuf {
        state_dir.join(STATE_FILE)
    }

    pub fn load(state_dir: &Path) -> Self {
        let path = Self::path_under(state_dir);
        let Ok(raw) = std::fs::read_to_string(&path) else {
            return Self::default();
        };
        serde_json::from_str(&raw).unwrap_or_default()
    }

    pub fn save(&self, state_dir: &Path) -> std::io::Result<()> {
        let path = Self::path_under(state_dir);
        if let Some(parent) = path.parent() {
            std::fs::create_dir_all(parent)?;
        }
        let tmp = path.with_extension("tmp");
        let body = serde_json::to_vec_pretty(self)
            .map_err(|e| std::io::Error::new(std::io::ErrorKind::InvalidData, e))?;
        std::fs::write(&tmp, body)?;
        std::fs::rename(&tmp, &path)?;
        Ok(())
    }

    pub fn cap_to(&mut self, max: usize) {
        if self.entries.len() <= max { return; }
        // Drop the lowest generation values (= least recently bumped).
        let mut sorted: Vec<_> = self.entries.iter()
            .map(|(k, v)| (k.clone(), v.generation))
            .collect();
        sorted.sort_by_key(|(_, gen)| *gen);
        let drop_count = self.entries.len() - max;
        for (k, _) in sorted.into_iter().take(drop_count) {
            self.entries.remove(&k);
        }
    }
}

#[cfg(test)]
mod tests {
    use super::*;
    use tempfile::tempdir;

    #[test]
    fn round_trip() {
        let d = tempdir().unwrap();
        let mut s = GenerationStore::default();
        s.entries.insert("x.jsonl".into(), DashboardGenerationState { fingerprint: 7, generation: 3 });
        s.save(d.path()).unwrap();
        let loaded = GenerationStore::load(d.path());
        assert_eq!(loaded.entries.get("x.jsonl").unwrap().generation, 3);
    }

    #[test]
    fn cap_drops_lowest() {
        let mut s = GenerationStore::default();
        for i in 0..1_005u64 {
            s.entries.insert(format!("p{i}"), DashboardGenerationState { fingerprint: 0, generation: i });
        }
        s.cap_to(1_000);
        assert_eq!(s.entries.len(), 1_000);
        assert!(!s.entries.contains_key("p0"));      // dropped
        assert!(s.entries.contains_key("p1004"));    // kept (highest gen)
    }
}
```

## Acceptance Criteria

- [ ] `crates/roko-cli/src/tui/dashboard_gen_persist.rs` exists.
- [ ] `cargo build -p roko-cli` and `cargo test -p roko-cli --lib dashboard_gen_persist` pass.
- [ ] After a TUI session, `.roko/state/dashboard-gen.json` exists and contains the latest counters.
- [ ] On the next session, the counter resumes from where it was (verify via a test fixture: write the file, load, assert).
- [ ] Map size never exceeds 1 000 entries.

## Do NOT

- Do not save on every render frame; only when `generation` actually
  bumps. Hot-loop disk writes will burn SSD endurance.
- Do not use `serde_json::to_string_pretty` if size matters; for 1 000
  entries pretty is fine (~80 KB).
- Do not use `tokio::fs` for the save path. The current update path
  is sync; mixing introduces a runtime requirement where there's
  none.
