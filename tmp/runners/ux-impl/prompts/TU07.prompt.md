# TU07: Agent panel WS multiplex via aggregator /api/ws

## Goal

When the user opens the Agents tab, subscribe to the aggregator's
`/api/ws` multiplex and render incoming events in the agent detail
panel. Tear down on tab change.

## Context

Followup item 70. Today the Agents tab reads agent state from the
parsed `executor.json` snapshot and lags by the polling interval
(500 ms + write latency). The aggregator now multiplexes all agent
`/stream` WS into one `/api/ws` connection (already shipped in
`crates/roko-serve/src/routes/aggregator.rs`).

## Changes Required

1. New module `crates/roko-cli/src/tui/agent_streams.rs`:

```rust
//! Subscribe to roko-serve's /api/ws agent stream multiplex while the
//! Agents tab is active. Push events into TuiState::agent_streams.

use std::collections::{HashMap, VecDeque};
use std::sync::Arc;
use std::time::Duration;

use serde::Deserialize;
use tokio::sync::{Mutex, mpsc};
use tokio::task::JoinHandle;
use tokio_tungstenite::connect_async;
use futures::StreamExt;

const RING_PER_AGENT: usize = 200;
const RECONNECT_INITIAL: Duration = Duration::from_millis(250);
const RECONNECT_CAP: Duration = Duration::from_secs(10);

#[derive(Debug, Clone, Deserialize)]
pub struct StreamEvent {
    pub source: String,
    #[serde(flatten)]
    pub payload: serde_json::Value,
}

pub struct AgentStreams {
    rings: Mutex<HashMap<String, VecDeque<StreamEvent>>>,
}

impl AgentStreams {
    pub fn new() -> Arc<Self> {
        Arc::new(Self { rings: Mutex::new(HashMap::new()) })
    }

    pub async fn snapshot(&self, agent_id: &str) -> Vec<StreamEvent> {
        self.rings.lock().await.get(agent_id).cloned().map(|d| d.into_iter().collect()).unwrap_or_default()
    }

    pub async fn ingest(&self, ev: StreamEvent) {
        let mut g = self.rings.lock().await;
        let ring = g.entry(ev.source.clone()).or_insert_with(|| VecDeque::with_capacity(RING_PER_AGENT));
        if ring.len() == RING_PER_AGENT { ring.pop_front(); }
        ring.push_back(ev);
    }

    pub async fn clear(&self) {
        self.rings.lock().await.clear();
    }
}

pub struct AgentStreamsHandle {
    pub streams: Arc<AgentStreams>,
    join: JoinHandle<()>,
}

impl AgentStreamsHandle {
    /// Spawn a WS consumer connected to `ws_url` (e.g. `ws://127.0.0.1:6677/api/ws`).
    pub fn start(ws_url: String) -> Self {
        let streams = AgentStreams::new();
        let s = Arc::clone(&streams);
        let join = tokio::spawn(async move {
            let mut delay = RECONNECT_INITIAL;
            loop {
                match connect_async(&ws_url).await {
                    Ok((mut ws, _)) => {
                        delay = RECONNECT_INITIAL;   // reset back-off
                        while let Some(msg) = ws.next().await {
                            let Ok(msg) = msg else { break };
                            let Some(text) = msg.to_text().ok() else { continue };
                            let Ok(ev) = serde_json::from_str::<StreamEvent>(text) else { continue };
                            s.ingest(ev).await;
                        }
                    }
                    Err(err) => {
                        tracing::debug!(?err, ?ws_url, "WS connect failed; backing off");
                    }
                }
                tokio::time::sleep(delay).await;
                delay = (delay * 2).min(RECONNECT_CAP);
            }
        });
        Self { streams, join }
    }

    pub fn stop(self) {
        self.join.abort();
    }
}
```

2. In `crates/roko-cli/src/tui/state.rs`, add:

   ```rust
   pub agent_streams: Option<Arc<crate::tui::agent_streams::AgentStreams>>,
   ```

   Default `None`. Populated when the Agents tab activates.

3. In `crates/roko-cli/src/tui/views/agents_view.rs`:
   - On Agents-tab activation, if `state.agent_streams.is_none()`,
     spawn `AgentStreamsHandle::start(ws_url)` and store the handle on
     a tab-local field. Set `state.agent_streams = Some(handle.streams.clone())`.
   - On Agents-tab deactivation, call `handle.stop()` and set
     `state.agent_streams = None`.
   - Each render: for the currently-selected agent, call
     `state.agent_streams.as_ref().map(|s| s.snapshot_blocking(agent_id))`.
     Since the snapshot is async, store the latest value on tab state
     and refresh on each WS event.

   Concretely, you'll need a `tokio::sync::watch` channel that the WS
   task writes the most-recent snapshot into; the render thread reads
   the latest `borrow()`.

## Acceptance Criteria

- [ ] `crates/roko-cli/src/tui/agent_streams.rs` exists.
- [ ] `cargo build -p roko-cli` and `cargo test -p roko-cli --lib tui::agent_streams` pass.
- [ ] Activating the Agents tab opens exactly one WS connection to roko-serve's `/api/ws`.
- [ ] Switching to a different tab aborts the WS consumer.
- [ ] An event landing on the WS surfaces in the agent detail panel within ~200 ms.
- [ ] Per-agent ring is bounded at 200 events; older events drop off.
- [ ] WS reconnects with exponential back-off (250 ms → 500 ms → 1 s → ... cap 10 s).

## Do NOT

- Do not subscribe per-agent (one WS per agent). The aggregator
  multiplex is the right level. N*M scales poorly.
- Do not block the render thread on the WS. Forward via `mpsc` /
  `watch`; the render reads the latest snapshot.
- Do not unbound the ring buffer. 200 events × 5 KB worst-case ~=
  1 MB per agent, OK for an interactive tail.
- Do not use `std::sync::Mutex`. The ingest path is async.
