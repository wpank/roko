## Task
Replace ACP raw subprocess dispatch with roko-agent Dispatcher

## Runner Context
You are working in runner `mega-parity`, batch R3_F01.
This batch is part of Runner 3: agent-session-parity — Make interactive roko and one-shot use a real agent session via existing adapters.

## Problem
ACP's `runner.rs` spawns pipeline agents (Strategist, Implementer, AutoFixer, Reviewer) via raw `Command::new("claude").arg("--print").arg("--dangerously-skip-permissions")` subprocess calls. This bypasses the entire roko-agent dispatch infrastructure — no provider routing, no MCP passthrough, no tool dispatch, no session management.

The rest of roko uses `roko-agent::Dispatcher` which supports 8 backends (Claude CLI, Claude API, OpenAI-compat, Codex, Ollama, Gemini, Perplexity, Moonshot). ACP should use the same path.

## Architecture Contract
- CP-1: One session struct — ACP sessions still own their state
- CP-2: Delegate to existing adapters — use roko-agent Dispatcher, not hand-rolled subprocess
- ACP-1: One dispatch path — all pipeline phases go through Dispatcher

## Changes Required

1. **In `crates/roko-acp/Cargo.toml`**: Ensure `roko-agent` dependency includes the dispatcher module (it's already a dep, but verify features)

2. **In `crates/roko-acp/src/runner.rs`**: Replace `run_claude_cli()` helper with a call through the Dispatcher:

```rust
// BEFORE (raw subprocess):
fn run_claude_cli(prompt: &str, system_prompt: &str, model: &str) -> Result<String> {
    let output = Command::new("claude")
        .arg("--print")
        .arg("--dangerously-skip-permissions")
        .arg("--model").arg(model)
        .arg("--system-prompt").arg(system_prompt)
        .arg(prompt)
        .output()?;
    ...
}

// AFTER (through Dispatcher):
async fn dispatch_pipeline_phase(
    dispatcher: &Dispatcher,
    prompt: &str,
    system_prompt: &str,
    model: &str,
    role: &str,
) -> Result<String> {
    let request = DispatchRequest {
        model: model.to_string(),
        system_prompt: system_prompt.to_string(),
        messages: vec![Message::user(prompt)],
        tools: vec![], // pipeline phases don't use tools directly
        ..Default::default()
    };
    dispatcher.dispatch(request).await
}
```

3. **In `runner.rs`**: Initialize Dispatcher from workspace config at pipeline start. Pass it to each phase execution function.

4. **Remove** the `run_claude_cli` function entirely. All 4 call sites (SpawnStrategist, SpawnImplementer, SpawnAutoFixer, SpawnReviewer) use the new `dispatch_pipeline_phase`.
