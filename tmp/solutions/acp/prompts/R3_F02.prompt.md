## Task
Wire SystemPromptBuilder into ACP session dispatch

## Runner Context
You are working in runner `mega-parity`, batch R3_F02.
This batch is part of Runner 3: agent-session-parity — Make interactive roko and one-shot use a real agent session via existing adapters.

## Problem
ACP uses hardcoded static strings for system prompts:
```rust
const CODE_MODE_SYSTEM_PROMPT: &str = "You are a senior software engineer...";
const PLAN_MODE_SYSTEM_PROMPT: &str = "You are a planning assistant...";
const RESEARCH_MODE_SYSTEM_PROMPT: &str = "You are a research analyst...";
```

Meanwhile, `orchestrate.rs` uses the 9-layer `SystemPromptBuilder` from `roko-compose` that includes: base identity, role specialization, workspace context, tool inventory, safety constraints, playbook injection, knowledge context, session history, and task-specific instructions.

ACP should use the same builder so agents get the same rich context regardless of entry point.

## Architecture Contract
- ACP-2: One prompt assembly — use SystemPromptBuilder, remove static strings
- CP-2: Delegate to existing adapters

## Changes Required

1. **In `crates/roko-acp/Cargo.toml`**: Add `roko-compose = { path = "../roko-compose" }`

2. **In `crates/roko-acp/src/session.rs`**:
   - Remove `CODE_MODE_SYSTEM_PROMPT`, `PLAN_MODE_SYSTEM_PROMPT`, `RESEARCH_MODE_SYSTEM_PROMPT` constants
   - Add a method `build_system_prompt(&self) -> String` that calls SystemPromptBuilder

3. **In `crates/roko-acp/src/bridge_events.rs`**:
   - Where system_prompt is passed to dispatch, call `session.build_system_prompt()` instead of the static constant
   - For pipeline phases (Strategist, Implementer, etc.), pass appropriate role to builder

4. **Map ACP session state → builder params:**
   ```rust
   use roko_compose::SystemPromptBuilder;

   impl AcpSession {
       pub fn build_system_prompt(&self) -> String {
           let mut builder = SystemPromptBuilder::new();
           builder
               .with_mode(self.config_state.agent_mode.clone())
               .with_workspace_root(&self.workspace_root)
               .with_temperament(self.config_state.temperament.clone());

           // Inject conversation context if available
           if !self.conversation_history.is_empty() {
               builder.with_history_summary(&self.summarize_history());
           }

           builder.build()
       }
   }
   ```
