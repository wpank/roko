## Task
Wire safety contract enforcement into ACP dispatch

## Runner Context
You are working in runner `mega-parity`, batch R3_F03.
This batch is part of Runner 3: agent-session-parity — Make interactive roko and one-shot use a real agent session via existing adapters.

## Problem
ACP dispatches agents with no safety checks. In `orchestrate.rs`, every dispatch goes through `AgentContract` enforcement — role-based auth, tool restrictions, path constraints, and pre/post checks. ACP skips all of this.

This means an ACP session in "research" mode can still write files, and a "code" mode agent has no tool restrictions at all.

## Architecture Contract
- ACP-5: Session state is shared-nothing except via shared services
- CP-2: Delegate to existing adapters

## Changes Required

1. **In `crates/roko-acp/src/handler.rs`**: Before dispatching a prompt, load and check the safety contract:
   ```rust
   // Before dispatch
   let contract = load_contract_for_mode(&session.config_state.agent_mode)?;
   contract.validate_dispatch(&prompt, &session.config_state)?;
   ```

2. **In `crates/roko-acp/src/bridge_events.rs`**: After loading contract, pass tool restrictions to the Dispatcher so the agent doesn't see tools it shouldn't use:
   ```rust
   let allowed_tools = contract.allowed_tools();
   // Pass to dispatcher request
   ```

3. **Graceful fallback**: If contract YAML doesn't exist for a mode, log a warning and use permissive default (matches existing orchestrate.rs behavior):
   ```rust
   let contract = match load_contract_for_mode(mode) {
       Ok(c) => c,
       Err(_) => {
           tracing::warn!("No contract for mode {mode:?}, using permissive default");
           AgentContract::permissive()
       }
   };
   ```

4. **Mode → contract mapping:**
   - `code` → allows all tools, all paths
   - `plan` → allows read + analysis tools, restricts file writes
   - `research` → read-only (no file writes, no terminal execution)
