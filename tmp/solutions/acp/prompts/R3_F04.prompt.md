## Problem
ACP agents currently run with no permission checks. The ACP spec defines `session/request_permission` — a bidirectional RPC where the agent asks the editor for permission before performing a sensitive action (file edit, terminal command, etc.). The editor renders this as a dialog card with Allow / Always Allow / Reject buttons.

Without this, roko uses `--dangerously-skip-permissions` and the user has no opportunity to review or reject file edits, which is unsafe for production-tracked files.

- ACP-1: One dispatch path

## Changes Required

1. **In `crates/roko-acp/src/types.rs`**: Add permission types:

```rust
/// Permission request sent from agent to client.
#[derive(Debug, Clone, PartialEq, Serialize, Deserialize)]
#[serde(rename_all = "camelCase")]
pub struct RequestPermissionParams {
    pub session_id: String,
    /// The tool call that needs permission.
    pub tool_call: PermissionToolCall,
    /// Available permission options.
    pub options: Vec<PermissionOption>,
}

#[derive(Debug, Clone, PartialEq, Serialize, Deserialize)]
#[serde(rename_all = "camelCase")]
pub struct PermissionToolCall {
    pub tool_call_id: String,
    pub title: String,
    pub kind: ToolCallKind,
    /// Rich description (file path, line range, metadata).
    pub content: Vec<ContentBlock>,
}

#[derive(Debug, Clone, PartialEq, Serialize, Deserialize)]
#[serde(rename_all = "snake_case")]
pub enum PermissionOption {
    AllowOnce,
    AllowAlways,
    RejectOnce,
    RejectAlways,
}

#[derive(Debug, Clone, PartialEq, Serialize, Deserialize)]
#[serde(rename_all = "camelCase")]
pub struct PermissionResponse {
    pub tool_call_id: String,
    pub decision: PermissionOption,
}
```

2. **In `crates/roko-acp/src/handler.rs`**: Add `session/request_permission` as a server-to-client request. This is a **reverse direction** call — the ACP server sends a request to the editor and waits for a response.

3. **In `crates/roko-acp/src/runner.rs`**: Before file-modifying tool calls in pipeline phases, send a permission request and wait for the response:

```rust
// Before file edit
let response = request_permission(
    session_id,
    PermissionToolCall {
        tool_call_id: format!("perm-{}", uuid()),
        title: format!("Edit {}?", file_path),
        kind: ToolCallKind::Edit,
        content: vec![ContentBlock::Text {
            text: format!("{} · {}", line_range, file_tracking_status),
        }],
    },
    vec![PermissionOption::AllowOnce, PermissionOption::AllowAlways, PermissionOption::RejectOnce],
).await?;

match response.decision {
    PermissionOption::AllowOnce | PermissionOption::AllowAlways => { /* proceed */ }
    _ => { /* skip this edit, log warning */ }
}
```

4. **Track "always allow" decisions** in session state so repeated edits to the same path don't re-prompt.
