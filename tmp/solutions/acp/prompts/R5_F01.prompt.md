## Task
Wire ACP episode logging (begin_turn / end_turn / close)

## Runner Context
You are working in runner `mega-parity`, batch R5_F01.
This batch is part of Runner 5: telemetry-learning — Make cost, usage, episodes, learning, and cascade router feedback truthful.

## Problem
ACP sessions produce zero episode data. Every dispatch in `orchestrate.rs` logs episodes to `.roko/episodes.jsonl` via `EpisodeLogger`. ACP dispatches are invisible to the learning system — no record of what models were used, what succeeded, what failed, how many tokens were consumed.

This means the cascade router can't learn from editor usage, efficiency tracking is blind to ACP, and playbook generation has no ACP data to work from.

## Architecture Contract
- TL-1: Unknown ≠ zero — store None, not 0 for missing data
- TL-3: Model is known before logging
- ACP-3: No silent learning gaps — every dispatch MUST emit an episode turn

## Changes Required

1. **In `crates/roko-acp/Cargo.toml`**: Add `roko-learn = { path = "../roko-learn" }` if not already present

2. **In `crates/roko-acp/src/bridge_events.rs`**: Initialize EpisodeLogger and call it around dispatch:

```rust
use roko_learn::EpisodeLogger;

// At dispatch start:
let episode_id = logger.begin_episode(EpisodeStart {
    session_id: session.session_id.clone(),
    mode: session.config_state.agent_mode.clone(),
    model: resolved_model.clone(),
    entry_point: "acp".to_string(),
});

// Per turn:
logger.record_turn(EpisodeTurn {
    episode_id: episode_id.clone(),
    role: "user",
    content_preview: prompt[..200.min(prompt.len())].to_string(),
    tokens: None, // filled on response
});

// On response complete:
logger.record_turn(EpisodeTurn {
    episode_id: episode_id.clone(),
    role: "assistant",
    content_preview: response[..200.min(response.len())].to_string(),
    tokens: Some(usage.output_tokens),
});

// On session end or pipeline complete:
logger.close_episode(episode_id, EpisodeOutcome {
    success: !halted,
    gate_results: gate_verdicts.clone(),
    total_tokens: accumulated_tokens,
});
```

3. **In `crates/roko-acp/src/runner.rs`**: For pipeline workflows, each phase (strategist, implementer, auto-fixer, reviewer) gets its own turn logged within the same episode.

4. **Pass logger**: Thread `EpisodeLogger` (or `Arc<EpisodeLogger>`) from `run_acp_server()` through to bridge_events and runner.
