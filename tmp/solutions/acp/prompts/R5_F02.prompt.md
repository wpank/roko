## Task
Feed ACP dispatch outcomes into CascadeRouter

## Runner Context
You are working in runner `mega-parity`, batch R5_F02.
This batch is part of Runner 5: telemetry-learning — Make cost, usage, episodes, learning, and cascade router feedback truthful.

## Problem
The CascadeRouter learns which models work best for which task types by observing outcomes. ACP dispatches produce outcomes (success/failure, latency, cost) but never report them to the router. This means editor usage — likely the highest-volume dispatch path — contributes zero signal to model routing optimization.

## Architecture Contract
- TL-5: Learning reads what execution writes
- ACP-3: No silent learning gaps

## Changes Required

1. **In `crates/roko-acp/src/bridge_events.rs`**: After each dispatch completes, report to CascadeRouter:

```rust
use roko_learn::CascadeRouter;

// After successful dispatch:
router.record_observation(RouterObservation {
    model: resolved_model.clone(),
    task_hint: classify_prompt(&prompt), // "code", "plan", "research", "review"
    success: true,
    latency_ms: elapsed.as_millis() as u64,
    tokens_used: usage.map(|u| u.input_tokens + u.output_tokens),
    cost_usd: computed_cost,
});

// After failed dispatch (timeout, error, gate failure):
router.record_observation(RouterObservation {
    model: resolved_model.clone(),
    task_hint: classify_prompt(&prompt),
    success: false,
    latency_ms: elapsed.as_millis() as u64,
    tokens_used: None,
    cost_usd: None,
});
```

2. **In `crates/roko-acp/src/runner.rs`**: For pipeline workflows, report per-phase:
   - Strategist success/fail
   - Implementer success/fail
   - AutoFixer success/fail (counts as negative signal for original model)
   - Reviewer success/fail

3. **Query router for model suggestion** when `routing_mode == "auto"`:
```rust
if session.config_state.routing_mode == "auto" {
    let suggestion = router.suggest(task_hint, effort_level);
    resolved_model = suggestion.model;
}
```

4. **Respect manual override**: When `routing_mode == "manual"`, use session's configured model directly (no router query). Still record the outcome for learning.
