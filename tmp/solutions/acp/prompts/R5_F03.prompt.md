## Problem
`WorkflowRun` has `total_cost_usd: f64` and `total_tokens: u64` fields that are always 0. Provider responses include token counts (and sometimes cost), but this data is never accumulated. The editor shows $0.00 for every workflow run.

Additionally, ACP never emits `UsageUpdate` notifications to the editor — the type is defined in `types.rs` but never constructed or sent.

## Architecture Contract
- TL-1: Unknown ≠ zero — use None/NaN for missing data, not 0.0
- TL-2: One cost event per attempt
- ACP-4: No dead metrics — fields must reflect actual usage

## Changes Required

1. **In `crates/roko-acp/src/bridge_events.rs`**: After each provider response, extract usage:
   ```rust

   // Look for the "result" message type with usage field

       accumulated_tokens += usage.input_tokens + usage.output_tokens;
       // Cost calculation from model pricing in roko.toml
       let cost = calculate_cost(&model, usage.input_tokens, usage.output_tokens);
       accumulated_cost += cost;
   }
   ```

2. **In `crates/roko-acp/src/runner.rs`**: After each pipeline phase, update WorkflowRun:
   ```rust
   if let Some(run) = &mut active_run {
       run.total_tokens += phase_tokens;
       run.total_cost_usd += phase_cost;
   }
   ```

