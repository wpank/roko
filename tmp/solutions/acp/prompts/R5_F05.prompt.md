## Task
Query knowledge store and emit knowledge card at dispatch time

## Runner Context
You are working in runner `mega-parity`, batch R5_F05.
This batch is part of Runner 5: telemetry-learning — Make cost, usage, episodes, learning, and cascade router feedback truthful.

## Problem
When a user sends a prompt, roko should check its knowledge store (roko-neuro) for relevant playbooks, past episodes, and domain knowledge BEFORE dispatching to the agent. This serves two purposes:
1. Show the user what roko already knows (as a visible card in the editor)
2. Inject that knowledge into the agent's system prompt for better results

Currently ACP queries nothing — the agent starts from zero every time, even if roko has solved the exact same problem before.

## Architecture Contract
- TL-5: Learning reads what execution writes
- ACP-3: No silent learning gaps

## Changes Required

1. **In `crates/roko-acp/Cargo.toml`**: Add `roko-neuro = { path = "../roko-neuro" }`

2. **In `crates/roko-acp/src/bridge_events.rs`**: Before dispatching, query neuro store:

```rust
use roko_neuro::KnowledgeStore;

// Before agent dispatch:
let store = KnowledgeStore::open(&workdir.join(".roko/neuro"))?;
let hits = store.query(&prompt, 5)?; // top 5 matches

if !hits.is_empty() {
    // Emit knowledge card to editor
    let card_content = hits.iter().map(|hit| {
        ContentBlock::Text {
            text: format!("{:.2}  {}", hit.relevance, hit.summary),
        }
    }).collect();

    event_sender.send(CognitiveEvent::ToolCallStart {
        tool_call_id: "knowledge-query".to_string(),
        title: format!("Knowledge from neuro store — {} hits", hits.len()),
        kind: ToolCallKind::Other,
    }).await?;

    event_sender.send(CognitiveEvent::ToolCallComplete {
        tool_call_id: "knowledge-query".to_string(),
        status: ToolCallStatus::Completed,
        content: card_content,
    }).await?;

    // Inject into system prompt context
    let knowledge_context = hits.iter()
        .map(|h| format!("- [{:.2}] {}: {}", h.relevance, h.kind, h.summary))
        .collect::<Vec<_>>()
        .join("\n");
    // Append to system prompt or pass as context
}
```

3. **Knowledge hit types to surface**:
   - **Playbooks**: Prior solutions to similar problems (highest value)
   - **Episodes**: Past agent sessions with similar prompts
   - **Domain knowledge**: Distilled insights from research

4. **Graceful fallback**: If neuro store doesn't exist or query fails, log warning and proceed without knowledge (don't block dispatch).

## Do NOT
- Query knowledge store synchronously on the handler thread (use async)
- Block dispatch if store query fails — always proceed
- Include full knowledge text in the card (truncate to summaries)
- Return more than 5 hits (noise → signal ratio drops)
