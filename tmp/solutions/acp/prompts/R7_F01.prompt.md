## Problem
ACP sessions have a `conversation_history: Vec<ConversationTurn>` field that exists but is never populated. Each `session/prompt` dispatches the prompt in isolation — no memory of prior turns. This means:
- Multi-turn conversations don't carry context
- The agent can't reference previous responses
- Editor users must re-explain context every prompt

Additionally, the `includeContext` parameter from `session/prompt` is parsed but never injected into the agent's system prompt or messages.

## Architecture Contract
- MP-1: Polish does not bypass contracts
- ACP-5: Session state is shared-nothing except via shared services

## Changes Required

1. **In `crates/roko-acp/src/session.rs`**: After each prompt+response pair, append to history:
   ```rust
   pub fn record_turn(&mut self, user_prompt: &str, assistant_response: &str, model: &str) {
       self.conversation_history.push(ConversationTurn {
           role: "user".to_string(),
           content: user_prompt.to_string(),
           timestamp: Utc::now(),
       });
       self.conversation_history.push(ConversationTurn {
           role: "assistant".to_string(),
           content: assistant_response.to_string(),
           timestamp: Utc::now(),
       });

       // FIFO trim: keep last N turns that fit within token budget
       self.trim_history(64_000); // ~64K token budget for history
   }
   ```

2. **In `crates/roko-acp/src/bridge_events.rs`**: Before dispatch, inject history into messages:
   ```rust
   let mut messages = Vec::new();

   // Include prior conversation turns
   for turn in &session.conversation_history {
       messages.push(Message {
           role: turn.role.clone(),
           content: turn.content.clone(),
       });
   }

   // Include context from editor (open files, selections)
   if let Some(context) = &prompt_request.include_context {
       for item in context {
           messages.push(Message {
               role: "user".to_string(),
               content: format!("[Context: {}]\n{}", item.label, item.content),
           });
       }
   }

   // Current prompt
   messages.push(Message {
       role: "user".to_string(),
       content: prompt.clone(),
   });
   ```

3. **Token budget trimming**: Simple character-based estimate (4 chars ≈ 1 token):
   ```rust
   fn trim_history(&mut self, max_tokens: usize) {
       let max_chars = max_tokens * 4;
       let mut total_chars = 0;
       let mut keep_from = 0;

       for (i, turn) in self.conversation_history.iter().rev().enumerate() {
           total_chars += turn.content.len();
           if total_chars > max_chars {
               keep_from = self.conversation_history.len() - i;
               break;
           }
       }

       if keep_from > 0 {
           self.conversation_history.drain(..keep_from);
       }
   }
   ```

4. **Session persistence**: History is already part of the Session struct. Verify it serializes to `.roko/sessions/<id>.json` correctly (it should if the field derives Serialize).
