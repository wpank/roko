## Problem
After a pipeline workflow commits changes, the editor doesn't know which files were modified. ACP spec defines a `file_change` session update type, but it's never emitted. This means:
- Editor file tree doesn't refresh
- User doesn't see which files were touched without manually checking git
- No way to navigate to changed files from the workflow result

1. **In `crates/roko-acp/src/runner.rs`**: After the commit action succeeds, run `git diff --stat HEAD~1` and parse the output:

```rust
fn detect_file_changes() -> Result<Vec<FileChange>> {
    let output = Command::new("git")
        .args(["diff", "--name-status", "HEAD~1"])
        .output()?;

    let stdout = String::from_utf8_lossy(&output.stdout);
    let mut changes = Vec::new();

    for line in stdout.lines() {
        let parts: Vec<&str> = line.splitn(2, '\t').collect();
        if parts.len() == 2 {
            let change_type = match parts[0] {
                "A" => FileChangeType::Added,
                "M" => FileChangeType::Modified,
                "D" => FileChangeType::Deleted,
                s if s.starts_with('R') => FileChangeType::Renamed,
                _ => FileChangeType::Modified,
            };
            changes.push(FileChange {
                path: parts[1].to_string(),
                change_type,
            });
        }
    }

    Ok(changes)
}
```
