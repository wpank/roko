- Daemon mode (`roko daemon --acp-mode`) could serve multiple editors
- Future WebSocket transport would need concurrent session access
- It's a correctness issue that should be fixed now before it becomes a race condition bug

### Part 2: Session Concurrency

Wrap the sessions map:
```rust
// BEFORE:
pub struct SessionManager {
    sessions: HashMap<String, AcpSession>,
}

// AFTER:
use std::sync::Arc;
use tokio::sync::RwLock;

pub struct SessionManager {
    sessions: Arc<RwLock<HashMap<String, AcpSession>>>,
}
```

Update all access points:
- `session/new`: `self.sessions.write().await.insert(id, session)`
- `session/list`: `self.sessions.read().await.keys()`
- `session/load`: `self.sessions.write().await.insert(id, loaded)`
- `session/prompt`: `self.sessions.write().await.get_mut(id)`
- `session/config/update`: `self.sessions.write().await.get_mut(id)`
