## Task
Emit narrative text between pipeline phases

## Runner Context
You are working in runner `mega-parity`, batch R7_F05.
This batch is part of Runner 7: mori-polish — Polish interactive experience to match Mori behavior.

## Problem
Pipeline phases run silently between the agent's output. The user sees the agent's raw response text and gate results, but no contextual narrative explaining what happened and what's next. Good UX means the pipeline should narrate its own actions:

- After strategist: "Here's my approach: ..."
- After gates fail: "Two tests fail. The fix is right — those tests asserted the old behaviour. Moving to auto-fixer to update them."
- After review approve: "Code review passed. Committing changes."
- After permission granted: "Permission granted. Following the agent..."

This narrative makes the pipeline feel intelligent rather than mechanical.

## Architecture Contract
- MP-1: Polish does not bypass contracts
- MP-4: Improve truth before appearance

## Changes Required

1. **In `crates/roko-acp/src/runner.rs`**: After each phase result, emit a brief narrative as `AgentMessageChunk`:

```rust
// After strategist completes:
fn narrate_strategy(brief: &str) -> String {
    // Summarize the strategy in 1-2 sentences
    let first_sentence = brief.split('.').next().unwrap_or(brief);
    format!("{first_sentence}.\n\n")
}

// After gate failure:
fn narrate_gate_failure(gate: &str, output: &str) -> String {
    let error_count = output.matches("error").count();
    let test_failures = output.matches("FAILED").count();
    if test_failures > 0 {
        format!("{test_failures} tests fail. Moving to auto-fixer to update them.\n\n")
    } else {
        format!("{gate} failed with {error_count} errors. Moving to auto-fixer.\n\n")
    }
}

// After gate pass:
fn narrate_gate_pass() -> String {
    "All gates passed.\n\n".to_string()
}

// After review:
fn narrate_review(approved: bool, summary: &str) -> String {
    if approved {
        format!("Code review approved: {summary}\n\n")
    } else {
        "Review requested revisions. Re-implementing with feedback.\n\n".to_string()
    }
}

// After permission granted:
fn narrate_permission(path: &str) -> String {
    format!("Permission granted. Editing {path}.\n\n")
}
```

2. **Emit narratives** in the `run_workflow_pipeline()` main loop, right after processing each `PipelineAction` result:

```rust
PipelineAction::RunGates => {
    let gate_result = run_gates(...).await;
    match &gate_result {
        Ok(()) => {
            emit_narrative("All gates passed.", &event_sender).await;
        }
        Err(e) => {
            let narration = narrate_gate_failure("gates", &e.to_string());
            emit_narrative(&narration, &event_sender).await;
        }
    }
    action = match gate_result { ... };
}
```

3. **Helper function**:
```rust
async fn emit_narrative(text: &str, sender: &mpsc::Sender<CognitiveEvent>) {
    let _ = sender.send(CognitiveEvent::TokenChunk(text.to_string())).await;
}
```
