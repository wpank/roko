## Problem
When users attach context via @-mentions in the editor (e.g., `@services/checkout.go`, `@branch diff`, `@datadog logs`), ACP receives these as `ContentBlock::Resource` items in the prompt. Currently roko parses them but never resolves them — the attached file contents, diffs, and logs are not included in the agent's prompt.

For files, resolution means reading the file. For "branch diff", it means running `git diff`. For external sources like "datadog logs", it means querying an MCP server.

```rust
/// A registered context source that can provide content for @-mentions.
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct ContextProvider {
    pub id: String,         // "file", "git-diff", "git-log", "mcp:<server>"
    pub label: String,      // "branch diff", "datadog logs"
    pub description: String,
    pub source: ContextSource,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
#[serde(tag = "type", rename_all = "snake_case")]
pub enum ContextSource {
    /// Read a file from disk.
    File { path: String },
    /// Run a git command and capture output.
    Git { subcommand: Vec<String> },
    /// Query an MCP server resource.
    Mcp { server: String, resource: String },
}
```

3. **In `crates/roko-acp/src/bridge_events.rs`**: Before dispatch, resolve all context items:

```rust
async fn resolve_context(
    prompt_blocks: &[ContentBlock],
    providers: &[ContextProvider],
    workdir: &Path,
) -> Vec<String> {
    let mut context_parts = Vec::new();

    for block in prompt_blocks {
        match block {
            ContentBlock::Resource { resource: ResourceRef::File { uri } } => {
                // Read file from disk
                if let Ok(content) = tokio::fs::read_to_string(uri_to_path(uri, workdir)).await {
                    context_parts.push(format!("--- File: {uri} ---\n{content}\n"));
                }
            }
            ContentBlock::Text { text } if text.starts_with("@") => {
                // Look up provider by label
                let label = text.trim_start_matches('@').trim();
                if let Some(provider) = providers.iter().find(|p| p.label == label) {
                    if let Ok(content) = resolve_provider(provider, workdir).await {
                        context_parts.push(format!("--- {label} ---\n{content}\n"));
                    }
                }
            }
            _ => {}
        }
    }

    context_parts
}

async fn resolve_provider(provider: &ContextProvider, workdir: &Path) -> anyhow::Result<String> {
    match &provider.source {
        ContextSource::File { path } => {
            Ok(tokio::fs::read_to_string(workdir.join(path)).await?)
        }
        ContextSource::Git { subcommand } => {
            let output = Command::new("git")
                .args(subcommand)
                .current_dir(workdir)
                .output()
                .await?;
            Ok(String::from_utf8_lossy(&output.stdout).to_string())
        }
        ContextSource::Mcp { server, resource } => {
            // Query MCP server (future — stub for now)
            Ok(format!("[MCP query to {server}/{resource} — not yet wired]"))
        }
    }
}
```
