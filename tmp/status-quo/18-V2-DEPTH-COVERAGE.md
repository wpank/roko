## Summary

The v2-depth corpus is far **less aspirational than CLAUDE.md implies**. Roughly 70% of its named designs correspond to real code, and much of that code is wired into the runtime. The v2 protocol vocabulary is genuinely in the kernel: all 11 v2 protocol traits (`Store`, `ColdStore`, `Score`, `Verify`, `Route`, `Compose`, `React`, `Bus`, `Observe`, `Connect`, `Trigger`) are defined in `crates/roko-core/src/traits.rs:37-425` (Store 37, ColdStore 102, Score 167, Verify 214, Route 242, Compose 285, React 339, Bus 385, Observe 400, Connect 408, Trigger 420), `Pulse` exists (`roko-core/src/pulse.rs:75`), and two `Cell` traits exist (`roko-core/src/cell.rs:91`, `roko-graph/src/cell.rs:74`). Systems CLAUDE.md doesn't even mention are implemented: the relay wire protocol (`apps/agent-relay/`), gamma/theta/delta heartbeat (`roko-runtime/src/heartbeat.rs:23,26,29`), StateHub projections (`roko-runtime/src/state_hub.rs`), MAP-Elites skill evolution (**in `roko-dreams/src/phase2/evolution.rs`**, not skill_library — see drift note), morphogenetic coordination (`roko-orchestrator/src/coordination.rs`), and CaMeL-style dual-LLM safety (`roko-agent/src/safety/data_llm.rs`, 439 LOC, not a stub).

Four structural gaps dominate:

1. **The noun is still `Engram`.** `Signal` is only a re-export alias (`roko-core/src/signal.rs:6`); the struct lives in `roko-core/src/engram.rs:63`. Every crate's working vocabulary is Engram-shaped (🕰️). A duplicate `Engram` also lives in `roko-chain/src/identity_economy_markets.rs:653` (divergence risk).
2. **The graph engine is exposed but not production-live — and RunnerV2 is the *default*, not Graph.** Re-verified: `crates/roko-cli/src/main.rs:1301` marks `RunnerV2` with `#[default]`; Graph is opt-in via `--engine graph`. `roko-graph` (Cell/GraphEngine/HotGraph, `hot.rs`+`engine.rs`) is built, but Graph plan tasks dry-run through `TaskExecutorCell` (GAPS.md:8) and all **7** cognitive-loop cells are `PassthroughCell` stubs (`roko-graph/src/cells/stubs.rs:69-77`, GAPS.md:16). Earlier wording that Graph was "the current plan run Clap default" was **STALE** — corrected.
3. **Two v2 abstractions have zero code**: `Lens` (telemetry projections) and `Arena` — `grep 'trait Lens\|struct .*Lens'` and `Arena` both return no relevant hits in crates/.
4. **Protocols aren't unified onto Cell.** Docs model every protocol as a Cell conformance; in code only `Observe`/`Connect`/`Trigger` are `Cell` supertraits (3 of 11; `traits.rs:400-425`), and their only impls are in tests (`roko-core/tests/phase1_integration.rs:188,237,294`).

The 4 near-empty sections (04, 06, 08, 19) are INDEX-only **by design** ("spec sufficient") — but for triggers and extensions real code exists undocumented at depth, while for arenas neither docs nor code exist.

## Per-section notes

**00-index** — Thesis docs claim the 5-layer taxonomy derives from the protocol dependency lattice. The lattice's protocols all exist (traits.rs:37-420) but crate organization doesn't mirror the layers, and `implementation-readiness.md`'s phase A–E build sequence is tracking a migration that stalled at "engine as default" (`.roko/GAPS.md:11-13`).

**01-signal** — Signal internals real but under the v1 name: `Engram` struct (engram.rs:63) with `HdcFingerprint`, taint fields, scores; `Signal` is a pure alias (signal.rs:6). `Pulse` is a distinct real type (pulse.rs:75) with graduation config (`roko-core/src/config/graduation.rs`) and a `GraduationCell` (`roko-graph/src/cells/graduation.rs`) — the Store/Bus bridge from the docs exists. The semiring/functor formalism and Gesell demurrage rate law are doc-only math; runtime has simpler decay/GC.

**02-block** — The flagship section, and it mostly describes reality. All 9+ protocols are traits; gates return `Verdict` never `Result<Verdict>` exactly as documented (`gate_pipeline.rs:209`, `diff_gate.rs:155` — re-verified this pass); the 7-rung pipeline is a real `enum Rung {Compile..Integration}` with `CANONICAL_ORDER: [Rung; 7]` (`roko-gate/src/rung_selector.rs:60-120`); the gate catalog matches. VCG re-verified: `vcg_allocate` is **defined** at `roko-compose/src/auction.rs:380` and **called** at `roko-compose/src/prompt.rs:1213`, guarded by a strategy branch (`prompt.rs:864-880`) where `CompositionStrategy::Vcg` selects the VCG path and density-greedy is the `else`. Both branches are live code — VCG is *not* dead — but which fires depends on `strategy.resolve()`; density-greedy dominates in practice because live bidders are rarely warmed, so treat VCG-in-production as reachable-but-cold rather than the default path. EFE/MVT context selection exists (`roko-compose/src/{foraging,scorer}.rs`) but is not the full v2 protocol story. Doc-only: the categorical protocol-morphism table, `StoreProtocol: Cell` unification, and parts of autonomous eval generation (though MAP-Elites lives in roko-learn/skill_library.rs).

**03-graph** — Orchestration substance (discovery, DAG, waves, worktrees, snapshots, conductor integration) is wired, but through `orchestrate.rs`'s runner, which predates the Cell/Graph engine. The "Conductor as 10 Lens Cells" reframe is vocabulary-only: roko-conductor's watchers are real but there is no Lens type. GraphEngine executes sequentially; `max_parallel` stored but unused (GAPS.md:13).
