## Summary

roko-core is a **faithful v1 implementation wearing a half-finished v2 costume**. The universal noun is still the `Engram` struct (`crates/roko-core/src/engram.rs:63`); `Signal` is a re-export alias — `pub use crate::engram::{Engram as Signal, ...}` (`crates/roko-core/src/signal.rs:6`) — explicitly labeled "forward-compatible alias… full Engram→Signal rename happens in Phase 1". The v2 spec itself blesses this bridge ("The Rust struct remains `Engram`; new code bridges with `type Signal = Engram;`", `docs/v2/01-SIGNAL.md:69`), so the alias is not a bug — but the ecosystem has split along it: roko-gate/-agent/-compose/-learn/-orchestrator import `Signal` exclusively while roko-cli/-serve/-fs/-graph/-std use `Engram`, and roko-conductor mixes both.

The v1 renames are **done**: Scorer→`Score`, Gate→`Verify`, Router→`Route`, Composer→`Compose`, Policy→`React`, Substrate→`Store` (with a blanket `Substrate: Store` legacy alias, `traits.rs:427-429`). Score (7-axis), Decay (4 variants), Provenance+typed Taint, Kind (compound lattice), Pulse+graduation all match the v1 docs closely.

The v2 deltas are **mostly missing**: no 5-axis Score, no calibration stack (temperature scaling/ECE/Beta-Binomial/isotonic), no demurrage runtime (trait exists, `balance` field exists, but nothing ticks it), no `tier`/`demurrage_paid`/`last_touched_at` on the Signal, no taint-lattice `join`, no `SignalRef`/ULID id, and the `Cell` protocol unification exists in two divergent copies (roko-core vs roko-graph).

The sharpest operational finding: **two persistence files with contradictory "legacy" labels and two live writers**. `.roko/engrams.jsonl` (10 lines, via FileSubstrate) is "main" per `roko-fs/src/layout.rs:202`, while `.roko/signals.jsonl` (467 lines, newer: 2026-05-08) is "legacy" — yet the plan-runner still appends **non-Engram-shaped** raw JSON gate verdicts to it (`roko-cli/src/runner/event_loop.rs:1147`), and the v2 spec declares `signals.jsonl` the *primary* log (`docs/v2/01-SIGNAL.md:1371`). The code renamed in the opposite direction from the spec.
