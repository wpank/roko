## Summary

Three orchestration generations coexist. **The default `roko plan run` path today is the Graph Engine, and it does not execute anything real**: `--engine` defaults to `graph` (`crates/roko-cli/src/main.rs:1361`), the converter maps every task to a `task-executor` node (`crates/roko-graph/src/convert.rs:63`), `default_registry()` instantiates `TaskExecutorCell::default()` (`crates/roko-graph/src/engine.rs:356-358`) whose default is `dry_run: true` (`crates/roko-graph/src/cells/task_executor.rs:32`), and live mode "falls back to dry-run with a warning" (`task_executor.rs:81-86`). Execution is sequential-topological (`engine.rs:1-4,123,136`), no gates nodes are emitted, no resume (`commands/plan.rs:260-264` prints "snapshots will be ignored"), no cost/episodes. So the CLAUDE.md self-hosting step 5 (`roko plan run plans/`) is a no-op synthetic pass unless `--engine runner-v2` is given.

**Runner v2 is the real production engine** — an event-driven `tokio::select!` loop (`crates/roko-cli/src/runner/event_loop.rs`) built to the `docs/v2/27-ORCHESTRATOR.md` spec, wrapping roko-orchestrator's pure `ParallelExecutor`. It is what every *other* entry point calls directly: `roko do` (`commands/do_cmd.rs:616`), `roko serve` runs (`serve_runtime.rs:304`), PRD auto-execution (`prd.rs:956`), and the cloud worker (`worker/cloud.rs:462`). It has streaming dispatch, parallel tasks, gates, resume, merge queue, budgets, and learning sinks — but **no worktree isolation** (in-place execution, `runner/merge.rs:148-150`) and only prompt-level replan.

**orchestrate.rs (v1, 23,676 lines, ~250 methods on `PlanRunner`) is no longer compiled by default** — gated behind the non-default `legacy-orchestrate` feature (`lib.rs:94-95`, Cargo.toml default = `["legacy-runner-v2"]` at `crates/roko-cli/Cargo.toml:15`). CLAUDE.md's component table ("Plan discovery + DAG executor — Wired — orchestrate.rs", plus ~15 rows citing orchestrate.rs) is **stale**: none of that code is in the shipping binary. Worktree isolation, agent-driven replan, stigmergy, dreams triggers, chain attestation, C-factor summaries, VCG bidders — all live only inside this dead-by-default module.

Feature-flag documentation is internally inconsistent: `legacy-runner-v2` is claimed to gate the Runner v2 path (Cargo.toml:17-19, GAPS.md Task 102 line 12) but **actually gates only 4 test files** (`tests/phase0_wiring.rs:12`, `tests/cost_dedup.rs:8`, `tests/smoke.rs:199,311`, `tests/common/mod.rs:261`) — no `cfg` exists in `commands/plan.rs`. And `PlanEngine`'s `#[default]` is `RunnerV2` (`main.rs:1301-1303`) while the clap arg default is `"graph"` (`main.rs:1362`). Worst concrete bug: **`roko resume` is broken** — it builds `PlanCmd::Run { engine: PlanEngine::Graph, resume_plan: Some(snapshot), .. }` (`main.rs:2697-2709`), so the snapshot is ignored and tasks dry-run.
