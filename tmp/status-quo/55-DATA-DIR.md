## Summary

`.roko/` is written by **at least six independent subsystems that do not share one path authority**. There are *three* competing path catalogs — `roko_fs::RokoLayout` (`crates/roko-fs/src/layout.rs`), `roko_core::Workspace` (`crates/roko-core/src/workspace.rs`), and `roko-cli`'s ad-hoc `workspace_paths.rs`/inline `join(".roko")` calls — and they disagree on canonical locations (episodes: root vs `learn/` vs `memory/`; plans: `.roko/plans/` vs repo-root `plans/`; daimon state: `daimon/affect.json` vs `state/daimon.json`).

For the normalized canonicalization map and path-migration checklist, see [60-STATE-PERSISTENCE-LEDGER.md](60-STATE-PERSISTENCE-LEDGER.md).

**The engrams/signals tension, resolved:** the naming history is *two* renames moving in opposite directions.
1. **File-level (done, v1):** the engram log was renamed `signals.jsonl` → `engrams.jsonl`. `layout.rs:202-215` calls `engrams.jsonl` "the main engram log" and `signals.jsonl` the "Legacy path … (pre-rename)"; `roko init` actively migrates `signals.jsonl` → `engrams.jsonl` (`commands/util.rs:135-150`).
2. **Type-level (aliased, v2):** the v2 architecture renamed the noun Engram → Signal, but only as an alias: `pub use crate::engram::{Engram as Signal, EngramBuilder as SignalBuilder}` (`roko-core/src/signal.rs:6`); the real struct is still `Engram` (`roko-core/src/engram.rs:63`). CLAUDE.md speaks v2 ("1 noun (Signal)", "Signal log: `.roko/signals.jsonl`") — **both CLAUDE.md claims contradict the code**, which treats `engrams.jsonl` as canonical.
3. **The live bug:** Runner v2 — the *newest* execution path — appends `GateVerdict` audit rows to `layout.signals_path()` (`runner/event_loop.rs:1147-1168`), i.e. it re-creates the *legacy-named* file with a *non-Engram schema* (flat `{"kind":"GateVerdict",...}` rows, verified on disk: 467 rows, mtime May 8) while `engrams.jsonl` holds full Engram records (10 rows, mtime May 6). If a fresh `roko init` ever finds only `signals.jsonl`, the migration renames it wholesale into `engrams.jsonl`, mixing GateVerdict rows into the Engram store.

Other headline findings: **~86 MB of unrotated logs** (`events.jsonl` 44.9 MB, `chain-watcher.log` 23.3 MB, `roko.log` 12.2 MB, `state/run-ledger.jsonl` 6 MB — none covered by the retention module, `roko-serve/src/retention.rs:115-174`); a **triplicated episode store** (root / `learn/` / `memory/`); **11 leaked `cascade-router.json.tmp.*`** atomic-write temp files; **66 accumulated PTY demo dirs** under `workspaces/`; `GAPS.md` last touched May 5 and no longer containing entries that six source files still cite; and `VERSION` = `1` with `LayoutVersion::V1` the only variant ever defined (`layout.rs:30-39`).

Freshest activity (Jul 6 14:56): `roko.log`, `serve-tui.log`, `runtime/roko.lock`, `INDEX.md`, `prd/INDEX.md`, `research/INDEX.md` — i.e. a `roko serve` TUI session ran yesterday; the plan-execution stores have been idle since May 8-9.
