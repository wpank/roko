# Docs Convergence Plan

**Verified**: 2026-07-08 against `main` @ `5852c93c05`.

The docs are useful but inconsistent. This plan describes how to converge them without
losing design history.

## The Four Layers (and the fifth: code)

Per `tmp/doc-convergence/README.md`, roko's specs sit in four disconnected layers that do
not reference each other, plus the code that matches none of them fully:

| Layer | Location | Files | Vocabulary |
|---|---|---|---|
| v1 | `docs/v1/` | ~417 | Old (Engram, Gate, Substrate, Scorer) |
| v2 | `docs/v2/` | 34 | New/target (Signal, Cell, Graph, Protocol) |
| v2-depth | `docs/v2-depth/` | ~185 | Mixed (part absorbed from v1) |
| tmp designs | `tmp/prds/`, `tmp/doc-convergence/**` | many | Old vocabulary, impl-oriented |
| **code** | `crates/` | 35 members | **Engram**, `Engine`/Runner v2, feature-gated legacy `orchestrate.rs` |

Key reconciliation fact: **the code uses v1 vocabulary (`Engram`, `roko-core/src/engram.rs:63`),
not v2's `Signal`.** The v2 `Signal/Cell/Graph/Protocol` model is *target* vocabulary. Any
convergence effort must state this explicitly or it will re-introduce the Signal-vs-Engram
drift. The `tmp/doc-convergence/` effort (prompts 01-05, scripts, `status/PROGRESS.md`) aims
to produce a `docs/v3/` canonical spec; its unfinished output means v3 is not yet a source
of truth — treat `tmp/doc-convergence/**` as SCRATCH until promoted.

## Convergence Workflow

1. Pick one status-quo file as the source for a topic.
2. Update root/user docs from that file.
3. Run/generate the source-doc manifest and confirm the source file status tag.
4. Add a short "historical" banner to superseded v1/v2/tmp docs.
5. Add route/state/config/test proof links where possible.
6. Move copied findings from tmp scratch into maintained docs.
7. Archive scratch sources after copied.

## Archive Banners

Use these labels:

- `CURRENT`: Maintained and expected to match code.
- `AUTHORITATIVE-PARTIAL`: Contains current design decisions but not full implementation truth.
- `HISTORICAL`: Useful background only.
- `SUPERSEDED`: Do not use for implementation decisions.
- `SCRATCH`: Generated or exploratory; findings must be copied before use.
