## Headline finding (P0, STILL-OPEN)

**The default `roko plan run plans/` executes NO real agents.** `plan run` defaults to
`engine = "graph"` (`crates/roko-cli/src/main.rs:1361`). The Graph path
(`crates/roko-cli/src/commands/plan.rs:258,1567`) builds a graph whose task nodes resolve to
`TaskExecutorCell` via `roko_graph::default_registry()`
(`crates/roko-graph/src/engine.rs:311,357`). `TaskExecutorCell::default()` has `dry_run: true`
(`crates/roko-graph/src/cells/task_executor.rs:32`), and **even its live branch is a stub** that
logs "live dispatch not yet implemented; using dry-run fallback" and emits a synthetic
`task-output:stub:` engram (`task_executor.rs:83-92`). Real agent execution only happens on the
**non-default** `--engine runner-v2` path (`plan.rs:269-270`). This directly falsifies CLAUDE.md's
"Plan discovery + DAG executor: Wired" claim for the default surface. Docs **08** and **21** are
the top P0. (Note: a prior verification pass mis-called this FIXED because `engine.execute()` is
invoked — but the *cell body* it invokes is a dry-run stub.)
