## Summary

The v2 kernel specs describe a system where "everything is a Graph of Cells processing Signals through Bus and Store." The code is roughly one-third of the way there, and the third that exists is split across **two parallel realities**: a hollow v2-shaped skeleton (roko-graph's Cell/Graph/Engine — the *default* `roko plan run` path, which dry-runs every task) and a working v1-shaped production engine (Runner v2 in `roko-cli/src/runner/`, reachable via `--engine runner-v2` and used by `roko do`/`serve`/PRD auto-exec) that implements most of 27-ORCHESTRATOR but almost none of the kernel vocabulary (no Cells, no Pulse lifecycle, no Bus, prompts assembled by a "lightweight local mirror" instead of the 9-layer builder).

Per-spec verdicts:

- **01-SIGNAL** 🟡🕰️ — The noun is still `Engram` (`roko-core/src/engram.rs:63`); `Signal` is an alias (`signal.rs:6`). Graduation exists; projection, 5-axis Score, the entire calibration stack (temperature/ECE/Beta-Binomial/isotonic), demurrage runtime, taint lattice ops, SignalId/SignalRef, and v2 knowledge Kinds are missing from core. Persistence filenames are *inverted* vs spec (code says `engrams.jsonl` primary, spec says `signals.jsonl`).
- **02-CELL** 🟡 — All 9 protocol verbs exist as traits (`roko-core/src/traits.rs:37-420`) and the v1→v2 renames are done, but the Verdict redesign (continuous `reward`, hard/soft criteria, 19 typed EvidenceKinds, verify_pre) is entirely absent — `Verdict` is the v1 pass/score/reason shape (`roko-core/src/verdict.rs:51-70`). Two incompatible `Cell` traits coexist (roko-core vs roko-graph). VCG compose is built (`roko-compose/src/auction.rs:380`) but not on the default dispatch path. No Bradley-Terry, no free monad, no adjacency matrix, no Cost-in-microcents.
- **03-GRAPH** 🟡 — Graph/loader/topo/engine/hot-loop exist and are tested (~2.6K LOC), but with no NodeKind variants, no conditional-edge evaluation (evaluator built, never called), no policy/budget, no snapshot/resume, no Workflow/Activity split, no Graph-as-Cell.
- **04-EXECUTION** ❌/🔌 — Nothing implements the `Engine` API (start/resume/pause/register_hot/estimate). The 7-cell cognitive loop exists as a TOML + `PassthroughCell` stubs ticking at 1 Hz doing nothing. Gamma/theta/delta heartbeats + `CorticalState` are *real and sophisticated* (`roko-runtime/src/heartbeat.rs:228`) but wired only from feature-gated-off orchestrate.rs.
- **05-AGENT** 🟡 — Vitality phases (roko-daimon), Regime + adaptive clock + CorticalState (roko-runtime), PAD affect, somatic retrieval bias (roko-neuro) all exist; the composite Agent (type-state lifecycle, Space, slots, 16 T0 probes, reflex store, cognitive energy, emergent goals, 15% contrarian retrieval) does not. "CognitiveWorkspace" exists in core but as an *audit record*, not the spec's VCG attention market.
- **06-MEMORY** 🟡 — The strongest kernel area (roko-neuro): kinds, tiers, falsifier-bearing heuristics, AntiKnowledge, distillation D1-D3, dreams with a real post-plan trigger in Runner v2. But the HDC feature is compiled out of every shipped binary, demurrage has taxes without income (reinforcement unwired, live balances 0.0), hindsight relabeling has **zero code**, and knowledge is a separate JSONL store, not Knowledge-as-Signal.
- **07-LEARNING** 🟡 — L2 (routing) and most of L3 (consolidation) are real and wired; episodes, playbooks, experiments, efficiency, section effects, post-gate reflection, reflection-derived playbook rules all ship in the default runner. L1 exists only as scattered EMA fragments (adaptive gate thresholds not consulted by the runner's rung path), L4 (structural adaptation, RecursiveSafetyMonitor, autonomy levels, c-factor gate, HGM/CycleQD) has essentially no code, and predict-publish-correct exists as one `CalibrationPolicy` module rather than a universal per-Cell mechanism.
- **27-ORCHESTRATOR** ✅ (core) / 🟡 (gaps) — Runner v2 matches the spec's architecture (§2-§7, §14) closely; of the 12 mori-parity gaps: 4 (reflection) and 8 (neuro→router) are wired, 9/10/11 are built-and-mostly-wired via runtime_feedback, 1/2/3 have types-but-no-runner-wiring, 6 (WarmPool) is a container with capacity hardcoded to 0, and 5 (KnowledgeConfig scoping) plus express mode and `apply_rustc_fixes` have no code. §13 (`roko plan enrich` as a separate command) does not exist.

**The single most consequential defect** (inherited scope from GAPS.md Tasks 101-103, independently re-verified at HEAD 5852c93c): `roko plan run` defaults to `--engine graph` (`roko-cli/src/main.rs:1361`) whose `TaskExecutorCell` defaults to `dry_run: true` (`roko-graph/src/cells/task_executor.rs:20,30-32`), so the documented self-hosting entry point completes "successfully" while doing no real work, persisting no episodes, and honoring no snapshots (`roko resume` hardcodes `engine: PlanEngine::Graph` and discards the snapshot, `main.rs:2699`). This is the **P0 that blocks the whole self-hosting claim** in CLAUDE.md: the loop the README advertises (`plan run` → agents → gates → persist) is, on the default path, a no-op that prints SUCCESS.

## Per-spec deep notes

### 01-SIGNAL — v1 core wearing a v2 alias

**Spec demands**: dual-medium kernel (durable Signal + ephemeral Pulse), ULID id + SHA-256 content hash, 5-axis Score with a full calibration stack (§5.3-5.9), Gesell demurrage with tiers/Markov transitions/telemetry (§6), HDC semiring incl. resonate (§7), taint lattice with action-time gating and custody (§10), graduation/projection adjunction (§12), Bus with replay, `signals.jsonl` as primary log.

**What exists**: the v1 `Engram` faithful implementation plus a `Signal` alias explicitly blessed by the spec itself (`01-SIGNAL.md:69` says the Rust struct stays `Engram`). Graduation is real (`pulse.rs:138`). The verb-trait renames are complete. Balance field exists but nothing ticks it in core.

**Exact deltas**: (1) no projection, no Pulse `source`/`lineage_hint`/`trace_id`, no Bus replay — so the predict-publish-correct join-by-lineage described in 02/07 cannot be built on today's Pulse; (2) zero calibration code (no temperature scaling, ECE, Beta-Binomial, isotonic); (3) demurrage machinery exists in three disconnected places (core trait test-only, roko-agent/roko-runtime modules test-only, roko-neuro flat-rate wired-for-decay-only); (4) persistence naming is inverted vs spec and the runner appends non-Engram JSON to the "legacy" file (`runner/event_loop.rs:1147-1160`) — records one path writes are unreadable by the other; (5) ecosystem is split ~50/50 on `Signal` vs `Engram` imports (30-audit census).

### 02-CELL — protocols renamed, contracts not upgraded

**Spec demands**: single `Cell` trait carrying schemas/capabilities/protocol conformance/cost; 9 protocols all `: Cell`; a four-role Verdict (reward function, relabeling oracle, safety boundary via verify_pre, economic attestation) with conjunctive-hard + Pareto-soft criteria and 19 typed evidence kinds; EFE routing; VCG compose with 8 bidders and Beta section effects; three-layer capability pullback; 5 implementation tiers; free-monad programs.

**What exists**: all 9 verbs as traits with Datum polymorphism and `verify_stream`; a working gate catalog (11+ gates, 7-rung pipeline via `rung_dispatch.rs` used by Runner v2); `vcg_allocate` + a real bidder vocabulary; a `CalibrationPolicy` that literally names predict-publish-correct (LEARN-09).

**Exact deltas**: (1) **Verdict is untouched v1** — no `reward`, no criteria vectors, no evidence typing; every downstream "four roles" behavior (routing reward, hindsight relabeling, pre-action veto, reputation) is therefore structurally impossible today; (2) two `Cell`/`CellContext` families (roko-core vs roko-graph) and a third de-facto family (`NodeOutput`-based AgentCell/ComposeCell inside roko-graph, invisible to the engine); (3) the default dispatch path doesn't run the Compose machinery at all — `dispatch/prompt_builder.rs:712-715` states wiring into the 9-layer `SystemPromptBuilder` "is exposed as a follow-up — see .roko/GAPS.md" (note: this item is *not* in GAPS.md — stale pointer); (4) no Bradley-Terry, adjacency matrix, capability pullback, or Cost-as-integer.

### 03-GRAPH — skeleton wired, organs missing

Spec's Graph = typed nodes (10 NodeKinds), conditional edges with a total Expr language, policies with 8 failure strategies, Workflow/Activity split, Hot Graphs with feedback edges and clock bindings, run storage + FlowSnapshot, merge queue as Cell, worktree isolation. Code has: TOML→petgraph→topo→sequential interpret (real, tested, CLI-surfaced via `roko graph run/validate/show`), a plan→graph converter, and a hot tick loop. Everything else in the spec's §2-§12 is absent or built-not-wired (condition evaluator, BudgetTracker). All six GAPS.md Task 101-103 items re-verified true (dry-run TaskExecutorCell, no engine snapshot/resume, sequential-only, PassthroughCell stubs, dead `persist_tick_state`, no `[graph.policy.hot]` parsing, unconditional edges). 3 of 7 example TOMLs don't parse under the current loader schema (31-audit).

### 04-EXECUTION — the Engine does not exist; its pieces do

No type implements the spec's `Engine` API. The two candidate substrates each hold half: roko-graph has interpretation-of-graphs but no lifecycle/budget/resume/pulses; Runner v2 has lifecycle/budget/resume/persistence/streaming but no graphs/cells. The genuinely spec-shaped artifacts — gamma/theta/delta adaptive clocks with Regime multipliers matching the spec's table, `CorticalState` with atomic accessors and snapshots, theta/delta consumers — live in roko-runtime and are driven **only** by orchestrate.rs, which is compiled out by default (`roko-cli/src/lib.rs:94`, `legacy-orchestrate` non-default). Net effect: the most v2-faithful runtime code in the repo is unreachable from any shipping binary path, while the shipping default (`plan run --engine graph`) executes a no-op pipeline and prints SUCCESS.

### 05-AGENT — affect real, agent absent

Real: `VitalityPhase` (roko-daimon mortality), PAD affect (roko-primitives/pad.rs + daimon), somatic bias applied to knowledge retrieval scoring (roko-neuro/context.rs:1488-1522), Regime-adaptive clocks, `CognitiveWorkspace` audit objects flowing into section-outcome learning. Missing entirely: the composite `Agent<S>` type-state runtime, Space isolation, SlotManager, all 16 T0 probes, reflex store, cognitive energy/fatigue/zones, emergent goals/ZPD/intrinsic motivation, contrarian retrieval. The 9-step pipeline exists only as the 7-node stub TOML (04). Name hazard: core's `CognitiveWorkspace` is an invocation audit record — a reviewer reading spec 05 §16 will expect the VCG attention market and find a different object.

### 06-MEMORY — most complete, with two dead switches

roko-neuro implements the knowledge lifecycle end-to-end and Runner v2 now triggers dream consolidation after plan completion (`event_loop.rs:1952`), closing the loop CLAUDE.md still lists as partial. Two systemic defects (both re-verified from the 39-audit): the `hdc` cargo feature is enabled by no consumer, so fingerprinting/repulsion/resonance/cross-domain transfer are compiled out of production (`.roko/neuro/knowledge.jsonl` has `"hdc_vector": null`); and the demurrage economy only taxes (serve heartbeat decays balances; `RuntimeKnowledgeLifecycle` reinforcement has zero external callers; balance is ignored by query scoring). Spec-06 features with zero code: hindsight relabeling, Resonator Networks, event calculus, 3-tier temporal memory (Allen algebra itself is built, uncalled), consensus freeze/challenge, Kind::Worldview-as-Signal, pheromone runtime.

### 07-LEARNING — L2/L3 alive, L1 fragmentary, L4 absent

The learning surface that CLAUDE.md brags about is real and wired through Runner v2's `FeedbackFacade` sinks (episodes, efficiency, routing observations, knowledge ingestion, section outcomes, post-gate reflection, playbook rules). But mapped onto the spec's four loops: L2 ✅ (CascadeRouter — still LinUCB-based, spec says EFE replaces it; an active-inference tier-selector exists at `cascade_router.rs:436` as a partial bridge); L3 🟡 (dreams + tier promotion + AntiKnowledge, minus hindsight and minus reinforcement economics); L1 🟡 (adaptive gate thresholds exist and persist to `.roko/learn/gate-thresholds.json` but the Runner v2 rung path doesn't consult them — only gate_service consumers do; no ParamRange/rollback/freeze framework); L4 ❌ (no proposals, monitor, approval workflow, c-factor gate, HGM, CycleQD, spec-as-artifact — MAP-Elites `skill_library.rs` is the only related artifact). Of the seven compounding loops only C5 (playbooks) is fully closed; C1 is half-open (tax without income), C4 has data surfaces but no learner/gate, C3/C6/C7 are absent or stubs. The c-factor governance apparatus (Lens cells, WisdomGate, anti-groupthink React cells, anti-metrics) has no code at all.

### 27-ORCHESTRATOR — the one spec the code mostly obeys

Runner v2 is a faithful build of §2-§7/§14 (module layout matches §4 exactly; streaming, atomic flush-per-task, tasks.toml-only loading, executor-pure/runner-I/O split all verified). Deviations: event_loop.rs is 16× the spec'd size (god-object regrowth); worktree isolation was dropped rather than ported; replan is prompt-context-only. Gap scorecard (12 mori-parity gaps): **wired** 4, 8; **built + partially wired** 9, 10, 11, 12, 3; **types-only** 1, 2; **container-only** 6; **missing** 5, express mode, apply_rustc_fixes. §11's migration phases are mid-flight and internally inconsistent: Phase C (deprecate orchestrate.rs) is done via feature gate, but Phase B's flag story drifted (`legacy-runner-v2` gates only 4 test files; clap default `graph` contradicts `PlanEngine::default()` = RunnerV2) and Phase D was started (engine default) before its precondition (live TaskExecutorCell) landed.
