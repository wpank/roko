## Summary

roko-primitives is a clean, zero-workspace-dep kernel: a 10,240-bit `HdcVector` (bind/bundle/permute/similarity, deterministic FNV-1a‚Üísplitmix64 seeding, serde-as-1280-bytes) that matches the v2 spec numerically, plus `InferenceTier` T0/T1/T2 and a `TierRouter` that matches v1/16-heartbeat/08 verbatim. It also carries ~3.2K LOC of "technical analysis" math (tropical, manifold, sheaf, TDA, robust stats) with **zero consumers anywhere in the workspace**.

**Reconciling the roko-neuro cross-finding (39-NEURO-KNOWLEDGE.md ¬ß2): both facts are true, at different layers.** The `hdc` cargo features declared by roko-neuro, roko-compose, roko-fs, and roko-serve are enabled by *no* dependent crate (grep of every Cargo.toml: only the 4 feature *definitions* exist, zero activations) ‚Äî so neuro's encoders, AntiKnowledge repulsion, MemoryIndex, resonance, FileSubstrate/webhook fingerprint tagging are all compiled out of every binary (`hdc_vector: null` live). **But roko-primitives itself is a *non-optional* dependency of roko-core, roko-learn, roko-runtime, and roko-dreams** (Cargo.toml: roko-core:15, roko-learn:18, roko-runtime:24, roko-dreams:26), so HDC code is linked into every binary transitively and **does execute at runtime through the roko-learn path**: episode fingerprint-per-episode (orchestrate.rs:3184-3186 ‚Üí 10766/13024), pattern-discovery k-medoids every 20 episodes (runtime_feedback.rs:2486), and dream-cycle structure vectors (roko-dreams/cycle.rs:1410-1419). Live proof: 25/27 episodes in `.roko/episodes.jsonl` carry a 1708-char base64 (=1280-byte) fingerprint, latest 2026-05-07.

CLAUDE.md's "Fully wired (tier routing + HDC fingerprint-per-episode)" is accurate for those two headline items but hides four holes: (1) `TierRouter::select_model` itself has **zero production callers** ‚Äî model choice moved to roko-learn's `CascadeRouter` keyed on `OperatingFrequency` (git 319108ff8 "config-driven tier routing"); (2) episode fingerprints are **hash-like, not locality-sensitive** (`from_seed(json{prompt,outcome})`) so similarity over them only detects exact duplicates; (3) the `Engram.fingerprint` field + `Store::query_similar` are dead plumbing (never populated, default no-op, no overrides); (4) **roko-index re-implements the entire HDC core from scratch** with its own incompatible `HdcFingerprint` type ‚Äî the v2 cross-domain promise ("code fingerprint matches knowledge fingerprint", 22-code-intelligence/03) is structurally impossible today.

## V2-aligned

- Dimensions, ops, tie-break, similarity range, 0.526 resonance threshold, FNV-1a/splitmix64 deterministic seeding, 1280-byte serialization all match v2-depth/11-memory/02 exactly (hdc.rs:8-11; codebook.rs:16-21).
- `BundleAccumulator`/`DecayingBundleAccumulator` (streaming, weighted, decay-factor with ~13.5-add half-life at 0.95) match the v2 spec including the documented half-life test (hdc.rs:713-717).
- T0/T1/T2 semantics + 0.3 vitality threshold match v1/16-heartbeat/08 ¬ßInferenceTier/¬ßTierRouter verbatim; Gamma/Theta/Delta ‚Üî tier mapping (roko-core operating_frequency.rs:34-38) is the doc's dual-process mapping; Delta‚Üídream trigger implements v2 heartbeat "Delta Flow triggered not periodic".

- Episode-fingerprint-at-write + fingerprint-informed consolidation (dreams/pattern discovery) are the two v2 pipeline touchpoints that genuinely run.

## Old paradigm & tech debt

- **`TierRouter` is a spec fossil**: hardcoded model IDs will rot (claude-haiku-4-5 etc.); the live router is CascadeRouter-on-OperatingFrequency. Either delete or make it the cascade's cold-start fallback (tier.rs:60-79).
- **roko-index parallel HDC** (index/hdc.rs:12-118): blocks v2 cross-domain resonance (code‚Üîknowledge‚Üîepisode all need one vector space); f32-vs-f64 similarity, no shared roles/codebooks.
- **Hash-like episode fingerprints** stored under a name ("HDC fingerprint") that implies semantic similarity; `HdcFingerprint.encoder_version` exists precisely to allow migration (engram.rs:22-27) but is used nowhere.
- **`hdc` feature fragmentation**: 4 crates gate ~43 cfg sites on a feature nothing enables ‚Äî worst of both worlds (code rots untested in CI binaries, users get nulls). Note roko-fs cheats with `cfg(any(test, feature = "hdc"))` (file_substrate.rs:15) so its tests pass while prod is a no-op.
