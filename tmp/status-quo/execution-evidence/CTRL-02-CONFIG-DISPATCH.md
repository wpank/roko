Implementation:
- Design and invariants: both `ServerBuilder::start_background` and public `run_server_with_state` call one startup normalization/validation helper before restore, watcher/subscriber/bridge/saver/job/archive task creation, router/listener work, or any other fallible startup operation. All PUT, explicit reload, and unchanged watcher reload calls cross one process-wide ownership gate and monotonic generation counter. Async HTTP callers execute the synchronous transaction on Tokio's blocking pool; the watcher retains its public synchronous API; no blocking mutex is held across an `.await`. A PUT transaction now selects raw project source under that gate, merges and canonicalizes the source candidate, derives the prospective effective candidate through the same runtime-layer pipeline as ordinary loading, validates and serializes both projections before any write, atomically replaces only the source file, synchronously propagates only source bytes, then infallibly publishes only the effective ArcSwap value and generation before releasing ownership. Once the blocking transaction starts, request-future cancellation cannot cancel it; the gate prevents an older accepted generation from propagating after a newer generation. Resolution remains inherited: trimmed exact keys win, unique aliases become canonical keys, and duplicate/unresolved references are fatal with an empty-registry compatibility escape. Tier and role keys are sorted before validation, making the first nested error deterministic.
- Source/effective separation: `resolve_runtime_layers` is now the single loader pipeline for ordinary, provenance, and in-memory candidate resolution: global merge, named env, hierarchical env, interpolation, file-secret resolution, and post-merge provider validation retain the same order. Reload publishes that effective object without a source write. PUT never snapshots effective `AppState` for persistence; it reads `workdir/roko.toml`, or when absent seeds the new local file from unified discovery's `ValidatedConfig.raw`, then resolves an owned clone in memory. Thus unified startup and reload can leave effective state in AppState without any later PUT promoting global registries, env values, interpolated strings, or literal file-secret contents into project or replica source.
