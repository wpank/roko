- Design and invariants: both `ServerBuilder::start_background` and public `run_server_with_state` call one startup normalization/validation helper before restore, watcher/subscriber/bridge/saver/job/archive task creation, router/listener work, or any other fallible startup operation. All PUT, explicit reload, and unchanged watcher reload calls cross one process-wide ownership gate and monotonic generation counter. Async HTTP callers execute the synchronous transaction on Tokio's blocking pool; the watcher retains its public synchronous API; no blocking mutex is held across an `.await`. A PUT transaction includes snapshot selection, merge, normalization/validation, fallible response/TOML serialization, atomic main-file replacement, synchronous atomic ephemeral propagation, infallible live ArcSwap commit, and generation publication before ownership release. Once that blocking transaction starts, request-future cancellation cannot cancel it; the gate prevents an older accepted generation from propagating after a newer generation. Resolution remains inherited: trimmed exact keys win, unique aliases become canonical keys, and duplicate/unresolved references are fatal with an empty-registry compatibility escape. Tier and role keys are sorted before validation, making the first nested error deterministic.
- Source/effective separation: reload uses the unified loader to apply global config, named and hierarchical environment overrides, interpolation, and file-secret resolution, then normalizes, validates, and publishes that effective object under the shared gate. It performs no project-file write, including when canonical alias normalization changes the effective object. Therefore explicit reload, watcher reload, and daemon use preserve the exact source bytes and never promote resolved secrets or runtime-only overlays into project `roko.toml`.
- Files/symbols changed: `crates/roko-core/src/config/loader.rs` sorts nested dispatch maps and repeats multi-invalid insertion-order regressions. `crates/roko-serve/src/lib.rs` factors shared startup validation and proves an invalid caller-supplied `AppState` leaves the workdir, cancellation state, process supervisor, and probe port untouched. `crates/roko-serve/src/routes/config.rs` owns serialized PUT/reload ordering, source-preserving effective reload, cancellation-independent blocking transactions, synchronous atomic replica propagation, and deterministic source/race/error regressions. Reload deliberately does not persist canonicalization or merged effective values: source remains source, while validated global/environment/interpolated/file-secret values remain live-only. Existing deployments with no configured models retain their legacy string references unchanged.

Invalid reloads return HTTP 400 without changing source or live truth. Main-file atomic replacement failure leaves the previous live generation untouched.
