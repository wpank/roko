Implementation:
- Design and invariants:
  - `roko-runtime` now owns the serde contract for an exact task-attempt timeout terminal. Its idempotency key is `(run_id, plan_id, task_id, attempt)`; a byte-equivalent semantic duplicate is ignored and a different fact for an occupied key fails closed.
  - The CLI performs exhaustive conversions between runner timeout/effect enums and the runtime contract, and rejects an effect whose embedded attempt does not equal the terminal attempt.
  - Exact timeout cancellation is linearized through the ownership claim. External cleanup completes first, the typed ledger fact is durably appended while the claim remains held, the claim is closed, and only then is the terminal runner event emitted. Completion and expiry therefore cannot both win the same phase/effect generation.
  - Each deadline scan is a bounded candidate snapshot. Eligibility is rechecked immediately before cancellation, making completions, replacements, earlier expiries, duplicate scans, and sibling drainage stale no-ops.
  - Resume replays only exact typed timeout records for the current run, restores cancellation/timeout state and failed-DAG blocking, and rejects a durable timeout that conflicts with a different terminal status.
  - Persisted unix time remains audit/display metadata. Restart intent uses only the persisted monotonic elapsed duration; absurd wall timestamps cannot be converted into a process-local monotonic baseline.
- Files/symbols changed:
  - `crates/roko-runtime/src/run_ledger.rs`: `TimeoutLedgerEntry`, `TaskTimeoutTerminal`, exact attempt/effect/kind types, `TimeoutTerminalReplay`, `TimeoutDeadlineIntent`, and `RunLedger::record_timeout_terminal`.
  - `crates/roko-cli/src/runner/event_loop.rs`: typed conversion/load/persist/replay helpers; resume reconciliation; failed-DAG seeding; exact owner recheck; cleanup -> ledger -> owner close -> terminal ordering; deterministic race, replay, idempotence, stale-effect, write-failure, absent/stale-gate-map, absurd-wall-time, and bounded-sibling tests.
  - `crates/roko-cli/src/runner/projection.rs`: one shared normalization entry point and exact `ProjectionEvent` equality proof for live versus replayed timeout events.
- Compatibility/migration: the tagged JSON shape remains `{"kind":"timeout_recorded","run_id":...,"timeout":...}`, so existing exact-attempt timeout lines deserialize into the new runtime type. Existing global timeout audit lines have no attempt and remain preserved but are intentionally excluded from task-terminal replay. Unrelated legacy ledger kinds are ignored. No manifest, lockfile, configuration, CLI event vocabulary, state schema, or persistence-helper change is required.
- Failure/recovery/security behavior: malformed or conflicting exact timeout facts fail resume/persistence closed; missing ledger files replay as empty. A ledger append or conversion failure restores the exact held claim as nonterminal and emits `CancellationFailed`, so no timeout terminal is published without its durable fact. Sibling cleanup attempts only the initial owned snapshot and cannot retry indefinitely. No network, credential, remote, or external-service mutation occurs.
