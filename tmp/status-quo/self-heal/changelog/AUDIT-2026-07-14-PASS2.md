## Bugs found and fixed

### Critical: `claim_cancellation_exact` partial-mutation (attempt_ownership.rs)

**Before**: The method set `slot.claimed = true` and advanced the nonce *before* checking
if `slot.resource.is_some()`. If the resource was `None`, the slot became permanently
stuck: `claimed = true` but no `AttemptClaim` was returned to the caller. No code path
could recover it except `discard_for_cleanup`.

**Fix**: Added `slot.resource.is_none()` to the eligibility guard *before* any mutation.
The same pattern was fixed in `take_resource`. Both now use `.unwrap()` after the
pre-validated guard instead of fallible `.ok_or()`.

### Bug: `break` only exits inner loop in stdout reader (agent_stream.rs)

**Before**: When `stdout_tx.send(event).await.is_err()` detected a closed channel, `break`
exited only the inner `for event in parse_stream_line(...)` loop. The outer
`while let Ok(Some(line)) = lines.next_line().await` continued consuming stdout lines
from the pipe with no consumer, wasting CPU/IO until the process exited.

**Fix**: Changed `break` to `return`, which exits the entire reader task immediately.

### Bug: PID registration window (agent_stream.rs)

**Before**: `register_spawned_pid(pid)` was called at line 375, *after* reader tasks were
spawned at lines 335 and 354. If a panic occurred between spawning readers and
registration, the PID would never be registered, and the orphan cleanup handler would not
find it.

**Fix**: Moved `register_spawned_pid(pid)` to immediately after the PID is obtained
(after `child.id()`), closing the registration window.

### Bug: SSE never-loop (sse.rs)

**Before**: A `loop { match ... { Ok => return, Err(Lagged) => return, Err(Closed) => return } }`
where every arm returned. Clippy correctly flagged this as `never_loop`.

**Fix**: Removed the unnecessary `loop` wrapper, leaving a direct `match`.

### Bug: Redundant `continue` in staging loop (atomic.rs)

**Before**: `continue` at the end of a match arm inside a for loop, which is the default
behavior and triggers `clippy::needless_continue`.

**Fix**: Changed to `{}` (empty block).

## Design improvements applied

### deadlines.rs â€” Separation of concerns

1. **Moved `owner_expiry` from `DeadlineTracker` static method to free function.** It
   used no `self` state and had no conceptual relationship to `DeadlineTracker`'s global
   clocks. Now `pub(crate) fn owner_expiry(...)` at module level.

2. **Replaced Vec allocation with inline running minimum.** The old code allocated a
   `Vec<(u64, TimeoutKind, Duration)>` on every call (hot path, per-owner per-tick), then
   called `.into_iter().min_by_key()`. Replaced with a `let mut earliest: Option<...>`
   that updates inline â€” zero allocation.

3. **Extracted `duration_millis_u64` helper.** Replaced 3 occurrences of
   `u64::try_from(d.as_millis()).unwrap_or(u64::MAX)` with a named function.

4. **Added doc comments** to all 5 public types: `MonotonicTime`, `OwnershipTiming`,
   `DeadlinePolicy`, `DeadlineExpiry`, `DeadlineTracker`.

5. **Added priority comment** to `global_expiry` documenting that check order encodes
   HardRun > SchedulerNoProgress priority.

### attempt_ownership.rs â€” Dedup and cleanup

1. **Extracted `advance_claim_nonce` helper.** The identical 4-line nonce-assignment
   pattern appeared in both `claim_cancellation_exact` and `take_resource`. Now a single
   private associated function.

2. **Removed dead code.** `restore_cancellation_claim()` (zero callers) and `resource()`
   immutable accessor (zero callers, only `resource_mut()` is used) deleted.

### worktree.rs â€” Robustness

1. **Added `validate_id` in `try_reattach`.** Prevents path traversal via crafted
   plan_id strings like `"../../etc"`.

2. **Fixed `as i64` truncating cast.** Replaced with
   `i64::try_from(d.as_millis()).unwrap_or(i64::MAX)` for consistency.

3. **Collapsed triple-lock in `ensure_for_plan`.** The old code acquired the mutex 3
   times (get, touch, get) for a single "refresh and return" operation. Now a single lock
   scope with atomic touch-and-clone.

4. **Added `#[derive(Debug)]` to `WorktreeManager`.**

### persist.rs â€” Consistency

1. **Removed redundant `create_dir_all`** in `GateThresholds::save` â€” `atomic_write`
   already creates parent directories.

2. **Added `.context("serializing state snapshot")`** to bare `?` in
   `save_state_snapshot` for consistency with every other serializer.

### workspace_lock.rs â€” Safety

1. **Added `#[must_use]`** to `WorkspaceLockGuard` to prevent accidental immediate drops.
