# Self-heal audit — 2026-07-14

## Verdict

The committed SH01 work is a substantial and generally well-factored lifecycle
foundation, but the complete six-batch self-heal programme is not finished.
The manifest is the authoritative status record: SH01 is 26/28 complete and
SH02–SH06 are 0/N. This audit therefore does not relabel partial precursors as
completed tasks.

The audit reviewed the 88 commits from the plan-catalogue introduction through
`1649c18b2`, the current working-tree continuation, all six task manifests, and
the referenced subsystem acceptance themes. It applied bounded fixes where the
failure and desired invariant were concrete; larger unfinished tasks remain in
their dependency order.

## Corrective changes applied

### Exact timeout and effect ownership

- Agent activity now refreshes the silence clock only for the eligible exact
  attempt/effect.
- Lost gate, merge, CLI, and bridge producers are detected at their deadline and
  classified as `LostEffect` through the exact ownership claim.
- Timeout ledger persistence now occurs after external cleanup succeeds but
  while the exact claim is still held. A ledger failure restores ownership and
  records nonterminal `CancellationFailed`; it cannot leave a false terminal
  timeout claim.
- Lost-effect tests now parse canonical events, replay them into `RunState`,
  compare live/replayed terminal status, and validate one typed ledger timeout.
- Added a regression for timeout-ledger failure preserving ownership.
- Standardized plan verification on `RUNG_PLAN_VERIFY` rather than the unrelated
  `u32::MAX` sentinel.
- Rejected incoming four-hour global timeout inflation and per-step timeout
  multiplication. The established hard/task/gate/silence/progress defaults are
  asserted explicitly, and the plan-verification outer timeout matches the
  default owned gate-effect deadline.

### Worktree and lock recovery

- Existing plan worktrees can be rediscovered on resume.
- Reattachment fails closed unless the candidate has the same canonical Git
  common directory, exact `roko/plan/<plan_id>` branch, non-detached HEAD, and a
  HEAD equal to that branch tip.
- Added positive reattachment coverage plus wrong-branch and foreign-repository
  rejection tests.
- Workspace lock contenders no longer truncate the live owner's PID before
  obtaining the advisory lock.
- Lock diagnostics are synced after acquisition and cleared during normal
  release; stale diagnostics are replaced only by the new owner.

### Persistence primitives

- Consolidated runner atomic persistence onto `roko-fs` rather than maintaining
  another write/rename implementation.
- Atomic writes now use unique, exclusively created sibling staging files,
  retry name collisions, sync file contents, rename, and sync the parent
  directory on Unix.
- Concurrent writers cannot share staging state, and pre-existing forensic
  debris is not overwritten.
- JSONL recovery now clears a wholly invalid or wholly partial file, allowing a
  later valid append to become readable instead of preserving permanent
  corruption.

### State publication and reconnects

- StateHub serializes snapshot commit, optional event-log append, sequence
  assignment, replay-ring insertion, and broadcast ordering.
- Subscribers now observe event N only after the materialized state includes N.
- Added an atomic snapshot update API and migrated TUI/server callers away from
  lossy read-modify-write replacement.
- SSE reconnect starts after `Last-Event-ID` instead of replaying the last
  acknowledged event.
- A lagged SSE client receives an explicit `gap` event with a materialized
  snapshot for resynchronization.

### Dispatch configuration

- Added a runner dispatch-boundary validator for model routing.
- Unique provider-facing slug aliases normalize to stable model keys.
- Duplicate slugs and unresolved default, fallback, tier, or role references
  stop runner dispatch deterministically.
- Ordinary config loading remains compatible; the fatal policy is applied at
  the dispatch boundary.

### TUI bounded state and layout

- Connected task-output rings are treated as authoritative snapshots rather
  than deltas, preventing the same output from being appended on every refresh.
- Retained output is bounded to 50 lines and stale task tails are removed.
- Route-table overhead arithmetic now preserves one data row when the bordered
  panel has room for a header plus one route.
- Added regressions for repeated snapshots, bounds, stale cleanup, capacity
  arithmetic, and actual ratatui buffer rendering.

## Remaining acceptance gaps

### SH01

- SH01-T06C4 remains `ready`. The working tree covers lost-producer expiry and
  ledger ordering, but does not yet prove every completion-versus-expiry race,
  duplicate/stale expiry case, or resume restoration invariant.
- SH01-T07 remains `ready`. `build_report` still derives task counts from the
  plan phase, cannot report blocked/skipped/cancelled/orphaned categories, and
  can contradict lifecycle state.
- Timeout JSONL is still an ad-hoc runner record rather than a first-class,
  idempotent `roko-runtime::RunLedger` replay type.

### SH02

- Per-plan `meta.max_parallel` is not enforced; the runner uses one global task
  semaphore.
- Capacity is not retained across the whole preflight → agent → gate → commit →
  merge lifecycle, and dispatch still polls on a 100 ms tick.
- Isolation remains plan-scoped instead of task-attempt-scoped.
- A passed task can still have undeclared dirty edits or a non-durable/no-change
  commit outcome.
- Dirty worktree quarantine and ownership attribution on resume remain open.

### SH03

- Terminal snapshots omit exact failed/blocked/skipped IDs, complete DAG state,
  and attempt ownership needed for equivalent resume.
- There is no rotating run-scoped last-known-good, pre-terminal, and terminal
  checkpoint series; the checkpoint writer is unused.
- Ledger records do not yet have stable idempotency keys across all lifecycle
  effects.
- StateHub now has correct commit-before-broadcast ordering, but complete
  durable gap/resync semantics across every consumer still need batch-level
  proof.

### SH04

- Dashboard events do not carry complete structured run/plan/task/attempt/agent
  identity; current reducers still depend on incompatible display-ID parsing.
- Approval mode still chooses `NoopSink`, dropping sink-only telemetry.
- Nonempty subprocess stderr is still promoted to `AgentEvent::Error`, so normal
  diagnostic output can appear fatal.
- Preflight step/output/heartbeat events, diagnosis summaries, liveness states,
  estimated/final token reconciliation, asynchronous bounded Git refresh, and
  operational event-health logging remain open.

### SH05

- Symmetric dispatch lifecycle and bounded jittered 429/529 retry are not
  implemented as a complete contract.
- Process-tree work from SH01 is a strong precursor, but bounded channels and
  deterministic end-to-end shutdown semantics remain to be proved.
- Real per-attempt cost attribution, pre-dispatch budget enforcement, and
  resumable durable budget halts remain open.

### SH06

- `runner_crash_recovery.rs` is absent.
- Existing TUI tests cover navigation metadata rather than connected event,
  buffer, and operational-log invariants.
- Existing resume tests cover persistence helpers rather than real interruption
  boundaries.
- The self-host E2E remains ignored; there is no deterministic, network-free
  crash-chain or fake-provider self-host proof.
