## Task

Replace the TUI's disk-polling refresh with streaming from StateHub when available,
keeping disk-polling as a fallback for standalone mode.

## Implementation

### 1. Add snapshot receiver to App (`app.rs`)

Add a new field:
```rust
pub snapshot_rx: Option<tokio::sync::watch::Receiver<roko_core::DashboardSnapshot>>,
```

Add a new constructor:
```rust
pub fn new_connected(root: impl AsRef<Path>, state_hub: &roko_core::SharedStateHub) -> Self {
    let mut app = Self::new(root);
    app.snapshot_rx = Some(state_hub.snapshot());
    app
}
```

### 2. Add update_from_dashboard_snapshot to TuiState (`state.rs`)

```rust
pub fn update_from_dashboard_snapshot(&mut self, snap: &roko_core::DashboardSnapshot) {
    // Map snap.plans → self.plans (Vec<PlanEntry>)
    // Map snap.agents → self.agents (Vec<AgentRow>)
    // Map snap.gate_verdicts → self.gate_results (Vec<GateResultEntry>)
    // Map snap.errors → notification entries
    // Preserve navigation state (selected_plan, selected_agent, scroll, etc.)
}
```

### 3. Integrate into main loop (`app.rs`)

In the main event loop, before handling events:
```rust
// Connected mode: zero-copy borrow from StateHub
if let Some(rx) = &mut self.snapshot_rx {
    if rx.has_changed().unwrap_or(false) {
        let snapshot = rx.borrow_and_update();
        self.tui_state.update_from_dashboard_snapshot(&snapshot);
    }
}
// Standalone mode: existing data_rx polling path unchanged
```

### 4. Wire cmd_dashboard (`main.rs`)

Update the `dashboard` command handler to accept an optional `SharedStateHub`.
When run as part of `plan run`, pass the hub; when run standalone, pass `None`.

## Rules

1. The existing polling path MUST continue to work when `snapshot_rx` is `None`.
2. Do not remove `data_rx` or any existing background thread.
3. Navigation state (selected_plan, selected_agent, scroll offsets, active_tab)
   must be preserved across snapshot updates — never reset them.
4. Add `tokio` to roko-cli's Cargo.toml dependencies if not already present
   (it likely is — check first).
