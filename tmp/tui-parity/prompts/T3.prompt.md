## Task

Wire an IPC channel so the orchestrator (PlanRunner) can send approval requests
to the TUI, and the TUI can respond with approve/deny.

## Implementation

### 1. Create `approval_ipc.rs` (new file)

```rust
use tokio::sync::{mpsc, oneshot};

/// Request sent from orchestrator to TUI
pub struct ApprovalRequest {
    pub role: String,
    pub command: String,
    pub approval_id: String,
    pub response_tx: oneshot::Sender<bool>,  // true = approve, false = deny
}

/// Channel pair for approval IPC
pub struct ApprovalChannel {
    pub tx: mpsc::Sender<ApprovalRequest>,
    pub rx: mpsc::Receiver<ApprovalRequest>,
}

impl ApprovalChannel {
    pub fn new(buffer: usize) -> Self {
        let (tx, rx) = mpsc::channel(buffer);
        Self { tx, rx }
    }
}
```

### 2. Wire into App (`app.rs`)

Add field:
```rust
pub approval_rx: Option<mpsc::Receiver<ApprovalRequest>>,
```

In main loop, check for incoming approval requests on each tick:
```rust
if let Some(rx) = &mut self.approval_rx {
    if let Ok(req) = rx.try_recv() {
        self.tui_state.pending_approval = Some(PendingApproval {
            role: req.role.clone(),
            command: req.command.clone(),
        });
        self.pending_approval_response = Some(req.response_tx);
        self.active_modal = Some(ModalState::Approval {
            role: req.role,
            command: req.command,
        });
    }
}
```

### 3. Handle approval modal response (`input.rs`)

When the user presses `y` or `n` in the approval modal:
```rust
// y → send true via oneshot
// n → send false via oneshot
if let Some(tx) = self.pending_approval_response.take() {
    let _ = tx.send(approved);
}
self.active_modal = None;
self.tui_state.pending_approval = None;
```

### 4. Wire orchestrator (`orchestrate.rs`)

When PlanRunner encounters an approval_required situation:
```rust
if let Some(approval_tx) = &self.approval_tx {
    let (response_tx, response_rx) = oneshot::channel();
    approval_tx.send(ApprovalRequest {
        role: agent_role.to_string(),
        command: command.clone(),
        approval_id: id.clone(),
        response_tx,
    }).await?;
    let approved = response_rx.await.unwrap_or(false);
    if !approved { /* skip or abort task */ }
}
```

### 5. Wire main.rs

When running `plan run --approval`, create the ApprovalChannel and pass
the tx side to PlanRunner and rx side to App.

## Rules

1. The approval channel is optional — standalone dashboard mode works without it.
2. The existing approval modal UI in `modals/approval.rs` should be reused.
3. If no approval request is pending, the TUI behaves exactly as before.
4. The oneshot channel ensures the orchestrator blocks until the user responds.
