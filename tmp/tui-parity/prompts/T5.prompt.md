## Task

Bridge the parallel_pool widget to use live AgentRow data, add context gauge
columns, populate execution_waves from active tasks or DashboardSnapshot,
wire wave progress into the dashboard bottom ribbon, and add wave navigation keys.

## Implementation

### 1. Bridge parallel_pool to AgentRow data (`parallel_pool.rs`)

The parallel_pool widget should read from `tui_state.agents: Vec<AgentRow>` instead
of any placeholder data. Each agent row shows:
- Agent ID (truncated)
- Role
- Model
- Current task
- Progress indicator
- **New**: Context gauge (tokens used / context_limit)

```rust
// Add context gauge column
let ctx_ratio = agent.input_tokens as f64 / agent.context_limit.max(1) as f64;
let gauge = Gauge::default()
    .ratio(ctx_ratio.min(1.0))
    .label(format!("{}k/{}k", agent.input_tokens / 1000, agent.context_limit / 1000));
```

### 2. Populate execution_waves (`state.rs`)

In `update_from_snapshot()` or `update_from_dashboard_snapshot()`:
```rust
// Group plans by wave from DashboardSnapshot or DashboardData
// If wave info is available in executor state, use it directly
// Otherwise derive from plan dependencies
self.execution_waves = derive_waves(&self.plans);
```

### 3. Wave progress ribbon (`dashboard_view.rs`)

Add a bottom ribbon showing wave progress:
```
Wave 1/3 ████████░░░░ 4/6 plans │ Wave 2/3 ░░░░░░░░░░░░ 0/4 plans │ Wave 3/3 — 0/2 plans
```

Use the `wave_progress` widget if it exists, or create a simple inline renderer.

### 4. Wave navigation keys (`input.rs`)

- `Shift+Left`: select previous wave
- `Shift+Right`: select next wave
- When a wave is selected, highlight its plans in the plan tree

Update `selected_wave_idx` in TuiState.

### 5. Preserve wave expanded state

Ensure `Wave.expanded` persists across data refreshes. When updating waves,
match by wave index and preserve the expanded flag.
